# Audit Report

## Title
Negative Port Values Bypass Validation in `fetchMirrorNodeNetwork`, Poisoning the Network Map with Invalid Addresses

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` deserializes `port` into a signed `int` and only rejects entries where `ep.Port == 0`. A crafted mirror node response supplying negative port values (e.g., `-1`) passes the guard, produces syntactically-formed but semantically invalid addresses (e.g., `"1.2.3.4:-1"`), and populates the returned network map. Because `len(network) > 0`, the function returns success with no error, the retry/backoff loop in `buildNetworkFromMirrorNodes` is never triggered, and the pinger operates against an entirely poisoned node list for the lifetime of the run.

## Finding Description
**File:** `pinger/mirror_node_client.go`
**Function:** `fetchMirrorNodeNetwork()`, lines 74–132

**Root cause — signed field accepts negative JSON integers:**
```go
// line 33
Port int `json:"port"`   // signed — accepts any int64 value from JSON
```

**Insufficient guard — only zero is excluded:**
```go
// line 118
if host == "" || ep.Port == 0 {
    continue   // negative ports (-1, -65535, …) are NOT skipped
}
```

**Poisoned address construction:**
```go
// line 122
addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
// ep.Port = -1  →  addr = "1.2.3.4:-1"
network[addr] = nodeAccountId   // stored without error
```

**Silent success path bypasses retry:**
```go
// lines 127–131
if len(network) == 0 {
    return nil, false, fmt.Errorf("no usable service_endpoints found …")
}
return network, false, nil   // returns successfully with all-invalid addresses
```

If every `service_endpoint` carries a negative port, `len(network) > 0` is satisfied, no error is returned, and the caller at line 54 (`if err == nil { return network, nil }`) immediately accepts the poisoned map without retrying.

## Impact Explanation
The pinger's sole purpose is to dial Hedera consensus nodes and measure liveness. With a poisoned network map, every dial attempt targets an address like `"0.0.3:-1"`, which fails at the TCP layer. The pinger reports all nodes unreachable, fully disabling the monitoring service. Because `fetchMirrorNodeNetwork` signals success, the retry/backoff logic in `buildNetworkFromMirrorNodes` is not triggered — the bad map is used as-is for the entire run.

## Likelihood Explanation
The precondition is controlling the mirror node HTTP response. This is achievable via:
1. **HTTP (non-TLS) deployment** — a network-adjacent attacker can inject a crafted response.
2. **DNS poisoning / BGP hijack** — redirects the mirror node hostname to an attacker-controlled server.
3. **Compromised mirror node operator** — the mirror node is an external service outside the pinger operator's trust domain.

No authentication of the mirror node response is performed. The attack is repeatable on every restart or network-refresh cycle.

## Recommendation
1. Change `Port int` to `Port uint16` in `serviceEndpoint`. `uint16` constrains the value to `[0, 65535]` at the type level and eliminates the negative-port class entirely.
2. If `uint16` is not used, add an explicit range check:
   ```go
   if host == "" || ep.Port <= 0 || ep.Port > 65535 {
       continue
   }
   ```
3. Consider treating a response that yields zero usable endpoints after filtering as a retryable error (set `retry = true`) so that a transiently bad response does not permanently poison the run.

## Proof of Concept
Craft a mirror node JSON response where all `service_endpoints` carry `"port": -1`:
```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{"ip_address_v4": "1.2.3.4", "port": -1}]
  }]
}
```
`fetchMirrorNodeNetwork` will:
1. Pass the `ep.Port == 0` guard (since `-1 != 0`).
2. Construct `addr = "1.2.3.4:-1"` via `net.JoinHostPort`.
3. Return `map["1.2.3.4:-1"] = 0.0.3`, `false`, `nil`.
4. `buildNetworkFromMirrorNodes` accepts the map immediately (no retry).
5. Every subsequent dial to `"1.2.3.4:-1"` fails at the TCP layer; the pinger reports all nodes unreachable.
