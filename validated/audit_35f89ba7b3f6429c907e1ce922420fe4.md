Audit Report

## Title
Gas Throttle Bypass via `eth_estimateGas` Binary Search Amplification

## Summary
A single `eth_estimateGas` request triggers up to 21 full EVM executions (1 initial probe + up to 20 binary search iterations), while the gas throttle bucket is charged for only 1 execution. The binary search iterations in `BinaryGasEstimator.search()` call `doProcessCall(..., estimate=true)`, which unconditionally skips all gas-throttle accounting, making them invisible to the rate limiter.

## Finding Description

**Exact code path:**

`ContractController.call()` calls `throttleManager.throttle(request)`, which consumes `scaleGas(request.getGas())` tokens once from the gas bucket. [1](#0-0) [2](#0-1) 

`ContractExecutionService.estimateGas()` then performs two phases:

**Phase 1 — Initial probe:** `callContract(params, context)` → `doProcessCall(params, params.getGas(), false)` with `estimate=false`. The `finally` block runs `restoreGasToBucket()`, returning unused gas to the bucket. [3](#0-2) [4](#0-3) 

**Phase 2 — Binary search:** `binaryGasEstimator.search(...)` runs up to `maxGasEstimateRetriesCount = 20` iterations, each calling `doProcessCall(params, gas, true)` with `estimate=true`. [5](#0-4) [6](#0-5) 

**Root cause:** In `ContractCallService.doProcessCall()`, the `finally` block is guarded by `if (!estimate)`. When `estimate=true` (all binary search iterations), `restoreGasToBucket()` is never called and no gas-throttle interaction occurs — neither consumption nor restoration. [7](#0-6) 

**Failed assumption:** The gas throttle was designed assuming one EVM execution per gas-bucket deduction. For `eth_estimateGas`, the design silently allows up to 21 EVM executions per single bucket deduction.

**Default configuration values confirmed in code:**
- `maxGasLimit = 15_000_000` [8](#0-7) 
- `maxGasEstimateRetriesCount = 20` [9](#0-8) 
- `gasPerSecond = 7_500_000_000` (scaled to 750,000 bucket tokens via `scaleGas()` which divides by `GAS_SCALE_FACTOR = 10_000`) [10](#0-9) 
- `requestsPerSecond = 500` [11](#0-10) 

## Impact Explanation

A single `eth_estimateGas` request against a contract consuming near the 15M gas limit causes up to 21 EVM executions while the gas throttle accounts for only 1. The gas bucket is debited `scaleGas(15_000_000) = 1500` tokens once, but up to 21 near-maximum-gas EVM runs execute. An attacker sending requests at the permitted 500 RPS rate can drive actual EVM execution throughput far beyond the intended rate, potentially saturating CPU on the mirror node web3 service without triggering any throttle rejection. The `requestsPerSecond` bucket still limits to 500 RPS, but the gas throttle — which is the primary mechanism for bounding EVM compute cost — is effectively bypassed for all binary search iterations. [12](#0-11) 

## Likelihood Explanation

No privileges, API keys, or special accounts are required. Any user with network access to the `/api/v1/contracts/call` endpoint can exploit this. The attack requires only targeting any existing contract that consumes close to the 15M gas limit. The binary search will consistently run near the maximum iteration count because when gas usage is near the upper bound, `lo` and `hi` remain far apart for many iterations, as confirmed by the test data showing `(low=35000, high=15_000_000)` produces 14 iterations. [13](#0-12) 

## Recommendation

The binary search iterations should interact with the gas throttle proportionally. Options include:

1. **Charge per iteration:** Before each `doProcessCall(..., true)` call in `BinaryGasEstimator.search()`, consume tokens from the gas bucket proportional to the `mid` gas value being tested.
2. **Charge upfront multiplier:** At the controller level, multiply the gas token deduction by an estimate of the expected number of binary search iterations (e.g., `log2(maxGasLimit / minGas)`).
3. **Restore per iteration:** Remove the `if (!estimate)` guard in `doProcessCall()`'s `finally` block so that `restoreGasToBucket()` is called for binary search iterations as well, ensuring each iteration is accounted for in the throttle.

The simplest fix is option 3: removing the `if (!estimate)` guard so that `restoreGasToBucket()` is always called, ensuring every EVM execution — including binary search iterations — is reflected in the gas throttle accounting. [7](#0-6) 

## Proof of Concept

```
1. Deploy or identify any contract that consumes ~14,900,000 gas (e.g., a loop of SLOAD opcodes).

2. Send a POST to /api/v1/contracts/call with:
   {
     "to": "<contract_address>",
     "gas": 15000000,
     "estimate": true,
     "data": "<calldata>"
   }

3. Observe server-side:
   - ThrottleManagerImpl.throttle() is called once, consuming scaleGas(15_000_000) = 1500 tokens.
   - ContractExecutionService.estimateGas() calls callContract() once (estimate=false) → restoreGasToBucket() is called.
   - BinaryGasEstimator.search() runs up to 20 iterations, each calling doProcessCall(..., true).
   - In each iteration, the finally block's if (!estimate) guard prevents any throttle interaction.
   - Total EVM executions: up to 21. Total throttle deductions: 1 (net of the initial probe refund).

4. Repeat at 500 RPS (the requestsPerSecond limit).
   - The rateLimitBucket rejects beyond 500 RPS.
   - The gasLimitBucket is not exhausted because binary search iterations don't consume it.
   - Actual EVM execution rate: up to 500 × 21 = 10,500 executions/second.
   - Intended EVM execution rate (per throttle design): ~500/second.

    protected final EvmTransactionResult doProcessCall(
            CallServiceParameters params, long estimatedGas, boolean estimate) throws MirrorEvmTransactionException {
        EvmTransactionResult result = null;
        var status = ResponseCodeEnum.SUCCESS.toString();

        try {
            result = transactionExecutionService.execute(params, estimatedGas);

            if (!estimate) {
                validateResult(result, params);
            }
        } catch (IllegalStateException | IllegalArgumentException e) {
            throw new MirrorEvmTransactionException(e.getMessage(), EMPTY);
        } catch (MirrorEvmTransactionException e) {
            // This result is needed in case of exception to be still able to call restoreGasToBucket method
            result = e.getResult();
            status = e.getMessage();
            throw e;
        } finally {
            if (!estimate) {
                restoreGasToBucket(result, params.getGas());

                // Only record metric if EVM is invoked and not inside estimate loop
                if (result != null) {
                    updateMetrics(params, result.gasUsed(), 1, status);
                }
            }
        }
        return result;
    }

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
