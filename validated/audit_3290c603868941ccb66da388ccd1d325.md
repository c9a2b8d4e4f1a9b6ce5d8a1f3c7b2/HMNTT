Audit Report

## Title
Future-Timestamp Mtime Bypass Silently Disables Pinger Liveness Probe

## Summary
The liveness healthcheck in `pinger/cmd/healthcheck/main.go` computes heartbeat age as `time.Since(fi.ModTime())`. Because `time.Duration` is a signed `int64`, a future-dated mtime on `/tmp/alive` produces a large negative duration that is never greater than `2*time.Minute`, causing the probe to unconditionally exit 0 (healthy) even when the pinger process is dead.

## Finding Description
**Location:** `pinger/cmd/healthcheck/main.go`, lines 34–38.

```go
age := time.Since(fi.ModTime())   // negative if ModTime is in the future
if age > 2*time.Minute {          // always false for negative age
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
os.Exit(0)                        // unconditionally reached
```

`time.Since(t)` returns `time.Now().Sub(t)`, a signed `int64` nanoseconds value. If `fi.ModTime()` is set to a future date (e.g., year 2099), the result is approximately `−2.3 × 10¹⁸` ns. The comparison `−2.3e18 > 120_000_000_000` is always `false`, so `os.Exit(0)` is always reached.

**Root cause:** The staleness check assumes `age` is non-negative. There is no lower-bound guard (`age < 0`).

The heartbeat file is written by `pinger/main.go` line 36 with mode `0644` under UID 1000 — the same UID the pinger runs as — making the file owner-writable via `utimes()` without any elevated privilege.

## Impact Explanation
The pinger's sole purpose is to submit periodic Hedera transactions and prove network reachability. Its liveness probe is the only automated mechanism that causes Kubernetes to restart a dead pinger. With the probe permanently bypassed, a crashed or hung pinger is never restarted, silently eliminating network health monitoring. The monitoring blind spot can mask real network degradation indefinitely until a human operator intervenes. Severity: Medium (operational integrity / availability).

## Likelihood Explanation
The precondition is code execution inside the container as UID 1000 (the pinger's own UID), achievable via RCE in the pinger binary or its Hedera Go SDK dependency. Once achieved, the manipulation is a single `utimes()` syscall — no elevated privileges required, as the attacker owns the file. The effect is persistent across all subsequent probe cycles and survives until the container is manually restarted or the file is deleted. Repeatability is trivial.

## Recommendation
Add a lower-bound check so that a negative (future-dated) age is treated as stale:

```go
age := time.Since(fi.ModTime())
if age < 0 || age > 2*time.Minute {
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
```

This ensures any mtime that is in the future is rejected as invalid rather than silently accepted as fresh.

## Proof of Concept
1. Gain code execution inside the pinger container as UID 1000.
2. Call `utimes("/tmp/alive", ...)` to set mtime to a far-future timestamp (e.g., 2099-01-01).
3. Kill the pinger goroutine or the entire process.
4. Trigger the Kubernetes liveness probe (`/healthcheck live`): `os.Stat("/tmp/alive")` succeeds, `time.Since(future_time)` returns a large negative duration, `negative > 2*time.Minute` is `false`, probe exits 0.
5. Kubernetes never restarts the container; the dead pinger is masked indefinitely.
