Audit Report

## Title
`Optional.empty()` Cached by Incorrect `unless` Condition in `findRuntimeBytecode()` Enables Targeted Contract DoS

## Summary
`ContractRepository.findRuntimeBytecode()` is annotated with `@Cacheable(unless = "#result == null")`, but its return type is `Optional<byte[]>`. Spring Cache evaluates `#result` against the actual returned object — the `Optional` wrapper — which is never `null`. `Optional.empty()` is a non-null singleton, so the `unless` guard never fires for a missing bytecode row, and the empty result is permanently stored in the Caffeine cache. Any caller who queries a `contractId` before its bytecode is written to the database permanently poisons the cache entry for that contract, making it unreachable via the web3 API for as long as the attacker refreshes the entry.

## Finding Description
**Location:** `web3/src/main/java/org/hiero/mirror/web3/repository/ContractRepository.java`, lines 16–18 [1](#0-0) 

```java
@Cacheable(cacheNames = CACHE_NAME_CONTRACT, cacheManager = CACHE_MANAGER_CONTRACT, unless = "#result == null")
@Query(value = "select runtime_bytecode from contract where id = :contractId", nativeQuery = true)
Optional<byte[]> findRuntimeBytecode(final Long contractId);
```

**Root cause:** Spring's `@Cacheable` `unless` SpEL expression `#result == null` is evaluated against the value returned by the method. Since the return type is `Optional<byte[]>`, `#result` is always an `Optional` object. `Optional.empty()` — returned when no DB row exists — is a non-null singleton (`Optional.EMPTY`), so `#result == null` evaluates to `false` unconditionally. The empty `Optional` is stored in the cache as if it were a valid positive result.

**Cache configuration:** `expireAfterAccess=1h,maximumSize=1000,recordStats` [2](#0-1) 

`expireAfterAccess` resets the TTL on every read, so a single request per hour keeps the poisoned entry alive indefinitely.

**Downstream consumer:** `ContractBytecodeReadableKVState.readFromDataSource()`, lines 42–46 [3](#0-2) 

```java
return contractRepository
        .findRuntimeBytecode(entityId.getId())
        .map(Bytes::wrap)
        .map(Bytecode::new)
        .orElse(null);
```

When the cache returns `Optional.empty()`, `.orElse(null)` returns `null`, and the EVM treats the address as having no deployed bytecode.

## Impact Explanation
Every EVM execution path that loads bytecode for a contract goes through `ContractBytecodeReadableKVState.readFromDataSource()`. With a poisoned cache entry, the EVM receives `null` bytecode and treats the target address as an EOA (no code). All `eth_call`, `eth_estimateGas`, and `eth_getCode` requests to the affected contract silently return empty data or revert. This is a targeted, contract-specific denial-of-service against the mirror node's web3 API. An attacker can selectively disable any contract they choose, with no privileged access required, and can poison up to 1,000 distinct contracts simultaneously (the cache's `maximumSize`).

## Likelihood Explanation
The precondition — querying a `contractId` before its bytecode is indexed — is trivially achievable. Hedera transaction receipts include the assigned `contractId` immediately upon submission confirmation, while mirror-node ingestion typically lags by seconds to tens of seconds. Any observer of the Hedera network (or the contract deployer themselves) can race this window. The attack requires only standard, unauthenticated HTTP calls to the public web3 endpoint. Keeping the cache entry alive requires one request per hour, which is negligible effort.

## Recommendation
Change the `unless` condition to inspect the contents of the `Optional`, not the wrapper itself:

```java
@Cacheable(
    cacheNames = CACHE_NAME_CONTRACT,
    cacheManager = CACHE_MANAGER_CONTRACT,
    unless = "#result == null || !#result.isPresent()"
)
```

This prevents `Optional.empty()` from being stored in the cache, so subsequent calls will re-query the database after ingestion completes. Alternatively, change the return type to `byte[]` (returning `null` for absent rows) and keep the existing `unless = "#result == null"` guard, which would then work correctly.

## Proof of Concept
1. Deploy a contract on Hedera. Note the assigned `contractId` from the transaction receipt (available before mirror-node ingestion completes).
2. During the ingestion lag window (typically seconds), send an `eth_getCode` (or `eth_call`) request to the mirror node's web3 endpoint for that `contractId`.
3. The DB has no row yet → `findRuntimeBytecode()` returns `Optional.empty()` → `unless` guard evaluates `Optional.empty() == null` → `false` → `Optional.empty()` is stored in the Caffeine cache.
4. Wait for mirror-node ingestion to complete and confirm the bytecode is in the DB.
5. Send another `eth_getCode` for the same `contractId` — the cache returns `Optional.empty()`, the DB is never queried, and the response shows no bytecode.
6. Send one request per hour to reset the `expireAfterAccess` timer. The contract remains permanently unreachable via the web3 API.
