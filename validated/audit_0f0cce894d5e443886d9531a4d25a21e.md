Audit Report

## Title
Unbounded `account_aliases` Input in `ConstructionMetadata()` Enables DB Connection Pool Exhaustion via Concurrent Requests

## Summary
`resolveAccountAliases()` iterates over every comma-separated alias in the `account_aliases` option with no count cap, issuing one synchronous DB query per alias. With Traefik middleware disabled by default, an unauthenticated attacker can exhaust the DB connection pool by flooding `/construction/metadata` with large alias lists, denying service to all DB-dependent endpoints.

## Finding Description

**`ConstructionMetadata()` — unconditional call to `resolveAccountAliases()`:**

At lines 159–163 of `construction_service.go`, `resolveAccountAliases()` is called whenever `account_aliases` is present in the request options, with no prior validation of the alias string's length or element count. [1](#0-0) 

**`resolveAccountAliases()` — unbounded per-alias DB query loop:**

Lines 575–586 split the raw string on commas and call `c.accountRepo.GetAccountId()` synchronously for every element. There is no goroutine, no batching, and no cap on the number of iterations. [2](#0-1) 

**`GetAccountId()` — one DB connection acquired per call:**

Lines 146–158 of `account.go` call `ar.dbClient.GetDbWithContext(ctx)` for every alias that has an alias form, executing `selectCurrentCryptoEntityByAlias` before releasing the connection. [3](#0-2) 

**Root cause — failed assumption:**

`ConstructionPreprocess()` only ever sets a single alias (the payer), so the design assumes at most one alias per request. There is no enforcement of this assumption in `resolveAccountAliases()`. [4](#0-3) 

**Traefik middleware disabled by default:**

The entire middleware chain (`inFlightReq: 5`, `rateLimit: average: 10`) is only deployed when `global.middleware` is `true`. Its default value is `false`. [5](#0-4) 

The middleware template confirms the guard condition: [6](#0-5) 

The middleware definitions themselves (inFlightReq, rateLimit) are present but inert by default: [7](#0-6) 

**DB pool configuration:**

The `Pool` struct exposes `MaxOpenConnections`, confirming a finite, configurable pool ceiling exists. [8](#0-7) 

## Impact Explanation
When the pool is saturated by attacker-controlled requests, every other endpoint requiring a DB connection — block queries, balance queries, and the Kubernetes liveness (`/health/liveness`) and readiness (`/health/readiness`) probes — blocks or returns errors. Sustained probe failures cause the pod to be restarted or removed from the load balancer, resulting in a complete denial of service for the Rosetta API. [9](#0-8) [10](#0-9) 

## Likelihood Explanation
- No authentication is required to call `/construction/metadata`.
- Default Helm deployments have `global.middleware: false`, leaving no application-level rate limiting active.
- A single attacker machine can open many concurrent HTTP connections, each carrying a JSON body with a long comma-separated alias string — trivially scriptable.
- The attack is repeatable after any pod restart.

## Recommendation
1. **Enforce an alias count cap** in `resolveAccountAliases()`: reject requests where the number of comma-separated aliases exceeds 1 (matching the single-alias contract of `ConstructionPreprocess()`).
2. **Enable the Traefik middleware by default** (`global.middleware: true`) or document it as a required production setting.
3. **Add a maximum length check** on the raw `account_aliases` string before splitting.
4. Consider **batching alias lookups** into a single SQL `IN` query rather than N sequential queries.

## Proof of Concept

```bash
# Generate a large alias list (e.g., 500 valid-format aliases)
ALIASES=$(python3 -c "print(','.join(['0.0.' + str(i) + '-aabbccdd' for i in range(1,501)]))")

# Flood with 100 concurrent requests
for i in $(seq 1 100); do
  curl -s -X POST http://<rosetta-host>/construction/metadata \
    -H 'Content-Type: application/json' \
    -d "{\"network_identifier\":{\"blockchain\":\"Hedera\",\"network\":\"mainnet\"},\"options\":{\"operation_type\":\"CRYPTO_TRANSFER\",\"account_aliases\":\"$ALIASES\"}}" &
done
wait
```

Each concurrent request enters `resolveAccountAliases()` and holds DB connections for the duration of 500 sequential queries. With the pool exhausted, subsequent requests to any DB-dependent endpoint (including health probes) will block or fail. [11](#0-10) [12](#0-11)

```go
func (c *constructionAPIService) resolveAccountAliases(
	ctx context.Context,
	options map[string]any,
) (emptyResult string, nilErr *rTypes.Error) {
	if options[optionKeyAccountAliases] == nil {
		return
	}

	if !c.BaseService.IsOnline() {
		return emptyResult, errors.ErrEndpointNotSupportedInOfflineMode
	}

	accountAliases, ok := options[optionKeyAccountAliases].(string)
	if !ok {
		return emptyResult, errors.ErrInvalidOptions
	}

	var accountMap []string
	for accountAlias := range strings.SplitSeq(accountAliases, ",") {
		accountId, err := types.NewAccountIdFromString(accountAlias, c.systemShard, c.systemRealm)
		if err != nil {
			return emptyResult, errors.ErrInvalidAccount
		}

		found, rErr := c.accountRepo.GetAccountId(ctx, accountId)
		if rErr != nil {
			return emptyResult, rErr
		}
		accountMap = append(accountMap, fmt.Sprintf("%s:%s", accountAlias, found))
	}

	return strings.Join(accountMap, ","), nilErr
}
```

**File:** rosetta/app/persistence/account.go (L138-159)
```go
func (ar *accountRepository) GetAccountId(ctx context.Context, accountId types.AccountId) (
	zero types.AccountId,
	_ *rTypes.Error,
) {
	if !accountId.HasAlias() {
		return accountId, nil
	}

	db, cancel := ar.dbClient.GetDbWithContext(ctx)
	defer cancel()

	var entity domain.Entity
	if err := db.Raw(selectCurrentCryptoEntityByAlias, sql.Named("alias", accountId.GetAlias())).First(&entity).Error; err != nil {
		if errors.Is(err, gorm.ErrRecordNotFound) {
			return zero, hErrors.ErrAccountNotFound
		}

		return zero, hErrors.ErrDatabaseError
	}

	return types.NewAccountIdFromEntityId(entity.Id), nil
}
```
