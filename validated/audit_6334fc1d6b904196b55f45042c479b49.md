## Audit Report

## Title
Unauthenticated DoS via Uncached, Unthrottled `getSupply()` Endpoint Causing Unbounded Database Load

## Summary
The `GET /api/v1/network/supply` endpoint in the `rest-java` module accepts unauthenticated requests with no rate limiting and no result caching. Every request triggers a full native SQL aggregation over the `entity` table. An attacker flooding this endpoint can exhaust the PostgreSQL connection pool and degrade all API consumers sharing the same database instance.

## Finding Description

`NetworkController.getSupply()` is a public `@GetMapping` with no authentication or throttle guard: [1](#0-0) 

It delegates to `NetworkServiceImpl.getSupply()`, which — when no timestamp is provided — calls `entityRepository.getSupply()` directly with no caching layer: [8](#0-7) 

`EntityRepository.getSupply()` executes a native SQL `SUM`/`MAX` aggregation joined with `unnest()` across 7 ID ranges on every single invocation: [3](#0-2) 

The 7 default ranges are defined in `NetworkProperties`: [7](#0-6) 

**Root cause — failed assumptions:**

1. **No caching**: `EntityRepository.getSupply()` in `rest-java` has no `@Cacheable`. By contrast, `NodeStakeRepository.findAllStakeByConsensusTimestamp()` in the `grpc` module is explicitly cached, demonstrating the pattern is known and available but not applied here. [4](#0-3) 

2. **No rate limiting in `rest-java`**: `ThrottleConfiguration` (bucket4j-based) exists exclusively in the `web3` module. [9](#0-8) 

   The only filter registered in `rest-java` is `ShallowEtagHeaderFilter`, which reduces response bandwidth but still executes the full DB query before ETag comparison — providing zero protection against DB-level load: [6](#0-5) 

## Impact Explanation
Each request issues a `SUM`/`MAX` aggregation joined with `unnest()` across 7 ID ranges on the `entity` table, which on a production mirror node contains millions of rows. Flooding this endpoint saturates the PostgreSQL connection pool and increases I/O load, degrading response times for all other API consumers on the same database — including `/api/v1/network/fees` which shares the same DB instance.

## Likelihood Explanation
The endpoint requires zero authentication, zero API keys, and zero special headers. Any external user can issue `GET /api/v1/network/supply` in a tight loop from a single machine or distributed botnet. No existing application-layer control in `rest-java` will reject or slow these requests.

## Recommendation
1. **Add caching**: Apply `@Cacheable` to `EntityRepository.getSupply()` (or at the service layer in `NetworkServiceImpl.getSupply()`) with a TTL aligned to the importer ingestion cycle. The supply result is static between ingestion cycles, making it an ideal cache candidate — consistent with the pattern already used in `NodeStakeRepository`.
2. **Add rate limiting**: Port the bucket4j-based `ThrottleConfiguration` pattern from `web3` into `rest-java`, or register a servlet filter on `/api/v1/network/supply` that enforces a per-IP or global request rate.
3. **Infrastructure-level protection**: Apply rate limiting at the reverse proxy/load balancer layer as a defense-in-depth measure independent of application code.

## Proof of Concept
```bash
# Flood the endpoint from a single machine
while true; do
  curl -s "https://<mirror-node-host>/api/v1/network/supply" -o /dev/null &
done
```
Or using the k6 load test the project ships for this exact endpoint, which confirms the attack surface is already documented internally. Each concurrent request issues the full `SUM`/`MAX`/`unnest()` aggregation against the `entity` table with no application-layer gate.

    public NetworkSupply getSupply(Bound timestamp) {
        final NetworkSupply networkSupply;

        final var bounds = networkProperties.getUnreleasedSupplyRangeBounds();
        final var lowerBounds = bounds.lowerBounds();
        final var upperBounds = bounds.upperBounds();

        if (timestamp.isEmpty()) {
            networkSupply = entityRepository.getSupply(lowerBounds, upperBounds);
        } else {
            var minTimestamp = timestamp.getAdjustedLowerRangeValue();
            final var maxTimestamp = timestamp.adjustUpperBound();

            // Validate timestamp range
            if (minTimestamp > maxTimestamp) {
                throw new IllegalArgumentException("Invalid range provided for timestamp");
            }

            final var optimalLowerBound = getFirstDayOfMonth(maxTimestamp, -1);
            minTimestamp = Math.max(minTimestamp, optimalLowerBound);

            networkSupply =
                    accountBalanceRepository.getSupplyHistory(lowerBounds, upperBounds, minTimestamp, maxTimestamp);
        }

        if (networkSupply.consensusTimestamp() == 0L) {
            throw new EntityNotFoundException("Network supply not found");
        }

        return networkSupply;
    }

@Configuration(proxyBeanMethods = false)
@RequiredArgsConstructor
public class ThrottleConfiguration {

    public static final String GAS_LIMIT_BUCKET = "gasLimitBucket";
    public static final String RATE_LIMIT_BUCKET = "rateLimitBucket";
    public static final String OPCODE_RATE_LIMIT_BUCKET = "opcodeRateLimitBucket";

    private final ThrottleProperties throttleProperties;

    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    @Bean(name = GAS_LIMIT_BUCKET)
    Bucket gasLimitBucket() {
        long gasLimit = throttleProperties.getGasPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(gasLimit)
                .refillGreedy(gasLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder()
                .withSynchronizationStrategy(SynchronizationStrategy.SYNCHRONIZED)
                .addLimit(limit)
                .build();
    }

    @Bean(name = OPCODE_RATE_LIMIT_BUCKET)

        long rateLimit = throttleProperties.getOpcodeRequestsPerSecond();

                .build();
