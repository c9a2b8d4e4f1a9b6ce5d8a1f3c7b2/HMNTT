Audit Report

## Title
`Optional.empty()` Cached Permanently Due to Broken `unless` Condition in `findRuntimeBytecode`, Enabling Cache Poisoning

## Summary
The `@Cacheable` annotation on `findRuntimeBytecode()` uses `unless = "#result == null"` to prevent caching of absent results. Because the method returns `Optional<byte[]>`, Spring SpEL's `#result` refers to the `Optional` wrapper object — not its contents. `Optional.empty()` is a non-null singleton, so the guard never fires for empty results. Any unauthenticated caller can poison the cache for a specific contract ID with `Optional.empty()`, causing the mirror node to serve empty bytecode for that contract for up to 1 hour (or indefinitely via repeated access).

## Finding Description
**Exact location:** `web3/src/main/java/org/hiero/mirror/web3/repository/ContractRepository.java`, line 16:

```java
@Cacheable(cacheNames = CACHE_NAME_CONTRACT, cacheManager = CACHE_MANAGER_CONTRACT, unless = "#result == null")
Optional<byte[]> findRuntimeBytecode(final Long contractId);
``` [1](#0-0) 

**Root cause:** Spring's `unless` SpEL expression evaluates `#result` against the actual return value. Since the return type is `Optional<byte[]>`, `#result` is the `Optional` object itself. `Optional.empty()` is a non-null singleton, so `#result == null` is always `false` for the empty case. The intended guard — "don't cache absent results" — is never triggered.

**Cache configuration:** `web3/src/main/java/org/hiero/mirror/web3/repository/properties/CacheProperties.java`, line 22:

```java
private String contract = "expireAfterAccess=1h,maximumSize=1000,recordStats";
``` [2](#0-1) 

`expireAfterAccess` resets the TTL on every read, meaning a repeatedly-polled poisoned entry never expires.

**Consumer code:** `web3/src/main/java/org/hiero/mirror/web3/state/keyvalue/ContractBytecodeReadableKVState.java`, lines 42–46:

```java
return contractRepository
        .findRuntimeBytecode(entityId.getId())
        .map(Bytes::wrap)
        .map(Bytecode::new)
        .orElse(null);
``` [3](#0-2) 

A cached `Optional.empty()` causes `.orElse(null)` to return `null` to the EVM, which treats the contract as having no bytecode.

**Why existing tests fail to catch this:**
- `findRuntimeBytecodeFailCall` (lines 34–38) only asserts that an empty Optional is returned for a non-existent ID; it does not verify that a subsequent DB insert becomes visible through the cache.
- `findRuntimeBytecodeSuccessfulCall` (lines 17–31) only tests cache retention for a previously-found value, not the negative-cache poisoning scenario. [4](#0-3) 

## Impact Explanation
Any contract deployed on the Hedera network can be rendered permanently inaccessible via the mirror node's web3 API. All `eth_call`, `eth_getCode`, and contract execution requests for the targeted contract ID will fail or return empty bytecode. Because the cache uses `expireAfterAccess`, an attacker who polls the endpoint every ~59 minutes can keep the poisoned entry alive indefinitely. This directly causes the mirror node to misrepresent actual on-chain state for targeted contracts.

## Likelihood Explanation
The attack requires no credentials, no special permissions, and no on-chain capability. The only precondition is knowledge of a contract ID that is about to be (or has just been) imported — which is publicly observable by monitoring the Hedera network. The timing window (between on-chain finality and mirror node import) is well-known and typically seconds to minutes wide, making the race condition reliably exploitable. The attack is repeatable for any new contract deployment.

## Recommendation
Fix the `unless` expression to correctly handle `Optional` return types:

```java
@Cacheable(
    cacheNames = CACHE_NAME_CONTRACT,
    cacheManager = CACHE_MANAGER_CONTRACT,
    unless = "#result == null || !#result.isPresent()"
)
Optional<byte[]> findRuntimeBytecode(final Long contractId);
```

Additionally, add a test that:
1. Queries a non-existent contract ID (gets `Optional.empty()`).
2. Inserts the contract into the DB.
3. Queries again and asserts the new value is returned (not the cached empty).

## Proof of Concept
1. Identify a contract ID `X` that is on-chain but not yet imported by the mirror node importer.
2. Send an unauthenticated `eth_getCode` request for contract `X`. This routes through `ContractBytecodeReadableKVState.readFromDataSource()` → `contractRepository.findRuntimeBytecode(X)`.
3. DB has no row yet → returns `Optional.empty()`.
4. Spring evaluates `unless = "#result == null"` → `Optional.empty() != null` → `false` → **caches `Optional.empty()` for key `X`**.
5. Importer writes the contract's `runtime_bytecode` to the DB.
6. All subsequent calls to `findRuntimeBytecode(X)` return the cached `Optional.empty()` without hitting the DB.
7. `readFromDataSource()` calls `.orElse(null)` on the empty Optional and returns `null` to the EVM.
8. The EVM treats the contract as having no bytecode; all calls to it revert or fail.
9. Repeat the `eth_getCode` request every ~59 minutes to reset the `expireAfterAccess` TTL and keep the poisoned entry alive indefinitely.
