Audit Report

## Title
Missing Port Range Validation in `fetchMirrorNodeNetwork` Enables Mirror-Server-Controlled Network Partition

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` deserializes the `Port` field of each `serviceEndpoint` into a Go `int` with no upper-bound check. The only guard (`ep.Port == 0`) does not reject values above 65535. An attacker who can influence the mirror REST response can return out-of-range port values that are accepted, formatted into `host:port` strings, and handed to the Hiero SDK, causing every gRPC dial to fail while no error is surfaced to the caller.

## Finding Description
**Location:** `pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`.

`serviceEndpoint.Port` is declared as a plain Go `int` with no range annotation:

```go
// lines 30–34
type serviceEndpoint struct {
    DomainName  string `json:"domain_name"`
    IPAddressV4 string `json:"ip_address_v4"`
    Port        int    `json:"port"`
}
```

`json.Unmarshal` will decode `"port": 99999` or `"port": 2147483647` without error. Inside the loop the only filter is:

```go
// lines 118–120
if host == "" || ep.Port == 0 {
    continue
}
```

Values such as `99999` pass this check and are formatted directly into the network map:

```go
// lines 122–124
addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
network[addr] = nodeAccountId
```

After the loop the only downstream guard is:

```go
// lines 127–129
if len(network) == 0 {
    return nil, false, fmt.Errorf("no usable service_endpoints found …")
}
```

If the attacker returns at least one node with a valid host and an invalid port, `len(network) > 0` is true, no error is raised, and the poisoned map is returned to `buildNetworkFromMirrorNodes` → `hiero.ClientForNetwork(netmap)`.

**Transport exposure:** The HTTP client is constructed with no TLS and no certificate pinning (`&http.Client{Timeout: cfg.mirrorNodeClientTimeout}`), and the documented default REST URL is `http://mirror-rest:5551` (plain HTTP), making the response trivially interceptable by any on-path attacker.

## Impact Explanation
The pinger's entire node network is derived exclusively from this mirror REST call when `network=other`. A poisoned response with all ports > 65535 causes every gRPC dial attempt by the Hiero SDK to fail — the OS rejects port numbers outside 0–65535 at the socket layer. The pinger cannot submit any transactions, producing a sustained, silent network partition. No error is surfaced to the caller because the network map is non-empty. **Severity: High** — complete loss of pinger functionality with no self-healing until the mirror REST response is corrected.

## Likelihood Explanation
The default mirror REST URL uses plain HTTP. Any attacker with network adjacency (same Kubernetes cluster, compromised sidecar, ARP/BGP poisoning) can perform a MITM and inject a crafted `nodesEnvelope`. No credentials or special privileges on the target system are required — only the ability to intercept one unauthenticated HTTP response. The attack is repeatable on every retry cycle. Alternatively, a compromised mirror node can serve the malicious response directly.

## Recommendation
1. **Add port range validation** in the endpoint loop:
   ```go
   if host == "" || ep.Port <= 0 || ep.Port > 65535 {
       continue
   }
   ```
2. **Use HTTPS** for the mirror REST URL and enforce TLS certificate validation in the HTTP client to eliminate the MITM vector.
3. **Distinguish empty-map from all-invalid-port cases** so that a response containing only invalid endpoints is treated as a retryable error rather than a silent success.

## Proof of Concept
Serve the following JSON from the mirror REST endpoint (or inject it via MITM on the plain-HTTP connection):

```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        { "ip_address_v4": "1.2.3.4", "port": 99999 }
      ]
    }
  ],
  "links": { "next": null }
}
```

`fetchMirrorNodeNetwork` will:
1. Parse `port: 99999` into `ep.Port = 99999` without error.
2. Pass the `ep.Port == 0` check (99999 ≠ 0).
3. Build `addr = "1.2.3.4:99999"` and insert it into `network`.
4. Return the map (len = 1, no error).
5. The Hiero SDK receives `{"1.2.3.4:99999": AccountID{0,0,3}}` and every gRPC dial fails at the OS socket layer, partitioning the pinger from the network indefinitely.
