Audit Report

## Title
Unauthenticated HTTP 500 via qs Nested Object Injection in `requestQueryParser` for `order`/`result` Parameters

## Summary
The `requestQueryParser` function configures `qs` with `depth: 1` and `strictDepth: true`. This permits exactly one bracket nesting level, so `?order[key]=val` is accepted and parsed into `{order: {key: 'val'}}`. The `lowerCaseQueryValue` canonicalizer silently returns the object unchanged. Every downstream consumer that calls `.split(':')` or `.toLowerCase()` on the value throws a `TypeError`, which the error handler maps to HTTP 500. Any unauthenticated caller can trigger this on any endpoint accepting `order` or `result`.

## Finding Description

**Step 1 — Parser accepts nested object.** [1](#0-0) 
`qs` with `depth: 1` parses `order[key]=val` (depth = 1, at the limit) into `{order: {key: 'val'}}`. `strictDepth: true` only throws when depth *exceeds* the configured limit; depth 1 is explicitly permitted.

**Step 2 — `canonicalizeValue` passes the object to `lowerCaseQueryValue`.** [2](#0-1) 
`{key: 'val'}` is not an array, so `canonicalizationFunc(value)` is called directly.

**Step 3 — `lowerCaseQueryValue` silently returns the object unchanged.** [3](#0-2) 
`typeof {key:'val'} === 'string'` is `false`; the object is returned as-is and stored in `req.query.order`.

**Step 4 — `buildComparatorFilter` crashes on `.split(':')`.** [4](#0-3) 
`filter.split` is `undefined` on a plain object → `TypeError: filter.split is not a function`.

**Step 5 — `paramValidityChecks` crashes on `.split(':')`.** [5](#0-4) 
Same TypeError when `opAndVal` is an object.

**Step 6 — `isValidValueIgnoreCase` crashes on `.toLowerCase()`.** [6](#0-5) 
`value.toLowerCase` is `undefined` on a plain object → `TypeError`.

**Step 7 — OpenAPI validator does not block this input.** [7](#0-6) 
`validateRequests: false` means the OpenAPI middleware performs no request validation.

**Step 8 — Error handler maps TypeError to HTTP 500.** [8](#0-7) 
`TypeError` is not an `InvalidArgumentError`, `NotFoundError`, or `DbError`, so it falls through to `defaultStatusCode = httpStatusCodes.INTERNAL_ERROR` (HTTP 500). The server process does not crash; only the affected request receives 500.

**Root cause:** `queryCanonicalizationMap` assumes `order`/`result` values are always strings or arrays of strings. The `depth: 1` qs option contradicts this by allowing objects. `lowerCaseQueryValue`'s non-string guard silently propagates the object rather than rejecting it, and no type guard exists in `canonicalizeValue` or `requestQueryParser` to reject non-string, non-array values. [9](#0-8) 

## Impact Explanation
Any REST API endpoint that accepts `order` or `result` as a query parameter returns HTTP 500 instead of HTTP 400 for the malformed input `?order[key]=val`. This is repeatable on every request with no authentication required. While the server process itself does not crash (the error is caught by Express error-handling middleware), every such request produces an incorrect 500 response, degrading service availability for affected endpoints.

## Likelihood Explanation
Zero authentication or privilege is required. The payload is a single standard HTTP query string (`?order[key]=val`) that any HTTP client can send. It is trivially repeatable and automatable. The OpenAPI validator is configured with `validateRequests: false` and provides no protection.

## Recommendation
1. **Add a type guard in `canonicalizeValue`** (or in `requestQueryParser` before storing into `caseInsensitiveQueryString`) to reject any value that is not a string or an array of strings, throwing an `InvalidArgumentError` (which maps to HTTP 400).
2. **Alternatively, set `depth: 0`** in `queryOptions` to prevent qs from ever producing nested objects from query strings. This is the simplest fix and aligns with the intended use of flat query parameters.
3. **Add a type check in `paramValidityChecks` and `buildComparatorFilter`** as a defense-in-depth measure: if `opAndVal` is not a string, return `false` / throw `InvalidArgumentError` rather than calling `.split`.

## Proof of Concept
```

Host: <mirror-node-host>
```
Expected response: HTTP 400 (invalid parameter)
Actual response: HTTP 500 (Internal Server Error)

The same applies to `?result[x]=success` and any other endpoint accepting `order` or `result`.
