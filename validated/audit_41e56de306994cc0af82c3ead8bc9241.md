Audit Report

## Title
Plaintext gRPC by Default Enables MITM-Triggered Block Node Deactivation and Block Stream Ingestion Halt

## Summary
`BlockNodeProperties.requiresTls` defaults to `false`, causing `ManagedChannelBuilderProviderImpl` to open unauthenticated plaintext gRPC channels. A network-path attacker can inject malformed block stream responses that fail cryptographic verification, triggering `onError()` on each `BlockNode`. After `maxSubscribeAttempts` (default: 3) failures per node, the node is marked inactive. Sustaining this across all configured block nodes halts block stream ingestion entirely.

## Finding Description

**Root cause — TLS off by default:**

`BlockNodeProperties.requiresTls` is a Java primitive `boolean`, defaulting to `false`: [1](#0-0) 

`ManagedChannelBuilderProviderImpl.get()` calls `builder.usePlaintext()` when `useTls=false`, establishing an unencrypted, unauthenticated gRPC channel: [2](#0-1) 

The channel is built directly from `properties.isRequiresTls()`: [3](#0-2) 

**Exploit propagation chain:**

1. A MITM attacker intercepts the plaintext gRPC stream and injects a malformed `SubscribeStreamResponse`.

2. `BlockAssembler.onEndOfBlock()` assembles the block and calls `blockStreamConsumer.accept(new BlockStream(...))`: [4](#0-3) 

3. The consumer is the lambda `(stream) -> onBlockStream(stream, node.getProperties().getEndpoint())` from `BlockNodeSubscriber.doGet()`: [5](#0-4) 

4. `AbstractBlockSource.onBlockStream()` calls `blockStreamVerifier.verify(blockFile)` at line 53, which throws `HashMismatchException` or `InvalidStreamFileException` on injected malformed data: [6](#0-5) 

5. The exception propagates through the `while` loop in `streamBlocks()` and is caught by the `catch` blocks, both of which call `onError()`: [7](#0-6) 

6. `onError()` increments the error counter. After `maxSubscribeAttempts` (default: **3**) errors, it sets `active = false` and enforces a `readmitDelay` (default: **1 minute**) cooldown: [8](#0-7) [9](#0-8) 

7. `BlockNodeSubscriber.getNode()` skips inactive nodes. A fallback attempts to force-readmit inactive nodes that report having the block via `getBlockRange()` — but this gRPC call is also over the plaintext channel and can be intercepted. When all nodes are inactive and none can be force-readmitted, it throws `BlockStreamException("No block node can provide block " + nextBlockNumber.get())`, halting ingestion: [10](#0-9) 

**Why existing checks are insufficient:**

`BlockStreamVerifier.verify()` performs correct cryptographic hash-chain and signature verification: [11](#0-10) 

However, the system cannot distinguish between a legitimately misbehaving block node and a MITM-injected failure. Both paths call `onError()` identically. The attacker does not need to bypass cryptographic verification — they only need to cause it to fail, which is trivially achieved by injecting any malformed protobuf payload over the plaintext channel.

## Impact Explanation

With all block nodes marked inactive, the importer stops processing new blocks. The mirror node's database falls behind the chain, causing stale query results and unavailability of recent transaction data. This constitutes a complete halt of block stream ingestion.

## Likelihood Explanation

The attacker requires a MITM position on the network path between the importer and at least one block node. This is realistic in cloud environments (shared VPC, misconfigured security groups), Kubernetes clusters without strict network policies, or environments susceptible to ARP spoofing or DNS poisoning. With `maxSubscribeAttempts=3` (default), only **3 injected malformed responses per node** are needed to deactivate it. The `readmitDelay` of 1 minute means the attacker must sustain the attack at a low rate (3 injections per minute per node) to keep all nodes permanently inactive. This requires no special cryptographic capability.

## Recommendation

1. **Change the default for `requiresTls` to `true`** in `BlockNodeProperties.java`: [1](#0-0) 

2. **Add a validation constraint** (e.g., `@AssertTrue`) or a startup warning when `requiresTls=false` is explicitly configured, to prevent accidental plaintext deployment.

3. **Consider distinguishing error sources** in `onError()` — errors caused by local verification failure (which may indicate a bad node) versus transport-level errors — to make the deactivation logic more resilient to MITM-induced false positives. [8](#0-7) 

## Proof of Concept

```
1. Deploy a MITM proxy (e.g., mitmproxy with a custom script) on the network path
   between the importer and a block node configured with requiresTls=false (default).

2. Intercept any SubscribeStreamResponse of type END_OF_BLOCK.

3. Replace the BlockEnd payload with a response containing a corrupted block number
   or tampered block items, causing BlockStreamVerifier.verify() to throw
   HashMismatchException or InvalidStreamFileException.

4. Repeat 3 times per node (maxSubscribeAttempts=3).
   → BlockNode.onError() sets active=false for that node.

5. Repeat across all configured block nodes.
   → BlockNodeSubscriber.getNode() finds no active node with the required block
     and throws BlockStreamException("No block node can provide block N").

6. Block stream ingestion halts. Mirror node database stops advancing.

7. To sustain: re-inject 3 malformed responses per node every ~60 seconds
   (before readmitDelay expires) to keep all nodes permanently inactive.
```

    private BlockNode getNode(final AtomicLong nextBlockNumber) {
        final var nodeList = getBlockNodes();
        final var inactiveNodes = new ArrayList<BlockNode>();
        for (final var node : nodeList) {
            if (!node.tryReadmit(false).isActive()) {
                inactiveNodes.add(node);
                continue;
            }

            if (hasBlock(nextBlockNumber, node)) {
                return node;
            }
        }

        // find the first inactive node with the block and force activating it
        for (final var node : inactiveNodes) {
            if (hasBlock(nextBlockNumber, node)) {
                node.tryReadmit(true);
                return node;
            }
        }

        throw new BlockStreamException("No block node can provide block " + nextBlockNumber.get());
    }
