Audit Report

## Title
Integer Overflow in `NumberRangeParameter.getInclusiveValue()` Bypasses `hook.id` Range Filter in `HooksController`, Exposing All Hook Records for an Account

## Summary
Supplying `hook.id=gt:9223372036854775807` (`Long.MAX_VALUE`) causes `getInclusiveValue()` to silently overflow to `Long.MIN_VALUE`. `Math.max(0L, Long.MIN_VALUE)` resolves back to `0L`, nullifying the lower-bound filter entirely. The downstream repository query then returns every hook record for the target account owner, as if no filter had been applied.

## Finding Description

**Root cause — unchecked arithmetic in `NumberRangeParameter.getInclusiveValue()`:**

`value + 1` is performed with no overflow guard for the `GT` operator. [1](#0-0) 

**Input validation gap — `getNumberValue()` only rejects negatives:**

`Long.MAX_VALUE` (9223372036854775807) is positive, so it passes the `value < 0` guard without error. [2](#0-1) 

**Overflow silently discarded — `HooksController.hooksRequest()` line 142:**

`Math.max(0L, Long.MIN_VALUE)` evaluates to `0L`, leaving `lowerBound` at its default. [3](#0-2) 

**Missing post-loop guard — contrast with `NetworkServiceImpl.getNetworkNodes()`:**

`NetworkServiceImpl` throws `IllegalArgumentException` when `lowerBound > upperBound` after the same loop pattern. `hooksRequest()` has no equivalent guard. [4](#0-3) 

**Downstream query — `HookServiceImpl.getHooks()` line 40:**

With `lowerBound = 0` and `upperBound = Long.MAX_VALUE`, the repository call becomes a full-range scan for the owner. [5](#0-4) 

## Impact Explanation
An attacker who knows (or can enumerate) a valid `ownerId` can retrieve the complete set of hook records for that account — including `admin_key`, `contract_id`, `extension_point`, and associated timestamps — without any authentication. The intended ID-range filter is not weakened; it is completely nullified. The result set is identical to an unfiltered query for the owner.

## Likelihood Explanation
The endpoint is a public, unauthenticated HTTP GET. [6](#0-5) 

No credentials, tokens, or prior knowledge beyond a valid `ownerId` are required. The payload is a single, trivially constructed query parameter. The exploit is deterministic, requires no brute-force or timing, and is repeatable by any external caller who can reach the API.

## Recommendation

1. **Add an upper-bound cap in `getNumberValue()`** — reject values equal to `Long.MAX_VALUE` when the operator is `GT` (or cap the maximum accepted value to `Long.MAX_VALUE - 1`): [2](#0-1) 

2. **Add a post-loop `lowerBound > upperBound` guard in `hooksRequest()`**, mirroring the existing guard in `NetworkServiceImpl.getNetworkNodes()`: [7](#0-6) 

3. **Use `Math.addExact()` instead of `+` in `getInclusiveValue()`** to throw `ArithmeticException` on overflow rather than silently wrapping: [8](#0-7) 

## Proof of Concept

```
GET /api/v1/accounts/0.0.123/hooks?hook.id=gt:9223372036854775807
```

Step-by-step:
1. `getNumberValue("9223372036854775807")` → `Long.MAX_VALUE`; passes `value < 0` check.
2. `getInclusiveValue()` (GT branch): `Long.MAX_VALUE + 1` → Java long overflow → `Long.MIN_VALUE` (-9223372036854775808).
3. `Math.max(0L, Long.MIN_VALUE)` → `lowerBound = 0L`.
4. `upperBound` is untouched at `Long.MAX_VALUE`.
5. No `lowerBound > upperBound` guard fires (0 < Long.MAX_VALUE).
6. `findByOwnerIdAndHookIdBetween(123, 0, Long.MAX_VALUE, page)` — returns all hooks for account `0.0.123`.

    private HooksRequest hooksRequest(
            EntityIdParameter ownerId, NumberRangeParameter[] hookIdFilters, int limit, Sort.Direction order) {
        final var hookIds = new TreeSet<Long>();
        long lowerBound = 0L; // The most restrictive lower bound (max of all gt/gte)
        long upperBound = MAX_VALUE; // The most restrictive upper bound (min of all lt/lte)

        for (final var hookIdFilter : hookIdFilters) {
            if (hookIdFilter.operator() == RangeOperator.EQ) {
                hookIds.add(hookIdFilter.value());
            } else if (hookIdFilter.hasLowerBound()) {
                lowerBound = Math.max(lowerBound, hookIdFilter.getInclusiveValue());
            } else if (hookIdFilter.hasUpperBound()) {
                upperBound = Math.min(upperBound, hookIdFilter.getInclusiveValue());
            }
        }

        return HooksRequest.builder()
                .hookIds(hookIds)
                .lowerBound(lowerBound)
                .ownerId(ownerId)
                .limit(limit)
                .order(order)
                .upperBound(upperBound)
                .build();
    }
