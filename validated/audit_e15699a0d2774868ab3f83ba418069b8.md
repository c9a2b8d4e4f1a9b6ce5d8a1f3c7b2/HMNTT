## Audit Report

## Title
Long Overflow in `NumberRangeParameter.getInclusiveValue()` Bypasses Filter in `resolveRegisteredNodeIdBounds()`

## Summary
Passing `gt:9223372036854775807` (`GT:Long.MAX_VALUE`) to the `registerednode.id` query parameter causes a signed-integer overflow in `getInclusiveValue()`. The overflowed value (`Long.MIN_VALUE`) is silently absorbed by `Math.max`, resetting the lower bound to `0`. The subsequent guard never fires, and the query returns all registered nodes instead of an empty result set.

## Finding Description

**`NumberRangeParameter.getInclusiveValue()`** [1](#0-0) 

```java
public long getInclusiveValue() {
    if (operator == RangeOperator.GT) {
        return value + 1;   // Long.MAX_VALUE + 1 wraps to Long.MIN_VALUE
    }
    ...
}
```

**`getNumberValue()`** only rejects negative input; `Long.MAX_VALUE` passes: [2](#0-1) 

**`resolveRegisteredNodeIdBounds()`** — three sequential failures: [3](#0-2) 

1. Line 165: `Math.max(0L, Long.MIN_VALUE)` evaluates to `0L` — overflow silently absorbed.
2. Line 171: `0 > Long.MAX_VALUE` is `false` — guard never throws.
3. Line 175: `Range.closed(0, Long.MAX_VALUE)` is returned — full table scan.

**Downstream call:** [5](#0-4) 

`findByRegisteredNodeIdBetweenAndDeletedIsFalseAndTypeIs(0, Long.MAX_VALUE, ...)` returns every registered node.

## Impact Explanation
A request that semantically means "nodes with ID greater than the maximum possible value" (i.e., an empty result) instead returns the complete set of registered nodes. This violates the filter contract of the API and causes mirror nodes consuming this endpoint to receive unfiltered node data, corrupting their view of the registered node set.

## Likelihood Explanation
No authentication is required. The exploit is a single HTTP GET request with one crafted query parameter. It is deterministic, requires no special tooling, and is trivially repeatable by any party with network access to the REST API.

## Recommendation
Add an explicit upper-bound check in `getNumberValue()` or `getInclusiveValue()` to reject values that would overflow when the `GT` operator is applied:

```java
private static long getNumberValue(String number) {
    var value = Long.parseLong(number);
    if (value < 0) {
        throw new IllegalArgumentException("Invalid range value");
    }
    return value;
}

public long getInclusiveValue() {
    if (operator == RangeOperator.GT) {
        if (value == Long.MAX_VALUE) {
            throw new IllegalArgumentException(
                "GT operator with Long.MAX_VALUE produces no valid results");
        }
        return value + 1;
    }
    ...
}
```

Alternatively, use `Math.addExact(value, 1)` which throws `ArithmeticException` on overflow, and map that to a 400 Bad Request response. The same fix should be applied to the analogous `LT` path (`value - 1` when `value == Long.MIN_VALUE`) and to the identical pattern in `getNetworkNodes()`. [6](#0-5) 

## Proof of Concept

```
GET /api/v1/network/nodes/registered?registerednode.id=gt:9223372036854775807
```

Step-by-step:
1. `NumberRangeParameter.valueOf("gt:9223372036854775807")` → `new NumberRangeParameter(GT, 9223372036854775807L)`
2. `getNumberValue("9223372036854775807")` → `9223372036854775807L` (passes `< 0` guard)
3. `getInclusiveValue()` → `9223372036854775807L + 1` = `-9223372036854775808L` (`Long.MIN_VALUE`)
4. `Math.max(0L, -9223372036854775808L)` = `0L`
5. `lowerBound = 0`, `upperBound = Long.MAX_VALUE`; guard `0 > Long.MAX_VALUE` = `false`
6. `findByRegisteredNodeIdBetweenAndDeletedIsFalseAndTypeIs(0, Long.MAX_VALUE, ...)` → **all registered nodes returned**

    private static Range<Long> resolveRegisteredNodeIdBounds(List<NumberRangeParameter> registeredNodeIdRanges) {
        long lowerBound = 0L;
        long upperBound = MAX_VALUE;

        for (final var range : registeredNodeIdRanges) {
            if (range.operator() == RangeOperator.EQ) {
                if (registeredNodeIdRanges.size() > 1) {
                    throw new IllegalArgumentException("The 'eq' operator cannot be combined with other operators");
                }
                return Range.closed(range.value(), range.value());
            } else if (range.hasLowerBound()) {
                lowerBound = Math.max(lowerBound, range.getInclusiveValue());
            } else if (range.hasUpperBound()) {
                upperBound = Math.min(upperBound, range.getInclusiveValue());
            }
        }

        if (lowerBound > upperBound) {
            throw new IllegalArgumentException("Invalid range: lower bound exceeds upper bound");
        }

        return Range.closed(lowerBound, upperBound);
    }
