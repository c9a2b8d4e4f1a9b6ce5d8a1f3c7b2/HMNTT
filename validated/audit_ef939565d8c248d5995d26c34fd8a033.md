Audit Report

## Title
Unauthenticated HTTP Redirect Injection Allows Full Node-Map Poisoning via `buildNetworkFromMirrorNodes()`

## Summary
`buildNetworkFromMirrorNodes()` in `pinger/mirror_node_client.go` constructs an `http.Client` with only a timeout and no `CheckRedirect` policy. Combined with the plaintext HTTP default (`http://mirror-rest:5551`), a network-adjacent attacker can intercept the initial request, reply with an HTTP 301/302 redirect to an attacker-controlled server, and have Go's default redirect-following silently fetch a crafted `nodesEnvelope`. The resulting poisoned network map is passed directly to `hiero.ClientForNetwork()`, routing all subsequent pinger transactions to attacker-controlled gRPC endpoints.

## Finding Description
**File:** `pinger/mirror_node_client.go`
**Function:** `buildNetworkFromMirrorNodes()`, line 46:
```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
```
No `CheckRedirect` field is set. Go's `net/http` default (`defaultCheckRedirect`) follows up to 10 redirects unconditionally, including cross-origin redirects to arbitrary hosts. [1](#0-0) 

The URL is built from `cfg.mirrorRest` (env var `HIERO_MIRROR_PINGER_REST`), whose documented example is `http://mirror-rest:5551` — plaintext HTTP. [5](#0-4) 

`fetchMirrorNodeNetwork()` checks the HTTP status code only **after** all redirects have already been followed. It does not inspect `resp.Request.URL` to verify the final host matches the originally configured mirror REST origin. [2](#0-1) 

The returned `network` map is passed without further validation to `hiero.ClientForNetwork(netmap)`: [3](#0-2) 

**Root cause:** `http.Client` is created with only a timeout; no `CheckRedirect` to block cross-origin redirects, no `Transport` enforcing TLS, and no post-request URL origin check.

## Impact Explanation
An attacker who poisons the network map gains full control over where the pinger sends Hedera transactions:
- **Transaction suppression:** Attacker's gRPC servers drop all transactions, causing the pinger to report continuous failures — a complete monitoring/liveness blackout.
- **Transaction inspection/replay:** Attacker receives fully signed transactions (including operator key signatures) and can log, delay, or selectively replay them.
- **False confirmation:** Attacker can return synthetic success responses, masking a real network outage.

## Likelihood Explanation
- The default and documented deployment uses plaintext HTTP (`http://mirror-rest:5551`), making passive interception trivial on any shared network segment.
- In Kubernetes, any compromised pod on the same CNI overlay can perform ARP/NDP spoofing or DNS poisoning without cluster-admin privileges.
- The attack is repeatable: `buildNetworkFromMirrorNodes()` is called at startup; every pinger restart re-poisons the map if the MITM is sustained.
- No authentication, no TLS, no redirect guard — only a MITM position is required, not any credential or system privilege on the target.

## Recommendation
1. **Block cross-origin redirects** by setting a `CheckRedirect` policy on the `http.Client`:
   ```go
   httpClient := &http.Client{
       Timeout: cfg.mirrorNodeClientTimeout,
       CheckRedirect: func(req *http.Request, via []*http.Request) error {
           if len(via) > 0 && req.URL.Host != via[0].URL.Host {
               return fmt.Errorf("redirect to different host %s blocked", req.URL.Host)
           }
           return nil
       },
   }
   ```
2. **Enforce TLS** for the mirror REST endpoint in production deployments; reject `http://` scheme URLs at startup.
3. **Post-request origin check:** After `httpClient.Do(req)`, verify `resp.Request.URL.Host` matches the originally configured host before decoding the response body.

## Proof of Concept
1. Attacker achieves MITM on the plaintext HTTP path (e.g., ARP spoofing, DNS poisoning, or rogue pod on the same Kubernetes overlay).
2. Attacker intercepts `GET http://mirror-rest:5551/api/v1/network/nodes`.
3. Attacker replies: `HTTP/1.1 301 Moved Permanently` + `Location: http://attacker.example.com/nodes`.
4. Go's `http.Client` silently follows the redirect and issues `GET http://attacker.example.com/nodes`.
5. Attacker's server returns a valid-looking `nodesEnvelope` JSON with all `node_account_id` values mapped to attacker-controlled `service_endpoints`.
6. `fetchMirrorNodeNetwork()` decodes the payload, builds the `network` map, and returns it without error (status 200 from attacker server passes the `< 200 || >= 300` check).
7. `hiero.ClientForNetwork(netmap)` configures the Hedera SDK client to use only attacker endpoints.
8. Every subsequent `submitWithRetry` call sends signed transactions to the attacker's gRPC servers.
