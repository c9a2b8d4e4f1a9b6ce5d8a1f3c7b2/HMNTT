Audit Report

## Title
Cache Poisoning via Non-Existent Storage Slots Causes Unbounded DB Query Amplification in `findStorageBatch()`

## Summary
`ContractStateServiceImpl` never writes a sentinel value to `contractStateCache` for storage slots that do not exist in the database. As a result, every request for a non-existent slot bypasses the cache guard and calls `findStorageBatch()`, which accumulates all previously requested non-existent slot keys in the per-contract `contractSlotsCache` and issues a batch DB query containing the full accumulated key list. An attacker can grow this list to the cache size limit (1500 keys), causing every future storage lookup for that contract to execute an increasingly expensive SQL `IN (...)` query.

## Finding Description

**Step 1 — Cache guard in `findStorage()`:**

The guard at line 65 only short-circuits when `cachedValue != null && cachedValue != EMPTY_VALUE`. [1](#0-0) 

For a non-existent slot, `contractStateCache.get(...)` returns `null` (no sentinel is ever written for absent slots), so the condition is `false` and `findStorageBatch()` is called unconditionally on every request for that slot.

**Step 2 — Unconditional accumulation in `findStorageBatch()`:**

Every call to `findStorageBatch()` adds the requested key to the per-contract `contractSlotsCache` via `putIfAbsent`, regardless of whether the slot exists in the DB: [2](#0-1) 

**Step 3 — Full accumulated key set sent to DB:**

All keys currently in the per-contract cache are collected and passed to the repository batch query: [3](#0-2) 

**Step 4 — Only DB-returned slots are written to `contractStateCache`:**

The loop that populates `contractStateCache` only iterates over rows returned by the DB. Non-existent slots are never written, so they never satisfy the guard in Step 1: [4](#0-3) 

**Cache size limits (confirmed):**

- `slotsPerContract`: `expireAfterAccess=5m, maximumSize=1500`
- `contractSlots` (outer): `expireAfterAccess=5m, maximumSize=3000`
- `contractState`: `expireAfterWrite=2s, maximumSize=25000` [5](#0-4) 

**Root cause:** The design assumes that `contractSlotsCache` only holds keys whose values are "not yet fetched," but it never cleans up keys for slots confirmed absent by the DB. There is no sentinel write-back for missing slots, so the cache guard in `findStorage()` can never short-circuit for them.

## Impact Explanation

Once an attacker fills the per-contract `slotsPerContract` cache with 1500 non-existent slot keys, every call to `findStorageBatch()` for that contract — triggered by any legitimate storage miss — executes a SQL query with up to 1500 32-byte keys in the `IN (...)` clause: [6](#0-5) 

Because `contractStateCache` expires after `expireAfterWrite=2s`, even legitimate callers continuously re-trigger `findStorageBatch()`, each time paying the cost of the inflated query. The attack is persistent for the 5-minute `expireAfterAccess` window and trivially renewable. This constitutes a sustained, amplified DB load increase achievable without any privileged access.

## Likelihood Explanation

Any user who can submit `eth_call` or `eth_estimateGas` requests can trigger this. No authentication, tokens, or special role is required. The attacker only needs a valid contract address and sends requests reading from 1500 distinct non-existent storage slots — trivially scriptable. The attack is repeatable every 5 minutes to keep the poisoned cache alive indefinitely. [7](#0-6) 

## Recommendation

Write a sentinel value (e.g., `EMPTY_VALUE`) into `contractStateCache` for slots confirmed absent by the DB query. After the `findStorageBatch()` DB call, for every key in `cachedSlots` that was **not** returned by the DB, explicitly call:

```java
contractStateCache.put(generateCacheKey(contractId, missingSlotKey), EMPTY_VALUE);
```

Then update the guard in `findStorage()` to also short-circuit on `EMPTY_VALUE` (returning `Optional.empty()`):

```java
if (cachedValue != null) {
    return cachedValue == EMPTY_VALUE ? Optional.empty() : Optional.of(cachedValue);
}
```

This ensures non-existent slots are cached as absent and never re-accumulate in the batch query key set. [8](#0-7) 

## Proof of Concept

```
1. Identify a valid deployed contract address (contractId = X).
2. Send 1500 eth_call requests, each reading a distinct storage slot
   that does not exist in the DB (e.g., slot keys 0x0001...0001 through 0x0001...05DC).
3. Each request causes findStorageBatch() to add the key to contractSlotsCache.
   After 1500 requests, the per-contract cache is full with 1500 non-existent keys.
4. Now send any eth_call that reads any storage slot of contract X.
   - contractStateCache misses (expireAfterWrite=2s).
   - findStorageBatch() is called.
   - cachedSlots contains all 1500 non-existent keys.
   - DB receives: SELECT slot, value FROM contract_state
                  WHERE contract_id = X AND slot IN (<1500 keys>)
5. Repeat step 4 continuously (or let legitimate traffic do it).
   Every 2 seconds, the contractStateCache expires and the inflated query fires again.
6. Repeat the poisoning every 5 minutes to renew the expireAfterAccess window.
``` [8](#0-7)

        if (!cacheProperties.isEnableBatchContractSlotCaching()) {

    private Optional<byte[]> findStorageBatch(final EntityId contractId, final byte[] key) {
        final var contractSlotsCache = ((CaffeineCache) this.contractSlotsCache.get(
                contractId, () -> cacheManagerSlotsPerContract.getCache(contractId.toString())));
        final var wrappedKey = ByteBuffer.wrap(key);
        // Cached slot keys for contract, whose slot values are not present in the contractStateCache
        contractSlotsCache.putIfAbsent(wrappedKey, EMPTY_VALUE);
        final var cachedSlotKeys = contractSlotsCache.getNativeCache().asMap().keySet();

        final var cachedSlots = new ArrayList<byte[]>(cachedSlotKeys.size());
        boolean isKeyEvictedFromCache = true;

        for (var slot : cachedSlotKeys) {
            cachedSlots.add(((ByteBuffer) slot).array());
            if (wrappedKey.equals(slot)) {
                isKeyEvictedFromCache = false;
            }
        }

        final var contractSlotValues = contractStateRepository.findStorageBatch(contractId.getId(), cachedSlots);
        byte[] cachedValue = null;

        for (final var contractSlotValue : contractSlotValues) {
            final byte[] slotKey = contractSlotValue.getSlot();
            final byte[] slotValue = contractSlotValue.getValue();
            contractStateCache.put(generateCacheKey(contractId, slotKey), slotValue);

            if (Arrays.equals(slotKey, key)) {
                cachedValue = slotValue;
            }
        }

        // If the cache key was evicted and hasn't been requested since, the cached value will be null.
        // In that case, fall back to the original query.
        if (isKeyEvictedFromCache) {
            return contractStateRepository.findStorage(contractId.getId(), key);
        }
        return Optional.ofNullable(cachedValue);
    }
