Audit Report

## Title
Throttle Budget Permanently Consumed via `IllegalArgumentException` from Odd-Length Hex `data` Field Bypassing `InvalidParametersException` Catch Block

## Summary
The `call()` method in `ContractController` restores throttle tokens only on `InvalidParametersException`. However, odd-length hex strings in the `data` field pass `@Hex` bean validation, then cause `hexToBytes()` to throw `IllegalArgumentException` — which is not caught by the restoration block. Each such request permanently consumes gas tokens from the throttle bucket, enabling unauthenticated DoS against the gas throttle subsystem.

## Finding Description

**Step 1 — Throttle consumed unconditionally.**
`throttleManager.throttle(request)` is called at line 40 before any parameter processing. The only restoration path is the `catch (InvalidParametersException e)` block at line 46. [1](#0-0) 

**Step 2 — `@Hex` validator does not enforce even-length.**
The `HexValidator` compiles the pattern `^(0x)?[0-9a-fA-F]{%d,%d}$`, which matches any number of hex characters including odd counts. There is no modulo-2 check. [2](#0-1) 

The `data` field on `ContractCallRequest` carries only `@Hex` with default bounds, so odd-length strings like `"1"` or `"0xa"` pass `@Valid` without rejection. [3](#0-2) 

**Step 3 — The try/catch in `constructServiceParameters()` is dead code and does not protect `hexToBytes()`.**
The try block at lines 67–72 wraps only the null-check assignment `data = request.getData() != null ? request.getData() : HEX_PREFIX;`, which can never throw. The actual `hexToBytes(data)` call is at line 80, **outside** this try/catch. [4](#0-3) 

**Step 4 — `hexToBytes()` throws `IllegalArgumentException` for odd-length input.**
`org.apache.commons.codec.binary.Hex.decodeHex()` requires an even number of hex characters. `BytesDecoder.hexToBytes()` catches the resulting `DecoderException` and re-throws it as `IllegalArgumentException` — not `InvalidParametersException`. [5](#0-4) 

**Step 5 — `IllegalArgumentException` bypasses the restoration block.**
The `IllegalArgumentException` propagates out of `constructServiceParameters()` and up to `call()`, where the `catch (InvalidParametersException e)` block does not match it. `throttleManager.restore()` is never called. The exception is ultimately caught by `GenericControllerAdvice.badRequest()`, which returns HTTP 400 but performs no token restoration. [6](#0-5) 

## Impact Explanation
Each malformed request permanently consumes `throttleProperties.scaleGas(request.getGas())` tokens from the `gasLimitBucket`. With the default `gas = 15_000_000`, an attacker can drain the bucket in a small number of requests, causing all subsequent legitimate callers to receive HTTP 429 (Too Many Requests). The `ThrottleManager.restore(long gas)` interface exists precisely to prevent this, but is never invoked on this code path. [7](#0-6) [8](#0-7) 

## Likelihood Explanation
The endpoint `/api/v1/contracts/call` is unauthenticated. The payload requires only a single odd-length hex character in the `data` field (e.g., `{"data":"1"}`). The attack is trivially automatable and fully repeatable. The existing test suite already exercises this exact path and confirms the HTTP 400 response with "Odd number of characters", proving the `IllegalArgumentException` escapes the controller in production-equivalent conditions. [9](#0-8) 

## Recommendation

**Primary fix — move `hexToBytes()` inside a try/catch that throws `InvalidParametersException`:**

In `constructServiceParameters()`, replace the current dead try/catch with one that wraps the `hexToBytes(data)` call at line 80:

```java
byte[] callData;
try {
    callData = hexToBytes(data);
} catch (IllegalArgumentException e) {
    throw new InvalidParametersException(
        "data field '%s' contains invalid hex characters: %s".formatted(data, e.getMessage()));
}
```

This ensures the `catch (InvalidParametersException e)` block in `call()` is reached and `throttleManager.restore()` is called.

**Secondary fix — enforce even-length in `HexValidator`:**

Add a length-parity check in `HexValidator.isValid()`:

```java
String stripped = value.startsWith("0x") ? value.substring(2) : value;
if (stripped.length() % 2 != 0) return false;
```

This rejects odd-length hex at the `@Valid` boundary (before throttle tokens are consumed), returning HTTP 400 via `MethodArgumentNotValidException` before `throttleManager.throttle()` is ever called. [10](#0-9) 

## Proof of Concept

```http
POST /api/v1/contracts/call HTTP/1.1
Content-Type: application/json

{"data":"1","gas":15000000,"to":"0x0000000000000000000000000000000000000001"}
```

1. `@Valid` passes — `"1"` matches `^(0x)?[0-9a-fA-F]{0,1048576}$`.
2. `throttleManager.throttle(request)` consumes 15,000,000 gas tokens.
3. `constructServiceParameters()` reaches `hexToBytes("1")` at line 80.
4. `Hex.decodeHex("1")` throws `DecoderException("Odd number of characters.")`.
5. `BytesDecoder` re-throws as `IllegalArgumentException`.
6. `catch (InvalidParametersException e)` in `call()` does not match; `restore()` is not called.
7. `GenericControllerAdvice.badRequest()` returns HTTP 400.
8. Repeat until the gas bucket is exhausted; legitimate callers receive HTTP 429. [1](#0-0) [11](#0-10)
