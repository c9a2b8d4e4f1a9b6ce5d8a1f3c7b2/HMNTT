Audit Report

## Title
Index Out-of-Bounds Panic in `getSdkPayerAccountId` via Crafted `account_map` Metadata

## Summary
An unauthenticated caller of the `/construction/payloads` endpoint can supply a `Metadata["account_map"]` value whose comma-separated entries match the payer alias prefix but contain no `:` separator. `strings.Split(aliasMap, ":")` returns a single-element slice, and the unconditional access to `mapping[1]` triggers a runtime index-out-of-range panic, aborting the goroutine and returning HTTP 500.

## Finding Description
**Exact location:** `rosetta/app/services/construction_service.go`, function `getSdkPayerAccountId`, lines 497–498. [1](#0-0) 

```go
mapping := strings.Split(aliasMap, ":")
if payer, err = hiero.AccountIDFromString(mapping[1]); err != nil {  // panics when len(mapping)==1
```

**Data flow from user input to panic site:**

1. `ConstructionPayloads` (line 218) receives a `*rTypes.ConstructionPayloadsRequest` whose `Metadata` map is entirely attacker-controlled. [2](#0-1) 

2. Line 243 passes `request.Metadata[metadataKeyAccountMap]` directly as the second argument to `getSdkPayerAccountId` — no sanitisation, no schema validation. [3](#0-2) 

3. Inside `getSdkPayerAccountId` (line 467), the guards before the split are:
   - `payerAccountId.HasAlias()` must be true (line 471) — satisfied by crafting operations whose first signer is an alias-based account.
   - `accountMapMetadata != nil` (line 480) — trivially satisfied.
   - Type assertion to `string` (line 484) — trivially satisfied.
   - `strings.HasPrefix(aliasMap, payerAlias)` (line 492) — satisfied by making the entry start with the payer alias. [4](#0-3) 

4. None of these guards verify that a `:` separator is present. `strings.Split("aliasNOCOLON", ":")` returns `[]string{"aliasNOCOLON"}` (length 1). `mapping[1]` is an unconditional out-of-bounds access → runtime panic. [1](#0-0) 

**Root cause:** Missing `len(mapping) >= 2` guard between lines 497 and 498. The `strings.HasPrefix` check only filters entries that do not begin with the payer alias; it does not guarantee a `:` is present.

## Impact Explanation
Go's `net/http` server recovers the panic per-goroutine, preventing full-process termination. However:
- The HTTP response for the crafted request is a 500 with no body.
- A full goroutine stack trace is written to the server error log on every triggered request.
- The construction/payloads endpoint is rendered non-functional for every request carrying the crafted metadata.
- Repeated requests can saturate log storage and obscure legitimate error signals.

## Likelihood Explanation
The `/construction/payloads` endpoint is a standard, unauthenticated Rosetta API surface. No credentials, tokens, or privileged network position are required. The attacker only needs to:
1. Know the alias string of the payer (derivable from the operations they craft themselves).
2. Set `Metadata["account_map"]` to `"<payerAlias>NOCOLON"`.

A single HTTP request is sufficient to trigger the panic; no flooding is required to demonstrate the bug.

## Recommendation
Add a bounds check immediately after the split:

```go
mapping := strings.Split(aliasMap, ":")
if len(mapping) != 2 {
    return zero, errors.ErrInvalidAccount
}
if payer, err = hiero.AccountIDFromString(mapping[1]); err != nil {
    return zero, errors.ErrInvalidAccount
}
```

Alternatively, use `strings.Cut(aliasMap, ":")` which returns `(before, after string, found bool)` and makes the missing-separator case explicit without requiring an index access.

## Proof of Concept

```http
POST /construction/payloads HTTP/1.1
Content-Type: application/json

{
  "network_identifier": { "blockchain": "Hedera", "network": "testnet" },
  "operations": [
    {
      "operation_identifier": { "index": 0 },
      "type": "CRYPTO_TRANSFER",
      "account": { "address": "0.0.AABBCCDD" },
      "amount": { "value": "-100", "currency": { "symbol": "HBAR", "decimals": 8 } }
    }
  ],
  "metadata": {
    "account_map": "0.0.AABBCCDDNOCOLON"
  }
}
```

`0.0.AABBCCDD` is an alias-based account (`HasAlias()` returns true). The `account_map` entry starts with the payer alias (`HasPrefix` passes) but contains no `:`. `strings.Split` returns a 1-element slice, `mapping[1]` panics, and the server returns HTTP 500. [5](#0-4)
