# Audit Report

## Title
Unauthenticated Rate-Unlimited `account()` GraphQL Query Enables Connection Pool Exhaustion DoS

## Summary
The GraphQL module exposes a publicly accessible `account()` query with no rate limiting, no authentication, and no concurrency control. An attacker can exhaust the shared HikariCP database connection pool with a small number of concurrent requests, causing all subsequent legitimate queries to time out.

## Finding Description

**`AccountController.account()`** accepts any unauthenticated POST to `/graphql/alpha` and immediately calls `entityService.getByIdAndType()`, which acquires a HikariCP connection from the shared pool. [1](#0-0) 

**No rate limiting exists in the graphql module.** `GraphQlConfiguration` registers only `MaxQueryComplexityInstrumentation(200)` and `MaxQueryDepthInstrumentation(10)`, plus JSON/parser size limits. None of these limit request rate or concurrency per client. [2](#0-1) 

A grep for `rateLimiter`, `RateLimiter`, `throttle`, `bucket4j`, or `resilience4j` across all `graphql/**/*.java` files returns **zero results**, confirming the absence of any rate-limiting mechanism.

**Asymmetric protection:** The `web3` module explicitly implements token-bucket rate limiting via `ThrottleConfiguration` (bucket4j) with `requestsPerSecond` and `gasPerSecond` limits. [3](#0-2) 

The `graphql` module has no equivalent. This is a deliberate design gap.

**Shared HikariCP pool with no graphql-specific override.** `CommonConfiguration` creates a `HikariDataSource` bound to `spring.datasource.hikari` properties. [4](#0-3) 

No `hikari` or `maximumPoolSize` configuration exists anywhere in `graphql/src/main/resources/` (the resources directory contains only GraphQL schema files and `banner.txt`, no `application.yml`), so HikariCP's default of **10 connections** applies.

**No authentication.** No `SecurityFilterChain`, `HttpSecurity`, or Spring Security configuration exists in `graphql/src/main/java/`. The only filter, `LoggingFilter`, exclusively logs requests and enforces nothing. [5](#0-4) 

## Impact Explanation

With 10 HikariCP connections (default) and no rate limiting, an attacker holding â‰Ą10 concurrent long-running queries exhausts the entire pool. Every subsequent legitimate request queues for up to 30 seconds (HikariCP default `connectionTimeout`) then fails with `Connection is not available, request timed out`. This is a complete application-layer partition from the database â€” all GraphQL queries fail for all users until the attacker stops.

## Likelihood Explanation

**Preconditions:** None. The endpoint is unauthenticated and publicly accessible.

**Attacker capability:** Any script with `curl -Z` or `ab` (Apache Bench). No credentials, tokens, or special knowledge required.

**Repeatability:** Fully repeatable and automatable. Only 10 concurrent HTTP connections are needed to saturate the default pool â€” well within reach of a single machine.

## Recommendation

1. **Add application-level rate limiting** to the graphql module using bucket4j (already a dependency in `web3`), mirroring `ThrottleConfiguration`.
2. **Configure an explicit `maximumPoolSize`** via `spring.datasource.hikari.maximum-pool-size` in `graphql/src/main/resources/application.yml` appropriate for expected concurrency.
3. **Set a `connectionTimeout`** lower than HikariCP's 30-second default to fail fast under pool exhaustion.
4. **Consider deploying behind an API gateway or reverse proxy** (e.g., nginx, Envoy) with per-IP connection/rate limits as a defense-in-depth layer.

## Proof of Concept

```bash
# Exhaust the 10-connection pool with 10 concurrent blocking queries
for i in $(seq 1 10); do
  curl -s -X POST http://<host>:8083/graphql/alpha \
    -H 'Content-Type: application/json' \
    -d '{"query":"{ account(input:{id:\"0.0.1\"}) { balance } }"}' &
done

# All subsequent legitimate requests will time out (up to 30s)
curl -X POST http://<host>:8083/graphql/alpha \
  -H 'Content-Type: application/json' \
  -d '{"query":"{ account(input:{id:\"0.0.2\"}) { balance } }"}'
# Expected: Connection is not available, request timed out after 30000ms
```
