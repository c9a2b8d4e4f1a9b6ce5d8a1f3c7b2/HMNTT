Audit Report

## Title
Missing Length Validation in `getByEvmAddressAndType` Causes Unhandled `BufferUnderflowException`

## Summary
`GraphQlUtils.decodeEvmAddress()` performs no length check on the decoded byte array. `EntityServiceImpl.getByEvmAddressAndType()` unconditionally wraps the result in a `ByteBuffer` and performs three sequential reads totalling 20 bytes. Any hex input between 12 and 19 bytes whose first 12 bytes are all zeros passes the short-circuit `if` condition and causes the second `buffer.getLong()` to throw `java.nio.BufferUnderflowException`. `CustomExceptionResolver` does not handle this exception type, so Spring GraphQL propagates it as a generic `INTERNAL_ERROR` instead of a clean `ValidationError`.

## Finding Description

**`decodeEvmAddress()` — no length check:** [1](#0-0) 

The function strips the `0x` prefix and calls `Hex.decodeHex()`. No check is made that the resulting byte array is exactly 20 bytes.

**`getByEvmAddressAndType()` — unconditional `ByteBuffer` reads:** [2](#0-1) 

- `buffer.getInt()` consumes bytes 0–3 (requires ≥ 4 bytes)
- First `buffer.getLong()` consumes bytes 4–11 (requires ≥ 12 bytes total)
- Second `buffer.getLong()` consumes bytes 12–19 (requires exactly 20 bytes total)

If the input decodes to exactly 12 bytes of all zeros, the first two reads succeed and return `0`, the `if` branch is entered, and the third read throws `BufferUnderflowException` because 0 bytes remain.

**`CustomExceptionResolver` — gap in exception handling:** [3](#0-2) 

`BufferUnderflowException` is not a subtype of `IllegalStateException`, `IllegalArgumentException`, or `MirrorNodeException`. The resolver returns `null`, causing Spring GraphQL to fall back to its default `INTERNAL_ERROR` handler.

**Call path:**

`AccountController.account()` (line 52–54) → `entityService.getByEvmAddressAndType(evmAddress, EntityType.ACCOUNT)` → `decodeEvmAddress(evmAddress)` → `ByteBuffer` reads. [4](#0-3) 

## Impact Explanation
Every request with a crafted short all-zero hex address reliably triggers an unhandled `BufferUnderflowException`. The service thread is not killed (Spring GraphQL catches unresolved exceptions at the framework level), so this does not cause a total outage. The realistic impact is: (a) the caller receives a generic `INTERNAL_ERROR` GraphQL response that may leak the exception class name depending on Spring GraphQL configuration, and (b) an attacker can repeatedly trigger this to generate noise in error logs and degrade observability. Severity is **Medium** — reliable, unauthenticated error injection, but not a crash or data-exposure issue.

## Likelihood Explanation
No authentication is required. The `evmAddress` field is a plain string accepted by the public GraphQL endpoint. The payload is trivial to construct. Any external user can reproduce it deterministically and repeatedly.

## Recommendation
Add an explicit length check in `decodeEvmAddress()` or at the top of `getByEvmAddressAndType()`:

```java
if (evmAddressBytes.length != 20) {
    throw new IllegalArgumentException(
        "evmAddress must decode to exactly 20 bytes, got " + evmAddressBytes.length);
}
```

`IllegalArgumentException` is already handled by `CustomExceptionResolver` and will produce a clean `ValidationError` response. [1](#0-0) 

## Proof of Concept

```graphql
query {
  account(input: { evmAddress: "000000000000000000000000" }) {
    id
  }
}
```

- Input: `"000000000000000000000000"` — 24 hex chars = 12 bytes, all zeros.
- `decodeEvmAddress` returns `byte[12]` with no error.
- `buffer.getInt()` → `0` (condition true).
- First `buffer.getLong()` → `0` (condition true; `if` branch entered).
- Second `buffer.getLong()` → 0 bytes remain → `java.nio.BufferUnderflowException` thrown.
- `CustomExceptionResolver.resolveToSingleError` returns `null`.
- Spring GraphQL returns `INTERNAL_ERROR` to the caller. [2](#0-1)

```java
    public static byte[] decodeEvmAddress(String evmAddress) {
        if (evmAddress == null) {
            return ArrayUtils.EMPTY_BYTE_ARRAY;
        }

        try {
            evmAddress = Strings.CS.removeStart(evmAddress, HEX_PREFIX);
            return Hex.decodeHex(evmAddress);
        } catch (DecoderException e) {
            throw new IllegalArgumentException("Unable to decode evmAddress: " + evmAddress);
        }
    }
```

```java
    public Optional<Entity> getByEvmAddressAndType(String evmAddress, EntityType type) {
        byte[] evmAddressBytes = decodeEvmAddress(evmAddress);
        var buffer = ByteBuffer.wrap(evmAddressBytes);
        if (buffer.getInt() == 0 && buffer.getLong() == 0) {
            return entityRepository.findById(buffer.getLong()).filter(e -> e.getType() == type);
        }
        return entityRepository.findByEvmAddress(evmAddressBytes).filter(e -> e.getType() == type);
    }
```

```java
    protected GraphQLError resolveToSingleError(Throwable ex, DataFetchingEnvironment env) {
        if (ex instanceof IllegalStateException
                || ex instanceof IllegalArgumentException
                || ex instanceof MirrorNodeException) {
            return GraphqlErrorBuilder.newError()
                    .errorType(ErrorType.ValidationError)
                    .message(ex.getMessage())
                    .path(env.getExecutionStepInfo().getPath())
                    .location(env.getField().getSourceLocation())
                    .build();
        } else {
            return null;
        }
    }
```
