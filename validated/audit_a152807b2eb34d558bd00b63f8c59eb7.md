Audit Report

## Title
Mirror Node Response Poisoning Injects Dead gRPC Endpoints, Permanently Stalling All Pinger Transactions

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` accepts any syntactically valid `host:port` from the mirror node REST response without reachability validation. Because `newClient()` in `pinger/sdk_client.go` builds the Hiero SDK client exactly once at startup and `pinger/main.go` reuses it forever, an attacker who controls the mirror node REST response at startup can inject permanently unreachable endpoints, causing every subsequent transaction to time out for the lifetime of the process.

## Finding Description

**`fetchMirrorNodeNetwork()` — no reachability check:**

The function's only content-level guards are an empty-host/zero-port skip and an empty-map rejection. Any syntactically valid but unreachable address (e.g., `0.0.0.0:50211`) passes all guards and is inserted into the network map. [1](#0-0) 

Specifically, the skip condition on line 118 only rejects blank hosts or zero ports — a non-empty, non-zero but permanently unreachable address is never filtered: [2](#0-1) 

**Client built once, never refreshed:**

`newClient()` calls `buildNetworkFromMirrorNodes()` and immediately passes the result to `hiero.ClientForNetwork()`: [3](#0-2) 

`main()` calls `newClient()` once at line 41 and the ticker loop reuses the same `client` pointer on every tick with no rebuild path: [4](#0-3) 

**Liveness probe is independent of transaction success:**

The heartbeat goroutine writes `/tmp/alive` on its own 15-second ticker, completely decoupled from whether `submitWithRetry()` succeeds: [5](#0-4) 

**Attack surface is plain HTTP by default in `network=other` mode:**

The `mirrorRest` flag documentation explicitly uses a plain `http://` example: [6](#0-5) 

## Impact Explanation

The pinger's sole purpose is to periodically execute HBAR transfers to verify network liveness. With a poisoned network map, every `Execute(client)` call blocks until the SDK's gRPC deadline and returns a timeout error. Because the client is never rebuilt, all retries also time out. The pinger cannot execute any transaction for the entire lifetime of the process. Critically, the Kubernetes liveness probe (`/tmp/alive`) continues to be written successfully, so the pod is never restarted — the freeze is self-sustaining and invisible to the orchestrator. The only observable symptom is repeated `transfer failed` log lines.

## Likelihood Explanation

This path is only reachable when `HIERO_MIRROR_PINGER_NETWORK=other`, the custom-network deployment mode. In that mode, the mirror REST URL is operator-configured and the documented example uses plain `http://`. Over plain HTTP, a network-adjacent attacker (same LAN, compromised router, ARP or DNS poisoning) can inject a crafted `nodesEnvelope` JSON response with no cryptographic capability. The attack is:
- **Persistent**: the poisoned client survives until process restart.
- **Stealthy**: the liveness probe keeps passing; only transfer logs reveal failures.
- **Low-skill**: crafting a valid JSON payload with dead endpoints requires no special privileges on the target host.

## Recommendation

1. **Probe endpoints before use**: Before inserting an address into the network map, attempt a short-timeout TCP dial (e.g., `net.DialTimeout("tcp", addr, 2*time.Second)`). Discard any address that does not accept a connection.
2. **Periodically refresh the client**: Add a background goroutine that re-fetches the mirror node network on a configurable interval and calls `client.SetNetwork()` to replace dead endpoints without restarting the process.
3. **Enforce HTTPS for the mirror REST URL**: Reject `http://` schemes in `loadConfig()` when `network=other`, or at minimum warn loudly. TLS prevents trivial MITM injection.
4. **Require a minimum number of reachable endpoints**: If fewer than N endpoints are reachable after probing, treat the response as invalid and retry rather than building a client from a potentially poisoned subset.

## Proof of Concept

1. Stand up a local HTTP server that responds to `GET /api/v1/network/nodes` with:
```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{
      "ip_address_v4": "0.0.0.0",
      "port": 50211
    }]
  }],
  "links": {}
}
```
2. Start the pinger with:
```
HIERO_MIRROR_PINGER_NETWORK=other
HIERO_MIRROR_PINGER_REST=http://<attacker-controlled-server>:8080
```
3. `fetchMirrorNodeNetwork()` accepts `"0.0.0.0"` (non-empty) and `50211` (non-zero), inserts `"0.0.0.0:50211" → AccountID{0,0,3}` into the map, and returns it. [7](#0-6) 
4. `hiero.ClientForNetwork({"0.0.0.0:50211": ...})` creates a client pointing exclusively at a dead endpoint. [8](#0-7) 
5. Every `submitWithRetry(ctx, client, cfg)` call blocks until the gRPC deadline, then logs `transfer failed: ...`. [9](#0-8) 
6. `/tmp/alive` continues to be updated every 15 seconds; the Kubernetes liveness probe never fires. [10](#0-9) 
7. The pinger remains permanently stalled until the process is restarted against a clean mirror node response.
