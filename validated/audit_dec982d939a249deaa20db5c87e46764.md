Audit Report

## Title
Unauthenticated Mirror-Node Response Allows Arbitrary gRPC Endpoint Injection via Unvalidated ServiceEndpoints

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` fetches the consensus-node address list over a plain, unauthenticated HTTP connection and blindly trusts every IP/hostname and port returned in `ServiceEndpoints`. Any party who can influence the HTTP response — via DNS hijacking, MITM on the unencrypted channel, or a misconfigured `mirrorRest` URL — can inject arbitrary gRPC endpoints, causing the Hiero SDK client to route all signed transactions to attacker-controlled infrastructure.

## Finding Description
`fetchMirrorNodeNetwork()` (`pinger/mirror_node_client.go`, lines 74–132) issues a plain HTTP GET with no TLS enforcement, no certificate pinning, and no response authentication:

```go
// lines 79-84
req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
resp, err := httpClient.Do(req)
```

For every `serviceEndpoint` in the decoded JSON body, the function unconditionally builds a network map entry:

```go
// lines 113-124
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId
}
```

The sole guard (`host == "" || ep.Port == 0`) does not reject loopback addresses (`127.0.0.0/8`, `::1`), RFC-1918 private ranges, link-local addresses, or attacker-controlled domain names. The resulting map is passed directly to `hiero.ClientForNetwork(netmap)` in `pinger/sdk_client.go` line 22, which the SDK uses as the exclusive set of gRPC targets. This path is active whenever `cfg.network == "other"`, the configuration mode intended for custom/private deployments where a `mirrorRest` URL is explicitly set.

## Impact Explanation
- **Transaction redirection / denial of write**: All transactions signed with the operator's private key are sent to the attacker's endpoint. No consensus record is ever created. The pinger reports no Go-level error if the attacker's server returns a plausible gRPC response, so the operator has no alarm.
- **Key-material exposure**: The attacker receives fully signed, serialized Hiero transactions including the operator account ID and cryptographic signature, enabling replay or offline analysis.
- **Persistence**: `buildNetworkFromMirrorNodes` is called exactly once at startup (`sdk_client.go` line 18, `main.go` line 41). A single poisoned response persists for the entire process lifetime.

## Likelihood Explanation
- The default `mirrorRest` value is an `http://` URL (`http://mirror-rest:5551`), so TLS is not used in the default deployment.
- In Kubernetes, DNS spoofing within a namespace is feasible without cluster-admin privileges when `NetworkPolicy` is absent, which is common in default installs.
- The attack requires only the ability to serve a valid JSON body at the mirror REST URL — no knowledge of the operator key is needed.
- The SECURITY.md "Lack of SSL/TLS best practices" exclusion applies only to "Websites and Apps"; this is a backend daemon. No other exclusion covers unauthenticated network-map injection.

## Recommendation
1. **Enforce TLS**: Require `https://` for `mirrorRest` in production configurations; reject or warn on plain `http://` at startup.
2. **Validate returned addresses**: After parsing each `host`, resolve it and reject any address that falls within loopback (`127.0.0.0/8`, `::1`), RFC-1918 (`10/8`, `172.16/12`, `192.168/16`), or link-local (`169.254/16`, `fe80::/10`) ranges.
3. **Authenticate the response**: Use mutual TLS or a signed/HMAC-protected payload so that a MITM cannot substitute a crafted response.
4. **Periodic refresh with change detection**: If the network map is refreshed at runtime, log and alert on unexpected changes to the node set.

## Proof of Concept
1. Stand up a local HTTP server that returns:
```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{"ip_address_v4": "127.0.0.1", "port": 50211}]
  }],
  "links": {}
}
```
2. Set `HIERO_MIRROR_PINGER_REST=http://127.0.0.1:<attacker_port>` and `HIERO_MIRROR_PINGER_NETWORK=other`.
3. Run the pinger. `fetchMirrorNodeNetwork` accepts the response (HTTP 200, valid JSON, non-empty host, non-zero port) and returns `{"127.0.0.1:50211": AccountID{0,0,3}}`.
4. `hiero.ClientForNetwork` builds a client whose only known node is `127.0.0.1:50211`.
5. Every `submitWithRetry` tick sends a fully signed transaction to the attacker's listener. No transaction reaches the Hashgraph.
