Audit Report

## Title
Cache Null-Miss for Non-Existent Contract Storage Slots Causes Unbounded Repeated DB Queries (DoS)

## Summary
`ContractStateServiceImpl.findStorage()` never writes a sentinel value to `contractStateCache` for storage slots that do not exist in the database. Every subsequent call for such a slot finds `null` in the cache (indistinguishable from a first-time miss) and unconditionally re-invokes `findStorageBatch()`, which issues a live database query. An unprivileged attacker can exploit the public `/api/v1/contracts/call` endpoint to sustain amplified database load by repeatedly reading uninitialized storage slots.

## Finding Description

In `ContractStateServiceImpl.findStorage()`: [1](#0-0) 

`contractStateCache.get(...)` returns `null` for any slot that was never written. The guard `cachedValue != null && cachedValue != EMPTY_VALUE` fails, so `findStorageBatch()` is called unconditionally â€” on every invocation, for every non-existent slot.

Inside `findStorageBatch()`: [2](#0-1) 

The non-existent slot key is accumulated in the per-contract `contractSlotsCache` via `putIfAbsent`. On the next call, the key is already present (no-op), but the entire accumulated key-set is still collected and passed to the batch query: [3](#0-2) 

Because the DB returns nothing for non-existent slots, the `for` loop body never executes, and no entry is ever written to `contractStateCache` for that key. The next call to `findStorage()` for the same slot again finds `null` and again calls `findStorageBatch()`.

**Root cause:** The code has the infrastructure to handle a null-miss sentinel â€” the check `cachedValue != EMPTY_VALUE` on line 65 implies `EMPTY_VALUE` was intended as a sentinel for "not found" in `contractStateCache`. However, `EMPTY_VALUE` is only ever stored in `contractSlotsCache` (line 90), never in `contractStateCache`. The failed assumption is that a cache miss (`null`) is distinguishable from a "slot confirmed absent" state; it is not.

**Amplification:** Each new non-existent slot key is accumulated in the per-contract slots cache (default `maximumSize=1500`, `expireAfterAccess=5m`): [4](#0-3) 

Once 1500 non-existent keys are accumulated, every future miss for any of those keys triggers a single batch query scanning all 1500 non-existent slots against the database.

## Impact Explanation

Every `eth_call` that reads an uninitialized storage slot bypasses the cache entirely and hits the database. The `contractState` cache has `expireAfterWrite=2s` for real values: [5](#0-4) 

Non-existent slots are never written to `contractStateCache`, so there is no expiry â€” the miss is permanent until the per-contract slots cache entry itself expires (default `expireAfterAccess=5m`). With 1500 accumulated non-existent keys, a single `findStorage()` call for any of those keys issues a batch query scanning all 1500 non-existent slots. This allows sustained, amplified database load from a small number of API requests.

## Likelihood Explanation

The `/api/v1/contracts/call` endpoint requires no authentication. The global rate limiter allows up to 500 RPS: [6](#0-5) 

The gas-per-second throttle: [7](#0-6) 

A single `eth_call` can read many storage slots within the gas budget. An attacker needs only to: (1) identify any deployed contract, (2) craft calldata that reads from uninitialized slots (e.g., slot keys that were never written), and (3) repeat at the allowed rate. No special privileges, tokens, or on-chain transactions are required. The attack is trivially scriptable.

## Recommendation

Store `EMPTY_VALUE` as a sentinel in `contractStateCache` for slots confirmed absent by the database. In `findStorageBatch()`, after the result loop, for any slot key in `cachedSlots` that was not returned by the DB, write `EMPTY_VALUE` into `contractStateCache`:

```java
// After the result loop in findStorageBatch():
for (byte[] slot : cachedSlots) {
    SimpleKey cacheKey = generateCacheKey(contractId, slot);
    if (contractStateCache.get(cacheKey, byte[].class) == null) {
        contractStateCache.put(cacheKey, EMPTY_VALUE);
    }
}
```

Then in `findStorage()`, the existing check `cachedValue != null && cachedValue != EMPTY_VALUE` already correctly short-circuits for the sentinel, returning `Optional.empty()` without hitting the DB: [8](#0-7) 

This makes the `EMPTY_VALUE` sentinel in `contractStateCache` consistent with its already-present handling logic.

## Proof of Concept

1. Deploy or identify any contract on the network (or use any existing contract address).
2. Craft an `eth_call` payload that executes `SLOAD` on slot keys that have never been written (e.g., slots `0xdead0001`, `0xdead0002`, ..., `0xdead05DC` â€” 1500 distinct non-existent slots).
3. Send the first batch of requests to accumulate 1500 non-existent slot keys in the per-contract `contractSlotsCache`.
4. Continue sending requests at up to 500 RPS. Each request for any of those 1500 slots:
   - Misses `contractStateCache` (nothing was ever written for non-existent slots)
   - Calls `findStorageBatch()` â†’ issues a `SELECT slot, value FROM contract_state WHERE contract_id = ? AND slot IN (?, ?, ...[1500 entries])` query
5. Observe sustained, amplified database load: 500 RPS Ă— 1500-slot batch queries, with no cache relief, for the full 5-minute `expireAfterAccess` window of the per-contract slots cache. [9](#0-8)

```java
    private Optional<byte[]> findStorageBatch(final EntityId contractId, final byte[] key) {
        final var contractSlotsCache = ((CaffeineCache) this.contractSlotsCache.get(
                contractId, () -> cacheManagerSlotsPerContract.getCache(contractId.toString())));
        final var wrappedKey = ByteBuffer.wrap(key);
        // Cached slot keys for contract, whose slot values are not present in the contractStateCache
        contractSlotsCache.putIfAbsent(wrappedKey, EMPTY_VALUE);
        final var cachedSlotKeys = contractSlotsCache.getNativeCache().asMap().keySet();

        final var cachedSlots = new ArrayList<byte[]>(cachedSlotKeys.size());
        boolean isKeyEvictedFromCache = true;

        for (var slot : cachedSlotKeys) {
            cachedSlots.add(((ByteBuffer) slot).array());
            if (wrappedKey.equals(slot)) {
                isKeyEvictedFromCache = false;
            }
        }

        final var contractSlotValues = contractStateRepository.findStorageBatch(contractId.getId(), cachedSlots);
        byte[] cachedValue = null;

        for (final var contractSlotValue : contractSlotValues) {
            final byte[] slotKey = contractSlotValue.getSlot();
            final byte[] slotValue = contractSlotValue.getValue();
            contractStateCache.put(generateCacheKey(contractId, slotKey), slotValue);

            if (Arrays.equals(slotKey, key)) {
                cachedValue = slotValue;
            }
        }

        // If the cache key was evicted and hasn't been requested since, the cached value will be null.
        // In that case, fall back to the original query.
        if (isKeyEvictedFromCache) {
            return contractStateRepository.findStorage(contractId.getId(), key);
        }
        return Optional.ofNullable(cachedValue);
    }
```
