Audit Report

## Title
Unauthenticated Mirror-Node Response Allows Arbitrary gRPC Endpoint Injection into Hiero SDK Client

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` fetches consensus-node addresses over plain HTTP and inserts them into the SDK network map without any IP-address validation. An attacker who can intercept or spoof the mirror-node REST response can inject arbitrary addresses as `service_endpoints`, causing `hiero.ClientForNetwork(netmap)` to direct all signed transactions to an attacker-controlled endpoint.

## Finding Description

**Verified code references:**

The HTTP client is constructed with no TLS at `pinger/mirror_node_client.go` line 46: [1](#0-0) 

The endpoint parsing loop at lines 113–124 performs only an emptiness check — no validation against loopback, link-local, or RFC-1918 ranges: [2](#0-1) 

The resulting map is passed verbatim to `hiero.ClientForNetwork` at `pinger/sdk_client.go` line 22: [3](#0-2) 

The documented default REST URL uses plain HTTP (`http://mirror-rest:5551`), confirmed at `pinger/config.go` line 37: [4](#0-3) 

**Root cause:** The only guard before inserting an address into the SDK network map is `host == "" || ep.Port == 0`. There is no check that the resolved host is not a loopback address, a private/internal IP, or any other restricted range. Because the HTTP channel is unencrypted, a network-level attacker can supply any address in the JSON response.

## Impact Explanation

- **Transaction loss:** If nothing listens on the injected address, every `submitWithRetry()` call silently fails with a gRPC connection error. The pinger logs the error but continues running, so operators may not notice immediately. [5](#0-4) 

- **Transaction interception:** If the attacker controls a gRPC listener at the injected address (e.g., a sidecar in the same pod), they receive fully-signed, ready-to-submit `CryptoTransfer` transactions and can replay, delay, or selectively drop them.

- **SSRF pivot:** The injected address need not be loopback; any internal cluster IP (e.g., a cloud metadata service at `169.254.169.254`) can be targeted, turning the pinger into an SSRF vector.

The client is built once at startup, so a single poisoned response permanently redirects all transactions for the lifetime of the process. [6](#0-5) 

## Likelihood Explanation

- Precondition: `network=other` mode, which is the exact use-case this code path serves (private/custom deployments).
- Plain HTTP makes passive interception trivial on any shared L2 segment (cloud VPC, Kubernetes overlay network with a compromised node).
- DNS poisoning of an internal service name (`mirror-rest`) is a well-known intra-cluster attack vector requiring no credentials.
- The attack is repeatable and persistent for the process lifetime.

## Recommendation

1. **Enforce HTTPS for the mirror REST URL.** Validate that `cfg.mirrorRest` uses the `https://` scheme and configure the `http.Client` with a strict TLS config (certificate verification, no `InsecureSkipVerify`). [1](#0-0) 

2. **Validate resolved endpoints before inserting into the network map.** After parsing `host`, resolve it and reject addresses that are loopback (`127.0.0.0/8`), link-local (`169.254.0.0/16`), or RFC-1918 private ranges unless an explicit allowlist is configured. [7](#0-6) 

3. **Reject an empty network map with a fatal error** rather than silently proceeding, so a poisoned response that yields zero valid endpoints causes an immediate startup failure rather than a silent misconfiguration. [8](#0-7) 

## Proof of Concept

1. Deploy pinger with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551`.
2. On the same network segment, intercept the HTTP GET to `/api/v1/network/nodes` (via ARP spoofing, DNS poisoning, or a rogue DNS server for `mirror-rest`).
3. Return the crafted response:
   ```json
   {"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"ip_address_v4":"127.0.0.1","port":50211}]}]}
   ```
4. `fetchMirrorNodeNetwork()` accepts this, builds `network["127.0.0.1:50211"] = AccountID{0,0,3}`, and returns it without error. [9](#0-8) 
5. `hiero.ClientForNetwork(netmap)` configures the SDK to send all gRPC traffic to `127.0.0.1:50211`. [10](#0-9) 
6. Every subsequent `submitWithRetry()` tick sends a fully-signed `CryptoTransfer` to the loopback interface. [11](#0-10)
