Audit Report

## Title
`cryptoTransferTransactionConstructor.Parse()` Silently Ignores Token and NFT Transfers in `TransferTransaction`, Enabling Hidden Operation Injection

## Summary
`ConstructionParse()` delegates to `cryptoTransferTransactionConstructor.Parse()`, which reads only HBAR transfers via `GetHbarTransfers()`. A `TransferTransaction` carrying fungible token or NFT transfers alongside HBAR transfers passes the type check without any content inspection; the non-HBAR transfers are never read and are absent from the returned operations slice. A caller relying on the parse response as a pre-signing verification step receives an incomplete picture of what the transaction will execute on-chain.

## Finding Description

**`ConstructionParse()` entry point** — `rosetta/app/services/construction_service.go` lines 183–214: [1](#0-0) 

The method deserializes any hex-encoded transaction and passes it directly to `c.transactionHandler.Parse()` with no content-level validation.

**Dispatch in `compositeTransactionConstructor.Parse()`** — `rosetta/app/services/construction/composite_transaction_constructor.go` lines 51–64: [2](#0-1) 

Routing is done purely on the Go type name (`reflect.TypeOf(transaction).Name()`). A `TransferTransaction` carrying token transfers is still a `TransferTransaction` and routes to `cryptoTransferTransactionConstructor`.

**Root cause — `cryptoTransferTransactionConstructor.Parse()`** — `rosetta/app/services/construction/crypto_transfer_transaction_constructor.go` lines 54–88: [3](#0-2) 

Line 59 performs a Go type assertion (`transaction.(hiero.TransferTransaction)`) that succeeds for any `TransferTransaction` regardless of its payload. Line 68 calls only `GetHbarTransfers()`. `GetTokenTransfers()` and `GetNftTransfers()` are never called anywhere in the method. The operations slice is sized and populated exclusively from the HBAR transfer map: [4](#0-3) 

**`Construct()` is HBAR-only by design**, confirming the implementation never intended to handle token transfers through this path: [5](#0-4) 

The failed assumption is that every `TransferTransaction` reaching `Parse()` was produced by `Construct()` and therefore contains only HBAR transfers. The endpoint accepts arbitrary externally-supplied hex, so this assumption does not hold.

## Impact Explanation
Any client that uses `POST /construction/parse` as a pre-signing verification step — the canonical Rosetta Construction API pattern — can be presented with a crafted `TransferTransaction` that embeds token or NFT transfers invisible to the parse response. The client sees only the HBAR operation, believes the transaction is a trivial HBAR transfer, signs it, and submits it. The network executes the full transaction including the hidden token/NFT transfers; the mirror node records the complete on-chain state. For exchanges or wallets using this Rosetta implementation, this enables silent asset theft with a single crafted transaction.

## Likelihood Explanation
No privilege is required. Constructing a valid serialized `TransferTransaction` with both HBAR and token transfers is trivial using the public Hiero SDK (the monitor test suite already does exactly this). The victim must be using `/construction/parse` for pre-signing verification, which is the intended use of the endpoint. The attack is fully repeatable and requires no special on-chain state or account permissions.

## Recommendation
Inside `cryptoTransferTransactionConstructor.Parse()`, after the type assertion, add explicit checks that `GetTokenTransfers()` and `GetNftTransfers()` both return empty maps. If either is non-empty, return `errors.ErrInvalidTransaction` (or a dedicated error). This ensures the endpoint rejects any `TransferTransaction` that carries non-HBAR transfers, eliminating the discrepancy between what `Parse()` reports and what the network will execute.

```go
if len(transferTransaction.GetTokenTransfers()) != 0 ||
    len(transferTransaction.GetNftTransfers()) != 0 {
    return nil, nil, errors.ErrInvalidTransaction
}
```

## Proof of Concept

Using the Hiero Go SDK:

```go
tx := hiero.NewTransferTransaction().
    // visible HBAR transfer — Parse() will report this
    AddHbarTransfer(victimAccountId, hiero.HbarFromTinybar(-1)).
    AddHbarTransfer(attackerAccountId, hiero.HbarFromTinybar(1)).
    // hidden token transfer — Parse() silently ignores this
    AddTokenTransfer(tokenId, victimAccountId, -1_000_000).
    AddTokenTransfer(tokenId, attackerAccountId, 1_000_000).
    SetTransactionID(hiero.TransactionIDGenerate(victimAccountId))

bytes, _ := hiero.TransactionToBytes(tx)
hexTx := hex.EncodeToString(bytes)
```

1. POST `hexTx` to `POST /construction/parse` → response contains only the −1 tinybar HBAR operation.
2. Victim inspects the response, sees a trivial HBAR transfer, signs the transaction bytes.
3. POST signed transaction to `POST /construction/submit`.
4. Network executes both the HBAR transfer and the 1,000,000-token transfer; mirror node records both.
