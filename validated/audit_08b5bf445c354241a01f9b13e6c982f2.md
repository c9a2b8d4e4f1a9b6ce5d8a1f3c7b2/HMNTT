Audit Report

## Title
Gas Throttle Amplification via `estimate=true` Binary Search — Up to 21× EVM Executions Per Single Rate-Limit Token

## Summary
`ContractController.call()` invokes `throttleManager.throttle(request)` exactly once per HTTP request, consuming one `rateLimitBucket` token and `scaleGas(request.getGas())` `gasLimitBucket` tokens. When `estimate=true`, `ContractExecutionService.estimateGas()` performs one initial EVM execution followed by up to `maxGasEstimateRetriesCount` (default: 20) additional EVM executions inside `BinaryGasEstimator.search()`. None of the binary search iterations interact with the throttle, creating a deterministic amplification factor of up to 21× per rate-limit token consumed.

## Finding Description

**Code path:**

`ContractController.call()` calls `throttleManager.throttle(request)` once at entry: [1](#0-0) 

`ThrottleManagerImpl.throttle()` deducts 1 `rateLimitBucket` token and `scaleGas(request.getGas())` `gasLimitBucket` tokens — no further deductions occur during processing: [2](#0-1) 

`ContractExecutionService.estimateGas()` first calls `callContract()` (which internally calls `doProcessCall(..., false)`), then passes `gas -> doProcessCall(params, gas, true)` to `binaryGasEstimator.search()`: [3](#0-2) 

`BinaryGasEstimator.search()` loops up to `maxGasEstimateRetriesCount` times, calling `safeCall(mid, call)` (i.e., `doProcessCall`) on every iteration with no throttle interaction: [4](#0-3) 

`maxGasEstimateRetriesCount` defaults to 20: [5](#0-4) 

The critical gate: `doProcessCall` with `estimate=true` explicitly skips `restoreGasToBucket` and all throttle interaction. The `if (!estimate)` guard means the binary search loop is entirely invisible to the throttle: [6](#0-5) 

**Root cause:** The throttle gate is a one-time check at the HTTP boundary. The `gasLimitBucket` is charged for `request.getGas()` once, but the binary search can execute up to `maxGasEstimateRetriesCount` full EVM simulations, each consuming CPU proportional to the gas value passed to that iteration. The failed assumption is that one HTTP request equals one EVM execution for throttle accounting purposes.

## Impact Explanation

With `maxGasEstimateRetriesCount = 20` and `gas = 15,000,000` (the default maximum, enforced by `evmProperties.getMaxGasLimit()`), one HTTP request triggers up to 21 EVM executions (1 initial + up to 20 binary search iterations) while consuming only 1 `rateLimitBucket` token and `scaleGas(15_000_000) = 1,500` `gasLimitBucket` tokens. [7](#0-6) 

The default `requestsPerSecond = 500` and `gasPerSecond = 7,500,000,000` (scaled: 750,000 tokens/s) allow 500 req/s at max gas. With the amplification, an attacker sustains up to ~10,500 EVM executions/second against the server's effective throttle of 500. This enables CPU saturation and thread pool starvation with a modest request rate. [8](#0-7) 

## Likelihood Explanation

No authentication or special privileges are required. Any external caller can POST to `/api/v1/contracts/call` with `estimate: true` and `gas: 15000000`. The attack requires no on-chain state, is trivially repeatable from a single HTTP client, and the amplification factor is deterministic and bounded only by `maxGasEstimateRetriesCount`. The endpoint is publicly accessible per the controller mapping: [9](#0-8) 

## Recommendation

1. **Charge throttle tokens per binary search iteration:** Inside `BinaryGasEstimator.search()`, call `throttleManager.throttleOpcodeRequest()` (or a dedicated estimate-iteration bucket) on each iteration, so the throttle reflects actual EVM invocations.
2. **Pre-charge for the full estimate cost:** At the HTTP boundary, when `request.isEstimate() == true`, multiply the gas token deduction by `maxGasEstimateRetriesCount` (or a configurable factor), and restore unused tokens after completion.
3. **Reduce the default `maxGasEstimateRetriesCount`:** Lowering this value reduces the amplification ceiling, though it does not eliminate the root cause.

The cleanest fix is option 1 or 2, ensuring the `gasLimitBucket` accurately reflects the number of EVM executions triggered per request.
