Audit Report

## Title
Reference-Equality Bypass on `BlockType.EARLIEST` Causes `findByIndex(0)` Instead of `findEarliest()` for Block Parameter `"0"`

## Summary
`RecordFileServiceImpl.findByBlockType()` guards the earliest-block path with a Java reference-equality check (`block == BlockType.EARLIEST`). When a caller supplies the decimal string `"0"` as the block parameter, `BlockType.of("0")` constructs a **new** `BlockType("0", 0L)` instance that is a distinct heap object from the `EARLIEST` singleton. The guard is never taken, execution falls through to `recordFileRepository.findByIndex(0)`, and on any mirror node whose earliest stored block has an index other than 0, the two queries return different (or no) results.

## Finding Description

**Root cause — reference equality on a value type**

`BlockType` is a Java `record`: [1](#0-0) 

The `EARLIEST` singleton is allocated once at class-load time: [2](#0-1) 

When a caller passes `"0"`, `BlockType.of()` matches the decimal branch and allocates a **fresh** object: [3](#0-2) 

This produces `new BlockType("0", 0L)` — same field values as `EARLIEST` (`name="earliest"`, `number=0L` vs `name="0"`, `number=0L`) but a distinct heap object. The guard in `findByBlockType()` uses `==`: [4](#0-3) 

`block == BlockType.EARLIEST` evaluates to `false`, so execution falls through to `recordFileRepository.findByIndex(block.number())` with `number() == 0`.

**Divergence between the two queries**

`findEarliest()` returns the record file with the **lowest** index, regardless of what that index value is: [5](#0-4) 

`findByIndex(0)` performs an exact match on `index = 0`: [6](#0-5) 

The importer exposes `updateIndex(long offset)` which shifts every row's index by an arbitrary offset: [7](#0-6) 

After any such rebase, or on a partial mirror that starts ingestion from a non-zero block height, no row with `index = 0` exists. `findByIndex(0)` returns `Optional.empty()` while `findEarliest()` would return the actual genesis record.

**Note on `toString()`:** The same reference-equality pattern appears in `BlockType.toString()` at line 93, confirming this is a systemic assumption throughout the class that `"0"` and `"earliest"` are treated as distinct inputs. [8](#0-7) 

## Impact Explanation
Any `eth_call` or historical simulation that relies on the `"earliest"` / `"0"` block equivalence is broken. On partial or reindexed mirror nodes — a common production topology — the call either returns an error instead of the genesis state, or silently executes against a wrong block. Smart-contract logic that depends on genesis-state values (initial balances, constructor storage) will produce incorrect results. Severity is **Medium**: no direct fund loss, but incorrect historical EVM execution is delivered to callers who expect canonical earliest-block semantics.

## Likelihood Explanation
The trigger requires zero privileges — any JSON-RPC client can pass `"blockNumber": "0"` or `"blockNumber": "0x0"`. The condition (mirror node whose earliest stored index ≠ 0) is common in production deployments that start from a checkpoint or have undergone index rebase via `updateIndex()`. The bug is deterministic and trivially repeatable.

## Recommendation

**Option A — Normalize in `BlockType.of()`:** In the decimal branch, return the `EARLIEST` singleton when the parsed value is `0`:
```java
// In BlockType.of(), decimal branch:
long n = Long.parseLong(decimal, 10);
if (n == 0L) return EARLIEST;
return new BlockType(value, n);
```

**Option B — Fix the guard in `findByBlockType()`:** Replace reference equality with a value-based check:
```java
if (block == BlockType.EARLIEST || (!block.isHash() && block.number() == 0L)) {
    return recordFileRepository.findEarliest();
}
```

Option A is preferred because it fixes the root cause at the factory level and keeps `findByBlockType()` clean. It also fixes the same latent issue in `toString()`.

## Proof of Concept

```java
// Demonstrates the bypass
BlockType byTag    = BlockType.of("earliest");  // returns EARLIEST singleton
BlockType byNumber = BlockType.of("0");          // returns new BlockType("0", 0L)

assert byTag    == BlockType.EARLIEST;  // true  — takes findEarliest() path
assert byNumber == BlockType.EARLIEST;  // FALSE — falls through to findByIndex(0)
assert byTag.number() == byNumber.number();  // true — same numeric value, different object
```
