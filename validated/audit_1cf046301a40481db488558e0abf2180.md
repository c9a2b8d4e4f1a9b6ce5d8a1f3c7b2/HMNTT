# Audit Report

## Title
Silent Long Overflow in `NumberRangeParameter.getInclusiveValue()` Causes `node.id=gt:Long.MAX_VALUE` to Return All Nodes Instead of Empty Set

## Summary
When a caller supplies `node.id=gt:9223372036854775807` to `/api/v1/network/nodes`, `NumberRangeParameter.getInclusiveValue()` computes `Long.MAX_VALUE + 1` in plain Java arithmetic, silently wrapping to `Long.MIN_VALUE`. The subsequent `Math.max(0L, Long.MIN_VALUE)` call in `NetworkServiceImpl.getNetworkNodes()` resolves to `0L`, and the range-inversion guard `lowerBound > upperBound` (`0 > Long.MAX_VALUE`) evaluates to `false`. The query executes with `lowerBound=0` and `upperBound=Long.MAX_VALUE`, returning all network nodes instead of an empty result set.

## Finding Description

**Root cause:** `NumberRangeParameter.getNumberValue()` accepts `Long.MAX_VALUE` because `Long.parseLong("9223372036854775807")` succeeds and the only guard is `value < 0`. [1](#0-0) 

`getInclusiveValue()` performs unchecked addition for the `GT` case:
```java
if (operator == RangeOperator.GT) {
    return value + 1;   // Long.MAX_VALUE + 1 = Long.MIN_VALUE (silent wrap)
}
``` [2](#0-1) 

In `NetworkServiceImpl.getNetworkNodes()`:
- Line 114: `Math.max(0L, Long.MIN_VALUE)` = `0L` — overflow is silently discarded.
- Line 120: `0L > Long.MAX_VALUE` = `false` — the range-inversion guard does not fire.
- Lines 135–136: `findNetworkNodes(fileId, [], 0L, Long.MAX_VALUE, ...)` — full table scan, all nodes returned. [8](#0-7) 

The same unchecked `lowerBound++` pattern also exists in `Bound.getAdjustedLowerRangeValue()`, affecting other endpoints that use the `Bound` abstraction. [9](#0-8) 

**Failed assumption:** The code assumes that any value accepted by `getNumberValue()` can safely have `+1` applied without overflow. This assumption is violated at `Long.MAX_VALUE`.

## Impact Explanation
The endpoint returns network topology data (node IDs, service endpoints, public keys). While this data is not secret, the filter contract is broken: a request semantically meaning "nodes with ID > Long.MAX_VALUE" (an empty set) instead returns every node in the address book. Any rate-limiting, pagination, or access-control logic that relies on the filter being honored is bypassed. **Severity: Low** — no authentication bypass or sensitive data exposure, but the input validation guarantee is violated and the API contract is incorrect.

## Likelihood Explanation
Exploitation requires only an unauthenticated HTTP GET request with a crafted query parameter. No special knowledge beyond the public API schema is needed. The value `9223372036854775807` is exactly 19 digits and matches the documented `\d{1,19}` pattern for `node.id`, so it passes any regex-level validation as well. [5](#0-4) 

## Recommendation

1. **In `NumberRangeParameter.getNumberValue()`**, add an explicit upper-bound check:
   ```java
   if (value < 0 || value == Long.MAX_VALUE) {
       throw new IllegalArgumentException("Invalid range value");
   }
   ```
   This prevents `Long.MAX_VALUE` from being stored as a `GT` operand.

2. **In `getInclusiveValue()`**, use `Math.addExact` to throw `ArithmeticException` on overflow:
   ```java
   if (operator == RangeOperator.GT) {
       return Math.addExact(value, 1);
   }
   ```

3. Apply the same fix to `Bound.getAdjustedLowerRangeValue()` and `Bound.getInclusiveRangeValue()`, which contain identical unchecked `lowerBound++` / `value += 1L` patterns. [9](#0-8) [10](#0-9) 

## Proof of Concept

```
GET /api/v1/network/nodes?node.id=gt:9223372036854775807
```

**Expected behavior:** Empty result set (`nodes: []`), because no node can have an ID greater than `Long.MAX_VALUE`.

**Actual behavior:** All nodes in the address book are returned, equivalent to calling `/api/v1/network/nodes` with no filter.

**Trace:**
1. `NumberRangeParameter.valueOf("gt:9223372036854775807")` → `new NumberRangeParameter(GT, 9223372036854775807L)` ✓ (no error)
2. `getInclusiveValue()` → `9223372036854775807L + 1` → `-9223372036854775808L` (`Long.MIN_VALUE`)
3. `Math.max(0L, -9223372036854775808L)` → `0L`
4. `0L > Long.MAX_VALUE` → `false` (guard skipped)
5. `findNetworkNodes(fileId, [], 0L, 9223372036854775807L, "ASC", limit)` → returns all nodes
