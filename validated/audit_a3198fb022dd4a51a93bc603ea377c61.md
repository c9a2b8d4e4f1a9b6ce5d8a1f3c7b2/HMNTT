## Title
Directory Substitution Attack on `/tmp/alive` Bypasses Liveness Probe Indefinitely

## Summary
An attacker with write access to `/tmp` inside the pinger container can create `/tmp/alive` as a directory. Because `os.WriteFile` silently discards its error and `os.Stat` succeeds on directories without any `IsRegular()` guard, the liveness probe permanently reports the container as alive even when the heartbeat mechanism is completely broken.

## Finding Description
**Heartbeat writer** — `pinger/main.go`, line 36:
```go
_ = os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644)
```
The return value of `os.WriteFile` is unconditionally discarded. On Linux, writing to a path that is a directory returns `EISDIR`; this error is silently swallowed every 15 seconds, so the heartbeat timestamp is never updated.

**Liveness checker** — `pinger/cmd/healthcheck/main.go`, lines 27–39:
```go
fi, err := os.Stat("/tmp/alive")
if err != nil { ... os.Exit(1) }
age := time.Since(fi.ModTime())
if age > 2*time.Minute { ... os.Exit(1) }
os.Exit(0)
```
`os.Stat` succeeds on a directory exactly as it does on a regular file. There is no `fi.Mode().IsRegular()` check. The only liveness guard is `ModTime()` freshness, which an attacker fully controls on a directory by creating and removing entries inside it (`touch /tmp/alive/x && rm /tmp/alive/x` updates directory mtime).

**Root cause:** Both the writer and the checker assume `/tmp/alive` is a regular file. Neither enforces this invariant.

## Impact Explanation
The Kubernetes liveness exec probe calls `healthcheck live`. If it exits 0, the pod is considered alive and is never restarted. With this attack, the probe exits 0 indefinitely even though the pinger's heartbeat mechanism is completely non-functional. The pod is stuck in a permanently "alive but non-functional" state, defeating the entire purpose of the liveness probe and preventing automatic recovery. Severity: Medium (availability impact).

## Likelihood Explanation
**Precondition:** any process that can write to `/tmp` inside the container — e.g., a compromised sidecar sharing a volume mount, an init container, or a partial container escape. `/tmp` is world-writable by default on Linux. The attack is a single `mkdir /tmp/alive` executed before or during pinger startup (after removing the existing file). Refreshing the mtime requires only `touch /tmp/alive/x && rm /tmp/alive/x`, which is trivially scriptable and fully repeatable.

## Recommendation
1. **Enforce file type in the liveness checker** — add an `IsRegular()` check after `os.Stat`:
   ```go
   if !fi.Mode().IsRegular() {
       fmt.Fprintln(os.Stderr, "heartbeat path is not a regular file")
       os.Exit(1)
   }
   ```
2. **Log heartbeat write errors** — replace `_ = os.WriteFile(...)` with explicit error handling so failures are visible in container logs.
3. **Use `O_WRONLY|O_CREATE|O_TRUNC` with an explicit `open` + type check** — open the file with `os.OpenFile` and verify the resulting `FileInfo` is a regular file before writing, making the writer itself resilient to directory substitution.

## Proof of Concept
```bash
# Step 1: Remove the legitimate file (if it exists) and substitute a directory
rm -f /tmp/alive
mkdir /tmp/alive

# Step 2: The pinger heartbeat goroutine now silently fails every 15 s (EISDIR)

# Step 3: Keep the directory mtime fresh to pass the 2-minute staleness check
while true; do
    touch /tmp/alive/x && rm /tmp/alive/x
    sleep 60
done &

# Step 4: Verify the liveness probe exits 0 indefinitely
healthcheck live; echo "exit: $?"   # prints "exit: 0" forever
```
The pod is never restarted by Kubernetes despite the pinger being functionally dead.
