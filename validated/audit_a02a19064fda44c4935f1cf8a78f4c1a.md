Audit Report

## Title
Incorrect `unless` Condition in `@Cacheable` Causes Indefinite Negative-Result Cache Poisoning in `findRuntimeBytecode()`

## Summary
`ContractRepository.findRuntimeBytecode()` returns `Optional<byte[]>` but its `@Cacheable` annotation uses `unless = "#result == null"`. Because `Optional.empty()` is never `null`, the guard is always `false`, and every result — including "contract not found" — is unconditionally written to the Caffeine cache. An unauthenticated attacker can pre-query any contract ID before the mirror node indexes it, permanently poisoning the cache entry so that the contract appears to have no bytecode even after it is fully indexed.

## Finding Description
**Exact location:** `web3/src/main/java/org/hiero/mirror/web3/repository/ContractRepository.java`, lines 16–18.

```java
@Cacheable(cacheNames = CACHE_NAME_CONTRACT, cacheManager = CACHE_MANAGER_CONTRACT, unless = "#result == null")
@Query(value = "select runtime_bytecode from contract where id = :contractId", nativeQuery = true)
Optional<byte[]> findRuntimeBytecode(final Long contractId);
``` [1](#0-0) 

Spring's `@Cacheable` evaluates the `unless` SpEL expression against the actual return value. Because the method signature guarantees the return value is always an `Optional` object, `#result == null` is always `false`. Therefore every result — including `Optional.empty()` representing "no contract found" — is stored in the cache.

The cache is configured with `expireAfterAccess=1h,maximumSize=1000`: [2](#0-1) 

`expireAfterAccess` resets the TTL on every read, so a poisoned entry survives indefinitely as long as it is periodically accessed.

The cache manager is wired in `EvmConfiguration`: [3](#0-2) 

The cached result is consumed in `ContractBytecodeReadableKVState.readFromDataSource()`, which maps `Optional.empty()` to `null` via `.orElse(null)`, causing the EVM to see no bytecode for the contract: [4](#0-3) 

The existing test `findRuntimeBytecodeFailCall` exercises the empty-Optional path but does not assert that the result was not cached, leaving the bug undetected: [5](#0-4) 

## Impact Explanation
Any contract that an attacker queries before the mirror node indexes it will appear to have no bytecode for up to one hour (or indefinitely with periodic re-queries). `eth_getCode` returns `0x`; `eth_call` and `eth_estimateGas` behave as if the target is an EOA, causing all calls to revert or return empty data. This breaks DeFi integrations, wallets, and tooling that rely on the mirror node's web3 API for contract interaction. The attacker can also fill all 1,000 cache slots with poisoned entries, evicting legitimate bytecode and degrading service for all contracts.

## Likelihood Explanation
No authentication or rate-limiting is required. The attacker only needs to know (or enumerate) a contract's numeric entity ID before the importer finishes indexing it. Hedera entity IDs are sequential and publicly observable on-chain, making enumeration trivial. The attack is repeatable, low-cost, and requires only standard HTTP requests to the public web3 endpoint.

## Recommendation
Change the `unless` condition to also exclude empty Optionals:

```java
@Cacheable(
    cacheNames = CACHE_NAME_CONTRACT,
    cacheManager = CACHE_MANAGER_CONTRACT,
    unless = "#result == null || !#result.isPresent()"
)
```

This mirrors the correct pattern already used in other repositories in the codebase. Additionally, add a test that verifies an empty-Optional result is not served from cache on a subsequent call after the underlying data has been inserted.

## Proof of Concept
1. Identify a Hedera contract entity ID (e.g., `12345`) that has been created on-chain but not yet indexed by the mirror node importer.
2. Send an unauthenticated JSON-RPC request: `{"jsonrpc":"2.0","method":"eth_getCode","params":["<long-zero-address-of-12345>","latest"],"id":1}`.
3. The DB returns no row → Spring Data JPA returns `Optional.empty()` → `Optional.empty() != null` → `unless` is `false` → `Optional.empty()` is written to the Caffeine cache under key `12345`.
4. Wait for the importer to index the contract (DB now has `runtime_bytecode`).
5. Repeat the same `eth_getCode` request. The cache is hit and returns `Optional.empty()` → `readFromDataSource()` returns `null` → response is `0x`.
6. To make the poisoning indefinite, repeat step 5 at intervals shorter than 1 hour to reset the `expireAfterAccess` TTL.
