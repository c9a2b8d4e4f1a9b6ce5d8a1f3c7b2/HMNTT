# Audit Report

## Title
Unbounded JSON Body Decode in `fetchMirrorNodeNetwork` Enables OOM-Based DoS

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` decodes `resp.Body` directly into a Go struct with no byte-size limit. A compromised or attacker-controlled mirror node endpoint can stream an arbitrarily large JSON body, exhausting heap memory and crashing the pinger process. Because this function is called at startup and on every network refresh, the crash is repeatable on every restart, rendering the pinger permanently unable to submit liveness transactions until the endpoint is restored and the process is manually restarted.

## Finding Description
**File:** `pinger/mirror_node_client.go`
**Function:** `fetchMirrorNodeNetwork()`
**Line:** 96

```go
var payload nodesEnvelope
if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {   // line 96
    return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
}
```

`resp.Body` is passed directly to `json.NewDecoder` with no byte-count guard. The only protection present is the `http.Client` wall-clock timeout at line 46:

```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
```

This is a **time limit, not a byte limit**. On a 100 Mbps or faster connection, a malicious server can deliver multiple gigabytes of syntactically valid JSON well within a typical 30-second timeout window. There is no:
- `Content-Length` pre-check
- `http.MaxBytesReader` wrapping `resp.Body`
- `io.LimitReader` wrapping `resp.Body`

The status-code check at lines 90–93 only rejects non-2xx responses; a crafted 200 response bypasses it entirely.

## Impact Explanation
The pinger's sole function is to submit periodic liveness transactions to the Hedera network. `buildNetworkFromMirrorNodes` (which calls `fetchMirrorNodeNetwork`) is invoked at startup and on every network refresh cycle. A successful OOM crash halts all transaction submission. If the pinger is not auto-restarted, this is a **complete, persistent denial-of-service** of the pinger component. Even with auto-restart, the attacker can crash it on every boot since each restart re-invokes `fetchMirrorNodeNetwork`. This directly impairs the ability to detect and respond to network outages, which is the pinger's core purpose.

## Likelihood Explanation
The mirror node REST URL is a single operator-configured string (`cfg.mirrorRest`). An attacker who can compromise that one HTTP endpoint — via server exploit, DNS poisoning, BGP hijacking, or MITM on a plaintext HTTP deployment — can trigger this with a single crafted HTTP response. No authentication or cryptographic verification of the mirror node response is performed. The attack is repeatable: every restart re-calls `fetchMirrorNodeNetwork`, so the attacker can crash the pinger on every boot cycle.

## Recommendation
Wrap `resp.Body` with a byte-count limit before passing it to the JSON decoder. Choose a limit that comfortably exceeds any legitimate response (e.g., 10 MB):

```go
const maxBodyBytes = 10 * 1024 * 1024 // 10 MB
limited := http.MaxBytesReader(nil, resp.Body, maxBodyBytes)
if err := json.NewDecoder(limited).Decode(&payload); err != nil {
    return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
}
```

Alternatively, use `io.LimitReader`:

```go
limited := io.LimitReader(resp.Body, maxBodyBytes)
if err := json.NewDecoder(limited).Decode(&payload); err != nil {
    return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
}
```

Optionally, add a `Content-Length` pre-check before reading the body to reject oversized responses early.

## Proof of Concept
1. Stand up an HTTP server that responds to any GET request with HTTP 200 and a JSON body of the form:
   ```json
   {"nodes": [{"node_account_id": "0.0.3", "service_endpoints": [{"domain_name": "AAAA...A (repeated millions of times)", "port": 50211}]}, ... (millions of entries)]}
   ```
2. Configure the pinger's `mirrorRest` to point to this server.
3. Start the pinger. `buildNetworkFromMirrorNodes` calls `fetchMirrorNodeNetwork`, which calls `json.NewDecoder(resp.Body).Decode(&payload)`.
4. The Go runtime allocates heap memory proportional to the response size. With a sufficiently large body (e.g., 2–4 GB of valid JSON), the OS OOM killer terminates the pinger process.
5. Restart the pinger — it crashes again on the next `fetchMirrorNodeNetwork` call, confirming the attack is repeatable.

func fetchMirrorNodeNetwork(
	ctx context.Context,
	httpClient *http.Client,
	url string,
) (map[string]hiero.AccountID, bool, error) {
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		return nil, false, err
	}

	resp, err := httpClient.Do(req)
	if err != nil {
		return nil, true, fmt.Errorf("GET %s failed: %w", url, err)
	}
	defer resp.Body.Close()

	if resp.StatusCode < 200 || resp.StatusCode >= 300 {
		retry := resp.StatusCode == http.StatusTooManyRequests || resp.StatusCode >= 500
		return nil, retry, fmt.Errorf("GET %s returned %s", url, resp.Status)
	}

	var payload nodesEnvelope
	if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
		return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
	}

	network := make(map[string]hiero.AccountID)

	for _, n := range payload.Nodes {
		if n.NodeAccountID == "" {
			continue
		}

		nodeAccountId, err := hiero.AccountIDFromString(n.NodeAccountID)
		if err != nil {
			continue
		}

		// Use service_endpoints for node gRPC (what the SDK wants)
		for _, ep := range n.ServiceEndpoints {
			host := strings.TrimSpace(ep.DomainName)
			if host == "" {
				host = strings.TrimSpace(ep.IPAddressV4)
			}
			if host == "" || ep.Port == 0 {
				continue
			}

			addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
			network[addr] = nodeAccountId
		}
	}

	if len(network) == 0 {
		return nil, false, fmt.Errorf("no usable service_endpoints found from %s", url)
	}

	return network, false, nil
}
