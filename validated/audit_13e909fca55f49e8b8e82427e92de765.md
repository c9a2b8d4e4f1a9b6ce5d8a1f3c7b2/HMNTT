# Audit Report

## Title
Binary Search Gas Estimation Multiplier Bypasses Throttle: Up to 21× EVM Execution Per Single Throttle-Deducted Request

## Summary
`ThrottleManagerImpl.throttle()` deducts gas tokens from the bucket exactly once per request, based on `request.getGas()`. When `estimate=true`, `ContractExecutionService.estimateGas()` runs one initial EVM execution followed by up to 20 additional binary-search iterations via `BinaryGasEstimator.search()`. Each binary-search iteration calls `doProcessCall(params, gas, true)`, which skips `restoreGasToBucket` and never deducts additional tokens from the throttle bucket. A single unauthenticated request therefore causes up to 21 full EVM executions while consuming only one request's worth of throttle tokens.

## Finding Description

**Entry point** — `ContractController.call()` calls `throttleManager.throttle(request)` exactly once, consuming `scaleGas(request.getGas())` tokens from the gas bucket before any processing occurs. [1](#0-0) 

`ThrottleManagerImpl.throttle()` deducts `scaleGas(request.getGas())` from the `gasLimitBucket`. `scaleGas` divides by `GAS_SCALE_FACTOR = 10_000`, so `gas=15,000,000` → 1,500 tokens consumed. [2](#0-1) [3](#0-2) 

**Estimate path** — `ContractExecutionService.estimateGas()` performs two steps:

- Step 1: `callContract(params, context)` → `doProcessCall(params, params.getGas(), false)` — `estimate=false`, so `restoreGasToBucket` runs and returns some tokens to the bucket.
- Step 2: `binaryGasEstimator.search(gas -> doProcessCall(params, gas, true), ...)` — up to `maxGasEstimateRetriesCount = 20` iterations, each a full EVM execution. [4](#0-3) 

**Root cause** — `doProcessCall` with `estimate=true` skips the `restoreGasToBucket` call entirely. No additional tokens are deducted from the throttle bucket for any binary-search iteration, and no tokens are restored either. [5](#0-4) 

`maxGasEstimateRetriesCount` defaults to 20, and `BinaryGasEstimator.search()` loops up to this limit. [6](#0-5) [8](#0-7) 

**Failed assumption**: The throttle assumes one gas deduction ≈ one EVM execution. For estimate requests, the ratio is 1 deduction : up to 21 executions.

## Impact Explanation

With `gas=15,000,000` (the default maximum) and bytecode that consumes ~14M gas per execution:

| Metric | Value |
|---|---|
| Throttle tokens deducted | 1,500 (net ~500 after initial-call restore) |
| Actual EVM executions | up to 21 (1 initial + 20 binary search) |
| Actual gas-equivalent work | up to ~294,000,000 |
| Throttle-equivalent work | ~15,000,000 |
| Multiplier | **up to ~21×** |

A single request causes up to 21× the EVM computation that the throttle accounts for. Repeated at the rate limit (500 req/s default), this can saturate node CPU and memory, causing denial of service for all other users.

## Likelihood Explanation

- **No authentication required**: any external user can POST to `/api/v1/contracts/call`
- **No special knowledge needed**: setting `estimate=true` and providing any gas-heavy bytecode (e.g., a loop-heavy constructor) is sufficient
- **Repeatable**: the rate-limit bucket (500 req/s) still allows 500 such amplified requests per second
- **Bytecode is not validated** for computational complexity before the binary search begins

## Recommendation

1. **Account for binary-search iterations in the throttle**: Before entering `BinaryGasEstimator.search()`, pre-deduct additional gas tokens proportional to `maxGasEstimateRetriesCount` (or a configurable estimate multiplier). Restore any unused portion after the search completes.
2. **Alternatively, reduce `maxGasEstimateRetriesCount`** for unauthenticated callers, or apply a separate, stricter rate limit for `estimate=true` requests.
3. **Add a dedicated throttle check inside `BinaryGasEstimator.search()`** per iteration, so each EVM execution consumes tokens proportional to the gas value used in that iteration.
4. **Consider a configurable cap on the total gas budget** for the entire estimate operation (initial call + all search iterations combined), enforced against the throttle bucket.
