Audit Report

## Title
Unbounded Prometheus Label Cardinality via User-Controlled Path in `toOpenApiPath()`

## Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to the raw `req.path` value for any request that does not match a registered Express route. Because `req.path` is fully attacker-controlled, an unauthenticated user can inject an unbounded number of unique `path` label values into the four per-route OpenTelemetry instruments, causing label cardinality explosion that exhausts Node.js heap memory and degrades or destroys the monitoring plane.

## Finding Description
**Exact code path:**

`rest/middleware/metricsHandler.js`, `toOpenApiPath()`, lines 13–31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // only set for sub-router-matched routes

  if (!path) {
    if (!req.route) {
      path = req.path;                        // ← raw, user-supplied path
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }

  path = path.replace(/:([^/]+)/g, '{$1}');

  if (!path.startsWith(apiPrefix)) {
    return apiPrefix + '/' + path;            // ← concatenation, not normalization
  }
  return path;
};
``` [1](#0-0) 

`res.locals[requestPathLabel]` (`'requestPath'`) is only written by `recordRequestPath` in `rest/routes/index.js` for sub-router-matched requests:

```js
const recordRequestPath = async (req, res) => {
  const path = req.route?.path;
  if (path && !path.startsWith(apiPrefix) && !res.locals[requestPathLabel]) {
    res.locals[requestPathLabel] = `${req.baseUrl}${req.route.path}`.replace(/\/+$/g, '');
  }
};
``` [4](#0-3) 

For any request that does not match a registered route (every 404), both `res.locals[requestPathLabel]` and `req.route` are `null`, so `path = req.path` — the raw URL path — is used verbatim as the `path` label in all four per-route instruments:

```js
const labels = {method, path, code};
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
``` [5](#0-4) 

**Root cause / failed assumption:** The code assumes only a bounded set of known route patterns will ever reach the `!req.route` branch. In reality, every 404 response passes through this branch with an arbitrary attacker-supplied path.

**Doubled-prefix sub-case:** A request to `GET /foo` (not starting with `/api/v1`) produces the label `/api/v1//foo` via the concatenation at line 27. A request to `GET /foo/api/v1/accounts` produces `/api/v1//foo/api/v1/accounts`. Each such string is distinct from every legitimate route label and from every other attacker-generated label.

## Impact Explanation
Each unique `{method, path, code}` triple causes the OpenTelemetry SDK to allocate a new time-series accumulator in its in-process accumulation map. With four instruments (`requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, `responseSizeHistogram`) — the histograms each having multiple bucket boundaries — a single unique path generates O(tens) of in-memory objects. Sending N distinct paths generates O(N × tens) objects. At scale this exhausts Node.js heap memory, crashes the mirror-node REST process, and destroys the metrics pipeline, directly impairing the ability to observe and operate the network. **Severity: High** (availability impact on the API and monitoring plane).

## Likelihood Explanation
No authentication is required. Any HTTP client can send GET requests to arbitrary paths. The attack is trivially scriptable: a loop generating `GET /api/v1/<random-uuid>` requests produces a new unique label on every iteration. The metrics middleware is applied globally (`app.useExt(metricsHandler())`) [6](#0-5) 
and fires on every request including 404s. The only gate is `config.metrics.enabled`, which is expected to be `true` in production deployments.

## Recommendation
Replace the `path = req.path` fallback with a fixed sentinel string (e.g., `'unknown'` or `'unmatched'`) so that all unmatched routes collapse to a single label value:

```js
if (!req.route) {
  path = 'unknown';   // ← bounded, not user-controlled
} else {
  path = (req.baseUrl ?? '') + req.route?.path;
}
```

This eliminates the cardinality explosion while still allowing matched routes to be tracked accurately. Optionally, a small allowlist of known static paths (e.g., `/health/liveness`, `/health/readiness`) can be handled before the fallback.

## Proof of Concept
```bash
# Send 10,000 requests with unique UUID paths
for i in $(seq 1 10000); do
  curl -s "http://<mirror-node-host>/api/v1/$(uuidgen)" &
done
wait
```

Each request returns a 404, but `toOpenApiPath()` records the full UUID path as the `path` label. After N iterations, the OpenTelemetry SDK holds N × 4 unique instrument accumulators (plus histogram bucket entries), growing the Node.js heap unboundedly. Scraping `/swagger/metrics/` will show an ever-growing number of unique `path` label values across `api_request_total`, `api_request_duration_milliseconds`, `api_request_size_bytes`, and `api_response_size_bytes`.
