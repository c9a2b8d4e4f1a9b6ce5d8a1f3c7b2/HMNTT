Audit Report

## Title
Missing Null-Byte Sanitization on `ContractResult.errorMessage` Causes PostgreSQL COPY Failure in Batch Insert

## Summary
`ContractResult.errorMessage` is populated from protobuf without calling `DomainUtils.sanitize()`, which strips null bytes (U+0000) that PostgreSQL unconditionally rejects in `text` columns. The `BatchInserter` persists records via `COPY ... FROM STDIN WITH CSV`. A single contract transaction that reverts with a null byte in its revert reason will cause every importer instance to throw a `ParserException` on that batch, halting stream ingestion at that consensus timestamp.

## Finding Description
`DomainUtils.sanitize()` exists precisely to strip null bytes before database insertion:

```java
// DomainUtils.java:239-241
public static String sanitize(String input) {
    return StringUtils.isNotEmpty(input) ? input.replace(NULL_CHARACTER, NULL_REPLACEMENT) : input;
}
``` [9](#0-8) 

It is applied to `memo` in `AbstractEntity` and `AbstractToken`, but the analogous field `errorMessage` in `ContractResult` is assigned without it in two places:

**`ContractResultServiceImpl.java:251`** (record stream path):
```java
contractResult.setErrorMessage(functionResult.getErrorMessage()); // no sanitize()
``` [2](#0-1) 

**`AbstractBlockTransactionTransformer.java:148`** (block stream path):
```java
contractResultbuilder.setErrorMessage(evmTransactionResult.getErrorMessage()); // no sanitize()
``` [10](#0-9) 

The unsanitized value flows into `ContractResult.errorMessage` (a plain `String` field) and is then serialized by `BatchInserter` directly into a PostgreSQL COPY stream:

```java
// BatchInserter.java:85
sql = String.format("COPY %s(%s) FROM STDIN WITH CSV", this.tableName, columnsCsv);
// ...
// BatchInserter.java:128
writer.writeValue(pgCopyOutputStream, items); // null byte → PostgreSQL error
``` [4](#0-3) [5](#0-4) 

PostgreSQL raises `ERROR: invalid byte sequence for encoding "UTF8": 0x00`, which is caught and re-thrown as a `ParserException`: [6](#0-5) 

## Impact Explanation
All mirror node importer instances consume the same canonical record stream. A single crafted transaction that produces a null-byte `errorMessage` will cause every importer to fail its batch insert at that consensus timestamp, throw a `ParserException`, and be unable to advance past that point. This halts ingestion of all subsequent transactions across the entire mirror node fleet simultaneously.

## Likelihood Explanation
Any unprivileged user can deploy a Solidity contract and trigger `REVERT` with arbitrary byte data. Hedera consensus nodes record the revert reason in `ContractFunctionResult.error_message`. A contract executing `revert("\x00poison")` produces a null byte in the recorded field. No special privileges, rate-limit bypass, or brute force is required — a single transaction is sufficient to trigger the condition on all importer instances.

## Recommendation
Apply `DomainUtils.sanitize()` to `errorMessage` at both assignment sites:

```java
// ContractResultServiceImpl.java:251
contractResult.setErrorMessage(DomainUtils.sanitize(functionResult.getErrorMessage()));

// AbstractBlockTransactionTransformer.java:148
contractResultbuilder.setErrorMessage(DomainUtils.sanitize(evmTransactionResult.getErrorMessage()));
```

Alternatively, add sanitization inside `ContractResult.setErrorMessage()` itself (analogous to how `AbstractEntity` and `AbstractToken` handle `memo`) to ensure all future callers are protected by default.

## Proof of Concept
1. Deploy a Solidity contract:
   ```solidity
   contract Poison {
       function trigger() external pure {
           revert("\x00poison");
       }
   }
   ```
2. Call `trigger()` via a `ContractCall` transaction on Hedera testnet.
3. The consensus node records `\u0000poison` in `ContractFunctionResult.error_message`.
4. The mirror node importer reads this value, assigns it to `ContractResult.errorMessage` without sanitization, and attempts to COPY the batch to PostgreSQL.
5. PostgreSQL rejects the null byte; `BatchInserter.persist()` throws `ParserException`.
6. The importer halts at that consensus timestamp and cannot advance, affecting all importer instances processing the same stream.
