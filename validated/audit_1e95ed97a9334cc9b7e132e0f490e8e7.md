Audit Report

## Title
Non-Retryable DoS via Crafted Empty-Endpoint Mirror Node Response in `fetchMirrorNodeNetwork()`

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` returns `retry=false` when a structurally valid HTTP 200 JSON response contains no usable `service_endpoints`. This causes `buildNetworkFromMirrorNodes()` to exit its retry loop on the very first attempt, regardless of the configured retry count. `main()` then calls `log.Fatalf`, permanently terminating the pinger process. An attacker who can serve or intercept the mirror REST response can trigger this with a single crafted reply.

## Finding Description

**Endpoint filtering — `pinger/mirror_node_client.go` lines 113–124:**

Each `serviceEndpoint` is silently dropped when both `DomainName` and `IPAddressV4` are empty strings (or `Port == 0`). [1](#0-0) 

**Non-retryable error — lines 127–129:**

When the resulting `network` map is empty, the function returns `(nil, false, error)` — the `false` marks the error as non-retryable. [2](#0-1) 

**Retry loop short-circuit — lines 58–61:**

`buildNetworkFromMirrorNodes()` breaks out of its retry loop immediately when `!retry` is true, bypassing all configured retries (`mirrorNodeClientMaxRetries` defaults to 10). [3](#0-2) 

**Process termination — `pinger/main.go` line 43:**

The error propagates to `main()`, which calls `log.Fatalf`, terminating the process. [4](#0-3) 

**Why existing checks are insufficient:**
- The HTTP status-code check (lines 90–93) only retries on 429/5xx; a 200 response bypasses it entirely. [5](#0-4) 
- The JSON decode check (lines 96–98) only catches malformed JSON; structurally valid JSON with empty strings passes. [6](#0-5) 
- The `http.Client` is constructed with only a timeout (line 46), with no TLS certificate pinning or response signing. [7](#0-6) 

**Root cause / failed assumption:** The code treats "valid JSON with no usable host fields" as a permanent, non-transient condition. This assumption is wrong: an attacker can deliberately craft exactly this response at will, on every restart.

## Impact Explanation

The pinger process terminates via `log.Fatalf`, permanently halting its function of submitting periodic transfers to verify Hiero network health. The `/tmp/ready` readiness file (written at `main.go` line 47, after `newClient`) is never created, so Kubernetes readiness probes fail and the pod is removed from service. [8](#0-7) 

The attack requires only one successful HTTP response, is immediately effective, and has no automatic recovery path — every restart will fail as long as the attacker continues to serve the malicious response.

## Likelihood Explanation

The mirror REST URL is configured via `HIERO_MIRROR_PINGER_REST`, with the documented example being `http://mirror-rest:5551` — plaintext HTTP. [9](#0-8) 

In any deployment using plaintext HTTP (common in internal Kubernetes clusters), a network-adjacent attacker — compromised sidecar, ARP/DNS poisoning on the cluster network, or a rogue service registered under the same DNS name — can perform a trivial MITM with no credentials required. Even with HTTPS, DNS poisoning or a compromised mirror REST server achieves the same result. The crafted payload is trivial to construct and requires no authentication or response signing anywhere in the code.

## Recommendation

1. **Change `retry=false` to `retry=true`** for the empty-network-map case (line 128). An empty endpoint list from a live server is plausibly transient (e.g., a deployment in progress) and should be retried.
2. **Add a minimum-node-count guard** — if the response contains fewer than N usable nodes, treat it as retryable rather than fatal.
3. **Use HTTPS** for the mirror REST endpoint in all deployments and document plaintext HTTP as unsupported for production.
4. **Consider response validation** — e.g., require at least one node with a valid account ID and endpoint before accepting the response as authoritative.

## Proof of Concept

Send the following HTTP 200 response from the mirror REST server (or via MITM) to `GET /api/v1/network/nodes`:

```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        { "domain_name": "", "ip_address_v4": "", "port": 50211 }
      ]
    }
  ],
  "links": { "next": null }
}
```

**Result:**
- Lines 114–119 filter the single endpoint (both host fields are empty strings). [10](#0-9) 
- `network` map remains empty; line 128 returns `(nil, false, error)`. [2](#0-1) 
- Line 59 evaluates `!retry` → `true` → `break`; all 10 configured retries are skipped. [11](#0-10) 
- `main()` calls `log.Fatalf` → process exits permanently. [12](#0-11)
