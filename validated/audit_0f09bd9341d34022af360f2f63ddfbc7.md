# Audit Report

## Title
Unthrottled Binary Gas Estimator Amplification via Historical Block Estimation

## Summary
A single unauthenticated POST to `/api/v1/contracts/call` with `estimate=true` and `gas=15000000` causes up to 21 full EVM executions while consuming throttle tokens for only one. The binary search loop in `BinaryGasEstimator` runs up to `maxGasEstimateRetriesCount` iterations, each invoking `doProcessCall` with `estimate=true`, which explicitly skips the `restoreGasToBucket` path — meaning no throttle accounting occurs for any binary search iteration.

## Finding Description

**Throttle gate (one-time, HTTP layer):**
`ContractController.call()` calls `throttleManager.throttle(request)` exactly once before delegating to the service layer. [1](#0-0) 

**Initial EVM call (throttle-aware):**
`ContractExecutionService.estimateGas()` calls `callContract(params, context)`, which internally calls `doProcessCall(params, params.getGas(), false)` — with `estimate=false`. The `finally` block in `doProcessCall` therefore executes `restoreGasToBucket`, partially returning unused gas to the throttle bucket. [2](#0-1) [3](#0-2) 

**Binary search loop (throttle-blind):**
`binaryGasEstimator.search()` is then called, passing `gas -> doProcessCall(params, gas, true)` as the call function. The loop runs while `lo + 1 < hi && iterationsMade < properties.getMaxGasEstimateRetriesCount()`. [4](#0-3) [5](#0-4) 

**Root cause — `estimate=true` skips all throttle accounting:**
In `doProcessCall`, the entire `finally` block — including `restoreGasToBucket` and metric recording — is guarded by `if (!estimate)`. When the binary search passes `estimate=true`, neither gas restoration nor any throttle deduction occurs for any of the up to 20 iterations. [6](#0-5) 

**Historical block I/O amplification:**
Each `callContract` invocation sets a `blockSupplier` that calls `recordFileService.findByBlockType(params.getBlock())`. For a historical block, this triggers a time-travel database query on every binary search iteration. [7](#0-6) 

## Impact Explanation
One request with `estimate=true` + historical block + `gas=15_000_000` causes up to **21 EVM executions** (1 initial + up to 20 binary search iterations) and up to 21 historical-state database queries, while consuming throttle tokens for only 1 request and approximately `gasUsed_initial` gas tokens (after restoration). An attacker operating at the rate limit drives actual EVM and I/O load at up to 21× the rate the throttle was designed to permit. Under sustained load this exhausts CPU and database connection pools, degrading or denying service to legitimate users.

## Likelihood Explanation
No authentication is required. The endpoint is public and documented. The attacker needs only a standard HTTP client. The worst-case amplification is reliably triggered by supplying a contract call with very low initial gas usage relative to the 15 M ceiling, maximizing binary search depth. Historical block support is part of the public OpenAPI spec, so the parameter combination is publicly known and trivially scriptable.

## Recommendation

1. **Account for binary search iterations in the throttle.** Before each iteration in `BinaryGasEstimator.search()`, deduct tokens proportional to the `mid` gas value being tested, or deduct a fixed per-iteration token from the rate-limit bucket.
2. **Alternatively, apply a multiplier at the HTTP layer.** When `estimate=true`, multiply the throttle cost by a factor reflecting the expected number of binary search iterations (e.g., `ceil(log2(maxGas / minGas))`).
3. **Cap `maxGasEstimateRetriesCount` as a resource-protection bound**, not just a correctness bound, and expose it as a tunable property with a conservative default.
4. **For historical block estimates**, consider requiring a separate, lower rate limit or disabling gas estimation against historical state entirely if not required by the API contract.

## Proof of Concept

```
POST /api/v1/contracts/call
Content-Type: application/json

{
  "to": "<deployed_contract>",
  "data": "<calldata_with_low_gas_usage>",
  "gas": 15000000,
  "estimate": true,
  "block": "0x<historical_block_number>"
}
```

1. `throttleManager.throttle(request)` deducts 15 M gas tokens + 1 rate token.
2. `estimateGas()` → `callContract()` → `doProcessCall(..., false)`: 1 EVM execution; gas partially restored.
3. `binaryGasEstimator.search()` loops up to 20 times, each calling `doProcessCall(..., true)`: 20 EVM executions, 0 throttle tokens deducted, 20 historical DB queries.
4. Total: **21 EVM executions + 20 historical DB queries** for 1 throttle deduction.
5. Repeat at the rate limit to sustain 21× amplified load.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
