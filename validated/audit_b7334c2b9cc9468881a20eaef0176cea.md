Audit Report

## Title
Silent Long Overflow in `getAdjustedLowerRangeValue()` Bypasses Range Validation in `Bound`

## Summary
An unprivileged user can supply `account.id=gt:511.65535.274877906943` — a fully valid Hedera entity ID whose encoded `long` value equals `Long.MAX_VALUE` — causing `Bound.getAdjustedLowerRangeValue()` to silently overflow to `Long.MIN_VALUE` via an unchecked `lowerBound++`. The subsequent range-validity guard `adjustedLower > adjustedUpper` evaluates to `Long.MIN_VALUE > Long.MAX_VALUE` = `false`, so no `IllegalArgumentException` is thrown and the structurally invalid `Bound` is forwarded to the repository layer.

## Finding Description

**Root cause — `Bound.getAdjustedLowerRangeValue()` (`Bound.java` lines 86–97):**

```java
public long getAdjustedLowerRangeValue() {
    if (this.lower == null) { return 0; }
    long lowerBound = this.lower.value();
    if (this.lower.operator() == RangeOperator.GT) {
        lowerBound++;   // ← unchecked; wraps to Long.MIN_VALUE when value == Long.MAX_VALUE
    }
    return lowerBound;
}
``` [1](#0-0) 

**Guard that is bypassed — `Bound` constructor (`Bound.java` lines 56–60):**

```java
long adjustedLower = getAdjustedLowerRangeValue(); // Long.MIN_VALUE after overflow
long adjustedUpper = adjustUpperBound();            // Long.MAX_VALUE (no upper supplied)
if (primarySortField && adjustedLower > adjustedUpper) { // MIN > MAX == false → skipped
    throw new IllegalArgumentException(...);
}
``` [2](#0-1) 

**Same bypass in `NftAllowanceServiceImpl.checkOwnerSpenderParamValidity` (lines 41–44):**

```java
if (!ownerOrSpenderParams.hasLowerAndUpper()
        && tokenParams.getAdjustedLowerRangeValue() > tokenParams.adjustUpperBound()) {
    throw new IllegalArgumentException("Invalid range provided for token.id");
}
``` [3](#0-2) 

**Why `Long.MAX_VALUE` is a reachable, valid entity-ID value:**

`EntityId.encode()` validates each component independently against its bit-width mask (shard ≤ 1023 / 10 bits, realm ≤ 65535 / 16 bits, num ≤ 274877906943 / 38 bits). [4](#0-3) 

`EntityId.of(511, 65535, 274877906943)` passes all three guards. The encoding `(274877906943 & NUM_MASK) | (65535 << 38) | (511 << 54)` sets bits 0–62 to 1 and leaves bit 63 (the sign bit) as 0, producing exactly `Long.MAX_VALUE` (9223372036854775807). `EntityIdRangeParameter.valueOf("gt:511.65535.274877906943")` parses this without error and produces `EntityIdRangeParameter(GT, Long.MAX_VALUE)`. [5](#0-4) 

**Identical overflow in `getInclusiveRangeValue()` (`Bound.java` lines 107–118):**

```java
if (operator == GT) {
    value += 1L;   // ← same unchecked increment
}
``` [6](#0-5) 

## Impact Explanation
The server silently accepts a semantically impossible range (`GT Long.MAX_VALUE`) instead of returning HTTP 400. Because the `lower` field retains the original `GT:Long.MAX_VALUE` operator/value, the generated SQL predicate `spender > 9223372036854775807` returns zero rows — no data is disclosed. However, the overflowed value (`Long.MIN_VALUE`) is also consumed by `hasEqualBounds()` and `adjustLowerRange()`, which could produce incorrect query rewrites if the code path is reached with a combined lower+upper bound. Severity is **low**: no authorization boundary is crossed (NFT allowances are public data), and the limit parameter is still enforced. The primary risk is logic corruption in range-rewrite paths and silent contract violation of the validation layer.

## Likelihood Explanation
Exploitability requires zero privileges — only a valid HTTP GET to `/api/v1/accounts/{id}/allowances/nfts`. The trigger value (`511.65535.274877906943`) is a syntactically normal Hedera entity ID that passes every existing input-validation layer. Any user who discovers the boundary condition (e.g., by fuzzing entity-ID ranges) can reproduce it deterministically on every request.

## Recommendation
Add an overflow guard in `getAdjustedLowerRangeValue()` before incrementing:

```java
if (this.lower.operator() == RangeOperator.GT) {
    if (lowerBound == Long.MAX_VALUE) {
        throw new IllegalArgumentException(
            "Range lower bound overflow: GT Long.MAX_VALUE is not representable");
    }
    lowerBound++;
}
```

Apply the same guard to `getInclusiveRangeValue()` (`value += 1L` branch) and to `adjustUpperBound()` (`upperBound--` branch, symmetric underflow). Alternatively, use `Math.addExact` / `Math.subtractExact` and catch `ArithmeticException`, converting it to `IllegalArgumentException` with an appropriate message.

## Proof of Concept

```
GET /api/v1/accounts/0.0.1/allowances/nfts?account.id=gt:511.65535.274877906943
```

1. `EntityIdRangeParameter.valueOf("gt:511.65535.274877906943")` produces `(GT, 9223372036854775807)`.
2. `new Bound(accountIds, true, "account.id", field)` is constructed.
3. Inside the constructor, `getAdjustedLowerRangeValue()` computes `Long.MAX_VALUE + 1` → `Long.MIN_VALUE`.
4. Guard: `Long.MIN_VALUE > Long.MAX_VALUE` = `false` → no exception thrown.
5. `checkOwnerSpenderParamValidity` is called; the same overflowed comparison at line 42 also evaluates to `false`.
6. `repository.findAll(request, id)` is invoked with the invalid `Bound`.
7. Server returns HTTP 200 with an empty `allowances` array instead of HTTP 400.

    public static EntityIdRangeParameter valueOf(String entityIdRangeParam) {
        if (StringUtils.isBlank(entityIdRangeParam)) {

        }

        var splitVal = entityIdRangeParam.split(":");
        return switch (splitVal.length) {
            case 1 -> new EntityIdRangeParameter(RangeOperator.EQ, getEntityId(splitVal[0]));
            case 2 -> new EntityIdRangeParameter(RangeOperator.of(splitVal[0]), getEntityId(splitVal[1]));
            default -> throw new IllegalArgumentException("Invalid range operator. Should have format 'operator:ID'");
        };
    }

    private static EntityId getEntityId(String entityId) {
        List<Long> parts = Splitter.on('.')
                .splitToStream(Objects.requireNonNullElse(entityId, ""))
                .map(Long::valueOf)
                .filter(n -> n >= 0)
                .toList();

        if (parts.size() != StringUtils.countMatches(entityId, ".") + 1) {
            throw new IllegalArgumentException("Invalid entity ID");
        }

        var properties = CommonProperties.getInstance();
        return switch (parts.size()) {
            case 1 -> EntityId.of(properties.getShard(), properties.getRealm(), parts.get(0));
            case 2 -> EntityId.of(properties.getShard(), parts.get(0), parts.get(1));
            case 3 -> EntityId.of(parts.get(0), parts.get(1), parts.get(2));
            default -> throw new IllegalArgumentException("Invalid entity ID");
        };
    }
