Audit Report

## Title
Negative Cache Poisoning via `Optional.empty()` in `findRuntimeBytecode()` Allows Targeted DoS of Newly Deployed Contracts

## Summary
The `@Cacheable` annotation on `ContractRepository.findRuntimeBytecode()` uses `unless = "#result == null"` to suppress caching of absent results. This guard is ineffective because the method returns `Optional<byte[]>`: Spring evaluates `#result` against the `Optional` object itself, and `Optional.empty()` is a non-null singleton. Any request for a `contractId` that arrives before the importer writes the bytecode row will cache `Optional.empty()`, and the 1-hour `expireAfterAccess` TTL blocks the correct bytecode from being served for up to an hour after deployment.

## Finding Description

**Exact location:**

`web3/src/main/java/org/hiero/mirror/web3/repository/ContractRepository.java`, lines 16–18:
```java
@Cacheable(cacheNames = CACHE_NAME_CONTRACT, cacheManager = CACHE_MANAGER_CONTRACT, unless = "#result == null")
@Query(value = "select runtime_bytecode from contract where id = :contractId", nativeQuery = true)
Optional<byte[]> findRuntimeBytecode(final Long contractId);
``` [1](#0-0) 

**Root cause — failed assumption in `unless` condition:**

Spring's `@Cacheable` with an `Optional` return type evaluates the `unless` SpEL expression against the `Optional` object itself (not the unwrapped value). `Optional.empty()` is a non-null singleton, so `#result == null` evaluates to `false`. Spring then stores the unwrapped value (`null`) as a `NullValue` sentinel in the Caffeine cache. On subsequent cache hits, Spring returns `Optional.ofNullable(null)` = `Optional.empty()`. The correct fix is `unless = "#result == null || !#result.isPresent()"`.

**Cache TTL:**

`web3/src/main/java/org/hiero/mirror/web3/repository/properties/CacheProperties.java`, line 22:
```java
private String contract = "expireAfterAccess=1h,maximumSize=1000,recordStats";
``` [2](#0-1) 

`expireAfterAccess` resets the TTL on every read, so an attacker who keeps polling extends the denial indefinitely.

**Cache manager configuration:**

`web3/src/main/java/org/hiero/mirror/web3/evm/config/EvmConfiguration.java`, lines 67–73:
```java
@Bean(CACHE_MANAGER_CONTRACT)
CacheManager cacheManagerContract() {
    final CaffeineCacheManager caffeineCacheManager = new CaffeineCacheManager();
    caffeineCacheManager.setCacheNames(Set.of(CACHE_NAME_CONTRACT));
    caffeineCacheManager.setCacheSpecification(cacheProperties.getContract());
    return caffeineCacheManager;
}
``` [3](#0-2) 

`CaffeineCacheManager` defaults to `allowNullValues = true`, so the `NullValue` sentinel is stored and served on cache hits.

**Downstream impact in `ContractBytecodeReadableKVState`:**

`web3/src/main/java/org/hiero/mirror/web3/state/keyvalue/ContractBytecodeReadableKVState.java`, lines 42–46:
```java
return contractRepository
        .findRuntimeBytecode(entityId.getId())
        .map(Bytes::wrap)
        .map(Bytecode::new)
        .orElse(null);
``` [4](#0-3) 

A cached `Optional.empty()` causes `readFromDataSource` to return `null`, which the EVM layer interprets as a non-existent contract.

## Impact Explanation
Any contract deployed on the Hedera network can be rendered inaccessible via the mirror node's web3 API for up to 1 hour (or indefinitely with polling). `eth_getCode` returns empty bytecode; `eth_call` against the contract fails or behaves as if calling an EOA. The denial is targeted (per contractId), repeatable, and requires no authentication.

## Likelihood Explanation
Hedera entity IDs are monotonically increasing and publicly observable on-chain. The importer introduces a processing lag (typically seconds to tens of seconds) between on-chain finality and the DB write, providing a reliable window to send the poisoning request. No special privileges, tokens, or network access beyond the public web3 API endpoint are needed. The attack is trivially scriptable and repeatable for every new contract deployment.

## Recommendation
Fix the `unless` condition to also suppress caching of empty Optionals:

```java
@Cacheable(
    cacheNames = CACHE_NAME_CONTRACT,
    cacheManager = CACHE_MANAGER_CONTRACT,
    unless = "#result == null || !#result.isPresent()"
)
```

This ensures that only a present (non-empty) `Optional` is stored in the cache, while absent results always fall through to the DB on the next request.

## Proof of Concept
1. Observe or predict the next contractId (Hedera IDs are sequential and publicly visible before importer processing).
2. Before the importer writes the `runtime_bytecode` row, send an `eth_getCode` or `eth_call` request to the web3 API for that contractId.
3. `findRuntimeBytecode()` queries the DB, receives `Optional.empty()`, and caches it (the `unless` guard does not fire because `Optional.empty() != null`).
4. The importer subsequently writes the correct bytecode to the DB.
5. All future calls to `findRuntimeBytecode()` for that contractId return the cached `Optional.empty()` without hitting the DB.
6. `ContractBytecodeReadableKVState.readFromDataSource()` maps `Optional.empty()` to `null` via `.orElse(null)`, so every EVM execution sees the contract as non-existent.
7. Polling the endpoint resets the `expireAfterAccess` timer, extending the denial indefinitely.
