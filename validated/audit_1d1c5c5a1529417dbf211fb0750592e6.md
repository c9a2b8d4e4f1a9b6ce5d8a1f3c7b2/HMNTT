# Audit Report

## Title
Directory Substitution Attack on `/tmp/alive` Bypasses Liveness Probe Indefinitely

## Summary
An actor with write access to `/tmp` inside the pinger container can create `/tmp/alive` as a directory. Because `os.WriteFile` silently fails on a directory (`EISDIR`) and the error is discarded, the real heartbeat timestamp is never updated. The healthcheck's `os.Stat` call succeeds on a directory and returns a controllable `ModTime()`, so the liveness probe permanently exits 0 even though the pinger's heartbeat mechanism is broken.

## Finding Description
**Heartbeat writer** — `pinger/main.go`, line 36:
```go
_ = os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644)
```
The return value of `os.WriteFile` is unconditionally discarded. If `/tmp/alive` is a directory, the call returns `EISDIR` and the timestamp is never written — silently. [4](#0-3) 

**Liveness checker** — `pinger/cmd/healthcheck/main.go`, lines 27–38:
```go
fi, err := os.Stat("/tmp/alive")
if err != nil { ... os.Exit(1) }
age := time.Since(fi.ModTime())
if age > 2*time.Minute { ... os.Exit(1) }
os.Exit(0)
```
`os.Stat` succeeds on a directory. There is no `fi.Mode().IsRegular()` check. The only guard is `ModTime()` freshness, which the attacker controls by creating/removing entries inside the directory (`touch /tmp/alive/x && rm /tmp/alive/x` updates directory mtime on Linux). [2](#0-1) 

**Root cause / failed assumption:** Both the writer and the checker assume `/tmp/alive` is always a regular file. Neither enforces this invariant.

## Impact Explanation
The Kubernetes liveness exec probe calls `healthcheck live`. If it exits 0, the pod is considered alive and is never restarted. With this attack, the probe exits 0 indefinitely even though the pinger's heartbeat mechanism is broken. The pod is stuck in a permanently "alive but non-functional" state, defeating the entire purpose of the liveness probe and preventing automatic recovery. Severity: Medium (availability / griefing impact).

## Likelihood Explanation
Precondition: write access to `/tmp` inside the container (e.g., a compromised sidecar, a shared `emptyDir` volume mount, or a partial container escape). `/tmp` is world-writable by default on Linux. The attack is a single `mkdir /tmp/alive` executed before or during pinger startup. Refreshing the mtime requires only `touch /tmp/alive/x && rm /tmp/alive/x`, which is trivially scriptable. Fully repeatable.

## Recommendation
1. **Enforce regular-file invariant in the checker** — add an `fi.Mode().IsRegular()` check immediately after `os.Stat`:
   ```go
   if !fi.Mode().IsRegular() {
       fmt.Fprintln(os.Stderr, "heartbeat path is not a regular file")
       os.Exit(1)
   }
   ```
2. **Log or handle the write error** — replace `_ = os.WriteFile(...)` with explicit error handling so failures are visible and actionable:
   ```go
   if err := os.WriteFile("/tmp/alive", ...); err != nil {
       log.Printf("heartbeat write failed: %v", err)
   }
   ```
3. **Use `O_WRONLY|O_CREATE|O_TRUNC` with explicit `open` flags** — opening with `os.OpenFile` and explicit flags will fail fast if the path is a directory, making the error impossible to miss.

## Proof of Concept
```bash
# Inside the container, before or while pinger is running:
rm -f /tmp/alive
mkdir /tmp/alive

# Keep mtime fresh every 30 seconds (well within the 2-minute window):
while true; do
    touch /tmp/alive/x && rm /tmp/alive/x
    sleep 30
done &

# Verify: healthcheck exits 0 even though no real heartbeat is written
/app/healthcheck live; echo "exit: $?"
# => exit: 0  (probe reports alive indefinitely)
```
