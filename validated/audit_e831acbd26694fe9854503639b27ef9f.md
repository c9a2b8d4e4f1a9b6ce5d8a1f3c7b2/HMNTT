# Audit Report

## Title
Nil Pointer Dereference in `AccountBalance()` via `AllAccountBalances()` with Missing `AccountIdentifier`

## Summary
`AllAccountBalances()` in `rosetta/app/services/account_service.go` delegates to `AccountBalance()` by copying `request.AccountIdentifier` directly, without a nil guard. `AccountBalance()` unconditionally dereferences `request.AccountIdentifier.Address` at line 45. Because the Rosetta SDK asserter treats `account_identifier` as optional for `AllAccountBalancesRequest` (per the Rosetta spec), a request to `/account/balance/all` with no `account_identifier` passes all upstream validation and causes a nil pointer dereference panic inside the handler goroutine.

## Finding Description
**Vulnerable code path:**

`AllAccountBalances()` (lines 88–113) constructs a delegated request by copying fields verbatim:

```go
singleAccountBalanceRequest := rTypes.AccountBalanceRequest{
    NetworkIdentifier: request.NetworkIdentifier,
    AccountIdentifier: request.AccountIdentifier,  // nil propagates here
    BlockIdentifier:   request.BlockIdentifier,
    Currencies:        request.Currencies,
}
response, err := a.AccountBalance(ctx, &singleAccountBalanceRequest)
``` [1](#0-0) 

`AccountBalance()` then immediately dereferences `AccountIdentifier` without any nil check:

```go
accountId, err := types.NewAccountIdFromString(request.AccountIdentifier.Address, a.systemShard, a.systemRealm)
``` [2](#0-1) 

**Root cause:** The code assumes `AccountIdentifier` is always non-nil. This assumption holds for direct `/account/balance` calls (where the Rosetta SDK asserter enforces it as required), but not for `/account/balance/all`, where `account_identifier` is optional per the Rosetta specification. There is no nil guard in `AllAccountBalances()` before the delegated call, and no nil guard at the top of `AccountBalance()`.

**Why existing checks fail:** The `rosettaAsserter.NewServer(...)` asserter registered in `main.go` validates `AccountBalanceRequest.AccountIdentifier` as required for `/account/balance`, but `AllAccountBalancesRequest.AccountIdentifier` is optional per the Rosetta spec, so the asserter allows a nil value through to `AllAccountBalances()`. [5](#0-4) 

**Test coverage gap:** All test helpers always supply a non-nil `AccountIdentifier`. The nil path is never exercised in the test suite. [3](#0-2) 

## Impact Explanation
When triggered, the handler goroutine panics. Go's `net/http` server recovers per-handler panics, so the process does not exit, but the client receives a connection reset or 500 response. An attacker can repeat this indefinitely, causing continuous panic log entries, potential log-storage exhaustion, and degraded observability. On nodes where log-based alerting or rate-limiting is absent, this constitutes a reliable, low-cost availability degradation vector.

## Likelihood Explanation
The `/account/balance/all` endpoint is unauthenticated and publicly reachable. No credentials, tokens, or prior knowledge are required. The payload is a single well-formed JSON object with `account_identifier` omitted or set to `null`. Any external user can craft and replay this request in a tight loop with standard HTTP tooling.

## Recommendation
Add a nil guard for `AccountIdentifier` at the top of `AllAccountBalances()` before delegating to `AccountBalance()`:

```go
func (a *AccountAPIService) AllAccountBalances(
    ctx context.Context,
    request *rTypes.AllAccountBalancesRequest,
) (*rTypes.AllAccountBalancesResponse, *rTypes.Error) {
    if request.AccountIdentifier == nil {
        return nil, errors.ErrInvalidAccount
    }
    // ... existing delegation logic
}
```

Alternatively, add a nil guard at the top of `AccountBalance()` itself, since it is a shared entry point. Additionally, add a unit test for `AllAccountBalances()` with a nil `AccountIdentifier` to prevent regression. [8](#0-7) 

## Proof of Concept
```bash
curl -X POST http://<rosetta-host>:<port>/account/balance/all \
  -H "Content-Type: application/json" \
  -d '{
    "network_identifier": {
      "blockchain": "Hedera",
      "network": "mainnet"
    }
  }'
```

This request omits `account_identifier`. The Rosetta SDK asserter permits it (optional field). `AllAccountBalances()` receives `request.AccountIdentifier == nil`, copies it into the delegated `AccountBalanceRequest`, and `AccountBalance()` panics at line 45 when dereferencing `nil.Address`. [2](#0-1)
