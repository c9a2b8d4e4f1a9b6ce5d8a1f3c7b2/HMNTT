## Audit Report

## Title
Gas Throttle Bypass via Non-Existent Alias Addresses Enables Sustained Database Lookup Amplification

## Summary
An unauthenticated attacker can POST to `/api/v1/contracts/call` with a non-long-zero `from` address that does not exist in the database. Each such request triggers a live database lookup with no cross-request caching, while the gas throttle is completely bypassed — either because `gas ≤ 10,000` scales to zero tokens, or because the full gas amount is restored in the `finally` block when the pre-EVM exception leaves `result == null`. The only effective rate control is the global 500 RPS bucket, which is shared across all clients with no per-IP subdivision.

## Finding Description

**Verified code path:**

`ContractCallService.doProcessCall()` → `transactionExecutionService.execute()` → `buildContractCallTransactionBody()` → `defaultTransactionBodyBuilder()` → `getSenderAccountID()` → `getSenderAccountIDAsNum()` → `aliasesReadableKVState.get(convertAddressToProtoBytes(senderAddress))` → `AliasesReadableKVState.readFromDataSource()` → `commonEntityAccessor.get(alias.value(), timestamp)` → **database query**.

**Root cause 1 — No cross-request cache (confirmed):**

`ContractCallContext` is instantiated fresh per request: [1](#0-0) 

Its `readCache` is a plain `HashMap` stored in a `ScopedValue` that is discarded when the scoped call returns: [2](#0-1) 

`AliasedAccountCacheManager.putAccountNum()` writes only into this per-request cache via `ContractCallContext.get().getReadCacheState(...)`: [3](#0-2) 

There is no shared, cross-request cache. Every request with a unique non-existent address unconditionally hits the database. The `ReadableKVStateBase.get()` intra-request cache (using a `marker` sentinel for null values) only deduplicates lookups *within* a single request, not across requests. [4](#0-3) 

**Root cause 2 — Gas throttle bypass (confirmed):**

`ThrottleProperties.scaleGas()` returns `0` for any `gas ≤ 10,000`: [5](#0-4) 

`ThrottleManagerImpl.throttle()` calls `gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))`. When `scaleGas` returns `0`, `tryConsume(0)` always succeeds, consuming no tokens: [6](#0-5) 

For `gas > 10,000`, tokens are consumed but then fully restored. `doProcessCall()` initialises `result = null` and catches `MirrorEvmTransactionException` thrown by `throwPayerAccountNotFoundException()` before EVM execution. The exception carries no `EvmTransactionResult`, so `result` remains `null` in the `finally` block: [7](#0-6) 

`restoreGasToBucket()` with `result == null` restores `gasLimitToRestoreBaseline = gasLimit * gasLimitRefundPercent / 100`. With the default `gasLimitRefundPercent = 100`, the full gas limit is restored — net gas cost is zero: [8](#0-7) 

`ThrottleManagerImpl.restore()` also passes the restored amount through `scaleGas()`, so the bucket ends up in exactly the same state as before the request. [9](#0-8) 

**Root cause 3 — No per-IP limiting (confirmed):**

`ThrottleConfiguration` creates a single global `Bucket` for the RPS limit, shared across all clients with no per-source-IP subdivision: [10](#0-9) 

The default `requestsPerSecond` is 500: [11](#0-10) 

**Trigger point:**

`getSenderAccountIDAsNum()` calls `aliasesReadableKVState.get()` for every non-long-zero sender address. For a non-existent address, `readFromDataSource()` returns `null`, `throwPayerAccountNotFoundException()` is called, and the request terminates — but the DB query has already executed: [12](#0-11) [13](#0-12) 

## Impact Explanation

At the default RPS cap of 500/s, an attacker sustains 500 uncached database alias lookups per second at zero gas cost. Each lookup searches for a non-existent entity, producing an index miss. Sustained over time this elevates database CPU, I/O, and connection utilization. In deployments where the importer and web3 module share the same PostgreSQL instance, this can degrade or deny service to legitimate users. The `requestsPerSecond` default of 500 is the sole remaining control.

## Likelihood Explanation

No authentication, API key, or account is required. The attacker only needs to POST to `/api/v1/contracts/call` with a valid JSON body, a non-zero `from` address, and `gas ≤ 10,000`. Unique addresses are trivially generated (random 20-byte hex strings). The attack is repeatable indefinitely and requires no special tooling beyond a standard HTTP client capable of 500 req/s.

## Recommendation

1. **Introduce a cross-request negative-result cache** for alias lookups. A bounded, short-TTL (e.g., 5–30 s) Caffeine or Redis cache keyed on the alias bytes would prevent repeated DB hits for the same non-existent address.
2. **Fix the gas-restoration logic for pre-EVM failures.** When `result == null` (exception thrown before EVM execution), do not restore gas to the bucket. The gas was consumed as a cost of the lookup; restoring it nullifies the throttle.
3. **Add per-IP (or per-client) rate limiting** in addition to the global RPS bucket, so a single attacker cannot consume the entire global allowance.
4. **Raise the `GAS_SCALE_FACTOR` floor or enforce a minimum gas cost** for requests that reach the alias-lookup stage, so that even small-gas requests consume a non-zero number of tokens.

## Proof of Concept

```bash
# Generate 500 unique random non-existent EVM addresses and fire them at 500 req/s
for i in $(seq 1 500); do
  ADDR="0x$(openssl rand -hex 20)"
  curl -s -X POST http://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d "{\"from\":\"$ADDR\",\"to\":\"0x0000000000000000000000000000000000000001\",\"gas\":10000,\"data\":\"0x\"}" &
done
wait
```

Each request: passes the RPS bucket (1 token consumed), passes the gas bucket (`scaleGas(10000) == 0`, 0 tokens consumed), reaches `getSenderAccountIDAsNum()`, triggers a DB lookup for a non-existent alias, throws `MirrorEvmTransactionException`, restores 0 gas tokens (since `scaleGas(0) == 0`). Net cost to the gas bucket: zero. Net DB queries: 500 per second, sustained indefinitely.

            CallServiceParameters params, long estimatedGas, boolean estimate) throws MirrorEvmTransactionException {
        EvmTransactionResult result = null;
        var status = ResponseCodeEnum.SUCCESS.toString();

        try {
            result = transactionExecutionService.execute(params, estimatedGas);

            if (!estimate) {
                validateResult(result, params);
            }

                // Only record metric if EVM is invoked and not inside estimate loop
                if (result != null) {
                    updateMetrics(params, result.gasUsed(), 1, status);
                }
            }
        }
        return result;

    private void restoreGasToBucket(EvmTransactionResult result, long gasLimit) {
        // If the transaction fails, gasUsed is equal to gasLimit, so restore the configured refund percent
        // of the gasLimit value back in the bucket.
        final var gasLimitToRestoreBaseline = (long) (gasLimit * throttleProperties.getGasLimitRefundPercent() / 100f);
        if (result == null || (!result.isSuccessful() && gasLimit == result.gasUsed())) {
            throttleManager.restore(gasLimitToRestoreBaseline);
        } else {
            // The transaction was successful or reverted, so restore the remaining gas back in the bucket or
            // the configured refund percent of the gasLimit value back in the bucket - whichever is lower.
            final var gasRemaining = gasLimit - result.gasUsed();
            throttleManager.restore(Math.min(gasRemaining, gasLimitToRestoreBaseline));
        }
    }

    private AccountID getSenderAccountIDAsNum(final Address senderAddress) {
        AccountID accountIDNum;
        if (senderAddress != null && !ConversionUtils.isLongZero(senderAddress)) {
            // If the address is an alias we need to first check if it exists and get the AccountID as a num.
            accountIDNum = aliasesReadableKVState.get(convertAddressToProtoBytes(senderAddress));
            if (accountIDNum == null) {
                throwPayerAccountNotFoundException(SENDER_NOT_FOUND);
            }
        } else {
            final var senderAccountID = accountIdFromEvmAddress(senderAddress);
            // If the address was passed as a long-zero address we need to convert it to the correct AccountID type.
            accountIDNum = AccountID.newBuilder()
                    .accountNum(senderAccountID.getAccountNum())
                    .shardNum(senderAccountID.getShardNum())
                    .realmNum(senderAccountID.getRealmNum())
                    .build();
        }
        return accountIDNum;
    }
