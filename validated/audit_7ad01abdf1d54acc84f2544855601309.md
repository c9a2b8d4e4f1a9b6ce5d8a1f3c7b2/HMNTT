Audit Report

## Title
Empty-Credential Authentication Bypass in Metrics Endpoint `authenticate()`

## Summary
The `authenticate()` function in `rest/middleware/metricsHandler.js` performs no validation that the configured `username` and `password` are non-empty before comparing them via `tsscmp`. When both config values are empty strings (e.g., from unset Kubernetes secrets), an attacker can send `Authorization: Basic Og==` (base64 of `:`) to obtain a truthy `{name: '', pass: ''}` object from `basicAuth()`, causing both `tsscmp` calls to return `true` and granting full access to the Prometheus metrics endpoint.

## Finding Description
**File:** `rest/middleware/metricsHandler.js`, function `authenticate()`, lines 137–144.

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

The function reads `username` and `password` from `config.metrics.config` and immediately passes them to `tsscmp` with no guard against empty-string values. [1](#0-0) 

The documented defaults for these fields are `mirror_api_metrics` and `mirror_api_metrics_pass` respectively. [2](#0-1) 

However, if an operator overrides them with empty strings (e.g., via `HIERO_MIRROR_REST_METRICS_CONFIG_USERNAME=` and `HIERO_MIRROR_REST_METRICS_CONFIG_PASSWORD=`), the only existing guard — `credentials &&` — does not prevent bypass. The `basic-auth` package parses `Authorization: Basic Og==` (base64 of `:`) as the truthy object `{name: '', pass: ''}`, and `tsscmp('', '')` returns `true` for both comparisons, so `authenticate()` returns `true` and the metrics payload is served. [3](#0-2) 

## Impact Explanation
An attacker gains unauthenticated read access to the full Prometheus metrics output, which includes per-route request counts, durations, error rates, in-flight request counts, and Node.js process memory/CPU telemetry. This leaks internal operational state (traffic patterns, error spikes, resource usage) that can be used to fingerprint the deployment, time attacks, or confirm the success of denial-of-service attempts. The metrics endpoint is explicitly documented as access-controlled (`authentication: true` by default), so this bypass directly violates the intended security boundary. [4](#0-3) 

## Likelihood Explanation
The precondition — empty credentials — is realistic in container/Kubernetes deployments where secrets are injected via environment variables. An unset or accidentally blank secret value silently produces an empty string in the config. The attacker needs no privileges, only network access to the REST API port (default 5551). The exploit is a single HTTP request with a known, fixed header value (`Authorization: Basic Og==`). It is repeatable and leaves no distinguishing log entry beyond a normal 200 response. [5](#0-4) 

## Recommendation
Add an explicit guard in `authenticate()` to reject empty-string credentials before performing the `tsscmp` comparison:

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;
  }
  if (!username || !password) {
    return false; // Treat empty/missing credentials as misconfigured → deny all
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

Alternatively, add startup validation in `config.js` (analogous to the existing `parseUsersConfig` validation that already rejects empty `username`/`password` for the `users` array) to throw an `InvalidConfigError` when `metrics.config.authentication` is `true` but either `username` or `password` is empty. [6](#0-5) 

## Proof of Concept
**Precondition:** Deploy with `HIERO_MIRROR_REST_METRICS_CONFIG_USERNAME=` and `HIERO_MIRROR_REST_METRICS_CONFIG_PASSWORD=` (empty), leaving `authentication` at its default of `true`.

**Request:**
```
GET /swagger/metrics/ HTTP/1.1
Host: <mirror-node-host>:5551
Authorization: Basic Og==
```

(`Og==` is the base64 encoding of `:`, representing empty username and empty password per RFC 7617.)

**Expected (secure) response:** `401 Unauthorized`

**Actual response:** `200 OK` with full Prometheus metrics body, because:
1. `basicAuth(req)` parses `Basic Og==` → `{name: '', pass: ''}` (truthy object, not `null`/`undefined`)
2. `tsscmp('', '')` → `true` (both strings are equal)
3. `tsscmp('', '')` → `true` (both strings are equal)
4. `authenticate()` returns `true`; metrics are served at lines 161–164 [7](#0-6)
