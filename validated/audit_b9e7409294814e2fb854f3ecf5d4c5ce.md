# Audit Report

## Title
Silent Importer Stall via Malformed Data Files Uploaded to Cloud Storage Bucket

## Summary
In `Downloader.verifySignatures()`, when `streamFileReader.read()` throws `InvalidStreamFileException` or `StreamFileReaderException` (both unchecked exceptions not listed in the named multi-catch), the generic `catch (Exception e)` block silently logs an error and advances to the next CONSENSUS_REACHED node. If all such nodes serve malformed data files, `verifySignatures()` returns `false`, `onVerified()` is never called, `lastStreamFile` is never advanced, and the importer retries the same batch on every scheduled invocation — permanently stalling stream processing.

## Finding Description
**File:** `importer/src/main/java/org/hiero/mirror/importer/downloader/Downloader.java`

In `verifySignatures()` (lines 320–383), for each CONSENSUS_REACHED signature, the code calls:

```java
T streamFile = streamFileReader.read(streamFileData);   // line 336
verify(streamFile, signature);                           // line 338 — never reached if read() throws
```

The catch structure at lines 363–379 is:
```java
} catch (FileOperationException | HashMismatchException | TransientProviderException e) {
    log.warn(...);
} catch (Exception e) {
    log.error("Error downloading data file from node {} ...", nodeId, ...);
    // continues to next node
}
```

`InvalidStreamFileException` and `StreamFileReaderException` (thrown by `CompositeRecordFileReader`) are not listed in the named multi-catch. They fall to the generic handler, which logs and continues. Neither is imported in `Downloader.java`.

`CompositeRecordFileReader.read()` (lines 39–56) reads the first 4 bytes as a version integer and throws `InvalidStreamFileException` for any value not in `{1, 2, 5, 6}`:

```java
default:
    throw new InvalidStreamFileException(
        String.format("Unsupported record file version %d in file %s", version, filename));
```

Any `IOException` during reading is wrapped in `StreamFileReaderException` (line 63).

When all CONSENSUS_REACHED nodes are exhausted, `verifySignatures()` returns `false` (line 382). Back in `verifySigsAndDownloadDataFiles()` (lines 308–311), this only emits a log error — no exception is thrown, no state is advanced. `lastStreamFile` (line 86) is never updated because `onVerified()` (line 361) is never reached. On the next scheduled invocation, `getStartAfterFilename()` (lines 253–263) reads the unchanged `lastStreamFile` and returns the same filename, causing the same batch to be downloaded and fail again.

## Impact Explanation
The importer permanently stalls on the targeted batch file. No subsequent record files, balance files, or event files are processed. The mirror node falls arbitrarily far behind the network. Because the failure is only logged (not alerted or escalated), operators may not notice until the mirror node's data is visibly stale. This constitutes a sustained denial-of-service against the mirror node's entire stream processing pipeline.

## Likelihood Explanation
The precondition is write access to the cloud storage bucket. This is achievable via misconfigured bucket ACLs (public write), compromised cloud credentials, or a supply-chain/insider attack on the storage layer. No Hedera network keys or node access are required. Once write access is obtained, the attack is trivially repeatable: uploading a 4-byte file with an unsupported version integer (e.g., `\x00\x00\x00\x07`) to the data file path for each CONSENSUS_REACHED node reliably triggers `InvalidStreamFileException`. The attack persists until an operator manually removes the malformed files and restarts the importer.

## Recommendation
1. **Catch parsing exceptions explicitly:** Add `InvalidStreamFileException` and `StreamFileReaderException` to the named multi-catch at line 363 so they receive the same `warn`-and-continue treatment as `FileOperationException`, making the behavior intentional and visible.
2. **Limit retries on persistent parse failures:** Track consecutive parse failures per batch filename. After a configurable threshold (e.g., 3 attempts), throw a fatal exception or disable the downloader rather than retrying indefinitely.
3. **Alert on stall:** Expose a metric or health indicator that fires when `lastStreamFile` has not advanced within a configurable time window, enabling operator alerting before the node falls significantly behind.
4. **Validate data file integrity before parsing:** Where possible, verify the file hash against the consensus signature hash *before* passing bytes to the reader, so a structurally invalid file is rejected at the hash-check stage rather than crashing the parser.

## Proof of Concept
1. Obtain write access to the cloud storage bucket (e.g., via a publicly writable bucket or misconfigured ACL).
2. For each CONSENSUS_REACHED node, replace the current pending data file (e.g., `2024-01-01T00_00_00.000000000Z.rcd`) with a 4-byte file containing `\x00\x00\x00\x07` (version 7, unsupported).
3. Wait for the importer's next scheduled invocation.
4. Observe in logs: repeated `log.error("Error downloading data file from node ...")` entries followed by `"None of the data files could be verified"` — with no advancement of `lastStreamFile`.
5. Confirm stall: `getStartAfterFilename()` returns the same filename on every subsequent invocation, and no new stream files are processed.

```java
    private boolean verifySignatures(Collection<StreamFileSignature> signatures, StreamFilename earliestFilename) {
        Instant endDate = importerProperties.getEndDate();

        for (var signature : signatures) {
            // Ignore signatures that didn't validate or weren't in the majority
            if (signature.getStatus() != StreamFileSignature.SignatureStatus.CONSENSUS_REACHED) {
                continue;
            }

            var nodeId = signature.getNode().getNodeId();

            try {
                var dataFilename = signature.getDataFilename();
                var node = signature.getNode();
                var streamFileData = Objects.requireNonNull(
                        streamFileProvider.get(dataFilename).block());
                T streamFile = streamFileReader.read(streamFileData);

                verify(streamFile, signature);

                var archiveDestinationFolder = importerProperties.getArchiveDestinationFolderPath(streamFileData);

                if (downloaderProperties.isWriteFiles()) {
                    Utility.archiveFile(streamFileData.getFilePath(), streamFile.getBytes(), archiveDestinationFolder);
                }

                if (downloaderProperties.isWriteSignatures()) {
                    signatures.forEach(s -> Utility.archiveFile(
                            s.getFilename().getBucketFilePath(), s.getBytes(), archiveDestinationFolder));
                }

                if (!downloaderProperties.isPersistBytes()) {
                    streamFile.setBytes(null);
                }

                if (dataFilename.getInstant().isAfter(endDate)) {
                    downloaderProperties.setEnabled(false);
                    log.warn("Disabled polling after downloading all files <= endDate ({})", endDate);
                    return false;
                }

                onVerified(streamFileData, streamFile);
                return true;
            } catch (FileOperationException | HashMismatchException | TransientProviderException e) {
                final var previous =
                        lastStreamFile.get().map(StreamFile::getName).orElse("None");
                log.warn(
                        "Failed processing signatures after {} from node {} corresponding to {}. Earliest failure in batch is {}. {}",
                        previous,
                        nodeId,
                        signature.getFilename(),
                        earliestFilename,
                        e.getMessage());
            } catch (Exception e) {
                log.error(
                        "Error downloading data file from node {} corresponding to {}. Will retry another node",
                        nodeId,
                        signature.getFilename(),
                        e);
            }
        }

        return false;
    }
```
