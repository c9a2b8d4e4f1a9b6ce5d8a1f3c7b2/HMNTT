## Audit Report

## Title
Unauthenticated `estimate=true` + Historical `block` Combination Triggers Up-to-21x DB-Load Amplification via Binary Gas Estimator

## Summary
Any unauthenticated caller can POST to `/api/v1/contracts/call` with both `estimate=true` and a historical `block` number. The controller passes both fields through without cross-field validation, causing `ContractExecutionService.estimateGas()` to invoke `BinaryGasEstimator.search()`, which performs up to 21 EVM executions (1 initial + up to 20 binary-search iterations), each querying expensive historical-state SQL (`_history` tables with CTEs). The throttle is consumed only once per request, so the attacker receives up to 21× the database load for the cost of a single throttle token.

## Finding Description

**[1] `constructServiceParameters()` blindly combines both fields without cross-field validation:**

`ContractController.constructServiceParameters()` sets `callType` from `request.isEstimate()` and `block` from `request.getBlock()` independently, with no guard preventing their combination: [1](#0-0) 

**[2] `ContractCallRequest` has no cross-field validation:**

`ContractCallRequest` only validates `hasFrom()` and `hasTo()`. There is no `@AssertTrue` or any other constraint that rejects `estimate=true` when `block` is a historical (non-latest) value: [2](#0-1) 

**[3] `processCall()` unconditionally branches into `estimateGas()` when `isEstimate=true`:** [3](#0-2) 

**[4] `estimateGas()` performs one initial `callContract()` call, then unconditionally invokes `binaryGasEstimator.search()`:** [4](#0-3) 

**[5] `BinaryGasEstimator.search()` loops up to `maxGasEstimateRetriesCount` (default 20) times:** [5](#0-4) [6](#0-5) 

**[6] The throttle is consumed exactly once before the entire estimate operation:** [7](#0-6) 

**[7] `doProcessCall()` explicitly skips `restoreGasToBucket()` and metric updates when `estimate=true`, making binary-search iterations invisible to the throttle:** [8](#0-7) 

**[8] Historical block context is preserved across all binary-search iterations:**

`callContract()` sets a memoized `blockSupplier` on the context using `recordFileService.findByBlockType(params.getBlock())`. `ContractCallContext.reset()` — called at the start of each binary-search iteration — only clears `writeCache`, not `blockSupplier`: [9](#0-8) [10](#0-9) 

This means every binary-search iteration executes against historical state, triggering expensive CTE queries over `_history` tables (e.g., `nft_allowance_history`, `token_history`): [11](#0-10) [12](#0-11) 

**[9] The test layer itself acknowledges this combination is unintended:**

`TestWeb3jService.ethCall()` explicitly skips estimate-gas logic for historical blocks with the comment *"estimate gas is not supported for historical blocks"*, but no equivalent guard exists in the production `ContractController`: [13](#0-12) 

## Impact Explanation

Each malicious request causes up to 21 EVM executions (1 initial + up to 20 binary-search iterations) against historical DB state. Historical queries use multi-table CTEs over `*_history` tables with `timestamp_range` GiST index scans, making each iteration substantially more expensive than a current-state call. An attacker sending concurrent requests at the throttle rate amplifies effective DB load by up to 21× compared to what the throttle was designed to permit. This can degrade historical-state availability for legitimate users and potentially cause DB connection exhaustion or query queue saturation.

## Likelihood Explanation

No authentication, API key, or special privilege is required. The endpoint is publicly accessible. The attack requires only a valid historical block number, which is freely obtainable from the public `/api/v1/blocks` endpoint. The amplification factor is deterministic and bounded only by `maxGasEstimateRetriesCount` (default 20), a server-side configuration value not exposed to the attacker.

## Recommendation

1. **Add cross-field validation** in `ContractCallRequest` (or in `constructServiceParameters()`) to reject requests where `estimate=true` and `block` resolves to a historical (non-latest) value. This aligns with the existing test-layer assumption that gas estimation is not supported for historical blocks.
2. **Alternatively**, if historical gas estimation is intentionally supported, update the throttle model to account for the full cost of binary-search iterations (e.g., charge `maxGasEstimateRetriesCount` throttle tokens upfront for historical estimate requests, or restore tokens per binary-search iteration).
3. **Consider resetting `blockSupplier`** in `ContractCallContext.reset()` if historical estimate support is added, to ensure each iteration re-resolves the block consistently.

## Proof of Concept

```bash
# Obtain a valid historical block number
BLOCK=$(curl -s https://<mirror-node>/api/v1/blocks?limit=1&order=asc | jq '.blocks[0].number')

# Send a single unauthenticated request that triggers up to 21 historical EVM executions
curl -X POST https://<mirror-node>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{
    "from": "0x0000000000000000000000000000000000000001",
    "to":   "0x0000000000000000000000000000000000000002",
    "gas":  15000000,
    "estimate": true,
    "block": "'$BLOCK'"
  }'
```

This single request, consuming one throttle token, causes `ContractExecutionService.estimateGas()` to invoke `callContract()` once (initial probe) and then `BinaryGasEstimator.search()` for up to 20 additional iterations, each executing against historical DB state via `_history` table CTEs — yielding up to 21× the DB load the throttle was designed to permit.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
