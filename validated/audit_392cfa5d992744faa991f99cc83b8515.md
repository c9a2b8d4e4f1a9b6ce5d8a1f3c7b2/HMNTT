# Audit Report

## Title
Unauthenticated HTTP Fetch of gRPC Network Topology Enables MITM Node Substitution in Pinger

## Summary
When `HIERO_MIRROR_PINGER_NETWORK=other`, the pinger bootstraps its entire gRPC node map by issuing a plain HTTP GET to the configured `HIERO_MIRROR_PINGER_REST` URL. The response is decoded and trusted without TLS enforcement, signature verification, or allowlist validation. A network-positioned attacker who intercepts or spoofs this HTTP response can inject arbitrary gRPC endpoints, causing every subsequent `cryptoTransfer` to be delivered to an attacker-controlled server.

## Finding Description
**Code path:**

`pinger/sdk_client.go` lines 17‚Äì22: when `cfg.network == "other"`, `buildNetworkFromMirrorNodes` is called and its return value is passed directly to `hiero.ClientForNetwork(netmap)` with no further validation. [1](#0-0) 

`pinger/mirror_node_client.go` lines 74‚Äì98: `fetchMirrorNodeNetwork` constructs a plain `http.Client` GET request to the configured URL and JSON-decodes the response body into `nodesEnvelope` with no TLS requirement, no certificate pinning, and no response authentication. [5](#0-4) 

`pinger/mirror_node_client.go` lines 100‚Äì131: the network map is built entirely from attacker-controllable `ip_address_v4`/`domain_name` and `port` fields. The sole guard (`len(network) == 0`) only rejects an empty response; it does not validate addresses against any known-good set. [6](#0-5) 

`pinger/config.go` line 37: the documented example URL is `http://mirror-rest:5551` ‚Äî plain HTTP ‚Äî and no code enforces HTTPS on the `mirrorRest` field. [3](#0-2) 

**Root cause:** The network topology bootstrap is treated as trusted data despite being fetched over an unauthenticated, unencrypted channel. The `nodesEnvelope` struct accepts arbitrary `ip_address_v4`/`domain_name` and `port` values with no constraints. [7](#0-6) 

## Impact Explanation
An attacker's gRPC server receives every signed `CryptoTransfer` transaction submitted by the pinger:

- **Transaction interception / replay suppression:** The attacker can silently drop all transactions, causing the pinger to report failures while the real network sees nothing ‚Äî completely defeating its liveness-monitoring purpose.
- **Fake receipt injection:** The attacker can return a synthetic `SUCCESS` receipt, masking the fact that no real transfer occurred.
- **Signed transaction exfiltration:** The attacker obtains valid, time-bounded signed transaction bytes and can forward them selectively or at a chosen time within the validity window, controlling when (or whether) the transfer settles on-chain.

Direct theft of HBAR is not possible because the transaction body is already signed with fixed transfer parties and amounts. However, the integrity of the pinger's monitoring function is completely destroyed and the operator's signed transactions are exposed to an untrusted third party.

## Likelihood Explanation
The attack requires network-path positioning between the pinger pod and the mirror REST service. This is realistic in:

- Shared Kubernetes clusters where a compromised sidecar or CNI plugin can intercept pod-to-pod HTTP traffic.
- Cloud environments where an attacker controls a load balancer, proxy, or DNS record for the mirror service hostname.
- DNS poisoning of the `mirror-rest` hostname (the default example uses a bare hostname, not an IP).

No credentials or privileges on the pinger host itself are required. The attack is repeatable on every pinger restart (the network map is fetched once at startup) and is silent ‚Äî the pinger logs no TLS or certificate errors because none are expected.

## Recommendation
1. **Enforce HTTPS:** Reject any `mirrorRest` URL that does not use the `https` scheme at config-load time in `loadConfig()`. [8](#0-7) 
2. **Pin or validate the CA:** Use a custom `tls.Config` with a pinned CA or system trust store in the `http.Client` constructed in `buildNetworkFromMirrorNodes`, rather than the default (which accepts any valid cert for the hostname). [9](#0-8) 
3. **Allowlist returned addresses:** After fetching the node list, validate each resolved IP against a configurable allowlist of known Hedera node CIDRs before adding it to the network map.
4. **Periodic re-fetch with consistency check:** If the topology is re-fetched at runtime, compare it against the previously known topology and alert/abort on unexpected changes.

## Proof of Concept
1. Attacker intercepts the HTTP GET to `<mirrorRest>/api/v1/network/nodes` (e.g., via ARP spoofing, DNS poisoning, or a compromised Kubernetes CNI).
2. Attacker returns a crafted JSON body:
   ```json
   {"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"ip_address_v4":"attacker.example.com","port":50211}]}]}
   ```
3. `fetchMirrorNodeNetwork` (lines 100‚Äì124) builds `network["attacker.example.com:50211"] = AccountID{0,0,3}` and returns it. [10](#0-9) 
4. `hiero.ClientForNetwork(netmap)` (line 22) creates a client whose sole gRPC target is the attacker's server. [11](#0-10) 
5. Every `cryptoTransfer.Execute(client)` in `submitWithRetry` sends fully-signed transaction bytes to the attacker's gRPC endpoint instead of the real Hedera network.

```go
func fetchMirrorNodeNetwork(
	ctx context.Context,
	httpClient *http.Client,
	url string,
) (map[string]hiero.AccountID, bool, error) {
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		return nil, false, err
	}

	resp, err := httpClient.Do(req)
	if err != nil {
		return nil, true, fmt.Errorf("GET %s failed: %w", url, err)
	}
	defer resp.Body.Close()

	if resp.StatusCode < 200 || resp.StatusCode >= 300 {
		retry := resp.StatusCode == http.StatusTooManyRequests || resp.StatusCode >= 500
		return nil, retry, fmt.Errorf("GET %s returned %s", url, resp.Status)
	}

	var payload nodesEnvelope
	if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
		return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
	}

	network := make(map[string]hiero.AccountID)

	for _, n := range payload.Nodes {
		if n.NodeAccountID == "" {
			continue
		}

		nodeAccountId, err := hiero.AccountIDFromString(n.NodeAccountID)
		if err != nil {
			continue
		}

		// Use service_endpoints for node gRPC (what the SDK wants)
		for _, ep := range n.ServiceEndpoints {
			host := strings.TrimSpace(ep.DomainName)
			if host == "" {
				host = strings.TrimSpace(ep.IPAddressV4)
			}
			if host == "" || ep.Port == 0 {
				continue
			}

			addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
			network[addr] = nodeAccountId
		}
	}

	if len(network) == 0 {
		return nil, false, fmt.Errorf("no usable service_endpoints found from %s", url)
	}

	return network, false, nil
```
