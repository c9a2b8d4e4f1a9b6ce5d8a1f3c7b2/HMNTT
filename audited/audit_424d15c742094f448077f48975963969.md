### Title
Unauthenticated, Rate-Limit-Free `getNodes()` gRPC Endpoint Enables DoS via Address Book Exhaustion

### Summary
The `getNodes()` endpoint in `NetworkController.java` accepts requests from any unauthenticated caller with no application-level rate limiting. The only concurrency control — `maxConcurrentCallsPerConnection = 5` — is scoped per HTTP/2 connection and is trivially bypassed by opening many parallel connections. Each call drives multiple synchronous database queries, allowing an attacker to exhaust the DB connection pool and thread executor, denying legitimate clients access to the address book.

### Finding Description

**Exact code path:**

`NetworkController.getNodes()` (lines 33–43) immediately delegates to `networkService::getNodes` with no authentication check, no caller identity check, and no rate-limiting gate:

```java
public void getNodes(final AddressBookQuery request, final StreamObserver<NodeAddress> responseObserver) {
    final var disposable = Mono.fromCallable(() -> toFilter(request))
            .flatMapMany(networkService::getNodes)   // DB hit on every call
            ...
``` [1](#0-0) 

`NetworkServiceImpl.getNodes()` (lines 55–77) performs at minimum three synchronous DB queries per invocation: `addressBookRepository.findLatestTimestamp()`, `nodeStakeRepository.findLatestTimestamp()`, and `nodeStakeRepository.findAllStakeByConsensusTimestamp()`, followed by paginated `addressBookEntryRepository` reads inside a transaction: [2](#0-1) 

**The only concurrency control** is `maxConcurrentCallsPerConnection = 5` (default), applied per HTTP/2 connection in `GrpcConfiguration`: [3](#0-2) [4](#0-3) 

This is a per-connection limit. An attacker opening N connections gets N×5 concurrent calls. There is no global connection cap configured.

**No interceptors exist** in the grpc module for authentication or rate limiting. The `config/` directory contains only `CacheConfiguration`, `GrpcConfiguration`, `GrpcHealthIndicator`, `MetricsConfiguration`, `NettyProperties`, `RedisConfiguration`, and `RuntimeHintsConfiguration` — none of which implement request throttling or caller authentication:


**The web3 module has Bucket4j-based rate limiting** (`ThrottleConfiguration.java`) but this is entirely absent from the grpc module: [5](#0-4) 

**The infrastructure-level GCP backend policy** sets `maxRatePerEndpoint: 250`, but:
1. It is gated on `gateway.gcp.enabled: true` (not the default)
2. The comment explicitly states it "Requires a change to HPA to take effect"
3. HPA is disabled by default (`hpa.enabled: false`) [6](#0-5) [7](#0-6) 

### Impact Explanation

An attacker flooding `getNodes()` with concurrent connections exhausts:
- **HikariCP DB connection pool** — each call holds a read transaction while paginating; pool saturation causes all subsequent DB operations (including for legitimate subscribers) to queue and time out
- **Bounded thread executor** — the `applicationTaskExecutor` shared across all gRPC calls becomes saturated
- **Memory** — each in-flight `AddressBookContext` + `nodeStakeMap` allocation accumulates

Legitimate clients that cannot retrieve a fresh address book fall back to cached/stale node addresses. Stale addresses may point to decommissioned nodes (causing transaction failures) or, in a targeted attack combined with DNS/BGP manipulation, to adversary-controlled nodes (enabling transaction interception). The `GrpcHighDBConnections` Prometheus alert threshold is 75% utilization — by the time it fires, service is already degraded: [8](#0-7) 

### Likelihood Explanation

**Preconditions:** None. The gRPC port (5600) is publicly exposed via Ingress for the `NetworkService` path. No credentials, tokens, or prior access are required. [9](#0-8) 

**Feasibility:** A single attacker machine can open thousands of HTTP/2 connections using standard gRPC client libraries (e.g., `grpcurl`, Python `grpc`, Go `google.golang.org/grpc`). Each connection spawns 5 concurrent streaming calls. 200 connections = 1,000 simultaneous DB-hitting calls, far exceeding typical HikariCP pool sizes. The attack is stateless, requires no authentication, and is trivially repeatable.

### Recommendation

1. **Application-level rate limiting:** Add a gRPC `ServerInterceptor` using Bucket4j (already a dependency in the web3 module) to enforce per-IP or global request rate limits on `NetworkService/getNodes`, mirroring the pattern in `ThrottleConfiguration.java`.
2. **Global connection limit:** Configure `NettyServerBuilder.maxConnectionAge()`, `maxConnectionIdle()`, and a global `maxConcurrentCallsPerConnection` ceiling that accounts for total connections, not just per-connection.
3. **Enable and correctly configure the GCP backend rate limit:** Ensure `maxRatePerEndpoint` is active and HPA is configured so the infrastructure-level throttle is enforced unconditionally.
4. **Enforce a mandatory `limit` field:** Currently `limit=0` maps to `Long.MAX_VALUE` entries returned. Enforce a server-side maximum (e.g., 1000) regardless of client input. [10](#0-9) 

### Proof of Concept

```bash
# Install grpcurl: https://github.com/fullstorydev/grpcurl
# Target: publicly exposed gRPC mirror node at <HOST>:5600

# Single call to confirm no auth required
grpcurl -plaintext -d '{"fileId": {"fileNum": 102}}' \
  <HOST>:5600 com.hedera.mirror.api.proto.NetworkService/getNodes

# Flood: open 500 parallel connections, each with a streaming call (no limit set = Long.MAX_VALUE)
for i in $(seq 1 500); do
  grpcurl -plaintext -d '{"fileId": {"fileNum": 102}}' \
    <HOST>:5600 com.hedera.mirror.api.proto.NetworkService/getNodes &
done
wait

# Expected result: DB connection pool exhaustion visible in metrics:
# hikaricp_connections_active / hikaricp_connections_max > 0.75
# Legitimate getNodes() calls begin returning UNAVAILABLE or timing out
# Clients fall back to stale cached address book entries
```

    public Flux<AddressBookEntry> getNodes(AddressBookFilter filter) {
        var fileId = filter.getFileId();
        if (!getValidFileIds().contains(fileId)) {
            throw new IllegalArgumentException(INVALID_FILE_ID);
        }

        long addressBookTimestamp = addressBookRepository
                .findLatestTimestamp(fileId.getId())
                .orElseThrow(() -> new EntityNotFoundException(fileId));
        long nodeStakeTimestamp = nodeStakeRepository.findLatestTimestamp().orElse(NODE_STAKE_EMPTY_TABLE_TIMESTAMP);
        var nodeStakeMap = nodeStakeRepository.findAllStakeByConsensusTimestamp(nodeStakeTimestamp);
        var context = new AddressBookContext(addressBookTimestamp, nodeStakeMap);

        return Flux.defer(() -> page(context))
                .repeatWhen(RepeatSpec.create(c -> !context.isComplete(), Long.MAX_VALUE)
                        .jitter(0.5)
                        .withFixedDelay(addressBookProperties.getPageDelay())
                        .withScheduler(Schedulers.boundedElastic()))
                .take(filter.getLimit() > 0 ? filter.getLimit() : Long.MAX_VALUE)
                .doOnNext(context::onNext)
                .doOnSubscribe(s -> log.info("Querying for address book: {}", filter))
                .doOnComplete(() -> log.info("Retrieved {} nodes from the address book", context.getCount()));
    }

@Configuration(proxyBeanMethods = false)
@RequiredArgsConstructor
public class ThrottleConfiguration {

    public static final String GAS_LIMIT_BUCKET = "gasLimitBucket";
    public static final String RATE_LIMIT_BUCKET = "rateLimitBucket";
    public static final String OPCODE_RATE_LIMIT_BUCKET = "opcodeRateLimitBucket";

    private final ThrottleProperties throttleProperties;

    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    @Bean(name = GAS_LIMIT_BUCKET)
    Bucket gasLimitBucket() {
        long gasLimit = throttleProperties.getGasPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(gasLimit)
                .refillGreedy(gasLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder()
                .withSynchronizationStrategy(SynchronizationStrategy.SYNCHRONIZED)
                .addLimit(limit)
                .build();
    }

    @Bean(name = OPCODE_RATE_LIMIT_BUCKET)
    Bucket opcodeRateLimitBucket() {
        long rateLimit = throttleProperties.getOpcodeRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }
