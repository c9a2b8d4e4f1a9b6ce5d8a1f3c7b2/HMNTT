### Title
Unauthenticated CPU Exhaustion DoS via `/construction/combine` with Invalid Signature Type

### Summary
The `ConstructionCombine` handler in the Hiero Rosetta service performs multiple expensive protobuf serialization/deserialization operations before reaching the `ErrInvalidSignatureType` check. No rate limiting exists on any Rosetta endpoint. An unprivileged attacker can flood the endpoint with crafted requests bearing a valid transaction payload but an unsupported signature type, exhausting CPU resources and degrading or denying service to legitimate users.

### Finding Description
**Exact code path:**

`rosetta/app/services/construction_service.go`, `ConstructionCombine`, lines 54–97:

```
1. len(request.Signatures) == 0  → cheap, O(1)
2. unmarshallTransactionFromHexString(request.UnsignedTransaction)
      → hex.DecodeString                    (line 659)
      → hiero.TransactionFromBytes          (protobuf binary deserialization, line 664)
      → isSupportedTransactionType          (type switch, line 669)
3. getFrozenTransactionBodyBytes(transaction)
      → hiero.TransactionString(transaction) (serialize to text proto, line 650)
      → prototext.Unmarshal(...)             (text proto deserialization, line 651)
4. for _, signature := range request.Signatures {
       if signature.SignatureType != rTypes.Ed25519 {
           return nil, errors.ErrInvalidSignatureType   ← check happens HERE
       }
```

Steps 2 and 3 each involve full protobuf round-trips (binary deserialization + text-proto serialization + text-proto deserialization) before the signature type is ever inspected.

**Root cause / failed assumption:** The implementation assumes that invalid requests will be rare or that an upstream layer will throttle them. Neither assumption holds. The middleware stack applied in `main.go` (lines 217–219) is exclusively `MetricsMiddleware` → `TracingMiddleware` → `CorsMiddleware` — none of which perform rate limiting. The throttling infrastructure that exists in the codebase (`web3/src/main/java/org/hiero/mirror/web3/config/ThrottleConfiguration.java`) belongs to the Java web3 module and is entirely absent from the Go Rosetta service.

**Why existing checks are insufficient:**
- `http.Server` timeouts (`ReadTimeout`, `WriteTimeout`, etc.) limit per-connection duration but do not bound the number of concurrent connections or requests per second.
- The `rosettaAsserter` validates Rosetta protocol structure but does not rate-limit.
- No IP-based throttle, token bucket, or connection semaphore exists anywhere in the Rosetta Go service.

### Impact Explanation
Each malicious request forces the server to: hex-decode an attacker-controlled byte blob, binary-deserialize a protobuf transaction, re-serialize it to text proto, and then text-proto-deserialize it again — four non-trivial CPU operations — before returning a 500-byte JSON error. At sustained high request rates (thousands/second from a single host or botnet), this saturates available CPU cores, causing legitimate `/construction/combine`, `/construction/submit`, and other Rosetta calls to time out or queue indefinitely. Because the Rosetta API is used by exchanges and custodians to construct and submit Hedera transactions, a sustained attack degrades the ability of those integrators to operate, directly impacting network liquidity and transaction throughput.

### Likelihood Explanation
Preconditions are minimal: the attacker needs only a valid hex-encoded protobuf `TransferTransaction` or `AccountCreateTransaction` (both are fully documented in the Hiero SDK and Rosetta spec) and the ability to send HTTP POST requests. No account, key, or privileged access is required. The attack is trivially scriptable with `curl` or any HTTP load tool (e.g., `wrk`, `hey`, `k6` — the project itself ships a k6 test at `tools/k6/src/rosetta/test/constructionCombine.js` that demonstrates the exact request shape). The attack is repeatable and stateless; each request is independent.

### Recommendation
1. **Add a token-bucket rate limiter** in the Rosetta Go service, applied as HTTP middleware before routing, analogous to the `ThrottleConfiguration` / `ThrottleManagerImpl` pattern already used in the web3 module. Limit requests per IP and globally per endpoint.
2. **Move the signature type check before the expensive deserialization steps.** The `SignatureType` field is available directly on `request.Signatures[i]` without any transaction parsing; validate it first and return `ErrInvalidSignatureType` immediately if any signature is not `Ed25519`, before calling `unmarshallTransactionFromHexString` or `getFrozenTransactionBodyBytes`.
3. **Cap request body size** at the HTTP server level to prevent oversized transaction payloads from amplifying per-request cost.
4. **Set `http.Server.MaxHeaderBytes`** and consider `http.Server` connection limits or a reverse-proxy (e.g., nginx) with `limit_req` in front of the Rosetta port.

func (c *constructionAPIService) ConstructionCombine(
	_ context.Context,
	request *rTypes.ConstructionCombineRequest,
) (*rTypes.ConstructionCombineResponse, *rTypes.Error) {
	if len(request.Signatures) == 0 {
		return nil, errors.ErrNoSignature
	}

	transaction, rErr := unmarshallTransactionFromHexString(request.UnsignedTransaction)
	if rErr != nil {
		return nil, rErr
	}

	frozenBodyBytes, rErr := getFrozenTransactionBodyBytes(transaction)
	if rErr != nil {
		return nil, rErr
	}

	for _, signature := range request.Signatures {
		if signature.SignatureType != rTypes.Ed25519 {
			return nil, errors.ErrInvalidSignatureType
		}

		pubKey, err := hiero.PublicKeyFromBytes(signature.PublicKey.Bytes)
		if err != nil {
			return nil, errors.ErrInvalidPublicKey
		}

		if !ed25519.Verify(pubKey.Bytes(), frozenBodyBytes, signature.Bytes) {
			return nil, errors.ErrInvalidSignatureVerification
		}

		_, _ = hiero.TransactionAddSignature(transaction, pubKey, signature.Bytes)
	}

	transactionBytes, err := hiero.TransactionToBytes(transaction)
	if err != nil {
		return nil, errors.ErrTransactionMarshallingFailed
	}

	return &rTypes.ConstructionCombineResponse{
		SignedTransaction: tools.SafeAddHexPrefix(hex.EncodeToString(transactionBytes)),
	}, nil
}

func getFrozenTransactionBodyBytes(transaction hiero.TransactionInterface) ([]byte, *rTypes.Error) {
	signedTransaction := services.SignedTransaction{}
	data, _ := hiero.TransactionString(transaction)
	if err := prototext.Unmarshal([]byte(data), &signedTransaction); err != nil {
		return nil, errors.ErrTransactionUnmarshallingFailed
	}

	return signedTransaction.BodyBytes, nil
}

func unmarshallTransactionFromHexString(transactionString string) (hiero.TransactionInterface, *rTypes.Error) {
	transactionBytes, err := hex.DecodeString(tools.SafeRemoveHexPrefix(transactionString))
	if err != nil {
		return nil, errors.ErrTransactionDecodeFailed
	}

	transaction, err := hiero.TransactionFromBytes(transactionBytes)
	if err != nil {
		return nil, errors.ErrTransactionUnmarshallingFailed
	}

	if rErr := isSupportedTransactionType(transaction); rErr != nil {
		return nil, rErr
	}

	return transaction, nil
}
