### Title
Unvalidated IP Addresses from Mirror REST Server Enable Network Partition via Broadcast/Multicast Injection

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` blindly accepts any non-empty string from `ServiceEndpoints[].ip_address_v4` in the mirror REST response and passes it directly to `hiero.ClientForNetwork()` without validating against reserved, broadcast, or multicast address ranges. An attacker who controls the mirror REST server response can inject `255.255.255.255` or multicast addresses (e.g., `224.0.0.1`) as node endpoints, causing the SDK client to route all consensus transactions to invalid network destinations, creating a complete network partition outside design parameters.

### Finding Description
**Code path:** `pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 113–124.

The function iterates over `payload.Nodes[].ServiceEndpoints` (line 113). For each endpoint, it selects `DomainName` first, then falls back to `IPAddressV4` (lines 114–116). The only guards are:

```go
if host == "" || ep.Port == 0 {
    continue
}
```

(line 118). There is no call to `net.ParseIP()`, no check against `ip.IsLoopback()`, `ip.IsMulticast()`, `ip.IsLinkLocalUnicast()`, `ip.IsUnspecified()`, or any broadcast sentinel (`255.255.255.255`). The raw string is passed to `net.JoinHostPort()` (line 122) and inserted into the `network` map, which is then handed to `hiero.ClientForNetwork(netmap)` in `sdk_client.go` line 22.

**Root cause:** The code assumes the mirror REST server is a trusted source of valid unicast addresses. This assumption is not enforced in code.

**Exploit flow:**
1. Attacker controls the mirror REST server response (see Likelihood section).
2. Returns a `nodesEnvelope` where every `nodeEntry.service_endpoints` entry has `"ip_address_v4": "255.255.255.255"` (or `"224.0.0.1"`) and a non-zero port (e.g., `50211`).
3. `host` = `"255.255.255.255"`, `ep.Port` = `50211` — both guards pass.
4. `addr` = `"255.255.255.255:50211"` is inserted into `network`.
5. `hiero.ClientForNetwork(network)` is called with this map.
6. All subsequent consensus gRPC calls target `255.255.255.255:50211`.

**Why existing checks fail:** The only checks (`host == ""` and `ep.Port == 0`) are syntactic, not semantic. `"255.255.255.255"` is a non-empty string with a valid port, so it passes both guards unconditionally.

### Impact Explanation
The pinger's sole purpose is to submit consensus transactions and verify liveness. If the SDK network map is poisoned with broadcast or multicast addresses, every `TransferTransaction` or `AccountBalanceQuery` issued by the pinger will be sent to an invalid destination. The pinger will report all nodes as unreachable, triggering false-positive alerts or masking a real outage. In a monitoring/SRE context this constitutes a complete denial-of-service of the health-check function and a network partition from the perspective of the pinger component. Severity: **High**.

### Likelihood Explanation
The `cfg.mirrorRest` URL is operator-configured. Attack vectors that do not require privileged access to the pinger host itself:

- **HTTP (no TLS):** If the operator configures an `http://` mirror URL, any on-path network attacker can inject a crafted JSON response — no credentials required.
- **DNS hijacking / BGP hijack:** The mirror REST hostname can be redirected to an attacker-controlled server.
- **Mirror server compromise:** A compromised mirror node (which is a separate service from the consensus nodes) is sufficient; no access to the pinger process or its config is needed.

The `http.Client` on line 46 has no TLS pinning, no certificate fingerprint check, and no HMAC/signature verification of the response body. The attack is repeatable on every pinger restart or network refresh cycle.

### Recommendation
After parsing each `IPAddressV4` string, validate it with `net.ParseIP()` and reject reserved ranges before adding to the network map:

```go
ip := net.ParseIP(host)
if ip != nil {
    if ip.IsLoopback() || ip.IsMulticast() || ip.IsLinkLocalUnicast() ||
       ip.IsUnspecified() || host == "255.255.255.255" {
        continue
    }
}
```

Additionally:
- Enforce HTTPS for `cfg.mirrorRest` and validate the server certificate.
- Reject any response where 100% of endpoints resolve to non-routable addresses (defense-in-depth).
- Consider pinning the expected IP ranges to known consensus-node CIDR blocks if the deployment topology is fixed.

### Proof of Concept
1. Stand up a local HTTP server that returns:
```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{"ip_address_v4": "255.255.255.255", "port": 50211}]
  }],
  "links": {}
}
```
2. Configure the pinger with `MIRROR_REST=http://localhost:8080` and `NETWORK=other`.
3. Start the pinger. `buildNetworkFromMirrorNodes()` returns `{"255.255.255.255:50211": AccountID{0,0,3}}` without error.
4. `hiero.ClientForNetwork()` is called with this map.
5. All consensus calls (e.g., `AccountBalanceQuery`) time out or error — the pinger is fully partitioned from the real network.
6. Repeat with `"224.0.0.1"` (multicast) to confirm the same path is taken.
