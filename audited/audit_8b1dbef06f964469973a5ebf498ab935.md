### Title
Unauthenticated `/metrics` Endpoint Exposes `status_code` Labels Revealing Hashgraph History Gaps

### Summary
The Rosetta `/metrics` endpoint is registered with `promhttp.Handler().ServeHTTP` and no authentication wrapper, making it publicly accessible to any unprivileged user. The `hiero_mirror_rosetta_request_duration` histogram records every request with `route` and `status_code` labels, so an attacker can read the full Prometheus dump and observe which Rosetta API routes (e.g., `/block`, `/block/transaction`) are returning 404 errors, directly inferring gaps in the mirror node's Hashgraph history.

### Finding Description

**Exact code path:**

`rosetta/app/middleware/metrics.go`, `Routes()`, lines 64–73: [1](#0-0) 

The route handler is `promhttp.Handler().ServeHTTP` — a bare Prometheus HTTP handler with zero authentication logic.

The histogram that leaks status codes is defined at lines 28–32: [2](#0-1) 

Labels `["method", "route", "status_code", "ws"]` are recorded for every request. The `route` label carries the API path template (e.g., `/block`, `/block/transaction`) and `status_code` carries the HTTP response code (e.g., `404`).

The endpoint is wired into the live router with no middleware guard in `rosetta/main.go`: [3](#0-2) 

**Root cause / failed assumption:** The code assumes `/metrics` is only reachable by internal Prometheus scrapers (e.g., behind a network boundary), but no enforcement of that assumption exists in the application layer. The REST API's metrics handler (`rest/middleware/metricsHandler.js`) correctly implements an `authenticate()` check before serving metrics: [4](#0-3) 

No equivalent guard exists anywhere in the Rosetta metrics path.

**Exploit flow:**

1. Attacker sends `GET /metrics` with no credentials to the Rosetta port.
2. Server responds `200 OK` with the full Prometheus text exposition.
3. Attacker reads `hiero_mirror_rosetta_request_duration_count{route="/block", status_code="404", ...}` and `hiero_mirror_rosetta_request_duration_count{route="/block/transaction", status_code="404", ...}`.
4. By polling `/metrics` repeatedly, the attacker builds a time-series of 404 rates per route, revealing when and how frequently block/transaction lookups fail — i.e., which time windows have gaps in the mirror node's ingested Hashgraph history.
5. The attacker then uses this signal to target specific block-index ranges with direct Rosetta API calls to enumerate exact missing blocks.

### Impact Explanation

An unprivileged external user can passively monitor the operational state of the mirror node's Hashgraph history without making any block queries themselves. The `status_code=404` rate on `/block` and `/block/transaction` routes is a direct indicator of history gaps. This constitutes information disclosure about the integrity and completeness of the mirror node's ledger data, which could be used to time attacks, identify sync failures, or undermine trust in the node's data.

### Likelihood Explanation

The Rosetta API is designed to be publicly accessible (it is a blockchain data API). Any user who can reach the Rosetta port — which is the intended audience — can also reach `/metrics` with a single unauthenticated GET request. No special tooling, credentials, or network position is required. The attack is trivially repeatable and scriptable.

### Recommendation

Add an authentication wrapper to the `/metrics` route handler in `Routes()`, mirroring the pattern used by the REST API's `metricsHandler.js`. At minimum, restrict the endpoint to localhost or a configurable IP allowlist via middleware, or require HTTP Basic Auth credentials checked before delegating to `promhttp.Handler()`.

### Proof of Concept

```bash
# No credentials required
curl -s http://<rosetta-host>:<port>/metrics | grep 'hiero_mirror_rosetta_request_duration'

# Example output revealing 404s on /block route:
# hiero_mirror_rosetta_request_duration_count{application="rosetta",method="POST",route="/block",status_code="404",ws="false"} 37
# hiero_mirror_rosetta_request_duration_count{application="rosetta",method="POST",route="/block/transaction",status_code="404",ws="false"} 12

# Poll repeatedly to build a time-series of gap activity:
watch -n 10 'curl -s http://<rosetta-host>:<port>/metrics | grep status_code=\"404\"'
```
