### Title
Throttle Token Leak via Odd-Length Hex `data` Field Bypassing `@Hex` Validator

### Summary
The `@Hex` validator on `ContractCallRequest.data` uses a regex that permits odd-length hex strings (e.g., `"0x1"`, `"0xabc"`). When such a value reaches `constructServiceParameters()`, the `hexToBytes()` call at line 80 sits **outside** the try/catch block and throws `IllegalArgumentException`. The `call()` method only restores throttle tokens on `InvalidParametersException`, so every such request permanently consumes throttle capacity without performing any useful work.

### Finding Description

**`@Hex` validator does not enforce even-length:** [1](#0-0) 

The compiled pattern is `^(0x)?[0-9a-fA-F]{0,1048576}$`. It counts hex characters but imposes no parity constraint, so `"0x1"`, `"0xabc"`, `"1"`, `"abc"` all pass.

**`data` field uses this validator with defaults:** [2](#0-1) 

**The try/catch in `constructServiceParameters()` wraps only the trivial getter, not `hexToBytes()`:** [3](#0-2) 

`request.getData()` is a plain Lombok getter that never throws. The catch block is dead code for its stated purpose. The actual dangerous call is at line 80: [4](#0-3) 

**`hexToBytes()` throws `IllegalArgumentException` for odd-length input:** [5](#0-4) 

`org.apache.commons.codec.binary.Hex.decodeHex` rejects odd-length strings with `DecoderException`, which is re-wrapped as `IllegalArgumentException`.

**`call()` only restores tokens for `InvalidParametersException`:** [6](#0-5) 

`throttleManager.throttle(request)` runs at line 40. The `IllegalArgumentException` from `hexToBytes()` propagates past the catch at line 46 (which only handles `InvalidParametersException`), so `throttleManager.restore(request.getGas())` is never called. The existing test at `ContractControllerTest.java:476-485` confirms the 400 response but does not assert token restoration.

### Impact Explanation
An attacker can repeatedly send requests with odd-length `data` values (e.g., `"0x1"`) to drain the throttle bucket without triggering any real EVM execution. Once the throttle is exhausted, legitimate callers are rate-limited or rejected, constituting a denial-of-service against the `/api/v1/contracts/call` endpoint. No authentication or special privilege is required.

### Likelihood Explanation
The attack requires only an HTTP POST with a crafted JSON body — zero privileges, no account, no on-chain state. The odd-length hex bypass is trivially discoverable by reading the validator regex or by fuzzing the `data` field. It is fully repeatable and automatable in a tight loop.

### Recommendation
1. **Fix the validator** to enforce even-length hex character counts (excluding the `0x` prefix):
   Change the pattern to `^(0x)?([0-9a-fA-F]{2})*$` (or `^(0x)?[0-9a-fA-F]{even}$` via a custom check).
2. **Move `hexToBytes()` inside the try/catch** (or wrap the entire `constructServiceParameters()` call in `call()`) so that any `IllegalArgumentException` is caught, converted to `InvalidParametersException`, and triggers `throttleManager.restore()`.
3. **Broaden the restore guard** in `call()` to also restore tokens on `IllegalArgumentException` (or any non-execution exception), as a defense-in-depth measure.

### Proof of Concept
```
POST /api/v1/contracts/call
Content-Type: application/json

{
  "data": "0x1",
  "to": "0x0000000000000000000000000000000000000001",
  "gas": 21000
}
```
**Preconditions:** None. Any unauthenticated HTTP client.

**Steps:**
1. Send the above request. The `@Hex` validator accepts `"0x1"` (1 hex char, matches regex).
2. `throttleManager.throttle(request)` consumes gas tokens for `gas=21000`.
3. `constructServiceParameters()` is called; the try/catch wraps only `request.getData()` (no-op).
4. `.callData(hexToBytes("0x1"))` throws `IllegalArgumentException("Odd number of characters")`.
5. The exception propagates to `call()`, which catches only `InvalidParametersException` — no match.
6. `throttleManager.restore()` is never called; 21 000 gas units of throttle capacity are permanently consumed.
7. Repeat in a loop until the throttle bucket is empty; legitimate requests begin receiving throttle-rejection responses.
