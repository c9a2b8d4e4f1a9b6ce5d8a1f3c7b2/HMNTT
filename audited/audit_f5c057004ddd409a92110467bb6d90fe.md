### Title
Unbounded Prometheus Label Cardinality via Unmatched Route Paths Enables Heap Exhaustion DoS

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to the raw `req.path` when no Express route matches a request. An unprivileged attacker can flood the server with requests to unique, non-existent paths, causing each unique path string to be registered as a distinct Prometheus label value across four per-route instruments, growing the OpenTelemetry SDK's in-memory label store without bound until the Node.js process exhausts heap memory and crashes.

### Finding Description

**Exact code path:**

In `toOpenApiPath()` ( [1](#0-0) ), when `res.locals[requestPathLabel]` is not set **and** `req.route` is falsy (i.e., no Express route matched), the function assigns `path = req.path` directly — the raw, attacker-controlled URL path string:

```js
if (!path) {
  if (!req.route) {
    path = req.path;   // ← raw attacker-controlled value
  } else { ... }
}
```

`res.locals[requestPathLabel]` is only populated by `recordRequestPath` in `rest/routes/index.js` [2](#0-1) , which itself reads `req.route?.path` — also undefined for unmatched routes. So for any request that does not match a registered route, both guards are bypassed and the raw path flows through.

That raw path is then used as the `path` label in **four** per-route instruments on every `finish` event:

```js
requestTotalCounter.add(1, labels);        // line 207
durationHistogram.record(duration, labels); // line 208
requestSizeHistogram.record(..., labels);   // line 209
responseSizeHistogram.record(..., labels);  // line 210
``` [3](#0-2) 

Each unique `{method, path, code}` label combination creates a new time-series entry in the OpenTelemetry SDK's in-memory `AttributesHashMap`. There is no cardinality cap, no label sanitization, and no rate-limiting middleware anywhere in `rest/middleware/`.

**Why existing checks fail:**

- `res.locals[requestPathLabel]` is only set for matched routes via `recordRequestPath`. [2](#0-1) 
- `req.route` is `undefined` for unmatched routes by Express design.
- The `responseHandler` throws `NotFoundError` for unmatched routes [4](#0-3) , but the `res.on('finish', ...)` callback in `metricsMiddleware` still fires after the 404 is sent, recording the raw path as a label. [5](#0-4) 
- No rate-limiting middleware was found in the codebase.

### Impact Explanation

Every unique path string permanently allocates a new label-set entry in the OpenTelemetry SDK's in-process metric store. With four instruments each holding the same label, memory grows at ~4× the rate of unique paths. Sending tens of thousands of unique paths exhausts the Node.js V8 heap, triggering an OOM crash and a complete denial of service of the REST API. Recovery requires a process restart, and the attack can be repeated immediately.

### Likelihood Explanation

The attack requires zero authentication, zero privileges, and only standard HTTP GET requests. Any internet-accessible deployment is vulnerable. The attacker needs only a script that generates unique path segments (e.g., sequential integers or UUIDs appended to any non-existent API prefix). The attack is fully repeatable and can be automated trivially.

### Recommendation

1. **Normalize unmatched paths to a sentinel value** in `toOpenApiPath`: replace the `path = req.path` fallback with a fixed string such as `"unknown"` or `"unmatched"`:
   ```js
   if (!req.route) {
     path = 'unmatched';  // never use req.path as a label
   }
   ```
2. **Enforce a cardinality cap** at the OpenTelemetry SDK level using an `AttributesProcessor` that drops or truncates label sets exceeding a configured limit.
3. **Add rate limiting** (e.g., `express-rate-limit`) as middleware before `metricsMiddleware` to bound the request rate per IP.

### Proof of Concept

```bash
# Send 50,000 requests with unique path segments to a non-existent route
for i in $(seq 1 50000); do
  curl -s "http://<target>/api/v1/nonexistent/id_${i}" &
done
wait
# Monitor heap: the Node.js process will OOM-crash as label entries accumulate
```

Each request causes `toOpenApiPath` to return `/api/v1/nonexistent/id_<N>` as the `path` label value, creating a new time-series entry in all four per-route instruments. After enough unique values, the process crashes with a JavaScript heap out-of-memory error.

const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];

  return path;
};
