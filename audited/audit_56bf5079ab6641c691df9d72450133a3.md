### Title
Uncaught `InvalidEntityException` in `SyntheticLogListener.getContractAddress()` via Empty `EntityId` Passed to `DomainUtils.toEvmAddress(EntityId)`

### Summary
When a synthetic contract log is created for a token transfer whose contract-related parent transaction record contains neither a `contractCallResult` nor a `contractCreateResult`, `contractId` is set to `EntityId.EMPTY`. The `SyntheticLogListener.SyntheticLogUpdater.getContractAddress()` fallback branch then calls `DomainUtils.toEvmAddress(contractLog.getContractId())` — which is also `EntityId.EMPTY` — unconditionally throwing `InvalidEntityException`. This exception propagates uncaught out of `onEnd()`, causing the entire record file's transaction to be rolled back and retried, effectively blocking the importer from advancing past that record file.

### Finding Description

**Step 1 — Empty `contractId` is stored in the `ContractLog`:**

In `SyntheticContractLogServiceImpl.create()`, when the parent record item exists but has neither `contractCallResult` nor `contractCreateResult`:

```java
// SyntheticContractLogServiceImpl.java:54-60
if (parentTransactionRecord.hasContractCallResult() || parentTransactionRecord.hasContractCreateResult()) {
    contractId = EntityId.of(parentTransactionRecord.getContractCallResult().getContractID());
} else {
    contractId = EntityId.EMPTY;   // ← set to EMPTY
}
...
contractLog.setContractId(contractId);  // line 85 — EMPTY stored in log
```

**Step 2 — `SyntheticLogUpdater` is created despite empty `contractId`:**

In `SyntheticLogListener.onContractLog()`, the guard only skips creation when ALL three IDs are empty:

```java
// SyntheticLogListener.java:83
if (!(EntityId.isEmpty(contractId) && EntityId.isEmpty(senderId) && EntityId.isEmpty(receiverId))) {
    final var updater = new SyntheticLogUpdater(contractId, senderId, receiverId, contractLog);
```

If `contractId` is `EMPTY` but `senderId` or `receiverId` is non-empty (valid token transfer participants), the updater is created with `contractId = EntityId.EMPTY`.

**Step 3 — `getContractAddress()` fallback calls `toEvmAddress` with `EntityId.EMPTY`:**

```java
// SyntheticLogListener.java:172-175
private byte[] getContractAddress(final EntityId contractEntityId, ...) {
    if (EntityId.isEmpty(contractEntityId)) {
        return DomainUtils.toEvmAddress(contractLog.getContractId()); // ← contractLog.getContractId() is ALSO EMPTY
    }
```

**Step 4 — `toEvmAddress(EntityId)` throws unconditionally:**

```java
// DomainUtils.java:393-396
public static byte[] toEvmAddress(EntityId contractId) {
    if (EntityId.isEmpty(contractId)) {
        throw new InvalidEntityException("Empty contractId");
    }
```

**Step 5 — Exception propagates uncaught through `onEnd()`:**

```java
// SyntheticLogListener.java:64-75
public void onEnd(RecordFile recordFile) {
    ...
    for (var updater : logUpdaters) {
        updater.updateContractLog(entityMap);  // ← throws InvalidEntityException, no catch
    }
}
```

`onEnd()` has no try-catch. The exception propagates up through `AbstractStreamFileParser` into `RecordFileParser.parse()`, which is `@Transactional` (rolling back all writes for the file) and `@Retryable` (retrying indefinitely up to max attempts).

### Impact Explanation

The uncaught `InvalidEntityException` causes the `@Transactional` record file parse to roll back, discarding all records in that file from the database. The `@Retryable` wrapper retries the same file repeatedly; since the triggering transaction is part of the immutable record stream, every retry hits the same exception. After max retries the importer halts, and all records in the affected file — and every subsequent file — are permanently absent from the mirror node's database. This is a complete, persistent denial-of-service on the importer's record processing pipeline.

### Likelihood Explanation

An unprivileged user needs only to submit a token transfer transaction (e.g., `CryptoTransfer` with fungible tokens) as a child of a contract call that the consensus node records without a `contractCallResult` or `contractCreateResult` (e.g., a call to a non-existent contract, or a call that fails before EVM execution begins). The consensus node still records the child token transfer in the record stream. This is a standard, permissionless Hedera transaction pattern requiring no special privileges. The condition is deterministic and reproducible on every retry.

### Recommendation

Fix `getContractAddress()` to handle the case where `contractLog.getContractId()` is also empty, rather than blindly delegating to `toEvmAddress`:

```java
private byte[] getContractAddress(final EntityId contractEntityId, ...) {
    if (EntityId.isEmpty(contractEntityId)) {
        var logContractId = contractLog.getContractId();
        if (EntityId.isEmpty(logContractId)) {
            return new byte[DomainUtils.EVM_ADDRESS_LENGTH]; // zero address fallback
        }
        return DomainUtils.toEvmAddress(logContractId);
    }
    ...
}
```

Alternatively, guard the `SyntheticLogUpdater` creation to skip when `contractId` is empty AND the log's `contractId` is also empty, or add a null/empty check before calling `toEvmAddress` in the fallback branch.

### Proof of Concept

1. Submit a `ContractCall` transaction to a non-existent or pre-deleted contract on Hedera mainnet/testnet. The consensus node produces a record with no `contractCallResult`.
2. Ensure the same transaction (or a child transaction) includes a fungible token transfer between two valid accounts (non-zero `topic1`/`topic2` in the resulting synthetic log).
3. The importer processes the record file containing this transaction.
4. `SyntheticContractLogServiceImpl.create()` sets `contractId = EntityId.EMPTY` (line 59).
5. `SyntheticLogListener.onContractLog()` creates a `SyntheticLogUpdater` because `senderId`/`receiverId` is non-empty (line 83-86).
6. During `onEnd()`, `updateContractLog()` → `getContractAddress(EntityId.EMPTY, ...)` → `DomainUtils.toEvmAddress(EntityId.EMPTY)` → `InvalidEntityException` thrown.
7. Exception propagates uncaught; `@Transactional` rolls back the file; `@Retryable` retries indefinitely.
8. Importer is permanently blocked on this record file; all records in it and all subsequent files are missing from the mirror node.
