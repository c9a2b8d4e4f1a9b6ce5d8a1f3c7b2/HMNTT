### Title
Unauthenticated Mirror Node REST API Used for Node Discovery Enables MITM-Based Malicious Node Injection and Operator Fund Drain

### Summary
`NodeSupplier.getAddressBook()` fetches the Hedera network node list from the mirror node REST API via `restApiClient::getNodes` with no authentication, no certificate pinning, and no integrity verification of the response. An attacker who can intercept or spoof this HTTP response can inject attacker-controlled gRPC endpoints into the client pool. Because the `certHash` used for TLS verification is sourced from the same spoofable response, and because node validation (`validateNode`) can be bypassed by a malicious gRPC server returning a fake `SUCCESS` receipt, the operator's signing key is then used to submit real transactions to attacker-controlled nodes, enabling fund drain.

**Root cause — three compounding failures:**

1. **No authentication or integrity check on the REST response.** `RestApiClient` builds a plain `WebClient` with only `Accept: application/json`. There is no API key, HMAC signature, or response signing. [4](#0-3) 

2. **`certHash` is sourced from the same spoofable response.** `toNodeProperties()` sets `certHash` directly from `networkNode.getNodeCertHash()`. If the attacker controls the REST response, they also control the cert hash that is loaded into the `NodeAddressBook` and passed to `client.setNetworkFromAddressBook()`. The attacker simply supplies the cert hash of their own TLS certificate, neutralising the only cryptographic check. [5](#0-4) 

3. **Node validation is bypassable.** `validateNode()` sends 1 tinyhbar and checks the receipt status via the *same* client that is pointed exclusively at the injected node. A malicious gRPC server under attacker control can return a well-formed `SUCCESS` receipt for any transaction, causing the node to pass validation and be added to `nodes`. [6](#0-5) 

**Why the `switchIfEmpty` fallback does not save you:** The REST API path is only skipped when `monitorProperties.getNodes()` is non-empty. In the default/recommended deployment where no static nodes are configured, the REST API is always consulted. [7](#0-6) 

### Impact Explanation

Once malicious nodes pass validation and enter the `nodes` list, `toClient()` in `TransactionPublisher` calls `client.setOperator(operatorId, operatorPrivateKey)`, binding the operator's private key to a client that routes all transactions to attacker-controlled endpoints. Every subsequent `publish()` call signs and submits real transactions (including `TransactionRecordQuery` with a 1 HBAR query payment) to the attacker's server. The attacker can collect signed transactions, replay them, or simply observe the operator's private key usage pattern. Severity: **Critical** — direct, repeatable loss of operator funds with no user interaction required after initial injection. [8](#0-7) 

### Likelihood Explanation

**Preconditions:**
- Static `nodes` list is empty (default deployment).
- `nodeValidation.retrieveAddressBook` is `true` (default).
- Attacker can intercept or spoof the mirror node REST API response (DNS poisoning, ARP spoofing on a shared cloud VPC/LAN, BGP hijack, or a compromised CDN/reverse proxy in front of the mirror node).

None of these require OS-level privilege on the monitor host. DNS spoofing and ARP poisoning are achievable by any host on the same network segment. In cloud environments (shared Kubernetes clusters, shared VPCs), this is a realistic attacker position. The attack is fully repeatable on every `refresh()` cycle (periodic via `Flux.interval` in `NodeSupplier.init()`). [9](#0-8) 

### Recommendation

1. **Enforce HTTPS with certificate pinning** on the `WebClient` used in `RestApiClient`. Pin the mirror node's TLS certificate or CA, independent of the data returned by the API itself.
2. **Verify node cert hashes out-of-band.** Do not trust `nodeCertHash` values from the same REST response used to discover node IPs. Maintain a separately-configured, operator-controlled trust anchor (e.g., a pinned address book or a signed genesis file).
3. **Validate receipts via multiple independent nodes**, not solely via the node under test, so a single malicious node cannot forge a `SUCCESS` receipt.
4. **Require static node configuration** in security-sensitive deployments, disabling the `retrieveAddressBook` path entirely.

### Proof of Concept

1. Deploy a malicious gRPC server at `attacker.example.com:50211` that accepts any transaction and returns a `SUCCESS` `TransactionReceipt`.
2. Poison DNS or ARP so that the monitor's HTTP request to the mirror node REST API (`/network/nodes`) resolves to an attacker-controlled HTTP server.
3. Return a crafted `NetworkNodesResponse` JSON with `serviceEndpoints` pointing to `attacker.example.com:50211` and `nodeCertHash` matching the attacker's TLS certificate.
4. The monitor calls `NodeSupplier.refresh()` → `getAddressBook()` → receives the injected node list.
5. `validateNode()` connects to `attacker.example.com:50211`, sends 1 tinyhbar; the malicious server returns `SUCCESS`. The node is added to `nodes`.
6. `TransactionPublisher.getClients()` builds a `Client` with `setOperator(operatorId, operatorPrivateKey)` pointed at the attacker's server.
7. All subsequent `publish()` calls sign transactions with the operator key and submit them to the attacker's server. The attacker collects signed transactions and drains operator funds.

    private Client toClient(List<NodeProperties> nodes) {
        var operatorId = AccountId.fromString(monitorProperties.getOperator().getAccountId());
        var operatorPrivateKey =
                PrivateKey.fromString(monitorProperties.getOperator().getPrivateKey());

        // setNetworkFromAddressBook() doesn't support in-process URIs so we have to set network too.
        // Fix issue #11908 when different nodes in address book have the same service endpoint(s).
        var network = nodes.stream()
                .collect(Collectors.toMap(
                        NodeProperties::getEndpoint, p -> AccountId.fromString(p.getAccountId()), (a, b) -> a));
        var accountIds = network.values().stream().map(AccountId::toString).collect(Collectors.toSet());
        var nodeAddresses = nodes.stream()
                .filter(n -> accountIds.contains(n.getAccountId()))
                .map(NodeProperties::toNodeAddress)
                .toList();
        var nodeAddressBook = NodeAddressBook.newBuilder()
                .addAllNodeAddress(nodeAddresses)
                .build()
                .toByteString();

        var client = Client.forNetwork(Map.of());
        client.setNetworkFromAddressBook(com.hedera.hashgraph.sdk.NodeAddressBook.fromBytes(nodeAddressBook));
        client.setNetwork(network);
        client.setNodeMaxBackoff(publishProperties.getNodeMaxBackoff());
        client.setOperator(operatorId, operatorPrivateKey);
        return client;

    private Flux<NodeProperties> getAddressBook() {
        if (!monitorProperties.getNodeValidation().isRetrieveAddressBook()) {
            return Flux.empty();
        }

        var count = new AtomicInteger(0);

        return Flux.defer(restApiClient::getNodes)
                .filter(n -> !CollectionUtils.isEmpty(n.getServiceEndpoints()))
                .flatMapIterable(this::toNodeProperties)
                .doOnNext(n -> count.incrementAndGet())
                .doOnComplete(() -> log.info("Retrieved {} nodes from address book", count));
    }

    private List<NodeProperties> toNodeProperties(NetworkNode networkNode) {
        var nodeValidation = monitorProperties.getNodeValidation();
        var tlsPredicate = nodeValidation.getTls().getPredicate();
        return networkNode.getServiceEndpoints().stream()
                .filter(s -> tlsPredicate.test(s.getPort()))
                .limit(nodeValidation.getMaxEndpointsPerNode())
                .map(serviceEndpoint -> {
                    var host = StringUtils.isNotBlank(serviceEndpoint.getDomainName())
                            ? serviceEndpoint.getDomainName()
                            : serviceEndpoint.getIpAddressV4();
                    var nodeProperties = new NodeProperties();
                    nodeProperties.setAccountId(networkNode.getNodeAccountId());
                    nodeProperties.setCertHash(Strings.CS.remove(networkNode.getNodeCertHash(), "0x"));
                    nodeProperties.setHost(host);
                    nodeProperties.setNodeId(networkNode.getNodeId());
                    nodeProperties.setPort(serviceEndpoint.getPort());
                    return nodeProperties;
                })
                .toList();

    boolean validateNode(NodeProperties node) {
        if (!monitorProperties.getNodeValidation().isEnabled()) {
            nodes.addIfAbsent(node);
            log.info("Adding node {} without validation", node.getAccountId());
            return true;
        }

        log.info("Validating node {}", node);
        Hbar hbar = Hbar.fromTinybars(1L);
        AccountId nodeAccountId = AccountId.fromString(node.getAccountId());

        try (Client client = toClient(node)) {
            Status receiptStatus = new TransferTransaction()
                    .addHbarTransfer(nodeAccountId, hbar)
                    .addHbarTransfer(client.getOperatorAccountId(), hbar.negated())
                    .setNodeAccountIds(node.getAccountIds())
                    .execute(client)
                    .getReceipt(client)
                    .status;

            if (receiptStatus == SUCCESS) {
                log.info("Validated node {} successfully", nodeAccountId);
                nodes.addIfAbsent(node);
                return true;
            }

            log.warn("Unable to validate node {}: invalid status code {}", node, receiptStatus);
        } catch (TimeoutException e) {
            log.warn("Unable to validate node {}: Timed out", node);
        } catch (Exception e) {
            log.warn("Unable to validate node {}: ", node, e);
        }

        nodes.remove(node);
        return false;
