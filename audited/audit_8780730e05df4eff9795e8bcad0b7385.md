### Title
Unauthenticated HTTP Network-Map Injection in `fetchMirrorNodeNetwork()` Enables Targeted Node Overload

### Summary
`fetchMirrorNodeNetwork()` issues a plain, unauthenticated HTTP GET to the operator-configured `mirrorRest` URL and blindly trusts the returned `nodesEnvelope` to build the entire Hiero SDK network map. No TLS is enforced, no response signature is verified, and no minimum node-count or cross-validation against a known baseline is performed. An attacker who can intercept or spoof that HTTP response can inject a crafted node list containing only a targeted subset of real consensus nodes, causing the pinger to concentrate all transaction traffic on those nodes until they are overwhelmed and shut down.

### Finding Description

**Exact code path**

`pinger/sdk_client.go` `newClient()` (lines 17-22) calls `buildNetworkFromMirrorNodes()` for `network=other`, which calls `fetchMirrorNodeNetwork()` once at startup and passes the result directly to `hiero.ClientForNetwork(netmap)`. The client is never refreshed.

`pinger/mirror_node_client.go` `fetchMirrorNodeNetwork()` (lines 74-131):

- Line 46 / 79: A bare `http.Client` with no TLS configuration issues an unauthenticated `GET` to `cfg.mirrorRest + "/api/v1/network/nodes"`.
- Lines 95-98: The response body is decoded into `nodesEnvelope` with no signature, HMAC, or any integrity check.
- Lines 100-125: The network map is built exclusively from whatever `payload.Nodes` contains.
- Line 127: The only guard is `len(network) == 0`; any non-empty list is accepted unconditionally.

**Root cause / failed assumption**

The code assumes the HTTP response from `mirrorRest` is authoritative and trustworthy. The configuration documentation (`config.go` line 37) explicitly shows `http://mirror-rest:5551` as the example URL, meaning plain HTTP is the intended and documented transport. There is no mechanism to authenticate the server, verify the response payload, or cross-check the returned nodes against a known-good set.

**Exploit flow**

1. Attacker identifies the `mirrorRest` hostname used by the pinger (e.g., `mirror-rest` in a Kubernetes cluster, or a public hostname).
2. Attacker performs DNS spoofing, ARP poisoning, BGP prefix hijacking, or compromises an upstream resolver to redirect the hostname to an attacker-controlled server.
3. Attacker's server responds with a valid-looking `nodesEnvelope` JSON containing only the `service_endpoints` of a targeted 30%+ subset of real consensus nodes (valid `node_account_id` values, real IPs/ports).
4. `fetchMirrorNodeNetwork()` accepts the response (non-empty, HTTP 200, valid JSON), builds a network map with only those nodes, and returns it.
5. `hiero.ClientForNetwork(netmap)` creates the SDK client with only those nodes.
6. Every subsequent `submitWithRetry()` tick sends all pinger transactions exclusively to the targeted nodes.
7. The concentrated load causes those nodes to degrade and shut down, meeting the ≥30% threshold.

### Impact Explanation

The pinger's network map is fixed at startup from the injected response and never refreshed. All transaction load is permanently concentrated on the attacker-chosen subset of nodes for the lifetime of the process. If the targeted nodes represent ≥30% of the network's processing capacity, their overload and shutdown constitutes a medium-severity network disruption without brute force. The attack is surgical: the attacker selects which nodes to target and the pinger becomes the unwitting amplifier.

### Likelihood Explanation

The attack requires network-path control between the pinger and `mirrorRest`. For deployments using plain HTTP to an internal hostname (the documented default), DNS spoofing within the same cluster namespace, a compromised sidecar, or a misconfigured ingress is sufficient — no credentials or privileged access to the pinger host are needed. For public-facing `mirrorRest` endpoints, BGP hijacking or DNS cache poisoning are realistic vectors for a sophisticated but unprivileged external attacker. The attack is repeatable: if the pinger is restarted, the attacker can inject again.

### Recommendation

1. **Enforce HTTPS**: Reject any `mirrorRest` URL that does not use the `https://` scheme at config validation time (`config.go` `loadConfig()`).
2. **Pin the CA or certificate**: Configure the `http.Client` with a custom `tls.Config` that pins the expected CA or leaf certificate for the mirror-rest endpoint.
3. **Validate minimum node count**: After building the network map, reject it if it contains fewer nodes than a configurable minimum (e.g., the known full network size minus an acceptable delta).
4. **Cross-validate against a hardcoded baseline**: For well-known networks, embed the expected node account IDs and reject any response that omits known nodes.
5. **Periodic refresh with drift detection**: Refresh the network map periodically and alert/abort if the new map is a strict subset of the previous one by more than a threshold.

func fetchMirrorNodeNetwork(
	ctx context.Context,
	httpClient *http.Client,
	url string,
) (map[string]hiero.AccountID, bool, error) {
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		return nil, false, err
	}

	resp, err := httpClient.Do(req)
	if err != nil {
		return nil, true, fmt.Errorf("GET %s failed: %w", url, err)
	}
	defer resp.Body.Close()

	if resp.StatusCode < 200 || resp.StatusCode >= 300 {
		retry := resp.StatusCode == http.StatusTooManyRequests || resp.StatusCode >= 500
		return nil, retry, fmt.Errorf("GET %s returned %s", url, resp.Status)
	}

	var payload nodesEnvelope
	if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
		return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
	}

		}
	}
