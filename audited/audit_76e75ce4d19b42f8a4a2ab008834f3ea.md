### Title
Unauthenticated `/metrics` Endpoint Lacks Concurrency Control, Enabling CPU-Exhaustion via Thundering Herd

### Summary
The `/metrics` endpoint registered in `Routes()` uses `promhttp.Handler()` with no `MaxRequestsInFlight` limit, no authentication, and no rate limiting. The Traefik ingress middleware (which enforces `inFlightReq: 5` and `rateLimit: average: 10`) explicitly omits `/metrics` from its protected paths, leaving the endpoint fully exposed. An unprivileged attacker can flood it with concurrent requests, causing every goroutine to simultaneously call `prometheus.DefaultGatherer.Gather()` and serialize the full registry, spiking CPU and starving legitimate traffic.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/metrics.go`, `Routes()`, lines 64â€“73:
```go
func (c *metricsController) Routes() server.Routes {
    return server.Routes{
        {
            "metrics",
            "GET",
            metricsPath,           // "/metrics"
            promhttp.Handler().ServeHTTP,  // no MaxRequestsInFlight
        },
    }
}
```
`promhttp.Handler()` is a convenience wrapper for `promhttp.HandlerFor(prometheus.DefaultGatherer, promhttp.HandlerOpts{})`. The zero-value `HandlerOpts` leaves `MaxRequestsInFlight` at `0`, meaning **unlimited** concurrent serializations are permitted.

**Why existing checks fail:**

The Traefik middleware chain (`inFlightReq: amount: 5`, `rateLimit: average: 10`) is attached to the ingress, but the ingress `paths` list in `charts/hedera-mirror-rosetta/values.yaml` lines 125â€“133 enumerates only:
```

```

At the application layer (`main.go` lines 217â€“227), the middleware stack is `MetricsMiddleware â†’ TracingMiddleware â†’ CorsMiddleware`. None of these impose a concurrency cap on `/metrics`. The `http.Server` has read/write timeouts but no per-endpoint goroutine limit.

**Exploit flow:**

Each concurrent HTTP request causes Go's `net/http` to spawn a goroutine that calls `prometheus.DefaultGatherer.Gather()`. The default gatherer acquires a per-collector read lock and invokes every registered collector (including all default Go runtime collectors: memory stats, GC, goroutine counts, plus the 4 custom histograms/gauges). All goroutines proceed in parallel, each allocating its own `[]dto.MetricFamily` slice and running the text encoder. With N concurrent requests, N full serializations run simultaneously.

### Impact Explanation
CPU and allocator pressure scale linearly with concurrent `/metrics` requests. Because the Go HTTP server spawns one goroutine per connection with no backpressure, a sustained flood (e.g., 500â€“1000 concurrent connections) can saturate all available CPU cores on the node. This degrades or blocks responses to legitimate Rosetta API calls (`/network/status`, `/block`, `/construction/*`) that the mirror node depends on for blockchain data availability. In the worst case the node becomes unresponsive, effectively removing it from the network â€” matching the stated scope of â‰Ą30% processing node shutdown without brute force.

### Likelihood Explanation
No authentication, no API key, no IP allowlist protects `/metrics`. The endpoint is reachable through the nginx reverse proxy in docker-compose (`location /rosetta/ { rewrite â€¦ }` in `docker-compose.yml` line 235) and directly on port 5700 if exposed. Any external actor with network access can issue concurrent HTTP GET requests using standard tools (`ab`, `wrk`, `hey`). The attack is stateless, requires no prior knowledge of the system, and is trivially repeatable.

### Recommendation
1. **Set `MaxRequestsInFlight`** when constructing the handler:
   ```go
   promhttp.HandlerFor(prometheus.DefaultGatherer, promhttp.HandlerOpts{
       MaxRequestsInFlight: 2,
       Timeout:             5 * time.Second,
   }).ServeHTTP
   ```
2. **Add `/rosetta/metrics` to the Traefik ingress paths** in `charts/hedera-mirror-rosetta/values.yaml` so the existing `inFlightReq` and `rateLimit` middleware applies.
3. **Restrict access** to `/metrics` to internal/monitoring networks only (e.g., via a Kubernetes `NetworkPolicy` or nginx `allow`/`deny` directives), since Prometheus scraping is an internal operation and the endpoint should never be publicly reachable.
