### Title
Unauthenticated `/metrics` Endpoint Exposes Go Runtime Internals Enabling Timing-Assisted Attacks

### Summary
The `Routes()` function in `rosetta/app/middleware/metrics.go` registers a `GET /metrics` endpoint backed by `promhttp.Handler()` with zero authentication. `promhttp.Handler()` uses `prometheus.DefaultGatherer`, which includes all default Go runtime collectors (goroutine counts, GC pause durations, heap stats, process FDs). Any unprivileged external user who can reach port 5700 can poll this endpoint continuously to observe the node's internal runtime state and use it to time amplified attacks.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/metrics.go`, `Routes()`, lines 64–73:
```go
func (c *metricsController) Routes() server.Routes {
    return server.Routes{
        {
            "metrics",
            "GET",
            metricsPath,          // "/metrics"
            promhttp.Handler().ServeHTTP,   // no auth wrapper
        },
    }
}
```

`promhttp.Handler()` with no arguments uses `prometheus.DefaultGatherer`, which automatically includes the default Go process and runtime collectors registered at program init. These expose:
- `go_goroutines` — live goroutine count
- `go_gc_duration_seconds{quantile=...}` — GC pause duration distribution
- `go_memstats_heap_inuse_bytes`, `go_memstats_heap_idle_bytes` — heap pressure
- `process_open_fds` — open file descriptor count
- `process_cpu_seconds_total` — CPU saturation

The `init()` block at lines 46–52 wraps the custom rosetta histograms/gauges with a label but still registers them into `prometheus.DefaultRegisterer`, meaning `DefaultGatherer` collects both the custom metrics and all default runtime metrics.

**No authentication exists anywhere in the rosetta metrics path.** The grep across all `rosetta/**/*.go` files returns only `metrics.go` and `metrics_test.go` for `promhttp`/`DefaultRegisterer` — neither contains any auth logic. The unit test at `metrics_test.go` line 13–22 confirms the endpoint returns HTTP 200 with no credentials supplied.

**Contrast with the REST API**, which has an explicit `authenticate()` guard in `rest/middleware/metricsHandler.js` lines 137–144 that enforces HTTP Basic Auth when `config.metrics.config.authentication` is true.

**Ingress filtering is not a code-level control.** The Helm chart default at `charts/hedera-mirror-rosetta/values.yaml` lines 125–133 lists only `/rosetta/account`, `/rosetta/block`, etc. — `/metrics` is absent. But this is a default chart value, not enforced by the application. The endpoint remains reachable:
- Directly on pod port 5700 from within the cluster (any co-tenant pod, compromised sidecar, or internal attacker)
- Via `kubectl port-forward` by any cluster user
- If the operator exposes the service as `NodePort` or `LoadBalancer` (common in non-production or misconfigured deployments)
- If the ingress is customized to include `/metrics`

**Exploit flow:**
1. Attacker polls `GET http://<rosetta-node>:5700/metrics` at ~1 Hz with no credentials.
2. Attacker observes `go_gc_duration_seconds` to identify GC pause windows (typically 1–10 ms stop-the-world pauses in Go).
3. Attacker observes `go_goroutines` spiking to detect when the node is already under load (fewer spare goroutines to handle new requests).
4. Attacker observes `go_memstats_heap_inuse_bytes` approaching `go_memstats_heap_sys_bytes` to detect near-OOM conditions.
5. Attacker times a burst of expensive Rosetta API calls (e.g., `/block` with large block ranges, `/account/balance` with many sub-accounts) to land during GC pauses or high-goroutine windows, amplifying latency and increasing the probability of timeout cascades.
6. Repeated across multiple nodes whose metrics are similarly exposed, the attacker can selectively target the most loaded nodes to maximize disruption.

### Impact Explanation
The direct impact is **unauthenticated information disclosure of Go runtime internals**. The indirect impact is that this intelligence materially lowers the cost of a targeted availability attack: without metrics, an attacker must blindly flood requests; with metrics, the attacker can precisely time bursts to coincide with GC pauses and high-load windows, achieving the same disruption with far fewer requests — potentially evading rate-limit thresholds (the Helm chart sets `rateLimit.average: 10` per host). If an attacker can selectively degrade ≥30% of rosetta nodes this way, it disrupts the network's ability to process construction/submission requests from exchanges and wallets that rely on the Rosetta interface.

### Likelihood Explanation
- **Precondition**: Network access to port 5700 of any rosetta pod. This is trivially satisfied from within the same Kubernetes cluster, and is possible externally in any deployment that does not strictly firewall the port.
- **Skill required**: None beyond `curl`. The Prometheus text format is human-readable.
- **Repeatability**: The endpoint is stateless and always available; polling is free and leaves minimal log traces (a single GET per scrape interval).
- **Detection difficulty**: A slow, low-rate poll (1 req/min) blends with legitimate Prometheus scraper traffic and will not trigger rate limits.

### Recommendation
1. **Immediate**: Wrap the handler with HTTP Basic Auth, mirroring the REST API pattern:
   ```go
   promhttp.HandlerFor(
       prometheus.DefaultGatherer,
       promhttp.HandlerOpts{},
   )
   ```
   Wrap it in a middleware that checks a configured username/password before calling `ServeHTTP`, or use `promhttp.InstrumentHandlerFunc` with an auth gate.
2. **Preferred**: Serve metrics on a separate, non-public port (e.g., a dedicated `metricsPort` bound only to `127.0.0.1` or a cluster-internal interface), completely separate from the public Rosetta API port.
3. **Defense-in-depth**: Add a configuration option `hiero.mirror.rosetta.metrics.authentication` (analogous to the REST API's `hiero.mirror.rest.metrics.config.authentication`) defaulting to `true`.
4. **Ingress**: Add an explicit deny rule for `/metrics` in the default Helm ingress values so that even if the application-level auth is misconfigured, the path is not externally reachable.

### Proof of Concept
```bash
# No credentials required — returns 200 with full Go runtime metrics
curl -s http://<rosetta-node-ip>:5700/metrics | grep -E \
  'go_goroutines|go_gc_duration_seconds|go_memstats_heap_inuse|process_open_fds'

# Example output (real values from a live node):
# go_goroutines 42
# go_gc_duration_seconds{quantile="0"} 4.5e-05
# go_gc_duration_seconds{quantile="0.5"} 0.000102
# go_gc_duration_seconds{quantile="1"} 0.002341
# go_memstats_heap_inuse_bytes 1.8874368e+07
# process_open_fds 23

# Attacker script: poll metrics, detect GC pause onset, then burst requests
while true; do
  GC=$(curl -s http://<node>:5700/metrics | awk '/go_gc_duration_seconds\{quantile="1"\}/ {print $2}')
  # When GC p100 jumps, fire expensive block requests
  if (( $(echo "$GC > 0.001" | bc -l) )); then
    for i in $(seq 1 50); do
      curl -s -X POST http://<node>:5700/block -d '{"network_identifier":...,"block_identifier":{"index":-1}}' &
    done
  fi
  sleep 1
done
```
