### Title
Gas Estimation Throttle Bypass via Binary Search Amplification (20x EVM Execution per Throttle Token)

### Summary
The `ContractController.call()` endpoint applies the gas throttle exactly once per HTTP request, but when `estimate=true`, `ContractExecutionService.processCall()` delegates to `BinaryGasEstimator.search()`, which executes the EVM up to `maxGasEstimateRetriesCount` (default: 20) times per request. An unprivileged attacker can submit requests with `estimate=true` and `gas=15000000` to trigger up to 20 full EVM executions while consuming only one throttle token, achieving a 20x amplification of CPU/thread work. This can exhaust the web3 module's EVM execution thread pool, causing the mirror node's `/api/v1/contracts/call` endpoint to become unresponsive to all other users.

### Finding Description

**Code path:**

1. `ContractController.call()` ΓÇË [1](#0-0)  ΓÇË calls `throttleManager.throttle(request)` once (line 40), then `validateContractMaxGasLimit(request)` (line 41), then `contractExecutionService.processCall(params)` (line 44). The throttle is consumed once for the full `gas` value.

2. `validateContractMaxGasLimit()` ΓÇË [2](#0-1)  ΓÇË only rejects `gas > maxGasLimit` (default 15,000,000). Setting `gas=15000000` passes this check.

3. `BinaryGasEstimator.search()` ΓÇË [3](#0-2)  ΓÇË loops `while (lo + 1 < hi && iterationsMade < properties.getMaxGasEstimateRetriesCount())`, calling `safeCall(mid, call)` each iteration. With `lo=21000` and `hi=15000000`, this runs up to `maxGasEstimateRetriesCount = 20` iterations. [4](#0-3) 

4. `ContractCallService.doProcessCall()` ΓÇË [5](#0-4)  ΓÇË `restoreGasToBucket` and metrics are explicitly skipped when `estimate=true` (`if (!estimate)`). No per-iteration throttle check exists.

**Root cause:** The throttle is applied once at the HTTP layer for the declared `gas` value, but the binary search loop multiplies actual EVM work by up to 20x without any additional throttle deduction per iteration. The design assumption that one throttle token Γëê one EVM execution is violated for estimate calls.

**Failed assumption:** The throttle design assumes each request maps to approximately one EVM execution. For `estimate=true`, this assumption is false by a factor of up to 20.

### Impact Explanation

An attacker sending concurrent requests with `estimate=true` and `gas=15000000` causes up to 20 synchronous EVM executions per request. The `gasPerSecond` default of 7.5B gas/second allows ~500 such requests/second through the gas throttle, but each triggers up to 20 EVM executions ΓÇË effectively 10,000 EVM executions/second. The `requestsPerSecond = 500` limit is also insufficient since each request carries 20x the expected work. This exhausts the web3 module's EVM execution threads, causing the `/api/v1/contracts/call` endpoint to stop responding to legitimate users (DoS of the mirror node's web3 API). [6](#0-5) 

### Likelihood Explanation

No authentication or special privilege is required. The endpoint is publicly accessible. The exploit requires only standard HTTP POST requests with `estimate=true` and `gas=15000000`. It is trivially scriptable and repeatable. Any attacker with network access to the mirror node's web3 API can execute this. The amplification factor (up to 20x) means even a single attacker with modest bandwidth can saturate the EVM thread pool.

### Recommendation

1. **Multiply the throttle cost for estimate calls**: Before invoking `BinaryGasEstimator.search()`, deduct `gas * maxGasEstimateRetriesCount` tokens from the throttle bucket (or a configurable fraction thereof), not just `gas` once.
2. **Alternatively, throttle per binary search iteration**: Pass the throttle manager into `BinaryGasEstimator.search()` and deduct gas on each `safeCall` invocation.
3. **Reduce `maxGasEstimateRetriesCount`** or add a separate, lower rate limit specifically for `estimate=true` requests.
4. **Add a per-IP or per-connection rate limit** at the HTTP layer independent of gas accounting.

### Proof of Concept

```bash
# Send 50 concurrent estimate requests with max gas
for i in $(seq 1 50); do
  curl -s -X POST http://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{
      "data": "0x<expensive_contract_calldata>",
      "to": "0x<deployed_contract_address>",
      "gas": 15000000,
      "estimate": true
    }' &
done
wait

# Each request triggers up to 20 EVM executions in BinaryGasEstimator.search()
# while consuming only 1 throttle token worth of 15M gas.
# Observe: legitimate /api/v1/contracts/call requests begin timing out
# as the EVM execution thread pool is saturated.
```

Preconditions: Mirror node web3 API is reachable; a deployed contract address is known (or `to` can be any address ΓÇË even a failing call runs the binary search loop). No credentials required.
