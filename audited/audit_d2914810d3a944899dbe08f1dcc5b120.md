### Title
Unauthenticated `/metrics` Endpoint Exposes Prometheus Operational Data in Rosetta Service

### Summary
The Rosetta service registers the `/metrics` endpoint using a bare `promhttp.Handler().ServeHTTP` with no authentication wrapper. Any unprivileged external user who can reach the service port can issue a plain `GET /metrics` and receive the full Prometheus text exposition, including all metric series carrying the `application="rosetta"` label. This contrasts directly with the REST service, which gates its equivalent endpoint behind a configurable `authenticate()` check.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/metrics.go`, `Routes()`, lines 64–73: [1](#0-0) 

The handler is wired as:
```go
promhttp.Handler().ServeHTTP   // no auth wrapper
```

`init()` (lines 46–52) registers all four metric families against `prometheus.DefaultRegisterer` wrapped with the constant label `application="rosetta"`: [2](#0-1) 

The controller is added unconditionally to the public router in `rosetta/main.go` lines 106 and 118: [3](#0-2) 

There is no IP allowlist, no token check, and no middleware applied to this route before the handler fires.

**Root cause / failed assumption:** The developer assumed `/metrics` would be shielded by network-layer controls (firewall, ingress policy). No such enforcement exists in the application itself, and the route is registered on the same public port as all other Rosetta API endpoints.

**Why existing checks fail:** The REST service (`rest/middleware/metricsHandler.js` lines 137–144) implements a configurable `authenticate()` guard that returns HTTP 401 when credentials are absent or wrong: [4](#0-3) 

No equivalent guard exists anywhere in the Rosetta middleware stack.

### Impact Explanation
A successful request returns the full Prometheus text exposition, which includes:
- `hiero_mirror_rosetta_request_bytes{method, route, application="rosetta"}` — reveals which API routes are being called and approximate payload sizes.
- `hiero_mirror_rosetta_request_duration{method, route, status_code, ws, application="rosetta"}` — reveals latency distribution per route, useful for timing-based reconnaissance.
- `hiero_mirror_rosetta_request_inflight{method, route, application="rosetta"}` — reveals real-time concurrency, useful for load-timing attacks.
- `hiero_mirror_rosetta_response_bytes{method, route, application="rosetta"}` — reveals response sizes per route.
- All default Go runtime and `promhttp` internal metrics (GC pauses, goroutine counts, memory usage, etc.).

This constitutes an **information disclosure** vulnerability: an attacker learns the internal traffic profile, active routes, and runtime health of the node without any credential.

### Likelihood Explanation
- **Precondition:** The attacker must be able to reach the Rosetta HTTP port (default 8082). In many deployments this port is exposed to the internet or to a semi-trusted network.
- **Skill required:** None — a single `curl` suffices.
- **Repeatability:** Unlimited; the endpoint is stateless and has no rate-limiting applied to it in the code.
- **Detection:** The request appears as a normal GET in access logs with no distinguishing marker.

### Recommendation
Apply an authentication wrapper to the `/metrics` route, mirroring the pattern already used in the REST service. Concretely:

1. Add a configurable `metrics.authentication` flag to the Rosetta config (analogous to `config.metrics.config.authentication` in the REST service).
2. Wrap `promhttp.Handler()` in an `http.HandlerFunc` that validates HTTP Basic credentials before delegating to the Prometheus handler.
3. Alternatively, bind the metrics endpoint to a separate, non-public port (e.g., `localhost:9090`) so it is never reachable from external networks without an explicit tunnel.

### Proof of Concept
```bash
# No credentials required; replace <host>:<port> with the Rosetta service address
curl -s http://<host>:<port>/metrics | grep 'application="rosetta"'

# Expected output (excerpt):
# hiero_mirror_rosetta_request_bytes_bucket{application="rosetta",method="POST",route="/construction/submit",le="512"} 0
# hiero_mirror_rosetta_request_duration_sum{application="rosetta",method="POST",route="/construction/submit",...} 1.23
# hiero_mirror_rosetta_request_inflight{application="rosetta",method="POST",route="/construction/submit"} 0
```
The response is returned with HTTP 200 and `Content-Type: text/plain; version=0.0.4`, confirming zero access control is enforced.

const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
