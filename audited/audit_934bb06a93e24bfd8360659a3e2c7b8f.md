### Title
Historical Query Path Exposes Deleted Hook Storage Records via Unprivileged Timestamp Parameter

### Summary
Any unauthenticated user can supply a `timestamp` query parameter to `GET /api/v1/accounts/{ownerId}/hooks/{hookId}/storage`, causing `HookStorageRequest.isHistorical()` to return `true` and routing execution to `getHookStorageChange()`. Unlike the non-historical path, the historical query path does not filter out records where `deleted=true`, exposing logically-deleted hook storage slots to any caller.

### Finding Description
**Code path:**

In `HookStorageRequest.isHistorical()` ( [1](#0-0) ), the method returns `!timestamp.isEmpty()`. `Bound.isEmpty()` returns `true` only when both `lower` and `upper` are `null` ( [2](#0-1) ). Any valid `timestamp` query parameter (e.g., `?timestamp=gt:0`) produces a non-empty `Bound`, making `isHistorical()` return `true`.

In `HookServiceImpl.getHookStorage()`, this branches to `getHookStorageChange()` instead of the non-historical path: [3](#0-2) 

**Non-historical path** explicitly filters deleted records at the DB level via Spring Data method naming:
- `findByOwnerIdAndHookIdAndKeyBetweenAndDeletedIsFalse` [4](#0-3) 
- `findByOwnerIdAndHookIdAndKeyInAndDeletedIsFalse` [5](#0-4) 

**Historical path** queries compute `deleted` as a derived column but apply **no `WHERE` filter** to exclude deleted records: [6](#0-5) 

The `deleted` flag is computed as `(value_written is null or length(value_written) = 0)` but the result set is returned unfiltered. Neither `getHookStorageChange()` nor `HookStorageMapper` applies any post-query filter on the `deleted` field: [7](#0-6) [8](#0-7) 

### Impact Explanation
Deleted hook storage slots — which represent intentionally erased state — are returned in API responses to any caller who provides a timestamp parameter. This breaks the data-integrity guarantee of the mirror node export: consumers of the API receive records that should not exist in the current or historical view of active storage. Depending on what data is stored in hook storage slots, this could leak sensitive contract state that was deliberately cleared.

### Likelihood Explanation
The endpoint is publicly accessible with no authentication required. The exploit requires only appending `?timestamp=gt:0` (or any valid timestamp value) to a standard storage query. No special privileges, credentials, or knowledge of internal state are needed. The attack is trivially repeatable by any external user.

### Recommendation
Add a `WHERE (value_written IS NOT NULL AND length(value_written) > 0)` predicate to both queries in `HookStorageChangeRepository`, mirroring the `DeletedIsFalse` constraint enforced by `HookStorageRepository`. Alternatively, filter deleted records in `getHookStorageChange()` after the query, or add a post-query filter in the mapper. The fix must be applied to both `findByKeyBetweenAndTimestampBetween` and `findByKeyInAndTimestampBetween`.

### Proof of Concept
1. Create a hook storage slot for account `0.0.123`, hook ID `1`, with key `0x000...001`.
2. Delete the storage slot (set `value_written` to null or empty in `hook_storage_change`).
3. Confirm the slot is absent from: `GET /api/v1/accounts/0.0.123/hooks/1/storage` (non-historical, correctly filtered).
4. Issue: `GET /api/v1/accounts/0.0.123/hooks/1/storage?timestamp=gt:0`
5. Observe the deleted storage slot appears in the response with `deleted: true` and a null/empty `value`, confirming the bypass.

```java
    private HookStorageResult getHookStorageChange(HookStorageRequest request) {
        final var page = request.getPageRequest();
        final var ownerId = entityService.lookup(request.getOwnerId());
        final long hookId = request.getHookId();

        final var keys = request.getKeys();
        final boolean requestHasKeys = !keys.isEmpty();
        final var keysInRange = request.getKeysInRange();

        if (keysInRange.isEmpty() && requestHasKeys) {
            return new HookStorageResult(ownerId, List.of());
        }

        final var timestamp = request.getTimestamp();
        final long timestampLowerBound = timestamp.getAdjustedLowerRangeValue();
        final long timestampUpperBound = timestamp.adjustUpperBound();

        List<HookStorage> changes;

        if (requestHasKeys) {
            changes = hookStorageChangeRepository.findByKeyInAndTimestampBetween(
                    ownerId.getId(), hookId, keys, timestampLowerBound, timestampUpperBound, page);
        } else {
            changes = hookStorageChangeRepository.findByKeyBetweenAndTimestampBetween(
                    ownerId.getId(),
                    hookId,
                    request.getKeyLowerBound(),
                    request.getKeyUpperBound(),
                    timestampLowerBound,
                    timestampUpperBound,
                    page);
        }

        return new HookStorageResult(ownerId, changes);
    }
```
