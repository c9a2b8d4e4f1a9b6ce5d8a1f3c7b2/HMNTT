### Title
`0.0.0.0` IP Address Bypasses Empty-String Check in `fetchMirrorNodeNetwork`, Redirecting gRPC Traffic to Localhost

### Summary
In `fetchMirrorNodeNetwork()`, the only guard against an invalid `ip_address_v4` value is a check for an empty string (`host == ""`). The reserved address `"0.0.0.0"` is non-empty, so it passes this check and is inserted into the SDK network map. On Linux (the typical container OS), a `connect()` to `0.0.0.0` resolves to the loopback interface (`127.0.0.1`), silently redirecting all consensus-node gRPC traffic — including signed transactions — to localhost.

### Finding Description
**File:** `pinger/mirror_node_client.go`
**Function:** `fetchMirrorNodeNetwork()`
**Lines:** 113–124

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)   // "0.0.0.0" is non-empty
    }
    if host == "" || ep.Port == 0 {                // ← only guard; "0.0.0.0" passes
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId                  // "0.0.0.0:<port>" inserted
}
```

**Root cause:** The code assumes that any non-empty, non-zero-port pair is a valid, routable consensus-node address. It never validates whether the IP is a reserved/special address (`0.0.0.0`, `127.x`, RFC-1918 ranges, etc.).

**Exploit flow:**
1. Attacker controls the mirror node HTTP response (compromised mirror node, DNS hijack of `cfg.mirrorRest`, or HTTP MITM — the connection is plain HTTP per the config default `http://mirror-rest:5551`).
2. Response JSON contains `"ip_address_v4": "0.0.0.0"` with a valid port (e.g., 50211).
3. `fetchMirrorNodeNetwork` inserts `"0.0.0.0:50211"` → `nodeAccountId` into the returned map.
4. `newClient` calls `hiero.ClientForNetwork(netmap)` with this poisoned map (`sdk_client.go` line 22).
5. The Hiero SDK opens gRPC connections to `0.0.0.0:50211`; on Linux this resolves to `127.0.0.1:50211`.
6. If the attacker has a gRPC listener on that port within the same pod/host (trivially achievable in a shared-node Kubernetes cluster), they receive all subsequent signed transactions.

**Why existing checks fail:** The sole guard is `host == ""` (line 118). `"0.0.0.0"` is a six-character string — it is never empty, so the check is bypassed unconditionally.

### Impact Explanation
The pinger submits real on-chain transfers every tick (default: 10 000 tinybar, configurable). Redirecting its gRPC stream to a localhost listener allows an attacker to:
- Observe every signed `CryptoTransfer` transaction, including the serialised transaction bytes and the operator's signature.
- Silently drop transactions, causing the pinger to report false node-health failures and masking real outages.
- In a worst-case shared-node environment, replay captured signed transactions against the real network.

Severity matches the stated scope: Layer-1 network-routing code producing unintended consensus-client behavior; direct fund loss is low (tinybar amounts) but monitoring integrity and transaction confidentiality are concretely compromised.

### Likelihood Explanation
The mirror node REST URL is configured via the environment variable `HIERO_MIRROR_PINGER_REST` and is fetched over plain HTTP (no TLS enforced in the default config). An attacker who can:
- Compromise or impersonate the mirror node (realistic in a self-hosted deployment), or
- Perform a DNS/ARP spoof on the cluster network (realistic in a multi-tenant Kubernetes environment),

can inject this payload with zero authentication required. The attack is repeatable on every pinger restart or network-refresh cycle.

### Recommendation
Add explicit IP validation before accepting an endpoint. At minimum, reject the IANA-reserved "unspecified" address and loopback ranges:

```go
import "net/netip"

func isRoutableIPv4(s string) bool {
    addr, err := netip.ParseAddr(s)
    if err != nil || !addr.Is4() {
        return false
    }
    return !addr.IsUnspecified() &&   // 0.0.0.0
           !addr.IsLoopback()    &&   // 127.x.x.x
           !addr.IsLinkLocalUnicast() // 169.254.x.x
}
```

Apply this check at line 116 before assigning `host`. Additionally, enforce HTTPS (or mutual TLS) for the mirror node REST URL to prevent MITM injection of the response.

### Proof of Concept
1. Stand up a mock mirror node that returns:
   ```json
   {"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"ip_address_v4":"0.0.0.0","port":50211}]}],"links":{}}
   ```
2. Start a gRPC listener on `127.0.0.1:50211` (e.g., `nc -l 50211`).
3. Run the pinger with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://<mock>`.
4. Observe that the pinger's SDK connects to `127.0.0.1:50211` (confirmed via `ss -tp` or `strace`), and the listener receives the raw gRPC bytes of the signed `CryptoTransfer`. [1](#0-0) [2](#0-1) [3](#0-2)
