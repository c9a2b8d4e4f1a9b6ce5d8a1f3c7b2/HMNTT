### Title
IP Spoofing via Client-Controlled Headers in Rosetta `getClientIpAddress()` Enables Audit Log Manipulation

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` unconditionally trusts the `X-Real-IP` and `X-Forwarded-For` headers supplied by the HTTP client. An unprivileged external attacker can send `X-Real-IP: ` (empty value) to pass the `len(ipAddress) == 0` guard, then supply an arbitrary `X-Forwarded-For` value that is written verbatim into the Rosetta access log. This corrupts source-IP attribution for every logged Rosetta operation, including `/call` (smart contract calls), making forensic audit trails unreliable.

### Finding Description
**Exact code path** — `rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)      // line 64 – reads X-Real-IP

    if len(ipAddress) == 0 {                       // line 66 – empty OR absent
        ipAddress = r.Header.Get(xForwardedForHeader) // line 67 – reads X-Forwarded-For
    }

    if len(ipAddress) == 0 {                       // line 70
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    return ipAddress
}
```

**Root cause** — Go's `r.Header.Get()` returns `""` for both *absent* headers and headers *present with an empty value*. The `len(ipAddress) == 0` check cannot distinguish the two cases. A client that deliberately sends `X-Real-IP: ` (empty value) satisfies the guard and causes the code to fall through to the fully attacker-controlled `X-Forwarded-For` value.

**Deployment enablement** — The Kubernetes Traefik configuration explicitly enables insecure forwarded-header passthrough:

```yaml
globalArguments:
  - --entryPoints.web.forwardedHeaders.insecure    # trusts ALL sources
  - --entryPoints.websecure.forwardedHeaders.insecure
``` [1](#0-0) 

With this flag, Traefik passes client-supplied `X-Forwarded-For` unmodified to the Rosetta service. Traefik does not set `X-Real-IP` by default, so the first header read returns `""`, the guard fires, and the attacker's `X-Forwarded-For` value is used.

The resulting spoofed IP is embedded directly into the log message at line 52–53:

```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
``` [2](#0-1) 

### Impact Explanation
Every Rosetta API request — including `/rosetta/call` (smart contract invocations) — is logged with the attacker-supplied IP as the source. Audit trails, incident response, and rate-limit forensics that rely on these logs will attribute malicious or erroneous smart contract calls to a fabricated address. While no on-chain state or funds are directly altered, the integrity of the network-layer audit record is fully compromised, satisfying the "unintended smart contract behavior / no concrete funds at direct risk" severity band.

### Likelihood Explanation
- **No authentication required** — any HTTP client reachable by the Rosetta endpoint can perform this.
- **Trivially repeatable** — a single crafted request is sufficient; no race condition or timing dependency.
- **Traefik misconfiguration is present in the shipped chart** — the `forwardedHeaders.insecure` flags are committed defaults, meaning every default Kubernetes deployment is affected without additional attacker effort.
- **docker-compose is not affected** — nginx overwrites `X-Real-IP` with `$remote_addr`, neutralising the attack in that deployment path.

### Recommendation
1. **Validate the header value is a syntactically valid IP address** before trusting it (use `net.ParseIP()` after extracting the first token from a comma-separated `X-Forwarded-For` list).
2. **Restrict trusted proxy sources** — only accept `X-Real-IP` / `X-Forwarded-For` when the connection originates from a known trusted proxy CIDR; reject or ignore these headers otherwise.
3. **Remove `--entryPoints.*.forwardedHeaders.insecure`** from the Traefik chart and replace it with an explicit `trustedIPs` list scoped to the cluster's internal CIDR.
4. Prefer `r.RemoteAddr` as the authoritative source when no trusted-proxy relationship can be established.

### Proof of Concept
```bash
# Against a Kubernetes deployment with default Traefik chart values:
curl -H "X-Real-IP: " \
     -H "X-Forwarded-For: 10.evil.attacker.ip" \
     https://<rosetta-host>/network/list

# Expected log entry (Rosetta pod stdout):
# level=info msg="10.evil.attacker.ip GET /network/list (200) in 3ms"
```

The spoofed IP `10.evil.attacker.ip` (or any arbitrary string) appears as the authenticated source of the request in the Rosetta access log, replacing the real client address.
