### Title
Unauthenticated `/metrics` Endpoint Exposes Internal Prometheus Metrics in Rosetta Go Service

### Summary
The `Routes()` function in `rosetta/app/middleware/metrics.go` registers the `/metrics` endpoint using `promhttp.Handler().ServeHTTP` directly, with no authentication wrapper of any kind. The entire rosetta Go server middleware chain (`MetricsMiddleware` → `TracingMiddleware` → `CorsMiddleware`) contains no authentication layer, meaning any network-reachable client can issue `GET /metrics` and receive a full Prometheus metrics dump. This contrasts with the `rest/` Node.js service, which explicitly gates its metrics endpoint behind a configurable `authenticate()` check.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/metrics.go`, `Routes()`, lines 64–73:
```go
func (c *metricsController) Routes() server.Routes {
    return server.Routes{
        {
            "metrics",
            "GET",
            metricsPath,           // "/metrics"
            promhttp.Handler().ServeHTTP,  // raw handler, no auth
        },
    }
}
``` [1](#0-0) 

`NewMetricsController()` is passed directly into `server.NewRouter()` in both `newBlockchainOnlineRouter` (line 118) and `newBlockchainOfflineRouter` (line 152) in `rosetta/main.go`, with no per-route auth wrapper applied. [2](#0-1) [3](#0-2) 

The full server middleware chain assembled in `main()` is:
```
MetricsMiddleware(router) → TracingMiddleware → CorsMiddleware → http.Server
``` [4](#0-3) 

None of these layers add authentication. There is no IP allowlist, no token check, no Basic Auth, and no conditional flag to disable the endpoint.

**Root cause:** The failed assumption is that `/metrics` would be protected at the network/ingress layer in all deployments. The code itself provides zero defense-in-depth; the handler is `promhttp.Handler()` with no wrapping.

**Contrast with the Node.js `rest/` service**, which explicitly calls `authenticate(req)` before serving metrics and returns HTTP 401 if the check fails: [1](#0-0) [1](#0-0) 

```js
// rest/middleware/metricsHandler.js lines 157-160
if (!authenticate(req)) {
    res.set('WWW-Authenticate', 'Basic realm="Metrics"');
    return res.status(401).send('Unauthorized');
}
``` [5](#0-4) 

No equivalent guard exists anywhere in the rosetta Go service.

### Impact Explanation
A successful `GET /metrics` response leaks:
- All registered route names (`method`, `route` labels on `hiero_mirror_rosetta_request_bytes`, `hiero_mirror_rosetta_request_duration`, `hiero_mirror_rosetta_request_inflight`, `hiero_mirror_rosetta_response_bytes`)
- Per-route request counts, duration histograms, and inflight gauges
- Go runtime and `promhttp` internal metrics (GC pauses, goroutine counts, memory usage)

This operational intelligence allows an attacker to identify high-traffic routes, measure backend processing times, and time griefing requests (e.g., flooding during peak inflight periods) with precision. Severity is Medium: no funds are at risk, but the data materially assists denial-of-service and targeted griefing campaigns. [6](#0-5) 

### Likelihood Explanation
Precondition is that the port is reachable. The Kubernetes ingress in `charts/hedera-mirror-rosetta/values.yaml` does not include `/metrics` in its path allowlist, providing a default network-level barrier. [7](#0-6) 

However, any deployment that exposes the rosetta port directly (NodePort, LoadBalancer, host-network pod, misconfigured ingress, internal cluster access) bypasses this. The exploit requires zero credentials, zero tooling beyond `curl`, and is trivially repeatable. Any developer, internal service, or attacker with network access to the rosetta port can exploit this.

### Recommendation
Wrap the `promhttp.Handler()` with an authentication middleware before registering it as the route handler. Mirror the pattern already used in the `rest/` service: read a configurable `metrics.authentication` flag and credentials from config, and return HTTP 401 if the check fails. Example:

```go
func authMiddleware(cfg *config.MetricsAuth, next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        if cfg.Enabled {
            user, pass, ok := r.BasicAuth()
            if !ok || !tsscmp(user, cfg.Username) || !tsscmp(pass, cfg.Password) {
                w.Header().Set("WWW-Authenticate", `Basic realm="Metrics"`)
                http.Error(w, "Unauthorized", http.StatusUnauthorized)
                return
            }
        }
        next.ServeHTTP(w, r)
    })
}
```

Then in `Routes()`:
```go
promhttp.Handler()  →  authMiddleware(cfg, promhttp.Handler())
``` [1](#0-0) 

### Proof of Concept
**Preconditions:** Rosetta service port is reachable (e.g., `localhost:5700` in a default local deployment).

**Steps:**
```bash
# 1. Send unauthenticated GET to /metrics
curl -v http://<rosetta-host>:<port>/metrics

# 2. Expected response: HTTP 200 with full Prometheus text exposition
# Example output excerpt:
# hiero_mirror_rosetta_request_inflight{application="rosetta",method="POST",route="/network/list"} 0
# hiero_mirror_rosetta_request_duration_bucket{application="rosetta",method="POST",route="/block",...} 42
# go_goroutines 18
# ...
```

**Result:** Full metrics dump returned with HTTP 200, no credentials required, confirming zero authentication on the endpoint. [1](#0-0) [8](#0-7)
