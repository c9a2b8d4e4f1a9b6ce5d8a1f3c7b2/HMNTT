### Title
Unbounded Metric Label Cardinality via User-Controlled Path in `metricsMiddleware()`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` helper falls back to the raw `req.path` value when a request does not match any Express route (`req.route` is `undefined`) and `res.locals[requestPathLabel]` is unset. This raw, user-controlled string is then used as the `path` label on four per-route OpenTelemetry instruments (`durationHistogram`, `requestTotalCounter`, `requestSizeHistogram`, `responseSizeHistogram`). An unprivileged attacker can flood the service with requests to unique, never-matched paths, creating an unbounded number of unique label attribute sets in the `MeterProvider`, exhausting the SDK's cardinality budget and causing memory pressure that degrades the Node.js process hosting the fee-estimation API.

### Finding Description

**Exact code path:**

In `toOpenApiPath()` (lines 13–31), when `res.locals[requestPathLabel]` is not set and `req.route` is `undefined` (i.e., the request matched no Express route), the function assigns the raw request path directly:

```js
// metricsHandler.js lines 16-21
if (!path) {
  if (!req.route) {
    path = req.path;          // ← raw, attacker-controlled value
  } else {
    path = (req.baseUrl ?? '') + req.route?.path;
  }
}
``` [1](#0-0) 

`res.locals[requestPathLabel]` is only populated by matched route handlers. In `responseHandler.js`, unmatched routes immediately throw `NotFoundError` before any label is stored:

```js
// responseHandler.js lines 27-31
if (responseData === undefined) {
  // unmatched route will have no response data, pass NotFoundError to next middleware
  throw new NotFoundError();
}
``` [2](#0-1) 

So for every 404 request, `requestPathLabel` is never written to `res.locals`, and `toOpenApiPath` returns the raw `req.path`.

Back in `metricsMiddleware()`, this raw path is used as a label on all four per-route instruments:

```js
// metricsHandler.js lines 189-210
const path = toOpenApiPath(req, res);
const code = String(res.statusCode);
const method = req.method;
const labels = {method, path, code};
...
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
``` [3](#0-2) 

**Root cause / failed assumption:** The code assumes that `toOpenApiPath()` always returns a bounded, template-style path (e.g., `/api/v1/transactions/{id}`). This assumption holds only for matched routes. For unmatched routes it silently returns the raw user input, creating an unbounded label space.

**No cardinality guard exists** anywhere in the application code. The `MeterProvider` is constructed with no explicit `cardinalityLimit` override:

```js
// metricsHandler.js lines 62-64
exporter = new PrometheusExporter({preventServerStart: true});
const meterProvider = new MeterProvider({readers: [exporter]});
``` [4](#0-3) 

The OpenTelemetry SDK default cardinality limit (2000 per instrument) is the only backstop. With four instruments, an attacker can register up to ~8000 unique attribute sets before the limit is hit. Once the limit is reached, new legitimate route combinations are silently dropped (overflow bucket), corrupting all per-route metrics. The memory consumed by 2000 histogram attribute sets (each holding bucket counts for the configured boundaries) is non-trivial and grows with each scrape cycle.

### Impact Explanation
- **Metrics corruption:** Once the cardinality limit is saturated with attacker-generated paths, legitimate route labels (e.g., `/api/v1/network/fees`) are dropped into the overflow bucket, making fee-estimation latency and error-rate data invisible to operators.
- **Memory pressure:** Each unique label set allocates histogram bucket arrays and is retained for the lifetime of the `MeterProvider`. At scale this causes measurable heap growth in the Node.js process.
- **Operational impact:** Operators relying on `durationHistogram` data to detect fee-API degradation lose visibility exactly when an attack is in progress, forcing reliance on fallback/default fee values outside normal design parameters.
- **Severity: Medium–High** (availability and observability impact; no direct data corruption, but monitoring blindness during an attack is a serious operational risk).

### Likelihood Explanation
- **No authentication required.** Any external client can send HTTP requests to arbitrary paths.
- **Trivially automatable.** A simple loop generating unique path segments (e.g., `/api/v1/x/AAAA0001` … `/api/v1/x/AAAA2001`) saturates the cardinality limit in a single burst.
- **Persistent effect.** The `MeterProvider` is initialized once at startup (`initMetrics()` called inside `metricsHandler()`) and never reset; the poisoned label sets persist until process restart.
- **Low cost to attacker.** Each request returns a 404 quickly; the attacker needs only ~2000 requests per instrument to saturate the limit.

### Recommendation
1. **Clamp unknown paths to a sentinel value.** In `toOpenApiPath()`, when `req.route` is `undefined` and `res.locals[requestPathLabel]` is unset, return a fixed string such as `"unknown"` or `apiPrefix + "/unknown"` instead of `req.path`:
   ```js
   if (!req.route) {
     path = 'unknown'; // do NOT use req.path
   }
   ```
2. **Set an explicit cardinality limit** on the `MeterProvider` to a small, known-good value matching the number of real API routes.
3. **Apply rate limiting** (e.g., `express-rate-limit`) before the metrics middleware to reduce the request volume an attacker can generate.

### Proof of Concept
```bash
# Saturate the cardinality limit with unique 404 paths (no auth needed)
for i in $(seq 1 2100); do
  curl -s -o /dev/null "https://<mirror-node-host>/api/v1/nonexistent/path${i}"
done

# After ~2000 requests, scrape metrics and observe:
# - api_request_duration_milliseconds for real routes (e.g. /api/v1/network/fees)
#   is no longer updated (overflow bucket absorbs it)
# - Heap usage of the Node.js process has grown measurably
curl http://<mirror-node-host>/swagger/metrics/
# Real route labels absent or stale; attacker-generated labels fill the output
```
