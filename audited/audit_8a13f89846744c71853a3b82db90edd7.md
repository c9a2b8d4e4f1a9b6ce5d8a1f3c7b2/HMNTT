### Title
Unbounded `maps.Clone(request.Options)` in `ConstructionMetadata()` Enables Memory Exhaustion DoS

### Summary
`ConstructionMetadata()` in `rosetta/app/services/construction_service.go` performs `maps.Clone(request.Options)` on line 156 with no prior bound on the number of entries or total byte size of the `Options` map. The Go HTTP server has no `MaxBytesReader` body-size guard, and the Traefik rate-limiting middleware is disabled by default (`global.middleware: false`). An unauthenticated attacker flooding the `/construction/metadata` endpoint with requests carrying arbitrarily large `Options` maps can exhaust process memory and crash the Rosetta service.

### Finding Description

**Exact code path:**

`rosetta/app/services/construction_service.go`, function `ConstructionMetadata()`, lines 137–180.

```go
// line 141-154: only validates that operation_type key exists and is a valid string
options := request.Options
if options == nil || options[optionKeyOperationType] == nil {
    return nil, errors.ErrInvalidOptions
}
operationType, ok := options[optionKeyOperationType].(string)
...
maxFee, rErr := c.transactionHandler.GetDefaultMaxTransactionFee(operationType)
...
// line 156: full shallow copy of the entire, unbounded map
metadata := maps.Clone(request.Options)
```

**Root cause – three compounding missing controls:**

1. **No HTTP body-size limit.** `rosetta/main.go` lines 220–227 configure the `http.Server` with only time-based limits (`ReadTimeout: 5 s`, `WriteTimeout: 10 s`). There is no `http.MaxBytesReader` wrapping the request body. On a fast link an attacker can deliver hundreds of megabytes within the 5-second read window.

2. **No application-level Options map size check.** Before `maps.Clone` is called, the only validation is that `optionKeyOperationType` is present and is a string (lines 142–149). There is no check on `len(request.Options)` or on the byte length of individual values. Every key-value pair in the JSON body is deserialized into `map[string]any` by the rosetta-sdk-go controller and then handed verbatim to `ConstructionMetadata()`.

3. **Traefik rate-limiting middleware is disabled by default.** `charts/hedera-mirror-rosetta/values.yaml` line 95 sets `global.middleware: false`. The `inFlightReq` (5 per IP) and `rateLimit` (10 req/s per host) rules defined at lines 149–166 are only applied when the middleware is explicitly enabled. A bare deployment has no per-IP or per-host throttle.

**Exploit flow:**

- Attacker sends `POST /construction/metadata` with a JSON body containing a valid `operation_type` value and thousands of additional keys, each with a large string value (e.g., 10 000 keys × 10 KB values = ~100 MB per request body).
- The rosetta-sdk-go controller deserializes the body into `map[string]any` (~100 MB allocation).
- `ConstructionMetadata()` calls `maps.Clone(request.Options)`, doubling the per-request heap footprint to ~200 MB.
- With Go's default goroutine-per-connection model and no in-flight limit, 10–20 concurrent such requests consume 2–4 GB of heap, triggering OOM.

### Impact Explanation

The Rosetta service is the sole path through which exchanges and integrators submit transactions to the Hiero network via this mirror node. Crashing it (OOM kill or unresponsive GC pause) prevents transaction routing for all clients of that node. Because the mirror node is stateless with respect to the Rosetta API, recovery requires a process restart, but the attacker can immediately repeat the attack. In a deployment without the Traefik middleware, there is no automatic recovery or throttle.

### Likelihood Explanation

The endpoint is unauthenticated and publicly reachable (Kubernetes Ingress exposes `/rosetta/construction`). No special knowledge beyond the Rosetta API specification is required. The only precondition is that `operation_type` must be a supported value (`cryptoTransfer` or `cryptoCreateAccount`), which is documented. The attack is trivially scriptable with `curl` or any HTTP client and is fully repeatable. Deployments that have not explicitly enabled the Traefik middleware (the default) have no mitigating control at the infrastructure layer.

### Recommendation

1. **Add `http.MaxBytesReader` in `rosetta/main.go`** before the router, e.g.:
   ```go
   Handler: http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
       r.Body = http.MaxBytesReader(w, r.Body, 1<<20) // 1 MB
       corsMiddleware.ServeHTTP(w, r)
   }),
   ```
2. **Validate `len(request.Options)` in `ConstructionMetadata()`** before line 156, returning `ErrInvalidOptions` if it exceeds a small constant (e.g., 20 keys).
3. **Enable the Traefik middleware by default** (`global.middleware: true`) or document it as a required security control.

import json, sys
opts = {'operation_type': 'cryptoTransfer'}
for i in range(5000):
    opts[f'key_{i}'] = 'A' * 10000

  'network_identifier': {'blockchain': 'Hiero', 'network': 'testnet'},
  'options': opts
})
sys.stdout.write(body)
" > payload.json

# Flood with 20 concurrent requests
for i in $(seq 1 20); do
  curl -s -X POST http://<rosetta-host>:5700/construction/metadata \
    -H 'Content-Type: application/json' \
    -d @payload.json &
done
wait
