### Title
Unauthenticated HTTP Node-Discovery Allows MITM Injection of Attacker-Controlled gRPC Endpoint

### Summary
When `network=other`, `buildNetworkFromMirrorNodes()` fetches the consensus-node list over a plain, unauthenticated HTTP connection with no TLS, no response signature, and no allowlist validation of the returned addresses. A network-adjacent attacker who can intercept that HTTP exchange can substitute a crafted JSON payload that maps every node account ID to an attacker-controlled gRPC host, causing `hiero.ClientForNetwork()` to build a client that routes every subsequent signed transfer transaction exclusively to the attacker's server.

### Finding Description

**Exact code path:**

`pinger/config.go` line 37 documents and defaults `mirrorRest` to a plain `http://` URL:
```
"mirror node REST base URL (required for other), e.g. http://mirror-rest:5551"
``` [1](#0-0) 

`pinger/mirror_node_client.go` line 46 creates a bare `http.Client` with no TLS configuration:
```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
``` [2](#0-1) 

`fetchMirrorNodeNetwork()` issues a plain HTTP GET and decodes whatever JSON body is returned:
```go
resp, err := httpClient.Do(req)
...
json.NewDecoder(resp.Body).Decode(&payload)
``` [3](#0-2) 

The only "validation" applied to each returned node entry is:
- `n.NodeAccountID == ""` (empty string check)
- `hiero.AccountIDFromString()` (format check only, e.g. `"0.0.3"` passes)
- `host == "" || ep.Port == 0` (non-empty check)

No allowlist of legitimate node IPs/domains, no certificate pinning, no response signature: [4](#0-3) 

The resulting map is passed directly to `hiero.ClientForNetwork()`:
```go
client = hiero.ClientForNetwork(netmap)
``` [5](#0-4) 

`submitWithRetry()` then executes signed `TransferTransaction`s against this client on every ticker tick: [6](#0-5) 

**Root cause:** The code assumes the HTTP channel delivering the node list is trustworthy. It is not — it is unauthenticated plaintext HTTP with no integrity protection.

**Failed assumption:** The mirror REST endpoint is assumed to be on a trusted, private network segment. In practice, Kubernetes cluster-internal DNS, ARP, or BGP-level interception makes this assumption invalid for a network-adjacent attacker.

### Impact Explanation

Once the attacker's gRPC server is the sole entry in the network map:

1. **Full transaction interception** — every signed `CryptoTransfer` gRPC call is delivered to the attacker's server. The attacker observes transaction IDs, amounts, sender, and recipient in plaintext.
2. **Persistent DoS** — the attacker drops all calls; the pinger logs failures but the `hiero.Client` never re-discovers real nodes (node list is fetched once at startup).
3. **Fake-success oracle** — the attacker's server returns a well-formed gRPC `OK` + fake `TransactionResponse`, causing `submitWithRetry()` to log `transfer success` and return `nil`, while no real transaction ever reaches the Hedera network. The operator's monitoring shows green while funds are never actually moved.
4. **Signed-transaction harvesting** — the attacker collects fully-signed, serialized transactions. While Hedera's transaction-ID/timestamp scheme limits direct replay, the attacker can forward them selectively or hold them for analysis.

Direct redirection of the *recipient* field is not possible because it is inside the signed body; however items 2 and 3 above constitute an effective loss of the service's operational integrity and can mask real fund-movement failures indefinitely.

### Likelihood Explanation

**Preconditions:**
- `HIERO_MIRROR_PINGER_NETWORK=other` (required for the `buildNetworkFromMirrorNodes` path to execute)
- `HIERO_MIRROR_PINGER_REST` set to a plain `http://` URL (the documented default example)
- Network-adjacent position: same Kubernetes namespace (pod-to-pod), same subnet, or control of a DNS/ARP entry

**Feasibility:** In a Kubernetes deployment a compromised sidecar, a rogue pod in the same namespace, or a DNS-spoofing attack against the cluster-internal service name (`mirror-rest`) is sufficient. No privileged credentials are required. The attack is repeatable on every pinger restart because node discovery runs once at startup with no re-validation.

### Recommendation

1. **Enforce HTTPS for `mirrorRest`**: Reject any `mirrorRest` URL that does not begin with `https://` in `loadConfig()`.
2. **Pin the expected node account IDs**: Accept a static allowlist of legitimate consensus-node account IDs (e.g. `0.0.3`–`0.0.28` for mainnet) and reject any response that introduces account IDs outside that set.
3. **Validate returned IP/domain against a known allowlist**: Cross-check returned `ip_address_v4` / `domain_name` values against a hardcoded or operator-supplied allowlist before adding them to the network map.
4. **Re-fetch and diff periodically**: Re-run node discovery on a schedule and alert/abort if the returned set diverges significantly from the previous one.
5. **Minimum node count guard**: Refuse to build a client if fewer than N (e.g. 3) distinct nodes are returned, making a single-entry injection immediately fatal.

### Proof of Concept

```
# 1. Start a fake gRPC server that accepts any CryptoTransfer and returns OK
#    (e.g. using grpcurl in server mode or a trivial Go gRPC stub)
attacker-grpc-server --listen 0.0.0.0:50211

# 2. Position as MITM for HTTP traffic to mirror-rest:5551
#    (DNS poisoning, ARP spoofing, or rogue Kubernetes Service pointing to attacker pod)

# 3. Serve the following JSON on GET /api/v1/network/nodes:
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{
      "ip_address_v4": "<attacker-ip>",
      "domain_name": "",
      "port": 50211
    }]
  }],
  "links": {"next": null}
}

# 4. Start the pinger with:
HIERO_MIRROR_PINGER_NETWORK=other \
HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551 \
HIERO_MIRROR_PINGER_OPERATOR_KEY=<real-key> \
./pinger

# 5. Observe: pinger logs "transfer success", attacker-grpc-server logs
#    the full serialized signed CryptoTransfer protobuf. No transaction
#    ever reaches the real Hedera network.
```
