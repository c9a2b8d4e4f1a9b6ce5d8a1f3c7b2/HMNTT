### Title
`contractFilterCheck()` Always Compares Long-Zero Address, Causing Permanent Trace Suppression for CREATE2 Contracts

### Summary
In `MirrorOperationActionTracer.tracePostExecution()`, after resolving the recipient entity from the frame's EVM address, the code computes the address for filter comparison using `toEvmAddress(entity.getId())`, which unconditionally produces a long-zero format address (12 zero bytes + 8-byte entity num). For contracts deployed via CREATE2 that have a custom EVM address stored in the entity record, this computed address will never match the CREATE2 address an operator places in the filter set, causing `contractFilterCheck()` to always return `true` (suppress the trace). Any unprivileged user calling such a contract causes the trace record to be permanently dropped, producing missing exports in mirror nodes.

### Finding Description

**Code path:**

`MirrorOperationActionTracer.tracePostExecution()` — lines 59–68:

```java
final var recipientAddress = frame.getRecipientAddress();
final var recipientNum = recipientAddress != null
        ? commonEntityAccessor.get(recipientAddress, Optional.empty())
        : Optional.empty();

if (recipientNum.isPresent()
        && traceProperties.contractFilterCheck(
                CommonUtils.hex(toEvmAddress(((Entity) recipientNum.get()).getId())))) {
    return;   // trace suppressed
}
```

`CommonEntityAccessor.get(Address, Optional<Long>)` — lines 29–36:

```java
if (ConversionUtils.isLongZeroAddress(addressBytes)) {
    return getEntityByMirrorAddressAndTimestamp(address, timestamp);
} else {
    return getEntityByEvmAddressTimestamp(address.toArrayUnsafe(), timestamp);
}
```

When `recipientAddress` is a CREATE2 address (non-long-zero), the accessor correctly finds the entity by its stored `evmAddress` column. The entity is returned with its numeric `id` (e.g., `1000`).

`DomainUtils.toEvmAddress(long num)` — lines 401–403:

```java
public static byte[] toEvmAddress(long num) {
    return Bytes.concat(MIRROR_PREFIX, Longs.toByteArray(num));
}
```

This **always** produces the long-zero format: `0x0000000000000000000000000000000000000003e8` for entity ID 1000.

`TraceProperties.contractFilterCheck(String contract)` — lines 33–35:

```java
public boolean contractFilterCheck(String contract) {
    return !getContract().isEmpty() && !getContract().contains(contract);
}
```

**Root cause:** The filter set is populated by the operator with hex EVM addresses (validated only for length/hex format by `@Hex`). If the operator adds a CREATE2 address (e.g., `"abcd1234..."`), the comparison is always between that CREATE2 address and the long-zero address derived from the entity ID. They can never match. `contractFilterCheck` always returns `true`, and the trace is always suppressed.

**Failed assumption:** The code assumes `toEvmAddress(entity.getId())` produces the same address format that operators use when configuring the filter. This holds only for contracts without a custom EVM address. For CREATE2 contracts, the entity's canonical address is its `evmAddress` field, not the long-zero derivation from its numeric ID — as correctly handled in `CommonEntityAccessor.evmAddressFromId()` (lines 86–107), which checks `entity.getEvmAddress()` first.

### Impact Explanation
Any unprivileged user who submits a contract call targeting a CREATE2 contract that the operator has added to the trace filter will have their call's trace record silently dropped. The mirror node never receives the trace export for that execution. Operators relying on the filter to audit or monitor specific CREATE2 contracts receive no data, creating a blind spot. This is a deterministic, permanent integrity failure in the trace export pipeline for all CREATE2 contracts in the filter set.

### Likelihood Explanation
The precondition is that an operator has configured `hiero.mirror.web3.evm.trace.contract` with at least one CREATE2 address. This is a realistic operational scenario — operators commonly want to monitor specific high-value or suspicious contracts by their canonical EVM address. Once that configuration exists, any unprivileged user who calls the contract triggers the suppression. No special knowledge, credentials, or timing is required. The bug is deterministic and repeatable on every invocation.

### Recommendation
Replace `toEvmAddress(entity.getId())` with logic that respects the entity's actual canonical EVM address, mirroring the pattern already used in `CommonEntityAccessor.evmAddressFromId()`:

```java
// In MirrorOperationActionTracer.tracePostExecution()
Entity entity = (Entity) recipientNum.get();
byte[] addressBytes;
if (entity.getEvmAddress() != null && entity.getEvmAddress().length == EVM_ADDRESS_LENGTH) {
    addressBytes = entity.getEvmAddress();
} else {
    addressBytes = toEvmAddress(entity.getId());
}
if (traceProperties.contractFilterCheck(CommonUtils.hex(addressBytes))) {
    return;
}
```

Alternatively, normalize the filter set at configuration load time to always store long-zero addresses, and document that CREATE2 addresses must be entered as their long-zero equivalent.

### Proof of Concept

**Preconditions:**
1. A contract exists in the mirror node DB with entity ID `1000` and a CREATE2 EVM address `0xabcd1234abcd1234abcd1234abcd1234abcd1234`.
2. Operator sets `hiero.mirror.web3.evm.trace.contract=abcd1234abcd1234abcd1234abcd1234abcd1234` and `hiero.mirror.web3.evm.trace.enabled=true`.

**Trigger:**
Any unprivileged user submits a contract call to `0xabcd1234abcd1234abcd1234abcd1234abcd1234`.

**Execution trace:**
1. `frame.getRecipientAddress()` → `0xabcd1234abcd1234abcd1234abcd1234abcd1234`
2. `commonEntityAccessor.get(...)` → entity with `id=1000`, `evmAddress=0xabcd...`
3. `toEvmAddress(1000)` → `0x00000000000000000000000000000000000003e8`
4. `CommonUtils.hex(...)` → `"00000000000000000000000000000000000003e8"`
5. `contractFilterCheck("00000000000000000000000000000000000003e8")`:
   - filter set = `{"abcd1234abcd1234abcd1234abcd1234abcd1234"}`
   - `"00000000000000000000000000000000000003e8"` ∉ filter set → returns `true`
6. `return;` — trace is suppressed, record never exported to mirror node.

    public Address evmAddressFromId(EntityId entityId, final Optional<Long> timestamp) {
        Entity entity = timestamp
                .map(t -> entityRepository
                        .findActiveByIdAndTimestamp(entityId.getId(), t)
                        .orElse(null))
                .orElseGet(() -> entityRepository
                        .findByIdAndDeletedIsFalse(entityId.getId())
                        .orElse(null));
        if (entity == null) {
            return Address.ZERO;
        }

        if (entity.getEvmAddress() != null) {
            return Address.wrap(org.apache.tuweni.bytes.Bytes.wrap(entity.getEvmAddress()));
        }

        if (entity.getAlias() != null && entity.getAlias().length == EVM_ADDRESS_LENGTH) {
            return Address.wrap(org.apache.tuweni.bytes.Bytes.wrap(entity.getAlias()));
        }

        return toAddress(entityId);
    }
