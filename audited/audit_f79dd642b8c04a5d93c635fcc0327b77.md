### Title
Directory Substitution Attack on `/tmp/alive` Bypasses Liveness Probe Indefinitely

### Summary
An unprivileged user with filesystem access to `/tmp` can create `/tmp/alive` as a directory before or instead of the expected regular file. Because `os.WriteFile` in the heartbeat goroutine silently discards errors (via `_ =`), the real heartbeat is never written, yet `os.Stat` in the healthcheck succeeds on a directory and returns a controllable `ModTime()`, causing the liveness probe to permanently report the container as alive even when the service is functionally dead.

### Finding Description
**Heartbeat writer** — `pinger/main.go`, `main()`, line 36:
```go
_ = os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644)
```
The error from `os.WriteFile` is unconditionally discarded. If `/tmp/alive` is already a directory, `os.WriteFile` returns `EISDIR` and the heartbeat timestamp is never updated — silently.

**Liveness checker** — `pinger/cmd/healthcheck/main.go`, `main()`, lines 27–38:
```go
fi, err := os.Stat("/tmp/alive")
if err != nil { ... os.Exit(1) }
age := time.Since(fi.ModTime())
if age > 2*time.Minute { ... os.Exit(1) }
os.Exit(0)
```
`os.Stat` succeeds on a directory just as it does on a regular file. There is no `fi.Mode().IsRegular()` check. The only guard is `ModTime()` freshness, which an attacker controls on a directory by creating/removing entries inside it (`touch /tmp/alive/x && rm /tmp/alive/x` updates directory mtime).

**Root cause / failed assumption:** The code assumes `/tmp/alive` is always a regular file. Neither the writer nor the checker enforces this invariant. [1](#0-0) [2](#0-1) 

### Impact Explanation
The Kubernetes liveness exec probe calls `healthcheck live`. If it exits 0, the pod is considered alive and is never restarted. With this attack, the probe exits 0 indefinitely even though the pinger service's heartbeat mechanism is broken. The pod is stuck in a permanently "alive but non-functional" state, defeating the entire purpose of the liveness probe and preventing automatic recovery. Severity: Medium (griefing / availability impact, no economic loss).

### Likelihood Explanation
Precondition: any process or user that can write to `/tmp` inside the container (e.g., a compromised sidecar, a shared-volume mount, or a container escape partial). `/tmp` is world-writable by default on Linux. The attack is a single `mkdir /tmp/alive` command executed before the pinger starts, or at any point while the pinger is running (the directory replaces the file on the next write attempt if the attacker first removes the file and creates the directory). Refreshing the mtime requires only `touch /tmp/alive/x && rm /tmp/alive/x`, which is trivially scriptable. Fully repeatable.

### Recommendation
1. **In the healthcheck**, add a file-type check immediately after `os.Stat`:
   ```go
   if !fi.Mode().IsRegular() {
       fmt.Fprintln(os.Stderr, "alive is not a regular file")
       os.Exit(1)
   }
   ```
2. **In the heartbeat writer**, do not discard the error; log it so the broken state is visible:
   ```go
   if err := os.WriteFile("/tmp/alive", ...); err != nil {
       log.Printf("heartbeat write failed: %v", err)
   }
   ```
3. Optionally write to a path outside world-writable `/tmp`, or use a file descriptor opened at startup and kept open, so the path cannot be replaced. [3](#0-2) [4](#0-3) 

### Proof of Concept
```bash
# 1. Before (or while) the pinger container starts, inside the container:
rm -f /tmp/alive          # remove any existing file (if pinger already ran)
mkdir /tmp/alive           # create as directory

# 2. Pinger's heartbeat goroutine now silently fails every 15 s:
#    os.WriteFile("/tmp/alive", ...) => EISDIR, error discarded

# 3. Keep directory mtime fresh (run in background):
while true; do
    touch /tmp/alive/x && rm /tmp/alive/x
    sleep 60
done &

# 4. Trigger the liveness probe manually to confirm false-positive:
/healthcheck live; echo "exit: $?"
# => exit: 0   (reports alive, pod never restarted)

# 5. Confirm the real heartbeat content is absent:
cat /tmp/alive   # => Is a directory — no timestamp content
```
