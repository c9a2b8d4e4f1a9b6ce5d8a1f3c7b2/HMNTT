### Title
Unbounded Prometheus Label Cardinality via Arbitrary HTTP Method Strings in MetricsMiddleware

### Summary
`MetricsMiddleware` in `rosetta/app/middleware/metrics.go` delegates instrumentation to `weaveworks/common/middleware.Instrument`, which reads `r.Method` directly from the incoming HTTP request and uses it verbatim as the `method` label value in all four Prometheus metric vectors. Because Go's `net/http` server accepts any syntactically valid token as an HTTP method, an unprivileged attacker can send requests with an unbounded number of unique method strings, causing each unique value to allocate a new time series in every `HistogramVec`/`GaugeVec`, exhausting process memory.

### Finding Description
**Exact code location:** `rosetta/app/middleware/metrics.go`, lines 76–83, `MetricsMiddleware()`.

```go
func MetricsMiddleware(next http.Handler) http.Handler {
    return middleware.Instrument{
        Duration:         requestDurationHistogram,   // labels: method, route, status_code, ws
        InflightRequests: requestInflightGauge,       // labels: method, route
        RequestBodySize:  requestBytesHistogram,      // labels: method, route
        ResponseBodySize: responseBytesHistogram,     // labels: method, route
        RouteMatcher:     next.(middleware.RouteMatcher),
    }.Wrap(next)
}
```

`middleware.Instrument.Wrap()` (from `github.com/weaveworks/common/middleware`) records metrics on every request by calling `.With(prometheus.Labels{"method": r.Method, ...})` on each `HistogramVec`/`GaugeVec`. The value of `r.Method` is taken verbatim from the HTTP request line — Go's `net/http` parser accepts any RFC 7230 token as a method name (e.g., `FOOBAR`, `AAAA1`, `AAAA2`, …) without restricting it to the standard set (GET, POST, etc.).

**Root cause:** No allowlist, no normalization, and no cardinality cap is applied to the `method` label before it is passed to Prometheus. Each unique `(method, route, status_code, ws)` tuple causes `prometheus-client-go` to allocate a new internal `histogram` struct (with 6 bucket counters for `requestDurationHistogram`) and store it in a `sync.Map` inside the `HistogramVec`. This allocation is permanent for the lifetime of the process.

**Failed assumption:** The code assumes the `method` label will only ever take one of the ~9 standard HTTP method values. In reality, Go's HTTP server places no such restriction.

### Impact Explanation
Each unique method string creates new time series across all four metric vectors simultaneously. With 6 buckets in `requestDurationHistogram` and 5 in the size histograms, each new method string allocates on the order of hundreds of bytes of heap per unique `(method, route)` pair. Sending millions of requests with distinct method strings (trivially scriptable) will grow the process heap without bound until the OS OOM-killer terminates the Rosetta node, causing a complete denial of service. Because Rosetta is a blockchain API gateway, this disrupts all downstream consumers (wallets, exchanges, validators) that depend on it.

### Likelihood Explanation
The Rosetta HTTP port is exposed to the network with no authentication required for API calls. An attacker needs only a TCP connection and the ability to send raw HTTP requests — no credentials, no special knowledge of the API. The attack is trivially automatable: a simple loop sending `METHOD<N> /network/list HTTP/1.1` with an incrementing N is sufficient. The `ReadHeaderTimeout` and `ReadTimeout` configured in `main.go` (lines 221–226) throttle individual connection duration but do not limit the total number of connections or unique method values, so the attacker simply opens many short-lived connections.

### Recommendation
1. **Normalize the method label** before recording: replace any value not in the standard set (`GET`, `POST`, `PUT`, `DELETE`, `PATCH`, `HEAD`, `OPTIONS`, `CONNECT`, `TRACE`) with a sentinel such as `"unknown"`.
2. **Wrap `middleware.Instrument`** with a thin adapter that sanitizes `r.Method` before the weaveworks middleware sees it, or subclass/replace the label extraction logic.
3. **Add a Prometheus cardinality limit** using `prometheus.Labels` with a fixed allowlist, or use `promauto` with a `CurryWith` that pre-fixes the method label to a validated value.

Example sanitization:
```go
func sanitizeMethod(m string) string {
    switch m {
    case "GET", "POST", "PUT", "DELETE", "PATCH", "HEAD", "OPTIONS", "CONNECT", "TRACE":
        return m
    default:
        return "unknown"
    }
}
```

### Proof of Concept
```bash
# Send 100,000 requests with unique HTTP method strings to exhaust label cardinality
for i in $(seq 1 100000); do
  printf "METHOD${i} /network/list HTTP/1.1\r\nHost: localhost:8080\r\nContent-Type: application/json\r\nContent-Length: 2\r\n\r\n{}" \
    | nc -q1 localhost 8080 &
done
# Monitor RSS growth:
watch -n1 'ps aux | grep rosetta'
```
