### Title
Phantom Topic Subscription Resource Exhaustion via `checkTopicExists=false` Bypass

### Summary
When `hiero.mirror.grpc.checkTopicExists` is set to `false`, the `topicExists()` method in `TopicMessageServiceImpl` synthesizes a phantom `Entity` object for any topic ID not found in the database, allowing the subscription to proceed unconditionally. Any unauthenticated external user can exploit this to open unbounded subscriptions against arbitrary non-existent topic IDs, consuming DB connections, listener threads, and memory with no rate-limiting defense in the gRPC module.

### Finding Description

**Code path:**

`TopicMessageServiceImpl.java`, `topicExists()`, lines 94–106:

```java
private Mono<?> topicExists(TopicMessageFilter filter) {
    var topicId = filter.getTopicId();
    return Mono.justOrEmpty(entityRepository.findById(topicId.getId()))
            .switchIfEmpty(
                    grpcProperties.isCheckTopicExists()
                            ? Mono.error(new EntityNotFoundException(topicId))
                            : Mono.just(Entity.builder()   // ← phantom entity
                                    .memo("")
                                    .type(EntityType.TOPIC)
                                    .build()))
            .filter(e -> e.getType() == EntityType.TOPIC)
            .switchIfEmpty(Mono.error(new IllegalArgumentException("Not a valid topic")));
}
```

When `checkTopicExists=false` and the topic is absent from the DB, the `switchIfEmpty` branch at line 100 constructs a synthetic `Entity` with `type=TOPIC`. This passes the `.filter(e -> e.getType() == EntityType.TOPIC)` check at line 104, so `topicExists()` succeeds and the full subscription pipeline is activated.

**What the subscription pipeline then does** (`subscribeTopic()`, lines 59–92):

- Line 61: `TopicContext topicContext = new TopicContext(filter)` — created unconditionally before `topicExists()` resolves.
- Line 63: `topicMessageRetriever.retrieve(filter, true)` — issues a DB query for historical messages (returns nothing, but consumes a DB connection).
- Lines 67–70: A safety-check `Mono.delay(1s)` fires a gap-recovery DB query after 1 second.
- Line 120: `topicListener.listen(newFilter)` — registers a live listener subscription that persists indefinitely when no `endTime` is set.
- Lines 88–91: `subscriberCount` is incremented and the subscription is held open.

**Root cause:** The `checkTopicExists=false` branch makes a failed assumption — that the absence of a topic in the DB is a legitimate operational state (e.g., topic not yet indexed) rather than an attacker-supplied phantom ID. There is no secondary guard to limit how many such phantom subscriptions can be opened.

**Why existing checks fail:**

- `maxConcurrentCallsPerConnection=5` (`NettyProperties.java` line 14) is per-connection only. An attacker opens N connections × 5 calls = 5N concurrent phantom subscriptions.
- The `subscriberCount` gauge (line 48) is metrics-only; it enforces no ceiling.
- Rate limiting (`ThrottleConfiguration`, `ThrottleManagerImpl`) exists only in the `web3` module — there is **no** equivalent in the `grpc` module.
- `TopicMessageFilter` validation (`@Min`, `@NotNull`) only checks field format, not topic existence.

### Impact Explanation

Each phantom subscription holds: one or more DB connections (historical retrieval + safety-check polling), a live listener slot in the shared listener buffer (`maxBufferSize=16384`), and a `TopicContext` object in heap. With no end time, subscriptions are permanent until the client disconnects. An attacker sustaining many connections can exhaust the HikariCP connection pool (alerting threshold at 75% per the Prometheus rule), fill the listener buffer causing `BUFFER_OVERFLOW` errors for legitimate subscribers, and drive JVM heap toward the 80% alert threshold. This constitutes a practical denial-of-service against the gRPC service.

### Likelihood Explanation

Precondition: `checkTopicExists=false` must be configured. This is non-default (`true` is the default per `GrpcProperties.java` line 19 and docs line 409), but it is a documented operator option used when topics may not yet be indexed. Any deployment using this setting is fully exposed. The gRPC endpoint is unauthenticated and publicly reachable (port 5600). No exploit tooling beyond a standard gRPC client is required. The attack is trivially repeatable and scriptable.

### Recommendation

1. **Add a global subscriber ceiling**: Enforce a configurable maximum on `subscriberCount` and reject new subscriptions when the limit is reached, regardless of `checkTopicExists`.
2. **Add per-IP or per-connection subscription rate limiting** in the gRPC module, mirroring the `ThrottleConfiguration` pattern used in `web3`.
3. **Require an `endTime` for phantom subscriptions**: When `checkTopicExists=false` and the topic is not found, mandate a bounded `endTime` so phantom subscriptions self-terminate.
4. **Reconsider the `checkTopicExists=false` semantic**: Document that this mode should only be used in controlled/internal environments, or replace it with a whitelist of known-valid-but-not-yet-indexed topic IDs.

    public Flux<TopicMessage> subscribeTopic(TopicMessageFilter filter) {
        log.info("Subscribing to topic: {}", filter);
        TopicContext topicContext = new TopicContext(filter);

        Flux<TopicMessage> historical = topicMessageRetriever.retrieve(filter, true);
        Flux<TopicMessage> live = Flux.defer(() -> incomingMessages(topicContext));

        // Safety Check - Polls missing messages after 1s if we are stuck with no data
        Flux<TopicMessage> safetyCheck = Mono.delay(Duration.ofSeconds(1L))
                .filter(_ -> !topicContext.isComplete())
                .flatMapMany(_ -> missingMessages(topicContext, null))
                .subscribeOn(Schedulers.boundedElastic());

        Flux<TopicMessage> flux = historical
                .concatWith(Flux.merge(safetyCheck, live).takeUntilOther(pastEndTime(topicContext)))
                .filter(t -> {
                    TopicMessage last = topicContext.getLast();
                    return last == null || t.getSequenceNumber() > last.getSequenceNumber();
                });

        if (filter.getEndTime() != null) {
            flux = flux.takeWhile(t -> t.getConsensusTimestamp() < filter.getEndTime());
        }

        if (filter.hasLimit()) {
            flux = flux.take(filter.getLimit());
        }

        return topicExists(filter)
                .thenMany(flux.doOnNext(topicContext::onNext)
                        .doOnSubscribe(s -> subscriberCount.incrementAndGet())
                        .doFinally(s -> subscriberCount.decrementAndGet())
                        .doFinally(topicContext::finished));
    }
