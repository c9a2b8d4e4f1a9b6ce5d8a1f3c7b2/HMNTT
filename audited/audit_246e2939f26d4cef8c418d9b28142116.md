### Title
Throttle Budget Permanently Consumed via `IllegalArgumentException` from Odd-Length Hex `data` Field Bypassing `InvalidParametersException` Catch Block

### Summary
The `call()` method in `ContractController` catches only `InvalidParametersException` to trigger `throttleManager.restore()`. However, the `@Hex` Bean Validation constraint on the `data` field permits odd-length hex strings (e.g., `"1"`, `"1aa"`, `"0xa"`), which pass validation but cause `hexToBytes()` to throw `IllegalArgumentException` inside `constructServiceParameters()`. This exception is not caught by the narrow `catch (InvalidParametersException e)` block, so `throttleManager.restore()` is never called and the consumed gas tokens are permanently lost. Any unauthenticated user can exploit this to exhaust the gas throttle bucket and deny service to legitimate callers.

### Finding Description

**Exact code path:**

`call()` in `ContractController.java` (lines 38–51) consumes throttle tokens unconditionally at line 40, then calls `constructServiceParameters()` at line 43. The only restoration path is the `catch (InvalidParametersException e)` block at line 46. [1](#0-0) 

Inside `constructServiceParameters()`, line 80 calls `hexToBytes(data)`: [2](#0-1) 

`hexToBytes()` in `BytesDecoder.java` (lines 84–88) catches `DecoderException` and re-throws it as `IllegalArgumentException` — not `InvalidParametersException`: [3](#0-2) 

**Root cause — validator/decoder mismatch:**

The `@Hex` validator on the `data` field uses the regex `^(0x)?[0-9a-fA-F]{0,1048576}$`, which accepts any number of hex characters including odd counts: [4](#0-3) 

`Hex.decodeHex()` (Apache Commons Codec) requires an even number of hex characters and throws `DecoderException` for odd-length input. The `@Hex` validator never enforces even-length, so odd-length strings pass `@Valid` but fail inside `hexToBytes()`.

**Why existing checks fail:**

- `@Valid` / `@Hex` — passes odd-length hex strings; no even-length check.
- `catch (InvalidParametersException e)` — does not catch `IllegalArgumentException`; the exception propagates directly to `GenericControllerAdvice.badRequest()` (lines 89–93), which returns HTTP 400 but never calls `throttleManager.restore()`. [5](#0-4) 

The existing test `callBadRequestWithInvalidHexData` with values `{"1", "1aa"}` confirms this path returns 400 with "Odd number of characters" — proving the `IllegalArgumentException` escapes the controller's catch block: [6](#0-5) 

### Impact Explanation
Each such request consumes `throttleProperties.scaleGas(request.getGas())` tokens from the `gasLimitBucket` without restoration. An attacker sending requests at the maximum gas limit (`15_000_000` by default) with odd-length `data` can drain the gas throttle bucket rapidly, causing all subsequent legitimate requests to receive HTTP 429 (Too Many Requests). This is a targeted, low-cost DoS against the gas throttle subsystem. [7](#0-6) 

### Likelihood Explanation
The exploit requires zero privileges — only the ability to send an HTTP POST to `/api/v1/contracts/call`. The payload is trivial to construct (any odd-length hex string in the `data` field). It is fully repeatable and automatable. The existing test suite already demonstrates the exact failure mode, confirming the path is reachable in production.

### Recommendation
1. **Fix the `@Hex` validator** to enforce even-length hex character counts (after stripping the optional `0x` prefix), eliminating the mismatch with `hexToBytes()`.
2. **Broaden the catch block** in `call()` to also catch `IllegalArgumentException` (and ideally any `RuntimeException` that represents a user-input error), ensuring `throttleManager.restore()` is always called when no EVM processing has occurred:
   ```java
   } catch (InvalidParametersException | IllegalArgumentException e) {
       throttleManager.restore(request.getGas());
       throw e;
   }
   ```
3. **Remove the dead try/catch** around `request.getData()` in `constructServiceParameters()` (lines 67–72) — `getData()` is a plain getter and cannot throw; the catch block gives a false sense of safety. [8](#0-7) 

### Proof of Concept

```bash
# Single request — consumes max gas tokens, no restoration
curl -X POST http://<host>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{
    "to": "0x00000000000000000000000000000000000004e2",
    "data": "0xa",
    "gas": 15000000
  }'
# Response: HTTP 400 {"messages":[{"message":"Bad Request","detail":"Odd number of characters."}]}
# Effect: 15,000,000 gas tokens consumed, NOT restored.

# Repeat in a loop to exhaust the gas throttle bucket:
for i in $(seq 1 100); do
  curl -s -X POST http://<host>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{"to":"0x00000000000000000000000000000000000004e2","data":"0xa","gas":15000000}'
done
# Subsequent legitimate calls receive HTTP 429 "Gas per second rate limit exceeded."
```
