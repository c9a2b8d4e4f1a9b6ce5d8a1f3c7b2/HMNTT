### Title
Unauthenticated `/network/status` with Disabled-by-Default Rate Limiting Enables Distributed DB-Exhaustion Flood

### Summary
`NetworkStatus()` in `rosetta/app/services/network_service.go` performs two uncached database queries on every invocation with no HTTP-level authentication or API key check. The Traefik middleware chain that provides the only rate-limiting defense is disabled by default (`global.middleware: false`), and even when enabled, its per-source-IP in-flight limit is trivially bypassed by a distributed multi-IP attack, leaving the database connection pool exposed to exhaustion.

### Finding Description

**Exact code path:**

`NetworkStatus()` at [1](#0-0)  performs three operations per request:
1. `n.RetrieveGenesis(ctx)` — delegates to `blockRepository.RetrieveGenesis()` [2](#0-1)  which calls `initGenesisRecordFile` guarded by `sync.Once` [3](#0-2)  — **cached after first call, not a repeated DB hit**.
2. `n.RetrieveLatest(ctx)` — delegates to `blockRepository.RetrieveLatest()` [4](#0-3)  which executes `selectLatestWithIndex` (`SELECT ... FROM record_file ORDER BY index DESC LIMIT 1`) on **every single request** with no caching.
3. `n.addressBookEntryRepo.Entries(ctx)` — executes `latestNodeServiceEndpoints` [5](#0-4)  on **every single request**: a complex 4-table JOIN (`address_book`, `address_book_entry`, `node`, `address_book_service_endpoint`) with `string_agg` aggregation, also with no caching.

**Root cause — middleware disabled by default:**

The Traefik middleware chain is the sole rate-limiting layer. The template guard is:

```
{{ if and .Values.global.middleware .Values.middleware }}
``` [6](#0-5) 

The default value is `global.middleware: false` [7](#0-6)  — so **no middleware is deployed in a default installation**. There is no application-level authentication, no API key, and no connection-level throttle inside the Go service itself.

**Why IP-based limiting fails even when middleware is enabled:**

When an operator does enable the middleware, the two relevant rules are:
- `inFlightReq: amount: 5, sourceCriterion: ipStrategy: depth: 1` — limits 5 concurrent requests **per source IP** [8](#0-7) 
- `rateLimit: average: 10, sourceCriterion: requestHost: true` — limits 10 req/s **globally per request host** [9](#0-8) 

A distributed attacker using N source IPs gets N×5 concurrent slots through `inFlightReq` because each IP has its own independent bucket. The `rateLimit` is global (per host), so it caps total throughput at 10 req/s — but only when middleware is enabled. In the default deployment it is absent entirely.

### Impact Explanation

Each `/network/status` request issues two uncached SQL queries against the mirror-node PostgreSQL database. The `Entries()` query is a multi-table JOIN with aggregation across `address_book_entry`, `address_book_service_endpoint`, and `node`. Under a sustained flood (default deployment: unlimited; middleware-enabled deployment with distributed attack: N×5 concurrent connections), the database connection pool is exhausted, causing all other mirror-node services (REST API, gRPC, importer) that share the same database to fail or time out. This is a griefing/availability impact with no economic damage to on-chain users, consistent with the Medium severity classification.

### Likelihood Explanation

The preconditions require zero privileges: the endpoint is publicly reachable, accepts unauthenticated POST requests, and the Rosetta spec itself mandates it be open. Renting a small botnet or using cloud spot instances across multiple IP ranges is trivially cheap. The default Helm chart ships with `global.middleware: false`, meaning any operator who deploys without explicitly overriding this value has no rate limiting at all. The attack is fully repeatable and requires no special knowledge beyond the standard Rosetta API specification.

### Recommendation

1. **Enable middleware by default**: Change `global.middleware: false` to `true` in `charts/hedera-mirror-rosetta/values.yaml` line 95, or document it as a required production setting with a clear warning.
2. **Cache `Entries()` results**: Add a short TTL in-process cache (e.g., 30–60 seconds) inside `addressBookEntryRepository.Entries()` in `rosetta/app/persistence/address_book_entry.go`. The address book changes infrequently; re-querying it on every `/network/status` call is unnecessary.
3. **Switch `inFlightReq` criterion to global or add a global cap**: Replace `sourceCriterion: ipStrategy: depth: 1` with a global in-flight cap, or add a second `inFlightReq` rule without a source criterion to bound total concurrent DB connections regardless of attacker IP count.
4. **Add application-level concurrency limiting**: Implement a semaphore or connection-pool size guard inside the Go service so that even without Traefik, the number of simultaneous DB queries from this endpoint is bounded.

### Proof of Concept

```bash
# Default deployment (global.middleware: false) — no rate limiting present
# Run from 10+ distinct source IPs or via a proxy pool:

for i in $(seq 1 500); do
  curl -s -X POST https://<rosetta-host>/network/status \
    -H "Content-Type: application/json" \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' &
done
wait

# Each request triggers:
#   SELECT ... FROM record_file ORDER BY index DESC LIMIT 1
#   SELECT ... FROM address_book JOIN address_book_entry JOIN node JOIN address_book_service_endpoint ...
#
# With middleware disabled (default), all 500 requests hit the DB concurrently.
# With middleware enabled but N source IPs: each IP contributes 5 concurrent slots,
# bypassing the per-IP inFlightReq limit while the global rateLimit only applies
# when middleware is active.
#
# Observable result: pg_stat_activity shows connection pool saturation;
# other mirror-node API endpoints begin returning 503/timeout errors.
```

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}
