### Title
Unauthenticated Mirror-Node Response Allows Arbitrary gRPC Endpoint Injection via Unvalidated ServiceEndpoints

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` fetches the consensus-node address list over a plain, unauthenticated HTTP connection and blindly trusts every IP/hostname and port returned in `ServiceEndpoints`. There is no validation against loopback, private, or otherwise attacker-controlled addresses. Any party who can influence the HTTP response — via DNS hijacking, MITM on the unencrypted channel, or a misconfigured `mirrorRest` URL — can inject arbitrary gRPC endpoints, causing the Hiero SDK client to route all signed transactions away from real consensus nodes.

### Finding Description

**Exact code path:**

`pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 74–132.

The function issues a plain `http.Get` (no TLS, no certificate pinning, no response signing):

```go
// line 79-84
req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
...
resp, err := httpClient.Do(req)
```

It then decodes the JSON body and, for every `serviceEndpoint`, unconditionally builds a network map entry:

```go
// lines 113-124
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId   // ← no IP/range validation whatsoever
}
```

The **only** guard is `host == "" || ep.Port == 0` (line 118). There is no check for:
- loopback (`127.0.0.0/8`, `::1`)
- RFC-1918 private ranges
- link-local addresses
- domain names resolving to any of the above

The resulting `network` map is passed directly to `hiero.ClientForNetwork(netmap)` in `sdk_client.go` line 22, which the SDK uses as the exclusive set of gRPC targets for all subsequent transactions.

**Root cause / failed assumption:** The code assumes the mirror-node REST endpoint is authoritative and trustworthy. It makes no attempt to authenticate the response (no TLS mutual auth, no HMAC, no signed payload), and it performs no semantic validation of the addresses it receives.

**Exploit flow:**

1. Attacker positions themselves to influence the HTTP response from `cfg.mirrorRest`. Viable vectors:
   - **DNS poisoning** of the hostname in `HIERO_MIRROR_PINGER_REST` (e.g., `mirror-rest` in a Kubernetes cluster with lax network policy).
   - **HTTP MITM** on the same L2 segment (ARP spoofing) — trivially possible because the default URL is `http://` (no TLS).
   - **Misconfigured `HIERO_MIRROR_PINGER_REST`** pointing to an attacker-controlled host (supply-chain / config injection).
2. Attacker's server returns a well-formed `nodesEnvelope` JSON where each `nodeEntry` carries a legitimate-looking `node_account_id` (e.g., `0.0.3`) but `service_endpoints` with `ip_address_v4: "127.0.0.1"` and a port the attacker's local listener occupies.
3. `fetchMirrorNodeNetwork()` accepts the response (HTTP 200, valid JSON, non-empty host, non-zero port) and returns the poisoned map.
4. `hiero.ClientForNetwork(netmap)` builds a client whose only known nodes are `127.0.0.1:<attacker_port>`.
5. Every subsequent `submitWithRetry` call in `main.go` (line 63) sends signed, operator-keyed transactions to the attacker's listener instead of real consensus nodes. No transaction ever reaches the Hashgraph.

### Impact Explanation

- **Hashgraph history tampering / denial of write**: All transactions signed with the operator's private key are silently swallowed by the attacker's endpoint. No consensus record is ever created. From the operator's perspective the pinger appears healthy (no Go-level error if the attacker's server returns a plausible gRPC response).
- **Key-material exposure**: The attacker receives fully signed, serialized Hiero transactions including the operator account ID and the cryptographic signature, enabling replay or analysis.
- **Severity: High** — complete loss of the pinger's core function (writing Hashgraph history) with no operator-visible alarm.

### Likelihood Explanation

- The default `mirrorRest` value is an HTTP URL (`http://mirror-rest:5551`), meaning TLS is not used in the default deployment.
- In Kubernetes, DNS spoofing within a namespace is feasible without cluster-admin privileges if `NetworkPolicy` is absent (common in default installs).
- The attack is repeatable: the pinger only calls `buildNetworkFromMirrorNodes` once at startup, so a single poisoned response persists for the entire process lifetime.
- No special knowledge of the operator key is required; the attacker only needs to serve a valid JSON body.

### Recommendation

1. **Enforce HTTPS** for `mirrorRest` and reject non-TLS URLs at config validation time (`config.go`).
2. **Validate resolved addresses** in `fetchMirrorNodeNetwork()` before inserting into the network map — reject loopback (`net.IP.IsLoopback()`), link-local (`IsLinkLocalUnicast()`), and unspecified addresses.
3. **Optionally pin the mirror-node TLS certificate** or verify a signed node-list payload to prevent MITM even over HTTPS.
4. **Re-fetch the node list periodically** with the same validation, and alert/abort if the returned set diverges significantly from the previously known set.

### Proof of Concept

```bash
# 1. Start a fake gRPC sink on localhost:50211
nc -lk 127.0.0.1 50211 &

# 2. Start a fake mirror-node REST server returning poisoned node list
cat > /tmp/nodes.json <<'EOF'
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        { "ip_address_v4": "127.0.0.1", "port": 50211 }
      ]
    }
  ],
  "links": { "next": null }
}
EOF
python3 -m http.server 5551 --directory /tmp &

# 3. Run the pinger pointed at the fake mirror node
HIERO_MIRROR_PINGER_NETWORK=other \
HIERO_MIRROR_PINGER_REST=http://127.0.0.1:5551 \
HIERO_MIRROR_PINGER_OPERATOR_ID=0.0.2 \
HIERO_MIRROR_PINGER_OPERATOR_KEY=<valid_key> \
./pinger

# Result: all transactions are sent to 127.0.0.1:50211 (nc),
# zero transactions reach any real Hashgraph consensus node.
# nc output will show raw gRPC frames containing the signed transaction.
