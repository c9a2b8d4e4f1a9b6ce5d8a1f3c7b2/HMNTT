### Title
Prometheus Label Cardinality Injection via Unvalidated `req.method` in `metricsMiddleware`

### Summary
In `rest/middleware/metricsHandler.js`, the `metricsMiddleware` function records `req.method` directly as a Prometheus label without any validation or normalization. Because Express/Node.js passes arbitrary HTTP method strings from the request line through to middleware, an unauthenticated attacker can inject attacker-controlled strings into the `method` label of all per-route metrics, creating unbounded new Prometheus time series and causing a cardinality-explosion denial-of-service against the monitoring stack.

### Finding Description

**Exact code path:**

In `rest/middleware/metricsHandler.js`, inside the `res.on('finish', ...)` callback:

```js
// line 191
const method = req.method;
const labels = {method, path, code};   // line 192

// lines 207-210 — all four per-route instruments use these labels
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
``` [1](#0-0) 

`req.method` is taken verbatim from the HTTP request line. Node.js's `llhttp` HTTP parser accepts any RFC-7230 token as a method name, so `FAKEMETHOD`, `AAAAAAAAAA`, or any other token-safe string is forwarded to Express unchanged and surfaces in `req.method`.

**Why existing checks do not block this:**

1. **`openApiValidator`** (`rest/middleware/openapiHandler.js` lines 148–158) is configured with `validateRequests: false` and `ignoreUndocumented: true`. It performs no method validation and passes all requests through. [2](#0-1) 

2. **`authHandler`** (`rest/middleware/authHandler.js`) only checks Basic-Auth credentials for optional per-user rate limiting. It does not inspect or restrict the HTTP method. [3](#0-2) 

3. **Middleware registration order** in `server.js` (lines 63–92): `openApiValidator` → `authHandler` → `metricsHandler`. Even if the OpenAPI validator were enabled, it runs before the metrics middleware but with `validateRequests: false`, so it never rejects the request. [4](#0-3) 

### Impact Explanation

Prometheus stores one time series per unique label-value combination. The four per-route instruments (`api_request`, `api_request_duration_milliseconds`, `api_request_size_bytes`, `api_response_size_bytes`) each use `{method, path, code}` labels. Each unique `method` string the attacker injects multiplies the number of active time series by the number of distinct `(path, code)` pairs. Sending requests with N distinct method strings against M endpoints produces up to `4 × N × M` new time series. At scale this exhausts Prometheus memory, degrades scrape performance, and can crash the Prometheus instance, blinding operators to real service issues.

### Likelihood Explanation

No authentication, no special network position, and no prior knowledge of the system is required. Any HTTP client can send a request with an arbitrary method string (e.g., `curl -X FAKEMETHOD https://mirror-node/api/v1/transactions`). The attack is trivially scriptable, repeatable at high rate, and requires only public network access to the REST API port.

### Recommendation

Normalize `req.method` against an explicit allowlist of standard HTTP methods before using it as a label. Replace the raw assignment with something like:

```js
const ALLOWED_METHODS = new Set(['GET','POST','PUT','PATCH','DELETE','HEAD','OPTIONS']);
const method = ALLOWED_METHODS.has(req.method) ? req.method : 'UNKNOWN';
```

Apply this normalization at line 191 of `rest/middleware/metricsHandler.js` before constructing `labels`.

### Proof of Concept

```bash
# Send 1000 requests each with a unique fake HTTP method
for i in $(seq 1 1000); do
  curl -s -o /dev/null -X "FAKEMETHOD${i}" \
    http://<mirror-node-host>/api/v1/transactions &
done
wait

# Scrape metrics and observe thousands of new time series
curl http://<mirror-node-host>/api/v1/metrics/ | grep 'api_request_total' | grep 'FAKEMETHOD'
```

Each `FAKEMETHOD<N>` appears as a distinct `method` label value across all four per-route histograms/counters, confirming unbounded cardinality injection.
