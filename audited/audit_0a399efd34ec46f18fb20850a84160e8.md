### Title
Unbounded `ServiceEndpoints` from Mirror REST Server Causes SDK Connection Exhaustion

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` imposes no upper bound on the number of `ServiceEndpoints` entries per node returned by the mirror REST server. A malicious or compromised mirror REST server can return thousands of synthetic endpoints per node, inflating the `network` map passed to `hiero.ClientForNetwork()`, causing the Hiero SDK to attempt gRPC connections to all of them and exhausting connection resources — effectively partitioning the pinger from the real network.

### Finding Description
In `pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()` (lines 74–131) decodes the mirror REST response into a `nodesEnvelope` with no body-size limit and no cap on the number of entries:

```go
// line 95-98
var payload nodesEnvelope
if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil { ... }
```

The inner loop (lines 113–124) iterates every `ServiceEndpoint` for every node without any bounds check:

```go
for _, ep := range n.ServiceEndpoints {   // line 113 — unbounded
    host := strings.TrimSpace(ep.DomainName)
    if host == "" { host = strings.TrimSpace(ep.IPAddressV4) }
    if host == "" || ep.Port == 0 { continue }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId          // line 122 — map grows without limit
}
```

The only guard is the `host == "" || ep.Port == 0` check (line 118), which is trivially bypassed by supplying valid-looking but attacker-controlled `host:port` pairs. The resulting `network` map — potentially containing tens of thousands of entries — is returned to `newClient()` in `sdk_client.go` (line 22) and passed directly to `hiero.ClientForNetwork(netmap)`, which causes the SDK to attempt gRPC connections to every entry.

**Root cause:** The code assumes the mirror REST server is trusted and returns a reasonable number of endpoints. There is no `io.LimitReader` on the response body, no cap on `len(payload.Nodes)`, and no cap on `len(n.ServiceEndpoints)`.

### Impact Explanation
When the SDK receives a map with thousands of fake `host:port` entries, it attempts to establish gRPC connections to all of them. This exhausts OS-level file descriptors and goroutine/connection-pool resources. Real consensus nodes are either never reached (all slots consumed by fake endpoints) or are reached only after extreme latency, constituting a de facto network partition. The pinger's health-check purpose is completely defeated. Severity: **High** — full denial of service of the pinger's network connectivity with no recovery until restart with a trusted mirror URL.

### Likelihood Explanation
The precondition is control over the mirror REST endpoint configured in `cfg.mirrorRest`. This is achievable by:
- Operating a malicious or misconfigured mirror node that the pinger is pointed at (zero privilege on the pinger host required).
- DNS poisoning or BGP hijacking of the mirror REST hostname.
- A MITM on an unencrypted or improperly validated TLS connection to the mirror REST server.

No authentication or signature verification is performed on the mirror REST response. The attack is repeatable on every startup or retry cycle (lines 52–69 in `buildNetworkFromMirrorNodes`).

### Recommendation
1. **Limit response body size**: Wrap `resp.Body` with `io.LimitReader` before decoding (e.g., 1 MB).
2. **Cap endpoints per node**: Enforce a maximum (e.g., 10) on `len(n.ServiceEndpoints)` before iterating.
3. **Cap total network map size**: Reject or truncate responses that would produce more than a reasonable number of entries (e.g., 100 total).
4. **Validate endpoint values**: Reject obviously invalid IPs/ports (e.g., loopback, reserved ranges, port 0).

```go
const maxEndpointsPerNode = 10
const maxTotalEndpoints   = 100

// Wrap body:
body := io.LimitReader(resp.Body, 1<<20)
json.NewDecoder(body).Decode(&payload)

// Inside loop:
epCount := 0
for _, ep := range n.ServiceEndpoints {
    if epCount >= maxEndpointsPerNode { break }
    ...
    epCount++
}
if len(network) >= maxTotalEndpoints { break }
```

### Proof of Concept
1. Stand up an HTTP server at a URL reachable as `cfg.mirrorRest`.
2. Have it respond to `GET /api/v1/network/nodes` with:
```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        {"ip_address_v4": "192.0.2.<N>", "port": 50211}
        // ... repeat 10,000 times with unique IPs
      ]
    }
  ],
  "links": {"next": null}
}
```
3. Configure the pinger with `network=other` and `mirrorRest=<attacker URL>`.
4. Start the pinger. Observe: `hiero.ClientForNetwork()` receives a 10,000-entry map; the SDK spawns connection attempts to all 10,000 fake endpoints; real node connections are starved; the pinger reports all nodes unreachable. [1](#0-0) [2](#0-1) [3](#0-2)
