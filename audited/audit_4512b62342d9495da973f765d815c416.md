### Title
SSRF via Unvalidated `DomainName`/`IPAddressV4` in `fetchMirrorNodeNetwork()` — gRPC Connections to Attacker-Controlled Internal Hosts

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` fetches node endpoint data from a mirror REST server and builds a gRPC network map with zero validation of the returned `DomainName` or `IPAddressV4` fields. Any party able to control what the mirror REST server returns — including a compromised mirror node, a MITM on a plaintext HTTP connection, or a malicious custom mirror deployment — can inject internal hostnames or link-local IPs (e.g., `169.254.169.254`, `kubernetes.default.svc`) that the Hiero SDK will then dial via gRPC, constituting a server-side request forgery (SSRF) from the pinger pod.

### Finding Description

**Exact code path:**

`pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 113–124:

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId
}
```

The only guards are: non-empty `host` and non-zero `port`. There is no:
- IP range check (RFC 1918, link-local `169.254.0.0/16`, loopback `127.0.0.0/8`)
- Hostname allowlist or denylist
- DNS rebinding protection

The resulting `network` map is passed directly to `hiero.ClientForNetwork(netmap)` in `pinger/sdk_client.go` line 22, which instructs the Hiero SDK to open gRPC connections to every address in the map.

**Root cause:** The function blindly trusts the mirror REST server's response as a source of truth for gRPC dial targets, with no sanitization of the host component.

**Exploit flow:**

1. Attacker controls what the mirror REST server at `cfg.mirrorRest` returns (see preconditions below).
2. The server returns a JSON payload with a `service_endpoints` entry where `domain_name` is set to `kubernetes.default.svc` and `port` to `443`, or `ip_address_v4` set to `169.254.169.254` and `port` to `80`.
3. `fetchMirrorNodeNetwork()` accepts the entry (non-empty host, non-zero port) and inserts `kubernetes.default.svc:443` into the network map.
4. `hiero.ClientForNetwork(netmap)` stores this as a gRPC node endpoint.
5. On the next `submitWithRetry()` tick, the Hiero SDK dials `kubernetes.default.svc:443` over gRPC (HTTP/2 + TLS), sending the operator's signed transaction bytes to the Kubernetes API server.

**Why existing checks fail:**

The only validation present (`host == ""` / `ep.Port == 0`) is a liveness check, not a security check. It explicitly allows any non-empty string as a dial target.

### Impact Explanation

- **SSRF from the pinger pod:** The pinger establishes outbound TCP/gRPC connections to attacker-specified internal addresses. Even when the target does not speak gRPC, the TCP handshake completes, enabling internal port scanning.
- **Kubernetes API server interaction:** `kubernetes.default.svc:443` speaks TLS + HTTP/2. A gRPC dial will complete the TLS handshake and send HTTP/2 frames (including the operator's signed Hiero transaction payload as a gRPC body), constituting unsolicited traffic to the cluster control plane.
- **Cloud metadata exfiltration:** `169.254.169.254:80` (AWS/GCP/Azure IMDS) will receive a TCP connection; the gRPC HTTP/2 preface will be sent. While the metadata service responds with HTTP/1.1 (causing a protocol mismatch), the connection itself may trigger IAM credential issuance flows on some platforms or be logged as an access attempt.
- **Severity: High** — the pinger runs inside the cluster with a network identity; SSRF from it reaches services that are otherwise inaccessible from outside.

### Likelihood Explanation

**Preconditions:**
1. `HIERO_MIRROR_PINGER_NETWORK=other` must be set (custom deployment mode).
2. The attacker must control the mirror REST server's response. Realistic paths:
   - The mirror REST URL uses plain HTTP (the documented example is `http://mirror-rest:5551`); a network-adjacent attacker can MITM the unencrypted response.
   - The mirror REST server is a community-operated or third-party node that is compromised.
   - In a multi-tenant cluster, a tenant with write access to the mirror node's backing store can inject malicious node records.

The `network=other` mode is explicitly the custom/private-network deployment path, making it the most likely environment where a non-standard (potentially less-trusted) mirror node is used. The HTTP-without-TLS default makes MITM straightforward on any shared network segment.

### Recommendation

Add an IP/hostname allowlist or denylist before inserting endpoints into the network map. At minimum:

```go
import "net/netip"

func isSafeHost(host string) bool {
    addr, err := netip.ParseAddr(host)
    if err != nil {
        // It's a hostname — apply a domain allowlist or regex
        return isAllowedHostname(host)
    }
    return !addr.IsLoopback() &&
           !addr.IsLinkLocalUnicast() &&
           !addr.IsPrivate() &&
           !addr.IsUnspecified()
}
```

Apply this check at line 118 before `net.JoinHostPort`. Additionally, enforce HTTPS for `cfg.mirrorRest` to eliminate MITM as an attack vector, and consider pinning the expected mirror node domain via configuration.

### Proof of Concept

**Setup:** Pinger deployed with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://attacker-controlled-mirror:5551`.

**Malicious mirror REST response** at `GET /api/v1/network/nodes`:

```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        {
          "domain_name": "kubernetes.default.svc",
          "ip_address_v4": "",
          "port": 443
        },
        {
          "domain_name": "",
          "ip_address_v4": "169.254.169.254",
          "port": 80
        }
      ]
    }
  ],
  "links": { "next": null }
}
```

**Result:** `fetchMirrorNodeNetwork()` returns:
```
map["kubernetes.default.svc:443": AccountID{0,0,3}, "169.254.169.254:80": AccountID{0,0,3}]
```

`hiero.ClientForNetwork(netmap)` stores these as gRPC dial targets. On the next ticker tick, `submitWithRetry()` causes the SDK to dial `kubernetes.default.svc:443`, completing a TCP+TLS connection to the Kubernetes API server from within the cluster network. [1](#0-0) [2](#0-1) [3](#0-2)
