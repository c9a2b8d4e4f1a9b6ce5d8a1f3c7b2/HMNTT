### Title
Log Level Bypass via Query String in `TracingMiddleware` Internal Path Check

### Summary
`TracingMiddleware` uses `request.URL.RequestURI()` to obtain the request path, which includes the raw query string. The `internalPaths` map contains only bare paths (`/metrics`, `/health/liveness`, `/health/readiness`). Any unauthenticated request appending a query string (e.g., `/metrics?x=1`) produces a key not present in the map, causing the middleware to emit a `log.Info` entry instead of `log.Debug`, bypassing the suppression that normally silences high-frequency internal probes in production.

### Finding Description
**File:** `rosetta/app/middleware/trace.go`
**Lines:** 47, 55‚Äì59

```go
path := request.URL.RequestURI()          // line 47 ‚Äî includes "?foo=bar"
...
if internalPaths[path] {                  // line 55 ‚Äî exact map key lookup
    log.Debug(message)
} else {
    log.Info(message)                     // line 58 ‚Äî triggered for any query-string variant
}
```

`internalPaths` is defined at line 19 with three exact keys:
- `"/health/liveness"` (from `health.go` line 20)
- `"/health/readiness"` (from `health.go` line 21)
- `"/metrics"` (from `metrics.go` line 16)

`RequestURI()` returns the full request URI including query string per Go's `net/http` contract. A request to `/metrics?a=1` yields `path = "/metrics?a=1"`, which is not a key in `internalPaths`, so the `else` branch fires and `log.Info` is called.

The Go HTTP router strips query strings for routing purposes, so the underlying handler still responds normally (200 OK for `/metrics?a=1`), meaning the request is fully processed and the log entry is emitted at `Info` level.

The test at `trace_test.go` line 49 (`metricsPath + "s"` ‚Üí `/metricss`) confirms the team is aware of suffix-based bypasses but never tested the query-string variant.

### Impact Explanation
In production deployments, the log level is typically set to `Info` or higher. Under normal operation:
- Kubernetes liveness/readiness probes hit `/health/liveness` and `/health/readiness` every few seconds ‚Äî logged at `Debug`, **suppressed**.
- Prometheus scrapes `/metrics` every 15‚Äì30 seconds ‚Äî logged at `Debug`, **suppressed**.

An attacker appending any query string to these paths forces every such request to emit a `log.Info` entry. Because these paths are designed to be hit at high frequency by infrastructure tooling, an attacker who continuously sends requests to `/health/liveness?x=1` or `/metrics?x=1` generates a sustained stream of `Info`-level log entries that would otherwise produce zero output. This inflates log I/O, log aggregation pipeline throughput (e.g., Fluentd, Loki, CloudWatch), and storage costs. In environments where log ingestion is rate-limited or billed per byte, this constitutes measurable resource amplification with no authentication required.

### Likelihood Explanation
- **Precondition:** None. The Rosetta API is an externally reachable HTTP service; no credentials, tokens, or special headers are required.
- **Trigger:** A single HTTP GET to `/metrics?x=1` or `/health/liveness?x=1` is sufficient to produce one `Info` log entry. Repeating at a moderate rate (e.g., 1 req/s) produces continuous log amplification without triggering rate-limit thresholds that would flag brute force.
- **Repeatability:** Fully repeatable and automatable with a single `curl` loop or any HTTP client. No state, session, or prior knowledge of the system is needed.

### Recommendation
Replace the `RequestURI()` lookup with a path-only comparison by using `request.URL.Path` (which strips the query string), or normalize the key before the map lookup:

```go
// Option 1: use Path instead of RequestURI
path := request.URL.Path
if internalPaths[path] { ... }

// Option 2: strip query string before lookup
rawPath := request.URL.RequestURI()
pathOnly := strings.SplitN(rawPath, "?", 2)[0]
if internalPaths[pathOnly] { ... }
```

The `message` variable can still use the full `RequestURI()` for logging fidelity; only the map lookup key needs to be the bare path.

### Proof of Concept
```bash
# Start the Rosetta node (default port 5700)

# Normal request ‚Äî logged at Debug, suppressed in production (Info level)
curl -s http://localhost:5700/metrics

# Bypass ‚Äî logged at Info, appears in production logs
curl -s "http://localhost:5700/metrics?x=1"

# Sustained amplification (no brute-force rate, 1 req/s)
while true; do
  curl -s "http://localhost:5700/health/liveness?bypass=1" > /dev/null
  sleep 1
done
# Each iteration emits one log.Info entry that would otherwise be log.Debug (suppressed)
```

Observe with `log.SetLevel(log.InfoLevel)` that the bare path produces no output while the query-string variant produces a timestamped `INFO` line per request.
