### Title
Index Out-of-Bounds Panic in `ConstructionSubmit` via Crafted Transaction with No Node Account IDs

### Summary
`ConstructionSubmit` unconditionally accesses `transaction.GetNodeAccountIDs()[0]` at line 353 without first verifying the slice is non-empty. An unauthenticated attacker can submit a valid hex-encoded Hedera transaction (e.g., a `TransferTransaction`) that passes all existing validation but contains no node account IDs, triggering an index-out-of-bounds panic before `hiero.TransactionExecute` is reached and preventing transaction gossip.

### Finding Description
In `rosetta/app/services/construction_service.go`, `ConstructionSubmit` (lines 332–368) processes an externally-supplied signed transaction:

1. `unmarshallTransactionFromHexString` (line 340) decodes the hex, calls `hiero.TransactionFromBytes`, and checks only that the transaction type is `AccountCreateTransaction` or `TransferTransaction`. No check is made for the presence of node account IDs.
2. `hiero.TransactionGetTransactionHash` (line 345) and `hiero.TransactionGetTransactionID` (line 351) are called — both can succeed on a transaction with no node account IDs.
3. Line 353 then executes `transaction.GetNodeAccountIDs()[0]` with no length guard. If the deserialized transaction has an empty node account ID list (which is valid protobuf), this is a runtime panic: `runtime error: index out of range [0] with length 0`.

The failed assumption is that any transaction successfully deserialized from bytes will always have at least one node account ID. This is only true for transactions produced by the normal Rosetta construction flow (which calls `transactionSetNodeAccountId`), but is not enforced for externally-submitted transactions.

### Impact Explanation
A panic in a Go HTTP handler goroutine that is not caught by a recovery middleware will crash the entire server process, taking down the Rosetta online endpoint. Even with panic recovery middleware, the request fails and the transaction is not gossiped. This is a denial-of-service vector against the `/construction/submit` endpoint — the only endpoint responsible for broadcasting transactions to the Hedera network. Repeated requests can keep the service unavailable.

### Likelihood Explanation
No authentication or privilege is required. The `/construction/submit` endpoint is publicly accessible per the Rosetta API specification. Crafting a valid `TransferTransaction` protobuf with no `nodeAccountID` field set is trivial using any Hedera SDK or raw protobuf tooling. The attack is repeatable and requires minimal effort.

### Recommendation
Add an explicit bounds check immediately after unmarshalling, before the `log.Infof` call:

```go
nodeIDs := transaction.GetNodeAccountIDs()
if len(nodeIDs) == 0 {
    return nil, errors.ErrTransactionInvalidType // or a dedicated error
}
log.Infof("Submitting transaction %s (hash %s) to node %s", transactionId, hash, nodeIDs[0])
```

Alternatively, validate inside `unmarshallTransactionFromHexString` that the deserialized transaction has at least one node account ID, returning an appropriate error to the caller.

### Proof of Concept
1. Using the Hiero Go SDK, construct a `TransferTransaction` with a valid `TransactionID` and transfers, but **do not call `SetNodeAccountIDs`**.
2. Call `hiero.TransactionToBytes(tx)` to serialize it.
3. Hex-encode the bytes and prefix with `0x`.
4. POST to the online Rosetta server's `/construction/submit` endpoint:
   ```json
   {
     "network_identifier": {"blockchain": "Hedera", "network": "testnet"},
     "signed_transaction": "0x<crafted_hex>"
   }
   ```
5. The server panics at `transaction.GetNodeAccountIDs()[0]` (line 353), crashing the handler goroutine (and potentially the process), and the transaction is never submitted to the network.

func unmarshallTransactionFromHexString(transactionString string) (hiero.TransactionInterface, *rTypes.Error) {
	transactionBytes, err := hex.DecodeString(tools.SafeRemoveHexPrefix(transactionString))
	if err != nil {
		return nil, errors.ErrTransactionDecodeFailed
	}

	transaction, err := hiero.TransactionFromBytes(transactionBytes)
	if err != nil {
		return nil, errors.ErrTransactionUnmarshallingFailed
	}

	if rErr := isSupportedTransactionType(transaction); rErr != nil {
		return nil, rErr
	}

	return transaction, nil
}
