### Title
Case-Insensitive Query Key Merging Produces Array for `encoding`, Bypassing Validation and Crashing Topic Message Endpoint (DoS)

### Summary
`requestQueryParser()` lowercases all query parameter keys and merges collisions into arrays. Sending `encoding=base64&ENCODING=utf-8` causes `req.query['encoding']` to become `['base64', 'utf-8']`. Downstream validation in `validateReq()` iterates each array element individually — both pass — but `getTopicMessages()` passes the raw array to `encodeBinary()`, which calls `.toLowerCase()` on it, throwing a `TypeError` and returning a 500 to the caller. Any unauthenticated user can repeatably crash the `/api/v1/topics/{topicId}/messages` endpoint.

### Finding Description

**Step 1 — Array creation in `requestQueryParser()`**

`rest/middleware/requestHandler.js` lines 38–68: the parser lowercases every key and, on collision, calls `merge()` which converts the value to an array.

```
encoding=base64&ENCODING=utf-8
  → qs parses: { encoding: 'base64', ENCODING: 'utf-8' }
  → loop: 'encoding' stored; 'ENCODING' lowercased → 'encoding' already exists
  → merge('base64', 'utf-8') → ['base64', 'utf-8']
  → req.query['encoding'] = ['base64', 'utf-8']
``` [1](#0-0) 

**Step 2 — Validation passes silently**

`validateReq()` in `rest/utils.js` lines 463–477: when `req.query[key]` is an `Array`, it iterates each element and calls `paramValidityChecks(key, val)` per element. `'base64'` and `'utf-8'` each individually satisfy `isValidEncoding()`, so no `badParams` entry is added and no 400 is thrown. [2](#0-1) 

**Step 3 — Raw array extracted and forwarded**

`getTopicMessages()` in `rest/topicmessage.js` line 120 reads `encoding` directly from `req.query` before any type check:

```js
const encoding = req.query[constants.filterKeys.ENCODING]; // ['base64', 'utf-8']
```

It is then passed verbatim to `new TopicMessageViewModel(m, encoding)` at line 144. [3](#0-2) 

**Step 4 — TypeError in `encodeBinary()`**

`TopicMessageViewModel` constructor calls `encodeBinary(topicMessage.message, messageEncoding)` at line 28. [4](#0-3) 

`encodeBinary()` calls `isValidUtf8Encoding(encoding)` where `encoding` is the array `['base64', 'utf-8']`: [5](#0-4) 

`isValidUtf8Encoding()` checks `if (!query)` (array is truthy, passes), then calls `query.toLowerCase()`. Arrays have no `toLowerCase` method → **`TypeError: query.toLowerCase is not a function`** is thrown, propagating as an unhandled 500. [6](#0-5) 

### Impact Explanation
Every call to `GET /api/v1/topics/{topicId}/messages?encoding=base64&ENCODING=utf-8` crashes with a 500 before any message data is returned. This is a complete denial-of-service against the topic message history endpoint, which is the primary mechanism for reading Hashgraph HCS (Hedera Consensus Service) message history. No authentication or special privilege is required.

### Likelihood Explanation
The exploit requires only a crafted HTTP query string with two differently-cased copies of the same parameter — trivially reproducible with `curl` or any HTTP client. It is repeatable, stateless, and requires zero knowledge of the system beyond the public API path. Any external actor aware of the case-insensitive key merging behavior can trigger it on demand.

### Recommendation
Apply one or both of the following fixes:

1. **In `getTopicMessages()`**: After reading `encoding`, assert it is a string, not an array:
   ```js
   const encoding = req.query[constants.filterKeys.ENCODING];
   if (Array.isArray(encoding)) {
     throw InvalidArgumentError.forParams(constants.filterKeys.ENCODING);
   }
   ```

2. **In `encodeBinary()` / `isValidUtf8Encoding()`**: Guard against non-string input:
   ```js
   const isValidUtf8Encoding = (query) => {
     if (!query || typeof query !== 'string') return false;
     return /^(utf-?8)$/.test(query.toLowerCase());
   };
   ```

3. **In `validateReq()`**: When a parameter is an array but the endpoint does not permit repeated values for that key (e.g., `encoding`), reject it with a 400 rather than validating each element individually.

### Proof of Concept

```bash
# Trigger 500 on the topic messages endpoint
curl -v "https://<mirror-node-host>/api/v1/topics/7/messages?encoding=base64&ENCODING=utf-8"

# Expected: HTTP 500 (TypeError: query.toLowerCase is not a function)
# No authentication required
# Repeatable indefinitely
```

Trace:
1. `requestQueryParser` merges → `req.query.encoding = ['base64', 'utf-8']`
2. `validateReq` iterates elements → both valid → no 400
3. `getTopicMessages` reads array → passes to `TopicMessageViewModel`
4. `encodeBinary` → `isValidUtf8Encoding(['base64','utf-8'])` → `.toLowerCase()` on array → `TypeError` → 500

const requestQueryParser = (queryString) => {
  const merge = (current, next) => {
    if (!Array.isArray(current)) {
      current = [current];
    }

    if (Array.isArray(next)) {
      current.push(...next);
    } else {
      current.push(next);
    }

    return current;
  };

  // parse first to benefit from qs query handling
  const parsedQueryString = qs.parse(queryString, queryOptions);

  const caseInsensitiveQueryString = {};
  for (const [key, value] of Object.entries(parsedQueryString)) {
    const lowerKey = key.toLowerCase();
    const canonicalValue = canonicalizeValue(lowerKey, value);
    if (lowerKey in caseInsensitiveQueryString) {
      // handle repeated values, merge into an array
      caseInsensitiveQueryString[lowerKey] = merge(caseInsensitiveQueryString[lowerKey], canonicalValue);
    } else {
      caseInsensitiveQueryString[lowerKey] = canonicalValue;
    }
  }

  return caseInsensitiveQueryString;
