### Title
Unauthenticated Opcode Replay Exposes Full EVM Stack and Memory of Any Historical Transaction Including HTS Precompile Call Parameters

### Summary
The `getContractOpcodes()` endpoint in `OpcodesController` performs no authentication or authorization check, allowing any unprivileged external user to replay any historical transaction and receive the full EVM execution trace. When `stack=true` (the default) and `memory=true` are requested, the trace captures the raw EVM stack and memory at every opcode step, including the ABI-encoded calldata loaded into memory immediately before a CALL to an HTS precompile — exposing token IDs, amounts, and account addresses that were part of the original transaction's internal execution.

### Finding Description

**Exact code path:**

`OpcodesController.getContractOpcodes()` at [1](#0-0)  performs exactly three checks before executing the replay:

1. `properties.isEnabled()` — a feature flag, defaults to `false` but is operator-configurable. [2](#0-1) 
2. `validateAcceptEncodingHeader()` — requires `Accept-Encoding: gzip`. [3](#0-2) 
3. `throttleManager.throttleOpcodeRequest()` — rate limiting only. [4](#0-3) 

There is no Spring Security configuration found in the web3 module, no `@PreAuthorize`, no ownership check, and no identity verification of any kind. The controller class itself carries no security annotation. [5](#0-4) 

**Root cause — how HTS parameters are exposed:**

`OpcodeServiceImpl.processOpcodeCall()` reconstructs the original transaction parameters from the database and re-executes it through the EVM with `OpcodeActionTracer` attached. [6](#0-5) 

`AbstractOpcodeTracer.tracePostExecution()` (called via `OpcodeActionTracer`) captures the full EVM stack and memory after every single opcode executes. [7](#0-6) 

`captureMemory()` dumps the entire EVM memory as 32-byte hex words. `captureStack()` dumps every stack item as hex. [8](#0-7) 

When a smart contract calls an HTS precompile (e.g., `mintToken`, `transferToken`), the EVM sequence is:
1. The contract uses `MSTORE`/`MSTORE8` opcodes to write ABI-encoded calldata into memory (token address, amounts, account addresses).
2. It pushes CALL arguments onto the stack (gas, HTS precompile address `0x167`, value, argsOffset, argsLength, retOffset, retLength).
3. It executes `CALL`.

`tracePostExecution()` fires after each of these opcodes. At the step just before `CALL`, the stack contains the HTS precompile address and memory offsets; the memory contains the full ABI-encoded HTS function call including token IDs, amounts, and account addresses. All of this is serialized into the `OpcodesResponse`.

Note: `tracePrecompileResult()` intentionally passes empty stack/memory for the precompile result entry itself, [9](#0-8)  but this does not prevent the preceding opcode traces from already having captured the parameters.

**Why existing checks are insufficient:**

The gzip header requirement is a client-side hint for compression, not a security control. Rate limiting prevents abuse volume but does not restrict who can call the endpoint. The feature flag is a deployment toggle, not per-request authorization. No check verifies that the requester has any relationship to the transaction being replayed.

### Impact Explanation

When the opcode tracer feature is enabled, any anonymous internet user can:
- Replay any historical smart contract transaction by its publicly observable transaction ID or hash.
- With `stack=true` (the default) and `memory=true`, obtain the full internal EVM execution trace.
- Extract the ABI-encoded parameters of every HTS precompile call made during that transaction, including token IDs, transfer amounts, and account addresses that may not be directly visible in the standard transaction record (e.g., intermediate internal calls within complex DeFi contracts).
- Observe internal contract logic, intermediate computed values, and storage slot writes that are not exposed by any other public API.

This constitutes an unauthorized information disclosure of the complete internal execution state of any historical transaction on the network.

### Likelihood Explanation

Preconditions are minimal: the attacker needs only (1) the feature to be enabled by the operator, (2) knowledge of a transaction ID or hash (publicly available via block explorers or the mirror node's own REST API), and (3) the ability to send an HTTP GET request with `Accept-Encoding: gzip`. No credentials, no account, no on-chain presence required. The attack is trivially repeatable for any transaction in history and requires no special tooling.

### Recommendation

1. **Add authentication/authorization**: Protect the `/api/v1/contracts/results/{transactionIdOrHash}/opcodes` endpoint with an access control mechanism (e.g., API key, operator-configured allowlist, or Spring Security `@PreAuthorize`). This is a debug/forensic endpoint and should not be publicly accessible without identity verification.
2. **Restrict by default**: Even when the feature flag is enabled, require explicit opt-in credentials per request.
3. **Consider ownership checks**: Optionally, restrict replay to the original transaction payer or to authenticated operators only.

### Proof of Concept

**Preconditions:** The mirror node is deployed with `hiero.mirror.web3.opcode.tracer.enabled=true`.

**Steps:**

1. Obtain any historical HTS-interacting transaction hash from the public mirror node REST API:
   ```
   GET /api/v1/contracts/results?limit=1
   ```
   Extract `hash` from the response.

2. Call the opcode endpoint as an unauthenticated user:
   ```
   GET /api/v1/contracts/results/{hash}/opcodes?stack=true&memory=true
   Accept-Encoding: gzip
   ```

3. In the returned `opcodes` array, locate entries where `op` is `MSTORE` or `CALL`. The `memory` field at those steps contains the ABI-encoded HTS precompile calldata. The `stack` field at the `CALL` step contains the HTS precompile address (`0x0000...0167`) and memory offsets.

4. ABI-decode the memory bytes at the relevant offset/length to recover the original token address, transfer amounts, and account addresses passed to the HTS precompile.

**Result:** Full internal HTS call parameters are recovered with no credentials required.

    public void tracePostExecution(@NonNull final MessageFrame frame, @NonNull final OperationResult operationResult) {
        final var context = ContractCallContext.get();

        // Reset the balance call flag after BALANCE opcode completes
        if (frame.getCurrentOperation() != null
                && BALANCE_OPERATION_NAME.equals(frame.getCurrentOperation().getName())) {
            context.setBalanceCall(false);
        }

        final var options = context.getOpcodeContext();
        final var memory = captureMemory(frame, options);
        final var stack = captureStack(frame, options);
        final var storage = captureStorage(frame, options, context);

        final var revertReasonBytes = frame.getRevertReason().orElse(null);
        final var reason = revertReasonBytes != null ? revertReasonBytes.toHexString() : null;
        context.getOpcodeContext()
                .addOpcodes(createOpcode(frame, operationResult.getGasCost(), reason, stack, memory, storage));
    }

    protected final List<String> captureMemory(final MessageFrame frame, final OpcodeContext options) {
        if (!options.isMemory()) {
            return Collections.emptyList();
        }
        var size = frame.memoryWordSize();
        var hexMemoryEntries = new ArrayList<String>(size);

        final var wordSize = 32;
        final var memory = frame.readMutableMemory(0L, (long) wordSize * size).toArrayUnsafe();
        final var hex = HexFormat.of();

        for (var i = 0; i < size; i++) {
            var startIndex = i * wordSize;
            final var result = HEX_PREFIX + hex.formatHex(memory, startIndex, startIndex + wordSize);
            hexMemoryEntries.add(result);
        }

        return hexMemoryEntries;
    }

    protected final List<String> captureStack(final MessageFrame frame, final OpcodeContext options) {
        if (!options.isStack()) {
            return Collections.emptyList();
        }

        var size = frame.stackSize();
        var stack = new ArrayList<String>(size);
        for (var i = 0; i < size; ++i) {
            var item = frame.getStackItem(size - 1 - i);
            stack.add(hexCache.get(item, Bytes::toHexString));
        }

        return stack;
    }
