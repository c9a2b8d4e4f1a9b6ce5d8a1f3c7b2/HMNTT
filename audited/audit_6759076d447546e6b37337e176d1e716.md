### Title
Unbounded `log.Info()` in `TracingMiddleware` Enables Log Flooding / Disk Exhaustion by Unauthenticated Attackers

### Summary
`TracingMiddleware()` unconditionally calls `log.Info(message)` for every HTTP request whose path is not in `internalPaths`, with no application-level rate limiting or log-volume cap. The log message includes attacker-controlled content (`X-Forwarded-For`/`X-Real-IP` headers and the full request URI). An unauthenticated attacker can flood the log subsystem, exhaust disk space, or cause log-rotation thrashing with zero authentication required.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/trace.go`, `TracingMiddleware()`, lines 43–61:

```go
func TracingMiddleware(inner http.Handler) http.Handler {
    return http.HandlerFunc(func(responseWriter http.ResponseWriter, request *http.Request) {
        ...
        clientIpAddress := getClientIpAddress(request)   // reads X-Real-IP / X-Forwarded-For — attacker-controlled
        path := request.URL.RequestURI()                  // full URI including query string — attacker-controlled
        ...
        message := fmt.Sprintf("%s %s %s (%d) in %s",
            clientIpAddress, request.Method, path, ...)

        if internalPaths[path] {
            log.Debug(message)   // only /health/liveness, /metrics, /health/readiness
        } else {
            log.Info(message)    // ALL other paths — unconditional
        }
    })
}
``` [1](#0-0) 

**Attacker-controlled log content:**

`getClientIpAddress()` (lines 63–75) reads `X-Real-IP` then `X-Forwarded-For` with no length validation or sanitization. Combined with an arbitrarily long request URI, each log line can be made kilobytes in size. [2](#0-1) 

**Failed assumption — rate limiting is optional and off by default:**

The only rate-limiting control is a Traefik middleware defined in the Helm chart values:

```yaml
- rateLimit:
    average: 10
    sourceCriterion:
      requestHost: true
``` [3](#0-2) 

This block is gated by `{{ if and .Values.global.middleware .Values.middleware }}`, and `global.middleware` defaults to `false`: [4](#0-3) [5](#0-4) 

When deployed without Traefik or with the default `global.middleware: false`, **no rate limiting exists at any layer**. Even when enabled, the limit is per `requestHost` (not per source IP), trivially bypassed by rotating `Host` headers.

The `TracingMiddleware` is wired directly into the HTTP server with no intervening limiter: [6](#0-5) 

### Impact Explanation
- **Disk exhaustion**: Each request writes a log line containing attacker-supplied data. With a 10 KB `X-Forwarded-For` value and a long query string, a single connection at 10,000 req/s produces ~100 MB/s of log data, exhausting typical disk partitions in minutes.
- **Log rotation thrashing**: Rapid log file cycling degrades I/O performance for the entire host, potentially starving the database or other co-located services.
- **Log integrity loss**: Legitimate audit/debug entries are buried or rotated away, defeating the purpose of the tracing middleware.
- Severity: **Medium** (griefing / availability degradation, no direct fund loss).

### Likelihood Explanation
- **Zero authentication required**: any network-reachable client can trigger this.
- **Trivially scriptable**: a single `curl` loop or `ab`/`wrk` benchmark suffices.
- **Default configuration is vulnerable**: `global.middleware: false` means most deployments have no rate limiting.
- **Amplification**: attacker controls both the IP field and the URI, multiplying per-line disk cost.

### Recommendation
1. **Application-level log rate limiting**: use a token-bucket or leaky-bucket guard around `log.Info()` inside `TracingMiddleware`, e.g. via `golang.org/x/time/rate` or a structured-logging sampler.
2. **Sanitize and truncate attacker-controlled fields** before including them in log messages: cap `clientIpAddress` to 45 characters (max IPv6 length) and `path` to a reasonable maximum (e.g. 512 bytes).
3. **Validate `X-Forwarded-For` / `X-Real-IP`**: only accept well-formed IP addresses; reject or strip malformed values.
4. **Enable the Traefik rate-limit middleware by default** (`global.middleware: true`) and switch `sourceCriterion` to `ipStrategy` rather than `requestHost` to prevent Host-header bypass.
5. Consider logging at `Debug` level for 4xx responses from unknown paths to reduce noise from scanners.

### Proof of Concept
```bash
# Step 1: craft a large attacker-controlled IP header (no validation in getClientIpAddress)
FAKE_IP=$(python3 -c "print('A'*8000)")

# Step 2: craft a long URI
LONG_PATH="/network/list?$(python3 -c "print('x'*8000)")"

# Step 3: flood the endpoint (no auth, no rate limit in default deployment)
while true; do
  curl -s -o /dev/null \
    -H "X-Forwarded-For: $FAKE_IP" \
    "http://<rosetta-host>:8082${LONG_PATH}" &
done
# Each request writes ~16 KB to the log file.
# At 1000 concurrent connections → ~16 MB/s log growth → disk full in minutes.
```

func getClientIpAddress(r *http.Request) string {
	ipAddress := r.Header.Get(xRealIpHeader)

	if len(ipAddress) == 0 {
		ipAddress = r.Header.Get(xForwardedForHeader)
	}

	if len(ipAddress) == 0 {
		ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
	}

	return ipAddress
}
