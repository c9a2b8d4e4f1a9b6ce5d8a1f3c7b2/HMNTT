### Title
Floor Division in `scaleGas()` Allows ~2x EVM Work Per Gas Token, Enabling Disproportionate CPU Load

### Summary
`ThrottleProperties.scaleGas()` uses integer floor division with `GAS_SCALE_FACTOR = 10_000`, causing all gas values in the range `[10_001, 19_999]` to consume exactly 1 token from `gasLimitBucket`. An unprivileged attacker can submit requests with `gas=19_999` to perform nearly twice the EVM computation per throttle token compared to `gas=10_001`, without any additional throttle cost, increasing node CPU consumption by up to ~100% relative to a baseline of lower-gas requests — well above the 30% threshold — without exceeding the rate limit.

### Finding Description

**Exact code location:**

`ThrottleProperties.scaleGas()` — [1](#0-0) 

```java
public long scaleGas(long gas) {
    if (gas <= GAS_SCALE_FACTOR) {   // GAS_SCALE_FACTOR = 10_000
        return 0L;
    }
    return Math.floorDiv(gas, GAS_SCALE_FACTOR);
}
```

`ThrottleManagerImpl.throttle()` — [2](#0-1) 

```java
public void throttle(ContractCallRequest request) {
    if (!rateLimitBucket.tryConsume(1)) {
        throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
    } else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
        throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
    }
```

**Root cause:** `Math.floorDiv(gas, 10_000)` produces identical results for the entire range `[10_001, 19_999]`:
- `scaleGas(10_001)` → `1`
- `scaleGas(19_999)` → `1`
- `scaleGas(20_000)` → `2`

The failed assumption is that token cost scales proportionally with gas. It does not — within each 10,000-unit band, the cost is flat, creating a ~2x exploitation window at the top of each band.

**Exploit flow:**
1. Attacker sends requests with `gas=19_999` (no authentication required, public endpoint).
2. `throttle()` calls `scaleGas(19_999)` → `1` token consumed from `gasLimitBucket`.
3. EVM executes up to 19,999 gas of computation.
4. Compare to baseline requests at `gas=10_001`: also 1 token consumed, but only ~10,001 gas of EVM work.
5. Net result: attacker doubles EVM CPU work per token consumed, staying within the same rate limit.

**Why restore() does not mitigate this:**

`ContractCallService.restoreGasToBucket()` — [3](#0-2) 

If the EVM uses all 19,999 gas: `gasRemaining = 0`, `scaleGas(0) = 0` tokens restored. Net cost = 1 token. The attacker still gets full EVM execution for 1 token. If the EVM uses < 10,000 gas: `scaleGas(gasRemaining) = 0` tokens restored (since `gasRemaining ≤ 9,999 ≤ GAS_SCALE_FACTOR`), net cost = 0 tokens — the gas throttle is entirely bypassed for that request.

### Impact Explanation

With the default `requestsPerSecond = 500` and `gasPerSecond = 7_500_000_000` (scaled to 750,000 tokens/s):

| Scenario | Req/s | Gas/req | EVM gas/s |
|---|---|---|---|
| Baseline (gas=10,001) | 500 | 10,001 | ~5M |
| Attack (gas=19,999) | 500 | 19,999 | ~10M |

The attacker achieves ~100% more EVM computation per second without sending a single additional request, exceeding the 30% threshold. For requests where EVM uses < 10,000 gas, the gas throttle is completely bypassed (0 net tokens consumed), making the `gasLimitBucket` ineffective as a resource guard for that class of requests.

### Likelihood Explanation

- **No privileges required**: the `/api/v1/contracts/call` endpoint is public.
- **Trivially reproducible**: the attacker simply sets `gas: 19999` in the JSON body.
- **No rate increase needed**: the attack operates within the existing `requestsPerSecond` limit, making it indistinguishable from normal traffic in terms of request count.
- **Repeatable indefinitely**: the condition is structural, not transient.

### Recommendation

Replace floor division with ceiling division (or round-up) so that higher gas values within a band cost at least as many tokens as lower values:

```java
public long scaleGas(long gas) {
    if (gas <= GAS_SCALE_FACTOR) {
        return 0L;
    }
    // Use ceiling division: gas=19_999 → 2 tokens, gas=10_001 → 2 tokens
    return (gas + GAS_SCALE_FACTOR - 1) / GAS_SCALE_FACTOR;
}
```

Alternatively, reduce `GAS_SCALE_FACTOR` to a smaller value (e.g., `1_000`) to reduce the granularity window, or enforce a minimum token cost of 1 per 10,000 gas started (ceiling semantics). Also audit `restore()` to ensure the same ceiling logic is applied symmetrically.

### Proof of Concept

```bash
# Baseline: send 500 req/s with gas=10_001 (1 token each, ~5M EVM gas/s)
for i in $(seq 1 500); do
  curl -s -X POST http://<mirror-node>/api/v1/contracts/call \
    -H 'Content-Type: application/json' \
    -d '{"data":"0x<expensive_opcode_payload>","gas":10001,"to":"0x<contract>"}' &
done

# Attack: send 500 req/s with gas=19_999 (still 1 token each, ~10M EVM gas/s)
for i in $(seq 1 500); do
  curl -s -X POST http://<mirror-node>/api/v1/contracts/call \
    -H 'Content-Type: application/json' \
    -d '{"data":"0x<expensive_opcode_payload>","gas":19999,"to":"0x<contract>"}' &
done

# Observe: CPU usage approximately doubles with no change in request rate,
# no throttle rejection, and no authentication required.
# scaleGas(19999) = Math.floorDiv(19999, 10000) = 1  (same as gas=10001)
```

    private void restoreGasToBucket(EvmTransactionResult result, long gasLimit) {
        // If the transaction fails, gasUsed is equal to gasLimit, so restore the configured refund percent
        // of the gasLimit value back in the bucket.
        final var gasLimitToRestoreBaseline = (long) (gasLimit * throttleProperties.getGasLimitRefundPercent() / 100f);
        if (result == null || (!result.isSuccessful() && gasLimit == result.gasUsed())) {
            throttleManager.restore(gasLimitToRestoreBaseline);
        } else {
            // The transaction was successful or reverted, so restore the remaining gas back in the bucket or
            // the configured refund percent of the gasLimit value back in the bucket - whichever is lower.
            final var gasRemaining = gasLimit - result.gasUsed();
            throttleManager.restore(Math.min(gasRemaining, gasLimitToRestoreBaseline));
        }
