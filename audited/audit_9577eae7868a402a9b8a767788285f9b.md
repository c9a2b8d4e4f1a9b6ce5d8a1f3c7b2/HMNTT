### Title
Unauthenticated Prometheus `/metrics` Endpoint Exposed on Public Rosetta API Port

### Summary
The Rosetta API server registers a `GET /metrics` route via `metricsController.Routes()` that directly invokes `promhttp.Handler().ServeHTTP` with no authentication check. The endpoint is bound to the same public-facing port as all other Rosetta API routes, and no authentication middleware is applied anywhere in the request chain. Any unprivileged external user with network access to the server port can freely scrape all Prometheus metrics.

### Finding Description
**Exact code path:**

In `rosetta/app/middleware/metrics.go`, lines 64–73, `Routes()` registers the handler:

```go
func (c *metricsController) Routes() server.Routes {
    return server.Routes{
        {
            "metrics",
            "GET",
            metricsPath,   // = "/metrics"
            promhttp.Handler().ServeHTTP,  // no auth wrapper
        },
    }
}
```

In `rosetta/main.go`, lines 106–119 (online) and 148–152 (offline), `metricsController` is passed directly into `server.NewRouter(...)` alongside all other controllers. The full middleware chain assembled in `main()` (lines 217–219) is:

```
MetricsMiddleware(router) → TracingMiddleware → CorsMiddleware → http.Server
```

None of `MetricsMiddleware`, `TracingMiddleware`, or `CorsMiddleware` apply any authentication to the `/metrics` route. The server binds to a single port (`rosettaConfig.Port`), so `/metrics` is reachable on the same address as all Rosetta API endpoints.

**Root cause / failed assumption:** The developer assumed the `/metrics` endpoint would be protected by network-level controls (firewall, reverse proxy), but no such enforcement exists in the application itself. By contrast, the REST API's `rest/middleware/metricsHandler.js` (lines 137–144) explicitly implements an `authenticate()` function gated on `config.metrics.config.authentication` and returns HTTP 401 when credentials are absent or invalid — the Rosetta Go implementation has no equivalent.

**Why existing checks are insufficient:** The `MetricsMiddleware` wraps the router only to *instrument* requests (recording histograms/gauges); it does not gate access. `TracingMiddleware` and `CorsMiddleware` are also purely observability/header-manipulation layers. There is no `authMiddleware` in the Rosetta server at all.

### Impact Explanation
The `promhttp.Handler()` serves the full `prometheus.DefaultRegistry`, which includes not only the four custom histograms/gauges (`hiero_mirror_rosetta_request_bytes`, `hiero_mirror_rosetta_request_duration`, `hiero_mirror_rosetta_request_inflight`, `hiero_mirror_rosetta_response_bytes`) but also all default Go runtime collectors: goroutine counts, GC pause times, heap allocations, open file descriptors, and process CPU/memory. An attacker gains:

- **Traffic pattern intelligence**: per-route request counts and durations reveal which Rosetta endpoints are active and at what rate, enabling correlation of submitted construction transactions with observed metric deltas.
- **Inflight request timing**: `hiero_mirror_rosetta_request_inflight` allows an attacker to confirm whether their own submitted transaction is currently being processed, enabling precise timing of follow-on attacks.
- **Operational reconnaissance**: memory, GC, and goroutine metrics reveal server load and health, useful for timing DoS or resource-exhaustion attacks.

### Likelihood Explanation
Preconditions: none beyond network reachability of the Rosetta port (default 5700, publicly documented). Trigger: a single unauthenticated HTTP GET. Repeatability: unlimited, no rate limiting on the `/metrics` route itself. Any external attacker, automated scanner, or malicious exchange integrator can exploit this trivially.

### Recommendation
Wrap the `promhttp.Handler()` in an authentication-checking handler before registering it in `Routes()`, mirroring the pattern in `rest/middleware/metricsHandler.js`. Concretely:

1. Add an `authentication` field to the Rosetta config (username/password or token).
2. In `Routes()`, replace `promhttp.Handler().ServeHTTP` with a wrapper that validates credentials (using constant-time comparison) and returns HTTP 401 if they are absent or invalid.
3. Alternatively, bind the `/metrics` endpoint to a separate, non-public port (e.g., a dedicated management listener) so it is never reachable from the external network.
