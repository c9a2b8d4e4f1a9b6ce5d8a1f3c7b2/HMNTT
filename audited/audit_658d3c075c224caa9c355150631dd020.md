### Title
Unauthenticated `/mempool` Endpoint Lacks Rate Limiting, Enabling DoS Against Shared HTTP Server Including `/construction/submit`

### Summary
The `Mempool()` handler in `rosetta/app/services/mempool_service.go` accepts arbitrary unauthenticated POST requests, discards all input, and immediately returns `ErrNotImplemented`. No application-level rate limiting exists, and the only infrastructure-level rate limiting (Traefik middleware) is **disabled by default** via `global.middleware: false`. Because `/mempool` and `/construction/submit` share the same `http.Server` instance, flooding `/mempool` exhausts the shared connection/goroutine pool and degrades legitimate transaction submissions.

### Finding Description

**Exact code path:**

`rosetta/app/services/mempool_service.go`, lines 22–27 — the handler unconditionally returns `ErrNotImplemented` with zero work:
```go
func (m *mempoolAPIService) Mempool(
    _ context.Context,
    _ *types.NetworkRequest,
) (*types.MempoolResponse, *types.Error) {
    return nil, errors.ErrNotImplemented
}
```

**Router registration** (`rosetta/main.go`, lines 89–90, 111–119): `mempoolAPIController` and `constructionAPIController` are co-registered in `server.NewRouter(...)` and served by the **same** `http.Server` on the same port.

**Middleware chain** (`rosetta/main.go`, lines 217–219): only `MetricsMiddleware` → `TracingMiddleware` → `CorsMiddleware` are applied. None of these enforce rate limits or connection caps.

**Infrastructure rate limiting is off by default**: `charts/hedera-mirror-rosetta/values.yaml` line 95 sets `global.middleware: false`. The Traefik `Middleware` CRD (which contains `inFlightReq: amount: 5` and `rateLimit: average: 10`) is only rendered when `global.middleware && middleware` are both truthy (`charts/hedera-mirror-rosetta/templates/middleware.yaml`, line 3). With the default value, the CRD is never created, so no Traefik-level throttling is active.

**Root cause**: The failed assumption is that infrastructure-level rate limiting (Traefik) is always present. It is opt-in and off by default, leaving the application with no rate limiting on any endpoint, including `/mempool`.

### Impact Explanation

Each `/mempool` POST causes the Go HTTP server to spawn a goroutine, run the `rosetta-sdk-go` asserter (JSON decode + schema validation), call the trivial handler, encode the error response, and write it back. While each request is cheap, the Go runtime has no built-in cap on concurrent goroutines from `net/http`. An attacker sending thousands of concurrent connections saturates the OS TCP accept queue and the server's goroutine pool. Because `/construction/submit` shares the same listener and goroutine pool, legitimate transaction submissions are queued behind or dropped entirely. The `http.Server` timeouts (`ReadTimeout: 5s`, `WriteTimeout: 10s`) slow connection recycling but do not prevent new connections from being accepted faster than they are drained under flood conditions.

### Likelihood Explanation

No authentication, API key, or client certificate is required to reach `/mempool`. The endpoint is publicly listed in the Helm ingress paths (`values.yaml` line 131: `/rosetta/mempool`). Any attacker with network access and a basic HTTP flood tool (e.g., `wrk`, `hey`, `ab`) can execute this. The default deployment ships with rate limiting disabled, making this exploitable in any standard Helm install without operator intervention.

### Recommendation

1. **Enable rate limiting by default**: Change `global.middleware` default to `true` in `charts/hedera-mirror-rosetta/values.yaml` line 95, or decouple the middleware CRD creation from `global.middleware` so it is always rendered for the rosetta chart.
2. **Add application-level concurrency control**: Insert a `golang.org/x/time/rate` token-bucket limiter or `http.MaxBytesReader` + semaphore middleware in `rosetta/main.go` before the router, independent of Traefik.
3. **Remove or restrict the `/mempool` route**: Since the endpoint is permanently unimplemented, consider removing `mempoolAPIController` from the online router registration (`main.go` lines 89–90, 114) to eliminate the attack surface entirely.

### Proof of Concept

**Preconditions**: Default Helm deployment (`global.middleware: false`), server reachable on port 5700.

```bash
# Step 1 – confirm /mempool is live and returns 501
curl -s -X POST http://<host>:5700/mempool \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"testnet"}}' | jq .

# Step 2 – flood /mempool with concurrent requests
hey -n 100000 -c 500 -m POST \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"testnet"}}' \
  http://<host>:5700/mempool &

# Step 3 – simultaneously attempt a legitimate /construction/submit
curl -s -X POST http://<host>:5700/construction/submit \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"testnet"},"signed_transaction":"<valid_hex>"}' \
  --max-time 5

# Expected result: /construction/submit times out or returns connection-refused
# while the flood is active, demonstrating availability degradation.
```
