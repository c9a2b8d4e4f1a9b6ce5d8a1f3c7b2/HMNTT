### Title
`BufferUnderflowException` via Undersized EVM Address in `getByEvmAddressAndType()`

### Summary
`decodeEvmAddress()` performs no length validation on the decoded byte array before it is wrapped in a `ByteBuffer`. An unprivileged user can supply a hex string of exactly 8 characters (4 bytes), causing `buffer.getInt()` to succeed but the immediately following `buffer.getLong()` to throw an uncaught `BufferUnderflowException`. This exception is unhandled within the function and propagates up through the GraphQL layer, producing an uncontrolled error response on every such request.

### Finding Description

**Code path:**

`GraphQlUtils.decodeEvmAddress()` — [1](#0-0) 

The function strips the optional `0x` prefix and calls `Hex.decodeHex()`. The only guard is a null check (line 88). There is **no length assertion** — any even-length hex string is accepted and decoded to a byte array of arbitrary size.

`EntityServiceImpl.getByEvmAddressAndType()` — [2](#0-1) 

The returned byte array is immediately wrapped:
```java
var buffer = ByteBuffer.wrap(evmAddressBytes);          // line 36
if (buffer.getInt() == 0 && buffer.getLong() == 0) {   // line 37
    return entityRepository.findById(buffer.getLong())  // line 38
```

A valid EVM address is 20 bytes. The code consumes:
- 4 bytes via `getInt()` (line 37)
- 8 bytes via first `getLong()` (line 37)
- 8 bytes via second `getLong()` (line 38)

With a 4-byte input (8 hex chars, e.g. `0x00000000`):
1. `buffer.getInt()` reads all 4 bytes → returns `0`, succeeds.
2. `buffer.getLong()` attempts to read 8 bytes from an empty buffer → throws `java.nio.BufferUnderflowException`.

`BufferUnderflowException` is an unchecked `RuntimeException`. Neither `getByEvmAddressAndType()` nor `decodeEvmAddress()` catches it. [3](#0-2) 

**Why existing checks are insufficient:**

`decodeEvmAddress` only guards against `null` input and invalid hex characters. [4](#0-3)  There is no assertion that `evmAddressBytes.length == 20` (or any minimum length) before the `ByteBuffer` operations begin.

### Impact Explanation
Every request carrying a short hex EVM address triggers an unhandled `BufferUnderflowException`. The GraphQL framework will catch it at the dispatcher level and return an error response, but the exception bypasses normal application logic, may produce verbose stack traces in logs, and can be triggered repeatedly at zero cost by any unauthenticated caller. This constitutes a reliable griefing vector causing degraded observability and noisy error logs, consistent with the "Medium / griefing, no economic damage" scope classification.

### Likelihood Explanation
No authentication or authorization is required. The GraphQL `evmAddress` field accepts a plain string. An attacker needs only to submit a syntactically valid but short hex string (e.g., `0x00000000`). The condition is trivially reproducible, requires no special knowledge, and can be scripted for continuous bombardment.

### Recommendation
Add an explicit length check in `decodeEvmAddress` or at the top of `getByEvmAddressAndType()` before any `ByteBuffer` operations:

```java
byte[] evmAddressBytes = decodeEvmAddress(evmAddress);
if (evmAddressBytes.length != 20) {
    throw new IllegalArgumentException("evmAddress must be exactly 20 bytes");
}
```

Alternatively, enforce the constraint at the GraphQL schema level using a custom scalar or directive that rejects strings whose decoded length is not exactly 20 bytes, so invalid input is rejected before it reaches service code.

### Proof of Concept
1. Send the following GraphQL query to the mirror-node GraphQL endpoint (no credentials required):
```graphql
{
  account(input: { evmAddress: "0x00000000" }) {
    id
  }
}
```
2. The 8-character hex string `00000000` decodes to a 4-byte array.
3. `ByteBuffer.wrap([0x00,0x00,0x00,0x00]).getInt()` returns `0` (condition passes).
4. The subsequent `.getLong()` call on the now-empty buffer throws `BufferUnderflowException`.
5. The server returns a GraphQL error response; the exception and stack trace appear in server logs.
6. Repeat indefinitely with no rate limiting or authentication barrier.

```java
    public static byte[] decodeEvmAddress(String evmAddress) {
        if (evmAddress == null) {
            return ArrayUtils.EMPTY_BYTE_ARRAY;
        }

        try {
            evmAddress = Strings.CS.removeStart(evmAddress, HEX_PREFIX);
            return Hex.decodeHex(evmAddress);
        } catch (DecoderException e) {
            throw new IllegalArgumentException("Unable to decode evmAddress: " + evmAddress);
        }
    }
```

```java
    public Optional<Entity> getByEvmAddressAndType(String evmAddress, EntityType type) {
        byte[] evmAddressBytes = decodeEvmAddress(evmAddress);
        var buffer = ByteBuffer.wrap(evmAddressBytes);
        if (buffer.getInt() == 0 && buffer.getLong() == 0) {
            return entityRepository.findById(buffer.getLong()).filter(e -> e.getType() == type);
        }
        return entityRepository.findByEvmAddress(evmAddressBytes).filter(e -> e.getType() == type);
    }
```
