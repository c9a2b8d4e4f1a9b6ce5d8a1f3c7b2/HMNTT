### Title
Unauthenticated or Default-Credential Access to Process Metrics Endpoint Exposes Server Internals

### Summary
The `metricsMiddleware` in `rest/middleware/metricsHandler.js` serves `nodejs_process_memory_rss_bytes`, `nodejs_process_memory_heap_used_bytes`, and `nodejs_process_cpu_usage_percentage` on the same public port (5551) as the REST API. Authentication is bypassed entirely when `config.metrics.config.authentication` is falsy, and even when enabled, the default credentials (`mirror_api_metrics` / `mirror_api_metrics_pass`) are publicly documented, allowing any external actor to read live server resource metrics.

### Finding Description
**Code location:** `rest/middleware/metricsHandler.js`, lines 137–164; `rest/server.js`, lines 89–92.

The `authenticate` function reads the `authentication` flag from config:

```js
// metricsHandler.js lines 137-144
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;          // ← completely open when flag is falsy
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
``` [1](#0-0) 

When `authentication` is `false` (or absent), `authenticate` returns `true` unconditionally and the full Prometheus scrape — including `nodejs_process_memory_rss_bytes`, `nodejs_process_memory_heap_used_bytes`, and `nodejs_process_cpu_usage_percentage` — is served without any credential check:

```js
// metricsHandler.js lines 156-164
if (normalizedPath === metricsPath) {
  if (!authenticate(req)) {
    res.set('WWW-Authenticate', 'Basic realm="Metrics"');
    return res.status(401).send('Unauthorized');
  }
  return exporter.collect().then(({resourceMetrics}) => {
    res.set('Content-Type', 'text/plain; charset=utf-8');
    res.send(serializer.serialize(resourceMetrics));   // ← full metrics dump
  });
}
``` [2](#0-1) 

The metrics middleware is registered on the main Express app — the same process and port (5551) that serves the public REST API: [3](#0-2) 

Even when `authentication: true`, the default credentials are publicly documented as `mirror_api_metrics` / `mirror_api_metrics_pass`: [4](#0-3) 

The process metrics themselves are registered unconditionally at startup: [5](#0-4) 

### Impact Explanation
An attacker who can reach port 5551 (the public API port) can poll `/swagger/metrics/` to obtain:
- **`nodejs_process_memory_rss_bytes` / `nodejs_process_memory_heap_used_bytes`**: reveals memory pressure trends, useful for timing resource-exhaustion attacks or detecting when the server is under load.
- **`nodejs_process_cpu_usage_percentage`**: reveals CPU saturation, enabling an attacker to time requests to coincide with low-load windows (reducing detection probability) or to confirm that a DoS payload is having effect.
- **`api_all_request_in_processing_total`, `api_request_duration_milliseconds`**: reveals in-flight request counts and latency distributions, aiding timing-based inference attacks.

Severity: **Medium** — no direct code execution, but meaningful operational intelligence is leaked to unauthenticated or trivially-authenticated external actors.

### Likelihood Explanation
- **Path 1 (auth disabled):** Any deployment where `authentication: false` is set (e.g., internal clusters assumed to be network-isolated but inadvertently exposed) is fully open. No credentials required.
- **Path 2 (default credentials):** Any deployment that has not rotated the documented defaults is exploitable by anyone who reads the public configuration documentation. The credentials are a single HTTP Basic Auth header away.
- The endpoint is on the same port as the public API, so no firewall bypass is needed if the API is internet-facing.
- Polling is trivially repeatable (e.g., `watch -n 1 curl http://host:5551/swagger/metrics/`).

### Recommendation
1. **Enforce authentication unconditionally**: Remove the `if (!authentication) { return true; }` escape hatch, or at minimum emit a loud startup warning when it is disabled.
2. **Separate the metrics port**: Serve the metrics endpoint on a dedicated internal-only port (e.g., 9090) rather than co-hosting it on the public API port.
3. **Require credential rotation**: Reject startup if the metrics password equals the documented default value.
4. **Restrict metric scope**: Do not expose process-level OS metrics (`rss`, `heapUsed`, CPU %) on any externally reachable endpoint; reserve those for internal Prometheus scrapers only.

### Proof of Concept

**Precondition A — auth disabled (`authentication: false` in config):**
```bash
curl -s http://<mirror-node-host>:5551/swagger/metrics/ | grep -E 'nodejs_process_memory|nodejs_process_cpu'
```
Returns live values with no credentials.

**Precondition B — default credentials not rotated:**
```bash
curl -s -u mirror_api_metrics:mirror_api_metrics_pass \
  http://<mirror-node-host>:5551/swagger/metrics/ \
  | grep -E 'nodejs_process_memory_rss_bytes|nodejs_process_memory_heap_used_bytes|nodejs_process_cpu_usage_percentage'
```
Expected output (example):
```
nodejs_process_memory_rss_bytes 1.23456789e+08
nodejs_process_memory_heap_used_bytes 4.5678901e+07
nodejs_process_cpu_usage_percentage 12.3
```

Poll repeatedly to build a time-series of server load, memory pressure, and deployment events (memory spikes at restart, CPU spikes under attack).
