### Title
Empty `Optional` Cache Poisoning in `ContractRepository.findRuntimeBytecode()` Enables Persistent Contract DoS

### Summary
The `@Cacheable` annotation on `findRuntimeBytecode()` uses `unless = "#result == null"`, which only prevents caching of literal `null` values. Because `Optional.empty()` is a non-null Java object, a miss-query (contract not yet ingested) stores an empty `Optional` in the cache. The contract cache is configured with `expireAfterAccess=1h`, so the poisoned entry persists — and resets its TTL on every access — effectively making the contract invisible to the EVM for at least one hour per attacker request.

### Finding Description

**Exact code location:**

`web3/src/main/java/org/hiero/mirror/web3/repository/ContractRepository.java`, line 16–18:
```java
@Cacheable(cacheNames = CACHE_NAME_CONTRACT, cacheManager = CACHE_MANAGER_CONTRACT, unless = "#result == null")
@Query(value = "select runtime_bytecode from contract where id = :contractId", nativeQuery = true)
Optional<byte[]> findRuntimeBytecode(final Long contractId);
``` [1](#0-0) 

**Root cause:** Spring's `unless` SpEL expression `#result == null` evaluates to `false` when the result is `Optional.empty()`, because `Optional.empty()` is a non-null singleton object. Spring therefore stores the empty `Optional` in the Caffeine cache.

**Cache TTL:** The contract cache is configured as `expireAfterAccess=1h,maximumSize=1000,recordStats`. [2](#0-1) 

Because the policy is `expireAfterAccess` (not `expireAfterWrite`), every subsequent read of the poisoned entry resets the 1-hour clock, allowing an attacker to keep the entry alive indefinitely with periodic requests.

**Downstream impact path:** `ContractBytecodeReadableKVState.readFromDataSource()` calls `contractRepository.findRuntimeBytecode(entityId.getId())` and maps the result to a `Bytecode` object, returning `null` when the `Optional` is empty. [3](#0-2) 

A `null` return from `readFromDataSource` causes the EVM to treat the contract as non-existent, reverting all `eth_call` executions targeting it.

**Why the existing check fails:** `unless = "#result == null"` is the only guard. There is no check for `!#result.isPresent()`. The test `ContractRepositoryTest.findRuntimeBytecodeFailCall()` confirms that `findRuntimeBytecode` returns `Optional.empty()` (not `null`) for a missing contract. [4](#0-3) 

### Impact Explanation
Any contract whose `contractId` is queried before the mirror node has ingested its `runtime_bytecode` row will have an empty `Optional` cached. All subsequent `eth_call` / `eth_estimateGas` requests for that contract will receive EVM execution failures (contract treated as non-existent) for up to 1 hour — or indefinitely if the attacker periodically re-queries the same ID. This is a targeted, per-contract denial-of-service with no authentication required.

### Likelihood Explanation
Hedera contract IDs are sequential and publicly observable on the network. An attacker can:
1. Watch the mempool or consensus stream for a `ContractCreate` transaction.
2. Immediately send an `eth_call` to the mirror node for the new contract ID before the mirror node's ingestion pipeline has written the `runtime_bytecode` row.
3. The mirror node's ingestion lag (typically seconds to tens of seconds) provides a reliable race window.

No privileged access, no special tooling, and no on-chain funds are required. The attack is repeatable and scriptable.

### Recommendation
Change the `unless` condition to also exclude empty `Optional` results:

```java
@Cacheable(
    cacheNames = CACHE_NAME_CONTRACT,
    cacheManager = CACHE_MANAGER_CONTRACT,
    unless = "#result == null || !#result.isPresent()"
)
```

Additionally, consider switching the contract cache from `expireAfterAccess` to `expireAfterWrite` to prevent indefinite TTL extension via repeated reads of a stale (including poisoned) entry. [2](#0-1) 

### Proof of Concept
1. Observe a `ContractCreate` transaction on the Hedera network targeting a new contract ID, e.g. `0.0.5000`.
2. Before the mirror node ingests the transaction (within the ingestion lag window), send:
   ```
   POST /api/v1/contracts/call
   { "to": "0x0000000000000000000000000000000000001388", "data": "0x..." }
   ```
   (where `0x1388` = 5000 in hex, the long-zero EVM address for `0.0.5000`)
3. The mirror node queries `findRuntimeBytecode(5000)`, gets `Optional.empty()`, and caches it.
4. After the mirror node fully ingests the contract, repeat the same `eth_call`. The cached `Optional.empty()` is returned; `readFromDataSource` returns `null`; the EVM reverts with "contract does not exist."
5. The contract remains inaccessible for 1 hour from the last access. Sending a request every 59 minutes keeps it inaccessible indefinitely.
