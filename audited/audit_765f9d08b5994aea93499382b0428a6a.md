### Title
Unauthenticated HTTP Address Book Fetch Enables MITM Node Injection in NodeSupplier

### Summary
When `MirrorNodeProperties.RestProperties.port != 443`, `getBaseUrl()` constructs a plain `http://` URL, causing `RestApiClient.getNodes()` to fetch the network address book over unencrypted HTTP with no response authentication. A network-level attacker can intercept and replace the `NetworkNodesResponse` with crafted `NetworkNode` objects containing attacker-controlled endpoints and cert hashes. The `validateNode()` check is insufficient because the Hedera Java SDK does not cryptographically verify receipt signatures returned by the node, allowing an attacker-controlled gRPC server to return a synthetic `SUCCESS` receipt and pass validation.

### Finding Description

**Code path:**

`MirrorNodeProperties.RestProperties.getBaseUrl()` — [1](#0-0)  — selects `http://` when `port != 443`, with no override or enforcement of TLS.

`RestApiClient.getNodes()` — [2](#0-1)  — issues a plain unauthenticated GET to `/network/nodes?limit=25` and paginates through all pages. No HMAC, no signature, no certificate pinning on the REST response itself.

`NodeSupplier.getAddressBook()` — [3](#0-2)  — blindly converts every returned `NetworkNode` into `NodeProperties` via `toNodeProperties()`.

`toNodeProperties()` — [4](#0-3)  — copies `nodeAccountId`, `nodeCertHash`, `domainName`/`ipAddressV4`, and `port` directly from the REST response into `NodeProperties` with no allowlist or integrity check.

`toClient()` — [5](#0-4)  — builds a `NodeAddressBook` from the injected `certHash` and sets it on the SDK client via `setNetworkFromAddressBook`. This means the attacker-supplied cert hash becomes the TLS pin for the connection to the attacker's node.

**Why `validateNode()` fails as a defense:**

`validateNode()` — [6](#0-5)  — submits a `TransferTransaction` to the candidate node and checks `receiptStatus == SUCCESS`. The Hedera Java SDK's `getReceipt()` call queries the node itself for the receipt protobuf and checks the `status` field. It does **not** verify a cryptographic signature over the receipt. An attacker-controlled gRPC server can accept the connection (using the TLS cert whose hash was injected), receive the transaction, and return a hand-crafted `TransactionGetReceiptResponse` with `status = SUCCESS`. The SDK accepts this and `validateNode()` returns `true`, adding the attacker's node to the live `nodes` list.

**Root cause:** The failed assumption is that the mirror node REST API is always served over HTTPS with a trusted certificate, making the response tamper-proof. When `port != 443`, this assumption is broken by design in `getBaseUrl()`, and no compensating control (response signing, allowlist of known node IPs, or cert-hash cross-validation against a trusted source) exists.

### Impact Explanation
Once an attacker's node is in the `CopyOnWriteArrayList<NodeProperties> nodes` — [7](#0-6)  — the round-robin `get()` method — [8](#0-7)  — will route a fraction of all subsequent monitoring transactions to the attacker's node. The attacker can silently drop transactions (causing false negative health alerts), return fabricated receipts (corrupting monitoring data), or log transaction metadata. The scope classification of "no concrete funds at direct risk" is consistent: the monitor sends 1-tinyhbar validation transfers, not user funds. The impact is integrity of the network health signal and potential for targeted DoS of the monitoring layer.

### Likelihood Explanation
The precondition of `port != 443` is a non-default configuration but is explicitly supported by the code and documented by the conditional in `getBaseUrl()`. In non-production, staging, or internal deployments where HTTP is used, this is realistic. The attacker needs a MITM position on the network path between the monitor and the mirror node REST endpoint — achievable via ARP spoofing on a shared LAN, DNS hijacking, or a compromised network device. No application-level credentials are required. The attack is repeatable on every address book refresh cycle (default: once per day — [9](#0-8) ).

### Recommendation
1. **Enforce HTTPS unconditionally** in `getBaseUrl()`: remove the `port == 443` branch and always use `https://`, or add a configuration flag that explicitly requires operator opt-in for HTTP with a loud warning.
2. **Validate the address book response** against a known-good source (e.g., cross-check returned node cert hashes against the on-ledger address book retrieved via a trusted SDK call, not the same HTTP endpoint).
3. **Allowlist node account IDs**: reject any `NetworkNode` whose `nodeAccountId` is not in a configured set of known consensus node account IDs.
4. **Pin the REST API TLS certificate** or use mutual TLS for the mirror node REST connection.

### Proof of Concept
1. Configure the monitor with `mirrorNode.rest.port=80` (or any port other than 443). `getBaseUrl()` returns `http://`.
2. Position a MITM proxy (e.g., `mitmproxy`) on the network path between the monitor host and the mirror node REST host.
3. Intercept the GET `/api/v1/network/nodes?limit=25` response and replace it with a crafted JSON body containing a single `NetworkNode` with:
   - `node_account_id`: a valid-looking account ID (e.g., `0.0.3`)
   - `service_endpoints`: `[{"ip_address_v4": "<attacker-IP>", "port": 50211}]`
   - `node_cert_hash`: the SHA-384 hash of the attacker's self-signed TLS certificate
4. Run an attacker-controlled gRPC server at `<attacker-IP>:50211` that accepts any `TransactionProto` and returns a `TransactionGetReceiptResponse` with `status = SUCCESS`.
5. Observe that `validateNode()` connects to the attacker's server (using the injected cert hash for pinning), receives the fake `SUCCESS` receipt, and calls `nodes.addIfAbsent(node)`.
6. Subsequent calls to `NodeSupplier.get()` will return the attacker's node for transaction submission.

    private List<NodeProperties> toNodeProperties(NetworkNode networkNode) {
        var nodeValidation = monitorProperties.getNodeValidation();
        var tlsPredicate = nodeValidation.getTls().getPredicate();
        return networkNode.getServiceEndpoints().stream()
                .filter(s -> tlsPredicate.test(s.getPort()))
                .limit(nodeValidation.getMaxEndpointsPerNode())
                .map(serviceEndpoint -> {
                    var host = StringUtils.isNotBlank(serviceEndpoint.getDomainName())
                            ? serviceEndpoint.getDomainName()
                            : serviceEndpoint.getIpAddressV4();
                    var nodeProperties = new NodeProperties();
                    nodeProperties.setAccountId(networkNode.getNodeAccountId());
                    nodeProperties.setCertHash(Strings.CS.remove(networkNode.getNodeCertHash(), "0x"));
                    nodeProperties.setHost(host);
                    nodeProperties.setNodeId(networkNode.getNodeId());
                    nodeProperties.setPort(serviceEndpoint.getPort());
                    return nodeProperties;
                })
                .toList();
    }

    boolean validateNode(NodeProperties node) {
        if (!monitorProperties.getNodeValidation().isEnabled()) {
            nodes.addIfAbsent(node);
            log.info("Adding node {} without validation", node.getAccountId());
            return true;
        }

        log.info("Validating node {}", node);
        Hbar hbar = Hbar.fromTinybars(1L);
        AccountId nodeAccountId = AccountId.fromString(node.getAccountId());

        try (Client client = toClient(node)) {
            Status receiptStatus = new TransferTransaction()
                    .addHbarTransfer(nodeAccountId, hbar)
                    .addHbarTransfer(client.getOperatorAccountId(), hbar.negated())
                    .setNodeAccountIds(node.getAccountIds())
                    .execute(client)
                    .getReceipt(client)
                    .status;

            if (receiptStatus == SUCCESS) {
                log.info("Validated node {} successfully", nodeAccountId);
                nodes.addIfAbsent(node);
                return true;
            }

            log.warn("Unable to validate node {}: invalid status code {}", node, receiptStatus);
        } catch (TimeoutException e) {
            log.warn("Unable to validate node {}: Timed out", node);
        } catch (Exception e) {
            log.warn("Unable to validate node {}: ", node, e);
        }

        nodes.remove(node);
        return false;
    }
