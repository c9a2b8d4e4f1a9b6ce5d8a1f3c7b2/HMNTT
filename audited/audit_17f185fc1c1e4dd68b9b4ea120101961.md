### Title
Unauthenticated Pagination Amplification via `limit=1` on `/api/v1/network/registered-nodes`

### Summary
The `GET /api/v1/network/registered-nodes` endpoint accepts `limit=1` from any unauthenticated caller. Because `LinkFactoryImpl` faithfully copies the caller-supplied `limit` into every generated `next` link, an attacker can follow the chain of next links one node at a time, issuing N sequential database queries to enumerate N nodes instead of the ceil(N/100) queries that would result from using the maximum page size. No credentials, tokens, or special network access are required.

### Finding Description
**Root cause — `RegisteredNodesRequest.java` lines 31–35:**

```java
@Min(1)          // ← minimum is 1, not a higher floor
@Max(MAX_LIMIT)  // MAX_LIMIT = 100
@RestJavaQueryParam(name = LIMIT, required = false, defaultValue = DEFAULT_LIMIT)
private int limit = Integer.parseInt(DEFAULT_LIMIT);
``` [1](#0-0) 

`@Min(1)` is the only lower-bound guard. There is no minimum floor of, say, 10 or 25.

**No rate limiting exists** in `NetworkController` for this endpoint. [6](#0-5) 

### Impact Explanation
With N registered nodes in the database, an attacker using `limit=1` causes N individual SQL queries against the `registered_node` table. A legitimate client using `limit=100` (the maximum) causes only ceil(N/100) queries — a 100× reduction. Because the endpoint is public and unauthenticated, any external actor can sustain this amplified query rate continuously. The impact is elevated database CPU and I/O load (griefing / degraded availability), with no economic damage to network users as scoped.

### Likelihood Explanation
The attack requires zero privileges, zero tokens, and only a standard HTTP client. The `next` link is self-describing — the attacker does not need to reverse-engineer pagination; the API hands them the next URL. The attack is trivially scriptable (`while next_link: fetch(next_link)`) and can be run from multiple IPs simultaneously. The integration test already demonstrates the exact exploit path works end-to-end.

### Recommendation
Enforce a minimum page size that meaningfully limits amplification. Options:

1. **Raise `@Min` to a reasonable floor** (e.g., `@Min(10)` or `@Min(25)`) in `RegisteredNodesRequest.java`.
2. **Add a `MIN_LIMIT` constant** in `Constants.java` (analogous to `MAX_LIMIT`) and reference it in the annotation.
3. **Add per-IP or per-endpoint rate limiting** at the gateway/filter layer so that even with `limit=1` the total query rate is bounded.

Option 1 alone reduces the worst-case amplification factor from 100× to 10× or 4× respectively, and is the lowest-effort fix.

### Proof of Concept
```bash
# Step 1 – seed the amplification
NEXT="/api/v1/network/registered-nodes?limit=1"
BASE="http://<mirror-node-host>"

# Step 2 – iterate through every registered node one at a time
while [ -n "$NEXT" ]; do
    RESPONSE=$(curl -s "${BASE}${NEXT}")
    echo "$RESPONSE" | jq '.registered_nodes[0].registered_node_id'
    NEXT=$(echo "$RESPONSE" | jq -r '.links.next // empty')
done
# Result: N HTTP requests → N database queries
# vs. ceil(N/100) queries with limit=100
```

Each iteration issues one `SELECT … FROM registered_node WHERE registered_node_id >= :lb AND registered_node_id <= :ub AND deleted IS FALSE …` query. With 1 000 registered nodes this is 1 000 queries instead of 10.

```java
    public <T> Links create(List<T> items, Pageable pageable, Function<T, Map<String, String>> extractor) {
        if (CollectionUtils.isEmpty(items) || pageable.getPageSize() > items.size()) {
            return DEFAULT_LINKS;
        }

        final var servletRequestAttributes = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        if (servletRequestAttributes == null || servletRequestAttributes.getResponse() == null) {
            return DEFAULT_LINKS;
        }

        var request = servletRequestAttributes.getRequest();
        var lastItem = Objects.requireNonNull(CollectionUtils.lastElement(items));
        var nextLink = createNextLink(lastItem, pageable, extractor, request);

        // If nextLink is null, it means the pagination range would be empty - no more results
        if (nextLink == null) {
            return DEFAULT_LINKS;
        }

        servletRequestAttributes.getResponse().setHeader(HttpHeaders.LINK, LINK_HEADER.formatted(nextLink));
        return new Links().next(nextLink);
    }
```
