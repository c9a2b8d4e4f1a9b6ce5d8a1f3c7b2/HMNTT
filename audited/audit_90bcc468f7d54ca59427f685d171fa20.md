### Title
Unauthenticated DoS via Uncached Multi-Table SQL Query on `/network/status` Endpoint

### Summary
`NetworkStatus()` in `rosetta/app/services/network_service.go` calls `addressBookEntryRepo.Entries()` on every request with no in-process caching. Each call executes the `latestNodeServiceEndpoints` query — a multi-table JOIN across `address_book`, `address_book_entry`, `node`, and `address_book_service_endpoint` — up to twice per request. The Traefik middleware chain that provides rate limiting and in-flight request caps is disabled by default (`global.middleware: false`), leaving the endpoint fully unprotected against flooding by any unauthenticated caller.

### Finding Description

**Code path:**

`NetworkStatus()` at [1](#0-0)  unconditionally calls `n.addressBookEntryRepo.Entries(ctx)` at line 77 on every request.

`Entries()` at [2](#0-1)  executes the `latestNodeServiceEndpoints` raw SQL query (defined at lines 18–30) against the live database with no result cache. The query performs a correlated subquery on `address_book`, then JOINs `address_book_entry`, `node`, and `address_book_service_endpoint`: [3](#0-2) 

The loop at lines 63–75 may execute this query **twice** per request (once for file_id 101, once for 102 if 101 returns empty).

**No in-process caching exists** in the rosetta Go service for this result. Compare with the gRPC Java service, which applies `@Cacheable` on its equivalent repository method: [4](#0-3) 
No equivalent protection exists in the rosetta path.

**Rate limiting is disabled by default.** The Helm chart defines a Traefik middleware chain with `inFlightReq: amount: 5` per source IP and `rateLimit: average: 10` per host: [5](#0-4) 

However, the middleware template is gated on `global.middleware`: [6](#0-5) 

And `global.middleware` defaults to `false`: [7](#0-6) 

The Go HTTP server itself has no application-level rate limiting or concurrency cap — only HTTP timeouts: [8](#0-7) 

The DB connection pool (`SetMaxOpenConns`) bounds concurrent connections but does not prevent sequential request flooding from queuing work against the database. [9](#0-8) 

### Impact Explanation
An attacker flooding `POST /network/status` forces repeated multi-table JOIN scans on the shared PostgreSQL instance. Because the mirror node's importer and rosetta service share the same database host, I/O saturation degrades the importer's ability to write newly ingested transaction records, effectively stalling the mirror node's view of the network. The rosetta service itself becomes unavailable for legitimate Rosetta API consumers (exchanges, wallets, block explorers) that depend on it for transaction construction and submission.

### Likelihood Explanation
The `/network/status` endpoint requires no authentication (Rosetta API spec mandates it be public). No API key, session token, or IP allowlist is enforced at the application layer. The Traefik protections are opt-in and off by default, meaning any default deployment is unprotected. A single attacker with a modest HTTP flood tool (e.g., `wrk`, `hey`, or a simple loop) can sustain hundreds of requests per second from a single host. The attack is trivially repeatable and requires no special knowledge of the system beyond the public Rosetta API specification.

### Recommendation
1. **Add an in-process TTL cache** for the `Entries()` result in `addressBookEntryRepository`. The address book changes infrequently (governed by `nodeRefreshInterval`, default 24h); caching for even 30–60 seconds eliminates virtually all repeated DB hits under flood conditions.
2. **Enable the Traefik middleware by default** by changing `global.middleware` default to `true` in `values.yaml`, or move the rate-limiting/inFlightReq middleware into the ingress annotations unconditionally.
3. **Add application-level concurrency limiting** in the Go HTTP handler (e.g., a semaphore or `golang.org/x/time/rate` token bucket) so the protection is not solely dependent on infrastructure configuration.
4. **Set a `statementTimeout`** in the DB config to bound the maximum duration of any single query execution.

### Proof of Concept
**Preconditions:** Default Helm deployment with `global.middleware: false` (the default); rosetta running in online mode; `/network/status` reachable.

**Steps:**
```bash
# 1. Confirm endpoint is unauthenticated
curl -s -X POST http://<rosetta-host>/network/status \
  -H "Content-Type: application/json" \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}'
# Expect 200 OK

# 2. Flood with concurrent requests (no credentials needed)
hey -n 10000 -c 100 -m POST \
  -H "Content-Type: application/json" \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' \
  http://<rosetta-host>/network/status

# 3. Observe on the PostgreSQL host:
#    - pg_stat_activity shows many concurrent latestNodeServiceEndpoints queries
#    - I/O wait climbs; importer write latency increases
#    - rosetta /network/status begins returning 500 or timing out
```

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

const latestNodeServiceEndpoints = `select
                                    abe.node_id,
                                    coalesce(n.account_id, abe.node_account_id) as node_account_id,
                                    string_agg(ip_address_v4 || ':' || port::text, ','
                                      order by ip_address_v4,port) endpoints

func (aber *addressBookEntryRepository) Entries(ctx context.Context) (*types.AddressBookEntries, *rTypes.Error) {
	db, cancel := aber.dbClient.GetDbWithContext(ctx)
	defer cancel()

	nodes := make([]nodeServiceEndpoint, 0)
	// address book file 101 has service endpoints for nodes, resort to file 102 if 101 doesn't exist
	for _, fileId := range []int64{aber.addressBook101.EncodedId, aber.addressBook102.EncodedId} {
		if err := db.Raw(
			latestNodeServiceEndpoints,
			sql.Named("file_id", fileId),
		).Scan(&nodes).Error; err != nil {
			log.Error("Failed to get latest node service endpoints", err)
			return nil, errors.ErrDatabaseError
		}

		if len(nodes) != 0 {
			break
		}
	}

	entries := make([]types.AddressBookEntry, 0, len(nodes))
	for _, node := range nodes {
		entries = append(entries, node.toAddressBookEntry())
	}

	return &types.AddressBookEntries{Entries: entries}, nil
}
