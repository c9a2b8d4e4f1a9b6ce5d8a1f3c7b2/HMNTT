### Title
Liveness Check Bypassed via Directory Substitution at `/tmp/alive`

### Summary
The `live` mode health check in `pinger/cmd/healthcheck/main.go` calls `os.Stat("/tmp/alive")` and checks only the path's `ModTime()`, without verifying the path is a regular file. Because `os.Stat` succeeds on directories and directories have valid, controllable mtimes, an attacker who can place a directory at `/tmp/alive` can keep the liveness check permanently passing even when the actual pinger heartbeat has stopped.

### Finding Description
**Exact location:** `pinger/cmd/healthcheck/main.go`, `main()`, lines 27–38.

```go
fi, err := os.Stat("/tmp/alive")   // line 27 — succeeds for directories too
if err != nil { … os.Exit(1) }

age := time.Since(fi.ModTime())    // line 34 — directories have controllable mtimes
if age > 2*time.Minute {
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
os.Exit(0)                         // passes if mtime is fresh
```

**Root cause:** The code never calls `fi.Mode().IsRegular()` (or checks `fi.IsDir()`). `os.Stat` returns a valid `FileInfo` for any filesystem object — file, directory, symlink target, etc. A directory's mtime is updated whenever an entry inside it is created or removed, giving an attacker full control over the apparent "freshness."

**Exploit flow:**

1. The pinger process writes `/tmp/alive` as a regular file and updates it periodically. The attacker waits for the pinger to crash or be killed (so the file is no longer updated).
2. The attacker removes `/tmp/alive` (possible if they own it — same UID — or if it never existed yet, e.g., a startup race) and creates a directory: `mkdir /tmp/alive`.
3. The attacker keeps the directory's mtime fresh by periodically running `touch /tmp/alive/x && rm /tmp/alive/x` (or any equivalent), which updates the directory's mtime to "now."
4. Every subsequent `healthcheck live` invocation calls `os.Stat("/tmp/alive")`, gets a `FileInfo` for the directory, reads a mtime that is seconds old, and exits 0 — reporting the process as alive.

**Why existing checks are insufficient:** The only guard is `err != nil` from `os.Stat` and the 2-minute age window. Neither check distinguishes a regular file from a directory.

### Impact Explanation
The liveness probe is the mechanism by which Kubernetes (or any container orchestrator) detects a dead pinger and triggers a pod restart. By keeping a directory at `/tmp/alive` with a fresh mtime, an attacker prevents the orchestrator from ever seeing a failed liveness check. A crashed or hung pinger continues to appear healthy indefinitely, silently dropping all pinger functionality (monitoring, alerting, heartbeat forwarding) without any operator notification.

### Likelihood Explanation
The precondition — being able to place a directory at `/tmp/alive` — is achievable in several realistic scenarios:

- **Same-UID attacker** (compromised sidecar, another process running as the same service account): can unlink the file and `mkdir` the directory at any time.
- **Startup race**: if the attacker can execute code before the pinger first creates `/tmp/alive`, they create the directory first; the pinger's subsequent `os.WriteFile("/tmp/alive", …)` will fail (cannot overwrite a directory with a file), leaving the directory in place permanently.
- **Container environments without sticky-bit `/tmp`**: some minimal base images mount `/tmp` as a plain `tmpfs` without the sticky bit, making the path world-writable and world-deletable.

No elevated privileges are required in any of these cases.

### Recommendation
Add an explicit file-type check immediately after `os.Stat` succeeds:

```go
fi, err := os.Stat("/tmp/alive")
if err != nil {
    fmt.Fprintln(os.Stderr, err)
    os.Exit(1)
}
if !fi.Mode().IsRegular() {
    fmt.Fprintln(os.Stderr, "heartbeat path is not a regular file")
    os.Exit(1)
}
```

Additionally, consider using `os.Open` + `fi.Stat()` on the open file descriptor (to avoid TOCTOU between `Stat` and use), and write the heartbeat file atomically (write to a temp file, then `os.Rename`) so the path is never absent during an update.

### Proof of Concept
```bash
# 1. Simulate pinger stopping (stop updating /tmp/alive)
kill <pinger_pid>

# 2. Remove the stale heartbeat file (attacker owns it or it's absent)
rm -f /tmp/alive

# 3. Create a directory in its place
mkdir /tmp/alive

# 4. Keep directory mtime fresh in background
while true; do
    touch /tmp/alive/.hb && rm /tmp/alive/.hb
    sleep 60
done &

# 5. Liveness check now always passes despite pinger being dead
./healthcheck live   # exits 0
echo $?              # prints 0
``` [1](#0-0)
