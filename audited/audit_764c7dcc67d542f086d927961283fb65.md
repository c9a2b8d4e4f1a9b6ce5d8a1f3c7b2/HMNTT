### Title
Unauthenticated Mirror Node Response Injection via Empty JSON Causes Non-Retried Pinger DoS

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` makes a plain HTTP GET to the configured mirror node URL with no TLS enforcement, no response authentication, and no validation that the decoded `nodes` array is non-empty. A network-adjacent attacker who can intercept or spoof the HTTP response can return `{}` (syntactically valid JSON), causing `json.Decode` to succeed with a zero-value `nodesEnvelope` (nil `Nodes`), an empty `network` map, and the "no usable service_endpoints found" error — returned with `retry=false`, bypassing all configured retries and permanently disabling the pinger until restart.

### Finding Description
**Exact code path:**

`pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 95–129.

**Root cause — three compounding failures:**

1. **No transport security.** `buildNetworkFromMirrorNodes` constructs a bare `&http.Client{Timeout: cfg.mirrorNodeClientTimeout}` (line 46) with no TLS configuration, no certificate pinning, and no scheme enforcement. The documented default URL is `http://mirror-rest:5551` (plain HTTP, `config.go` line 37), making the request trivially interceptable.

2. **`{}` passes all decode checks.** At line 96, `json.NewDecoder(resp.Body).Decode(&payload)` returns `nil` error for `{}` because it is syntactically valid JSON. The `nodesEnvelope.Nodes` field remains its Go zero value: `nil`. The `for _, n := range payload.Nodes` loop at line 102 is a no-op over a nil slice, so `network` stays empty.

3. **`retry=false` on the empty-nodes path kills all retries.** Line 128 returns `nil, false, fmt.Errorf("no usable service_endpoints found...")`. Back in `buildNetworkFromMirrorNodes`, line 59 checks `if !retry || attempt == attempts` — because `retry` is `false`, the loop breaks immediately regardless of `mirrorNodeClientMaxRetries` (default 10). A single poisoned response causes permanent failure.

**Exploit flow:**
- Attacker intercepts the HTTP GET to `http://mirror-rest:5551/api/v1/network/nodes`
- Returns HTTP 200 with body `{}`
- Line 90–93 status check passes (200 is 2xx)
- Line 96 decode succeeds, `payload.Nodes == nil`
- Line 102 loop does not execute, `network` is empty
- Line 127–129: returns `(nil, false, error)`
- Line 59 in caller: `!retry` is true → `break` → all retries skipped
- `buildNetworkFromMirrorNodes` returns error; pinger cannot initialize

**Why existing checks are insufficient:**
- HTTP status check (lines 90–93): attacker returns 200, bypassed
- JSON decode error check (lines 96–98): `{}` is valid JSON, no error raised
- No minimum-nodes validation before returning the network map
- No HTTPS enforcement, no certificate validation, no response signing

### Impact Explanation
The pinger loses its entire consensus-node network map and cannot submit any transactions. Because the error is returned with `retry=false`, the retry loop in `buildNetworkFromMirrorNodes` exits after the very first poisoned response, making recovery impossible without a process restart. This constitutes a complete, persistent denial-of-service against the pinger's transaction-monitoring capability, which is the component responsible for detecting network freezes and block-time anomalies.

### Likelihood Explanation
The attack requires network-level interception of an HTTP (not HTTPS) connection. In Kubernetes or container-based deployments — the typical deployment target for this component — ARP spoofing, DNS poisoning, or a compromised sidecar/service-mesh component are all realistic vectors for a network-adjacent attacker with no privileges on the pinger host itself. The plain-HTTP default URL (`http://mirror-rest:5551`) is explicitly documented in `config.go` line 37, meaning many deployments will be vulnerable without operator intervention. The attack is repeatable: after a restart, a single intercepted request re-triggers the failure.

### Recommendation
1. **Enforce HTTPS.** Reject any `cfg.mirrorRest` URL whose scheme is not `https` at config-load time (`config.go`), or configure the `http.Client` with a strict TLS policy.
2. **Validate non-empty nodes after decode.** After line 98, add an explicit check: if `len(payload.Nodes) == 0`, return `nil, true, fmt.Errorf(...)` — note `retry=true` so transient empty responses are retried.
3. **Return `retry=true` for the empty-nodes case.** The current `retry=false` at line 128 means a single bad response is fatal. Transient or injected empty responses should be retried.
4. **Consider response authentication.** Use a pre-shared token or mTLS between the pinger and the mirror node REST endpoint to prevent unauthenticated response injection.

### Proof of Concept
```
# 1. Start a fake mirror node on the same network segment (or via DNS spoof):
python3 -c "
import http.server, json
class H(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        body = b'{}'
        self.send_response(200)
        self.send_header('Content-Type','application/json')
        self.send_header('Content-Length', str(len(body)))
        self.end_headers()
        self.wfile.write(body)
http.server.HTTPServer(('0.0.0.0', 5551), H).serve_forever()
"

# 2. Point the pinger at the fake server (or intercept DNS for mirror-rest):
export HIERO_MIRROR_PINGER_NETWORK=other
export HIERO_MIRROR_PINGER_REST=http://<attacker-ip>:5551

# 3. Run the pinger. Observe immediate fatal error with no retries:
# "attempt 1/1: no usable service_endpoints found from http://<attacker-ip>:5551/api/v1/network/nodes"
# Process exits; no transactions are submitted; network monitoring is disabled.
```
