### Title
Contract Filter Bypass via Unregistered Recipient Addresses Causes Unrestricted Trace Log Flooding

### Summary
In `MirrorOperationActionTracer.tracePostExecution()`, the call to `contractFilterCheck()` is guarded by `recipientNum.isPresent()`, meaning the filter is never evaluated when a recipient address is absent from the entity store. Any unprivileged user can submit a contract call that triggers internal calls to addresses not indexed in the entity store, causing every opcode across all affected call frames to be unconditionally logged — completely bypassing the operator-configured contract filter.

### Finding Description
**Exact code path:**

`MirrorOperationActionTracer.java` lines 59–68:
```java
final var recipientAddress = frame.getRecipientAddress();
final var recipientNum = recipientAddress != null
        ? commonEntityAccessor.get(recipientAddress, Optional.empty())
        : Optional.empty();

if (recipientNum.isPresent()
        && traceProperties.contractFilterCheck(
                CommonUtils.hex(toEvmAddress(((Entity) recipientNum.get()).getId())))) {
    return;  // only reached when entity IS found AND is filtered out
}

log.info(...);  // reached unconditionally when entity is NOT found
```

`TraceProperties.java` lines 33–35:
```java
public boolean contractFilterCheck(String contract) {
    return !getContract().isEmpty() && !getContract().contains(contract);
}
```

**Root cause:** `contractFilterCheck()` is only invoked when `recipientNum.isPresent()`. When `commonEntityAccessor.get(recipientAddress, Optional.empty())` returns `Optional.empty()` (address not in entity store), the short-circuit `&&` prevents `contractFilterCheck()` from ever being called. Execution falls directly to `log.info(...)`.

**Failed assumption:** The code assumes that any address worth filtering will always be resolvable in the entity store. Addresses that are not indexed (e.g., freshly deployed contracts not yet ingested, precompile-adjacent addresses, or deliberately crafted addresses) are invisible to the filter.

**Exploit flow:**
1. Operator enables tracing with a non-empty contract filter: `hiero.mirror.web3.evm.trace.contract = [0xAAA...]`
2. Attacker deploys or identifies contracts whose addresses are absent from the entity store
3. Attacker submits an `eth_call` or `eth_estimateGas` to a contract that internally calls those unregistered addresses (e.g., via `CALL`, `DELEGATECALL`, `STATICCALL`)
4. For each such internal call frame, `recipientNum.isEmpty()` → `contractFilterCheck()` is skipped → `log.info(...)` fires for every opcode in that frame
5. Attacker repeats at will, generating unbounded log volume

### Impact Explanation
When tracing is enabled with a contract filter, the filter's entire purpose is to restrict log output to specific contracts. This bypass causes the mirror node to emit a `log.info` entry for **every single EVM opcode** executed in every unregistered call frame. A single deeply nested contract call with hundreds of opcodes per frame, repeated across many requests, can rapidly exhaust disk space or saturate the logging pipeline. This is a griefing attack: no funds are at risk, but node availability and operator infrastructure are degraded. Severity is Medium, consistent with the declared scope.

### Likelihood Explanation
The precondition — tracing enabled with a non-empty contract filter — is an explicit operator opt-in, which limits the exposed population. However, once that configuration is active, **any** unprivileged user making a standard `eth_call` can trigger the bypass with zero special permissions. The attack is trivially repeatable: the attacker needs only to call a contract that internally dispatches to addresses not in the entity store, which is easy to arrange with a purpose-built contract. No authentication, no tokens, no privileged role required.

### Recommendation
Remove the `recipientNum.isPresent()` guard from the filter invocation. When the contract filter is non-empty and the recipient cannot be resolved in the entity store, the frame should be **filtered out** (not logged), because it cannot possibly match any configured contract. Replace lines 64–68 with:

```java
if (traceProperties.contractFilterCheck(
        recipientNum.isPresent()
                ? CommonUtils.hex(toEvmAddress(((Entity) recipientNum.get()).getId()))
                : (recipientAddress != null ? recipientAddress.toHexString() : ""))) {
    return;
}
```

Or more simply: if the contract filter is non-empty and the entity is not found, always return early:

```java
if (!traceProperties.getContract().isEmpty()) {
    if (recipientNum.isEmpty()) {
        return; // unknown address cannot match any configured contract filter
    }
    if (traceProperties.contractFilterCheck(
            CommonUtils.hex(toEvmAddress(((Entity) recipientNum.get()).getId())))) {
        return;
    }
}
```

### Proof of Concept
1. Configure the mirror node with:
   ```
   hiero.mirror.web3.evm.trace.enabled=true
   hiero.mirror.web3.evm.trace.contract=0000000000000000000000000000000000000001
   ```
   (a specific contract address that is NOT the target of the attack)

2. Deploy a contract `Dispatcher` that calls several addresses not present in the entity store:
   ```solidity
   contract Dispatcher {
       function attack() external {
           address(0xDEAD1).call("");
           address(0xDEAD2).call("");
           address(0xDEAD3).call("");
           // ... repeat N times
       }
   }
   ```

3. Submit `eth_call` to `Dispatcher.attack()`.

4. Observe: `log.info` fires for every opcode in every call frame targeting `0xDEAD1`, `0xDEAD2`, `0xDEAD3`, etc., even though none of them are in the configured contract filter. The filter is completely bypassed for all unregistered recipients.

5. Repeat the call in a loop to flood the log indefinitely with no rate limiting or authentication required.
