### Title
Gas Throttle Amplification via `estimate=true` Binary Search — Up to N×EVM Executions Per Single Rate-Limit Token

### Summary
The `ContractController.call()` method calls `throttleManager.throttle(request)` exactly once per HTTP request, consuming one `rateLimitBucket` token and `scaleGas(request.getGas())` `gasLimitBucket` tokens. When `estimate=true`, `ContractExecutionService.estimateGas()` then invokes `BinaryGasEstimator.search()`, which executes `doProcessCall()` up to `maxGasEstimateRetriesCount` times (≈20) in a binary search loop — none of which consume additional throttle tokens. An unprivileged attacker can therefore trigger up to 20× the EVM computation per rate-limit token consumed.

### Finding Description
**Code path:**

- `ContractController.call()` — [1](#0-0)  calls `throttleManager.throttle(request)` once at line 40, consuming 1 `rateLimitBucket` token and `scaleGas(request.getGas())` `gasLimitBucket` tokens.
- `ThrottleManagerImpl.throttle()` — [2](#0-1)  deducts tokens only at request entry; no further deductions occur during processing.
- `ContractExecutionService.estimateGas()` — [3](#0-2)  first calls `callContract()` once, then passes `gas -> doProcessCall(params, gas, true)` to `binaryGasEstimator.search()`.
- `BinaryGasEstimator.search()` — [4](#0-3)  loops `while (lo + 1 < hi && iterationsMade < properties.getMaxGasEstimateRetriesCount())`, calling `safeCall(mid, call)` (i.e., `doProcessCall`) on every iteration with no throttle interaction.

**Root cause:** The throttle gate is a one-time check at the HTTP boundary. The `gasLimitBucket` is charged for `request.getGas()` once, but the binary search can execute up to `maxGasEstimateRetriesCount` full EVM simulations, each consuming CPU proportional to the same gas value. The failed assumption is that one HTTP request equals one EVM execution for throttle accounting purposes.

### Impact Explanation
With `maxGasEstimateRetriesCount ≈ 20` and `gas = 15,000,000` (the default max), one HTTP request triggers up to 21 EVM executions (1 initial + 20 binary search) while consuming only 1 `rateLimitBucket` token and 1,500 `gasLimitBucket` tokens (15,000,000 / 10,000). The actual CPU cost is up to 21× what the throttle accounts for. At the default 500 req/s rate limit, an attacker can sustain ~10,500 EVM executions/second against the server's effective throttle of 500. This enables server-side resource exhaustion (CPU saturation, thread pool starvation) with a modest request rate.

### Likelihood Explanation
No authentication or special privileges are required. Any external caller can set `estimate: true` and `gas: 15000000` in a standard JSON POST to `/api/v1/contracts/call`. The attack is trivially repeatable, requires no on-chain state, and is effective from a single HTTP client. The amplification factor is deterministic and bounded only by `maxGasEstimateRetriesCount`.

### Recommendation
1. **Charge the gas bucket per binary search iteration**: Pass a callback into `BinaryGasEstimator.search()` that calls `throttleManager` (or directly deducts from `gasLimitBucket`) for each `mid` gas value used in the loop.
2. **Alternatively, multiply the initial gas deduction**: Before entering `estimateGas()`, deduct `scaleGas(request.getGas()) * maxGasEstimateRetriesCount` tokens from `gasLimitBucket` upfront, refunding the unused portion afterward (similar to the existing `restore()` pattern used for `InvalidParametersException`).
3. **Apply a separate estimate-specific rate limit**: Add a dedicated bucket for `estimate=true` requests with a lower per-second cap than the general `rateLimitBucket`.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
