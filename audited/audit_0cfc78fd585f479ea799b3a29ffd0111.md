### Title
Unauthenticated Circuit Breaker Exhaustion via Permanent HTTP 500 on `/mempool/transaction`

### Summary
`MempoolTransaction()` unconditionally returns `errors.ErrNotImplemented`, which the `coinbase/rosetta-sdk-go` server framework serializes as HTTP 500 per the Rosetta API specification. The Traefik circuit breaker middleware is applied globally across all Rosetta routes with a single `ResponseCodeRatio(500, 600, 0, 600) > 0.25` threshold. An unprivileged attacker who sends more than 25% of the total allowed request volume to this endpoint will trip the circuit breaker and block all legitimate Rosetta API traffic.

### Finding Description
**Exact code path:**

`rosetta/app/services/mempool_service.go`, lines 30–35:
```go
func (m *mempoolAPIService) MempoolTransaction(
    _ context.Context,
    _ *types.MempoolTransactionRequest,
) (*types.MempoolTransactionResponse, *types.Error) {
    return nil, errors.ErrNotImplemented
}
``` [1](#0-0) 

`ErrNotImplemented` is a `*types.Error` (Rosetta error code 111, `retriable: false`): [2](#0-1) 

The `coinbase/rosetta-sdk-go` server framework returns **HTTP 500** for every non-nil `*types.Error` return value, per the Rosetta API specification. This is confirmed by the Prometheus alerting rule that tracks `status_code=~"^5.."` for Rosetta errors: [3](#0-2) 

The Traefik middleware chain for the Rosetta ingress is: [4](#0-3) 

The circuit breaker is the **outermost** middleware and is applied to **all** Rosetta routes (`/rosetta/account`, `/rosetta/block`, `/rosetta/mempool`, `/rosetta/network`, etc.): [5](#0-4) 

The rate limit is `average: 10` per `requestHost` — meaning all clients sharing the same Host header share a single 10 req/s bucket. An attacker sending 3 req/s to `/rosetta/mempool/transaction` (all returning HTTP 500) while legitimate traffic accounts for 7 req/s (returning HTTP 200) produces a `ResponseCodeRatio` of 3/10 = **30% > 25%**, tripping the circuit breaker.

**Root cause:** The endpoint is permanently broken (always HTTP 500), the circuit breaker threshold is global across all routes, and the rate limit is per-host rather than per-source-IP, so an attacker can consume >25% of the shared capacity with guaranteed-500 requests.

### Impact Explanation
When the circuit breaker trips, Traefik returns HTTP 503 for **all** Rosetta API routes — including `/network/status`, `/block`, `/account/balance`, and `/construction/*`. This is a complete denial of service for all Rosetta API consumers. The circuit breaker remains open for a recovery period (Traefik default: 10 seconds), during which no legitimate requests are served. An attacker sustaining the flood keeps the breaker permanently open.

### Likelihood Explanation
No authentication, API key, or privileged access is required. The endpoint is publicly reachable at `POST /rosetta/mempool/transaction`. The attack requires only ~3 HTTP POST requests per second — well within reach of any client with a basic HTTP library. The `inFlightReq` limit of 5 per IP does not prevent this because the requests complete immediately (no backend I/O). The per-host rate limit of 10 req/s is shared with all legitimate users, making it trivially easy to consume >25% of the budget. The attack is repeatable and requires no special knowledge of the system.

### Recommendation
1. **Remove the endpoint from the public ingress** or return HTTP 404/405 instead of HTTP 500 for unimplemented endpoints, so the response does not count toward the `ResponseCodeRatio(500, 600, 0, 600)` threshold.
2. **Scope the circuit breaker per route** rather than globally, so errors on `/mempool/*` do not affect `/network/*`, `/block/*`, etc.
3. **Add per-source-IP rate limiting** (change `sourceCriterion` on `rateLimit` from `requestHost` to `ipStrategy`) to prevent a single attacker from consuming a disproportionate share of the request budget.
4. **Exclude known-unimplemented endpoints** from circuit breaker accounting, or return a non-5xx status (e.g., HTTP 501) for them — though this requires changes to the Rosetta SDK error handling.

### Proof of Concept
```bash
# Precondition: Rosetta API is publicly accessible at https://rosetta.example.com
# Step 1: Confirm the endpoint always returns HTTP 500
curl -s -o /dev/null -w "%{http_code}" \
  -X POST https://rosetta.example.com/rosetta/mempool/transaction \
  -H "Content-Type: application/json" \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"transaction_identifier":{"hash":"0x1234"}}'
# Expected: 500

# Step 2: Flood at ~3 req/s (sustained) to trip the circuit breaker
# (while legitimate traffic is ~7 req/s)
while true; do
  curl -s -o /dev/null \
    -X POST https://rosetta.example.com/rosetta/mempool/transaction \
    -H "Content-Type: application/json" \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"transaction_identifier":{"hash":"0x1234"}}' &
  sleep 0.33
done

# Step 3: Verify circuit breaker has tripped — legitimate endpoints now return 503
curl -s -o /dev/null -w "%{http_code}" \
  -X POST https://rosetta.example.com/rosetta/network/list \
  -H "Content-Type: application/json" \
  -d '{"metadata":{}}'
# Expected: 503 (circuit breaker open)
```
