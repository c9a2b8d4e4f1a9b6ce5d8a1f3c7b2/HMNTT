### Title
Missing gRPC Port Range Validation in `fetchMirrorNodeNetwork()` Allows Silent Node Removal via Malicious Mirror REST Response

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` accepts any `serviceEndpoint.Port` value from 1–65535 without validating whether it falls within a usable gRPC range. The only guard is `ep.Port == 0`, which skips that single value. Because the HTTP client is constructed with no TLS enforcement and the documented default URL is plaintext `http://`, a network-level attacker who can intercept or spoof the mirror REST response can inject arbitrary port numbers for ≥30% of nodes. The function returns success as long as at least one node survives, so the poisoned entries are silently added to the SDK network map and all subsequent gRPC dials to those nodes fail.

### Finding Description
**Exact code path:**

`pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 113–124:

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {   // ← only guard: skips port 0
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId     // ← any port 1-65535 accepted
}
```

**Root cause:** The failed assumption is that the mirror REST API will only return ports on which gRPC is actually listening. The code trusts the response entirely; it performs no range check (e.g., `ep.Port < 1 || ep.Port > 65535` is impossible for an `int`, but no check for well-known gRPC ports like 50211/50212 or even a basic `1–1023` exclusion exists).

**Transport exposure:** `config.go` line 37 shows the documented default is `http://mirror-rest:5551` (plaintext HTTP):

```go
flag.StringVar(&cfg.mirrorRest, "mirror-rest", ..., "... e.g. http://mirror-rest:5551")
```

The HTTP client at line 46 is:

```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
```

No TLS is enforced, no certificate pinning, no scheme validation. A network-level attacker on the same segment (e.g., another pod in the same Kubernetes namespace, or an ARP/DNS poisoning attacker on a flat network) can intercept the plaintext HTTP response and substitute a crafted `nodesEnvelope`.

**Why the only existing check is insufficient:**

The guard `ep.Port == 0` is the sole filter. Ports 1, 65535, or any other non-gRPC value pass through. The function's only aggregate check is:

```go
if len(network) == 0 {
    return nil, false, fmt.Errorf("no usable service_endpoints found from %s", url)
}
```

As long as one node has a valid endpoint, the function returns `nil` error even if every other node has a poisoned port. The caller in `sdk_client.go` line 22 passes the map directly to `hiero.ClientForNetwork(netmap)` with no further validation.

### Impact Explanation
The SDK network map is populated once at startup. Nodes whose addresses resolve to ports with no gRPC listener will fail every dial attempt. If ≥30% of nodes are poisoned this way, the effective consensus network shrinks below the threshold required for reliable transaction routing. The pinger will silently route all traffic through the remaining (attacker-chosen) nodes, or fail entirely if all nodes are poisoned. No log line, no error, no alert is produced by `fetchMirrorNodeNetwork()` for the poisoned entries — they are indistinguishable from legitimate entries until the SDK's own connection layer times out.

### Likelihood Explanation
The attack requires the ability to intercept or forge the HTTP response from the mirror REST endpoint. In a Kubernetes deployment using the documented `http://` scheme, any workload in the same network namespace (or a compromised sidecar/init container) can perform this interception without cluster-admin privileges. DNS spoofing of the service name `mirror-rest` is also feasible for a pod with `NET_ADMIN` capability or in a misconfigured CNI. The attack is repeatable: the network map is built once at startup, so a single poisoned response persists for the lifetime of the pinger process.

### Recommendation
1. **Validate port range** in `fetchMirrorNodeNetwork()` before adding an endpoint to the map:
   ```go
   if ep.Port <= 0 || ep.Port > 65535 {
       continue
   }
   // Optionally: warn if port is in the privileged range (< 1024) or not a known gRPC port
   ```
2. **Enforce HTTPS** for the mirror REST URL: reject `cfg.mirrorRest` values that do not begin with `https://`, or configure the `http.Client` with a strict TLS config.
3. **Log and count** skipped endpoints so operators can detect anomalous responses.
4. **Consider a minimum-node threshold**: if fewer than N nodes survive validation, treat the response as invalid and retry rather than proceeding with a degraded map.

### Proof of Concept
**Preconditions:** Attacker can intercept HTTP traffic between the pinger pod and `mirror-rest:5551` (e.g., via ARP poisoning, DNS spoofing, or a rogue pod on the same flat network).

**Steps:**
1. Intercept the GET request to `http://mirror-rest:5551/api/v1/network/nodes`.
2. Return a crafted JSON response where ≥30% of nodes have `service_endpoints` with `port: 1` (or `port: 65535`), and one node retains a valid port (e.g., 50211) so `len(network) > 0`:
   ```json
   {
     "nodes": [
       {"node_account_id": "0.0.3", "service_endpoints": [{"ip_address_v4": "1.2.3.4", "port": 1}]},
       {"node_account_id": "0.0.4", "service_endpoints": [{"ip_address_v4": "1.2.3.5", "port": 65535}]},
       {"node_account_id": "0.0.5", "service_endpoints": [{"ip_address_v4": "1.2.3.6", "port": 50211}]}
     ],
     "links": {}
   }
   ```
3. `fetchMirrorNodeNetwork()` returns `nil` error and a map containing all three entries.
4. `hiero.ClientForNetwork(netmap)` is called with the poisoned map.
5. All gRPC dials to `1.2.3.4:1` and `1.2.3.5:65535` fail; 66% of nodes are effectively removed from routing with no error surfaced to the operator. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
