### Title
Unprivileged `?internal=true` Removes `transaction_nonce = 0` Filter, Enabling Amplified DB Result Sets on `/contracts/:contractId/results`

### Summary
Any unauthenticated caller can append `?internal=true` to `GET /contracts/:contractId/results` to suppress the `cr.transaction_nonce = 0` SQL predicate that normally excludes internal (child) transactions. Without that predicate, the database query returns all contract results regardless of nonce, potentially orders of magnitude more rows than the default path. No privilege, API key, or special role is required.

### Finding Description
**Route registration** — `rest/routes/contractRoute.js` line 14 wires the endpoint directly to `ContractController.getContractResultsById` with no middleware guard:

```js
router.getExt('/:contractId/results', ContractController.getContractResultsById);
``` [1](#0-0) 

**Handler** — `getContractResultsById` in `rest/controllers/contractController.js` (lines 856–892) parses query parameters with `acceptedContractResultsParameters` and passes them straight into `extractContractResultsByIdQuery`:

```js
const {conditions, params, order, limit, skip} =
  await this.extractContractResultsByIdQuery(filters, contractId);
``` [2](#0-1) 

**Filter extraction** — Inside `extractContractResultsByIdQuery` (lines 415–542), the `internal` variable defaults to `false`. When the parsed filter list contains `filterKeys.INTERNAL`, its value is assigned directly:

```js
case filterKeys.INTERNAL:
  internal = filter.value;   // set to true if caller sends ?internal=true
  break;
``` [3](#0-2) 

**The guard** — The only protection against returning internal transactions is the conditional at lines 531–533:

```js
if (!internal) {
  conditions.push(`${ContractResult.getFullName(ContractResult.TRANSACTION_NONCE)} = 0`);
}
``` [4](#0-3) 

When `internal=true` is supplied by the caller, this block is skipped entirely. The resulting SQL query sent to the database has **no nonce predicate**, so it scans and returns every `contract_result` row for the given contract ID up to `limit`. Both `getContractResultsById` and `getContractResults` share the same `acceptedContractResultsParameters` and the same `extractContractResultsByIdQuery` path, confirming `internal` is an accepted, unguarded parameter on the per-contract endpoint. [5](#0-4) 

**No authentication check exists** anywhere in this call chain.

### Impact Explanation
A busy contract (e.g., a popular DeFi or token contract) can accumulate millions of internal call results with `transaction_nonce > 0`. Normally these are hidden by the `= 0` filter. With `?internal=true`, a single request forces the DB to scan and return up to `limit` rows from that much larger set, and the server must allocate memory to serialize each `ContractResultViewModel`. Repeated concurrent requests targeting high-activity contracts amplify DB I/O, CPU, and heap pressure, degrading service for all users — a classic griefing / resource-exhaustion scenario with no economic cost to the attacker.

### Likelihood Explanation
The exploit requires zero privileges, zero tokens, and zero knowledge beyond the public API documentation (the `internal` query parameter is documented in `openapi.yml`). It is trivially scriptable with `curl` or any HTTP client. Any actor who can reach the mirror-node REST API — including anonymous internet users if the node is public — can trigger it repeatedly. [6](#0-5) 

### Recommendation
1. **Remove `internal` from `acceptedContractResultsParameters` for the `/:contractId/results` endpoint**, or maintain a separate accepted-parameter list for that route that excludes `filterKeys.INTERNAL`.
2. If `internal` must remain available on that endpoint, gate it behind an authenticated/privileged role check before `extractContractResultsByIdQuery` is called.
3. As a defense-in-depth measure, enforce a strict per-IP rate limit on this endpoint at the reverse-proxy or middleware layer.

### Proof of Concept
```
# Default request — only nonce=0 rows returned (fast, small result)
GET /api/v1/contracts/0.0.1234/results

# Exploit — nonce filter removed, all internal results returned
GET /api/v1/contracts/0.0.1234/results?internal=true

# Amplified attack — concurrent flood against a high-activity contract
for i in $(seq 1 200); do
  curl -s "https://<mirror-node>/api/v1/contracts/0.0.1234/results?internal=true&limit=100" &
done
wait
```
No authentication header, API key, or special role is needed. The server will issue a full-table-scan-equivalent query for each request, consuming DB I/O, CPU, and heap proportional to the number of internal results stored for that contract.
