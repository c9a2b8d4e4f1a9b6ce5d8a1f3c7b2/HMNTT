### Title
Unauthenticated CPU Exhaustion via Crafted Alias in `AccountBalance()` — Expensive Cryptographic Parsing Without Application-Level Rate Limiting

### Summary
An unprivileged external user can send `POST /account/balance` requests with a hex-encoded address that decodes to a valid protobuf `Key_ECDSASecp256K1` message containing a well-formed 33-byte compressed-point prefix but an invalid secp256k1 curve point. Each such request causes `hiero.PublicKeyFromBytes()` to perform a full 256-bit modular exponentiation (point decompression) before returning an error. The Rosetta service has no application-level rate limiting, the Traefik middleware is optional and its `rateLimit` is trivially bypassed via the `requestHost` criterion, and the configured `retry: attempts: 3` middleware amplifies each attacker request into three server-side crypto operations.

### Finding Description

**Exact code path:**

`AccountBalance()` at [1](#0-0)  calls `types.NewAccountIdFromString()`.

`NewAccountIdFromString()` at [2](#0-1)  accepts any `0x`-prefixed string, hex-decodes it, and passes the raw bytes directly to `NewAccountIdFromAlias()`.

`NewAccountIdFromAlias()` at [3](#0-2)  calls `NewPublicKeyFromAlias(alias)` with no length or format pre-check beyond the empty-byte guard.

`NewPublicKeyFromAlias()` at [4](#0-3)  performs:
1. `proto.Unmarshal(alias, &key)` — cheap; attacker controls this to succeed. [5](#0-4) 
2. A key-type switch that extracts `rawKey` (33 bytes for ECDSA). [6](#0-5) 
3. `hiero.PublicKeyFromBytes(rawKey)` — for secp256k1 this decompresses the point: computes `y² = x³ + 7 mod p` then `y = y²^((p+1)/4) mod p`, a full 256-bit modular exponentiation, before returning an error if the point is not on the curve. [7](#0-6) 

**Root cause:** Attacker-controlled bytes reach an expensive elliptic-curve operation with no prior structural validation that would cheaply reject invalid inputs, and with no application-level concurrency or rate control.

**Retry amplification:** The Traefik middleware chain includes `retry: attempts: 3`. Rosetta errors are returned as HTTP 500, which Traefik retries, so each attacker request triggers **three** server-side modular exponentiations. [8](#0-7) 

### Impact Explanation

Each malicious request consumes CPU proportional to a 256-bit modular exponentiation (secp256k1 point decompression), multiplied by 3 due to the retry middleware. Without application-level rate limiting, a single attacker machine can saturate CPU threads on the mirror-node Rosetta process. At sufficient concurrency this degrades or halts the `/account/balance` endpoint, which is a core Rosetta API path, meeting the ≥30% processing-node degradation threshold for nodes running without Traefik or with the bypassable `requestHost`-scoped rate limit.

### Likelihood Explanation

- **No authentication required** — any internet client can call `/account/balance`.
- **Payload is trivial to craft** — a valid protobuf `Key_ECDSASecp256K1` field with a 33-byte value whose first byte is `0x02` and whose remaining 32 bytes are arbitrary (most random values are not valid secp256k1 x-coordinates).
- **Traefik middleware is optional** — gated on `{{ if and .Values.global.middleware .Values.middleware }}`; deployments without Traefik have zero rate limiting. [9](#0-8) 
- **`rateLimit` is bypassable** — scoped to `requestHost` (the HTTP `Host` header), not the client IP; an attacker rotates the `Host` header to bypass it. [10](#0-9) 
- **`inFlightReq` per-IP limit of 5** provides partial mitigation only when Traefik is deployed and the attacker uses a single IP. [11](#0-10) 
- Attack is fully repeatable and automatable with a simple HTTP client loop.

### Recommendation

1. **Add a cheap structural pre-check in `NewPublicKeyFromAlias()`** before calling `hiero.PublicKeyFromBytes`: validate that the alias length matches exactly the expected protobuf-encoded sizes for Ed25519 (34 bytes: tag `0x0a` + len `0x20` + 32 bytes) or ECDSA (36 bytes: tag `0x22` + len `0x21` + 33 bytes). Reject anything else immediately without protobuf parsing or crypto.
2. **Add application-level rate limiting** inside the Rosetta Go service (e.g., `golang.org/x/time/rate` per-IP token bucket) so protection is not solely dependent on optional infrastructure middleware.
3. **Fix the `rateLimit` criterion** from `requestHost` to `ipStrategy` (same as `inFlightReq`) so it cannot be bypassed by rotating the `Host` header.
4. **Scope the `retry` middleware** to exclude 5xx responses caused by invalid input (use a `retryExpression` that only retries on network-level errors, not application errors).

### Proof of Concept

```python
import socket, struct, threading, requests

# Craft protobuf Key { ecdsaSecp256k1: <0x02 + 32 bytes of 0xff> }
# Field 3 (ECDSASecp256K1), wire type 2 (length-delimited)
raw_key = b'\x02' + b'\xff' * 32          # valid prefix, invalid curve point
proto_key = b'\x22' + bytes([len(raw_key)]) + raw_key  # field 3, LEN, 33 bytes
hex_alias = '0x' + proto_key.hex()

payload = {
    "network_identifier": {"blockchain": "Hedera", "network": "mainnet"},
    "account_identifier": {"address": hex_alias}
}

def attack():
    while True:
        try:
            requests.post("http://<target>:5700/account/balance",
                          json=payload, timeout=5,
                          headers={"Host": f"bypass-{threading.get_ident()}.example.com"})
        except Exception:
            pass

threads = [threading.Thread(target=attack) for _ in range(200)]
for t in threads: t.start()
for t in threads: t.join()
```

Each request reaches `hiero.PublicKeyFromBytes` with a 33-byte value whose x-coordinate (`0xff…ff`) is not a valid secp256k1 point, triggering full modular exponentiation before returning an error. The `Host` header rotation bypasses the `requestHost`-scoped Traefik rate limit. The `retry: attempts: 3` middleware triples the server-side CPU work per attacker request.

func NewPublicKeyFromAlias(alias []byte) (zeroCurveType types.CurveType, zeroPublicKey PublicKey, _ error) {
	if len(alias) == 0 {
		return zeroCurveType, zeroPublicKey, errors.Errorf("Empty alias provided")
	}

	var key services.Key
	if err := proto.Unmarshal(alias, &key); err != nil {
		return zeroCurveType, zeroPublicKey, err
	}

	var curveType types.CurveType
	var rawKey []byte
	switch value := key.GetKey().(type) {
	case *services.Key_Ed25519:
		curveType = types.Edwards25519
		rawKey = value.Ed25519
	case *services.Key_ECDSASecp256K1:
		curveType = types.Secp256k1
		rawKey = value.ECDSASecp256K1
	default:
		return zeroCurveType, zeroPublicKey, errors.Errorf("Unsupported key type")
	}

	publicKey, err := hiero.PublicKeyFromBytes(rawKey)
	if err != nil {
		return zeroCurveType, zeroPublicKey, err
	}

	return curveType, PublicKey{publicKey}, nil
