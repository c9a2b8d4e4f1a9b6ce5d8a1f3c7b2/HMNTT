### Title
Unauthenticated DB Connection Exhaustion via Unbounded Concurrent `POST /api/v1/network/fees?mode=STATE` Requests

### Summary
The `estimateFees()` endpoint in `NetworkController` accepts unauthenticated requests with no rate limiting. When `mode=STATE` is specified with a token-transfer transaction type, each request triggers two uncached live database queries through `FeeTokenStore`. An attacker sending many concurrent requests can exhaust the HikariCP connection pool, causing all other mirror node API calls that require a database connection to block or time out.

### Finding Description

**Code path:**

`NetworkController.estimateFees()` (lines 110‚Äì124) accepts any unauthenticated `POST /api/v1/network/fees` with no throttle guard:

```java
@PostMapping(consumes = {"application/protobuf", "application/x-protobuf"}, value = "/fees")
FeeEstimateResponse estimateFees(
        @RequestBody @NotNull byte[] body,
        @RequestParam(defaultValue = "INTRINSIC", required = false) FeeEstimateMode mode,
        ...) {
    final var transaction = Transaction.PROTOBUF.parse(Bytes.wrap(body));
    return toResponse(feeEstimationService.estimateFees(transaction, mode, highVolumeThrottle));
}
```

When `mode=STATE`, `FeeEstimationService.estimateFees()` (lines 101‚Äì102) wraps the context with a live `FeeEstimationFeeContext` backed by the real `feeTokenStore`:

```java
final var context = mode == FeeEstimateMode.STATE
        ? txContext.withFeeContext(newFeeContext(txContext.body(), throttleUtilization))
        : txContext;
```

`FeeEstimationFeeContext.readableStore()` (lines 199‚Äì201) returns the real `feeTokenStore` for `ReadableTokenStore.class`. `FeeTokenStore.get()` (lines 30‚Äì35) then executes **two uncached live DB queries per call**:

```java
public Token get(@NonNull final TokenID id) {
    return tokenRepository
            .findById(id.tokenNum())                          // DB query 1
            .map(token -> toToken(id, token, customFeeRepository))
            .orElse(null);
}
// inside toToken ‚Üí getCustomFees:
customFeeRepository.findById(tokenId)                         // DB query 2
```

The `rest-java` `TokenRepository` and `CustomFeeRepository` are plain `CrudRepository` interfaces with **no `@Cacheable` annotation** (unlike the `web3` module's `TokenRepository` which has `@Cacheable`). Every STATE-mode request with a token transfer body hits the database twice.

**No rate limiting exists for this endpoint.** The `ThrottleManager`/`ThrottleConfiguration` in the codebase lives entirely in the `web3` module and is wired only to contract-call endpoints. There is no equivalent throttle, filter, or interceptor applied to `POST /api/v1/network/fees` in the `rest-java` module.

The HikariCP pool is shared across all rest-java DB operations. Default pool size is small (Spring Boot default: 10 connections). With N concurrent STATE-mode requests each holding a connection for two sequential queries, the pool saturates at N/2 concurrent requests, causing all other DB-dependent API calls to queue behind the HikariCP `connectionTimeout` (configured at 10 s for rest-java per `hiero.mirror.restJava.db.statementTimeout`).

### Impact Explanation

Exhausting the HikariCP connection pool blocks every other endpoint in the rest-java service that requires a database connection (account lookups, transaction queries, topic queries, etc.). The mirror node REST API becomes unavailable to all clients for the duration of the attack. While this does not affect Hedera consensus directly, the mirror node is the primary read API used by wallets, explorers, and dApps to confirm transaction status; its unavailability constitutes a significant service disruption. The attack requires no credentials, no special transaction content (any valid token-transfer body suffices), and is trivially repeatable.

### Likelihood Explanation

Any unprivileged external user can craft a minimal valid protobuf `Transaction` containing a `CryptoTransfer` with a token transfer list and POST it to `POST /api/v1/network/fees?mode=STATE` in a tight loop. No authentication, no API key, no prior knowledge beyond the public OpenAPI spec is required. The attack is low-cost (small request body, fast round-trip) and can be sustained indefinitely from a single machine or amplified with multiple sources.

### Recommendation

1. **Add rate limiting to `POST /api/v1/network/fees`** ‚Äî apply a per-IP or global request-rate bucket (analogous to `ThrottleManagerImpl` in the web3 module) before the DB path is reached.
2. **Add a request-body size cap** ‚Äî enforce a maximum byte length on the protobuf body to prevent amplification via large payloads.
3. **Cache `FeeTokenStore` lookups** ‚Äî add a short-lived (e.g., 1‚Äì5 s) in-process cache on `TokenRepository.findById` and `CustomFeeRepository.findById` within the rest-java module, consistent with the `@Cacheable` pattern already used in the web3 module's `TokenRepository`.
4. **Isolate the fee-estimation DB pool** ‚Äî use a dedicated, size-limited HikariCP pool for fee-estimation queries so that pool exhaustion cannot cascade to other endpoints.

### Proof of Concept

```bash
# Build a minimal CryptoTransfer with a token transfer (any valid token num)
# Encode as protobuf binary ‚Üí fee_tx.bin

# Fire 50 concurrent STATE-mode requests
for i in $(seq 1 50); do
  curl -s -X POST \
    "https://<mirror-node>/api/v1/network/fees?mode=STATE" \
    -H "Content-Type: application/protobuf" \
    --data-binary @fee_tx.bin &
done
wait

# Immediately probe another endpoint that requires DB
time curl "https://<mirror-node>/api/v1/accounts/0.0.1234"
# Expected: request hangs for up to connectionTimeout (10 s) or returns 503
```

    public FeeResult estimateFees(
            @NonNull final Transaction transaction,
            @NonNull final FeeEstimateMode mode,
            final int throttleUtilization) {
        try {
            final var txContext = new TransactionFeeContext(transaction);
            final var context = mode == FeeEstimateMode.STATE
                    ? txContext.withFeeContext(newFeeContext(txContext.body(), throttleUtilization))
                    : txContext;
            final SimpleFeeCalculator calculator = Objects.requireNonNull(feeManager.getSimpleFeeCalculator());
            return calculator.calculateTxFee(context.body(), context);
        } catch (ParseException e) {
            throw new IllegalArgumentException("Unable to parse transaction", e);
        } catch (NullPointerException e) {
            throw new IllegalArgumentException("Unknown transaction type", e);
        } catch (IllegalStateException e) {
            if (e.getCause() instanceof UnknownHederaFunctionality) {
                throw new IllegalArgumentException("Unknown transaction type", e);
            }
            throw e;
        }
    }

    private static Token toToken(
            final TokenID id,
            final org.hiero.mirror.common.domain.token.Token token,
            final CustomFeeRepository customFeeRepository) {
        return Token.newBuilder()
                .tokenId(id)
                .tokenType(
                        token.getType() == TokenTypeEnum.NON_FUNGIBLE_UNIQUE
                                ? TokenType.NON_FUNGIBLE_UNIQUE
                                : TokenType.FUNGIBLE_COMMON)
                .customFees(getCustomFees(token.getTokenId(), customFeeRepository))
                .build();
    }

    // Calculator only checks isEmpty(); CustomFee.DEFAULT is a safe placeholder.
    private static List<CustomFee> getCustomFees(final long tokenId, final CustomFeeRepository customFeeRepository) {
        return customFeeRepository
                .findById(tokenId)
                .filter(customFee -> !customFee.isEmptyFee())
                .map(customFee -> {
                    final var count = size(customFee.getFixedFees())
                            + size(customFee.getFractionalFees())
                            + size(customFee.getRoyaltyFees());
                    return Collections.nCopies(count, CustomFee.DEFAULT);
                })
                .orElseGet(List::of);
    }
