### Title
Unbounded Metric Label Cardinality via User-Controlled Path in `toOpenApiPath()` for Unmatched Routes

### Summary
In `rest/middleware/metricsHandler.js`, `toOpenApiPath()` falls back to `req.path` (the raw, user-controlled URL path) when no Express route matches (`req.route` is null). This value is used directly as the `path` label in four per-request Prometheus instruments. An unprivileged attacker can send an unbounded number of requests to unique 404 paths, creating a new in-memory time series for each unique label combination, exhausting the Node.js process heap.

### Finding Description
**Code path:**

`toOpenApiPath()` at [1](#0-0)  resolves the path label in this priority order:

1. `res.locals[requestPathLabel]` — set only by `recordRequestPath` in `routes/index.js` [2](#0-1)  when `req.route` exists (i.e., a route matched).
2. `req.route?.path` — the Express route pattern (safe, app-controlled).
3. **`req.path`** — the raw URL path from the HTTP request (user-controlled), taken when `req.route` is null.

For any request that does not match a registered route (HTTP 404), `req.route` is `null`, `res.locals[requestPathLabel]` is never populated, and the fallback `path = req.path` is used. [3](#0-2) 

The regex `path.replace(/:([^/]+)/g, '{$1}')` is a fixed pattern applied to the user-supplied string — it does not constitute regex injection (user input is the source, not the pattern). The actual root cause is that the resulting string, regardless of its content, is stored as the `path` label value in `labels = {method, path, code}`. [4](#0-3) 

This label set is then recorded into four instruments on every request finish: [5](#0-4) 

Each unique `(method, path, code)` triple creates a new time series stored in the OpenTelemetry SDK's in-process metric registry. There is no cardinality cap, no path normalization for unmatched routes, and no allowlist check.

### Impact Explanation
An attacker can exhaust the heap of the mirror-node REST API process by sending a high volume of requests to unique paths (e.g., `/api/v1/x/1`, `/api/v1/x/2`, … `/api/v1/x/N`). Each unique path creates a new set of time series across all four instruments. As the registry grows, GC pressure increases, throughput degrades, and eventually the Node.js process OOMs and crashes. This renders the REST API unavailable (denial of service). Note: this affects only the mirror-node REST API process — it does not affect Hedera consensus or block production, which are separate services.

### Likelihood Explanation
No authentication or rate limiting is required. Any HTTP client can send GET requests to arbitrary paths. The attack is trivially scriptable, repeatable, and requires no knowledge of the application internals beyond the public HTTP endpoint. The only practical constraint is network bandwidth and the rate at which the target processes requests.

### Recommendation
1. **Normalize unmatched-route paths to a fixed sentinel value** (e.g., `"unknown"` or `apiPrefix + "/unknown"`) instead of using `req.path` when `req.route` is null. [6](#0-5) 
2. Alternatively, skip per-route metric recording entirely for requests that produced a 404 (only increment the aggregate `allClientErrorCounter`).
3. Add a hard cardinality cap in the metric registry configuration, or use a middleware-level path allowlist before recording labels.

### Proof of Concept
```bash
# Send N requests to unique paths, each creating a new time series
for i in $(seq 1 100000); do
  curl -s "http://<mirror-node-rest>:5551/api/v1/nonexistent/path/$i" &
done
wait
# Monitor heap growth:
# curl http://<mirror-node-rest>:5551/swagger/metrics/ | grep nodejs_process_memory_heap_used_bytes
```
Each unique `/api/v1/nonexistent/path/$i` path is stored as a distinct `path` label value across `api_request_total`, `api_request_duration_milliseconds`, `api_request_size_bytes`, and `api_response_size_bytes`. Heap usage grows linearly with the number of unique paths until OOM.

```javascript
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];

  if (!path) {
    if (!req.route) {
      path = req.path;
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }

  path = path.replace(/:([^/]+)/g, '{$1}');

  if (!path.startsWith(apiPrefix)) {
    return apiPrefix + '/' + path;
  }

  return path;
};
```

  const path = req.route?.path;
  if (path && !path.startsWith(apiPrefix) && !res.locals[requestPathLabel]) {
