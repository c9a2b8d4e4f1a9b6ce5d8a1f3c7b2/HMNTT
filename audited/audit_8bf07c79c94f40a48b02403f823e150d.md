### Title
Metric Label Cardinality Explosion via Unmatched Route Path Injection in `toOpenApiPath()`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to the raw user-supplied `req.path` when a request does not match any registered Express route (i.e., `req.route` is `undefined` and `res.locals[requestPathLabel]` is unset). This raw path is then used directly as the `path` label value recorded into `requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, and `responseSizeHistogram`. An unauthenticated attacker can send an unbounded number of requests to unique, non-existent paths, creating a new unique label combination per request and exhausting process memory.

### Finding Description
**Exact code path:**

In `rest/middleware/metricsHandler.js` lines 13–31, `toOpenApiPath()` resolves the path label as follows:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // set only for matched routes

  if (!path) {
    if (!req.route) {
      path = req.path;                        // ← raw user input for 404s
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }

  path = path.replace(/:([^/]+)/g, '{$1}'); // only normalises :param tokens
  ...
  return path;
};
```

`res.locals[requestPathLabel]` is populated by `recordRequestPath` in `rest/routes/index.js` (lines 16–21) **only when `req.route?.path` exists**, i.e., only for matched routes. For any request that does not match a registered route, both `res.locals[requestPathLabel]` and `req.route` are `undefined`, so the branch at line 21 executes: `path = req.path`.

The resulting path is then used as a label on all four per-route instruments at lines 207–210:

```js
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
```

**Root cause:** The failed assumption is that every request reaching the `res.on('finish')` handler will have a bounded, template-derived path. In reality, 404 requests carry the raw, attacker-controlled `req.path` as the label value.

**Why the regex does not help:** `/:([^/]+)/g` only replaces Express-style `:param` tokens. A path like `/api/v1/AAAA1234` contains no such token and passes through unchanged, producing a unique label value for every unique string the attacker supplies.

### Impact Explanation
The OpenTelemetry SDK (`@opentelemetry/sdk-metrics`) stores every distinct label-set as a separate time series in memory. Each unique `{method, path, code}` triple allocates a new entry in the histogram/counter internal maps. With no upper bound on the `path` dimension, an attacker can grow these maps without limit, exhausting the Node.js heap. Once memory is exhausted, the mirror-node REST process crashes or becomes unresponsive, denying service to all legitimate callers — including those submitting or querying transaction authorization requests.

### Likelihood Explanation
Exploitation requires zero privileges: any HTTP client can send GET requests to arbitrary paths. The attack is trivially automatable (a simple loop generating unique path strings suffices), repeatable, and requires no knowledge of the application internals. The only practical constraint is network access to the REST API port, which is by design publicly reachable.

### Recommendation
1. **Allowlist-based path normalization:** Before recording the label, check whether the resolved path matches a known OpenAPI path template. If it does not match, replace it with a sentinel value such as `"unknown"` or `"unmatched"`.
2. **Hard cardinality cap:** Maintain a bounded set of seen `path` label values; once the cap is reached, map all new values to `"other"`.
3. **Do not fall back to `req.path`:** Replace the `path = req.path` branch with a static fallback string (e.g., `path = 'unmatched'`) so that all 404 traffic is collapsed into a single label value.

### Proof of Concept
```bash
# Send 100 000 requests to unique paths — no authentication required
for i in $(seq 1 100000); do
  curl -s "http://<mirror-node-host>/api/v1/nonexistent_path_${i}" &
done
wait
```

Each request produces a distinct `path` label value (`/api/v1/nonexistent_path_1`, `/api/v1/nonexistent_path_2`, …) recorded into all four per-route instruments. The Node.js heap grows proportionally until the process OOMs or becomes unresponsive, blocking all legitimate API traffic.
