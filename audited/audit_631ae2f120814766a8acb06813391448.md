### Title
`TracingMiddleware` `internalPaths` Bypass via Percent-Encoded Path Variants Causes Elevated Log-Level Emission

### Summary
`TracingMiddleware` in `rosetta/app/middleware/trace.go` extracts the request path using `request.URL.RequestURI()`, which preserves raw percent-encoding. The `internalPaths` map is keyed on canonical decoded strings (`/metrics`, `/health/liveness`, `/health/readiness`). A request to `/%6detrics` (or any other percent-encoded equivalent of `/metrics`) produces a `path` value that does not match any key in `internalPaths`, so the request is unconditionally logged at `log.Info` level instead of `log.Debug`. Any unauthenticated external caller can exploit this to inject a sustained stream of `Info`-level log entries, degrading signal-to-noise ratio in the log pipeline.

### Finding Description
**Exact code path:**

- `rosetta/app/middleware/trace.go`, `TracingMiddleware()`, line 47:
  ```go
  path := request.URL.RequestURI()
  ```
  `RequestURI()` delegates to `url.URL.EscapedPath()`, which returns `RawPath` when it is a valid encoding of `Path`. For an incoming request line `GET /%6detrics HTTP/1.1`, Go's HTTP parser sets `URL.Path = "/metrics"` and `URL.RawPath = "/%6detrics"`. `EscapedPath()` returns `/%6detrics` because it is a valid encoding of `/metrics`. Therefore `path = "/%6detrics"`.

- `rosetta/app/middleware/trace.go`, line 19:
  ```go
  var internalPaths = map[string]bool{livenessPath: true, metricsPath: true, readinessPath: true}
  ```
  `metricsPath = "/metrics"` (defined in `metrics.go` line 16). The map contains only the canonical, decoded strings.

- `rosetta/app/middleware/trace.go`, lines 55–59:
  ```go
  if internalPaths[path] {   // false for "/%6detrics"
      log.Debug(message)
  } else {
      log.Info(message)      // executed
  }
  ```

**Root cause:** The code assumes that every request whose decoded path equals an internal path will also have a raw path identical to that internal path. This assumption is false: Go's `net/url` preserves the original percent-encoding in `RawPath`, and `RequestURI()` returns that raw form.

**Exploit flow:**
1. Attacker sends a high-rate stream of `GET /%6detrics HTTP/1.1` requests (no credentials needed; the `/metrics` endpoint has no authentication).
2. The gorilla/mux router (used by coinbase/rosetta-sdk-go) does not decode percent-encoded segments for route matching, so the request does not match the `/metrics` route and returns HTTP 404.
3. `TracingMiddleware` records `path = "/%6detrics"`, which is absent from `internalPaths`, and emits `log.Info(message)` for every such request.
4. Legitimate `log.Info` and `log.Warn`/`log.Error` entries are buried in the flood.

**Why existing checks fail:** The `internalPaths` map lookup is a plain string equality check with no path normalization (no `url.PathUnescape`, no `path.Clean`, no `url.URL.Path` dereference). The test suite in `trace_test.go` only tests the exact canonical strings and a suffix-extended variant (`/metricss`); no percent-encoded variant is tested, so the gap is undetected.

### Impact Explanation
During normal operation the `/metrics` and health endpoints are polled at high frequency by Prometheus scrapers and Kubernetes probes. Those legitimate requests are suppressed to `Debug` level precisely to avoid log noise. An attacker who replays percent-encoded variants at the same or higher rate forces every such request to emit an `Info`-level line, potentially saturating the log pipeline. During a network partition—when the readiness check is already failing and operators rely on log observability to diagnose the incident—this flood can mask critical `Error`/`Warn` entries, delaying incident response. No data is exfiltrated and no state is modified, so severity is **Low–Medium** (availability/observability impact only).

### Likelihood Explanation
Exploitation requires zero privileges and zero knowledge of the application beyond the existence of a `/metrics` endpoint (standard for Prometheus-instrumented Go services). The percent-encoding `%6d` → `m` is trivially constructed. The attack is fully repeatable and scriptable with a single `curl` or `ab` invocation. Rate-limiting middleware (Traefik `rateLimit` in the Helm values) is configured per-source-IP and set to an average of 10 req/s, which is sufficient to produce a noticeable log stream but may limit large-scale flooding from a single IP.

### Recommendation
Replace the raw `RequestURI()` lookup with the already-decoded `request.URL.Path`:

```go
// Before (vulnerable):
path := request.URL.RequestURI()

// After (safe):
path := request.URL.Path   // always the decoded, canonical path
```

`request.URL.Path` is set by Go's HTTP parser to the percent-decoded path regardless of how the client encoded the request. The log message can still include the raw URI for forensic fidelity by keeping `request.URL.RequestURI()` only in the `message` format string, while using `request.URL.Path` exclusively for the `internalPaths` lookup.

### Proof of Concept
```bash
# Confirm canonical path is suppressed (Debug only, not visible at Info level)
curl -s http://<rosetta-host>/metrics

# Bypass: percent-encode the 'm' in /metrics
# %6d == 'm', so /%6detrics decodes to /metrics but is not in internalPaths
curl -v "http://<rosetta-host>/%6detrics"
# Expected: HTTP 404 (router does not match), but TracingMiddleware emits log.Info

# Flood the log pipeline
while true; do
  curl -s "http://<rosetta-host>/%6detrics" > /dev/null
done
# Observe: continuous stream of Info-level entries in the log output,
# indistinguishable from legitimate application traffic logs.
```
