### Title
Unvalidated User-Supplied `node_account_id` in `getTransactionNodeAccountId()` Causes Guaranteed Submission Failure and Leaks Internal SDK Error Details

### Summary
`getTransactionNodeAccountId()` accepts any syntactically valid `hiero.AccountID` string from the caller-controlled `metadata` map without checking whether it corresponds to a node in the configured network. When the resulting transaction is submitted via `ConstructionSubmit()`, the SDK fails to route it and the raw SDK error — which can include internal gRPC endpoint details — is returned verbatim to the caller in the `reason` field of `ErrTransactionSubmissionFailed`. No authentication or privilege is required to trigger this.

### Finding Description

**Code path:**

`getTransactionNodeAccountId()` at [1](#0-0)  performs only a format parse:

```go
nodeAccountId, err := hiero.AccountIDFromString(str)
if err != nil {
    return emptyAccountId, errors.ErrInvalidAccount
}
// no whitelist check — returns attacker-controlled value
return nodeAccountId, nilErr
```

The function that *correctly* enumerates valid nodes is `getRandomNodeAccountId()` at [2](#0-1) , which iterates `c.sdkClient.GetNetwork()`. `getTransactionNodeAccountId()` never calls `c.sdkClient.GetNetwork()` and has no equivalent whitelist check.

The returned attacker-controlled `nodeAccountId` is embedded into the frozen transaction body via `transactionSetNodeAccountId(nodeAccountId)` at [3](#0-2) .

In `ConstructionSubmit()`, the raw SDK error is forwarded to the caller: [4](#0-3) 

```go
_, err = hiero.TransactionExecute(transaction, c.sdkClient)
if err != nil {
    return nil, errors.AddErrorDetails(
        errors.ErrTransactionSubmissionFailed,
        "reason",
        fmt.Sprintf("%s", err),   // raw SDK error returned to caller
    )
}
```

`AddErrorDetails` clones the error and sets `Details["reason"]` to the raw string, which is then serialized into the HTTP response body. [5](#0-4) 

**Root cause / failed assumption:** The code assumes the caller will faithfully forward the `node_account_id` produced by `/construction/metadata`. There is no server-side enforcement of this assumption. The metadata map is entirely caller-controlled in the `/construction/payloads` request.

### Impact Explanation

1. **Information disclosure**: The Hiero Go SDK error for an unknown/unreachable node includes gRPC dial details — internal IP addresses, hostnames, and port numbers of the nodes the SDK attempted to contact. These are returned in the JSON response body under `details.reason`, exposing internal network topology to any unauthenticated caller.

2. **Guaranteed per-transaction DoS**: Any caller can force their own (or a relayed) transaction to fail deterministically at submission time by supplying a non-existent node ID, bypassing the randomised node selection that `/construction/metadata` provides as a reliability mechanism.

Severity: **Medium** — information disclosure of internal infrastructure details with no authentication barrier; the DoS is self-scoped to the attacker's own transactions.

### Likelihood Explanation

The `/construction/payloads` and `/construction/submit` endpoints are public with no authentication. The attack requires only:
- Knowledge of the Rosetta Construction API spec (public)
- Ability to send HTTP POST requests

Any external user can reproduce this in a single request cycle. It is trivially repeatable and requires no special tooling.

### Recommendation

1. **Whitelist-validate the supplied node account ID** inside `getTransactionNodeAccountId()` against `c.sdkClient.GetNetwork()`:

```go
network := c.sdkClient.GetNetwork()
valid := false
for _, knownId := range network {
    if knownId == nodeAccountId {
        valid = true
        break
    }
}
if !valid {
    log.Errorf("Node account id %s not in configured network", str)
    return emptyAccountId, errors.ErrInvalidAccount
}
```

2. **Sanitize the error returned to the caller** in `ConstructionSubmit()`. Replace the raw SDK error string with a generic message; log the full error server-side only:

```go
log.Errorf("Failed to execute transaction %s (hash %s): %s", transactionId, hash, err)
return nil, errors.ErrTransactionSubmissionFailed  // no Details["reason"]
```

### Proof of Concept

**Preconditions:** Online Rosetta mirror node reachable at `http://rosetta-host:5700`.

**Step 1 — Build a payloads request with a non-existent node:**
```bash
curl -s -X POST http://rosetta-host:5700/construction/payloads \
  -H 'Content-Type: application/json' \
  -d '{
    "network_identifier": {"blockchain":"Hiero","network":"testnet"},
    "operations": [
      {"operation_identifier":{"index":0},"type":"CRYPTO_TRANSFER",
       "account":{"address":"0.0.1234"},"amount":{"value":"-1","currency":{"symbol":"HBAR","decimals":8}}},
      {"operation_identifier":{"index":1},"type":"CRYPTO_TRANSFER",
       "account":{"address":"0.0.5678"},"amount":{"value":"1","currency":{"symbol":"HBAR","decimals":8}}}
    ],
    "metadata": {
      "node_account_id": "0.0.99999",
      "valid_start_nanos": "1700000000000000000"
    }
  }'
```
The service accepts `0.0.99999` — it passes `hiero.AccountIDFromString` — and returns an `unsigned_transaction` hex with node `0.0.99999` baked in.

**Step 2 — Sign and combine** (standard Rosetta flow with any valid key).

**Step 3 — Submit:**
```bash
curl -s -X POST http://rosetta-host:5700/construction/submit \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hiero","network":"testnet"},
       "signed_transaction":"<hex from step 2>"}'
```

**Expected response** (HTTP 500):
```json
{
  "code": 118,
  "message": "Transaction submission failed",
  "retriable": false,
  "details": {
    "reason": "failed to execute transaction: rpc error: code = Unavailable desc = connection error: ... 10.x.x.x:50211 ..."
  }
}
```

The `details.reason` field contains the raw gRPC error including internal node endpoint addresses, confirming the information disclosure.

```go
func (c *constructionAPIService) getTransactionNodeAccountId(metadata map[string]any) (
	emptyAccountId hiero.AccountID, nilErr *rTypes.Error,
) {
	value, ok := metadata[metadataKeyNodeAccountId]
	if !ok {
		return emptyAccountId, errors.ErrMissingNodeAccountIdMetadata
	}

	str, ok := value.(string)
	if !ok {
		return emptyAccountId, errors.ErrInvalidArgument
	}

	nodeAccountId, err := hiero.AccountIDFromString(str)
	if err != nil {
		log.Errorf("Invalid node account id provided in metadata: %s", str)
		return emptyAccountId, errors.ErrInvalidAccount
	}

	log.Infof("Use node account id %s from metadata", str)
	return nodeAccountId, nilErr
}
```

```go
func (c *constructionAPIService) getRandomNodeAccountId() (hiero.AccountID, *rTypes.Error) {
	nodeAccountIds := make([]hiero.AccountID, 0)
	seen := map[hiero.AccountID]struct{}{}
	// network returned from sdkClient is a map[string]AccountID, the key is the address of a node and the value is
	// its node account id. Since a node can have multiple addresses, we need the seen map to get a unique node account
	// id array
	for _, nodeAccountId := range c.sdkClient.GetNetwork() {
		if _, ok := seen[nodeAccountId]; ok {
			continue
		}

		seen[nodeAccountId] = struct{}{}
		nodeAccountIds = append(nodeAccountIds, nodeAccountId)
	}

	if len(nodeAccountIds) == 0 {
		return hiero.AccountID{}, errors.ErrNodeAccountIdsEmpty
	}

	maxValue := big.NewInt(int64(len(nodeAccountIds)))
	index, err := rand.Int(rand.Reader, maxValue)
	if err != nil {
		log.Errorf("Failed to get a random number, use 0 instead: %s", err)
		return nodeAccountIds[0], nil
	}

	return nodeAccountIds[index.Int64()], nil
}
```
