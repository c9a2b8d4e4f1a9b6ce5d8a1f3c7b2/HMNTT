### Title
Unbounded Percent-Encoded URL Logged Verbatim in `TracingMiddleware` Enables Log I/O Amplification

### Summary
`TracingMiddleware` captures the raw output of `request.URL.RequestURI()` and embeds it unsanitized into every log entry. Because `RequestURI()` returns the percent-encoded form of the URL (e.g., `%41` instead of `A`), an attacker can craft requests whose path segment is up to ~3× larger in encoded form than in decoded form, inflating every log line proportionally. No URL-length cap, no path sanitization, and no rate limiting exist in the rosetta middleware stack, so any unauthenticated caller can drive this amplification at will.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/trace.go`, function `TracingMiddleware()`:

```go
// line 47
path := request.URL.RequestURI()          // returns raw percent-encoded URI

// lines 52-53
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)  // path embedded verbatim

// line 55
if internalPaths[path] {                  // encoded path never matches map keys
    log.Debug(message)
} else {
    log.Info(message)                     // always Info for encoded paths
}
```

**Root cause:** `net/url.URL.RequestURI()` returns the URI in its wire-encoded form. A path composed entirely of percent-encoded octets (e.g., `/%41%41%41…`) is 3 bytes per logical character instead of 1. The middleware copies this string directly into the log message with no length check, no decoding, and no truncation.

**Secondary issue:** The `internalPaths` map contains literal strings (`/health/liveness`, `/health/readiness`, `/metrics`). An encoded variant such as `/health%2Fliveness` never matches, so the request is logged at `Info` level rather than `Debug`, bypassing the intended suppression for internal probes.

**No mitigating controls in the stack:**
- `rosetta/app/config/types.go` `Http` struct defines only `IdleTimeout`, `ReadHeaderTimeout`, `ReadTimeout`, `WriteTimeout` — no `MaxHeaderBytes` override is set on the `http.Server` in `rosetta/main.go` (lines 220-227), leaving Go's default 1 MiB header budget in place.
- No rate-limiting middleware exists anywhere in `rosetta/app/middleware/`.
- No URL-length guard exists before or inside `TracingMiddleware`.

### Impact Explanation
Go's default `MaxHeaderBytes` (1 MiB) bounds the request line + all headers. A URL path of ~900 KB filled with `%41` sequences is accepted by the server and produces a single log entry of roughly the same size. Under normal operation a log line is on the order of 100–200 bytes; a single malicious request inflates that to ~900 KB — a factor of ~4,500×. Sustained at even a modest request rate (e.g., 100 req/s), this generates ~90 MB/s of log I/O, which can saturate disk write bandwidth, exhaust log-storage quotas, crowd out legitimate log entries, and introduce synchronous I/O latency into request handling (logrus writes are synchronous by default). The secondary `internalPaths` bypass means health-check paths can also be forced to emit `Info`-level entries, defeating log-level filtering intended to reduce noise.

### Likelihood Explanation
No authentication or API key is required to reach the Rosetta HTTP endpoint. The exploit requires only a standard HTTP client capable of sending a long GET request — no special tooling, no prior knowledge of the system, and no brute force. The attack is trivially repeatable and scriptable. Any internet-accessible deployment is exposed.

### Recommendation
1. **Truncate or sanitize `path` before logging.** Decode the URI and cap its logged length, e.g.:
   ```go
   rawPath := request.URL.RequestURI()
   decoded, err := url.PathUnescape(rawPath)
   if err != nil { decoded = rawPath }
   const maxLoggedPath = 256
   if len(decoded) > maxLoggedPath {
       decoded = decoded[:maxLoggedPath] + "…"
   }
   path := decoded
   ```
2. **Set `MaxHeaderBytes` on the `http.Server`** (e.g., 8 KB) to reject oversized request lines before they reach middleware.
3. **Fix the `internalPaths` lookup** to compare against the decoded path so encoded variants of health/metrics paths are still suppressed at `Debug` level.
4. **Add rate limiting** (e.g., `golang.org/x/time/rate`) in the middleware chain to bound per-IP request rates.

### Proof of Concept
```bash
# Build a ~60 KB percent-encoded path (each 'A' → '%41', 20000 triplets = 60000 bytes)
PAYLOAD=$(python3 -c "print('/%41' * 20000, end='')")

# Send to the rosetta node (no credentials required)
curl -s -o /dev/null -w "%{http_code}\n" \
     -X GET "http://<rosetta-host>:5700${PAYLOAD}"

# Observe the log output: each line will be ~60 KB
# Repeat at 100 req/s to generate ~6 MB/s of log I/O from a single client:
while true; do
  curl -s -o /dev/null "http://<rosetta-host>:5700${PAYLOAD}" &
  sleep 0.01
done
```

The server accepts the request (returns 404 or 405 depending on routing), and `TracingMiddleware` emits a ~60 KB `Info`-level log line for every iteration. Scaling to multiple concurrent clients or longer paths (up to the ~1 MiB server limit) proportionally increases log I/O overhead.
