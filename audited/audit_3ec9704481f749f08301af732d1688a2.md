### Title
Unbounded HTTP Method Label Cardinality in Metrics Middleware Enables Memory Exhaustion DoS

### Summary
In `rest/middleware/metricsHandler.js`, the `metricsMiddleware()` function records `req.method` directly as a Prometheus/OpenTelemetry label without any allowlist validation or cardinality cap. An unprivileged attacker can send HTTP requests with arbitrary non-standard method strings, causing the OpenTelemetry SDK to allocate a new in-memory time-series for every unique method value across four histogram/counter instruments, leading to unbounded heap growth and eventual OOM crash of the REST API process.

### Finding Description
**Exact code path:**

In `rest/middleware/metricsHandler.js` lines 191–192 and 207–210:
```js
const method = req.method;           // line 191 — raw, unsanitized
const labels = {method, path, code}; // line 192

requestTotalCounter.add(1, labels);          // line 207
durationHistogram.record(duration, labels);  // line 208
requestSizeHistogram.record(..., labels);     // line 209
responseSizeHistogram.record(..., labels);    // line 210
``` [1](#0-0) 

`req.method` is taken verbatim from the Node.js HTTP parser (llhttp), which accepts any RFC 7230 token as a method name — including non-standard strings like `AAAA`, `AAAB`, `FOOBAR1`, etc. Express does not normalize or restrict `req.method`.

**Root cause:** The code assumes only a small, bounded set of HTTP methods (GET, POST, etc.) will appear as label values. There is no allowlist (e.g., `['GET','POST','PUT','DELETE','OPTIONS','HEAD','PATCH']`) and no cardinality cap configured on the OpenTelemetry `MeterProvider`.

**Why existing checks fail:**

- `authHandler` (lines 15–36 of `rest/middleware/authHandler.js`) only handles Basic Auth credential lookup; it performs zero validation of the HTTP method and does not block unauthenticated requests from reaching `metricsMiddleware`.
- `metricsMiddleware` runs for **all** requests (line 152: `return function metricsMiddleware(req, res, next)`), including those that will later be rejected by route handlers with 404/405.
- The `res.on('finish', ...)` callback at line 185 fires regardless of the response status code, so even rejected requests contribute new label combinations.
- No rate limiting is present in the middleware chain before metrics recording. [2](#0-1) [3](#0-2) 

### Impact Explanation
Each unique `{method, path, code}` triple creates a new time-series in the OpenTelemetry SDK's in-memory storage across all four instruments (`requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, `responseSizeHistogram`). A histogram with 4 buckets stores 6 float64 values per series (~48 bytes). Sending 1 million unique method strings against a single path generates ~4 million time-series, consuming hundreds of MB to several GB of heap, ultimately crashing the Node.js REST API process via OOM. The direct impact is **denial of service of the mirror node REST API**. Note: this does not directly delay block processing (the importer is a separate component), but it renders the REST API unavailable to all consumers.

### Likelihood Explanation
No authentication is required — the metrics middleware runs before any auth gate and instruments all requests including unauthenticated ones. The attack requires only a TCP connection and the ability to send HTTP/1.1 requests with non-standard method tokens, which any standard HTTP client (curl, Python requests with raw socket, etc.) can do. The attack is repeatable, automatable, and requires no special knowledge of the application. A single attacker with a modest script can exhaust memory within minutes.

### Recommendation
1. **Normalize `req.method` to an allowlist** before using it as a label:
   ```js
   const ALLOWED_METHODS = new Set(['GET','POST','PUT','DELETE','PATCH','HEAD','OPTIONS']);
   const method = ALLOWED_METHODS.has(req.method) ? req.method : 'UNKNOWN';
   ```
2. **Configure an OpenTelemetry cardinality limit** on the `MeterProvider` (e.g., `cardinalityLimit` per instrument) to cap the number of unique label combinations.
3. **Add a method validation middleware** early in the Express chain (before `metricsHandler`) that rejects requests with non-standard HTTP methods with a `405 Method Not Allowed`.

### Proof of Concept
```bash
# Send 10,000 requests with unique method strings to exhaust label cardinality
for i in $(seq 1 10000); do
  METHOD=$(cat /dev/urandom | tr -dc 'A-Z' | head -c 8)
  curl -s -X "$METHOD" http://<mirror-node-rest>:5551/api/v1/accounts &
done
wait
# Monitor REST API process memory: watch -n1 'ps aux | grep node'
# Heap will grow unboundedly; process will OOM-crash or become unresponsive
```

Each unique method string `AAAAAAAA`, `BBBBBBBB`, … creates a new set of time-series across all four OpenTelemetry instruments. With no cardinality cap, the SDK accumulates these indefinitely in the heap until the process is killed by the OS OOM killer. [4](#0-3) [5](#0-4)
