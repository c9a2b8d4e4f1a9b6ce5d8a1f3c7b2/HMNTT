### Title
Liveness Probe Bypass / Restart-Loop via Unix Domain Socket at `/tmp/alive`

### Summary
`healthcheck live` uses `os.Stat("/tmp/alive")` without verifying the file is a regular file. Because `stat(2)` succeeds for any filesystem entry — including Unix domain sockets — an unprivileged process that pre-creates `/tmp/alive` as a socket can silently block the pinger's heartbeat writer (`os.WriteFile`, whose error is discarded), causing the liveness probe to either pass falsely for up to two minutes or enter a permanent restart loop once the socket's mtime goes stale.

### Finding Description
**File:** `pinger/cmd/healthcheck/main.go`, `main()`, lines 27–38
**File:** `pinger/main.go`, heartbeat goroutine, line 36

```
// healthcheck — no file-type check
fi, err := os.Stat("/tmp/alive")   // succeeds for socket, FIFO, dir, etc.
if err != nil { os.Exit(1) }
age := time.Since(fi.ModTime())
if age > 2*time.Minute { os.Exit(1) }
os.Exit(0)

// pinger heartbeat — error silently discarded
_ = os.WriteFile("/tmp/alive", []byte(...), 0644)
```

**Root cause — two compounding flaws:**

1. `os.Stat()` returns successfully for any inode type. The code never calls `fi.Mode().IsRegular()`, so a socket, FIFO, or directory at `/tmp/alive` passes the existence check.

2. `os.WriteFile` internally calls `open(path, O_WRONLY|O_CREATE|O_TRUNC)`. On Linux, `open(2)` on a Unix domain socket returns `ENXIO`. The pinger discards this error (`_ = os.WriteFile(...)`), so the heartbeat silently stops updating the mtime of the socket. The socket's mtime is frozen at creation time.

**Exploit flow:**

- Attacker creates `/tmp/alive` as a Unix domain socket (e.g. via a sidecar container sharing an `emptyDir /tmp` volume, or any process with write access to `/tmp` before the pinger starts).
- Pinger starts; every 15 s it calls `os.WriteFile("/tmp/alive", ...)` which fails with `ENXIO` — silently.
- Liveness probe calls `healthcheck live`: `os.Stat` succeeds (socket exists), mtime is the socket's creation time.
- For the first ~2 minutes the probe returns exit 0 (false healthy).
- After 2 minutes the mtime goes stale → probe returns exit 1 → Kubernetes restarts the container.
- On restart the socket persists (emptyDir is not cleared between container restarts, only pod restarts) → the cycle repeats indefinitely.

**Existing checks are insufficient:** The only guards are `err != nil` from `os.Stat` and the 2-minute age threshold. Neither distinguishes a regular file from a socket.

### Impact Explanation
The pinger's sole purpose is to submit periodic Hedera transactions and signal liveness. A persistent restart loop prevents it from ever completing a transaction cycle. Kubernetes will apply exponential back-off (`CrashLoopBackOff`), making the pinger effectively unavailable for extended periods. No funds are at risk, but the monitoring/health-signalling function of the service is completely neutralised — matching the stated "griefing, no economic damage" severity.

### Likelihood Explanation
Precondition: write access to `/tmp` inside the pod before or concurrently with pinger startup. In Kubernetes this is achievable without elevated privileges via:
- A sidecar or init container in the same pod with a shared `emptyDir` at `/tmp` (common pattern for log shipping, proxies, etc.).
- Any other process sharing the container's mount namespace.

The Dockerfile runs as `USER 1000:1000` from `scratch`, so there is no shell or extra tooling inside the container itself — but the attack originates from *outside* the pinger container, in a co-located sidecar. The technique requires no kernel exploits, no root, and is fully repeatable.

### Recommendation
In `pinger/cmd/healthcheck/main.go`, add a regular-file type check immediately after `os.Stat`:

```go
fi, err := os.Stat("/tmp/alive")
if err != nil {
    fmt.Fprintln(os.Stderr, err)
    os.Exit(1)
}
if !fi.Mode().IsRegular() {
    fmt.Fprintln(os.Stderr, "heartbeat path is not a regular file")
    os.Exit(1)
}
```

Additionally, in `pinger/main.go` propagate (log) the `os.WriteFile` error instead of discarding it, so silent heartbeat failures are observable:

```go
if err := os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644); err != nil {
    log.Printf("heartbeat write failed: %v", err)
}
```

### Proof of Concept

```bash
# 1. In a sidecar / init container sharing emptyDir at /tmp:
python3 -c "
import socket, os
s = socket.socket(socket.AF_UNIX)
s.bind('/tmp/alive')
# socket is now at /tmp/alive; mtime = now
"

# 2. Pinger starts; heartbeat goroutine fires every 15 s:
#    os.WriteFile("/tmp/alive", ...) → open() returns ENXIO → silently discarded
#    mtime of /tmp/alive never advances past creation time

# 3. Within 2 minutes — liveness probe passes (false healthy):
/healthcheck live   # exits 0

# 4. After 2 minutes — liveness probe fails:
/healthcheck live   # exits 1, stderr: "heartbeat stale"
# Kubernetes restarts the container; emptyDir /tmp persists → loop repeats
