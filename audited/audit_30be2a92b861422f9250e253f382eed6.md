### Title
Log Injection via Unsanitized `X-Forwarded-For` / `X-Real-IP` Header in `getClientIpAddress()`

### Summary
The `getClientIpAddress()` function in `rosetta/app/middleware/trace.go` accepts the `X-Forwarded-For` (or `X-Real-IP`) header value verbatim and passes it directly into a log message without any sanitization. The only guard is `len(ipAddress) == 0`, which is bypassed by any single-byte value — including `\n` (0x0A). An unprivileged external attacker can inject arbitrary content into the log stream, corrupting structured log output and misleading or breaking downstream log consumers.

### Finding Description

**Exact code path:** [1](#0-0) 

`getClientIpAddress()` reads `X-Real-IP` first, then `X-Forwarded-For`. The only rejection condition is `len(ipAddress) == 0`. A header value of `"\n"` (one byte, 0x0A) has `len == 1`, so it passes both guards and is returned as-is. [2](#0-1) 

The returned value is then interpolated directly into `message` via `fmt.Sprintf` and passed to `log.Info` / `log.Debug` (logrus). No escaping, stripping, or validation of the header value occurs at any point.

**Root cause:** The code assumes the header contains a valid IP address string. It only checks for emptiness, not for content validity or the presence of control characters.

**Failed assumption:** `len(ipAddress) > 0` does **not** imply the value is a safe, printable IP address string.

### Impact Explanation

When logrus uses its default **TextFormatter**, each log entry is a single line. Injecting `\n` splits the entry, and anything after the newline is written as a new, attacker-controlled "log line." This enables:

- **Log forgery**: Fake entries (e.g., `time="..." level=info msg="admin login succeeded"`) appear as legitimate application log lines.
- **Log structure corruption**: Structured log parsers (Logstash, Fluentd, Splunk forwarders) that expect one-entry-per-line can misparse or drop subsequent real entries.
- **Security-monitoring bypass**: Injected lines can drown out or overwrite real audit events, defeating alerting rules.
- **ANSI escape injection**: Terminals or log viewers rendering ANSI codes can be manipulated (cursor movement, color flooding) causing display corruption.

Severity is **High** for a service exposed to external traffic, consistent with the scope label (non-network DoS / log integrity compromise affecting a significant market-cap project).

### Likelihood Explanation

- **No authentication required**: Any HTTP client can set arbitrary headers.
- **Trivially reproducible**: A single `curl -H "X-Forwarded-For: $'\n'fake entry"` suffices.
- **No rate-limiting or WAF assumed at the application layer**: The middleware itself imposes none.
- **Repeatable at will**: Every request is logged, so the attacker can flood logs continuously.

### Recommendation

1. **Validate the header as an IP address** before using it:
   ```go
   if net.ParseIP(ipAddress) == nil {
       ipAddress = "" // fall through to RemoteAddr
   }
   ```
2. **Strip or escape control characters** if a looser policy is required:
   ```go
   ipAddress = strings.Map(func(r rune) rune {
       if r < 0x20 || r == 0x7f { return -1 }
       return r
   }, ipAddress)
   ```
3. Apply the sanitization **inside `getClientIpAddress()`** before returning, so all callers are protected automatically.

### Proof of Concept

```bash
# Inject a fake log line via a 1-byte newline in X-Forwarded-For
curl -s \
  -H $'X-Forwarded-For: \ntime="2024-01-01T00:00:00Z" level=info msg="INJECTED: admin authenticated"' \
  http://<rosetta-host>/network/list
```

**Expected log output (corrupted):**
```
time="..." level=info msg=" POST /network/list (200) in 3ms"
time="2024-01-01T00:00:00Z" level=info msg="INJECTED: admin authenticated"
```

The second line is entirely attacker-controlled and indistinguishable from a real application log entry. The `\n` character is exactly 1 byte, satisfying `len(ipAddress) != 0` and bypassing the only existing guard.

func getClientIpAddress(r *http.Request) string {
	ipAddress := r.Header.Get(xRealIpHeader)

	if len(ipAddress) == 0 {
		ipAddress = r.Header.Get(xForwardedForHeader)
	}

	if len(ipAddress) == 0 {
		ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
	}

	return ipAddress
