### Title
Unauthenticated Access to Prometheus Metrics Endpoint via Insecure Default in `authenticate()`

### Summary
The `authenticate` function in `rest/middleware/metricsHandler.js` returns `true` (grants access) whenever `config.metrics.config.authentication` is any falsy value — including `undefined`, which is the state when no configuration is loaded. Any unauthenticated external user can reach the Prometheus metrics endpoint and obtain per-route request counts, error distributions, response sizes, and timing histograms — including for smart contract query endpoints — without supplying any credentials.

### Finding Description
**Exact code location**: `rest/middleware/metricsHandler.js`, `authenticate()`, lines 137–144.

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {   // ← returns true for undefined, null, false, 0, ""
    return true;
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

**Root cause**: The guard `if (!authentication) return true` conflates two distinct states — "authentication explicitly disabled by the operator" and "authentication key absent from config" — and treats both identically as "allow all." There is no hardcoded safe default in the code; no `application.yml` was found in the repository to supply one at the framework level. If the config file is absent, incomplete, or the key is omitted, `authentication` resolves to `undefined`, `!undefined` evaluates to `true`, and the function returns `true` unconditionally.

**Failed assumption**: The code assumes `config.metrics.config.authentication` will always be explicitly set to a truthy value in production. The documentation lists the default as `true` (`docs/configuration.md` line 569), but this is a documentation claim — the code itself has no fallback, making any deployment without a fully-loaded config silently open.

**Exploit flow**:
1. The `metricsMiddleware` at line 156–164 calls `authenticate(req)` as the sole access gate.
2. When `authenticate` returns `true`, the handler immediately calls `exporter.collect()` and serializes the full Prometheus metric set to the response.
3. No other middleware, IP filter, or secondary check exists between the network and the metrics response. [1](#0-0) [2](#0-1) 

### Impact Explanation
The exposed metrics include:

- `api_request_total` with labels `{method, path, code}` — reveals exactly which contract endpoints (`/api/v1/contracts/{id}`, `/api/v1/contracts/{id}/results`, etc.) are being queried, their HTTP status codes, and call volumes.
- `api_request_duration_milliseconds` histogram — reveals per-route latency, exposing computational complexity of specific contract queries.
- `api_response_size_bytes` histogram — reveals data volume returned per contract endpoint, which can indicate the size of contract state or result sets.
- `api_all_client_error` / `api_all_server_error` counters — reveals error rates, helping an attacker identify which contract queries are failing and at what rate. [3](#0-2) [4](#0-3) 

An attacker can use this intelligence to map active contract query patterns, identify high-error-rate endpoints worth targeting, infer response payload sizes to correlate with on-chain state, and time attacks to coincide with observed load patterns — all without any privileged access.

### Likelihood Explanation
The precondition is a deployment where `authentication` is not explicitly set to a truthy value. This is realistic in: containerized deployments that omit the config file, environments where config is partially overridden (e.g., setting `authentication: false` for testing and forgetting to revert), or any fresh deployment before the config is hardened. The attack itself requires only a single unauthenticated HTTP GET request — no credentials, no special tooling, fully repeatable and automatable.

### Recommendation
Replace the insecure falsy-check default with a secure-by-default pattern in the code itself:

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  // Treat absent/non-boolean config as authentication REQUIRED, not disabled
  if (authentication === false) {
    return true;  // Only explicitly opt-out disables auth
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

Additionally, add a startup warning when `authentication` is not `true`, and ensure the default `application.yml` explicitly sets `authentication: true` so the safe default is enforced at both the code and config layers. [1](#0-0) 

### Proof of Concept

```bash

# Expected result: full Prometheus text exposition, e.g.:
# api_request_total{method="GET",path="/api/v1/contracts/{id}/results",code="200"} 142
# api_request_duration_milliseconds_bucket{...le="100"} 89
# api_response_size_bytes_bucket{...le="1000"} 130

```

No credentials required. The response reveals per-route contract endpoint usage, error distributions, and response size distributions in plain text.
