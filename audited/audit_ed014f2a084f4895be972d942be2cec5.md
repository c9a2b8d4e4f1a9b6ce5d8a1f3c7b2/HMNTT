### Title
`getLatestBalanceSnapshot` Misclassifies `gorm.ErrRecordNotFound` as `ErrDatabaseError` When Treasury Has No Balance Snapshot in Range

### Summary
In `rosetta/app/persistence/account.go`, `getLatestBalanceSnapshot()` uses GORM's `.First()` to execute the `latestBalanceBeforeConsensus` query, which anchors entirely on the treasury account's balance snapshot. When no treasury snapshot exists within the computed `[@lower_bound, @timestamp]` range, the derived table `bt` returns zero rows, causing `.First()` to return `gorm.ErrRecordNotFound`. The error handler at lines 295–301 catches all errors without distinguishing `gorm.ErrRecordNotFound` from real DB failures, returning `hErrors.ErrDatabaseError` (code 125, retriable) instead of the semantically correct `hErrors.ErrNodeIsStarting` (code 110). The developer-intended guard at line 304 (`ab.ConsensusTimestamp == 0`) is structurally unreachable in this path.

### Finding Description

**Exact code path:**

`latestBalanceBeforeConsensus` (lines 30–50) selects from a derived table `bt`:
```sql
from (
  select consensus_timestamp
  from account_balance
  where account_id = @treasury_entity_id and
    consensus_timestamp >= @lower_bound and
    consensus_timestamp <= @timestamp
  order by consensus_timestamp desc
  limit 1
) as bt
```
If the treasury account has no row in `account_balance` satisfying `consensus_timestamp >= @lower_bound AND consensus_timestamp <= @timestamp`, `bt` is empty and the outer `SELECT` returns **zero rows**.

In `getLatestBalanceSnapshot()` (lines 289–301):
```go
if err := db.Raw(
    latestBalanceBeforeConsensus, ...
).First(ab).Error; err != nil {
    // catches gorm.ErrRecordNotFound AND real DB errors identically
    return 0, nil, hErrors.ErrDatabaseError
}
```
GORM's `.First()` returns `gorm.ErrRecordNotFound` on zero rows. The handler has no `errors.Is(err, gorm.ErrRecordNotFound)` branch, so it falls through to `return 0, nil, hErrors.ErrDatabaseError`.

**Why the existing guard fails:**

Line 304:
```go
if ab.ConsensusTimestamp == 0 {
    return 0, nil, hErrors.ErrNodeIsStarting
}
```
This is **never reached** when `.First()` returns an error — the function already returned at line 301. The guard only executes on a successful query result, which cannot have `ConsensusTimestamp == 0` because `bt.consensus_timestamp` is sourced directly from the treasury's non-zero `consensus_timestamp` column.

**Contrast with sibling functions in the same file** — both `GetAccountAlias` (lines 117–122) and `GetAccountId` (lines 150–155) explicitly check `errors.Is(err, gorm.ErrRecordNotFound)` and return appropriate semantic errors. `getLatestBalanceSnapshot` is the only `.First()`-using function that omits this check.

**Trigger condition:**

`timestamp` is derived from `block.ConsensusEndNanos` (account_service.go line 61), which comes from a user-supplied `BlockIdentifier` (account_service.go lines 52–56). `partitionLowerBound` is computed by `selectPreviousPartitionLowerBound` — if no matching partition exists, it stays `0`. Any valid block whose `ConsensusEndNanos` predates the earliest treasury `account_balance` row (or falls in a newly started partition with no treasury snapshot yet) satisfies the trigger.

### Impact Explanation

An unprivileged API caller querying `/account/balance` or `/account/balances` with a `block_identifier` pointing to a block whose timestamp precedes any treasury balance snapshot in the computed partition window receives `ErrDatabaseError` (code 125, `retriable: true`) instead of `ErrNodeIsStarting` (code 110, `retriable: true`). The semantic mismatch causes Rosetta clients to interpret a data-availability gap as a transient infrastructure failure, potentially triggering indefinite retry storms against the node. No HBAR balances are directly at risk, consistent with the medium/layer-0 classification.

### Likelihood Explanation

Any unprivileged user of the Rosetta API can trigger this by:
- Querying a block from before the mirror node's first ingested balance snapshot (common on partially-synced or freshly deployed nodes).
- Querying a block whose timestamp falls in a newly created `account_balance` partition that has not yet received a treasury snapshot (a normal operational window between partition creation and first snapshot ingestion).

No special privileges, credentials, or knowledge of internal state are required beyond knowing a valid block hash or index. The condition is repeatable and deterministic.

### Recommendation

Add a `gorm.ErrRecordNotFound` check in `getLatestBalanceSnapshot()`, mirroring the pattern already used in `GetAccountAlias` and `GetAccountId`:

```go
if err := db.Raw(
    latestBalanceBeforeConsensus, ...
).First(ab).Error; err != nil {
    if errors.Is(err, gorm.ErrRecordNotFound) {
        return 0, nil, hErrors.ErrNodeIsStarting
    }
    log.Errorf(...)
    return 0, nil, hErrors.ErrDatabaseError
}
```

This makes the `ab.ConsensusTimestamp == 0` guard at line 304 reachable only for genuine zero-value results (if any), and ensures clients receive the semantically correct `ErrNodeIsStarting` when no treasury snapshot exists in the queried range.

### Proof of Concept

**Preconditions:**
1. A running Hiero Mirror Node Rosetta service.
2. The `account_balance` table contains at least one block record but **no treasury account row** with `consensus_timestamp <= T` for some target timestamp `T` (e.g., a freshly started node mid-partition, or a node whose earliest ingested block predates balance snapshot ingestion).

**Steps:**
1. Identify a valid block index `N` whose `consensus_end` timestamp `T` is before any treasury `account_balance` row in the partition window.
2. Send:
```json
POST /account/balance
{
  "network_identifier": { "blockchain": "Hedera", "network": "mainnet" },
  "account_identifier": { "address": "0.0.1234" },
  "block_identifier": { "index": N }
}
```
3. **Expected (correct):** HTTP 200 with Rosetta error code 110 (`Node is starting`).
4. **Actual (buggy):** HTTP 200 with Rosetta error code 125 (`Database error`, `retriable: true`).

The mismatch is confirmed by the absence of `errors.Is(err, gorm.ErrRecordNotFound)` at lines 295–301 of `rosetta/app/persistence/account.go`, while the same pattern is correctly applied at lines 118 and 151 of the same file.
