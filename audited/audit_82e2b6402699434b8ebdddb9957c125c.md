### Title
Query String Bypass of Internal-Path Log Suppression in `TracingMiddleware`

### Summary
`TracingMiddleware` uses `request.URL.RequestURI()` to obtain the request path, which returns the full URI including any query string. The `internalPaths` map contains only bare paths (e.g., `/health/liveness`), so a request to `/health/liveness?x=1` produces a map key that does not match, causing the request to be logged at `Info` level instead of `Debug`. Any unprivileged external user can exploit this to flood the Info-level log with health-check noise, obscuring legitimate smart contract call traces.

### Finding Description
**File:** `rosetta/app/middleware/trace.go`
**Function:** `TracingMiddleware()`, lines 43–61

At line 47, the path variable is assigned via:
```go
path := request.URL.RequestURI()
```
`RequestURI()` returns the encoded path **plus** any query string (e.g., `/health/liveness?x=1`).

At line 19, the suppression map is:
```go
var internalPaths = map[string]bool{livenessPath: true, metricsPath: true, readinessPath: true}
```
where `livenessPath = "/health/liveness"`, `readinessPath = "/health/readiness"`, `metricsPath = "/metrics"` — all bare paths, no query strings.

At lines 55–59, the log-level decision is:
```go
if internalPaths[path] {
    log.Debug(message)
} else {
    log.Info(message)
}
```
When `path` is `/health/liveness?x=1`, the map lookup returns `false`, and the message is emitted at `Info` level. The existing check is the **only** guard; there is no normalization of `path` to strip the query component before the lookup.

### Impact Explanation
An attacker can continuously send health-check requests with arbitrary query strings (e.g., `GET /health/liveness?flood=1`) to inject a high volume of `Info`-level log entries. Because Rosetta's smart contract call traces are also logged at `Info`, this log pollution degrades the signal-to-noise ratio of operational logs, making it difficult or impossible for operators to detect anomalous smart contract behavior in real time. No funds are directly at risk, but the observability layer — the primary mechanism for detecting protocol-level abuse — is compromised.

### Likelihood Explanation
The exploit requires zero authentication, zero special privileges, and zero knowledge of the system beyond the public health endpoint paths (which are standard and documented). It is trivially repeatable with a single `curl` command or a scripted loop. The health endpoints are typically reachable from the internet or from any pod in a Kubernetes cluster, making this accessible to any external or internal attacker.

### Recommendation
Replace `request.URL.RequestURI()` with `request.URL.Path` at line 47, which returns only the path component without the query string:

```go
// Before (vulnerable):
path := request.URL.RequestURI()

// After (fixed):
path := request.URL.Path
```

This ensures the `internalPaths` map lookup always operates on the bare path regardless of any query string appended by the caller.
