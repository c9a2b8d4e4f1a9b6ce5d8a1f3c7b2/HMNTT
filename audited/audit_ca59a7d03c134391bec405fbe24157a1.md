### Title
Unauthenticated HTTP Mirror-Node Response Allows MITM JSON Injection to Permanently Prevent Pinger Startup

### Summary
The pinger fetches network topology over plain HTTP with no transport security or response integrity verification. An attacker on the same network segment can ARP-spoof the mirror node, inject a malformed JSON body, and cause `json.NewDecoder().Decode()` to return an error that is explicitly marked non-retryable, propagating fatally through `newClient()` into `log.Fatalf`, permanently killing the pinger process before it ever starts.

### Finding Description
**Code path:**

1. `pinger/main.go:41-43` — `newClient(cfg)` is called once at startup; any error triggers `log.Fatalf`, terminating the process. [1](#0-0) 

2. `pinger/sdk_client.go:18-21` — for `network=other`, `buildNetworkFromMirrorNodes` is called; its error is returned directly. [2](#0-1) 

3. `pinger/mirror_node_client.go:46` — the HTTP client is constructed with no TLS transport, no certificate pinning, and no response-integrity check. [3](#0-2) 

4. `pinger/mirror_node_client.go:96-98` — `json.NewDecoder(resp.Body).Decode(&payload)` on a tampered body returns an error; crucially, `retry=false` is returned. [4](#0-3) 

5. `pinger/mirror_node_client.go:59-61` — the retry loop breaks immediately when `retry=false`, so no subsequent attempt is made regardless of `mirrorNodeClientMaxRetries`. [5](#0-4) 

**Root cause:** The `mirrorRest` URL defaults to plain `http://` (documented example: `http://mirror-rest:5551`). No TLS is enforced anywhere in the HTTP client construction. A JSON decode failure is classified as a permanent (non-retryable) error, so a single injected bad response exhausts all retry budget instantly. [6](#0-5) 

### Impact Explanation
The pinger is the sole network health monitoring component. If it never starts, `/tmp/ready` is never written, liveness/readiness probes fail, and all downstream alerting on network health is silenced. The attacker achieves a complete, persistent DoS of the monitoring plane with a single intercepted HTTP exchange. No economic damage to on-chain users occurs, but operator visibility into network health is fully eliminated for as long as the attacker maintains the MITM position (or until the pinger is manually restarted into a clean network path). [7](#0-6) 

### Likelihood Explanation
Preconditions are realistic in a typical Kubernetes or bare-metal deployment where the pinger pod and the mirror-node REST pod share a flat L2 segment. ARP spoofing requires no special privileges beyond network adjacency. The attack is repeatable: the attacker simply needs to be present during the one startup window when `buildNetworkFromMirrorNodes` runs. Because the error is non-retryable, a single poisoned response is sufficient; the attacker does not need to sustain the MITM for the lifetime of the process. [8](#0-7) 

### Recommendation
1. **Enforce HTTPS** — validate that `cfg.mirrorRest` begins with `https://` and reject `http://` at config load time; configure the `http.Transport` with a non-nil `TLSClientConfig`.
2. **Retry on decode errors** — change `return nil, false, fmt.Errorf("decode mirror nodes: %w", err)` to `return nil, true, ...` so that a single bad response does not exhaust the retry budget.
3. **Response size cap** — wrap `resp.Body` with `io.LimitReader` before passing to the JSON decoder to prevent amplification via a large injected payload. [9](#0-8) 

### Proof of Concept
```
# Preconditions: attacker is on the same L2 segment as the pinger pod.
# 1. ARP-spoof the pinger so traffic to mirror-rest:5551 flows through attacker.
arpspoof -i eth0 -t <pinger-pod-IP> <mirror-rest-IP>

# 2. Use mitmproxy (or iptables + socat) to intercept the GET /api/v1/network/nodes
#    request and return a 200 OK with body: `{{{invalid json`

mitmproxy --mode transparent --script inject_bad_json.py
# inject_bad_json.py:
#   def response(flow):
#       if "/network/nodes" in flow.request.pretty_url:
#           flow.response.content = b"{{{invalid"

# 3. Start (or restart) the pinger with HIERO_MIRROR_PINGER_NETWORK=other.
#    Observed output:
#      pinger starting (version=development)
#      client error: attempt 1/1: decode mirror nodes: invalid character '{' ...
#      exit status 1
# /tmp/ready is never created; readiness probe fails indefinitely.
```
