### Title
Metrics Cardinality Explosion via Unbounded `path` Label on Unmatched Routes (DoS)

### Summary
The `metricsMiddleware` in `rest/middleware/metricsHandler.js` records per-route Prometheus metrics using a `path` label derived from `req.path` for any request that does not match a registered Express route. Because `req.path` is the raw, attacker-controlled URL path, an unprivileged external user can generate an unbounded number of unique label combinations, causing the OpenTelemetry `MeterProvider`'s in-memory time-series store to grow without limit until the Node.js process exhausts heap memory and crashes.

### Finding Description

**Middleware registration order** (`rest/server.js`, lines 89–92):
```js
if (config.metrics.enabled) {
  const {metricsHandler} = await import('./middleware/metricsHandler.js');
  app.useExt(metricsHandler());
}
```
`metricsHandler()` is registered **before** all route handlers. It attaches a `res.on('finish', ...)` callback to every request unconditionally.

**Unmatched-route branch in `toOpenApiPath`** (`rest/middleware/metricsHandler.js`, lines 13–31):
```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // undefined for unmatched routes

  if (!path) {
    if (!req.route) {                        // true for unmatched routes
      path = req.path;                       // ← raw attacker-controlled path
    } else { ... }
  }
  path = path.replace(/:([^/]+)/g, '{$1}');
  ...
  return path;
};
```

For any request that does not match a registered route:
- `res.locals[requestPathLabel]` is never set (the `recordRequestPath` middleware in `rest/routes/index.js` only runs inside matched sub-routers).
- `req.route` is `undefined`.
- Therefore `path = req.path` — the verbatim, unsanitized URL path supplied by the attacker.

**Label recording** (`rest/middleware/metricsHandler.js`, lines 185–210):
```js
res.on('finish', () => {
  const path = toOpenApiPath(req, res);   // unique per request for unmatched routes
  const labels = {method, path, code};

  requestTotalCounter.add(1, labels);
  durationHistogram.record(duration, labels);
  requestSizeHistogram.record(..., labels);
  responseSizeHistogram.record(responseSize, labels);
});
```
Each unique `{method, path, code}` triple creates a new time series in the `MeterProvider`'s in-memory accumulation map. There is no cardinality cap, no label allowlist, and no normalization for unmatched paths.

**Why existing checks are insufficient:**
- `responseHandler.js` (line 29–30) throws `NotFoundError` for unmatched routes, but this only affects the HTTP response — the `finish` event still fires and the metrics callback still executes with the raw path.
- The `recordRequestPath` middleware (`rest/routes/index.js`, lines 16–21) only runs inside matched sub-routers; it never fires for unmatched routes, so `res.locals[requestPathLabel]` remains unset.
- There is no rate-limiting, path-length cap, or cardinality guard anywhere in the metrics pipeline.

### Impact Explanation
Each unique path sent by the attacker creates four new time series (one per histogram/counter instrument) in the `MeterProvider`'s heap-allocated accumulation map. With sustained flooding of unique paths, heap memory grows monotonically. When the Node.js heap limit is reached, the process crashes with an OOM error, taking down the REST API entirely. This is a non-network-based DoS (memory exhaustion) requiring no authentication.

### Likelihood Explanation
The attack requires only the ability to send HTTP GET/POST requests to the public REST API endpoint (port 5551 by default). No credentials, tokens, or special protocol knowledge are needed. The attack is trivially scriptable: a simple loop generating random path suffixes (e.g., `GET /api/v1/<uuid>`) is sufficient. The attack is repeatable and can be sustained indefinitely. The metrics endpoint is enabled by default (`hiero.mirror.rest.metrics.enabled: true`).

### Recommendation
1. **Normalize unmatched-route paths to a fixed sentinel label** in `toOpenApiPath`: when `req.route` is `undefined` and `res.locals[requestPathLabel]` is unset, return a fixed string such as `"unknown"` or `"unmatched"` instead of `req.path`.
2. **Alternatively**, skip per-route metric recording entirely for unmatched routes (detect via `req.route === undefined` before calling `requestTotalCounter.add` / `durationHistogram.record`).
3. **Add a cardinality guard** in the MeterProvider configuration or via a custom View that drops or caps label values exceeding a known-good set.

### Proof of Concept

```bash
# Send 100,000 requests with unique paths to exhaust metrics cardinality
for i in $(seq 1 100000); do
  curl -s "http://<mirror-node-host>:5551/api/v1/$(cat /proc/sys/kernel/random/uuid)" &
done
wait
```

Each request:
1. Hits `metricsMiddleware` → `inFlightCounter.add(1)` → `next()`.
2. No route matches → `responseHandler` throws `NotFoundError` → `handleError` sends HTTP 404.
3. `res.on('finish')` fires → `toOpenApiPath` returns the unique UUID path → four new time series are created in the `MeterProvider` heap.
4. After sufficient iterations, Node.js heap is exhausted → process OOM-crashes → REST API is unavailable. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
