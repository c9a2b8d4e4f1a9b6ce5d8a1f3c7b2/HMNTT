### Title
Unbounded `X-Real-IP` Header Value Logged Without Validation Enables Log I/O Amplification DoS

### Summary
`TracingMiddleware` in `rosetta/app/middleware/trace.go` reads the `X-Real-IP` header value without any length validation and embeds it verbatim into a log message written via `log.Info()` for every non-internal path request. An unprivileged attacker who can reach port 5700 directly (bypassing the nginx proxy) can send concurrent requests with near-maximum-size `X-Real-IP` header values, causing the log sink to receive ~1 MB per request and driving disk write throughput and CPU consumption well above 30% of baseline.

### Finding Description
**Exact code path:**

In `rosetta/app/middleware/trace.go`, `getClientIpAddress()` (line 64) performs an unconstrained `r.Header.Get(xRealIpHeader)` — no length check, no format validation, no truncation:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)   // line 64 — raw, unbounded
    ...
    return ipAddress
}
```

The returned string is then embedded directly into the log message (lines 52–53) and written to the log sink for every non-internal path (line 58):

```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, ...)   // clientIpAddress can be up to ~1 MB

if internalPaths[path] {
    log.Debug(message)
} else {
    log.Info(message)       // line 58 — written for ALL non-internal paths
}
```

**Root cause:** The code assumes `X-Real-IP` contains a short IP address string. There is no length cap, no IP-format validation, and no truncation before the value is formatted into the log message.

**Why existing checks fail:**

1. Go's `http.Server` in `rosetta/main.go` (lines 220–227) does **not** set `MaxHeaderBytes`, so it defaults to Go's `http.DefaultMaxHeaderBytes` = 1,048,576 bytes (1 MB). This is the ceiling per header value, not a protection — it means each `X-Real-IP` value can be up to ~1 MB.
2. No rate limiting, no connection-count cap, and no request throttling middleware exists anywhere in the rosetta middleware chain (`MetricsMiddleware` → `TracingMiddleware` → `CorsMiddleware`).
3. The nginx proxy sets `X-Real-IP: $remote_addr` only for requests routed through it. Port 5700 is directly exposed (`ports: - 5700:5700` in `docker-compose.yml` line 151), so an attacker bypasses nginx entirely and controls the header freely.

### Impact Explanation
Each request to any non-internal path (e.g., `/network/list`, `/block`, `/construction/...`) with a ~1 MB `X-Real-IP` header causes `log.Info()` to write a ~1 MB log entry. With no concurrent-connection limit on the HTTP server, an attacker opening N connections and sending one request per `readHeaderTimeout` interval (default 3 s) generates approximately `N × 1 MB / 3 s` of sustained log I/O. At 100 concurrent connections this is ~33 MB/s of additional disk writes — a dramatic increase over the near-zero baseline log throughput of a lightly loaded node. CPU cost also rises due to `fmt.Sprintf` string allocation and repeated `write()` syscalls. The 30% threshold is easily exceeded even with a modest number of connections.

### Likelihood Explanation
The attack requires no credentials, no special protocol knowledge, and no prior access. Any network-reachable client can open TCP connections to port 5700 and send crafted HTTP requests. The exploit is trivially scriptable with `curl` or any HTTP client. It is repeatable and sustainable indefinitely. The only prerequisite is network reachability to port 5700, which is exposed by default in the reference deployment.

### Recommendation
1. **Truncate or validate the header value** in `getClientIpAddress()` before returning it — enforce a maximum length consistent with a valid IP address (e.g., 45 characters for IPv6):
   ```go
   const maxIpLength = 45
   if len(ipAddress) > maxIpLength {
       ipAddress = ipAddress[:maxIpLength]
   }
   ```
2. **Validate IP format** using `net.ParseIP()` and fall back to `r.RemoteAddr` if the header value is not a valid IP.
3. **Set `MaxHeaderBytes`** on the `http.Server` to a small value (e.g., 8 KB) to reject oversized header blocks before they reach any handler.
4. **Restrict direct external access** to port 5700; only the nginx proxy should be reachable from untrusted networks.

### Proof of Concept
```bash
# Generate a ~900 KB string to use as X-Real-IP
LARGE_IP=$(python3 -c "print('A' * 900000)")

# Send 100 concurrent requests directly to the Rosetta port, bypassing nginx
for i in $(seq 1 100); do
  curl -s -o /dev/null \
    -H "X-Real-IP: ${LARGE_IP}" \
    http://<rosetta-host>:5700/network/list \
    -X POST -H "Content-Type: application/json" \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' &
done
wait

# Observe disk write throughput spike (iostat -x 1) and log file growth rate
# Each request produces a ~900 KB log.Info() entry; 100 concurrent requests
# generate ~90 MB of log writes per round-trip, far exceeding the 30% threshold.
```
