### Title
Unbounded `X-Forwarded-For` Header Logged Without Length Validation in Rosetta Tracing Middleware

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` accepts the full, unsanitized value of the `X-Forwarded-For` header and embeds it verbatim into every log line. Because the Kubernetes/Traefik deployment is configured with `--entryPoints.web.forwardedHeaders.insecure`, an unauthenticated external client can inject an arbitrarily long comma-separated IP list that is passed through to the application unchanged, producing log entries up to Go's 1 MB per-header limit per request and permanently replacing the real source IP in the audit trail.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)          // line 64

    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader) // line 67 – no length cap
    }

    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }
    return ipAddress
}
```

The returned string is then formatted directly into the log message at lines 52–59:

```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, ...)   // clientIpAddress is the raw header value
log.Info(message)
```

**Root cause:** The only guard is `len(ipAddress) == 0` (an emptiness check), not a maximum-length check. Any non-empty `X-Forwarded-For` value — regardless of size or content — is accepted and logged.

**Why the proxy does not save you:**

- **Docker Compose / nginx** (`docker-compose.yml` line 204): nginx sets `proxy_set_header X-Real-IP $$remote_addr`, so `X-Real-IP` is always populated with the real TCP peer address. `getClientIpAddress()` checks `X-Real-IP` first (line 64), so `X-Forwarded-For` is never reached. This deployment is *not* vulnerable.
- **Kubernetes / Traefik** (`charts/hedera-mirror-common/values.yaml` lines 643–645): Traefik is configured with `--entryPoints.web.forwardedHeaders.insecure` for both HTTP and HTTPS entry points. This flag instructs Traefik to trust and forward all client-supplied `X-Forwarded-For` headers without stripping them. Traefik does **not** set `X-Real-IP` by default, so `X-Real-IP` arrives at the application empty. `getClientIpAddress()` therefore falls through to line 67 and uses the attacker-controlled `X-Forwarded-For` value verbatim. ### Impact Explanation
1. **Identity erasure in audit logs:** The real TCP source address (`r.RemoteAddr`) is never logged when `X-Forwarded-For` is non-empty. An attacker submitting malicious Rosetta `/construction/submit` calls (transaction gossip) can make every log line attribute the request to an arbitrary fake IP, defeating post-incident forensics.
2. **Log volume amplification:** Go's `net/http` default `MaxHeaderBytes` is 1 MB. A single request with a ~1 MB `X-Forwarded-For` value produces a ~1 MB log line. At modest request rates (e.g., 100 req/s) this generates ~100 MB/s of log data, exhausting disk or log-shipping quotas and potentially causing log loss for surrounding legitimate entries — including entries that would reveal a concurrent gossip-suppression or transaction-replay attack. [4](#0-3) 

### Likelihood Explanation
- **No authentication required.** The Rosetta API is a public HTTP endpoint; any network-reachable client can send headers.
- **Traefik `insecure` mode is the default Kubernetes configuration** in this repository, not an optional hardening bypass.
- **Trivially repeatable** with a single `curl` command; no special tooling or privileges needed.
- The attack is silent — it produces no error response to the client and no server-side alarm. [5](#0-4) 

### Recommendation
1. **Validate and truncate the header value** in `getClientIpAddress()` before logging. Accept only the first token (the leftmost IP, which is the client-claimed address) and enforce a maximum length (e.g., 45 characters covers the longest valid IPv6 address):

```go
func getClientIpAddress(r *http.Request) string {
    const maxIPLen = 45
    ipAddress := r.Header.Get(xRealIpHeader)

    if len(ipAddress) == 0 {
        xff := r.Header.Get(xForwardedForHeader)
        // Take only the first (leftmost) entry
        if idx := strings.IndexByte(xff, ','); idx != -1 {
            xff = xff[:idx]
        }
        ipAddress = strings.TrimSpace(xff)
    }

    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    if len(ipAddress) > maxIPLen {
        ipAddress = ipAddress[:maxIPLen]
    }
    return ipAddress
}
```

2. **Remove `--entryPoints.web.forwardedHeaders.insecure`** from the Traefik configuration and replace it with an explicit trusted-IP allowlist (`--entryPoints.web.forwardedHeaders.trustedIPs=<cluster-CIDR>`). This ensures only headers injected by known proxies are trusted.

3. **Configure Traefik to set `X-Real-IP`** (via a middleware) so the application always has the verified peer address available, mirroring the nginx behavior in the docker-compose deployment. [6](#0-5) [5](#0-4) 

### Proof of Concept

**Prerequisites:** Kubernetes deployment with default Traefik configuration (i.e., `--entryPoints.web.forwardedHeaders.insecure` active). Rosetta endpoint reachable at `$ROSETTA_URL`.

```bash
# Build a ~500 KB X-Forwarded-For value (thousands of fake IPs)
FAKE_IPS=$(python3 -c "
import random
ips = [f'{random.randint(1,254)}.{random.randint(0,255)}.{random.randint(0,255)}.{random.randint(1,254)}' for _ in range(30000)]
print(', '.join(ips))
")

# Send a valid Rosetta network/list request with the oversized header
curl -s -X POST "$ROSETTA_URL/network/list" \
  -H "Content-Type: application/json" \
  -H "X-Forwarded-For: $FAKE_IPS" \
  -d '{}'
```

**Expected result:** The application returns a normal JSON response (HTTP 200). Inspecting the application logs shows a single log line of ~500 KB containing thousands of fake IP addresses instead of the real client IP. Repeating at 10 req/s generates ~5 MB/s of log data from a single client, with no real source IP ever recorded.
