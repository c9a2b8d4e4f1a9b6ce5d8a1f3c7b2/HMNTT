### Title
Unauthenticated CPU Amplification DoS via `estimate=true` Binary Search in `ContractController.call()`

### Summary
Any unauthenticated caller can send a POST to `/api/v1/contracts/call` with `estimate=true` and `gas=15000000` targeting any address. The throttle gate consumes gas tokens exactly once per request, but the estimate path executes up to 21 full EVM invocations (1 initial + 20 binary-search iterations). The throttle never accounts for the 20× CPU multiplier, allowing a small number of concurrent requests to saturate server CPU.

### Finding Description

**Exact code path:**

1. `ContractController.call()` (line 40) calls `throttleManager.throttle(request)` — consumes gas tokens once for the declared `gas` value. [1](#0-0) 

2. `ContractExecutionService.processCall()` → `estimateGas()` (line 82) calls `callContract()` — one full EVM execution. [2](#0-1) 

3. If `gasUsedByInitialCall < params.getGas()` (line 86), `binaryGasEstimator.search()` is invoked with `lo=gasUsedByInitialCall`, `hi=15_000_000`. [3](#0-2) 

4. `BinaryGasEstimator.search()` loops up to `maxGasEstimateRetriesCount=20` times, calling `doProcessCall()` each iteration. [4](#0-3) 

5. Inside `doProcessCall()`, when `estimate=true`, `restoreGasToBucket` is **never called** (line 128 guard), so the throttle is never updated for any of the 20 binary-search EVM executions. [5](#0-4) 

**Root cause — failed assumption:** The throttle model assumes one gas-token deduction equals one EVM execution. For `estimate=true`, the actual execution count is `1 + iterations` (up to 21), but the token bucket is only debited once.

**Non-existent contract amplifies worst-case:** When the target address has no code, every `safeCall()` returns `null` (exception caught silently at line 70), setting `err=true` and `lo=mid` every iteration. [6](#0-5) 

Because `lo` only ever moves toward `hi` and `hi` never shrinks, the loop runs all 20 iterations. Additionally, `estimateIterationThreshold = (long)(lo * 0.10)` where `lo` starts at 0 (no gas used by a non-existent contract call), making the threshold 0 — the early-exit condition `Math.abs(prevGasLimit - mid) < 0` is mathematically impossible, guaranteeing all 20 iterations execute. [7](#0-6) 

### Impact Explanation
Each crafted request causes up to 21 full EVM setup/execute/teardown cycles while consuming only 1 request-worth of throttle tokens. A small number of concurrent connections (e.g., 10–50) can pin all available CPU threads, making the mirror-node web3 API unresponsive to all users. Because the endpoint is public and unauthenticated, this constitutes a complete denial of service of the ETH-compatible API surface with no special privileges required.

### Likelihood Explanation
The attack requires only an HTTP client and knowledge of the public API (documented). No wallet, token, or account is needed. The payload is a single JSON body: `{"to":"0x000...001","gas":15000000,"estimate":true}`. It is trivially scriptable, repeatable, and can be distributed across many source IPs to defeat any IP-level rate limiting that may exist at the network edge.

### Recommendation
1. **Charge throttle tokens proportionally for estimate calls.** Before entering `binaryGasEstimator.search()`, pre-deduct `gas × maxGasEstimateRetriesCount` tokens (or a configurable multiplier), and restore unused tokens after the search completes.
2. **Cap binary-search iterations based on remaining throttle budget**, not a fixed constant, so a depleted bucket aborts the search early.
3. **Validate contract existence before entering the estimate path.** If the target address has no deployed code and no value is being sent, return an error immediately rather than running the binary search.
4. **Apply a per-IP or per-connection concurrency limit** specifically for `estimate=true` requests at the servlet/filter layer, independent of the gas-token throttle.

### Proof of Concept

```bash
# Send 50 concurrent estimate requests targeting a non-existent address
for i in $(seq 1 50); do
  curl -s -X POST https://<mirror-node-host>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{"to":"0x00000000000000000000000000000000000DEAD1",
         "gas":15000000,
         "estimate":true,
         "data":"0x"}' &
done
wait
```

Each request triggers:
- 1 initial `callContract()` EVM execution (non-existent address → gasUsed=0)
- 20 binary-search `doProcessCall()` EVM executions (all fail, `err=true`, loop runs to limit)
- Total: **21 EVM executions per request**, throttle debited for **1**
- 50 concurrent requests → **1,050 simultaneous EVM executions** against a throttle budget sized for 50
