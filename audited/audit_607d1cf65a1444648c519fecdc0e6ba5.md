### Title
Prometheus Label Cardinality Injection via Unmatched Routes in `metricsMiddleware`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to `req.path` for requests that match no Express route and have no `res.locals[requestPathLabel]` set. Because `req.path` is attacker-controlled and the only guard is a `startsWith(apiPrefix)` check that passes for any path beginning with `/api/v1`, an unprivileged external user can inject an unbounded number of unique label values into `requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, and `responseSizeHistogram`. This causes Prometheus cardinality explosion, exhausting in-process memory in the OpenTelemetry SDK and potentially crashing the mirror node.

### Finding Description

**Exact code path:**

`rest/middleware/metricsHandler.js`, `toOpenApiPath()`, lines 13–31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // (1) not set for unmatched routes

  if (!path) {
    if (!req.route) {                        // (2) undefined for unmatched routes
      path = req.path;                       // (3) attacker-controlled value
    } else { ... }
  }

  path = path.replace(/:([^/]+)/g, '{$1}'); // (4) no-op if no :params

  if (!path.startsWith(apiPrefix)) {        // (5) passes for /api/v1/* — no prepend
    return apiPrefix + '/' + path;
  }

  return path;                              // (6) returned verbatim
};
```

Then at lines 207–210:
```js
requestTotalCounter.add(1, labels);         // labels.path = attacker string
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
```

**Root cause / failed assumption:** The code assumes that any path already prefixed with `/api/v1` must be a known, normalized route pattern. This is false for 404 requests: `res.locals[requestPathLabel]` is only written by matched route handlers (confirmed in `rest/middleware/responseHandler.js` line 33 and `rest/routes/index.js`), and `req.route` is `undefined` for unmatched routes. The `startsWith(apiPrefix)` guard at line 26 was designed to *add* the prefix when missing, but it simultaneously *skips all normalization* for any path that already carries it — including arbitrary attacker-supplied paths.

**Why existing checks fail:**
- `res.locals[requestPathLabel]` is never populated for unmatched routes (`responseHandler.js` line 28–30 throws `NotFoundError` before setting it).
- `req.route` is `undefined` for unmatched routes, so the safe branch (`req.baseUrl + req.route.path`) is never taken.
- The metrics endpoint itself is protected by `authenticate()` (line 157), but metric *recording* at lines 207–210 runs for every request with no path validation whatsoever. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation

Each unique `path` label value creates four new in-process time series (one per instrument). The OpenTelemetry SDK accumulates all active label sets in memory with no eviction. An attacker sending `N` requests with distinct paths like `/api/v1/a1`, `/api/v1/a2`, … `/api/v1/aN` creates `4N` time series. At scale this exhausts heap memory, causing the mirror node process to OOM-crash — a complete denial of service of the Hashgraph history REST API. Even at moderate volume it degrades Prometheus scrape performance and corrupts dashboards/alerts by flooding them with phantom endpoints. [4](#0-3) 

### Likelihood Explanation

No authentication, API key, or rate-limiting is required to trigger this. Any HTTP client reachable to the mirror node's REST port can send GET requests to `/api/v1/<random_string>` in a tight loop. The attack is trivially scriptable (`curl`, `ab`, `wrk`), fully repeatable, and requires zero knowledge of the application internals beyond the public `/api/v1` prefix documented in the OpenAPI spec. The 404 response returned to the attacker is irrelevant — the damage is done in the `res.on('finish')` callback regardless of status code. [5](#0-4) 

### Recommendation

Replace the raw `req.path` fallback with a static sentinel label for unrecognized routes instead of the actual path:

```js
if (!req.route) {
  path = 'unknown';   // or a fixed string like '/api/v1/unknown'
} else {
  path = (req.baseUrl ?? '') + req.route?.path;
}
```

Alternatively, maintain an allowlist of known OpenAPI path patterns and map any unrecognized path to a fixed `unknown` bucket before recording metrics. This eliminates unbounded label cardinality while preserving useful per-route instrumentation for legitimate endpoints. [6](#0-5) 

### Proof of Concept

```bash
# Send 10,000 requests with unique paths, each starting with /api/v1
for i in $(seq 1 10000); do
  curl -s "http://<mirror-node-host>:<port>/api/v1/fake-$(openssl rand -hex 8)" &
done
wait

# Scrape metrics and observe unbounded unique path label values
curl http://<mirror-node-host>:<port>/swagger/metrics/ | grep api_request_total | wc -l
# Returns ~10000+ unique time series instead of the ~30 legitimate endpoints
```

Each iteration: `req.route` is `undefined` → `path = req.path` = `/api/v1/fake-<random>` → `startsWith('/api/v1')` is `true` → returned verbatim → recorded as a unique label in all four per-route instruments → new time series allocated in heap. [7](#0-6) [8](#0-7)
