### Title
Unauthenticated Historical State DoS via Max-Gas Contract Calls with No Per-Historical-Request Throttle Differentiation

### Summary
An unprivileged user can submit `POST /api/v1/contracts/call` requests with `gas = evmProperties.getMaxGasLimit()` (15,000,000 by default) and a historical `block` parameter. The `validateContractMaxGasLimit()` check only enforces `gas <= maxGasLimit`, and the gas throttle is calibrated purely against EVM computation cost — not against the disproportionately higher database I/O cost of historical state reads. At the permitted rate limit (500 RPS), an attacker can drive thousands of timestamp-filtered UNION queries per second against `entity_history`, `contract_state_history`, and related tables, exhausting DB connection pool resources and degrading the historical state service for all users.

### Finding Description

**Code path:**

`ContractController.call()` (lines 38–51) calls `validateContractMaxGasLimit()` (lines 92–97), which only rejects `gas > maxGasLimit`:

```java
if (request.getGas() > evmProperties.getMaxGasLimit()) {
    throw new InvalidParametersException(...);
}
```

`gas == maxGasLimit` (15,000,000) passes. The `block` field defaults to `BlockType.LATEST` but accepts any historical block number via `BlockTypeDeserializer`. [1](#0-0) 

The throttle in `ThrottleManagerImpl.throttle()` consumes `scaleGas(request.getGas())` tokens from the gas bucket:

```java
gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))
``` [2](#0-1) 

`scaleGas` divides by `GAS_SCALE_FACTOR = 10,000`, so `scaleGas(15,000,000) = 1,500` tokens. The bucket capacity is `scaleGas(7,500,000,000) = 750,000` tokens/sec. After execution, `restoreGasToBucket` returns `min(gasRemaining, gasLimitToRestoreBaseline)` to the bucket: [3](#0-2) 

With `gasLimitRefundPercent = 100` (default), the net throttle cost equals actual EVM gas used. For a call consuming 14.7M gas (7,000 cold `SLOAD`s), net bucket cost ≈ 1,470 tokens, allowing ~510 such requests/second — bounded by the 500 RPS limit.

**Root cause:** The gas throttle is calibrated for EVM opcode computation cost. Historical state reads trigger expensive timestamp-filtered UNION queries against `entity_history`, `contract_state_history`, etc. for every `SLOAD`, account lookup, and token read — but these DB I/O costs are not reflected in EVM gas. There is no separate throttle, rate limit, or additional validation for historical requests. [4](#0-3) 

`ContractCallContext.useHistorical()` returns `true` whenever `block != LATEST`, routing all state reads through the historical (UNION) query path: [5](#0-4) 

### Impact Explanation

At 500 RPS with 7,000 cold `SLOAD`s per request, the attacker drives ~3,500,000 historical DB queries per second. Each query is a `UNION ALL` across `entity`/`entity_history` (or equivalent) with timestamp filtering. This exhausts the DB connection pool and statement execution capacity, causing query timeouts (`db.statementTimeout = 3,000ms`) and request timeouts (`requestTimeout = 10,000ms`) for all concurrent users. The historical Hashgraph state service becomes unavailable or severely degraded. Data integrity is not modified (reads only), but availability of the historical state service is destroyed. [6](#0-5) 

### Likelihood Explanation

No authentication, API key, or account is required. The endpoint is publicly accessible. The attack requires only an HTTP client capable of 500 RPS — trivially achievable from a single machine or small botnet. The `block` parameter accepts any valid historical block number (decimal or hex), and the `gas` field defaults to `15,000,000` in `ContractCallRequest`, meaning even default requests qualify. The attack is repeatable and sustainable indefinitely. [7](#0-6) 

### Recommendation

1. **Separate throttle for historical requests**: Add a dedicated `historicalRequestsPerSecond` rate limit in `ThrottleProperties` and enforce it in `ThrottleManagerImpl.throttle()` when `request.getBlock() != BlockType.LATEST`.
2. **Reduce effective gas limit for historical calls**: Apply a lower `maxGasLimit` for historical block requests (e.g., configurable `maxGasLimitHistorical`), checked in `validateContractMaxGasLimit()`.
3. **DB-level query cost accounting**: Instrument historical query execution time and feed it back into the throttle bucket rather than relying solely on EVM gas.
4. **Connection pool isolation**: Use a separate DB connection pool for historical queries to prevent historical DoS from starving latest-state queries.

### Proof of Concept

```bash
# Attacker sends 500 RPS, each with max gas and a historical block,
# calling a contract that performs many cold SLOADs (e.g., reads 7000 storage slots)

for i in $(seq 1 500); do
  curl -s -X POST https://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{
      "block": "0x1",
      "gas": 15000000,
      "to": "<storage-heavy-contract>",
      "data": "<calldata-triggering-7000-SLOADs>",
      "estimate": false
    }' &
done
wait
# Repeat in a tight loop. DB connection pool exhausts within seconds.
# Legitimate users querying historical state receive timeouts or errors.
```

Preconditions: Public network access to the mirror node web3 API. No credentials needed.
Trigger: 500 concurrent max-gas historical `eth_call` requests per second.
Result: DB connection pool exhaustion, `db.statementTimeout` (3s) violations, service degradation for all historical state queries.

    private void restoreGasToBucket(EvmTransactionResult result, long gasLimit) {
        // If the transaction fails, gasUsed is equal to gasLimit, so restore the configured refund percent
        // of the gasLimit value back in the bucket.
        final var gasLimitToRestoreBaseline = (long) (gasLimit * throttleProperties.getGasLimitRefundPercent() / 100f);
        if (result == null || (!result.isSuccessful() && gasLimit == result.gasUsed())) {
            throttleManager.restore(gasLimitToRestoreBaseline);
        } else {
            // The transaction was successful or reverted, so restore the remaining gas back in the bucket or
            // the configured refund percent of the gasLimit value back in the bucket - whichever is lower.
            final var gasRemaining = gasLimit - result.gasUsed();
            throttleManager.restore(Math.min(gasRemaining, gasLimitToRestoreBaseline));
        }
    }
```
