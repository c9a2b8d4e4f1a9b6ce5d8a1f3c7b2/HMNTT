### Title
Null Dereference (NPE) in `ContractBytecodeReadableKVState` via Crafted Long-Zero EVM Address with Out-of-Range `num`

### Summary
`DomainUtils.fromEvmAddress()` silently returns `null` when the 8-byte `num` field of a long-zero EVM address exceeds `EntityId.NUM_MASK` (274,877,906,943), catching the resulting `InvalidEntityException` only at debug level. `ContractBytecodeReadableKVState.toEntityId()` propagates this `null` without a guard, and `readFromDataSource()` immediately dereferences it via `entityId.getId()`, producing an unhandled `NullPointerException`. Any unprivileged caller of the web3 API can trigger this deterministically.

### Finding Description

**Step 1 — Silent null production (`DomainUtils.java` lines 285–297)**

```java
public static EntityId fromEvmAddress(byte[] evmAddress) {
    final var commonProperties = CommonProperties.getInstance();
    try {
        if (isLongZeroAddress(evmAddress)) {
            final var num = Longs.fromByteArray(Arrays.copyOfRange(evmAddress, 12, 20));
            return EntityId.of(commonProperties.getShard(), commonProperties.getRealm(), num);
        }
    } catch (InvalidEntityException ex) {
        log.debug("Failed to parse long zero evm address into EntityId", ex);  // silent!
    }
    return null;
}
```

`isLongZeroAddress()` only verifies that bytes 0–11 are zero; it places no constraint on bytes 12–19. `Longs.fromByteArray()` interprets those 8 bytes as a signed 64-bit integer, so any value > `NUM_MASK` (274,877,906,943) or any negative value causes `EntityId.encode()` to throw `InvalidEntityException`. That exception is swallowed at `DEBUG` level and `null` is returned. [1](#0-0) [2](#0-1) 

**Step 2 — Null propagated without guard (`ContractBytecodeReadableKVState.java` lines 49–64)**

```java
private EntityId toEntityId(@NonNull final ContractID contractID) {
    ...
    } else if (contractID.hasEvmAddress()) {
        final var evmAddress = contractID.evmAddress().toByteArray();
        if (isLongZeroAddress(evmAddress)) {
            return DomainUtils.fromEvmAddress(evmAddress);  // returns null — no check
        }
        ...
    }
    return EntityId.EMPTY;
}
```

When `isLongZeroAddress()` is true the method returns whatever `fromEvmAddress()` returns, including `null`. There is no fallback to `EntityId.EMPTY` or any null guard. [3](#0-2) 

**Step 3 — Null dereference (`ContractBytecodeReadableKVState.java` lines 39–47)**

```java
protected Bytecode readFromDataSource(@NonNull ContractID contractID) {
    final var entityId = toEntityId(contractID);          // null
    return contractRepository
            .findRuntimeBytecode(entityId.getId())        // NullPointerException
            ...
}
```

`entityId` is `null`; calling `.getId()` on it throws `NullPointerException`. [4](#0-3) 

**Why existing checks are insufficient**

`isLongZeroAddress()` only validates that the first 12 bytes are zero; it does not validate that the remaining 8 bytes encode a value within `[0, NUM_MASK]`. The `InvalidEntityException` guard in `fromEvmAddress()` is a catch-and-suppress pattern that converts a hard error into a silent `null`, and no caller in this path checks for that `null` before dereferencing. [5](#0-4) 

### Impact Explanation

The NPE is thrown inside the mirror node's web3 service during contract bytecode lookup (e.g., `eth_getCode`, `eth_call` targeting a contract). Depending on the framework's exception handling, this either returns a 500 error for every such request or, if the exception escapes a shared handler thread, can destabilize the web3 service. The Hedera consensus network itself is unaffected (the mirror node is read-only infrastructure), but the web3 API endpoint becomes unreliable or unavailable for any contract address matching the crafted pattern. This is a targeted, repeatable denial-of-service against the mirror node's web3 layer.

### Likelihood Explanation

The precondition is trivially met: the attacker needs only the ability to send an HTTP request to the mirror node's public web3 endpoint — no authentication, no privileged account, no on-chain transaction required. The crafted address (12 zero bytes + 8 bytes encoding a value > 274,877,906,943, e.g., `0x000000000000000000000000ffffffffffffffff`) is a single static input. The attack is deterministic and repeatable at zero cost.

### Recommendation

1. **In `ContractBytecodeReadableKVState.toEntityId()`**: replace the bare `return DomainUtils.fromEvmAddress(evmAddress)` with a null-safe fallback:
   ```java
   var id = DomainUtils.fromEvmAddress(evmAddress);
   return id != null ? id : EntityId.EMPTY;
   ```
2. **In `DomainUtils.fromEvmAddress()`**: validate that `num` is within `[0, NUM_MASK]` before calling `EntityId.of()`, or promote the log level from `DEBUG` to `WARN` so the silent suppression is at least observable.
3. **Audit all other callers** of `fromEvmAddress()` / `fromTrimmedEvmAddress()` that do not check for `null` before dereferencing (e.g., `EvmTokenUtils.entityIdFromEvmAddress()` which returns `null` directly to callers).

### Proof of Concept

```
# Craft a 20-byte long-zero EVM address where bytes 12-19 = 0xFFFFFFFFFFFFFFFF
# (decoded as -1L, which is < 0, triggering InvalidEntityException in EntityId.encode())
# Hex: 0x000000000000000000000000ffffffffffffffff

curl -X POST https://<mirror-node>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{
    "data": "0x",
    "to": "0x000000000000000000000000ffffffffffffffff"
  }'

# Expected result: NullPointerException in ContractBytecodeReadableKVState.readFromDataSource()
# at line: contractRepository.findRuntimeBytecode(entityId.getId())
# entityId is null because fromEvmAddress() caught InvalidEntityException silently
# and returned null, which toEntityId() propagated without a null guard.
```
