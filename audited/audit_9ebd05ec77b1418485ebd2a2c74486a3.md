### Title
Unbounded Prometheus Label Cardinality via Arbitrary HTTP Method Strings in `MetricsMiddleware`

### Summary
`MetricsMiddleware` in `rosetta/app/middleware/metrics.go` passes the raw HTTP request method (`r.Method`) directly as a Prometheus label to all four metric vectors, including `requestInflightGauge`. Go's `net/http` server does not validate or restrict HTTP method strings, so any unprivileged attacker can flood the server with requests bearing arbitrary method tokens, creating an unbounded number of unique label combinations in the Prometheus registry and exhausting server memory.

### Finding Description
In `rosetta/app/middleware/metrics.go`, four `prometheus.*Vec` metrics are declared with a `"method"` label:

```go
// lines 34–37
requestInflightGauge = prometheus.NewGaugeVec(prometheus.GaugeOpts{
    Name: "hiero_mirror_rosetta_request_inflight",
    Help: "Current number of inflight HTTP requests.",
}, []string{"method", "route"})
```

`MetricsMiddleware` (lines 76–84) delegates to `middleware.Instrument{}.Wrap(next)` from `github.com/weaveworks/common/middleware`. That library reads `r.Method` verbatim and calls `requestInflightGauge.With(prometheus.Labels{"method": r.Method, "route": ...})` for every incoming request, creating a new in-memory time series for each unique `(method, route)` pair.

In `main.go` the middleware chain is assembled as:

```go
// lines 217–219
metricsMiddleware := middleware.MetricsMiddleware(router)
tracingMiddleware := middleware.TracingMiddleware(metricsMiddleware)
corsMiddleware    := server.CorsMiddleware(tracingMiddleware)
```

Requests arrive at `corsMiddleware` first, then `tracingMiddleware`, then `metricsMiddleware`. The CORS middleware from `rosetta-sdk-go` adds CORS headers but does **not** reject requests based on HTTP method for non-preflight traffic; it passes all methods through. Go's `net/http` server similarly imposes no method allowlist — it parses whatever token appears in the request line and stores it verbatim in `r.Method`.

There is no rate limiting, no method normalization, and no allowlist anywhere in the rosetta Go codebase (confirmed by grep across `rosetta/**/*.go`).

### Impact Explanation
Each unique method string creates a new label combination in all four `*Vec` registries (`requestInflightGauge`, `requestDurationHistogram`, `requestBytesHistogram`, `responseBytesHistogram`). Prometheus stores these as in-process Go maps that grow without bound. Sending N distinct method strings produces O(N) new map entries across all four vectors simultaneously. At sufficient scale this exhausts heap memory, causing OOM termination of the server process. Even before OOM, the `/metrics` scrape output becomes arbitrarily large, degrading monitoring accuracy for all observers and potentially causing scrape timeouts in Prometheus.

### Likelihood Explanation
No authentication or network-level restriction is required. Any client that can reach the server's HTTP port can send a request with an arbitrary method string using standard tools (`curl -X FOOBAR123 http://target/`). The attack is trivially scriptable in a loop with unique method strings per iteration, requires no special knowledge of the application, and is repeatable indefinitely. The Rosetta API is designed to be publicly reachable, making this a realistic external threat.

### Recommendation
Normalize or allowlist the HTTP method before it is used as a Prometheus label. Options:

1. **Allowlist known methods**: Before passing to `middleware.Instrument`, wrap the handler to replace `r.Method` with `"unknown"` (or `"other"`) if it is not in `{GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS}`.
2. **Use a sanitizing wrapper**: Introduce a thin `http.Handler` that rewrites `r.Method` on the request before it reaches `MetricsMiddleware`.
3. **Upstream fix**: If the weaveworks library supports a method-normalization hook, configure it there.

The fix must be applied **before** `MetricsMiddleware` in the chain, i.e., as the outermost layer, so that the label value is already sanitized when `middleware.Instrument` reads `r.Method`.

### Proof of Concept
```bash
# Send 10,000 requests each with a unique arbitrary HTTP method
for i in $(seq 1 10000); do
  curl -s -o /dev/null -X "METHOD${i}" http://<rosetta-host>:<port>/network/list &
done
wait

# Observe unbounded label growth in the metrics output
curl http://<rosetta-host>:<port>/metrics | grep hiero_mirror_rosetta_request_inflight | wc -l
# Returns ~10,000 distinct time series instead of the expected handful
```

Each iteration creates a new `(METHOD${i}, /network/list)` label pair in all four Prometheus vectors. Memory consumption of the rosetta process grows linearly with the number of unique method strings sent. [1](#0-0) [2](#0-1) [3](#0-2)
