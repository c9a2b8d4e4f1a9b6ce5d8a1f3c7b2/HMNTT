### Title
Unvalidated `X-Real-IP` Header Logged Verbatim Enables Log Injection / Parser Confusion

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` accepts the `X-Real-IP` header value without any IP format validation and passes it directly into the log message via `fmt.Sprintf`. Because port 5700 is directly exposed in the default `docker-compose.yml`, an unprivileged attacker can bypass the nginx proxy (which would otherwise overwrite the header) and supply an arbitrary string that is written verbatim to the application log.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/trace.go`, `getClientIpAddress()` (lines 63–74):
```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)   // line 64 – no validation
    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader)
    }
    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }
    return ipAddress
}
```

The returned string is immediately interpolated into the log line in `TracingMiddleware` (lines 52–53):
```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
```

**Root cause:** The function performs only a length check (`len(ipAddress) == 0`). There is no call to `net.ParseIP()` or any regex/allowlist check. Any non-empty string supplied in the header is accepted and logged.

**Why the nginx mitigation is insufficient:** The nginx reverse proxy sets `proxy_set_header X-Real-IP $remote_addr` (`docker-compose.yml` line 204), which would overwrite a client-supplied value — but only for requests that transit nginx. `docker-compose.yml` line 151 exposes `5700:5700` directly on the host, so an attacker can connect to port 5700 directly, bypassing nginx entirely, and supply an arbitrary `X-Real-IP` value.

**Exploit flow:**
1. Attacker sends `GET / HTTP/1.1` directly to `<host>:5700` with header `X-Real-IP: not-an-ip-address` (or a structured payload such as `1.1.1.1 GET /admin (200) in 0s`).
2. `getClientIpAddress` returns the raw string unchanged.
3. `TracingMiddleware` writes `not-an-ip-address GET / (200) in Xms` to the log at `INFO` level.

### Impact Explanation
- **Log parser confusion / SIEM bypass:** Any downstream system (Splunk, ELK, Datadog, etc.) that tokenises the first field of the log line as an IP address will fail to parse or will silently misclassify the entry, potentially dropping it from security dashboards.
- **Log spoofing:** A crafted value such as `1.2.3.4 GET /admin (200) in 0s` produces a syntactically valid-looking but entirely fabricated log record, allowing an attacker to plant false audit evidence or mask their real activity.
- **Severity:** Medium — no direct code execution or data exfiltration, but integrity of the audit trail is compromised and monitoring tooling can be blinded.

### Likelihood Explanation
Port 5700 is exposed by default in the shipped `docker-compose.yml`. No authentication is required to reach the rosetta HTTP endpoint. The attack requires only a single HTTP request with a crafted header — trivially scriptable and repeatable. Any user who can reach the host network can exploit this.

### Recommendation
Validate the extracted value with `net.ParseIP()` before using it; fall back to `r.RemoteAddr` if the header value is not a valid IP:

```go
func getClientIpAddress(r *http.Request) string {
    for _, h := range []string{xRealIpHeader, xForwardedForHeader} {
        val := r.Header.Get(h)
        if val != "" {
            // take only the first token (handles "client, proxy1" XFF lists)
            val = strings.SplitN(val, ",", 2)[0]
            val = strings.TrimSpace(val)
            if ip := net.ParseIP(val); ip != nil {
                return ip.String()
            }
        }
    }
    ipAddress, _, _ := net.SplitHostPort(r.RemoteAddr)
    return ipAddress
}
```

Additionally, consider removing the direct host-port binding for `5700` in production deployments so that all traffic must transit the nginx proxy.

### Proof of Concept
```bash
# Direct connection to rosetta port, bypassing nginx
curl -s -o /dev/null \
  -H "X-Real-IP: not-an-ip-address" \
  http://<host>:5700/network/list

# Observe in rosetta container logs:
# time="..." level=info msg="not-an-ip-address POST /network/list (200) in 1.2ms"
```

A log-spoofing variant:
```bash
curl -s -o /dev/null \
  -H $'X-Real-IP: 10.0.0.1 GET /admin (200) in 0s\ntime="2024-01-01" level=info msg="fake entry"' \
  http://<host>:5700/network/list
```
(Go's `net/http` canonicalises headers and strips bare `\n`, so full newline injection is blocked, but the first-field spoofing in the single log line is fully reproducible.) [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
