### Title
Unbounded Recursive Key Traversal in `mapKey()` Enables CPU/Memory Exhaustion via `key: Object` Field

### Summary
The `mapKey()` method in `CommonMapper.java` recursively traverses protobuf `ThresholdKey`/`KeyList` structures with no depth limit, building an arbitrarily deep nested Java `Map` in heap memory. The GraphQL-level `MaxQueryDepthInstrumentation(10)` guard measures only the depth of the *query document* field-selection tree, not the depth of data returned by a scalar field — so `key: Object` at query depth 1 can return a map of unbounded depth. Any unauthenticated caller who knows (or discovers) an account with a complex multi-level threshold key can trigger disproportionate CPU and heap allocation with a single minimal query.

### Finding Description

**Exact code path**

`CommonMapper.java` lines 95–126: [1](#0-0) 

`mapKey(Key)` calls `mapKeyList()` for `KEYLIST` and `THRESHOLDKEY` cases; `mapKeyList()` iterates every child key and calls `mapKey()` again — mutual recursion with no depth counter, no visited-set, and no size cap.

**Root cause**

The developers placed three guards in `GraphQlConfiguration.java`: [2](#0-1) 

`MaxQueryDepthInstrumentation(10)` and `MaxQueryComplexityInstrumentation(200)` operate on the *parsed GraphQL document* (field-selection nesting). They are evaluated before any resolver runs. A query that selects `key` at depth 1 passes both checks regardless of how many recursive `Map` objects `mapKey()` will allocate at runtime.

The Jackson `StreamWriteConstraints.maxNestingDepth(100)`: [3](#0-2) 

only fires during JSON *serialization*, after the entire nested `Map` tree has already been constructed in heap. It also permits 100 levels of nesting, which is generous.

**Exploit flow**

1. `account(input: {entityId: {num: X}}) { key }` is submitted — a 1-field query, query depth = 2, complexity = 2, well within all document-level limits.
2. `AccountController.account()` resolves the entity from the DB.
3. `AccountMapper` calls `CommonMapper.mapKey(byte[])` on the stored protobuf key bytes.
4. `mapKey` → `mapKeyList` → `mapKey` recurses for every node in the key tree. For a threshold key of depth D with branching factor B, this allocates O(B^D) `Map`/`ArrayList` objects before returning.
5. The resulting nested `Map` is handed to `ExtendedScalars.Object` for serialization — no depth check there either. [4](#0-3) 

**Why existing checks fail**

| Guard | What it measures | Does it bound `mapKey()` recursion? |
|---|---|---|
| `MaxQueryDepthInstrumentation(10)` | GraphQL document field depth | No — scalar `key` counts as depth 1 |
| `MaxQueryComplexityInstrumentation(200)` | GraphQL document field count | No — `key` counts as 1 field |
| `StreamWriteConstraints.maxNestingDepth(100)` | JSON write nesting during serialization | No — heap already allocated; also 100 is permissive |
| Parser limits (`maxTokens`, `maxCharacters`) | Query document tokens | No — query is tiny |

### Impact Explanation

The Hedera network permits `KeyList` entries containing nested `ThresholdKey`/`KeyList` structures. A realistic worst-case key (e.g., a threshold-of-N key whose children are themselves threshold keys, 3–4 levels deep with 10–20 keys per level) produces hundreds to thousands of `Map` and `ArrayList` allocations per request. Each allocation is proportional to the key tree size, not to the query size. Repeated requests against such an account — or concurrent requests from multiple source IPs (the `inFlightReq` middleware limits 5 concurrent per IP, not across IPs) — can drive heap pressure and GC pauses, degrading or crashing the GraphQL service for all users.

### Likelihood Explanation

No authentication is required. The GraphQL endpoint is publicly accessible. Accounts with complex multi-level threshold keys exist on Hedera mainnet (used for governance and multi-party custody). An attacker can enumerate account IDs (they are sequential integers) to find one with a complex key, or create such an account on testnet to verify the behavior, then replay the same single-field query against mainnet. The query is trivially small and passes all document-level validation.

### Recommendation

Add a depth parameter to `mapKey()` and enforce a hard cap (e.g., 10):

```java
default Object mapKey(Key key) {
    return mapKey(key, 0);
}

default Object mapKey(Key key, int depth) {
    if (depth > MAX_KEY_DEPTH) {  // e.g. static final int MAX_KEY_DEPTH = 10
        logger.warn("Key depth exceeded maximum allowed depth");
        return null;
    }
    // ... existing switch, passing depth+1 to recursive calls
}
```

Also add a total-node counter to `mapKeyList()` to cap the total number of key nodes processed per request, independent of depth.

### Proof of Concept

**Precondition**: Identify (or create) a Hedera account whose `key` column in the mirror node DB contains a multi-level threshold key. On mainnet, governance accounts (e.g., `0.0.2`, `0.0.50`) use complex key structures.

**Request** (no authentication required):
```http
POST /graphql/alpha HTTP/1.1
Content-Type: application/json

{"query":"{ account(input: {entityId: {num: 2}}) { key } }"}
```

**Observation**: Profile the JVM during this request. `CommonMapper.mapKey` and `mapKeyList` will appear in the CPU flame graph with stack depth proportional to the key tree depth. Heap allocation rate spikes relative to a query for any other scalar field (e.g., `memo`). Sending this query in a tight loop from multiple IPs (each under the 5-concurrent limit) amplifies the effect without triggering the per-IP rate limit.

        var keyCase = key.getKeyCase();
        return switch (keyCase) {
            case CONTRACTID -> Map.of(CONTRACT_ID, mapContractId(key.getContractID()));
            case DELEGATABLE_CONTRACT_ID -> Map.of(keyCase.toString(), mapContractId(key.getDelegatableContractId()));
            case ECDSA_384 -> Map.of(keyCase.toString(), encodeBase64String(toBytes(key.getECDSA384())));
            case ECDSA_SECP256K1 -> Map.of(keyCase.toString(), encodeBase64String(toBytes(key.getECDSASecp256K1())));
            case ED25519 -> Map.of(keyCase.toString(), encodeBase64String(toBytes(key.getEd25519())));
            case KEYLIST -> Map.of(KEYS, mapKeyList(key.getKeyList()));
            case RSA_3072 -> Map.of(keyCase.toString(), encodeBase64String(toBytes(key.getRSA3072())));
            case THRESHOLDKEY ->
                Map.of(
                        THRESHOLD, key.getThresholdKey().getThreshold(),
                        KEYS, mapKeyList(key.getThresholdKey().getKeys()));
            default -> null;
        };
    }

    default List<Object> mapKeyList(KeyList keyList) {
        var keys = keyList.getKeysList();
        if (CollectionUtils.isEmpty(keys)) {
            return Collections.emptyList();
        }

        var target = new ArrayList<>(keys.size());

        for (var key : keys) {
            target.add(mapKey(key));
        }

        return target;
    }
