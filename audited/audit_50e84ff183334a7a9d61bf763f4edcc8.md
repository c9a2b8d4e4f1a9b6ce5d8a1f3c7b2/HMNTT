### Title
Unauthenticated Repeated Full-Range DB Scan via `GET /api/v1/network/registered-nodes` with No Caching or Rate Limiting

### Summary
The `GET /api/v1/network/registered-nodes` endpoint is publicly accessible with no authentication, no rate limiting, and no response caching in the `rest-java` module. When called with no filters, `getRegisteredNodes()` resolves bounds to `[0, Long.MAX_VALUE]` and issues a native SQL query against the `registered_node` table on every request. An unprivileged attacker sending a sustained stream of filter-free requests can drive repeated full-range index scans, measurably increasing DB I/O above the 24-hour baseline.

### Finding Description

**Exact code path:**

1. `NetworkController.getRegisteredNodes()` at [1](#0-0)  — publicly reachable `@GetMapping("/registered-nodes")` with no authentication guard. No `@PreAuthorize`, no security filter, no rate-limit interceptor exists anywhere in the `rest-java` config layer. [2](#0-1) 

2. `NetworkServiceImpl.getRegisteredNodes()` at [3](#0-2)  — calls `resolveRegisteredNodeIdBounds(request.getRegisteredNodeIds())`.

3. `resolveRegisteredNodeIdBounds()` at [4](#0-3)  — when `registeredNodeIds` is the default empty list (no filter supplied), the loop body never executes and the method returns `Range.closed(0L, Long.MAX_VALUE)`.

4. `RegisteredNodeRepository.findByRegisteredNodeIdBetweenAndDeletedIsFalseAndTypeIs(0, Long.MAX_VALUE, null, page)` at [5](#0-4)  — executes a native SQL query with `registered_node_id >= 0 AND registered_node_id <= 9223372036854775807 AND deleted IS FALSE AND (:type IS NULL OR type @> array[:type]::smallint[])`. With `type = null` the array-containment check is bypassed, and the range covers the entire table.

**Root cause:** `RegisteredNodesRequest.registeredNodeIds` defaults to `List.of()` [6](#0-5) , so an empty-filter request is fully valid. `resolveRegisteredNodeIdBounds` treats an empty list as "no constraint" and returns the widest possible range. There is no `@Cacheable` annotation on the service method or repository method, and the `ShallowEtagHeaderFilter` only helps cooperative clients — an attacker ignores ETags. The throttle infrastructure (`ThrottleConfiguration`, `ThrottleManagerImpl`) lives exclusively in the `web3` module and is not wired into `rest-java`. [7](#0-6) 

### Impact Explanation

Every unauthenticated request with no query parameters causes a full primary-key-range scan of `registered_node` plus an array-containment evaluation (`type @> array[...]::smallint[]`) on every row. The `LIMIT` cap (max 100) [8](#0-7)  reduces rows returned but does not prevent the DB from evaluating the predicate across the full index range before stopping. As the `registered_node` table grows (it accumulates history rows via `registered_node_history`), each scan becomes more expensive. A sustained moderate request rate (e.g., tens of requests per second from a single client or a small botnet) can push DB read I/O and CPU well above the 24-hour baseline, degrading query latency for all other consumers of the same DB instance.

### Likelihood Explanation

No credentials, API keys, or special network access are required. The endpoint is documented in the public OpenAPI spec. [9](#0-8)  Any external actor can discover and script repeated calls. The k6 load-test script for this endpoint already demonstrates the pattern. [10](#0-9)  No brute-force credential guessing is needed; the attack is purely volumetric GET requests to a single URL.

### Recommendation

1. **Add response caching** at the service or HTTP layer (e.g., `@Cacheable` with a short TTL, or a `Cache-Control: max-age` header) so repeated identical requests are served from memory without hitting the DB.
2. **Add rate limiting** in the `rest-java` filter chain (e.g., bucket4j per-IP or global, mirroring the `web3` `ThrottleConfiguration`) for all `/api/v1/network/*` endpoints.
3. **Require a mandatory lower bound or narrow the default range** — reject requests where no `registerednode.id` filter is supplied and the result set would exceed a configurable threshold, or require at least one filter parameter.
4. **Add a DB-level index** on `(deleted, registered_node_id)` if not already present, so the predicate `deleted IS FALSE AND registered_node_id BETWEEN 0 AND MAX` uses a covering index rather than a heap scan.

    private static Range<Long> resolveRegisteredNodeIdBounds(List<NumberRangeParameter> registeredNodeIdRanges) {
        long lowerBound = 0L;
        long upperBound = MAX_VALUE;

        for (final var range : registeredNodeIdRanges) {
            if (range.operator() == RangeOperator.EQ) {
                if (registeredNodeIdRanges.size() > 1) {
                    throw new IllegalArgumentException("The 'eq' operator cannot be combined with other operators");
                }
                return Range.closed(range.value(), range.value());
            } else if (range.hasLowerBound()) {
                lowerBound = Math.max(lowerBound, range.getInclusiveValue());
            } else if (range.hasUpperBound()) {
                upperBound = Math.min(upperBound, range.getInclusiveValue());
            }
        }

        if (lowerBound > upperBound) {
            throw new IllegalArgumentException("Invalid range: lower bound exceeds upper bound");
        }

        return Range.closed(lowerBound, upperBound);
    }
