### Title
Prometheus Cardinality Explosion via User-Controlled `req.path` Label in `toOpenApiPath()`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to `req.path` — the raw, user-controlled URL path — when `req.route` is absent (i.e., for any unmatched/404 request). This raw path is then used as the `path` label in `requestTotalCounter` and `durationHistogram`. An unprivileged attacker can flood the server with requests to unique arbitrary paths, creating an unbounded number of distinct Prometheus label combinations (cardinality explosion), which exhausts memory and corrupts or drops metric exports.

### Finding Description

**Exact code path:**

`toOpenApiPath()` at `rest/middleware/metricsHandler.js` lines 13–31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // line 14 — only set for matched routes

  if (!path) {
    if (!req.route) {
      path = req.path;                        // line 18 — RAW user-controlled path
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }
  path = path.replace(/:([^/]+)/g, '{$1}');
  ...
  return path;
};
```

The result is used directly as a metric label at lines 192 and 207–210:

```js
const labels = {method, path, code};
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
```

**Root cause — two failed guards:**

1. `res.locals[requestPathLabel]` is set only by `recordRequestPath` in `rest/routes/index.js` (lines 16–21), which only fires when `req.route?.path` is truthy — i.e., only for matched routes.
2. `req.route` is set by Express only when a route handler matches. For any unmatched path (resulting in a 404), `req.route` is `null`.

For a 404 request, both guards fail, so `path = req.path` is used verbatim.

**Why the error handler doesn't help:**

`httpErrorHandler.js` handles the `NotFoundError` thrown by `responseHandler.js` (line 30 of `responseHandler.js`), but it only sets the HTTP status code and sends a JSON body. It never sets `req.route` or `res.locals[requestPathLabel]`. The `res.on('finish', ...)` callback in `metricsHandler.js` (line 185) fires after the response is finished — after the error handler — and at that point `req.route` is still `null` and `res.locals[requestPathLabel]` is still unset.

**Exploit flow:**

An attacker sends a high volume of GET requests to unique paths:
```
GET /api/v1/accounts/AAAA1/extra/seg1
GET /api/v1/accounts/AAAA2/extra/seg2
GET /api/v1/accounts/AAAA3/extra/seg3
...
```
Each request:
- Gets a 404 (no route matches)
- `req.route` is `null`, `res.locals[requestPathLabel]` is unset
- `toOpenApiPath()` returns the raw `req.path` as the label value
- A new unique `{method, path, code}` label set is registered in `requestTotalCounter` and `durationHistogram`

### Impact Explanation
Prometheus stores one time series per unique label combination. Each histogram metric (`durationHistogram`) stores multiple series per label set (one per bucket boundary plus `_sum` and `_count`). With `N` unique attacker-supplied paths, the number of in-memory time series grows as `O(N × buckets)`. This exhausts the Node.js process heap, causing the OpenTelemetry `MeterProvider` to fail or drop data during collection, and the `/metrics` scrape endpoint to return corrupt or empty data. The mirror node's observability is effectively blinded, and the process may OOM-crash.

### Likelihood Explanation
No authentication or authorization is required. Any network-reachable client can send HTTP requests. The attack is trivially scriptable: a simple loop generating unique path segments (e.g., random UUIDs or sequential integers) is sufficient. The `res.on('finish', ...)` hook fires for every request including 404s, so there is no way for the application to avoid recording the label before the fix is applied. The attack is repeatable and can be sustained indefinitely.

### Recommendation
In `toOpenApiPath()`, replace the `req.path` fallback with a static sentinel value that does not embed user input:

```js
if (!req.route) {
  path = 'unknown';   // or apiPrefix + '/unknown'
} else {
  path = (req.baseUrl ?? '') + req.route?.path;
}
```

Alternatively, maintain an allowlist of known OpenAPI path templates and map unmatched requests to a single `unknown` bucket. Do not use any part of `req.path`, `req.url`, or `req.originalUrl` as a metric label value.

### Proof of Concept

```bash
# Send 10,000 requests with unique path segments (no auth required)
for i in $(seq 1 10000); do
  curl -s "http://<mirror-node-host>/api/v1/accounts/ATTACK${i}/extra/segment" &
done
wait

# Scrape metrics and observe unbounded unique 'path' label values
curl http://<mirror-node-host>/swagger/metrics/ | grep 'api_request_total' | wc -l
```

After this, the metrics output will contain thousands of unique `path=` label values derived directly from the attacker-supplied URL segments, confirming cardinality explosion. Memory usage of the Node.js process will grow proportionally.
