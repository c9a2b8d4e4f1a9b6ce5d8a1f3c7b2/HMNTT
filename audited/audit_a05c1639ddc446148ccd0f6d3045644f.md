### Title
Unbounded Repeated Allocation in `NetworkOptions()` with No Application-Level Rate Limiting Enables Griefing DoS

### Summary
`NetworkOptions()` unconditionally rebuilds the full `operationStatuses` slice (up to ~337 entries) on every invocation with no caching, no pagination, and no application-layer rate limiting. The Traefik middleware that would enforce per-IP in-flight limits and per-host rate limits is disabled by default (`global.middleware: false`). An unauthenticated attacker can flood the endpoint at arbitrary concurrency, forcing repeated heap allocations and map iterations that degrade server responsiveness without any economic cost to the attacker.

### Finding Description

**Exact code path:**

`rosetta/app/services/network_service.go`, `NetworkOptions()`, lines 35–55:

```go
func (n *networkAPIService) NetworkOptions(
    _ context.Context,
    _ *rTypes.NetworkRequest,
) (*rTypes.NetworkOptionsResponse, *rTypes.Error) {
    operationStatuses := make([]*rTypes.OperationStatus, 0, len(types.GetTransactionResults()))
    for value, name := range types.GetTransactionResults() {
        operationStatuses = append(operationStatuses, &rTypes.OperationStatus{
            Status:     name,
            Successful: persistence.IsTransactionResultSuccessful(value),
        })
    }
    return &rTypes.NetworkOptionsResponse{ ... }, nil
}
```

`GetTransactionResults()` returns a map populated at `init()` time with up to `MaxTransactionResult = 337` entries (all `ResponseCodeEnum` codes ≤ 337). On every request the function:
1. Allocates a new `[]*rTypes.OperationStatus` slice with capacity 337.
2. Iterates the full 337-entry map.
3. Calls `IsTransactionResultSuccessful()` for each entry.
4. JSON-serializes the full response (~337 statuses + ~58 operation types + ~42 error objects ≈ 5–15 KB).

None of this work is cached or memoized. The response is identical on every call because all three data sources (`transactionResults`, `operationTypes`, `errors.Errors`) are static after initialization.

**Why existing checks fail:**

The only defined rate-limiting mechanism is the Traefik middleware chain in `charts/hedera-mirror-rosetta/values.yaml` (lines 149–166), which includes `inFlightReq: amount: 5` per IP and `rateLimit: average: 10` per host. However, this middleware is gated by the Helm template condition:

```yaml
{{ if and .Values.global.middleware .Values.middleware -}}
```

The default value of `global.middleware` is `false` (line 95 of `values.yaml`). The middleware is therefore **not deployed** in default installations. The Go HTTP server itself applies only `MetricsMiddleware`, `TracingMiddleware`, and `CorsMiddleware` — none of which throttle requests. No authentication is required.

### Impact Explanation
An attacker can sustain thousands of concurrent POST requests to `/network/options` from a single host or distributed sources. Each request forces ~337 heap allocations and a full map scan on the server goroutine pool. Under sufficient concurrency this exhausts available goroutines, increases GC pressure, and delays or drops legitimate requests to all Rosetta endpoints sharing the same process. The impact is service degradation (griefing) with no economic cost to the attacker and no direct financial loss to network users, consistent with the medium severity classification.

### Likelihood Explanation
The endpoint is fully public, requires no credentials, and accepts a minimal POST body (`{"network_identifier": {...}}`). Any attacker with a commodity HTTP flood tool (e.g., `wrk`, `hey`, `ab`) can trigger this. The default deployment ships without the protective middleware, meaning the vast majority of real-world deployments are unprotected. The attack is trivially repeatable and requires no special knowledge of the protocol.

### Recommendation
1. **Cache the response**: Since all three components (`operationStatuses`, `operationTypes`, `errors.Errors`) are fully static after process startup, compute the `NetworkOptionsResponse` once in `NewNetworkAPIService` and store it as a field. `NetworkOptions()` then returns the pre-built pointer with zero allocation.
2. **Enable middleware by default**: Change `global.middleware` default to `true` in `values.yaml`, or add application-level rate limiting directly in the Go server (e.g., `golang.org/x/time/rate` per-IP limiter in a middleware layer) so protection is not opt-in.
3. **Add per-endpoint concurrency limits** in the Go HTTP layer independent of the Kubernetes ingress configuration, so bare deployments (non-Kubernetes) are also protected.

### Proof of Concept

**Preconditions:** Default Rosetta deployment with `global.middleware: false` (the default). Server reachable on port 5700.

**Steps:**
```bash
# 1. Confirm endpoint responds with no auth
curl -s -X POST http://<rosetta-host>:5700/network/options \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' \
  | python3 -m json.tool | head -20

# 2. Flood with concurrent requests (requires `hey` tool)
hey -n 100000 -c 500 -m POST \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' \
  http://<rosetta-host>:5700/network/options

# 3. Observe: server CPU spikes, GC pauses increase, latency on /network/status
# and other endpoints rises or requests begin timing out.
```

**Expected result:** Server goroutine pool saturates; legitimate requests to other endpoints experience elevated latency or connection refusals. No authentication, no rate-limit response (HTTP 429) is returned under default configuration.
