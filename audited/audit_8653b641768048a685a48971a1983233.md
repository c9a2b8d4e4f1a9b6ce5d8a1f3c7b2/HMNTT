### Title
Unsanitized `X-Forwarded-For` / `X-Real-IP` Header Enables Log Injection and IP Spoofing in Tracing Middleware

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` blindly trusts and returns the raw value of the `X-Forwarded-For` or `X-Real-IP` header without any validation, sanitization, or format check. Because the Rosetta server binds directly to a port with no upstream proxy enforcing these headers, any unauthenticated external client can supply an arbitrary string — including newline characters — that is then written verbatim into the application log, enabling both IP spoofing in audit records and log injection attacks.

### Finding Description
**Exact code path:**

`getClientIpAddress()` at `rosetta/app/middleware/trace.go` lines 63–74:
```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)          // line 64 – raw, unvalidated
    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader) // line 67 – raw, unvalidated
    }
    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr) // only fallback is safe
    }
    return ipAddress                                   // line 74 – returned verbatim
}
```

The returned value is immediately interpolated into the log message at `TracingMiddleware` lines 52–53:
```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
```

**Root cause:** The function performs only a length check (`len(ipAddress) == 0`) — it never calls `net.ParseIP()`, never extracts the first token from a comma-separated `X-Forwarded-For` list, and never strips control characters such as `\n` or `\r`.

**Why existing checks fail:** The test suite (`trace_test.go` lines 52–58) only exercises the happy path with a well-formed IP string `"10.0.0.100"`. No test covers malformed, multi-value, or control-character-containing inputs. There is no upstream proxy in the deployment path (`main.go` line 231 `httpServer.ListenAndServe()`) that would strip or override these headers before they reach the handler.

**Exploit flow:**
1. Attacker sends an HTTP request directly to the Rosetta port with a crafted header.
2. `getClientIpAddress()` returns the raw header value.
3. The value is written to the log via `log.Info(message)`.

### Impact Explanation
**IP spoofing / misleading audit records:** Any client can impersonate any IP address in the tracing log, making it impossible to attribute requests to real sources. This directly corrupts the "records exported to mirror nodes" that rely on these logs for audit and monitoring.

**Log injection:** A header value containing `\n` (e.g., `X-Forwarded-For: 1.2.3.4\nINFO fake-event admin-login success`) causes logrus to write a second, fully attacker-controlled log line. This can be used to forge security events, hide malicious activity, or poison log-aggregation pipelines (e.g., Loki, as referenced in `main.go` line 43).

Severity: **Medium** — no direct code execution, but audit integrity is broken and log-injection can mislead incident response.

### Likelihood Explanation
- **No authentication required** — the Rosetta API is a public HTTP endpoint.
- **Trivially reproducible** — a single `curl` command with a custom header is sufficient.
- **No rate-limiting or header-stripping** is present in the middleware chain (`main.go` lines 217–219: `MetricsMiddleware → TracingMiddleware → CorsMiddleware`).
- Exploitable by any network-reachable client, repeatedly and at zero cost.

### Recommendation
1. **Validate the extracted IP** using `net.ParseIP()` after extraction; if parsing fails, fall back to `r.RemoteAddr`.
2. **For `X-Forwarded-For`**, extract only the *first* (leftmost) token before the first comma, which represents the original client IP per RFC 7239.
3. **Strip or reject control characters** (`\n`, `\r`, `\t`) from any header-derived string before logging.

Example fix for `getClientIpAddress()`:
```go
func getClientIpAddress(r *http.Request) string {
    for _, header := range []string{xRealIpHeader, xForwardedForHeader} {
        val := r.Header.Get(header)
        if val == "" {
            continue
        }
        // X-Forwarded-For may be a comma-separated list; take the first entry
        ip := strings.TrimSpace(strings.SplitN(val, ",", 2)[0])
        if net.ParseIP(ip) != nil {
            return ip
        }
    }
    ip, _, _ := net.SplitHostPort(r.RemoteAddr)
    return ip
}
```

### Proof of Concept
```bash
# IP spoofing
curl -s -X POST http://<rosetta-host>:<port>/network/list \
  -H 'Content-Type: application/json' \
  -H 'X-Forwarded-For: 10.0.0.1, admin, internal' \
  -d '{"metadata":{}}'
# Log output will contain: "10.0.0.1, admin, internal POST /network/list (200) in ..."

# Log injection
curl -s -X POST http://<rosetta-host>:<port>/network/list \
  -H 'Content-Type: application/json' \
  -H $'X-Forwarded-For: 1.2.3.4\ntime="2026-01-01" level=info msg="admin login success"' \
  -d '{"metadata":{}}'
# A second, fully forged log line is written to stdout/Loki
