### Title
Case-Variant `hbar` Parameter Merging Causes Unhandled Array in `parseHbarParam`, Returning Internal Error Message to Unauthenticated Callers

### Summary
The `requestQueryParser()` function in `rest/middleware/requestHandler.js` intentionally merges query parameters that differ only by case into a single lowercase key as an array. When an unprivileged user sends `?hbar=true&HBAR=false`, the `merge()` helper produces `req.query.hbar = ['true', 'false']`. The downstream `parseHbarParam()` in `rest/utils.js` only handles `string` and `undefined`/`null` types; receiving an array bypasses both guards and unconditionally throws `InvalidArgumentError`. The error handler in `rest/middleware/httpErrorHandler.js` is configured to return the full `err.message` to the client for all 4xx responses, so the raw error string — including the array value — is sent back to the caller.

### Finding Description

**Step 1 — `requestQueryParser` merges mixed-case keys into an array**

`rest/middleware/requestHandler.js` lines 38–69:
```
hbar=true&HBAR=false
  → qs.parse → { hbar: 'true', HBAR: 'false' }
  → loop: 'hbar' stored as 'true'
  → loop: 'HBAR'.toLowerCase() == 'hbar' already present → merge('true','false') → ['true','false']
  → req.query.hbar = ['true', 'false']
``` [1](#0-0) 

The `queryCanonicalizationMap` only covers `order` and `result`; `hbar` is absent, so no canonicalization or array-guard is applied. [2](#0-1) 

**Step 2 — `parseHbarParam` has no array guard**

`rest/utils.js` lines 216–231:
```js
if (hbarParam === undefined || hbarParam === null) return true;   // array passes
if (typeof hbarParam === 'string') { ... }                        // array fails typeof check
throw new InvalidArgumentError(`Invalid hbar parameter value: ${hbarParam}. Must be 'true' or 'false'`);
```
`typeof ['true','false']` is `'object'`, not `'string'`, so both guards are bypassed and the throw is reached unconditionally. The array is coerced to the string `"true,false"` inside the template literal. [3](#0-2) 

`parseHbarParam` is called directly from three handlers in `contractController.js` — `getContractResultsByTimestamp`, `getContractResults`, and `getContractResultsByTransactionIdOrHash` — all before any per-parameter validation that might catch the array. [4](#0-3) [5](#0-4) [6](#0-5) 

**Step 3 — Error handler returns full message for all 4xx responses**

`rest/middleware/httpErrorHandler.js` lines 17–42:
```js
} else if (err instanceof InvalidArgumentError || err instanceof RangeError) {
  statusCode = httpStatusCodes.BAD_REQUEST;   // 400 → isClientError() == true
}
if (shouldReturnMessage(statusCode)) {
  errorMessage = err.message;                 // full message assigned
}
res.status(statusCode.code).json(errorMessageFormat(errorMessage));
```
`shouldReturnMessage` returns `true` for any client error (4xx), so `err.message` — the full `InvalidArgumentError` string — is serialised into the response body. [7](#0-6) [8](#0-7) 

**Response body delivered to the caller:**
```json
{
  "_status": {
    "messages": [
      { "message": "Invalid hbar parameter value: true,false. Must be 'true' or 'false'" }
    ]
  }
}
```

### Impact Explanation
Any unauthenticated caller can force every contract-result endpoint (`/api/v1/contracts/results`, `/api/v1/contracts/{id}/results/{timestamp}`, `/api/v1/contracts/results/{txIdOrHash}`) to return a 400 with the internal error string. The message confirms the existence and exact name of the `hbar` unit-conversion parameter, its accepted values, and the fact that the server received an array — information that, while partially documented in the OpenAPI spec, is now also confirmed via an unintended error path. The primary concrete impact is a reliable, zero-privilege mechanism to trigger an unhandled code path and receive internal diagnostic output. Severity is **Low**: no sensitive data (credentials, PII, DB internals) is exposed, and the information is largely already public.

### Likelihood Explanation
Exploitation requires no authentication, no special tooling, and no knowledge of the system beyond the public API. Any HTTP client can reproduce it with a single crafted URL. The trigger is stable across all three affected endpoints and is repeatable indefinitely. Automated scanners that fuzz query-parameter casing will discover this without human effort.

### Recommendation
1. **Guard `parseHbarParam` against non-string input** — add an explicit array check before the `typeof` test:
   ```js
   if (Array.isArray(hbarParam)) {
     throw new InvalidArgumentError(`Invalid hbar parameter value: ${hbarParam}. Must be 'true' or 'false'`);
   }
   ```
   Or, more defensively, only accept the first element if an array is received (last-value-wins semantics).

2. **Register `hbar` in `queryCanonicalizationMap`** or add a dedicated single-value enforcement path so that `requestQueryParser` rejects or truncates duplicate `hbar` keys before they reach controller logic.

3. **Consider sanitising user-supplied values from error messages** before returning them to callers, consistent with the pattern already used in the Java `GenericControllerAdvice` (`"Failed to convert '%s'"` — deliberately omits the value to prevent reflected-input issues).

const requestQueryParser = (queryString) => {
  const merge = (current, next) => {
    if (!Array.isArray(current)) {
      current = [current];
    }

    if (Array.isArray(next)) {
      current.push(...next);
    } else {
      current.push(next);
    }

    return current;
  };

  // parse first to benefit from qs query handling
  const parsedQueryString = qs.parse(queryString, queryOptions);

  const caseInsensitiveQueryString = {};
  for (const [key, value] of Object.entries(parsedQueryString)) {
    const lowerKey = key.toLowerCase();
    const canonicalValue = canonicalizeValue(lowerKey, value);
    if (lowerKey in caseInsensitiveQueryString) {
      // handle repeated values, merge into an array
      caseInsensitiveQueryString[lowerKey] = merge(caseInsensitiveQueryString[lowerKey], canonicalValue);
    } else {
      caseInsensitiveQueryString[lowerKey] = canonicalValue;
    }
  }

  return caseInsensitiveQueryString;
};

const parseHbarParam = (hbarParam) => {
  if (hbarParam === undefined || hbarParam === null) {
    return true; // Default to true for backward compatibility
  }

  if (typeof hbarParam === 'string') {
    const lower = hbarParam.toLowerCase();
    if (lower === 'true') {
      return true;
    }
    if (lower === 'false') {
      return false;
    }
  }

  throw new InvalidArgumentError(`Invalid hbar parameter value: ${hbarParam}. Must be 'true' or 'false'`);
};
