### Title
Shared DB Connection Pool Exhaustion via Unauthenticated Read Endpoints Starves Construction Service Account Validation

### Summary
The Rosetta API runs a single shared `sql.DB` connection pool (default `MaxOpenConnections: 100`) across all services — block, account, and construction. There is no application-level rate limiting, concurrency cap, or connection reservation for the construction service. An unauthenticated attacker can flood read-heavy endpoints (`/account/balance`, `/block`) that each consume multiple DB connections per request, exhausting the pool and blocking `ConstructionMetadata`'s account-alias DB lookup, which is a required step before a transaction can be gossiped.

### Finding Description

**Single shared pool, no partitioning:**
`rosetta/main.go` calls `db.ConnectToDb` exactly once and passes the resulting `dbClient` to every repository — `accountRepo`, `blockRepo`, `transactionRepo` — all of which share the same `sql.DB` pool. [1](#0-0) [2](#0-1) 

**Pool ceiling:**
`Pool.MaxOpenConnections` defaults to **100** (documented default). [3](#0-2) 

**Read endpoints consume multiple connections per request:**
`/account/balance` → `RetrieveBalanceAtBlock` issues three sequential DB queries: `getCryptoEntity`, `getLatestBalanceSnapshot`, `getBalanceChange`. Each acquires a connection from the pool for up to `StatementTimeout` (default 20 s). [4](#0-3) 

`/block` → `Block` calls `RetrieveBlock`, `FindBetween`, and per-operation `GetAccountAlias` — also multiple pool acquisitions. [5](#0-4) 

**Construction service DB dependency (pre-gossip validation):**
`ConstructionMetadata` calls `resolveAccountAliases` → `accountRepo.GetAccountId`, which issues a DB query to resolve an alias to a `shard.realm.num` account ID. This is a mandatory step in the Rosetta construction flow before a signed transaction can be submitted. [6](#0-5) [7](#0-6) 

**No application-level rate limiting or connection reservation:**
A `grep` across all `rosetta/**/*.go` for `inFlightReq`, `rateLimit`, `semaphore`, `throttle`, `concurrency` returns **zero matches**. The only middleware layers are metrics, tracing, and health — none enforce concurrency limits. [8](#0-7) 

**Traefik middleware is optional and bypassable:**
The Helm chart defines `inFlightReq: amount: 5` per source IP and `rateLimit: average: 10` per host, but this is a Kubernetes deployment artifact — not enforced at the application layer. An attacker using ≥20 source IPs (e.g., a small botnet or cloud VMs) trivially bypasses the per-IP cap. [9](#0-8) 

### Impact Explanation
With 100 max open connections and each `/account/balance` request holding up to 3 connections for 20 seconds, only **~34 concurrent requests** are needed to saturate the pool. Once saturated, any call to `ConstructionMetadata` that triggers `GetAccountId` blocks indefinitely waiting for a free connection. This prevents legitimate operators from completing the construction flow (preprocess → metadata → payloads → combine → submit), effectively halting transaction gossip from the mirror node's Rosetta interface. The `ConstructionSubmit` endpoint itself uses the Hiero SDK (not the DB), but it is unreachable without a valid metadata response containing the resolved account map and node account ID.

### Likelihood Explanation
The attack requires no credentials, no special protocol knowledge beyond the public Rosetta API spec, and no privileged network position. The attacker needs only HTTP access to port 5700 and the ability to issue ~34 concurrent POST requests to `/account/balance` with valid (or even invalid) account identifiers — invalid identifiers still reach the DB query path before returning an error. The attack is repeatable and can be sustained indefinitely. The only deployed mitigation (Traefik per-IP limit) is bypassable with multiple source IPs and is absent in non-Kubernetes deployments.

### Recommendation
1. **Reserve connections for the construction service**: Use a separate `sql.DB` pool (second `ConnectToDb` call with a dedicated DB user/pool config) for the construction service's `accountRepo`, isolating it from read traffic.
2. **Add application-level concurrency limits**: Implement a `golang.org/x/sync/semaphore` or `http.MaxBytesHandler`-style middleware in the Rosetta Go server to cap total in-flight DB-bound requests, independent of the Traefik layer.
3. **Reduce `StatementTimeout`**: Lower from 20 s to 2–5 s for read endpoints to release connections faster under load.
4. **Add per-endpoint connection accounting**: Track connections consumed per route and shed load (HTTP 503) before the pool is fully exhausted, preserving headroom for construction endpoints.

### Proof of Concept
```bash
# Exhaust the 100-connection pool with concurrent /account/balance requests
# Each request holds ~3 connections for up to 20s (StatementTimeout)
# 34 concurrent requests * 3 connections = 102 > MaxOpenConnections(100)

for i in $(seq 1 40); do
  curl -s -X POST http://<rosetta-host>:5700/account/balance \
    -H 'Content-Type: application/json' \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},
         "account_identifier":{"address":"0.0.98"}}' &
done

# While the above runs, attempt ConstructionMetadata (requires DB for alias resolution)
curl -X POST http://<rosetta-host>:5700/construction/metadata \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},
       "options":{"operation_type":"CRYPTOTRANSFER",
                  "account_aliases":"0x<alias_hex>"}}'
# Result: hangs or times out — pool exhausted, GetAccountId cannot acquire a connection
```

func (ar *accountRepository) GetAccountId(ctx context.Context, accountId types.AccountId) (
	zero types.AccountId,
	_ *rTypes.Error,
) {
	if !accountId.HasAlias() {
		return accountId, nil
	}

	db, cancel := ar.dbClient.GetDbWithContext(ctx)
	defer cancel()

	var entity domain.Entity
	if err := db.Raw(selectCurrentCryptoEntityByAlias, sql.Named("alias", accountId.GetAlias())).First(&entity).Error; err != nil {
		if errors.Is(err, gorm.ErrRecordNotFound) {
			return zero, hErrors.ErrAccountNotFound
		}

		return zero, hErrors.ErrDatabaseError
	}

	return types.NewAccountIdFromEntityId(entity.Id), nil
}

func (ar *accountRepository) RetrieveBalanceAtBlock(
	ctx context.Context,
	accountId types.AccountId,
	consensusEnd int64,
) (types.AmountSlice, string, []byte, *rTypes.Error) {
	var entityIdString string
	entity, err := ar.getCryptoEntity(ctx, accountId, consensusEnd)
	if err != nil {
		return nil, entityIdString, nil, err
	}

	if entity == nil && accountId.HasAlias() {
		return types.AmountSlice{&types.HbarAmount{}}, entityIdString, nil, nil
	}

	balanceChangeEndTimestamp := consensusEnd
	balanceSnapshotEndTimestamp := consensusEnd
	if entity != nil && entity.Deleted != nil && *entity.Deleted && entity.GetModifiedTimestamp() <= consensusEnd {
		// if an account / contract is deleted at t1, a balance snapshot at t1 (if exists) won't have info for the
		// entity, thus look for a balance snapshot at or before the deleted timestamp - 1
		// however, the balanceChangeEndTimestamp should be the deletion timestamp since the crypto delete transaction
		// may have a transfer which moves the remaining hbar balance to another account
		balanceChangeEndTimestamp = entity.GetModifiedTimestamp()
		balanceSnapshotEndTimestamp = balanceChangeEndTimestamp - 1
	}

	id := accountId.GetId()
	if accountId.HasAlias() {
		// entity can't be nil if accountId has alias
		id = entity.Id.EncodedId
	}
	snapshotTimestamp, hbarAmount, err := ar.getLatestBalanceSnapshot(
		ctx,
		id,
		balanceSnapshotEndTimestamp,
	)
	if err != nil {
		return nil, entityIdString, nil, err
	}

	hbarValue, err := ar.getBalanceChange(
		ctx,
		id,
		snapshotTimestamp,
		balanceChangeEndTimestamp,
	)
	if err != nil {
		return nil, entityIdString, nil, err
	}

	hbarAmount.Value += hbarValue

	amounts := make(types.AmountSlice, 0, 1)
	amounts = append(amounts, hbarAmount)

	var publicKey []byte
	if entity != nil {
		// return the entity id string in the format of 'shard.realm.num'
		entityIdString = entity.Id.String()
		publicKey = entity.Key
	}
	return amounts, entityIdString, publicKey, nil
}

func (s *blockAPIService) Block(
	ctx context.Context,
	request *rTypes.BlockRequest,
) (*rTypes.BlockResponse, *rTypes.Error) {
	block, err := s.RetrieveBlock(ctx, request.BlockIdentifier)
	if err != nil {
		return nil, err
	}

	if block.Transactions, err = s.FindBetween(ctx, block.ConsensusStartNanos, block.ConsensusEndNanos); err != nil {
		return nil, err
	}

	var otherTransactions []*rTypes.TransactionIdentifier
	if len(block.Transactions) > s.maxTransactionsInBlock {
		otherTransactions = make([]*rTypes.TransactionIdentifier, 0, len(block.Transactions)-s.maxTransactionsInBlock)
		for _, transaction := range block.Transactions[s.maxTransactionsInBlock:] {
			otherTransactions = append(otherTransactions, &rTypes.TransactionIdentifier{Hash: transaction.Hash})
		}
		block.Transactions = block.Transactions[0:s.maxTransactionsInBlock]
	}

	if err = s.updateOperationAccountAlias(ctx, block.Transactions...); err != nil {
		return nil, err
	}

	return &rTypes.BlockResponse{Block: block.ToRosetta(), OtherTransactions: otherTransactions}, nil

func (c *constructionAPIService) resolveAccountAliases(
	ctx context.Context,
	options map[string]any,
) (emptyResult string, nilErr *rTypes.Error) {
	if options[optionKeyAccountAliases] == nil {
		return
	}

	if !c.BaseService.IsOnline() {
		return emptyResult, errors.ErrEndpointNotSupportedInOfflineMode
	}

	accountAliases, ok := options[optionKeyAccountAliases].(string)
	if !ok {
		return emptyResult, errors.ErrInvalidOptions
	}

	var accountMap []string
	for accountAlias := range strings.SplitSeq(accountAliases, ",") {
		accountId, err := types.NewAccountIdFromString(accountAlias, c.systemShard, c.systemRealm)
		if err != nil {
			return emptyResult, errors.ErrInvalidAccount
		}

		found, rErr := c.accountRepo.GetAccountId(ctx, accountId)
		if rErr != nil {
			return emptyResult, rErr
		}
		accountMap = append(accountMap, fmt.Sprintf("%s:%s", accountAlias, found))
	}

	return strings.Join(accountMap, ","), nilErr
}
