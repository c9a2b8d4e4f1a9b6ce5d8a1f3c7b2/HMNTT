### Title
Unsanitized User-Controlled Path and IP Headers Logged Verbatim — HTML/Log Injection in `TracingMiddleware`

### Summary
`TracingMiddleware()` in `rosetta/app/middleware/trace.go` logs the raw request URI and the value of user-supplied `X-Real-IP` / `X-Forwarded-For` headers directly into log messages without any sanitization or encoding. An unprivileged external attacker can inject arbitrary HTML/XML markup into these log entries. If the log pipeline feeds into a web-based dashboard that renders log message content as HTML (e.g., certain Kibana, Graylog, or Splunk configurations), the injected markup executes in the operator's browser, degrading log-dashboard integrity.

### Finding Description
**File:** `rosetta/app/middleware/trace.go`
**Function:** `TracingMiddleware()`, lines 43–61
**Root cause:** Two distinct unsanitized user-controlled values are interpolated into the log message:

1. **Path** — line 47: `path := request.URL.RequestURI()` captures the raw request-target string. Go's `net/http` server is permissive; non-standard clients (e.g., `curl --path-as-is`) can transmit literal `<`, `>` characters in the request line that survive into `URL.RawPath` and are returned verbatim by `RequestURI()`.

2. **Client IP** — lines 64–68 (`getClientIpAddress`): the value of `X-Real-IP` or `X-Forwarded-For` is accepted as-is with no format validation. Any string, including `<script>alert(1)</script>`, is returned and used directly.

Both values are concatenated into `message` at line 52–53:
```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
```
and passed to `log.Info(message)` / `log.Debug(message)` (logrus) at lines 56–58 with no escaping. Logrus's default text formatter writes the string as-is; even the JSON formatter embeds the raw string value, which many log-viewer UIs then render.

**No existing checks** strip, encode, or validate either value before logging. The `internalPaths` guard (line 55) only controls log level, not content.

### Impact Explanation
An attacker who can reach the Rosetta API endpoint (no authentication required) can permanently inject HTML/XML payloads into the operator's log stream. In web-based log dashboards that render message content as HTML, this enables stored cross-site scripting against operators and security engineers reviewing logs — forging fake log entries, hiding real events, or executing scripts in the operator's browser session. The impact is confined to the integrity of the log-monitoring plane; no on-chain or financial assets are directly affected, consistent with the Medium/griefing classification.

### Likelihood Explanation
The precondition is trivially met: any unauthenticated HTTP client can reach the public Rosetta API. The `X-Forwarded-For` / `X-Real-IP` headers require no special privileges and are routinely forwarded by reverse proxies. The attack is repeatable and requires no prior knowledge of the system. Exploitability is conditional on the log viewer rendering HTML, which is a realistic deployment scenario for teams using Kibana or Graylog with default or lightly-configured index patterns.

### Recommendation
Sanitize both values before logging:

1. **Path**: strip or percent-encode `<`, `>`, `"`, `'`, and `&` characters, or use `url.PathEscape` / `html.EscapeString` before interpolation.
2. **IP address**: validate that `X-Real-IP` / `X-Forwarded-For` values conform to a valid IPv4/IPv6 format (e.g., using `net.ParseIP`) and reject or truncate non-conforming values before using them in the log message.
3. Consider using logrus structured fields (`log.WithFields`) instead of a pre-formatted string, so that log formatters can apply per-field encoding.

### Proof of Concept
```bash
# Inject via X-Real-IP header (no URL encoding involved — raw string logged as-is)
curl -s \
  -H 'X-Real-IP: <img src=x onerror=alert(document.cookie)>' \
  http://<rosetta-host>/network/list

# Inject via path (using --path-as-is to bypass client-side encoding)
curl -s --path-as-is \
  'http://<rosetta-host>/<script>alert(1)</script>'
```

After either request, the log entry written by `log.Info(message)` contains the raw HTML tag. In a Kibana/Graylog dashboard configured to display the `message` field with HTML rendering enabled, the injected payload executes in the operator's browser. [1](#0-0) [2](#0-1) [3](#0-2)
