### Title
Unauthenticated `/metrics` Access Silently Downgraded to Debug-Level Logging, Enabling Invisible Reconnaissance

### Summary
`TracingMiddleware` in `rosetta/app/middleware/trace.go` unconditionally routes any request whose path equals `/metrics` to `log.Debug`, while the production default log level is `info`. Because the `/metrics` endpoint carries no authentication, any unprivileged external caller can repeatedly scrape Prometheus telemetry and have every access suppressed from info-level logs, making the activity undetectable through normal log monitoring.

### Finding Description
**Code path:**

- `rosetta/app/middleware/trace.go`, `TracingMiddleware()`, line 19 defines the suppression set:
  ```go
  var internalPaths = map[string]bool{livenessPath: true, metricsPath: true, readinessPath: true}
  ```
- Lines 47 and 55ŌĆō58 apply it:
  ```go
  path := request.URL.RequestURI()   // line 47
  ...
  if internalPaths[path] {           // line 55
      log.Debug(message)             // line 56  ŌåÉ suppressed at info level
  } else {
      log.Info(message)              // line 58
  }
  ```
- `metricsPath = "/metrics"` is declared in `rosetta/app/middleware/metrics.go` line 16.
- The route handler registered for that path is `promhttp.Handler().ServeHTTP` with **no authentication wrapper** (`metrics.go` lines 64ŌĆō72).
- `main.go` line 156 initialises the logger at `"info"` before any config override, so debug messages are dropped in every standard deployment.

**Root cause / failed assumption:** The `internalPaths` map was designed to suppress log spam from internal scraping agents (Kubernetes probes, Prometheus). The implicit assumption is that only trusted internal systems reach these paths. That assumption is never enforced: there is no network policy, no IP allowlist, and no HTTP authentication on the `/metrics` route, so the suppression applies equally to any external caller who sends `GET /metrics`.

**Why existing checks fail:** `promhttp.Handler()` is wrapped directly with no middleware that checks credentials or source IP. `TracingMiddleware` itself only decides the log level; it does not gate access. There is no separate auth middleware applied to the metrics route in either `newBlockchainOnlineRouter` or `newBlockchainOfflineRouter`.

### Impact Explanation
An external attacker who GETs `/metrics` receives the full Prometheus exposition: Go runtime stats, process memory/CPU, and all four application histograms (`hiero_mirror_rosetta_request_bytes`, `_request_duration`, `_request_inflight`, `_response_bytes`). These counters reveal endpoint usage patterns, error rates, and timing distributions ŌĆö useful for fingerprinting the deployment and timing further attacks. Critically, none of these requests appear in info-level logs, so SIEM rules, log-based alerting, and audit trails that rely on the standard log stream are blind to the activity. Severity: **Medium** (information disclosure + audit evasion; no direct data corruption or privilege escalation).

### Likelihood Explanation
Preconditions are minimal: network reachability to the Rosetta port and knowledge that the standard Prometheus path `/metrics` exists (publicly documented). No credentials, tokens, or prior access are required. The attack is trivially repeatable with a single `curl` command and is completely passive from the server's perspective. Any attacker performing basic API enumeration will discover and exploit this automatically.

### Recommendation
1. **Add authentication to the `/metrics` route.** Wrap `promhttp.Handler()` with a middleware that enforces an allowlist of source CIDRs or requires a bearer/basic token before serving the handler.
2. **Separate internal-only ports.** Serve `/metrics`, `/liveness`, and `/readiness` on a distinct, non-public port (e.g., `8090`) so network policy can restrict access without application-level auth.
3. **If neither is done immediately**, remove `metricsPath` from `internalPaths` so that external accesses are at least visible in info-level logs until proper access control is added.

### Proof of Concept
```bash

# 2. From any external host with network access:
curl -s http://<rosetta-host>:8080/metrics | head -40
# ŌåÆ Returns full Prometheus text exposition (200 OK)

# 3. Inspect server logs ŌĆö no info-level entry for the /metrics request appears.
