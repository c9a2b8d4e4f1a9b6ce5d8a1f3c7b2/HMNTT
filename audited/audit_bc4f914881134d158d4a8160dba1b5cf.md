### Title
Slow-Body MITM Attack Causes Unbounded Startup Stall via Unguarded Response-Body Read in `fetchMirrorNodeNetwork`

### Summary
An attacker who can intercept the TCP connection between the pinger and the mirror REST endpoint can send a valid HTTP `200 OK` header immediately, then stall or drip-feed the response body indefinitely. Because `json.NewDecoder(resp.Body).Decode()` blocks on the body read — not on `httpClient.Do()` — and because the retry loop is driven by `context.Background()` (immune to SIGTERM), the pinger startup is blocked for up to `(mirrorNodeClientMaxRetries+1) × mirrorNodeClientTimeout + Σbackoffs` ≈ **621 seconds** with default settings, after which `log.Fatalf` terminates the process and no transactions are ever submitted.

### Finding Description

**Exact code path:**

`pinger/mirror_node_client.go`, `buildNetworkFromMirrorNodes()` → `fetchMirrorNodeNetwork()`:

- Line 46: `httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}` — timeout is set, but it only bounds a single attempt.
- Line 84: `resp, err := httpClient.Do(req)` — returns as soon as response **headers** are received; body is not yet read.
- Line 96: `json.NewDecoder(resp.Body).Decode(&payload)` — **this is where the blocking occurs**. If the attacker sends headers instantly but never completes the body, this call blocks until the per-attempt `mirrorNodeClientTimeout` (default 10 s) fires.

**Retry loop** (`buildNetworkFromMirrorNodes`, lines 52–69): runs for `max(mirrorNodeClientMaxRetries+1, 1)` = **11 attempts** by default, with exponential backoff `500ms × 2^(attempt-1)` between each.

**Critical amplifier** (`sdk_client.go`, line 18):
```go
netmap, err := buildNetworkFromMirrorNodes(context.Background(), cfg)
```
`context.Background()` is passed — **not** the signal-aware context from `main.go`. SIGTERM/SIGINT cannot cancel this loop.

**Root cause:** The code assumes the mirror REST server is trusted and will respond promptly. There is no `io.LimitReader` on the body, no separate per-read deadline, and no cancellable context. The `http.Client.Timeout` bounds each individual attempt but does not prevent an attacker from forcing every attempt to consume the full timeout.

**Total worst-case stall (defaults):**
- 11 attempts × 10 s = 110 s of HTTP timeouts
- Backoffs: 500 ms + 1 s + 2 s + 4 s + 8 s + 16 s + 32 s + 64 s + 128 s + 256 s ≈ 511.5 s
- **Total ≈ 621.5 s** before `newClient()` returns an error and `log.Fatalf` exits the process.

### Impact Explanation

For deployments using `network=other` (the only path that calls `buildNetworkFromMirrorNodes`), the pinger **never submits any transactions** during the stall window and then **exits permanently** after all retries are exhausted. With Hedera's ~1–2 s block time, the 500% threshold is 5–10 s; the 621 s stall exceeds it by ~60–120×. The liveness probe (`/tmp/alive`) continues to be written by a background goroutine, so Kubernetes may not detect the hang and restart the pod promptly.

### Likelihood Explanation

**Preconditions:**
1. `HIERO_MIRROR_PINGER_NETWORK=other` must be configured (required for any non-public network deployment, e.g., private testnets, staging environments).
2. The mirror REST URL uses plain HTTP (common for internal/cluster-local deployments such as `http://mirror-rest:5551`), or the attacker can perform TLS interception (e.g., via a compromised internal CA or a rogue sidecar proxy).
3. The attacker controls a network path: ARP spoofing on the same L2 segment, DNS poisoning if the URL uses a hostname, a compromised ingress/load-balancer, or a rogue container on the same pod network.

All three conditions are realistic in containerized/cloud-native environments. No credentials or privileged OS access are required — only network-path control, which is achievable by an unprivileged attacker on the same cluster network.

### Recommendation

1. **Wrap the body with `io.LimitReader`** before decoding to prevent reading an unbounded stream:
   ```go
   var payload nodesEnvelope
   if err := json.NewDecoder(io.LimitReader(resp.Body, 1<<20)).Decode(&payload); err != nil { ... }
   ```
2. **Pass the cancellable context** from `main.go` into `buildNetworkFromMirrorNodes` so SIGTERM aborts the retry loop:
   ```go
   // sdk_client.go line 18 — replace context.Background() with ctx
   netmap, err := buildNetworkFromMirrorNodes(ctx, cfg)
   ```
3. **Set a separate `ResponseHeaderTimeout`** on the transport to bound the time before headers arrive independently of the body timeout.
4. **Use TLS** (`https://`) for the mirror REST endpoint to prevent unauthenticated MITM body injection.

### Proof of Concept

```
# 1. On attacker machine (same network segment or DNS-poisoned host):
#    Serve a TCP listener that sends HTTP 200 headers then stalls.

python3 - <<'EOF'
import socket, time
s = socket.socket()
s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
s.bind(('0.0.0.0', 5551))
s.listen(1)
print("Listening on :5551")
while True:
    conn, addr = s.accept()
    print(f"Connection from {addr}")
    # Read the HTTP request
    conn.recv(4096)
    # Send valid 200 OK headers, but never send the body
    conn.sendall(b"HTTP/1.1 200 OK\r\nContent-Type: application/json\r\nTransfer-Encoding: chunked\r\n\r\n")
    # Stall — never send body chunks
    time.sleep(3600)
    conn.close()
EOF

# 2. Configure pinger to point at attacker's server:
export HIERO_MIRROR_PINGER_NETWORK=other
export HIERO_MIRROR_PINGER_REST=http://<attacker-ip>:5551

# 3. Start pinger — observe it hangs for ~621 s (11 × 10 s timeouts + backoffs)
#    then exits with log.Fatalf, having submitted zero transactions.
./pinger
```

**Observed result:** `newClient()` blocks for the full retry window; `main.go` line 43 calls `log.Fatalf("client error: ...")` and the process exits. Zero transactions are submitted.
