### Title
Shared Mutable `LinkedList` Backing `TRANSACTION_RECEIPTS_STATE_ID` Queue Enables Cross-Request State Corruption in Concurrent EVM Simulations

### Summary
`MirrorNodeState` is a singleton Spring bean that initializes a single `LinkedList` for the `TRANSACTION_RECEIPTS_STATE_ID` queue at startup. `getWritableStates()` caches and returns the same `ListWritableQueueState` instance (wrapping that shared `LinkedList`) to every caller via `computeIfAbsent`. Because `ListWritableQueueState.removeFromDataSource()` calls `backingStore.remove()` directly on the shared, non-thread-safe `LinkedList`, concurrent EVM simulations mutate each other's receipt data, causing incorrect simulation results and potential `ConcurrentModificationException`.

### Finding Description

**Exact code path:**

`MirrorNodeState` is annotated `@Named` (singleton scope): [1](#0-0) 

A single `LinkedList` is created once at startup and stored as the backing store for `TRANSACTION_RECEIPTS_STATE_ID`: [2](#0-1) 

`getWritableStates()` uses `computeIfAbsent` on the `writableStates` `ConcurrentHashMap`, meaning the **same** `MapWritableStates` (and the same `ListWritableQueueState` wrapping the same `LinkedList`) is returned to every caller for the same service name: [3](#0-2) 

When the EVM framework calls `poll()` on the writable queue, `WritableQueueStateBase` calls `removeFromDataSource()`, which directly removes the head element from the shared `LinkedList`: [4](#0-3) 

**Root cause and failed assumption:**

The design assumes that `getWritableStates()` is called in a single-threaded or per-request-isolated context. This assumption is false: `MirrorNodeState` is a singleton, `writableStates.computeIfAbsent` caches the result permanently, and `LinkedList` is not thread-safe. By contrast, KV states use `ContractCallContext.get().getWriteCacheState()` for per-request isolation: [5](#0-4) 

No equivalent per-request isolation exists for the queue state. The `ListWritableQueueState` has no write buffer — mutations go directly to the shared `LinkedList`.

**Why existing checks are insufficient:**

The `ConcurrentHashMap` for `writableStates` only protects the map's own structural integrity; it does not protect the `LinkedList` stored inside the cached `ListWritableQueueState`. The `LinkedList` itself has no synchronization. There is no copy-on-write, no per-request snapshot, and no rollback mechanism for the queue.

### Impact Explanation

During a contract simulation (e.g., `eth_call`, `eth_estimateGas`), the Hedera EVM framework adds transaction receipts to the `TRANSACTION_RECEIPTS_STATE_ID` queue and later polls them to determine execution status. With the shared `LinkedList`:

1. **Cross-simulation receipt theft**: Simulation A adds receipt R1; Simulation B's `poll()` removes R1 before Simulation A can read it. Simulation A sees an empty queue and behaves incorrectly (wrong status, wrong gas estimate, wrong return value).
2. **`ConcurrentModificationException`**: Concurrent `add()` and `remove()` on a non-thread-safe `LinkedList` can throw unchecked exceptions, crashing the simulation thread.
3. **Incorrect smart contract behavior**: Callers receive wrong simulation results, which can mislead dApps about contract state, gas costs, or revert conditions.

No funds are directly at risk (read-only simulation context), consistent with the Medium severity classification for Layer 0/1/2 unintended smart contract behavior.

### Likelihood Explanation

The mirror node's JSON-RPC web3 API is publicly accessible with no authentication required. Any external user can submit concurrent `eth_call` or `eth_estimateGas` requests. Under any moderate load (two or more concurrent simulations targeting `RecordCacheService` state), the race condition is triggered. It is repeatable and requires no special privileges — only the ability to send HTTP requests to the public endpoint.

### Recommendation

Apply the same per-request isolation pattern used by KV states. Options:

1. **Per-call queue copy**: In `getWritableStates()`, instead of caching the `ListWritableQueueState` in `writableStates`, create a new `ListWritableQueueState` backed by a **fresh copy** of the queue for each call (or use a `ThreadLocal`/`ContractCallContext`-scoped copy).
2. **No-op queue for simulation**: Since the mirror node never persists queue mutations, replace `ListWritableQueueState` with an in-memory, per-request queue (not backed by the shared `LinkedList`), analogous to how `FunctionWritableSingletonState.putIntoDataSource()` is a no-op.
3. **Remove caching for queue writable states**: Do not cache `WritableStates` for services that contain queue states, or invalidate the cache per request.

The fix for KV states (`ContractCallContext`-scoped write cache) should be replicated for queue states.

### Proof of Concept

1. Start the mirror node web3 service.
2. Send two concurrent HTTP requests to the JSON-RPC endpoint:
   ```
   POST /api/v1/contracts/call  (eth_call, any valid contract)
   POST /api/v1/contracts/call  (eth_call, any valid contract)
   ```
3. Both requests reach `getWritableStates(RecordCacheService.NAME)`, which returns the **same cached** `MapWritableStates` containing the **same** `ListWritableQueueState` backed by the **same** `LinkedList`.
4. Request A's EVM execution adds a receipt to the queue via `addToDataSource()` → `backingStore.add()`.
5. Request B's EVM execution calls `poll()` → `removeFromDataSource()` → `backingStore.remove()`, consuming Request A's receipt.
6. Request A's EVM execution calls `poll()` on an empty queue, receives `null`, and produces an incorrect simulation result (wrong status or exception).
7. Alternatively, if both requests call `add()` and `remove()` concurrently on the non-thread-safe `LinkedList`, a `ConcurrentModificationException` or internal `LinkedList` corruption occurs.

```java
    public WritableStates getWritableStates(@NonNull String serviceName) {
        return writableStates.computeIfAbsent(serviceName, s -> {
            final var serviceStates = states.get(s);
            if (serviceStates == null) {
                return new EmptyWritableStates();
            }
            final Map<Integer, Object> data = new ConcurrentHashMap<>();
            for (final var entry : serviceStates.entrySet()) {
                final var stateId = entry.getKey();
                final var state = entry.getValue();
                if (state instanceof Queue<?> queue) {
                    data.put(stateId, new ListWritableQueueState<>(serviceName, stateId, queue));
                } else if (state instanceof ReadableKVState<?, ?>) {
                    data.put(
                            stateId,
                            new MapWritableKVState<>(
                                    serviceName,
                                    stateId,
                                    getReadableStates(serviceName).get(stateId)));
                } else if (state instanceof SingletonState<?> ref) {
                    data.put(stateId, new FunctionWritableSingletonState<>(serviceName, stateId, ref));
                }
            }
            return new MapWritableStates(data, () -> readableStates.remove(serviceName));
        });
    }
```
