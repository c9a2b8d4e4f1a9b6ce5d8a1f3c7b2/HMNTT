### Title
Unauthenticated Access to Opcode Tracer Endpoint Exposes Contract Storage Write Traces

### Summary
The `getContractOpcodes()` endpoint in `OpcodesController` is publicly accessible with no authentication or authorization check. Any unprivileged external user can submit any known `transactionIdOrHash` with `storage=true` to trigger a full EVM replay of that transaction and receive the exact storage slot keys and written values from every SSTORE operation executed during it, including intermediate values that differ from the final on-chain state.

### Finding Description
**Code path:**
`web3/src/main/java/org/hiero/mirror/web3/controller/OpcodesController.java`, `getContractOpcodes()`, lines 52–68.

The handler performs exactly three checks before executing the replay:
1. `properties.isEnabled()` — a feature flag (`hiero.mirror.web3.opcode.tracer.enabled`, default `false`). When the feature is deployed and enabled, this gate is open to everyone.
2. `validateAcceptEncodingHeader()` — requires `Accept-Encoding: gzip`. Trivially satisfied by any HTTP client.
3. `throttleManager.throttleOpcodeRequest()` — a **global** token-bucket rate limit (default `opcodeRequestsPerSecond = 1`). This is a shared server-wide counter, not per-user or per-IP; it does not restrict which users may call the endpoint.

There is no `@PreAuthorize`, no Spring Security role check, no API-key validation, and no caller-identity verification anywhere in the controller or its service chain.

When `storage=true` is passed, `AbstractOpcodeTracer.captureStorage()` (lines 75–129 of `AbstractOpcodeTracer.java`) walks every `StorageAccess` recorded by the `RootProxyWorldUpdater` during the replay and emits a map of `{slot_key → written_value}` for every SSTORE encountered. Crucially, it records the **first** written value per key (line 116: `if (!result.containsKey(key))`), meaning it can expose intermediate values that were subsequently overwritten and are therefore invisible in the final on-chain state.

**Root cause:** The failed assumption is that the opcode-tracer endpoint would be protected by network-level controls (e.g., only reachable from internal networks) or that callers would be authenticated before the feature is enabled. Neither constraint is enforced in code.

### Impact Explanation
An attacker who knows (or can enumerate) any transaction ID or hash for a contract that performed SSTORE operations can obtain:
- The exact storage slot keys written during that transaction.
- The exact values written to those slots, including intermediate values overwritten within the same transaction (not visible via `eth_getStorageAt` or the `contract_state` table).
- Full stack and memory traces if `stack=true`/`memory=true` are also set.

For contracts that store sensitive application state (e.g., private keys derived on-chain, commitment schemes, encrypted data before it is hashed, or any value that was written and then deleted in the same transaction), this provides a side-channel that bypasses the normal "final state only" visibility of the blockchain. The replay also re-executes the transaction against historical state, so the exposure is permanent and retroactive for every historical transaction.

### Likelihood Explanation
- Transaction hashes are public (visible on any block explorer or via the mirror node's own `/api/v1/contracts/results` endpoint).
- The only barrier is the feature flag; once the operator enables the tracer for legitimate debugging use, the endpoint is immediately open to the entire internet.
- The global rate limit (1 req/s) does not prevent a single attacker from consuming the full quota; it only limits throughput, not access.
- No special tooling is required — a single `curl` with `Accept-Encoding: gzip` and `?storage=true` suffices.

### Recommendation
1. **Add authentication/authorization**: Protect the endpoint with a Spring Security rule (e.g., `@PreAuthorize("hasRole('ADMIN')")` or an API-key filter) so only authorized operators can invoke it.
2. **Per-caller rate limiting**: Replace or supplement the global bucket with a per-IP or per-authenticated-user bucket so one caller cannot monopolize the quota.
3. **Network-level restriction**: If the tracer is intended only for internal debugging, bind it to a separate internal port or require a reverse-proxy ACL, and document this as a deployment requirement.
4. **Audit log**: Log the caller identity and the queried transaction for every successful opcode-trace request.

### Proof of Concept
```
# Precondition: hiero.mirror.web3.opcode.tracer.enabled=true (operator has enabled the feature)
# Step 1: Obtain any transaction hash from the public mirror node API
TX_HASH=$(curl -s https://<mirror-node>/api/v1/contracts/results?limit=1 \
  | jq -r '.results[0].hash')

# Step 2: Call the opcode endpoint with storage=true as an unprivileged user
curl -s -H "Accept-Encoding: gzip" \
  "https://<mirror-node>/api/v1/contracts/results/${TX_HASH}/opcodes?storage=true&stack=false&memory=false" \
  | gunzip \
  | jq '[.opcodes[] | select(.storage | length > 0) | .storage]'

# Result: JSON array of {slot_key: written_value} maps for every SSTORE in the transaction,
# including intermediate values not visible in final contract state.

    OpcodesResponse getContractOpcodes(
            @PathVariable TransactionIdOrHashParameter transactionIdOrHash,

            validateAcceptEncodingHeader(acceptEncoding);
            throttleManager.throttleOpcodeRequest();

            final var request = new OpcodeRequest(transactionIdOrHash, stack, memory, storage);
            return opcodeService.processOpcodeCall(request);
        }

        throw new ResponseStatusException(HttpStatus.NOT_FOUND);
    }

    protected Map<String, String> captureStorage(
            final MessageFrame frame, final OpcodeContext options, final ContractCallContext context) {
        if (!options.isStorage()) {
            return Collections.emptyMap();
        }

        try {
            if (context.getOpcodeContext().getRootProxyWorldUpdater() == null) {

                var worldUpdater = frame.getWorldUpdater();
                var parent = worldUpdater.parentUpdater().orElse(null);
                while (parent != null) {
                    worldUpdater = parent;
                    parent = worldUpdater.parentUpdater().orElse(null);
                }

                if (!(worldUpdater instanceof RootProxyWorldUpdater rootProxyWorldUpdater)) {
                    // The storage updates are kept only in the RootProxyWorldUpdater.
                    // If we don't have one -> something unexpected happened and an attempt to
                    // get the storage changes from a ProxyWorldUpdater would result in a
                    // NullPointerException, so in this case just return an empty map.
                    return Collections.emptyMap();
                }

                context.getOpcodeContext().setRootProxyWorldUpdater(rootProxyWorldUpdater);
            }

            final var rootProxyWorldUpdater = context.getOpcodeContext().getRootProxyWorldUpdater();
            final var updates = rootProxyWorldUpdater
                    .getEvmFrameState()
                    .getTxStorageUsage(true)
                    .accesses();

            if (updates.isEmpty()) {
                return Collections.emptyMap();
            }

            final var result = new TreeMap<String, String>();
            for (final var storageAccesses : updates) {
                for (final var access : storageAccesses.accesses()) {
                    final var key = hexCache.get(access.key(), Bytes::toHexString);
                    if (!result.containsKey(key)) {
                        final var value = access.writtenValue() != null
                                ? hexCache.get(access.writtenValue(), Bytes::toHexString)
                                : hexCache.get(access.value(), Bytes::toHexString);
                        result.put(key, value);
                    }
                }
            }
            return result;

        } catch (final ModificationNotAllowedException e) {
            return Collections.emptyMap();
        }
    }
