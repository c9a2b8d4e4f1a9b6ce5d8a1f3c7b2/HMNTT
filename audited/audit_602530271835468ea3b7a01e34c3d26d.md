### Title
Gas Throttle Bypass via Estimate Mode: 21x EVM Execution Amplification per Single Throttle Deduction

### Summary
When a user submits a request with `estimate=true`, the gas throttle is deducted exactly once at the controller layer, yet `estimateGas()` internally triggers 1 initial EVM execution plus up to 20 additional binary-search EVM executions — all of which bypass the throttle entirely. This allows any unprivileged caller to amplify their server-side EVM workload by up to 21x per throttle token consumed.

### Finding Description

**Entry point — single throttle deduction:**

`ContractController.call()` calls `throttleManager.throttle(request)` exactly once before delegating to `processCall()`. This deducts 1 token from `rateLimitBucket` and `scaleGas(request.getGas())` tokens from `gasLimitBucket`. [1](#0-0) 

**Estimate branch — 1 + up to 20 EVM executions:**

Inside `ContractExecutionService.processCall()`, when `params.isEstimate()` is true, `estimateGas()` is called. It first calls `callContract(params, context)` (1 full EVM execution), then feeds the result into `binaryGasEstimator.search()` which loops up to `maxGasEstimateRetriesCount = 20` times, each iteration calling `doProcessCall(params, gas, true)`. [2](#0-1) [3](#0-2) 

**Binary search iterations bypass throttle entirely:**

`doProcessCall()` only calls `restoreGasToBucket()` and updates metrics when `estimate == false`. When `estimate == true` (all 20 binary search iterations), the throttle is never touched — no deduction, no restoration. [4](#0-3) 

**Max iterations confirmed at 20:**

`BinaryGasEstimator.search()` loops while `iterationsMade < properties.getMaxGasEstimateRetriesCount()`, which defaults to 20. [5](#0-4) 

**Root cause:** The throttle design assumes 1 throttle deduction = 1 EVM execution. The estimate path violates this assumption by triggering up to 21 EVM executions per deduction, with 20 of them completely invisible to the throttle.

### Impact Explanation

The default `requestsPerSecond = 500` and `gasPerSecond = 7,500,000,000` are calibrated for 1 EVM execution per request. An attacker sending 500 estimate requests/second (each with `gas = 15,000,000`, the maximum) passes both throttle checks but triggers up to **10,500 EVM executions/second** instead of the intended 500. This is a **21x CPU/memory amplification** that can exhaust server resources, degrade service for legitimate users, or cause OOM conditions under sustained load — all without any privileged access. [6](#0-5) [7](#0-6) 

### Likelihood Explanation

No authentication, special role, or privileged account is required. Any caller with network access to `POST /api/v1/contracts/call` can trigger this by setting `"estimate": true` in the JSON body. The attack is trivially scriptable, fully repeatable, and requires no on-chain state. The amplification factor is deterministic and bounded only by `maxGasEstimateRetriesCount`. [8](#0-7) 

### Recommendation

Apply one or more of the following mitigations:

1. **Charge the throttle proportionally for estimate requests.** Before entering `estimateGas()`, deduct `maxGasEstimateRetriesCount` additional tokens from `rateLimitBucket`, or multiply the gas deduction by the expected iteration count.
2. **Deduct per binary-search iteration.** Inside `BinaryGasEstimator.search()`, call `throttleManager` for each iteration so every EVM execution is accounted for.
3. **Apply a separate, lower rate limit for estimate requests.** Add a dedicated `estimateRateLimitBucket` with a capacity calibrated to the 21x amplification factor (e.g., `requestsPerSecond / maxGasEstimateRetriesCount`).
4. **Reduce `maxGasEstimateRetriesCount`** to a value that keeps the amplification within the throttle's safety margin.

### Proof of Concept

```bash
# Single request — triggers 1 throttle deduction but up to 21 EVM executions
curl -X POST http://<mirror-node>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{
    "to": "<deployed_contract_address>",
    "data": "<calldata_for_expensive_function>",
    "gas": 15000000,
    "estimate": true
  }'

# Amplification attack — flood with estimate requests at the rate limit
for i in $(seq 1 500); do
  curl -s -X POST http://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{"to":"<contract>","data":"<calldata>","gas":15000000,"estimate":true}' &
done
# Result: 500 throttle tokens consumed, up to 10,500 EVM executions triggered
```

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }

    protected final EvmTransactionResult doProcessCall(
            CallServiceParameters params, long estimatedGas, boolean estimate) throws MirrorEvmTransactionException {
        EvmTransactionResult result = null;
        var status = ResponseCodeEnum.SUCCESS.toString();

        try {
            result = transactionExecutionService.execute(params, estimatedGas);

            if (!estimate) {
                validateResult(result, params);
            }
        } catch (IllegalStateException | IllegalArgumentException e) {
            throw new MirrorEvmTransactionException(e.getMessage(), EMPTY);
        } catch (MirrorEvmTransactionException e) {
            // This result is needed in case of exception to be still able to call restoreGasToBucket method
            result = e.getResult();
            status = e.getMessage();
            throw e;
        } finally {
            if (!estimate) {
                restoreGasToBucket(result, params.getGas());

                // Only record metric if EVM is invoked and not inside estimate loop
                if (result != null) {
                    updateMetrics(params, result.gasUsed(), 1, status);
                }
            }
        }
        return result;
    }
