### Title
Unbounded HTTP Method Label Cardinality Inflation via Arbitrary Method Names in `metricsMiddleware`

### Summary
In `rest/middleware/metricsHandler.js`, the `metricsMiddleware()` function reads `req.method` directly from the incoming HTTP request and uses it as a raw label value in all four per-route OpenTelemetry instruments without any allowlisting or normalization. An unprivileged attacker can send requests with an unbounded set of syntactically valid but non-standard HTTP method tokens (e.g., `PROPFIND`, `SEARCH`, `CUSTOM`, `AAAA`, `BBBB`, …), causing the OpenTelemetry SDK to accumulate an ever-growing set of unique label combinations in memory, leading to unbounded heap growth and eventual OOM crash of the REST API process.

### Finding Description
**Exact code path:**

`rest/middleware/metricsHandler.js`, function `metricsMiddleware()`, lines 191–210:

```js
const method = req.method;           // line 191 — raw, unsanitized
const labels = {method, path, code}; // line 192

requestTotalCounter.add(1, labels);                                          // line 207
durationHistogram.record(duration, labels);                                  // line 208
requestSizeHistogram.record(parseInt(req.headers['content-length']…), labels); // line 209
responseSizeHistogram.record(responseSize, labels);                          // line 210
```

**Root cause:** `req.method` is taken verbatim from the HTTP request. Node.js's `llhttp` parser accepts any RFC 7230 token as a method name — this includes arbitrary uppercase/lowercase ASCII strings like `PROPFIND`, `SEARCH`, `FOOBAR`, `AAAA`, etc. There is no allowlist (e.g., `['GET','POST','PUT','DELETE','PATCH','HEAD','OPTIONS']`), no normalization, and no cap on distinct label values.

**Failed assumption:** The code implicitly assumes `req.method` will only ever be one of the small, fixed set of standard HTTP verbs. This assumption is wrong; the HTTP spec and Node.js both permit arbitrary token strings as method names.

**Exploit flow:**
1. Attacker sends HTTP requests with a rotating set of method names: `AAAA`, `AAAB`, `AAAC`, … (all valid HTTP tokens, no special characters needed).
2. Each unique method value, combined with any path and status code, creates a new label combination `{method, path, code}`.
3. The OpenTelemetry SDK (`@opentelemetry/sdk-metrics`) stores a separate in-memory data point per unique label set for each of the 4 instruments (`requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, `responseSizeHistogram`).
4. With N unique methods × M unique paths × K unique status codes × 4 instruments, heap usage grows as O(N·M·K). These data points are never evicted.
5. The process heap grows without bound until OOM.

**Why existing checks fail:** There are no checks. The aggregate counters (`allRequestCounter`, etc.) at lines 195–204 use no labels and are unaffected, but the per-route instruments at lines 207–210 consume all label combinations unconditionally.

### Impact Explanation
The REST API Node.js process will exhaust heap memory and crash (OOM kill), taking down the mirror node REST API entirely. Recovery requires a process restart, but the attack is trivially repeatable. All four per-route histograms/counters are affected simultaneously, multiplying the memory cost. This is a denial-of-service against the public-facing REST API with no authentication required.

### Likelihood Explanation
The attack requires zero privileges — any network-reachable client can send HTTP requests with arbitrary method names. The attack is fully automatable with a simple script (e.g., `curl -X AAAA`, `curl -X AAAB`, …). Node.js's HTTP parser accepts these without error. The attack is repeatable indefinitely and requires no knowledge of the application beyond the existence of a public HTTP endpoint.

### Recommendation
Normalize `req.method` against an explicit allowlist before using it as a label value. Unknown methods should be collapsed to a sentinel value such as `"OTHER"` or `"UNKNOWN"`:

```js
const ALLOWED_METHODS = new Set(['GET','POST','PUT','DELETE','PATCH','HEAD','OPTIONS']);
const method = ALLOWED_METHODS.has(req.method) ? req.method : 'OTHER';
const labels = {method, path, code};
```

This caps the `method` label cardinality to a fixed, small set regardless of what the attacker sends.

### Proof of Concept
```bash
# Send 10,000 requests with unique method names to any REST API endpoint
for i in $(seq 1 10000); do
  METHOD=$(printf 'M%05d' $i)
  curl -s -X "$METHOD" http://<mirror-node-rest>:5551/api/v1/transactions &
done
wait
# Observe heap growth via /metrics or process monitoring; process will OOM
```

Each iteration creates a new label combination in all four per-route instruments. With enough iterations (or combined with path variation), the process heap is exhausted. [1](#0-0)
