### Title
Publicly Documented Default Credentials Grant Unauthenticated Access to Process Memory Metrics

### Summary
The metrics endpoint at `/swagger/metrics/` is protected by HTTP Basic Auth, but the default credentials (`mirror_api_metrics` / `mirror_api_metrics_pass`) are hardcoded in the default configuration and publicly documented. Any external attacker who reads the project documentation can authenticate with these credentials and read real-time process memory gauges including `nodejs_process_memory_heap_used_bytes` and `nodejs_process_memory_rss_bytes`, enabling precision-timed memory exhaustion attacks.

### Finding Description
**Code location:** `rest/middleware/metricsHandler.js`, `authenticate()` lines 137–144; memory gauges registered at lines 125–134.

The `authenticate` function reads credentials from `config.metrics.config`:

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;          // auth entirely disabled → open access
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
``` [1](#0-0) 

When `authentication` is `true` (the default), the gate is only as strong as the secrecy of `username`/`password`. Those values are publicly documented defaults:

| Property | Default value |
|---|---|
| `hiero.mirror.rest.metrics.config.username` | `mirror_api_metrics` |
| `hiero.mirror.rest.metrics.config.password` | `mirror_api_metrics_pass` | [2](#0-1) 

The memory gauges that are exposed once authenticated:

```js
for (const [name, key] of [
  ['nodejs_process_memory_rss_bytes', 'rss'],
  ['nodejs_process_memory_heap_total_bytes', 'heapTotal'],
  ['nodejs_process_memory_heap_used_bytes', 'heapUsed'],
  ...
]) {
  meter.createObservableGauge(name, {unit: 'By'}).addCallback((result) => {
    result.observe(process.memoryUsage()[key]);
  });
}
``` [3](#0-2) 

**Root cause / failed assumption:** The design assumes operators will rotate the default credentials before deployment. Because the defaults are published in the open-source documentation, this assumption fails for any deployment that ships with the out-of-the-box configuration.

**Why the existing check is insufficient:** `tsscmp` provides timing-safe comparison, which is correct, but it compares against a value that is not secret. The authentication gate is structurally sound; the secret it guards is not secret.

### Impact Explanation
An attacker who successfully authenticates with the default credentials receives a full Prometheus text exposition containing live `nodejs_process_memory_heap_used_bytes` and `nodejs_process_memory_rss_bytes` readings. Polling this endpoint at low frequency (e.g., once per second) gives the attacker a real-time view of heap pressure on the transaction processing server. The attacker can then launch a high-cardinality label injection or large-payload flood precisely when heap utilisation is already near its peak, maximising the probability of an OOM kill or severe GC pause with minimal effort. The endpoint also leaks CPU usage percentage and in-flight request counts, further aiding attack timing.

### Likelihood Explanation
Preconditions are minimal: the attacker needs only network access to port 5551 (the default REST API port) and knowledge of the default credentials, both of which are available from the public repository. No account, token, or prior compromise is required. The attack is fully repeatable and scriptable. Deployments that follow the quickstart or Helm chart defaults without explicitly overriding `metrics.config.username`/`password` are vulnerable.

### Recommendation
1. **Remove hardcoded defaults** for `username` and `password` in the default `application.yml`; require operators to supply them explicitly, and fail startup if they are absent or still equal to the documented example values.
2. **Add a startup guard** in `config.js` (alongside `parseDbPoolConfig`) that rejects the well-known default credential strings:
   ```js
   if (metrics.config.authentication &&
       (metrics.config.password === 'mirror_api_metrics_pass')) {
     throw new InvalidConfigError('metrics password must be changed from the default');
   }
   ```
3. **Consider network-level restriction** of the `/swagger/metrics/` path to internal scraper IPs via a reverse proxy, so the endpoint is never reachable from the public internet regardless of credential configuration.
4. **Rotate credentials** on all existing deployments immediately.

### Proof of Concept
```bash
# 1. Encode the publicly documented default credentials
CREDS=$(echo -n 'mirror_api_metrics:mirror_api_metrics_pass' | base64)

# 2. Request the metrics endpoint (default path /swagger/metrics/)
curl -s -H "Authorization: Basic ${CREDS}" \
     http://<mirror-node-host>:5551/swagger/metrics/

# 3. Expected output (excerpt) on a deployment using default credentials:
# nodejs_process_memory_heap_used_bytes{...} 52428800
# nodejs_process_memory_rss_bytes{...} 104857600
# nodejs_process_cpu_usage_percentage{...} 12.4

# 4. Poll at 1-second intervals to track heap pressure:
watch -n1 "curl -s -H 'Authorization: Basic ${CREDS}' \
     http://<mirror-node-host>:5551/swagger/metrics/ \
     | grep nodejs_process_memory_heap_used_bytes"

# 5. Launch memory-exhaustion payload when heap_used_bytes approaches heap_total_bytes
```
