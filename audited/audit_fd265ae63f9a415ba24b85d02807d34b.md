### Title
Plaintext gRPC by Default Enables MITM-Triggered Block Node Deactivation and Block Stream Ingestion Halt

### Summary
The importer's `BlockNodeProperties` defaults `requiresTls` to `false`, causing `ManagedChannelBuilderProviderImpl` to open plaintext gRPC channels to block nodes. A network-path attacker can inject malformed block stream responses that cause `BlockStreamVerifier.verify()` to throw, which propagates back through `BlockNode.streamBlocks()` and triggers `onError()`, marking nodes inactive. Repeating this across all configured block nodes halts block stream ingestion entirely.

### Finding Description

**Root cause — TLS off by default:**

`BlockNodeProperties.requiresTls` is a Java `boolean` primitive, defaulting to `false`. [1](#0-0) 

`ManagedChannelBuilderProviderImpl.get()` calls `builder.usePlaintext()` when `useTls=false`, establishing an unencrypted, unauthenticated gRPC channel. [2](#0-1) 

The channel is built directly from this flag: [3](#0-2) 

**Exploit propagation chain:**

1. A MITM attacker intercepts the plaintext gRPC stream and injects a malformed `SubscribeStreamResponse` containing a `END_OF_BLOCK` message with corrupted block items.

2. `BlockAssembler.onEndOfBlock()` assembles the block and calls `blockStreamConsumer.accept(new BlockStream(...))`. [4](#0-3) 

3. The consumer is the lambda `(stream) -> onBlockStream(stream, node.getProperties().getEndpoint())` from `BlockNodeSubscriber.doGet()`. [5](#0-4) 

4. `AbstractBlockSource.onBlockStream()` calls `blockStreamVerifier.verify(blockFile)` at line 53, which throws `HashMismatchException`, `InvalidStreamFileException`, or `SignatureVerificationException` on the injected malformed data. [6](#0-5) 

5. The exception propagates back through the while loop in `streamBlocks()` and is caught by `catch (Exception ex)`, which calls `onError()`. [7](#0-6) 

6. `onError()` increments the error counter. After `maxSubscribeAttempts` (default: **3**) errors, it sets `active = false` and enforces a `readmitDelay` (default: **1 minute**) cooldown. [8](#0-7) 

7. `BlockNodeSubscriber.getNode()` skips inactive nodes. When all nodes are inactive and none can be force-readmitted with the required block, it throws `BlockStreamException("No block node can provide block " + nextBlockNumber.get())`, halting ingestion. [9](#0-8) 

**Why existing checks are insufficient:**

`BlockStreamVerifier.verify()` performs cryptographic hash-chain and signature verification — these are the correct integrity checks. However, the system cannot distinguish between a legitimately misbehaving block node and a MITM-injected failure. Both paths call `onError()` identically. The node deactivation mechanism was designed for reliability, but it is weaponized here: the attacker does not need to bypass cryptographic verification; they only need to cause it to fail, which is trivially achieved by injecting any malformed protobuf payload over the plaintext channel. [10](#0-9) 

### Impact Explanation

With all block nodes marked inactive, the importer stops processing new blocks. The mirror node's database falls behind the chain, causing stale query results and unavailability of recent transaction data. This constitutes a shutdown of block stream ingestion — matching the "≥30% of network processing nodes" severity threshold — achievable without brute force and without compromising any node's private key.

### Likelihood Explanation

The attacker requires a MITM position on the network path between the importer and at least one block node. This is realistic in:
- Cloud environments (shared VPC, misconfigured security groups)
- Kubernetes clusters without strict network policies
- Environments susceptible to ARP spoofing or DNS poisoning

With `maxSubscribeAttempts=3` (default), only **3 injected malformed responses per node** are needed to deactivate it. The `readmitDelay` of 1 minute means the attacker must sustain the attack at a low rate (3 injections per minute per node) to keep all nodes permanently inactive. This is highly repeatable and requires no special cryptographic capability.

### Recommendation

1. **Change the default**: Set `requiresTls = true` in `BlockNodeProperties` so all channels use TLS unless explicitly overridden. [1](#0-0) 

2. **Differentiate error types in `onError()`**: Cryptographic verification failures (`HashMismatchException`, `SignatureVerificationException`) should not count toward the node deactivation counter, since they may reflect attacker-injected data rather than a faulty node. Only transport/connectivity errors should trigger deactivation. [7](#0-6) 

3. **Enforce TLS in auto-discovery**: `BlockNodeDiscoveryService` should default discovered nodes to `requiresTls=true` unless the endpoint explicitly advertises otherwise. [11](#0-10) 

### Proof of Concept

**Preconditions:**
- Importer configured with one or more block nodes where `requiresTls` is `false` (the default)
- Attacker has MITM position on the network path (e.g., same L2 segment, ARP spoofing, or compromised network device)

**Steps:**

1. Identify the plaintext gRPC stream between the importer and a block node (default port 40840).

2. Intercept a `SubscribeStreamResponse` containing `END_OF_BLOCK` and replace the block items with a valid-looking but hash-mismatched payload (e.g., flip one byte in the block proof hash).

3. Forward the malformed response to the importer. `BlockStreamVerifier.verify()` throws `HashMismatchException`. `BlockNode.onError()` increments the error counter.

4. Repeat step 2–3 two more times (total: 3 injections). The node is marked `active=false` with a 1-minute readmit delay.

5. Repeat for each remaining block node.

6. **Result**: `BlockNodeSubscriber.getNode()` throws `BlockStreamException("No block node can provide block N")`. Block stream ingestion halts. Continue injecting at a rate of 3 per minute per node to prevent readmission and sustain the outage indefinitely.

    private BlockNode getNode(final AtomicLong nextBlockNumber) {
        final var nodeList = getBlockNodes();
        final var inactiveNodes = new ArrayList<BlockNode>();
        for (final var node : nodeList) {
            if (!node.tryReadmit(false).isActive()) {
                inactiveNodes.add(node);
                continue;
            }

            if (hasBlock(nextBlockNumber, node)) {
                return node;
            }
        }

        // find the first inactive node with the block and force activating it
        for (final var node : inactiveNodes) {
            if (hasBlock(nextBlockNumber, node)) {
                node.tryReadmit(true);
                return node;
            }
        }

        throw new BlockStreamException("No block node can provide block " + nextBlockNumber.get());
    }

            verifyHashChain(blockFile);
            verifySignature(blockFile);

            final var recordFile = blockFileTransformer.transform(blockFile);
