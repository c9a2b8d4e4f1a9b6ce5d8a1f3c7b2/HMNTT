### Title
Index Out-of-Range Panic in `getSdkPayerAccountId` via Colon-Free `account_map` Entry

### Summary
In `rosetta/app/services/construction_service.go`, the function `getSdkPayerAccountId` splits each comma-separated entry of the attacker-controlled `account_map` metadata on `:` and unconditionally accesses `mapping[1]` without first verifying the slice has at least two elements. An unprivileged caller can craft a `ConstructionPayloads` request whose `account_map` value matches the payer alias prefix but contains no colon, causing a runtime panic that aborts the request handler before any `ErrInvalidAccount` error is returned.

### Finding Description
**Exact location:** `rosetta/app/services/construction_service.go`, lines 491–501.

```go
for aliasMap := range strings.SplitSeq(accountMap, ",") {
    if !strings.HasPrefix(aliasMap, payerAlias) {
        continue
    }

    var err error
    mapping := strings.Split(aliasMap, ":")   // line 497
    if payer, err = hiero.AccountIDFromString(mapping[1]); err != nil {  // line 498 – no len check
        return zero, errors.ErrInvalidAccount
    }
    break
}
``` [1](#0-0) 

**Root cause:** `strings.Split(s, ":")` returns a one-element slice when `s` contains no colon. The code assumes the result always has at least two elements and directly indexes `mapping[1]` without a bounds check.

**Exploit flow:**

1. The attacker sends a `POST /construction/payloads` request with:
   - `operations[0].account.address` set to a valid hex alias (e.g., `0xdeadbeef…`), so `payerAccountId.HasAlias()` returns `true` and `payerAlias = "0xdeadbeef…"`.
   - `metadata["account_map"]` set to exactly `"0xdeadbeef…"` — the alias string with **no colon**.

2. Inside `getSdkPayerAccountId`:
   - `accountMapMetadata != nil` → passes the nil check (line 480).
   - Type assertion to `string` succeeds (line 484).
   - `strings.SplitSeq` yields the single token `"0xdeadbeef…"`.
   - `strings.HasPrefix("0xdeadbeef…", "0xdeadbeef…")` → `true`; the `continue` is skipped.
   - `strings.Split("0xdeadbeef…", ":")` → `["0xdeadbeef…"]` (length 1).
   - `mapping[1]` → **runtime panic: index out of range [1] with length 1**.

3. The panic propagates up through the handler. Go's `net/http` server recovers per-connection, so the process does not exit, but the connection is forcibly closed and no structured Rosetta error is returned to the client.

**Why existing checks are insufficient:**

- The nil check (line 480) and type assertion (line 484) only guard against absent or non-string metadata; they do not validate the internal format of the string.
- The `strings.HasPrefix` guard (line 492) is the exact condition that *enables* the panic: it passes when the attacker supplies the bare alias with no colon.
- The existing test suite (`TestConstructionPayloadsAliasError`) covers `"foobar:0.0.100"` (no prefix match) and `"%s:abc"` (bad account id after colon), but **never tests a matching prefix with no colon at all**. [2](#0-1) 

### Impact Explanation
Any unauthenticated caller of the `/construction/payloads` endpoint can trigger a per-request panic. While Go's HTTP server prevents a full process crash, the panic: (a) bypasses the intended `ErrInvalidAccount` error path, (b) returns no structured response to the client, and (c) can be repeated at will to degrade service availability. In a high-throughput deployment this constitutes a reliable, low-cost denial-of-service against the construction workflow.

### Likelihood Explanation
The `/construction/payloads` endpoint is a public, unauthenticated HTTP endpoint. The attacker needs only to know the hex alias of the payer account (which they themselves supply in `operations[0]`) and to omit the colon from the `account_map` value. No credentials, tokens, or prior state are required. The attack is trivially repeatable with a single HTTP request.

### Recommendation
Before accessing `mapping[1]`, verify the split produced at least two parts:

```go
mapping := strings.Split(aliasMap, ":")
if len(mapping) < 2 {
    return zero, errors.ErrInvalidAccount
}
if payer, err = hiero.AccountIDFromString(mapping[1]); err != nil {
    return zero, errors.ErrInvalidAccount
}
```

Alternatively, use `strings.Cut(aliasMap, ":")` which returns an explicit `found bool` and avoids the slice-indexing pattern entirely:

```go
_, after, found := strings.Cut(aliasMap, ":")
if !found {
    return zero, errors.ErrInvalidAccount
}
if payer, err = hiero.AccountIDFromString(after); err != nil {
    return zero, errors.ErrInvalidAccount
}
```

A corresponding test case with a colon-free matching entry should be added to `TestConstructionPayloadsAliasError`.
