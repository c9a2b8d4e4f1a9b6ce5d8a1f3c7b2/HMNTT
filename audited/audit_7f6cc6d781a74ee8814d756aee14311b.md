### Title
Log Amplification via `eth_estimateGas` Binary Search Loop When Tracing Is Enabled

### Summary
When `hiero.mirror.web3.evm.trace.enabled=true` is set by an operator, `MirrorOperationActionTracer.tracePostExecution()` emits a `log.info()` line for every EVM opcode executed. Because `stateFilterCheck()` returns `false` (no filtering) when `status` is empty (the default), and because the `eth_estimateGas` binary search loop executes the EVM up to 21 times per request with no check to suppress tracing during estimate iterations, any unprivileged external user can multiply log volume by up to 21× per request. At the default 500 RPS limit, this produces catastrophic log I/O.

### Finding Description

**Exact code path:**

`TraceProperties.stateFilterCheck()` — [1](#0-0) 

Returns `false` (meaning "do not filter, proceed to log") whenever `status` is empty, which is the default. So when `enabled=true`, every opcode state passes through.

`MirrorOperationActionTracer.tracePostExecution()` — [2](#0-1) 

Calls `log.info(...)` with full frame data for every opcode after passing the `enabled` and `stateFilterCheck` gates. There is no check for whether the current execution is inside an estimate gas loop.

`ContractExecutionService.estimateGas()` — [3](#0-2) 

Performs 1 initial EVM call via `callContract()`, then passes `gas -> doProcessCall(params, gas, true)` into the binary search loop.

`BinaryGasEstimator.search()` — [4](#0-3) 

Iterates up to `maxGasEstimateRetriesCount = 20` times, invoking a full EVM execution each iteration.

`doProcessCall()` with `estimate=true` — [5](#0-4) 

When `estimate=true`, `restoreGasToBucket` and metric updates are skipped. Critically, there is no mechanism here to suppress the `MirrorOperationActionTracer` during these iterations.

**Root cause:** The `enabled` flag gates tracing globally but there is no `isEstimate` awareness in the tracer. `stateFilterCheck()` with an empty `status` set (the default when tracing is enabled) always returns `false`, meaning all opcode states are logged. The binary search loop calls the full EVM stack — including the tracer — up to 20 additional times per request.

**Failed assumption:** The design assumes tracing is a low-volume diagnostic tool. It does not account for the fact that `eth_estimateGas` multiplies EVM executions by up to 21× per HTTP request, and that each execution fires the tracer for every single opcode.

### Impact Explanation

For a moderately complex contract executing ~5,000 opcodes per run:
- Per `eth_estimateGas` request: up to 21 × 5,000 = **105,000 `log.info()` calls**
- At the default 500 RPS throttle: **52,500,000 log lines/second**

Each log line includes frame type, operation name, call depth, contract address, sender, recipient, remaining gas, revert reason, input, output, and return data — substantial I/O per line. This can exhaust disk space, saturate I/O bandwidth, stall the logging subsystem (blocking the request thread if the appender is synchronous), and degrade or crash the JVM process. Affecting 30%+ of mirror nodes simultaneously is achievable by distributing requests across nodes.

### Likelihood Explanation

The precondition is `enabled=true`, which is non-default but is a documented, supported operator configuration intended for debugging and monitoring. [6](#0-5)  Once enabled, the attack requires zero privileges: `eth_estimateGas` is a public, unauthenticated JSON-RPC endpoint. The attacker needs only a deployed contract address and the ability to send HTTP requests. The attack is trivially repeatable and scriptable.

### Recommendation

1. **Suppress tracing during estimate iterations**: Pass an `isEstimate` flag through the call context (`ContractCallContext`) and check it in `MirrorOperationActionTracer.tracePostExecution()` before emitting any log.
2. **Alternatively**, check `params.isEstimate()` in `doProcessCall()` and set a context flag before invoking `transactionExecutionService.execute()`.
3. **Rate-limit log emission**: Add a per-second cap on the number of trace log lines emitted (e.g., using a token bucket inside the tracer), independent of the RPS throttle.
4. **Document the risk**: The configuration documentation should warn that enabling tracing on a public-facing node with no IP allowlist creates a log amplification surface.

### Proof of Concept

**Preconditions:**
- Mirror node running with `hiero.mirror.web3.evm.trace.enabled=true` and default `hiero.mirror.web3.evm.trace.status=[]`
- A deployed contract address `TARGET_CONTRACT` that executes a non-trivial number of opcodes (e.g., a loop or storage-heavy function)

**Steps:**

```bash
# 1. Craft an eth_estimateGas request targeting the complex contract
PAYLOAD='{"jsonrpc":"2.0","method":"eth_estimateGas","params":[{"to":"TARGET_CONTRACT","data":"0x<function_selector>"}],"id":1}'

# 2. Send at high rate (up to the 500 RPS limit)
for i in $(seq 1 500); do
  curl -s -X POST http://<mirror-node>:8545 \
    -H "Content-Type: application/json" \
    -d "$PAYLOAD" &
done
wait
```

**Observed result:** Each request triggers 1 initial EVM execution + up to 20 binary search iterations. With tracing enabled and `status=[]`, `MirrorOperationActionTracer.tracePostExecution()` fires `log.info()` for every opcode in every iteration. At 500 concurrent requests, log I/O spikes to tens of millions of lines per second, rapidly exhausting disk and degrading node availability.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
