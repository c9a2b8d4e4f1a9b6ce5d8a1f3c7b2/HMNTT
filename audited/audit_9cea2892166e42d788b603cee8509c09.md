### Title
Unauthenticated Access to Full EVM Execution Trace via `/opcodes` Endpoint

### Summary
The `getContractOpcodes()` endpoint in `OpcodesController` performs no authentication or authorization checks, allowing any unprivileged external user to supply any valid `transactionIdOrHash` — including one belonging to another user — and receive the complete EVM execution trace (stack, memory, storage at every opcode step). The only guards are a feature-enabled flag, a gzip header check, and a rate limiter, none of which restrict who can query whose transaction.

### Finding Description
**Code path:**
- `OpcodesController.getContractOpcodes()` — `web3/src/main/java/org/hiero/mirror/web3/controller/OpcodesController.java`, lines 52–68
- `OpcodeServiceImpl.buildCallServiceParameters()` — `web3/src/main/java/org/hiero/mirror/web3/service/OpcodeServiceImpl.java`, lines 76–118

**Root cause:**
`getContractOpcodes()` applies three checks before delegating to `opcodeService.processOpcodeCall()`:

```java
if (properties.isEnabled()) {           // (1) feature flag only
    validateAcceptEncodingHeader(...);   // (2) gzip header only
    throttleManager.throttleOpcodeRequest(); // (3) rate limit only
    final var request = new OpcodeRequest(transactionIdOrHash, stack, memory, storage);
    return opcodeService.processOpcodeCall(request);
}
```

None of these checks verify the identity of the caller or whether the caller has any relationship to the transaction being queried.

In `OpcodeServiceImpl.buildCallServiceParameters()`, the service resolves the transaction purely by the supplied identifier:
- **Hash path**: looks up `contractTransactionHashRepository.findByHash(...)` — no ownership check.
- **Transaction ID path**: looks up `transactionRepository.findByPayerAccountIdAndValidStartNsOrderByConsensusTimestampAsc(payerAccountId, validStartNs)` — the `payerAccountId` comes directly from the attacker-supplied path parameter, not from any authenticated session.

The `TransactionIdOrHashParameter` format (`{payerAccountId}-{validStartSeconds}-{validStartNanos}`) is entirely attacker-controlled. Transaction IDs and hashes are public on Hedera (visible in block explorers and transaction receipts).

**Exploit flow:**
1. Attacker observes victim's transaction ID (e.g., `0.0.12345-1700000000-123456789`) or hash from any public source.
2. Attacker sends:
   ```
   GET /api/v1/contracts/results/0.0.12345-1700000000-123456789/opcodes?stack=true&memory=true&storage=true
   Accept-Encoding: gzip
   ```
3. The service re-executes the victim's transaction on the EVM and returns the full opcode trace including every stack frame, memory word, and storage slot at each step.

**Why checks fail:**
- `properties.isEnabled()` — binary feature flag, not per-user access control.
- `validateAcceptEncodingHeader()` — trivially satisfied by any HTTP client.
- `throttleManager.throttleOpcodeRequest()` — rate-limits volume, does not authenticate identity.
- No Spring Security configuration exists for the web3 module (no `SecurityConfig*.java` found).
- CORS is explicitly open: `registry.addMapping("/api/v1/contracts/**").allowedOrigins("*")`.

### Impact Explanation
When the feature is enabled, any internet-accessible caller can replay any historical contract transaction and receive the full EVM execution trace. This exposes:
- **Stack contents**: function arguments, intermediate computed values, return data.
- **Memory contents**: ABI-encoded call data, return buffers, string/bytes data passed during execution.
- **Storage diffs**: contract storage slot values at each opcode step, revealing internal state that contracts may treat as private.

Contracts that handle sensitive data (e.g., private auction bids encoded in calldata, confidential parameters, access-control logic) have that data fully exposed through the trace. This is a confidentiality breach that cannot be mitigated at the contract level.

### Likelihood Explanation
- **Precondition**: The operator must set `hiero.mirror.web3.opcode.tracer.enabled=true`. This is `false` by default, but the endpoint is explicitly designed to be enabled for debugging/tracing use cases.
- **Attacker capability**: Zero — no credentials, no special network access, no on-chain interaction required. Transaction IDs and hashes are universally public.
- **Repeatability**: Fully repeatable for every historical contract transaction in the database.
- **Feasibility**: A single HTTP request with a known transaction ID suffices.

### Recommendation
Add authentication and authorization before processing the opcode request:
1. **Require authentication**: Protect the endpoint with Spring Security (e.g., API key, JWT, or IP allowlist) so only authorized operators/developers can call it.
2. **Restrict by caller identity**: If the endpoint is intended for end-users, verify that the authenticated caller's account matches the `payerAccountId` of the requested transaction before replaying it.
3. **Deployment-level control**: If this endpoint is strictly an operator/debug tool, expose it only on an internal/admin port or behind a network-level access control, not on the public-facing API surface.

### Proof of Concept
**Preconditions**: `hiero.mirror.web3.opcode.tracer.enabled=true`; victim's transaction ID is known (e.g., from Hashscan or any block explorer).

```bash
# Step 1: Obtain victim's transaction ID from any public source
VICTIM_TX="0.0.12345-1700000000-123456789"

# Step 2: As an unprivileged attacker, replay the victim's transaction
curl -s \
  -H "Accept-Encoding: gzip" \
  "https://<mirror-node-host>/api/v1/contracts/results/${VICTIM_TX}/opcodes?stack=true&memory=true&storage=true" \
  | gunzip

# Result: Full JSON opcode trace including stack[], memory[], storage{} at every EVM step
# revealing all function arguments, intermediate values, and storage state of the victim's tx
```

    @GetMapping(value = "/{transactionIdOrHash}/opcodes")
    OpcodesResponse getContractOpcodes(
            @PathVariable TransactionIdOrHashParameter transactionIdOrHash,
            @RequestParam(required = false, defaultValue = "true") boolean stack,
            @RequestParam(required = false, defaultValue = "false") boolean memory,
            @RequestParam(required = false, defaultValue = "false") boolean storage,
            @RequestHeader(value = HttpHeaders.ACCEPT_ENCODING) String acceptEncoding) {
        if (properties.isEnabled()) {
            validateAcceptEncodingHeader(acceptEncoding);
            throttleManager.throttleOpcodeRequest();

            final var request = new OpcodeRequest(transactionIdOrHash, stack, memory, storage);
            return opcodeService.processOpcodeCall(request);
        }

        throw new ResponseStatusException(HttpStatus.NOT_FOUND);
    }
