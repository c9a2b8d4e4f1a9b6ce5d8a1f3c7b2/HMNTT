### Title
Unbounded Block Index Allows Unauthenticated DB Resource Exhaustion via `AccountBalance`

### Summary
`FindByIndex` in `rosetta/app/persistence/block.go` only validates that the supplied index is non-negative, but imposes no upper-bound check against the current chain tip. An unauthenticated caller can supply `index = math.MaxInt64` in a `BlockIdentifier`, causing a guaranteed-miss SQL query to execute on every request. Because the Rosetta `/account/balance` endpoint requires no authentication, this is trivially repeatable and constitutes a griefing vector against the database connection pool.

### Finding Description

**Code path:**

1. `AccountBalance()` — `rosetta/app/services/account_service.go` lines 52–56: when `request.BlockIdentifier != nil`, delegates to `a.RetrieveBlock(ctx, request.BlockIdentifier)`. [1](#0-0) 

2. `RetrieveBlock()` — `rosetta/app/services/base_service.go` lines 82–83: when only `Index` is set (no `Hash`), calls `b.blockRepo.FindByIndex(ctx, *bIdentifier.Index)`. [2](#0-1) 

3. `FindByIndex()` — `rosetta/app/persistence/block.go` lines 171–181: the **only** guard is `if index < 0`. `math.MaxInt64` (9223372036854775807) is positive, so it passes straight through to `findBlockByIndex`. [3](#0-2) 

4. `findBlockByIndex()` — lines 210–223: checks `index < br.genesisBlock.Index` (also trivially bypassed by `math.MaxInt64`), then executes `selectRecordBlockByIndex` — a parameterized SQL query — against the database. No block at that index exists, so GORM returns `ErrRecordNotFound`, which is mapped to `ErrBlockNotFound` and returned to the caller. [4](#0-3) 

5. The SQL query `selectRecordBlockByIndex` (lines 75–85) performs `WHERE index = @index`. With `@index = math.MaxInt64`, the outer query returns zero rows; the correlated subquery never executes; GORM's `.First()` returns `ErrRecordNotFound`. No overflow occurs — the query simply always misses and consumes a DB connection round-trip. [5](#0-4) 

**Root cause:** `FindByIndex` validates only a lower bound (`< 0`) but never validates an upper bound against the latest known block index. Any index value above the chain tip will always produce a DB query that returns no rows.

**Why existing checks are insufficient:**
- `index < 0` check: does not prevent `math.MaxInt64`.
- `index < br.genesisBlock.Index` check: does not prevent values above the chain tip.
- Traefik `rateLimit: average: 10` in `charts/hedera-mirror-rosetta/values.yaml` (lines 157–160): this is an optional Kubernetes deployment artifact, not enforced at the application layer, and is keyed on `requestHost` (not source IP), making it bypassable by varying the `Host` header. [6](#0-5) 

### Impact Explanation
Each crafted request forces one DB connection acquisition and one index-lookup query that is guaranteed to miss. An attacker sending requests at the Traefik `inFlightReq` limit of 5 concurrent requests per IP (or more from multiple IPs) can continuously saturate the database connection pool, degrading or denying service to legitimate users. There is no economic damage to network participants; the impact is availability degradation of the Rosetta API and increased DB load. Severity is consistent with the stated "Medium: griefing with no economic damage" classification. [7](#0-6) 

### Likelihood Explanation
- **Precondition:** None beyond network access to the Rosetta endpoint. No authentication, no tokens, no privileged role required.
- **Repeatability:** Fully repeatable; the same payload always triggers the same DB query.
- **Attacker capability:** Any script-kiddie with `curl` or a simple loop. The payload is a standard Rosetta JSON body with a single integer field set to its maximum value.
- **Detection difficulty:** The server returns a normal `ErrBlockNotFound` HTTP error response, indistinguishable from a legitimate miss, making it hard to distinguish from benign traffic without index-range anomaly detection.

### Recommendation
Add an upper-bound validation in `FindByIndex` (or in `findBlockByIndex`) that rejects any index greater than the current latest block index before issuing the SQL query. Concretely:

1. After `initGenesisRecordFile`, call `RetrieveLatest` (or cache the latest index) and return `ErrBlockNotFound` immediately if `index > latestBlock.Index`, without touching the DB.
2. Alternatively, add a single pre-flight check in `RetrieveBlock` in `base_service.go`: if `bIdentifier.Index != nil && *bIdentifier.Index > currentTip`, return `ErrBlockNotFound` before delegating to the repository.
3. As a defense-in-depth measure, enforce application-level rate limiting (not just infrastructure-level Traefik config) on the `/account/balance` endpoint.

### Proof of Concept

```bash
# Replace <ROSETTA_HOST> and <VALID_ACCOUNT> with real values.
# No authentication required.

curl -s -X POST http://<ROSETTA_HOST>/account/balance \
  -H "Content-Type: application/json" \
  -d '{
    "network_identifier": {"blockchain": "Hedera", "network": "mainnet"},
    "account_identifier": {"address": "<VALID_ACCOUNT>"},
    "block_identifier": {"index": 9223372036854775807}
  }'
# Returns: {"code":...,"message":"Block not found",...}
# Each call executes: SELECT ... FROM record_file WHERE index = 9223372036854775807
# (guaranteed miss, DB resources consumed)

# Flood loop (no auth, no special privileges):
while true; do
  curl -s -o /dev/null -X POST http://<ROSETTA_HOST>/account/balance \
    -H "Content-Type: application/json" \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"account_identifier":{"address":"<VALID_ACCOUNT>"},"block_identifier":{"index":9223372036854775807}}'
done
```

Every iteration hits `findBlockByIndex` → executes `selectRecordBlockByIndex` with `index = 9223372036854775807` → guaranteed `ErrRecordNotFound` → DB connection consumed → `ErrBlockNotFound` returned to caller.

```go
func (br *blockRepository) FindByIndex(ctx context.Context, index int64) (*types.Block, *rTypes.Error) {
	if index < 0 {
		return nil, hErrors.ErrInvalidArgument
	}

	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	return br.findBlockByIndex(ctx, index)
}
```

```go
func (br *blockRepository) findBlockByIndex(ctx context.Context, index int64) (*types.Block, *rTypes.Error) {
	if index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectRecordBlockByIndex, sql.Named("index", index)).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	return rb.ToBlock(br.genesisBlock), nil
```
