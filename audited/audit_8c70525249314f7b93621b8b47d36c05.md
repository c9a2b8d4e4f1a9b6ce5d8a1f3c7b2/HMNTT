### Title
Unauthenticated `/network/status` Flood Triggers Retriable `ErrDatabaseError`, Enabling Amplified DoS via Client Retry Loop

### Summary
The `NetworkStatus()` handler in `rosetta/app/services/network_service.go` is publicly accessible with no application-level rate limiting. Each request issues multiple raw DB queries; if the database connection pool is exhausted under flood conditions, `addressBookEntryRepo.Entries()` returns `ErrDatabaseError`, which is statically declared `Retriable: true`. Rosetta clients that honor the Rosetta spec will retry indefinitely, and the optional Traefik retry middleware (when enabled) multiplies each attacker request up to 4× at the backend, creating a self-amplifying load loop.

### Finding Description

**Code path:**

`NetworkStatus()` at [1](#0-0)  calls `addressBookEntryRepo.Entries(ctx)` at line 77. That call reaches `Entries()` in [2](#0-1) , which executes up to two raw SQL queries (one per address book file ID). On any DB-level failure the function returns `errors.ErrDatabaseError` at line 69. [3](#0-2) 

**Root cause — `ErrDatabaseError` is `Retriable: true`:**

`ErrDatabaseError` is declared with `retriable: true` at [4](#0-3) . The Rosetta specification requires compliant clients to retry any error whose `retriable` field is `true`. This means every DB error returned from `/network/status` is a signal to clients to immediately re-issue the same request.

**No application-level rate limiting:**

The middleware stack assembled in `main.go` is only `MetricsMiddleware → TracingMiddleware → CorsMiddleware`. [5](#0-4)  There is no in-process rate limiter, concurrency cap, or circuit breaker.

**Traefik middleware is disabled by default:**

The Helm chart defines a middleware chain that includes `inFlightReq: 5`, `rateLimit: 10/s`, and a `retry: attempts: 3` policy, but the entire chain is gated on `{{ if and .Values.global.middleware .Values.middleware }}`. [6](#0-5)  The default value for `global.middleware` is `false`. [7](#0-6)  In a default deployment the Traefik middleware is entirely absent. When it *is* enabled, the `retry: attempts: 3` entry means Traefik itself retries each failed request three additional times, multiplying attacker-originated load by up to 4× before it ever reaches the DB. [8](#0-7) 

**Exploit flow:**

1. Attacker sends a high volume of concurrent unauthenticated POST requests to `/network/status`.
2. Each request causes `RetrieveGenesis` + `RetrieveLatest` + up to two `Entries` DB queries — 3–4 queries per request.
3. The DB connection pool saturates; subsequent `db.Raw(...).Scan(...)` calls return errors.
4. `Entries()` returns `ErrDatabaseError` (`Retriable: true`) to `NetworkStatus()`, which propagates it directly to the caller.
5. Rosetta-compliant clients (exchanges, validators, Rosetta CLI) receive `{"retriable": true}` and retry immediately.
6. If Traefik retry middleware is active, each attacker request is internally retried 3 more times before the error reaches the client, further amplifying DB pressure.
7. The feedback loop sustains DB saturation without the attacker needing to maintain the original flood rate.

### Impact Explanation

A sustained DB connection pool exhaustion degrades all endpoints that share the same `dbClient`, not only `/network/status`. `RetrieveGenesis` and `RetrieveLatest` (called before `Entries`) also return `ErrDatabaseError` under pool exhaustion, so the entire online Rosetta node becomes unavailable. Rosetta clients that depend on `/network/status` to determine the current block tip cannot construct or validate transactions, effectively halting Rosetta-mediated transaction submission for the duration of the attack.

### Likelihood Explanation

The endpoint requires no authentication, no API key, and no session. The attack requires only the ability to send HTTP POST requests — achievable from any host with internet access. The `Retriable: true` amplification means the attacker does not need to sustain the full flood indefinitely; once enough legitimate clients begin retrying, the load is self-sustaining. Default deployments (Traefik middleware disabled) have no automated circuit breaker to interrupt the loop.

### Recommendation

1. **Change `ErrDatabaseError` to `Retriable: false`**, or introduce a separate non-retriable error for transient DB overload conditions, so clients do not amplify load on failure.
2. **Add application-level concurrency limiting** (e.g., a semaphore or `golang.org/x/time/rate` limiter) inside the Go server, independent of any infrastructure middleware, so protection is present in all deployment configurations.
3. **Enable the Traefik middleware by default** (`global.middleware: true`) and ensure the `inFlightReq` and `rateLimit` entries are active in all production deployments.
4. **Remove the Traefik `retry` middleware** from the chain, or scope it only to safe idempotent endpoints, to prevent infrastructure-level retry amplification.

### Proof of Concept

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (aber *addressBookEntryRepository) Entries(ctx context.Context) (*types.AddressBookEntries, *rTypes.Error) {
	db, cancel := aber.dbClient.GetDbWithContext(ctx)
	defer cancel()

	nodes := make([]nodeServiceEndpoint, 0)
	// address book file 101 has service endpoints for nodes, resort to file 102 if 101 doesn't exist
	for _, fileId := range []int64{aber.addressBook101.EncodedId, aber.addressBook102.EncodedId} {
		if err := db.Raw(
			latestNodeServiceEndpoints,
			sql.Named("file_id", fileId),
		).Scan(&nodes).Error; err != nil {
			log.Error("Failed to get latest node service endpoints", err)
			return nil, errors.ErrDatabaseError
		}

		if len(nodes) != 0 {
			break
		}
	}

	entries := make([]types.AddressBookEntry, 0, len(nodes))
	for _, node := range nodes {
		entries = append(entries, node.toAddressBookEntry())
	}

	return &types.AddressBookEntries{Entries: entries}, nil
}
