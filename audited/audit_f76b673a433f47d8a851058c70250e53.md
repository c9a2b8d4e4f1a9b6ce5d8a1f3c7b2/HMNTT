### Title
Adversarial Block Node Can Permanently Evade Block Requests via Deliberate `serverStatus()` Timeout

### Summary
`BlockNode.getBlockRange()` catches all exceptions—including gRPC `DEADLINE_EXCEEDED` from a deliberately slow `serverStatus()` call—and silently returns `EMPTY_BLOCK_RANGE`. Unlike `streamBlocks()`, `getBlockRange()` never calls `onError()`, so a node that consistently times out is never penalized or marked inactive. If that node is the sole provider of certain blocks, those blocks are permanently unindexed by the mirror node.

### Finding Description

**Exact code path:**

`StreamProperties.java` line 35 sets `responseTimeout = Duration.ofMillis(400)` (minimum 100ms per `@DurationMin`). [1](#0-0) 

`BlockNode.getBlockRange()` applies this deadline to the `serverStatus()` call and catches **all** exceptions, returning `EMPTY_BLOCK_RANGE = Range.closedOpen(0L, 0L)` on any failure: [2](#0-1) 

`EMPTY_BLOCK_RANGE` is a Guava `Range.closedOpen(0L, 0L)`, which `.isEmpty()` returns `true`. In `BlockNodeSubscriber.hasBlock()`, this causes the node to be skipped: [3](#0-2) 

`getNode()` iterates all active nodes; if none pass `hasBlock()`, it throws `BlockStreamException`: [4](#0-3) 

**Root cause — failed assumption:** The code assumes that a `getBlockRange()` failure is a transient, self-correcting condition. It is not. There is no call to `onError()` from `getBlockRange()`, so the error counter is never incremented and the node is never marked inactive. The `onError()` / readmit mechanism exists only inside `streamBlocks()`: [5](#0-4) 

An adversarial node can therefore remain permanently in the active node list while always returning `EMPTY_BLOCK_RANGE`, creating a "ghost" node that is polled every cycle but never serves blocks and is never penalized.

**Node discovery surface:** Auto-discovery reads tier-1 block nodes from `registeredNodeRepository`. If `autoDiscoveryEnabled` is true, any on-chain registered node meeting the tier-1 API criteria is automatically added to the pool: [6](#0-5) 

### Impact Explanation

If the adversarial node is the **only** configured/discovered node that holds a particular block range, `getNode()` throws `BlockStreamException("No block node can provide block N")` and the mirror node fails to index those blocks indefinitely. Applications (explorers, wallets, compliance tools) relying on the mirror node for transaction history will see those blocks as missing. While the mirror node itself holds no funds, failure to index fund-transfer transactions causes incorrect balance/history reporting for downstream consumers. The severity is **medium-high** for availability/integrity of the mirror node's data.

### Likelihood Explanation

- **With manual config:** Requires the attacker to already be a trusted, configured block node — low likelihood.
- **With auto-discovery enabled:** Any entity that can register a tier-1 block node on-chain (STATUS + PUBLISH + SUBSCRIBE_STREAM APIs) is automatically added to the pool. On a permissionless network this is a realistic capability for a motivated attacker. The attack is trivially repeatable: simply never respond to `serverStatus()` within 400ms. No special runtime access to the mirror node is needed after initial registration.

### Recommendation

1. **Penalize `getBlockRange()` failures the same way `streamBlocks()` failures are penalized.** Call `onError()` (or an equivalent counter increment) inside the `catch` block of `getBlockRange()` so that a node which repeatedly times out is eventually marked inactive and subject to the readmit delay.
2. **Distinguish "node has no blocks" from "node failed to respond."** Return an `Optional<Range<Long>>` or a dedicated sentinel (not `EMPTY_BLOCK_RANGE`) on exception, and handle it separately in `hasBlock()` — e.g., skip the node for this cycle without treating it as having an empty range.
3. **Raise the minimum `responseTimeout` floor** or make it operator-configurable with a documented security note, since 100ms is trivially beatable on any WAN path.
4. **Audit auto-discovery trust model:** Ensure that auto-discovered nodes are not unconditionally trusted as sole providers; require a quorum or cross-check against a known-good node before accepting a block range as authoritative.

### Proof of Concept

**Preconditions:**
- Auto-discovery is enabled (`blockProperties.isAutoDiscoveryEnabled() == true`).
- Attacker registers a tier-1 block node on-chain (STATUS + PUBLISH + SUBSCRIBE_STREAM endpoints).
- The attacker's node is the only node advertising a specific block range (e.g., blocks 1000–2000).

**Steps:**
1. Attacker's block node accepts the TCP/gRPC connection but deliberately stalls the `serverStatus()` response for > 400ms (e.g., by sleeping in the handler before writing any bytes).
2. Mirror node calls `BlockNode.getBlockRange()` → `withDeadlineAfter(400ms)` fires → gRPC throws `StatusRuntimeException(DEADLINE_EXCEEDED)`.
3. `catch (Exception ex)` at line 108 of `BlockNode.java` catches it, logs an error, and returns `EMPTY_BLOCK_RANGE`.
4. `hasBlock()` evaluates `EMPTY_BLOCK_RANGE.isEmpty()` → `true` → returns `false`.
5. `getNode()` finds no node with the required block → throws `BlockStreamException("No block node can provide block 1000")`.
6. Blocks 1000–2000 are never fetched; all fund transfers in that range are absent from the mirror node's database.
7. Because `onError()` is never called, the node remains `active = true` and the cycle repeats on every polling interval with no recovery.

                    : EMPTY_BLOCK_RANGE;

    private BlockNode getNode(final AtomicLong nextBlockNumber) {
        final var nodeList = getBlockNodes();
        final var inactiveNodes = new ArrayList<BlockNode>();
        for (final var node : nodeList) {
            if (!node.tryReadmit(false).isActive()) {
                inactiveNodes.add(node);
                continue;
            }

            if (hasBlock(nextBlockNumber, node)) {
                return node;
            }
        }

        // find the first inactive node with the block and force activating it
        for (final var node : inactiveNodes) {
            if (hasBlock(nextBlockNumber, node)) {
                node.tryReadmit(true);
                return node;
            }
        }

        throw new BlockStreamException("No block node can provide block " + nextBlockNumber.get());
    }

    private static boolean hasBlock(final AtomicLong nextBlockNumber, final BlockNode node) {
        final var blockRange = node.getBlockRange();
        if (blockRange.isEmpty()) {
            return false;
        }

        if (nextBlockNumber.get() == EARLIEST_AVAILABLE_BLOCK_NUMBER) {
            nextBlockNumber.set(blockRange.lowerEndpoint());
            return true;
        } else {
            return blockRange.contains(nextBlockNumber.get());
        }
    }
