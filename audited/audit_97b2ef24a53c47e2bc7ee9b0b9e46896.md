### Title
Global Shared Rate/Gas Buckets Allow Single-User Exhaustion (DoS Against All Other Callers)

### Summary
`ThrottleManagerImpl.throttle()` enforces rate and gas limits using two process-wide singleton `Bucket` instances with no per-IP or per-user partitioning. A single unprivileged attacker can saturate both buckets within one second, causing every concurrent legitimate caller to receive a `ThrottleException` until the buckets refill. The design assumes fair sharing of a global quota, but nothing enforces that assumption.

### Finding Description
**Code location:** `web3/src/main/java/org/hiero/mirror/web3/throttle/ThrottleManagerImpl.java`, `throttle()`, lines 37–48; bucket construction in `web3/src/main/java/org/hiero/mirror/web3/config/ThrottleConfiguration.java`, lines 24–45.

Both `rateLimitBucket` and `gasLimitBucket` are Spring singleton beans — one shared instance for the entire JVM:

```
rateLimitBucket  → capacity = requestsPerSecond (default 500)
gasLimitBucket   → capacity = scaleGas(gasPerSecond) = 7_500_000_000 / 10_000 = 750_000 tokens
```

In `throttle()`:
```java
if (!rateLimitBucket.tryConsume(1)) { throw ... }
else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) { throw ... }
```

`scaleGas(maxGasLimit)` = `15_000_000 / 10_000` = **1,500 tokens per request**.  
Gas bucket capacity = **750,000 tokens**.  
Requests to exhaust gas bucket = `750_000 / 1_500` = **500 requests** — exactly the per-second rate limit.

An attacker sending 500 requests/second each with `gas = 15_000_000` (the configured `maxGasLimit`) drains both buckets simultaneously within a single 1-second window. No IP address, authentication token, or user identity is inspected anywhere in the throttle path. The `from` field in `ContractCallRequest` is present but never used for throttle partitioning. ### Impact Explanation
Once both buckets are exhausted, every request from every other client receives HTTP 429 (`ThrottleException`) until the next 1-second refill cycle. Because the buckets use `refillGreedy` (burst-at-start), the attacker can re-exhaust them immediately at the start of each new second. This is a sustained, repeatable denial-of-service against the `/api/v1/contracts/call` endpoint (and by extension any `eth_sendRawTransaction` path that flows through the same throttle), affecting all legitimate callers with no ability to distinguish attacker traffic from normal traffic at the application layer. [4](#0-3) 

### Likelihood Explanation
No privileges, accounts, or special network position are required. Any public internet client can open 500 concurrent HTTP connections and send max-gas contract-call requests. The attack is trivially scriptable, costs nothing (requests are rejected after throttle exhaustion so no EVM work is done), and is repeatable every second indefinitely. The only practical barrier is network-layer rate limiting (e.g., a WAF or load balancer), which is external to the application and not guaranteed to be present.

### Recommendation
Replace the single global buckets with per-IP (or per-authenticated-key) buckets, e.g. using a `ConcurrentHashMap<String, Bucket>` keyed on the resolved client IP (extracted from `X-Forwarded-For` / `RemoteAddr`). Retain the global bucket as a secondary hard ceiling. Alternatively, enforce a per-request gas cap that is a fraction of the global budget so no single caller can consume more than `1/N` of the quota, and add IP-level connection limiting at the ingress layer as a defence-in-depth measure. [5](#0-4) 

### Proof of Concept
```bash
# Precondition: mirror-node web3 service running with default config
# maxGasLimit = 15_000_000, requestsPerSecond = 500, gasPerSecond = 7_500_000_000

# Step 1 – Attacker floods 500 max-gas requests in <1 second from a single IP
for i in $(seq 1 500); do
  curl -s -o /dev/null -X POST http://<mirror-node>/api/v1/contracts/call \
    -H 'Content-Type: application/json' \
    -d '{"data":"0x","to":"0x0000000000000000000000000000000000000001","gas":15000000}' &
done
wait

# Step 2 – Legitimate user attempts a call in the same second
curl -X POST http://<mirror-node>/api/v1/contracts/call \
  -H 'Content-Type: application/json' \
  -d '{"data":"0x","to":"0x0000000000000000000000000000000000000001","gas":21000}'

# Expected result: HTTP 429 "Gas per second rate limit exceeded."
# (or "Requests per second rate limit exceeded" if rate bucket hits first)

# Step 3 – Repeat every second to maintain continuous denial of service
```

```java
    public void throttle(ContractCallRequest request) {
        if (!rateLimitBucket.tryConsume(1)) {
            throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
        } else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
            throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
        }

        for (var requestFilter : throttleProperties.getRequest()) {
            if (requestFilter.test(request)) {
                action(requestFilter, request);
            }
        }
```

```java
    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }
```

```java
    @Bean(name = GAS_LIMIT_BUCKET)
    Bucket gasLimitBucket() {
        long gasLimit = throttleProperties.getGasPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(gasLimit)
                .refillGreedy(gasLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder()
                .withSynchronizationStrategy(SynchronizationStrategy.SYNCHRONIZED)
                .addLimit(limit)
                .build();
    }
```
