### Title
Timestamp Range Bounding Bypassed in `getContractLogsById`, Enabling Unbounded Database Scan

### Summary
The `getContractLogsById()` handler in `rest/controllers/contractController.js` calls `extractContractLogsMultiUnionQuery(filters, contractId)` with a non-`undefined` `contractId`. The `optimizeTimestampFilters`/`bindTimestampRange` protection — which enforces a maximum time-window on queries — is gated behind `contractId === undefined` and is therefore **never invoked** for this endpoint. An unprivileged user can supply `timestamp=gte:0.000000000` (or any arbitrarily old timestamp with no upper bound), forcing the database to scan the entire `contract_log` index range for that contract from the beginning of time to now.

### Finding Description

**Code path:**

`getContractLogsById()` (line 779) calls `extractContractLogsMultiUnionQuery(filters, contractId)` (line 802) with a resolved `contractId`.

Inside `extractContractLogsMultiUnionQuery`, the timestamp-bounding logic is:

```js
// rest/controllers/contractController.js, lines 669-686
if (transactionHash !== undefined) {
  ...
} else if (contractId === undefined) {          // ← only path that calls optimizeTimestampFilters
  const {filters: timestampFilters, next} =
    await optimizeTimestampFilters(bounds.primary.getAllFilters(), order);
  ...
}
```

Because `getContractLogsById` always supplies a `contractId`, the `else if (contractId === undefined)` branch is **never taken**, and `optimizeTimestampFilters` is never called.

`optimizeTimestampFilters` (line 384) calls `bindTimestampRange` (in `rest/timestampRange.js`, line 19), which is the only place that enforces `maxTransactionsTimestampRangeNs` — the configured maximum query window. Without it, the raw user-supplied timestamp (e.g., `gte:0.000000000`) is passed directly into the SQL query with no lower-bound correction.

The resulting SQL query (built in `ContractService.getContractLogsQuery`, line 323) issues a `SELECT` against `contract_log` with `consensus_timestamp >= 0` and `contract_id = $1`, scanning every row ever written for that contract.

**Root cause:** The guard `contractId === undefined` was intended to distinguish the "global logs" endpoint from the "per-contract logs" endpoint, but it inadvertently disables the only timestamp-range bounding mechanism for the per-contract path.

**Why the `limit` check is insufficient:** `limit` caps the number of rows *returned*, but the database must still evaluate the full index range to find the first `limit` matching rows. On a high-volume contract, this is a full index scan of potentially millions of rows.

### Impact Explanation
Any unauthenticated/unprivileged caller can issue repeated requests to `GET /api/v1/contracts/{contractId}/results/logs?timestamp=gte:0.000000000` targeting a high-activity contract. Each request forces a full historical scan of `contract_log` for that contract. With concurrent requests this exhausts database I/O and CPU, degrading or halting query processing for all mirror-node consumers — consistent with the ≥30% network processing node shutdown severity described in the scope.

### Likelihood Explanation
No authentication is required. The endpoint is publicly accessible. The exploit requires only knowledge of a high-activity contract address (trivially discoverable from block explorers). It is trivially repeatable and scriptable, making sustained DoS straightforward.

### Recommendation
Remove the `contractId === undefined` gate so that `optimizeTimestampFilters` (and therefore `bindTimestampRange`) is applied for **all** calls to `extractContractLogsMultiUnionQuery`, regardless of whether a `contractId` is present:

```js
// Replace:
} else if (contractId === undefined) {

// With:
} else if (transactionHash === undefined) {
```

Alternatively, apply `bindTimestampRange` unconditionally before building the query, or enforce a mandatory upper-bound timestamp when no explicit upper bound is provided by the caller.

### Proof of Concept

1. Identify any high-activity contract (e.g., a popular ERC-20 on Hedera): `0.0.1234`
2. Send the following request (no authentication needed):
   ```
   GET /api/v1/contracts/0.0.1234/results/logs?timestamp=gte:0.000000000&limit=100
   ```
3. The mirror node builds a SQL query equivalent to:
   ```sql
   SELECT ... FROM contract_log cl
   WHERE cl.contract_id = 1234
     AND cl.consensus_timestamp >= 0
   ORDER BY cl.consensus_timestamp DESC, cl.index DESC
   LIMIT 100
   ```
4. The database scans the entire historical index for contract `1234` from timestamp `0` to present.
5. Repeat concurrently (e.g., 50 parallel requests) to saturate database resources and cause service degradation.

const optimizeTimestampFilters = async (timestampFilters, order) => {
  const filters = [];

  const {range, eqValues, neValues} = utils.parseTimestampFilters(timestampFilters, false, true, true, false, false);
  if (range?.isEmpty()) {
    return {filters};
  }

  const {range: optimizedRange, next} = eqValues.length === 0 ? await bindTimestampRange(range, order) : {range};
  if (optimizedRange?.begin) {
    filters.push({key: filterKeys.TIMESTAMP, operator: utils.opsMap.gte, value: optimizedRange.begin});
  }

  if (optimizedRange?.end) {
    filters.push({key: filterKeys.TIMESTAMP, operator: utils.opsMap.lte, value: optimizedRange.end});
  }

  eqValues.forEach((value) => filters.push({key: filterKeys.TIMESTAMP, operator: utils.opsMap.eq, value}));
  neValues.forEach((value) => filters.push({key: filterKeys.TIMESTAMP, operator: utils.opsMap.ne, value}));

  return {filters, next};
};

```javascript
const bindTimestampRange = async (range, order) => {
  if (!queryConfig.bindTimestampRange) {
    return {range};
  }

  const {maxTransactionsTimestampRangeNs} = queryConfig;
  const boundRange = Range(range?.begin ?? (await getFirstTransactionTimestamp()), range?.end ?? nowInNs(), '[]');
  if (boundRange.end - boundRange.begin + 1n <= maxTransactionsTimestampRangeNs) {
    return {range: boundRange};
  }

  let next;
  if (order === orderFilterValues.DESC) {
    next = boundRange.begin = boundRange.end - maxTransactionsTimestampRangeNs + 1n;
  } else {
    next = boundRange.end = boundRange.begin + maxTransactionsTimestampRangeNs - 1n;
  }

  return {range: boundRange, next: nsToSecNs(next)};
};
```
