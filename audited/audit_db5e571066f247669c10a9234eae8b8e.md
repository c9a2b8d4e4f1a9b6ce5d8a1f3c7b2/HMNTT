### Title
Gas Estimation Loop Amplifies EVM Compute 20x Beyond Throttle Accounting, Enabling Unauthenticated DoS

### Summary
The `BinaryGasEstimator.search()` method executes up to 20 full EVM calls per single gas-throttled request when `estimate=true`. The `ThrottleManager` consumes gas tokens only once at the HTTP layer (based on `request.getGas()`), but the estimation loop runs up to `maxGasEstimateRetriesCount=20` iterations, each executing the full EVM with up to `maxGasLimit=15,000,000` gas. The `safeCall()` wrapper silently swallows all exceptions, ensuring all 20 iterations always complete even when the target contract is gas-exhausting or reverts every time.

### Finding Description

**Code path:**

1. `ContractController.call()` — [1](#0-0)  — calls `throttleManager.throttle(request)` once, consuming gas tokens based on `request.getGas()`, then delegates to `contractExecutionService.processCall(params)`.

2. `ContractCallService.doProcessCall()` — [2](#0-1)  — when `estimate=true`, the `restoreGasToBucket` and metric update are **skipped entirely** (guarded by `if (!estimate)`), meaning no gas is ever returned to the throttle bucket for any of the 20 sub-calls.

3. `BinaryGasEstimator.search()` — [3](#0-2)  — loops `while (lo + 1 < hi && iterationsMade < properties.getMaxGasEstimateRetriesCount())` with default `maxGasEstimateRetriesCount=20`.

4. `safeCall()` — [4](#0-3)  — catches `Exception ignored` and returns `null`. When `null` is returned, `err=true`, `lo=mid`, and the loop **continues** to the next iteration rather than aborting.

5. `EvmProperties` defaults — [5](#0-4)  — `maxGasLimit=15,000,000` and `maxGasEstimateRetriesCount=20`.

**Root cause:** The throttle is a gas-bucket mechanism that accounts for exactly 1 EVM execution per request. For `estimate=true` requests, the actual EVM execution count is up to 20. The throttle is never informed of the 20 sub-executions; `restoreGasToBucket` is explicitly bypassed for estimate calls. The `safeCall()` exception suppression ensures a contract that OOGs or reverts on every iteration still drives all 20 iterations to completion.

**Failed assumption:** The throttle design assumes 1 EVM execution per throttled gas unit. This assumption breaks for `estimate=true` requests, which multiply actual EVM work by up to 20x.

### Impact Explanation

Each request with `estimate=true` and `gas=15,000,000` causes up to 20 full EVM executions, each consuming up to 15M gas worth of CPU/memory. The effective compute cost per throttle token is **20x** the intended limit. A small number of concurrent attackers (or even a single attacker with modest request rate) can saturate the JVM thread pool and CPU, causing the mirror node's web3 API to become unresponsive. Since the mirror node is the sole ETH-compatible JSON-RPC gateway for Hedera, this constitutes a total denial of service for all users relying on `eth_call` and `eth_estimateGas`.

### Likelihood Explanation

No authentication or API key is required. The endpoint `/api/v1/contracts/call` is publicly accessible. The attacker only needs to know (or deploy) any gas-intensive contract on the network — even a simple infinite-loop contract that always OOGs. The attack is trivially repeatable and scriptable. The 20x amplification means the attacker's required request rate to exhaust server resources is 20x lower than what the throttle was designed to handle.

### Recommendation

1. **Account for the estimation multiplier in the throttle:** Consume `request.getGas() * maxGasEstimateRetriesCount` tokens from the throttle bucket for `estimate=true` requests, or restore consumed gas after each sub-call inside the estimation loop.
2. **Abort the estimation loop on repeated failures:** If `safeCall()` returns `null` (exception/OOG) on the first iteration, abort immediately rather than running all 20 iterations. A contract that fails at maximum gas will fail at every midpoint.
3. **Add a per-IP / per-connection rate limit** at the HTTP layer (e.g., via a servlet filter or API gateway) independent of the gas-bucket throttle.
4. **Cap the wall-clock time** of the entire `search()` call with a timeout, interrupting the loop if it exceeds a configurable deadline.

### Proof of Concept

**Preconditions:**
- Mirror node web3 API is publicly reachable.
- A contract exists (or is deployed) at address `0xGAS_BOMB` containing an infinite loop: `while(true) {}`.

**Steps:**

```bash
# Send concurrent estimate requests targeting the gas-bomb contract
for i in $(seq 1 50); do
  curl -s -X POST https://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{
      "to": "0xGAS_BOMB",
      "gas": 15000000,
      "estimate": true,
      "data": "0x"
    }' &
done
wait
```

**Result:** Each of the 50 concurrent requests triggers 20 EVM executions of the infinite-loop contract, each running until OOG at 15M gas. `safeCall()` swallows the OOG exception, and the loop continues all 20 iterations. The server processes 50 × 20 = 1,000 maximum-gas EVM executions simultaneously. The throttle bucket is depleted as if only 50 × 15M = 750M gas was consumed, while actual compute equals 50 × 20 × 15M = 15B gas-equivalent. The web3 API becomes unresponsive to all users.

    protected final EvmTransactionResult doProcessCall(
            CallServiceParameters params, long estimatedGas, boolean estimate) throws MirrorEvmTransactionException {
        EvmTransactionResult result = null;
        var status = ResponseCodeEnum.SUCCESS.toString();

        try {
            result = transactionExecutionService.execute(params, estimatedGas);

            if (!estimate) {
                validateResult(result, params);
            }
        } catch (IllegalStateException | IllegalArgumentException e) {
            throw new MirrorEvmTransactionException(e.getMessage(), EMPTY);
        } catch (MirrorEvmTransactionException e) {
            // This result is needed in case of exception to be still able to call restoreGasToBucket method
            result = e.getResult();
            status = e.getMessage();
            throw e;
        } finally {
            if (!estimate) {
                restoreGasToBucket(result, params.getGas());

                // Only record metric if EVM is invoked and not inside estimate loop
                if (result != null) {
                    updateMetrics(params, result.gasUsed(), 1, status);
                }
            }
        }
        return result;
    }
