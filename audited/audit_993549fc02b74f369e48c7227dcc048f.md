### Title
Symlink Attack on `/tmp/alive` Allows Persistent Liveness Probe Failure via Silent Write Error

### Summary
`pinger/cmd/healthcheck/main.go` uses `os.Stat("/tmp/alive")` which follows symlinks, so an attacker with code execution in the container can replace `/tmp/alive` with a symlink to a mode-000 stale file. The pinger's heartbeat writer silently discards write errors (`_ = os.WriteFile(...)`), so the symlink target's mtime is never refreshed, causing every liveness probe to report "heartbeat stale" and triggering a Kubernetes pod restart after 5 consecutive failures.

### Finding Description
**File:** `pinger/cmd/healthcheck/main.go`, `main()`, lines 27–38
**Writer:** `pinger/main.go`, line 36

`os.Stat` in Go unconditionally follows symlinks and returns the target's metadata. The healthcheck trusts `fi.ModTime()` from that stat call to determine liveness:

```go
fi, err := os.Stat("/tmp/alive")   // follows symlinks
...
age := time.Since(fi.ModTime())
if age > 2*time.Minute {
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
```

The pinger refreshes the file every 15 seconds:

```go
_ = os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644)
```

`os.WriteFile` also follows symlinks on Linux (it calls `open(2)` without `O_NOFOLLOW`). The write error is **silently discarded** with `_`. If `/tmp/alive` is a symlink to a mode-000 file, `os.WriteFile` fails with `EACCES` every tick, the error is thrown away, and the target's mtime is never updated. The healthcheck then always reads the stale mtime through the symlink.

**Exploit flow:**
1. Attacker (user 1000, code execution in container) creates `/tmp/stale_file` and backdates its mtime: `touch -t 202001010000 /tmp/stale_file`
2. Removes write permission: `chmod 000 /tmp/stale_file`
3. Atomically replaces the heartbeat file: `rm /tmp/alive && ln -s /tmp/stale_file /tmp/alive`
4. Pinger ticks every 15 s, calls `os.WriteFile("/tmp/alive", ...)` → follows symlink → `open("/tmp/stale_file", O_WRONLY|O_CREAT|O_TRUNC)` → `EACCES` → error discarded
5. Healthcheck runs every 10 s, calls `os.Stat("/tmp/alive")` → follows symlink → `ModTime` of `/tmp/stale_file` (years old) → `age > 2*time.Minute` → exits 1
6. After `failureThreshold: 5` (50 s), Kubernetes kills and restarts the pod

**Why existing mitigations are insufficient:**
- `readOnlyRootFilesystem: true` does **not** protect `/tmp`; tmpfs mounts remain writable
- `runAsNonRoot: true` / `runAsUser: 1000` does not prevent the container's own user from creating symlinks in `/tmp`
- No `O_NOFOLLOW` flag, no `os.Lstat`, no symlink detection anywhere in the probe path

### Impact Explanation
An attacker who achieves code execution inside the container (e.g., via a vulnerability in the pinger binary or its dependencies) can cause the pod to be continuously restarted by Kubernetes. This is a targeted denial-of-service / griefing attack against the pinger's availability with no economic damage to the network, matching the stated Medium scope.

### Likelihood Explanation
Precondition is code execution inside the container as user 1000. Once that bar is met, the attack is trivially reproducible with three shell commands, requires no privileges beyond what the container user already has, and is persistent across pinger ticks because the write error is silently swallowed. The attack can be re-applied after each pod restart.

### Recommendation
1. **Use `os.Lstat` instead of `os.Stat`** in `pinger/cmd/healthcheck/main.go` line 27. `os.Lstat` does not follow symlinks; if `/tmp/alive` is a symlink, it returns the symlink's own mtime (which is recent if the pinger just created it) or the check can explicitly reject symlinks:
   ```go
   fi, err := os.Lstat("/tmp/alive")
   if err != nil { ... }
   if fi.Mode()&os.ModeSymlink != 0 {
       fmt.Fprintln(os.Stderr, "alive is a symlink")
       os.Exit(1)
   }
   ```
2. **Do not silently discard write errors** in `pinger/main.go` line 36. Log the error so that a failed heartbeat write is observable.
3. **Use `O_NOFOLLOW` when writing** the heartbeat file to prevent the pinger from writing through a symlink.

### Proof of Concept
```bash
# Inside the container as user 1000:

# 1. Create a stale, non-writable target
touch /tmp/stale_file
touch -t 202001010000 /tmp/stale_file   # backdate mtime
chmod 000 /tmp/stale_file

# 2. Replace the heartbeat file with a symlink
rm /tmp/alive
ln -s /tmp/stale_file /tmp/alive

# 3. Observe: pinger silently fails to update mtime every 15s
#    Healthcheck now exits 1 on every invocation:
/healthcheck live
# stderr: heartbeat stale
# exit code: 1

# 4. After 5 x 10s = 50s, Kubernetes restarts the pod
```
