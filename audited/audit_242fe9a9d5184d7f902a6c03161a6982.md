### Title
Gas Throttle Bypass via Non-Existent Block Lookup Enables Unauthenticated DB Query Flooding

### Summary
An unprivileged user can supply any syntactically valid but non-existent `BlockType` (block number or block hash) in a contract call request. When the lazy `blockSupplier` is evaluated during EVM execution, `recordFileService.findByBlockType()` returns `Optional.empty()`, throwing `BlockNumberNotFoundException`. Because this exception is not caught by `doProcessCall`'s catch blocks, the `finally` branch restores the full gas allowance back to the throttle bucket, completely neutralizing the gas-based rate limiter. For block-hash-typed lookups, `findByHash` carries no `@Cacheable` annotation, so every request unconditionally issues a database query. The only remaining protection is the per-second request-count bucket (default 500 req/s), which is insufficient to prevent connection-pool exhaustion under a distributed or sustained attack.

### Finding Description

**Code path:**

1. `ContractController.call()` calls `throttleManager.throttle(request)` (rate + gas buckets consumed), then `contractExecutionService.processCall(params)`. [1](#0-0) 

2. `ContractExecutionService.processCall()` → `callContract(params, ctx)` sets the lazy memoized supplier: [2](#0-1) 

3. `Suppliers.memoize` explicitly does **not** cache exceptions — if the delegate throws, `value.set(newValue)` is never reached, so every subsequent call re-invokes the delegate: [3](#0-2) 

4. During `transactionExecutionService.execute()`, `ContractCallContext.getRecordFile()` is called, which invokes `blockSupplier.get()`: [4](#0-3) 

5. For a non-existent block, `RecordFileServiceImpl.findByBlockType()` returns `Optional.empty()`, and `orElseThrow(BlockNumberNotFoundException::new)` fires: [5](#0-4) 

6. `BlockNumberNotFoundException` extends `InvalidInputException` → `Web3Exception` → `RuntimeException`. It is **not** `IllegalStateException`, `IllegalArgumentException`, or `MirrorEvmTransactionException`, so `doProcessCall`'s catch blocks do not intercept it: [6](#0-5) 

7. The `finally` block runs with `result == null`, so `restoreGasToBucket(null, gasLimit)` restores `gasLimitRefundPercent` (default **100%**) of the gas back to the bucket — completely negating the gas throttle for this failure path: [7](#0-6) 

8. For hash-typed blocks, `findByHash` has **no** `@Cacheable` annotation, so every request unconditionally hits the database: [8](#0-7) 

   By contrast, `findByIndex` is cached (`unless = "#result == null"`), but `Optional.empty()` is not `null`, so an empty result for a given index IS cached after the first hit. An attacker cycling through distinct non-existent block numbers still generates one DB query per unique number.

9. The gas throttle bucket uses `scaleGas`: requests with `gas ≤ 10 000` consume **zero** gas tokens, making the gas bucket entirely irrelevant for small-gas requests: [9](#0-8) 

10. The sole remaining guard is the request-count bucket (default **500 req/s**): [10](#0-9) 

**Root cause:** The design assumes the gas throttle will bound the cost of failed requests, but `BlockNumberNotFoundException` bypasses the gas accounting path entirely. The hash-lookup path additionally lacks any caching layer.

### Impact Explanation

Each attacker can sustain 500 DB queries/second at zero effective gas cost. The `findByHash` query (`WHERE hash LIKE concat(:hash, '%')`) is a prefix-scan that may be slow on large tables. Multiple concurrent attackers multiply the query rate linearly. A finite database connection pool (typically 10–50 connections) can be exhausted when connections are held by slow queries, causing all subsequent legitimate requests to queue or fail with connection-pool timeouts. This constitutes a denial-of-service against the entire web3 API surface, not just historical block queries.

### Likelihood Explanation

No authentication or special privilege is required. The attack requires only the ability to POST to `/api/v1/contracts/call` with a `block` field set to a syntactically valid but non-existent value (e.g., `"0x" + 64 random hex chars`). This is trivially scriptable. The 500 req/s per-IP rate limit can be circumvented by distributing requests across multiple source IPs. The attack is fully repeatable and stateless.

### Recommendation

1. **Validate block existence eagerly** — resolve `recordFileService.findByBlockType()` synchronously at the start of `callContract`, before entering `doProcessCall`, and throw `BlockNumberNotFoundException` before any gas is consumed from the throttle bucket (so no restoration is needed and the gas cost is correctly charged).
2. **Do not restore gas on `BlockNumberNotFoundException`** — treat it like other input-validation failures that should consume throttle budget.
3. **Add caching to `findByHash`** — apply `@Cacheable` with a short TTL and a `unless` condition that excludes empty results, consistent with the other repository methods.
4. **Tighten the rate limit** — consider a lower default for `requestsPerSecond` or add per-IP rate limiting at the ingress/load-balancer layer.

### Proof of Concept

```bash
# Flood with non-existent block hashes (no caching, gas fully restored each time)
while true; do
  curl -s -X POST https://<mirror-node>/api/v1/contracts/call \
    -H 'Content-Type: application/json' \
    -d "{
      \"to\": \"0x0000000000000000000000000000000000000001\",
      \"gas\": 10000,
      \"block\": \"0x$(openssl rand -hex 32)\"
    }" &
done
```

- `gas=10000` → `scaleGas(10000)=0` → gas bucket consumes 0 tokens.
- Random 32-byte hex hash → `findByHash` → uncached DB query every request.
- `BlockNumberNotFoundException` → `finally` restores gas → gas throttle never depletes.
- Only the 500 req/s rate-limit bucket applies; distribute across IPs to bypass it.
- Sustained load exhausts the DB connection pool, causing `QueryTimeoutException` for all concurrent legitimate requests.

```java
    protected final EvmTransactionResult doProcessCall(
            CallServiceParameters params, long estimatedGas, boolean estimate) throws MirrorEvmTransactionException {
        EvmTransactionResult result = null;
        var status = ResponseCodeEnum.SUCCESS.toString();

        try {
            result = transactionExecutionService.execute(params, estimatedGas);

            if (!estimate) {
                validateResult(result, params);
            }
        } catch (IllegalStateException | IllegalArgumentException e) {
            throw new MirrorEvmTransactionException(e.getMessage(), EMPTY);
        } catch (MirrorEvmTransactionException e) {
            // This result is needed in case of exception to be still able to call restoreGasToBucket method
            result = e.getResult();
            status = e.getMessage();
            throw e;
        } finally {
            if (!estimate) {
                restoreGasToBucket(result, params.getGas());

                // Only record metric if EVM is invoked and not inside estimate loop
                if (result != null) {
                    updateMetrics(params, result.gasUsed(), 1, status);
                }
            }
        }
        return result;
    }
```

```java
    private void restoreGasToBucket(EvmTransactionResult result, long gasLimit) {
        // If the transaction fails, gasUsed is equal to gasLimit, so restore the configured refund percent
        // of the gasLimit value back in the bucket.
        final var gasLimitToRestoreBaseline = (long) (gasLimit * throttleProperties.getGasLimitRefundPercent() / 100f);
        if (result == null || (!result.isSuccessful() && gasLimit == result.gasUsed())) {
            throttleManager.restore(gasLimitToRestoreBaseline);
        } else {
            // The transaction was successful or reverted, so restore the remaining gas back in the bucket or
            // the configured refund percent of the gasLimit value back in the bucket - whichever is lower.
            final var gasRemaining = gasLimit - result.gasUsed();
            throttleManager.restore(Math.min(gasRemaining, gasLimitToRestoreBaseline));
        }
    }
```

```java
    public static <T> Supplier<T> memoize(Supplier<T> delegate) {
        final var value = new AtomicReference<T>();
        return () -> {
            final T previousValue = value.get();
            if (previousValue != null) {
                return previousValue;
            }

            final T newValue = delegate.get();
            value.set(newValue);
            return newValue;
        };
    }
```

```java
    public Optional<RecordFile> findByBlockType(BlockType block) {

            return recordFileRepository.findEarliest();

            return recordFileRepository.findLatest();
```
