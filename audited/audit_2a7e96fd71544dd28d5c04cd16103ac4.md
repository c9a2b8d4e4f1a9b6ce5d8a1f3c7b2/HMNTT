### Title
Unauthenticated Mirror-Node Bootstrap Allows Full Network Topology Substitution via DNS/MITM

### Summary
`buildNetworkFromMirrorNodes()` fetches the Hiero node topology over a plain, unauthenticated HTTP connection whose hostname is resolved at runtime. Because the response is decoded and trusted without any validation of the returned addresses, an attacker who can redirect that DNS name (or intercept the plaintext HTTP stream) can supply a crafted `nodesEnvelope` that replaces every consensus-node endpoint with attacker-controlled infrastructure, causing the pinger to submit all signed transactions to the attacker's gRPC servers.

### Finding Description

**Exact code path:**

`pinger/mirror_node_client.go`, `buildNetworkFromMirrorNodes()` (lines 36–72) → `fetchMirrorNodeNetwork()` (lines 74–131).

1. The base URL is taken verbatim from `cfg.mirrorRest` (line 37). The config accepts any scheme, including plain `http://` (e.g. the documented default `http://mirror-rest:5551`). [1](#0-0) 

2. An `http.Client` is created with no TLS configuration, no certificate pinning, and no transport-level authentication (line 46). [2](#0-1) 

3. The JSON response body is decoded directly into `nodesEnvelope` (line 96). [3](#0-2) 

4. Every `DomainName`/`IPAddressV4` + `Port` tuple from `ServiceEndpoints` is inserted into the network map with zero validation — no IP allowlist, no RFC-1918 block, no cross-check against a known-good set (lines 113–124). [4](#0-3) 

5. The only guard is `len(network) == 0` (line 127) — it confirms *something* was returned, not that it is legitimate. [5](#0-4) 

6. The resulting map is passed directly to `hiero.ClientForNetwork(netmap)` in `sdk_client.go` (line 22), making it the sole source of truth for all subsequent gRPC connections. [6](#0-5) 

**Root cause:** The bootstrap URL is resolved via the OS/container DNS resolver at startup with no transport security and no response integrity check. The code's implicit assumption — that the DNS name always resolves to the legitimate mirror node — is not enforced anywhere in the code.

**Exploit flow:**

- **Precondition:** Attacker can poison the DNS cache for the hostname in `cfg.mirrorRest` (e.g. `mirror-rest`) *or* occupy a MITM position on the plaintext HTTP path (same L2 segment, BGP hijack, compromised CoreDNS in the same Kubernetes cluster, etc.).
- **Trigger:** On startup (or any restart), `buildNetworkFromMirrorNodes()` resolves the hostname to the attacker's IP and issues `GET /api/v1/network/nodes`.
- **Attacker response:** Return a valid-looking `nodesEnvelope` where every `service_endpoints` entry points to attacker-controlled gRPC servers (arbitrary IP/domain + port).
- **Result:** `hiero.ClientForNetwork()` is initialized with the fake topology; every `CryptoTransfer` in `transfer.go` is sent to the attacker's infrastructure.

### Impact Explanation

- **Network partition:** The pinger never reaches any legitimate consensus node. All transactions fail or are silently dropped, breaking the liveness/health-check signal the pinger is designed to provide.
- **Transaction interception:** The attacker receives fully-formed, operator-signed `CryptoTransfer` gRPC messages. While the private key is not exposed, the attacker can observe transaction IDs, amounts, and account IDs, and can selectively replay or drop them.
- **Persistent effect:** Because the client is built once at startup and never refreshed, the compromise persists for the entire lifetime of the process.

Severity: **High** — complete loss of the pinger's monitoring function plus exposure of signed transaction traffic.

### Likelihood Explanation

- In Kubernetes, CoreDNS is a shared, cluster-wide component. A pod with `cluster-admin` or DNS-write RBAC (not uncommon in misconfigured clusters) can inject a malicious A record for `mirror-rest` with no interaction from the pinger operator.
- Plain HTTP (no TLS) means any network-adjacent attacker (same node, same VPC subnet, or BGP peer) can MITM without touching DNS at all.
- The attack is repeatable on every pod restart or rolling update.
- Classic off-path DNS cache poisoning (Kaminsky-style) is harder but remains practical against resolvers without DNSSEC.

### Recommendation

1. **Enforce HTTPS** for `cfg.mirrorRest` and reject `http://` schemes at config-load time in `loadConfig()`.
2. **Pin the CA or certificate** of the mirror node REST endpoint using a custom `tls.Config` in the `http.Client`.
3. **Validate returned addresses** against an operator-supplied allowlist (CIDR ranges or explicit hostnames) before accepting them into the network map.
4. **Enable DNSSEC** validation in the container's resolver, or supply the mirror node address as a literal IP to eliminate DNS resolution entirely.
5. Optionally, **cross-check** the returned `NodeAccountID` values against a hardcoded or separately-fetched trusted set before using the endpoints.

func buildNetworkFromMirrorNodes(ctx context.Context, cfg config) (map[string]hiero.AccountID, error) {
	base := strings.TrimRight(strings.TrimSpace(cfg.mirrorRest), "/")

	var url string
	if strings.HasSuffix(base, "/api/v1") {
		url = base + "/network/nodes"
	} else {
		url = base + "/api/v1/network/nodes"
	}

	httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}

	attempts := max(cfg.mirrorNodeClientMaxRetries + 1, 1)

	var lastErr error

	for attempt := 1; attempt <= attempts; attempt++ {
		network, retry, err := fetchMirrorNodeNetwork(ctx, httpClient, url)
		if err == nil {
			return network, nil
		}

		lastErr = fmt.Errorf("attempt %d/%d: %w", attempt, attempts, err)
		if !retry || attempt == attempts {
			break
		}

		backoff := cfg.mirrorNodeClientBaseBackoff * time.Duration(1<<(attempt-1))
		select {
		case <-ctx.Done():
			return nil, ctx.Err()
		case <-time.After(backoff):
		}
	}

	return nil, lastErr
}

func fetchMirrorNodeNetwork(
	ctx context.Context,
	httpClient *http.Client,
	url string,
) (map[string]hiero.AccountID, bool, error) {
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		return nil, false, err
	}

	resp, err := httpClient.Do(req)
	if err != nil {
		return nil, true, fmt.Errorf("GET %s failed: %w", url, err)
	}
	defer resp.Body.Close()

	if resp.StatusCode < 200 || resp.StatusCode >= 300 {
		retry := resp.StatusCode == http.StatusTooManyRequests || resp.StatusCode >= 500
		return nil, retry, fmt.Errorf("GET %s returned %s", url, resp.Status)
	}

	var payload nodesEnvelope
	if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
		return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
	}

	network := make(map[string]hiero.AccountID)

	for _, n := range payload.Nodes {
		if n.NodeAccountID == "" {
			continue
		}

		nodeAccountId, err := hiero.AccountIDFromString(n.NodeAccountID)
		if err != nil {
			continue
		}

		// Use service_endpoints for node gRPC (what the SDK wants)
		for _, ep := range n.ServiceEndpoints {
			host := strings.TrimSpace(ep.DomainName)
			if host == "" {
				host = strings.TrimSpace(ep.IPAddressV4)
			}
			if host == "" || ep.Port == 0 {
				continue
			}

			addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
			network[addr] = nodeAccountId
		}
	}

	if len(network) == 0 {
		return nil, false, fmt.Errorf("no usable service_endpoints found from %s", url)
	}

	return network, false, nil
