### Title
Log Injection and `internalPaths` Bypass via Query String in `TracingMiddleware`

### Summary
`TracingMiddleware` captures the full request URI including query string via `request.URL.RequestURI()` and stores it in `path`, but the `internalPaths` map is keyed on bare paths (e.g., `/metrics`). Any request to `/metrics?<arbitrary>` fails the map lookup, causing the log entry to be emitted at `Info` level instead of `Debug`, and the raw query string content is written verbatim into the log message. No authentication or privilege is required.

`RequestURI()` in Go returns the unmodified request URI including the raw query string. The `internalPaths` map only contains the bare path strings. The lookup `internalPaths["/metrics?injected=block_hash_abc"]` evaluates to `false`, so the branch falls through to `log.Info`. The `message` string is built with `fmt.Sprintf` using `path` directly, with no sanitization of newlines, control characters, or query parameter content, enabling classic log injection.

### Impact Explanation
An attacker can:
1. **Bypass debug suppression**: Flood info-level logs with high-frequency metrics/liveness scrape traffic, drowning out legitimate operational log entries.
2. **Inject arbitrary log lines**: By embedding newline characters (`%0a`) or structured log tokens in the query string, an attacker can forge log entries that appear to be legitimate Rosetta API calls (e.g., fake `/block` or `/construction/submit` entries), corrupting audit trails and SIEM-based alerting. For example: `GET /metrics?%0a10.0.0.1 POST /construction/submit (200) in 12ms` produces a second, forged log line indistinguishable from a real one.

### Likelihood Explanation
Preconditions are zero: the Rosetta port is externally reachable (no auth on `/metrics`), and the attack requires only a standard HTTP GET with a crafted query string. It is trivially repeatable and automatable. Any unauthenticated network client qualifies as the attacker.

### Recommendation
Replace `request.URL.RequestURI()` with `request.URL.Path` for the `internalPaths` lookup and for the log message, so query strings are excluded from both the map key comparison and the logged string:

```go
path := request.URL.Path   // "/metrics" regardless of query string
```

If the query string must be logged for debugging purposes, sanitize it separately by stripping or escaping newline and carriage-return characters before including it in the message.
