### Title
Unbounded Prometheus Label Cardinality via Unmatched Routes in `metricsMiddleware`

### Summary
The `metricsMiddleware` in `rest/middleware/metricsHandler.js` uses `toOpenApiPath()` to derive the `path` label for four per-route Prometheus instruments. When a request matches no registered Express route, `req.route` is `undefined` and `res.locals[requestPathLabel]` is never set, causing `toOpenApiPath` to fall back to the raw `req.path`. Because the REST API has no rate-limiting middleware, an unauthenticated attacker can flood the server with requests to unique paths, creating an unbounded number of distinct label combinations that accumulate in process memory until the Node.js process is exhausted.

### Finding Description

**Exact code path:**

`toOpenApiPath` (lines 13–31, `rest/middleware/metricsHandler.js`) resolves the path label in this priority order:

1. `res.locals[requestPathLabel]` — set only by `recordRequestPath` (routes/index.js:16–19) when `req.route?.path` exists, or by specific controller workarounds (e.g., `contractController.js:791,827`).
2. `req.route.path` — set only when Express matched a registered route.
3. **Fallback: `req.path`** — the raw, un-normalized URL path (line 18).

```js
// metricsHandler.js lines 16-18
if (!path) {
  if (!req.route) {
    path = req.path;   // ← raw attacker-controlled path
  }
```

On `res.on('finish')` (line 185), the label set `{method, path, code}` is recorded into all four per-route instruments:

```js
// lines 207-210
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
```

The OpenTelemetry SDK stores a separate time-series entry for every unique label combination. Each histogram also allocates per-bucket counters (4 duration buckets + 4 response-size buckets by default). There is no cap, eviction, or cardinality guard anywhere in the metrics pipeline.

**Why existing checks fail:**

- `recordRequestPath` (routes/index.js:17–19) only fires for matched routes inside the three registered routers (`AccountRoutes`, `BlockRoutes`, `ContractRoutes`). Requests to any other path leave `res.locals[requestPathLabel]` unset.
- The `responseHandler` (responseHandler.js:28–31) throws `NotFoundError` for unmatched routes, but this happens *after* `metricsMiddleware` has already registered the `res.on('finish')` hook and *before* `req.route` is ever populated.
- The REST API (`rest/server.js`) has **no rate-limiting middleware** of any kind. The rate-limiting found in the codebase (`web3/ThrottleConfiguration.java`) belongs to the separate Java web3 service and does not protect the Node.js REST server.

### Impact Explanation

Each unique path value multiplied by HTTP method and status code creates a new in-memory time-series entry across four instruments. With default histogram bucket counts, each unique path consumes O(tens) of heap objects. Sending 1 million requests with unique paths (trivially achievable with a single-threaded HTTP client in minutes) creates millions of live objects that the GC cannot collect because the `MeterProvider` holds strong references to all registered time-series. The result is Node.js heap exhaustion, causing the mirror node REST API to crash or become unresponsive, denying smart contract query service to all users.

### Likelihood Explanation

The attack requires no authentication, no special headers, and no knowledge of the system beyond the public API hostname. Any HTTP client can generate unique paths (e.g., appending a counter or random hex string). The REST API is publicly exposed by design. The attack is repeatable, automatable, and requires no prior access or credentials.

### Recommendation

1. **Normalize unknown paths before labeling.** In `toOpenApiPath`, replace the `req.path` fallback with a static sentinel such as `"unknown"` or `apiPrefix + "/unknown"`:
   ```js
   if (!req.route) {
     path = apiPrefix + '/unknown'; // instead of req.path
   }
   ```
2. **Add a cardinality guard.** Maintain a bounded set of seen label values; if a new `path` value would exceed the limit, map it to `"other"`.
3. **Add rate limiting to the REST API.** Deploy an ingress-level or application-level rate limiter (e.g., `express-rate-limit`) analogous to the `ThrottleConfiguration` already present in the web3 service.

### Proof of Concept

```bash
# Send 100,000 requests to unique unmatched paths (no auth required)
for i in $(seq 1 100000); do
  curl -s "http://<mirror-node-rest>:5551/api/v1/nonexistent/path_${i}" &
done
wait

# Observe heap growth via the metrics endpoint itself:
curl http://<mirror-node-rest>:5551/swagger/metrics/ | grep nodejs_process_memory_heap_used_bytes
```

Each request to `/api/v1/nonexistent/path_N` matches no Express route, so `req.route` is `undefined`. `toOpenApiPath` returns the raw path `/api/v1/nonexistent/path_N` as the label. After N iterations, N distinct time-series entries exist in the `MeterProvider` heap. Repeating at scale exhausts process memory and crashes the REST service.
