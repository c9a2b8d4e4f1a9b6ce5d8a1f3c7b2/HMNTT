### Title
Publicly Documented Default Credentials Enable Unauthenticated Access to Metrics Endpoint Exposing System Error Counters

### Summary
The `metricsMiddleware` in `rest/middleware/metricsHandler.js` protects the `/swagger/metrics/` endpoint with HTTP Basic Auth, but the default credentials (`mirror_api_metrics` / `mirror_api_metrics_pass`) are hardcoded and publicly documented in the repository. Any external user who reads the project documentation can authenticate with these defaults and read `api_all_errors`, `api_all_client_error`, `api_all_server_error`, `api_all_request_in_processing_total`, CPU usage, and memory gauges. Additionally, the `authenticate()` function contains an explicit bypass: when `authentication` is falsy, it returns `true` unconditionally, making the endpoint fully open with zero credentials.

### Finding Description
**Code path:**

`rest/middleware/metricsHandler.js`, `authenticate()` (lines 137–144):
```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;          // ← complete bypass, no credentials needed
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

`metricsMiddleware` (lines 152–164) calls `authenticate()` and, on success, serializes and returns the full Prometheus metric dump including `api_all_errors`, `api_all_client_error`, `api_all_server_error`, `api_all_request_in_processing_total`, and process-level CPU/memory gauges.

**Root cause – two independent weaknesses:**

1. **Default credential disclosure.** `docs/configuration.md` lines 569–571 publicly document the defaults:
   - `hiero.mirror.rest.metrics.config.username` → `mirror_api_metrics`
   - `hiero.mirror.rest.metrics.config.password` → `mirror_api_metrics_pass`
   No `application.yml` override exists in the repository, so any deployment that does not explicitly rotate these credentials is exploitable with zero guessing.

2. **Hard bypass via `authentication: false`.** The `authenticate()` function returns `true` immediately when `authentication` is falsy. The configuration key `hiero.mirror.rest.metrics.config.authentication` is a plain boolean that can be set to `false` via environment variable (`HIERO_MIRROR_REST_METRICS_CONFIG_AUTHENTICATION=false`), completely removing the gate.

**Why existing checks fail:**
The `tsscmp` constant-time comparison is correct and prevents timing attacks on credential comparison, but it is irrelevant when the credentials themselves are publicly known or when the `!authentication` branch is taken before any comparison occurs.

### Impact Explanation
An attacker who successfully reaches `/swagger/metrics/` receives a full Prometheus text dump containing:
- `api_all_errors_total`, `api_all_client_error_total`, `api_all_server_error_total` — real-time error rates
- `api_all_request_in_processing_total` — current in-flight request count (system load indicator)
- `nodejs_process_cpu_usage_percentage`, heap/RSS memory gauges — process health

This constitutes **information disclosure of operational telemetry**. An attacker can poll the endpoint repeatedly to build a time-series picture of system load and error rates, identifying windows of elevated 5xx rates (server stress) or high in-flight counts. While the mirror node is a read-only API and does not itself accept Hedera transactions, this telemetry can be used to time requests to other Hedera infrastructure or to correlate with known stress conditions to craft more effective denial-of-service or query-amplification attacks against the REST API itself.

Severity: **Medium** — information disclosure with no direct write primitive, but enables informed timing of further attacks.

### Likelihood Explanation
- **Precondition:** Attacker can reach port 5551 (the default REST API port). In many deployments this is internet-facing.
- **Effort:** Zero — credentials are in the public documentation of this open-source repository. Any attacker who clones or browses the repo has them.
- **Repeatability:** Fully repeatable; polling is stateless and leaves no authentication log entry distinguishable from legitimate scraping.
- **Alternative path:** If the operator sets `HIERO_MIRROR_REST_METRICS_CONFIG_AUTHENTICATION=false` (e.g., to simplify Prometheus scraping), the endpoint requires no credentials at all.

### Recommendation
1. **Remove default credentials from documentation and code.** Require operators to explicitly set credentials; fail startup if `authentication: true` and credentials are at their default values.
2. **Restrict the metrics endpoint by network, not only by password.** Bind the metrics path to a separate internal-only port or require an IP allowlist middleware before `metricsMiddleware`.
3. **Treat `authentication: false` as a deployment risk.** Emit a loud startup warning (or refuse to start in production mode) when authentication is disabled.
4. **Consider separating the metrics endpoint** from the public REST API port entirely, following the Prometheus best practice of a dedicated scrape port not exposed externally.

```bash

     http://<mirror-node-host>:5551/swagger/metrics/

# Step 2: Parse error counters from the Prometheus text output
curl -s -u mirror_api_metrics:mirror_api_metrics_pass \
     http://<mirror-node-host>:5551/swagger/metrics/ \
  | grep -E 'api_all_errors_total|api_all_client_error_total|api_all_server_error_total|api_all_request_in_processing'

# Step 3: Poll repeatedly to build error-rate time series
while true; do
  curl -s -u mirror_api_metrics:mirror_api_metrics_pass \
       http://<mirror-node-host>:5551/swagger/metrics/ \
    | grep api_all_server_error_total
  sleep 5
done

# Alternative – if authentication is disabled:
curl http://<mirror-node-host>:5551/swagger/metrics/ | grep api_all_errors
```
