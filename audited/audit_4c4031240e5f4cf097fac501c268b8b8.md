### Title
Unauthenticated DB Connection Pool Exhaustion via Unrate-Limited `/network/status` Endpoint

### Summary
`NetworkStatus()` in `rosetta/app/services/network_service.go` issues multiple sequential database queries per request with no application-level rate limiting. The only defined rate-limiting controls are Traefik middleware CRDs that are **disabled by default** (`global.middleware: false`). An unprivileged attacker can sustain a modest request rate that keeps the 100-connection pool fully occupied, starving legitimate users of database access.

### Finding Description

**Code path:**

`NetworkStatus()` at [1](#0-0)  makes three sequential DB-bound calls per request:

1. `n.RetrieveGenesis(ctx)` → `blockRepository.RetrieveLatest` → `initGenesisRecordFile` (cached after first call via `sync.Once`) + `selectLatestWithIndex` query [2](#0-1) 
2. `n.RetrieveLatest(ctx)` → another `selectLatestWithIndex` DB query [3](#0-2) 
3. `n.addressBookEntryRepo.Entries(ctx)` → up to **two** SQL queries (address book 101 then 102 fallback), each calling `GetDbWithContext` [4](#0-3) 

Each `GetDbWithContext` call creates a context with a **20-second statement timeout** (default), meaning a connection can be held for up to 20 s per query: [5](#0-4) 

**Connection pool ceiling:**

`ConnectToDb` sets `SetMaxOpenConns` from config, defaulting to **100 open connections**: [6](#0-5) [7](#0-6) 

**No application-level rate limiting:**

The middleware chain assembled in `main()` is only `MetricsMiddleware → TracingMiddleware → CorsMiddleware` — none impose request or connection limits: [8](#0-7) 

**Infrastructure rate limiting is disabled by default:**

The Traefik middleware CRD (which defines `inFlightReq: amount: 5` per IP and `rateLimit: average: 10` per host) is gated on `global.middleware: false` — the default value: [9](#0-8) [10](#0-9) 

The rate-limit values themselves are defined but never deployed unless an operator explicitly sets `global.middleware: true`: [11](#0-10) 

**Root cause / failed assumption:** The code assumes an external proxy enforces per-IP throttling before requests reach the Go process. That assumption fails in the default Helm deployment and in any direct-access scenario (e.g., port-forward, NodePort, LoadBalancer without Traefik).

### Impact Explanation

With `maxOpenConnections = 100` and a 20-second statement timeout, an attacker sustaining only **5 requests/second** keeps ≈ 100 connections occupied (5 req/s × 20 s/req = 100 in-flight). Once the pool is saturated, every legitimate call to any endpoint that touches the database blocks or returns an error immediately. This is a complete denial-of-service for all DB-backed Rosetta endpoints (`/block`, `/account/balance`, `/construction/metadata`, etc.) with no economic cost to the attacker and no authentication required.

### Likelihood Explanation

The attack requires no credentials, no special knowledge of the network, and no amplification. Any HTTP client capable of sustaining ~5 req/s to a public endpoint can trigger it. The default Helm chart ships with the protection disabled, so any operator who deploys without explicitly enabling `global.middleware` is exposed. The endpoint is publicly documented as part of the Rosetta API specification, making it trivially discoverable.

### Recommendation

1. **Application-level guard (primary fix):** Add an in-process semaphore or token-bucket limiter in the `NetworkStatus()` handler (or as a middleware wrapping all online endpoints) that limits concurrent DB-consuming requests, independent of any infrastructure layer.
2. **Enable middleware by default:** Change `global.middleware` default to `true` in `charts/hedera-mirror-rosetta/values.yaml` so the `inFlightReq` and `rateLimit` Traefik CRDs are deployed out of the box.
3. **Reduce statement timeout or add per-request DB context cancellation:** Lowering `statementTimeout` reduces the window each connection is held, shrinking the attack surface.
4. **Add a response cache for `/network/status`:** Genesis block data is immutable and the latest block changes only every few seconds; a short TTL cache (e.g., 1–2 s) would collapse thousands of concurrent requests into a single DB round-trip.

### Proof of Concept

```bash
# Precondition: rosetta service reachable at $HOST:5700, global.middleware=false (default)
# Step 1 – saturate the pool with 110 concurrent slow requests
for i in $(seq 1 110); do
  curl -s -X POST http://$HOST:5700/network/status \
    -H 'Content-Type: application/json' \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' &
done
wait

# Step 2 – immediately issue a legitimate request; it will block or return a DB error
curl -X POST http://$HOST:5700/network/status \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}'
# Expected: timeout or {"code":500,"message":"Failed to connect to DB","retriable":true}
```

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}

			log.Error("Failed to get latest node service endpoints", err)
			return nil, errors.ErrDatabaseError
		}

		if len(nodes) != 0 {
			break
		}
	}
