### Title
Unbounded Metric Label Cardinality via User-Controlled Path in `toOpenApiPath`

### Summary
The `toOpenApiPath` function in `rest/middleware/metricsHandler.js` uses `req.path` directly as a Prometheus metric label for unmatched routes, with no sanitization or cardinality cap. For any path not starting with `/api/v1`, the function concatenates `apiPrefix + '/' + path`, and since `req.path` always begins with `/`, this produces double-slash labels like `/api/v1//attacker-path`. An unprivileged external user can flood the metrics system with unlimited unique `path` label values, exhausting Prometheus memory and making dashboards unreadable.

### Finding Description
**Code location:** `rest/middleware/metricsHandler.js`, `toOpenApiPath()`, lines 13–31.

```
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];       // not set for unmatched routes
  if (!path) {
    if (!req.route) {
      path = req.path;                           // ← fully user-controlled
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }
  path = path.replace(/:([^/]+)/g, '{$1}');
  if (!path.startsWith(apiPrefix)) {
    return apiPrefix + '/' + path;               // ← double-slash: '/api/v1/' + '/foo'
  }
  return path;
};
```

**Root cause:** The function assumes that when `path` does not start with `apiPrefix`, it is a bare segment without a leading slash. This assumption is wrong: `req.path` in Express always starts with `/`. The concatenation `'/api/v1/' + '/foo'` produces `/api/v1//foo`.

**Exploit flow:**
1. The metricsHandler is registered globally (`app.useExt(metricsHandler())`, `server.js` line 91), so it intercepts every request.
2. For any request to a path with no matching Express route, `req.route` is `undefined`, so `path = req.path` (the raw user-supplied URL path).
3. `toOpenApiPath` returns `apiPrefix + '/' + req.path`, e.g. `/api/v1//attack-unique-1`.
4. This value is used as the `path` label on four per-route instruments: `requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, `responseSizeHistogram` (lines 207–210).
5. Each unique label combination creates a new time series stored in the OpenTelemetry SDK's in-process registry — there is no cardinality limit.

**Why existing checks fail:** The only guard is `!path.startsWith(apiPrefix)` (line 26), which is a routing check, not a sanitization check. It does not prevent arbitrary strings from becoming label values, and it actively produces the double-slash artifact. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation
Every unique path an attacker sends creates four new Prometheus time series (one per histogram/counter instrument). With no rate limiting or cardinality cap, an attacker can exhaust the Node.js process heap by sending requests to millions of distinct paths. Even at moderate volume, dashboards become unreadable because the `path` dimension explodes with double-slash noise labels (`/api/v1//a`, `/api/v1//b`, …) that do not correspond to any real API route. This is a denial-of-service against the observability plane, classified as griefing with no direct economic damage. [4](#0-3) 

### Likelihood Explanation
No authentication, no API key, and no rate limiting is required. Any HTTP client reachable to the server can trigger this. The attack is trivially scriptable: a simple loop sending `GET /a1`, `GET /a2`, … `GET /aN` is sufficient. It is repeatable indefinitely and requires no knowledge of the application internals. [5](#0-4) 

### Recommendation
1. **Fix the double-slash:** Change line 27 to strip the leading slash from `path` before concatenation:
   ```js
   return apiPrefix + '/' + path.replace(/^\//, '');
   ```
2. **Normalize unknown paths to a sentinel:** For requests with no matching route (`req.route` is `undefined`), replace the raw path with a fixed label such as `unknown` or `unmatched` instead of using `req.path`:
   ```js
   if (!req.route) {
     path = 'unknown';
   }
   ```
3. **Add a cardinality cap:** Maintain a bounded set of known label values and fall back to `unknown` for any value not in the set. [6](#0-5) 

### Proof of Concept
```bash
# Send 10 000 requests to unique paths — no credentials needed
for i in $(seq 1 10000); do
  curl -s "http://<mirror-node-host>/attack-path-$i" > /dev/null
done

# Scrape metrics and observe thousands of unique path label values
curl http://<mirror-node-host>/swagger/metrics/ | grep 'api_request_total' | head -20
# Output will contain lines like:
# api_request_total{method="GET",path="/api/v1//attack-path-1",code="404"} 1
# api_request_total{method="GET",path="/api/v1//attack-path-2",code="404"} 1
# ...
```

Each iteration adds four new in-memory time series. At scale this exhausts heap and renders the metrics endpoint unusable. [5](#0-4) [7](#0-6)
