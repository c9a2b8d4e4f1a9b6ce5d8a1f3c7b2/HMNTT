### Title
Whitespace-Only `X-Real-IP`/`X-Forwarded-For` Header Bypasses Empty-String Check, Obfuscating Client IP in Rosetta Audit Logs

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` guards against missing headers with `len(ipAddress) == 0`, which does not catch strings consisting solely of whitespace (e.g., a single tab `\t`). Because the production Traefik ingress is configured with `--entryPoints.web.forwardedHeaders.insecure`, an unprivileged external attacker can supply `X-Real-IP: \t` and have it forwarded verbatim to the rosetta service. The tab character passes the length check and is written directly into the log line for every request, including `/construction/submit`, replacing the real client IP with invisible whitespace and breaking IP-based forensic correlation.

### Finding Description
**Exact code path** — `rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)       // line 64

    if len(ipAddress) == 0 {                        // line 66 — only catches ""
        ipAddress = r.Header.Get(xForwardedForHeader)
    }

    if len(ipAddress) == 0 {                        // line 70 — only catches ""
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    return ipAddress                                // line 74 — returned unsanitised
}
```

`len("\t") == 1`, so a tab-only value passes both guards and is returned as-is.

**Why the proxy does not save you** — `charts/hedera-mirror-common/values.yaml` lines 635–637 configure Traefik with:
```
--entryPoints.web.forwardedHeaders.insecure
--entryPoints.websecure.forwardedHeaders.insecure
```
This flag instructs Traefik to trust and forward all client-supplied forwarded headers without stripping or overriding them. Traefik does **not** set `X-Real-IP` by default, so a client-supplied `X-Real-IP: \t` is passed through unchanged. The rosetta function reads `X-Real-IP` first (line 64); `len("\t") == 1` satisfies neither guard; `"\t"` is returned and embedded in the log line at line 52–53:

```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, ...)   // clientIpAddress == "\t"
```

The test suite (`trace_test.go`) never exercises a whitespace-only header value, so this path has no regression coverage.

### Impact Explanation
Every `/construction/submit` call (the Rosetta transaction-broadcast endpoint) is logged at `INFO` level with the client IP as the first field. Replacing that field with a tab character makes the log line unparseable by standard IP-extraction regexes and SIEM correlation rules. An attacker submitting fraudulent or double-spend transactions can systematically suppress their IP from the audit trail for every submission, eliminating the primary post-incident forensic signal. The nginx docker-compose path is less affected because `proxy_add_x_forwarded_for` appends the real IP to the chain, but the Kubernetes/Traefik path (the stated production deployment) is fully exploitable.

### Likelihood Explanation
The precondition is zero-privilege: any HTTP client reachable by the Traefik ingress can set an arbitrary `X-Real-IP` header. The Traefik `insecure` flag is explicitly present in the shipped chart values. The attack is trivially repeatable with a single extra request header and requires no authentication, no special tooling, and no knowledge of internal state. An attacker planning a fraudulent `/construction/submit` sequence would naturally include this header to reduce traceability.

### Recommendation
Replace both `len(ipAddress) == 0` guards with a blank/whitespace check, and additionally validate that the extracted value is a syntactically valid IP address before trusting it:

```go
import "strings"
import "net"

func getClientIpAddress(r *http.Request) string {
    for _, header := range []string{xRealIpHeader, xForwardedForHeader} {
        val := strings.TrimSpace(r.Header.Get(header))
        if val == "" {
            continue
        }
        // X-Forwarded-For may be a comma-separated list; take the first entry
        ip := strings.TrimSpace(strings.SplitN(val, ",", 2)[0])
        if net.ParseIP(ip) != nil {
            return ip
        }
    }
    ipAddress, _, _ := net.SplitHostPort(r.RemoteAddr)
    return ipAddress
}
```

Additionally, consider removing `--entryPoints.web.forwardedHeaders.insecure` from the Traefik chart and replacing it with an explicit trusted CIDR list (`--entryPoints.web.forwardedHeaders.trustedIPs=...`) so that only the ingress layer can set forwarded-IP headers.

### Proof of Concept
```bash
# Direct request to rosetta (or through Traefik ingress)
curl -s \
  -H "X-Real-IP: $(printf '\t')" \
  -H "Content-Type: application/json" \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"testnet"},...}' \
  http://<rosetta-host>:5700/construction/submit

# Observe the log line on the rosetta container:
# time="..." level=info msg="	 POST /construction/submit (200) in 3.2ms"
#                              ^--- tab character, not a real IP
```

The leading tab character causes standard log parsers (e.g., `awk '{print $1}'`, Splunk field extraction, Loki label parsing) to emit an empty or whitespace token for the client-IP field, making IP-based correlation queries return no results for this transaction. [1](#0-0) [2](#0-1) [3](#0-2)
