### Title
Unauthenticated UNION ALL Query Amplification via `token_id=lte:X` + `serial_number=lte:Y` on NFT Account Endpoint

### Summary
Any unauthenticated user can supply `token_id=lte:X` and `serial_number=lte:Y` to the `/accounts/:id/nfts` endpoint, causing `NftService.getQuery()` to emit a two-subquery `UNION ALL` SQL statement instead of a single subquery. Because each subquery carries its own `LIMIT` clause and performs an independent index scan, every such request costs roughly twice the database work of a baseline request. Flooding the endpoint with this parameter combination is a trivially repeatable, zero-privilege amplification vector.

### Finding Description

**Code path:**

`accountController.js` `extractNftMultiUnionQuery()` (lines 34–82) parses user-supplied filters into `bounds.primary` (token\_id) and `bounds.secondary` (serial\_number), then calls:

```
lower = this.getLowerFilters(bounds)   // baseController.js:131-146
inner = this.getInnerFilters(bounds)   // baseController.js:154-168
upper = this.getUpperFilters(bounds)   // baseController.js:176-183
```

With `token_id=lte:X` and `serial_number=lte:Y` the bound state is:

| field | value |
|---|---|
| `primary.upper` | `{op: lte, val: X}` |
| `secondary.upper` | `{op: lte, val: Y}` |
| everything else | `null` |

**`validateUpperBounds`** (baseController.js:100–109) rejects only when `primary.upper.operator === 'lt'`; `lte` passes silently. [1](#0-0) 

**`getLowerFilters`**: `secondary.hasBound()` is true, `primary.hasLower()` is false, `primary.hasEqual()` is false → returns `[]`. [2](#0-1) 

**`getInnerFilters`**: both bounds have `hasBound() = true`; `primary.upper` exists and `secondary.hasUpper()` is true → returns `[{token_id < X}]`. [3](#0-2) 

**`getUpperFilters`**: both `primary.hasUpper()` and `secondary.hasUpper()` are true → returns `[{token_id = X}, {serial_number <= Y}]`. [4](#0-3) 

`NftService.getQuery()` then filters out empty arrays and joins the remaining two non-empty arrays with `UNION ALL`: [5](#0-4) 

```sql
(SELECT … WHERE account_id=$1 AND token_id < $3  ORDER BY … LIMIT $2)
UNION ALL
(SELECT … WHERE account_id=$1 AND token_id = $3 AND serial_number <= $4  ORDER BY … LIMIT $2)
ORDER BY … LIMIT $2
```

Each subquery has its own `LIMIT $2` and performs an independent index scan. The outer query adds a third sort+limit pass. A baseline request with no filters produces a single subquery. [6](#0-5) 

No authentication, API key, or elevated privilege is required to reach this endpoint. [7](#0-6) 

### Impact Explanation

Each crafted request forces the database to execute two full subqueries (each bounded by `limit`) plus an outer merge, compared to one subquery for a plain request. An attacker sending N requests/second with this parameter combination imposes ~2× the read I/O and CPU on the database compared to N plain requests. Because the mirror node's database is shared infrastructure, sustained amplification degrades query latency for all consumers of the mirror node API, including services that monitor network state.

### Likelihood Explanation

- **Zero privilege required**: the endpoint is public.
- **Trivial to craft**: a single curl command suffices.
- **Repeatable at scale**: the attacker can parallelize with any HTTP load tool.
- **No visible rate limiting** in the application layer code; infrastructure-level controls (if any) are outside this codebase.
- The parameter combination passes all existing validation checks without modification.

### Recommendation

1. **Reject or restrict the `lte`+`lte` combination** at the validation layer: add a check in `validateUpperBounds` (or a new `validateCombinedBounds`) that throws `InvalidArgumentError` when both `primary.upper` and `secondary.upper` are present without a corresponding `primary.equal`, since this is the only combination that silently produces a multi-subquery plan without a lower bound.
2. **Apply per-IP / per-account rate limiting** at the application layer for all NFT listing endpoints.
3. **Cap the `limit` parameter** to a lower maximum for multi-subquery paths, or detect the multi-subquery case and apply a stricter limit before executing.

### Proof of Concept

```bash
# Triggers a 2-subquery UNION ALL — no credentials needed
curl "https://<mirror-node>/api/v1/accounts/0.0.1234/nfts?token_id=lte:50000000000&serial_number=lte:9999999999&limit=100"

# Flood variant (bash)
for i in $(seq 1 500); do
  curl -s "https://<mirror-node>/api/v1/accounts/0.0.1234/nfts?token_id=lte:50000000000&serial_number=lte:9999999999&limit=100" &
done
wait
```

Each concurrent request causes the database to run two independent index scans plus an outer merge sort, approximately doubling per-request database cost relative to a plain `GET /api/v1/accounts/0.0.1234/nfts` call.

  getLowerFilters(bounds) {
    let filters = [];
    const {primary, secondary} = bounds;
    if (!secondary.hasBound()) {
      // no secondary bound filters or no secondary filters at all, everything goes into the lower part and there
      // shouldn't be inner or upper part.
      filters = [primary.equal, primary.lower, primary.upper, secondary.equal];
    } else if (primary.hasLower() && secondary.hasLower()) {
      // both have lower. If primary has lower and secondary doesn't have lower, the lower bound of primary
      // will go into the inner part.
      filters = [{...primary.lower, operator: utils.opsMap.eq}, secondary.lower];
    } else if (primary.hasEqual()) {
      filters = [primary.equal, primary.lower, primary.upper, secondary.lower, secondary.equal, secondary.upper];
    }
    return filters.filter((f) => !isNil(f));
  }
```

  getInnerFilters(bounds) {
    const {primary, secondary} = bounds;
    if (!primary.hasBound() || !secondary.hasBound()) {
      return [];
    }

    return [
      // if secondary has lower bound, the primary filter should be > ?
      {filter: primary.lower, newOperator: secondary.hasLower() ? utils.opsMap.gt : null},
      // if secondary has upper bound, the primary filter should be < ?
      {filter: primary.upper, newOperator: secondary.hasUpper() ? utils.opsMap.lt : null},
    ]
      .filter((f) => !isNil(f.filter))
      .map((f) => ({...f.filter, operator: f.newOperator || f.filter.operator}));
  }
```
