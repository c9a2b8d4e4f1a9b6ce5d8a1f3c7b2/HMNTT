### Title
Unauthenticated Concurrent Request Flood Exhausts DB Connection Pool via `/network/status`, Denying Transaction Confirmation Service

### Summary
The `NetworkStatus()` handler in `rosetta/app/services/network_service.go` performs two unconditional database queries per request (`RetrieveLatest()` and `addressBookEntryRepo.Entries()`) with no application-level rate limiting or concurrency control. The Traefik middleware that defines `inFlightReq` and `rateLimit` protections is **disabled by default** (`global.middleware: false`). An unprivileged attacker can flood the endpoint with concurrent unauthenticated requests, exhaust the 100-connection database pool, and starve all other Rosetta API operations — including transaction confirmation queries — of database connections.

### Finding Description

**Exact code path:**

`NetworkStatus()` at `rosetta/app/services/network_service.go` lines 59–88 makes three sequential database-bound calls per request: [1](#0-0) 

`RetrieveGenesis()` calls `initGenesisRecordFile()`. After the first successful initialization, `initGenesisRecordFile()` returns immediately via the guard at line 244 (no DB hit). However, `RetrieveLatest()` **always** executes a live database query: [2](#0-1) 

`addressBookEntryRepo.Entries(ctx)` also always hits the database. So every `/network/status` request consumes at least two database connections from the pool.

**Database pool configuration:**

The pool is configured with `MaxOpenConnections = 100` (default per documentation) and `StatementTimeout = 20` seconds: [3](#0-2) [4](#0-3) 

**No application-level rate limiting:**

The application middleware stack (`rosetta/app/middleware/`) contains only `TracingMiddleware`, `MetricsMiddleware`, and health handlers — no concurrency or rate-limit controls: [5](#0-4) 

**Traefik middleware disabled by default:**

The Helm chart defines `inFlightReq: amount: 5` (per source IP) and `rateLimit: average: 10` (per request host), but the entire middleware chain is gated on `global.middleware: true`: [6](#0-5) 

The default value is `false`: [7](#0-6) 

This means any deployment that does not explicitly set `global.middleware: true` (the default) has **zero rate limiting** in front of the endpoint.

**Additional amplifier — retry middleware:**

When the Traefik middleware chain is enabled, it includes `retry: attempts: 3`, which triples the effective request load on the backend for any request that receives an error response: [8](#0-7) 

**Root cause / failed assumption:** The design assumes an external reverse proxy will enforce rate limits, but that assumption is not enforced — the default configuration leaves the application directly exposed without any concurrency control.

### Impact Explanation

With 100 max open connections and each `/network/status` request consuming 2 connections for up to 20 seconds (statement timeout), an attacker sustaining ~50 concurrent requests saturates the entire pool. All other Rosetta API endpoints that require database access — including block queries and construction endpoints used for transaction confirmation — will block waiting for a free connection and eventually time out. For operators and exchanges relying on this Rosetta API to confirm Hedera transactions (e.g., via Coinbase Rosetta integration), this constitutes a complete denial of the transaction confirmation service for the duration of the attack.

### Likelihood Explanation

The endpoint is unauthenticated by Rosetta protocol design and publicly reachable. No special knowledge, credentials, or exploit code is required — only the ability to send concurrent HTTP POST requests. A single attacker with a modest number of connections (50–100) from one or more IPs is sufficient. The attack is trivially repeatable and can be sustained indefinitely. Default deployments (the majority, given `global.middleware: false`) are fully exposed.

### Recommendation

1. **Immediate:** Add application-level concurrency limiting (e.g., a semaphore or `golang.org/x/time/rate` token bucket) directly inside `NetworkStatus()` or as a Go HTTP middleware, independent of any external proxy configuration.
2. **Short-term:** Change the Helm chart default to `global.middleware: true`, or add a dedicated per-endpoint concurrency guard in the Go server.
3. **Consider caching:** Cache the `/network/status` response for a short TTL (e.g., 1–2 seconds) using an in-memory store. The genesis block is already cached via `sync.Once`; extend this pattern to `RetrieveLatest()` with a short TTL to collapse concurrent requests into a single DB query.
4. **Correct the `rateLimit` source criterion:** Change `sourceCriterion: requestHost: true` to `sourceCriterion: ipStrategy: depth: 1` so the rate limit applies per client IP rather than globally per destination host.

### Proof of Concept

```bash
# Requires: curl, GNU parallel or similar
# Target: default Rosetta deployment with global.middleware: false

ENDPOINT="http://<rosetta-host>:5700/network/status"
BODY='{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}'

# Send 200 concurrent requests
seq 200 | parallel -j 200 \
  curl -s -o /dev/null -w "%{http_code}\n" \
  -X POST "$ENDPOINT" \
  -H "Content-Type: application/json" \
  -d "$BODY"

# While flood is running, observe that legitimate queries time out:
curl -s -X POST "$ENDPOINT" \
  -H "Content-Type: application/json" \
  -d "$BODY"
# Expected: connection wait timeout or 500 ErrDatabaseError
# DB pool exhaustion visible in Prometheus metric:
#   hiero_mirror_rosetta_request_duration{status_code="500"}
```

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}
