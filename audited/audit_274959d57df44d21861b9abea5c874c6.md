### Title
`/tmp/ready` Directory Substitution Causes False Readiness Signal and Pinger Crash Loop

### Summary
The readiness healthcheck in `pinger/cmd/healthcheck/main.go` calls `os.Stat("/tmp/ready")` and discards the returned `FileInfo`, checking only whether the path exists — not whether it is a regular file. An unprivileged user with write access to `/tmp` can `mkdir /tmp/ready` before the pinger starts, causing two simultaneous effects: the readiness probe permanently reports "ready" (false positive), and `pinger/main.go`'s `os.WriteFile("/tmp/ready", …)` fails with "is a directory", triggering `log.Fatalf` and a crash loop.

### Finding Description
**Exact code locations:**

`pinger/cmd/healthcheck/main.go`, lines 20–23:
```go
if _, err := os.Stat("/tmp/ready"); err != nil {
    fmt.Fprintln(os.Stderr, "not ready")
    os.Exit(1)
}
``` [1](#0-0) 

`pinger/main.go`, lines 47–49:
```go
if err := os.WriteFile("/tmp/ready", []byte("ok\n"), 0o644); err != nil {
    log.Fatalf("failed to create readiness file /tmp/ready: %v", err)
}
``` [2](#0-1) 

**Root cause:** `os.Stat` returns a non-nil `FileInfo` (and `nil` error) for both regular files *and* directories. The healthcheck discards the `FileInfo` entirely (`_`) and only branches on `err != nil`. There is no `fi.Mode().IsRegular()` or `fi.IsDir()` guard. [3](#0-2) 

**Exploit flow:**
1. Attacker runs `mkdir /tmp/ready` before the pinger process starts.
2. `os.Stat("/tmp/ready")` in the healthcheck succeeds → probe exits 0 → Kubernetes marks pod "Ready" and begins routing traffic.
3. Simultaneously, `pinger/main.go` calls `os.WriteFile("/tmp/ready", …)` which fails with `EISDIR` ("is a directory") → `log.Fatalf` fires → container exits.
4. Kubernetes restarts the container. If `/tmp` is backed by a persistent `emptyDir` volume (not a fresh `tmpfs` per restart), the directory survives and the pinger crashes again on every restart — a permanent crash loop.
5. Throughout the crash loop, the healthcheck exec probe continues to return exit 0 (because the directory still exists), so Kubernetes never removes the pod from the Endpoints list.

**Why existing checks are insufficient:** There are no checks. The `_` discard of `FileInfo` is the entire gap. [3](#0-2) 

### Impact Explanation
Traffic is routed to a pod whose pinger process is not running and is crash-looping. Any Kubernetes service or ingress that relies on the readiness probe to gate traffic will send requests to a dead backend, causing connection failures for downstream consumers. The pinger's intended function — submitting periodic Hedera transfers to verify mirror-node liveness — is completely disabled while the crash loop persists. Severity is Medium (griefing/DoS, no direct fund loss). [2](#0-1) 

### Likelihood Explanation
Precondition is write access to `/tmp` inside the container. The Dockerfile runs as `USER 1000:1000` on a `FROM scratch` base. [4](#0-3)  In a Kubernetes pod with a shared `emptyDir` volume mounted at `/tmp`, any sidecar or init container running as uid 1000 (or with world-write on the volume) satisfies the precondition. The attack is a single `mkdir` command, requires no privileges, and is repeatable across every container restart as long as the directory persists.

### Recommendation
Replace the bare `os.Stat` existence check with an explicit regular-file check in `pinger/cmd/healthcheck/main.go`:

```go
fi, err := os.Stat("/tmp/ready")
if err != nil || !fi.Mode().IsRegular() {
    fmt.Fprintln(os.Stderr, "not ready")
    os.Exit(1)
}
```

Apply the same guard to the `"live"` branch for `/tmp/alive`. In `pinger/main.go`, consider calling `os.Remove("/tmp/ready")` before `os.WriteFile` to atomically replace any pre-existing directory or stale file, or use `os.OpenFile` with `O_CREATE|O_WRONLY|O_TRUNC` after asserting the path is not a directory. [1](#0-0) 

### Proof of Concept
```bash
# Inside the container (or a sidecar sharing /tmp), before pinger starts:
mkdir /tmp/ready

# Verify healthcheck falsely reports ready:
/healthcheck ready
echo $?   # prints 0 — "ready" — even though pinger has not started

# Verify pinger crashes:
/pinger
# Output: fatal: failed to create readiness file /tmp/ready: open /tmp/ready: is a directory
# Container exits with non-zero status; Kubernetes restarts it.

# On each restart the directory persists (emptyDir volume), so the loop is permanent.
# Meanwhile, every healthcheck invocation continues to exit 0.
```
