### Title
`BufferUnderflowException` via Under-Length EVM Address in `getByEvmAddressAndType()`

### Summary
`decodeEvmAddress()` in `GraphQlUtils` performs no length validation, accepting any even-length hex string. When a caller supplies an 8-hex-char (4-byte) all-zero address, `ByteBuffer.getInt()` consumes all 4 bytes and returns 0 (satisfying the first condition), then `ByteBuffer.getLong()` throws an uncaught `BufferUnderflowException` because 0 bytes remain. This unhandled exception propagates through the GraphQL layer, exposing internal implementation details to any unauthenticated caller.

### Finding Description
**Exact code path:**

`GraphQlUtils.decodeEvmAddress()` strips the optional `0x` prefix and calls `Hex.decodeHex()` with no length check: [1](#0-0) 

The test suite confirms this explicitly accepts non-20-byte results (e.g., 5-byte `0000AaAaAa`): [2](#0-1) 

`getByEvmAddressAndType()` then wraps the returned byte array in a `ByteBuffer` and unconditionally reads 4 + 8 + 8 = 20 bytes: [3](#0-2) 

**Exploit trace for input `"00000000"` (8 hex chars → 4 bytes `[0,0,0,0]`):**
1. `decodeEvmAddress("00000000")` → `[0x00,0x00,0x00,0x00]` (4 bytes, no error)
2. `ByteBuffer.wrap(...)` → buffer capacity=4, position=0
3. `buffer.getInt()` → reads 4 bytes, returns `0`, position=4 — condition `== 0` is **true**
4. `buffer.getLong()` → attempts to read 8 bytes, 0 remain → **`BufferUnderflowException` thrown**
5. No try/catch in `getByEvmAddressAndType()` or in `AccountController` [4](#0-3) 

**Why existing checks fail:** `@Valid` on `AccountInput` is present but no `@Size` or pattern constraint is placed on the `evmAddress` field. `decodeEvmAddress` only rejects odd-length hex strings (via `DecoderException`) and `null`; it never enforces the 20-byte EVM address length.

### Impact Explanation
The `BufferUnderflowException` propagates to Spring GraphQL's error handler, which returns it as a GraphQL error. Depending on the deployment's error-detail configuration, the response exposes the exception class name (`java.nio.BufferUnderflowException`) and potentially a partial stack trace, revealing that the server uses a `ByteBuffer`-based branching strategy to distinguish "long-zero" entity IDs from raw EVM addresses. This constitutes internal routing-logic disclosure. Additionally, any monitoring/alerting on unhandled exceptions creates noise that can mask legitimate incidents.

### Likelihood Explanation
The attack requires zero privileges — only the ability to send a GraphQL query with an `evmAddress` field. The payload is trivial (`"00000000"`). It is fully repeatable and deterministic. No rate-limiting or authentication is required to reach this code path.

### Recommendation
1. **Enforce length in `decodeEvmAddress`**: After `Hex.decodeHex`, assert the result is exactly 20 bytes; throw `IllegalArgumentException` otherwise.
2. **Add a `@Pattern` or `@Size` constraint** on the `evmAddress` field of `AccountInput` (e.g., `@Pattern(regexp = "^(0x)?[0-9a-fA-F]{40}$")`).
3. **Guard `ByteBuffer` reads** in `getByEvmAddressAndType` with an explicit length check (`if (evmAddressBytes.length != 20) ...`) before wrapping.

### Proof of Concept
Send the following GraphQL request to the mirror-node GraphQL endpoint (no authentication required):

```graphql
query {
  account(input: { evmAddress: "00000000" }) {
    entityId { shard realm num }
  }
}
```

**Step-by-step:**
1. `decodeEvmAddress("00000000")` returns `[0,0,0,0]` (4 bytes).
2. `ByteBuffer.wrap([0,0,0,0])` creates a 4-byte buffer.
3. `buffer.getInt()` returns `0` (short-circuit condition passes).
4. `buffer.getLong()` throws `java.nio.BufferUnderflowException`.
5. Spring GraphQL catches the unchecked exception and returns an error response containing the exception type, disclosing internal implementation details.

**Variant:** Any even-length hex string shorter than 40 chars whose first 4 bytes are `0x00000000` triggers the same path (e.g., `"0000000000000000"` — 8 bytes — causes `getInt()=0`, `getLong()=0`, then the second `getLong()` at line 38 throws). [5](#0-4)

    public static byte[] decodeEvmAddress(String evmAddress) {
        if (evmAddress == null) {
            return ArrayUtils.EMPTY_BYTE_ARRAY;
        }

        try {
            evmAddress = Strings.CS.removeStart(evmAddress, HEX_PREFIX);
            return Hex.decodeHex(evmAddress);
        } catch (DecoderException e) {
            throw new IllegalArgumentException("Unable to decode evmAddress: " + evmAddress);
        }
    }

        byte[] evmAddressBytes = decodeEvmAddress(evmAddress);
        var buffer = ByteBuffer.wrap(evmAddressBytes);
        if (buffer.getInt() == 0 && buffer.getLong() == 0) {
            return entityRepository.findById(buffer.getLong()).filter(e -> e.getType() == type);
