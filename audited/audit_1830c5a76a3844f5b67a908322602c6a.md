### Title
Symlink-Following `os.Stat` in Readiness Probe Allows Readiness Spoofing via `/tmp` Race

### Summary
`healthcheck/main.go` uses `os.Stat("/tmp/ready")` which unconditionally follows symlinks. An attacker with write access to `/tmp` (e.g., a sidecar container sharing an emptyDir volume) can place a symlink at `/tmp/ready` pointing to any existing file in the container filesystem before the pinger process creates the legitimate marker, causing the Kubernetes readiness probe to report the pod as ready while the pinger is either not yet initialized or has crashed.

### Finding Description
**Exact location:** `pinger/cmd/healthcheck/main.go`, `main()`, line 20:
```go
if _, err := os.Stat("/tmp/ready"); err != nil {
```
`os.Stat` resolves symlinks all the way to the final target. It returns `nil` error as long as the resolved target exists — it does not verify that the path is a regular file, that it is owned by the expected UID, or that it was written by the pinger process itself.

**How `/tmp/ready` is legitimately created** — `pinger/main.go` line 47:
```go
if err := os.WriteFile("/tmp/ready", []byte("ok\n"), 0o644); err != nil {
    log.Fatalf("failed to create readiness file /tmp/ready: %v", err)
}
```
`os.WriteFile` also follows symlinks. If `/tmp/ready` is a pre-placed symlink pointing to a non-writable target (e.g., `/etc/ssl/certs/ca-certificates.crt`, which is present in the image per `Dockerfile` line 38), `os.WriteFile` fails with EPERM, `log.Fatalf` kills the pinger, but **the symlink itself remains in `/tmp`**. The subsequent healthcheck call then resolves the dangling-but-valid symlink chain and exits 0.

**Exploit flow (crash-and-spoof path):**
1. Attacker (sidecar or init container sharing `/tmp` emptyDir) runs: `ln -s /etc/ssl/certs/ca-certificates.crt /tmp/ready`
2. Pinger starts; `os.WriteFile("/tmp/ready", ...)` follows the symlink, hits EPERM on the CA cert file, calls `log.Fatalf` → pinger process exits.
3. The symlink `/tmp/ready → /etc/ssl/certs/ca-certificates.crt` persists in `/tmp`.
4. Kubernetes exec-probe runs `/healthcheck ready`; `os.Stat("/tmp/ready")` resolves through the symlink, finds the CA cert file exists, returns `nil` → exits 0 → **probe reports READY**.
5. Kubernetes routes live traffic to the pod; the pinger is dead and no transfers are being submitted.

**No existing mitigations:** The code performs no `os.Lstat` check, no `fi.Mode().IsRegular()` check, no file-ownership check, and no content validation.

### Impact Explanation
The Kubernetes readiness gate is bypassed: the pod is marked ready and receives traffic while the pinger process is not running. Any downstream system relying on the pinger to submit Hedera transactions receives no activity, yet the orchestration layer believes the service is healthy. This defeats the purpose of the readiness probe entirely and can mask a persistent crash loop from operators and alerting systems.

### Likelihood Explanation
The precondition is write access to the pod's `/tmp`. This is satisfied by: (a) any sidecar container in the same pod with a shared emptyDir at `/tmp`, (b) an init container that runs before the pinger, or (c) a compromised co-located process. The container runs as UID 1000 (non-root, `Dockerfile` line 42) on a `FROM scratch` base, so there is no shell for interactive exploitation, but a purpose-built sidecar or init container requires no elevated privileges. The attack is repeatable across every pod restart.

### Recommendation
Replace `os.Stat` with `os.Lstat` and add an explicit regular-file check:
```go
fi, err := os.Lstat("/tmp/ready")
if err != nil || !fi.Mode().IsRegular() {
    fmt.Fprintln(os.Stderr, "not ready")
    os.Exit(1)
}
```
Apply the same fix to the `"live"` branch (line 27). Optionally, also verify `fi.Uid()` matches the expected UID (1000) using `fi.Sys().(*syscall.Stat_t).Uid`. On the writer side (`pinger/main.go`), use `os.OpenFile` with `os.O_CREATE|os.O_EXCL` so that a pre-existing symlink causes an explicit failure rather than a silent follow.

### Proof of Concept
```bash
# In a sidecar or init container sharing /tmp (emptyDir):

# Step 1 – place symlink before pinger starts
ln -s /etc/ssl/certs/ca-certificates.crt /tmp/ready

# Step 2 – pinger starts, WriteFile follows symlink, hits EPERM, fatalf → exits
# (observe: pinger container enters CrashLoopBackOff)

# Step 3 – probe still passes
/healthcheck ready
echo $?   # prints 0  ← pod marked Ready despite dead pinger

# Step 4 – confirm pinger is not running
ps aux | grep pinger   # no pinger process
```
