### Title
Empty-Credential Authentication Bypass in Metrics Endpoint `authenticate()`

### Summary
The `authenticate()` function in `rest/middleware/metricsHandler.js` performs no validation that the configured `username` and `password` are non-empty strings before comparing them. When an operator misconfigures either value as an empty string (e.g., via an unset environment variable resolving to `""`), an unauthenticated attacker can craft `Authorization: Basic Og==` (base64 of `:`) to produce `{name: '', pass: ''}` from `basicAuth()`, causing both `tsscmp` calls to return `true` and granting full access to the Prometheus metrics endpoint.

### Finding Description
**File:** `rest/middleware/metricsHandler.js`, function `authenticate()`, lines 137–144.

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

**Root cause:** The function reads `username` and `password` from config and immediately uses them in `tsscmp` comparisons with no guard against empty-string values. The only existing check — `credentials &&` — prevents `null`/`undefined` from `basicAuth()` but does **not** prevent a truthy object `{name: '', pass: ''}`.

**Exploit flow:**
1. Operator sets `HIERO_MIRROR_REST_METRICS_CONFIG_USERNAME=` and `HIERO_MIRROR_REST_METRICS_CONFIG_PASSWORD=` (empty, e.g., via an unset Kubernetes secret or a blank env var), causing `username = ''` and `password = ''` in `config.metrics.config`.
2. `authentication` remains `true` (the default), so the early-return bypass at line 139 is not triggered.
3. Attacker sends: `GET /swagger/metrics/ HTTP/1.1` with `Authorization: Basic Og==` (`Og==` is base64 of `:`).
4. `basicAuth(req)` parses this as `{name: '', pass: ''}` — a valid, truthy object.
5. `tsscmp('', '')` returns `true` for both comparisons.
6. `authenticate()` returns `true`; the metrics payload is served at line 161–164.

**Why the existing check fails:** `credentials &&` only guards against `basicAuth()` returning `undefined` (which it does for a completely absent or malformed header). A header of `Basic Og==` is syntactically valid RFC 7617 Basic Auth with empty username and empty password, so `basicAuth()` returns a truthy object, bypassing the only guard.

### Impact Explanation
An attacker gains unauthenticated read access to the full Prometheus metrics output, which includes per-route request counts, durations, error rates, in-flight request counts, and Node.js process memory/CPU telemetry. This leaks internal operational state (traffic patterns, error spikes, resource usage) that can be used to fingerprint the deployment, time attacks, or confirm the success of denial-of-service attempts. The metrics endpoint is documented as intentionally access-controlled (`authentication: true` by default), so this bypass directly violates the intended security boundary.

### Likelihood Explanation
The precondition — empty credentials — is realistic in container/Kubernetes deployments where secrets are injected via environment variables. An unset or accidentally blank secret value silently produces an empty string in the config. The attacker needs no privileges, only network access to the REST API port (default 5551). The exploit is a single HTTP request with a known, fixed header value (`Authorization: Basic Og==`). It is repeatable and leaves no distinguishing log entry beyond a normal 200 response.

### Recommendation
Add an explicit non-empty credential guard inside `authenticate()` before performing comparisons:

```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;
  }
  // Reject if credentials are not configured (empty or missing)
  if (!username || !password) {
    return false;
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};
```

Additionally, add startup validation that rejects a configuration where `authentication: true` but `username` or `password` is empty, failing fast rather than silently accepting an insecure state.

### Proof of Concept
**Precondition:** Deploy the REST API with:
```
HIERO_MIRROR_REST_METRICS_CONFIG_AUTHENTICATION=true
HIERO_MIRROR_REST_METRICS_CONFIG_USERNAME=
HIERO_MIRROR_REST_METRICS_CONFIG_PASSWORD=
```

**Trigger:**
```bash
curl -v \
  -H 'Authorization: Basic Og==' \
  http://<host>:5551/swagger/metrics/
```

**Expected (vulnerable) result:** HTTP 200 with full Prometheus metrics text output.

**Expected (patched) result:** HTTP 401 `Unauthorized`.
