### Title
Unbounded Operations Array in `getOperationSlice()` Enables Heap Exhaustion DoS via `/construction/payloads`

### Summary
`ConstructionPayloads()` calls `getOperationSlice(request.Operations)`, which performs `make(types.OperationSlice, 0, len(operations))` with no upper bound on the number of operations. No HTTP body size limit (`http.MaxBytesReader`) is applied anywhere in the rosetta server stack. An unauthenticated attacker can POST a crafted request with a massive `operations` array to exhaust heap memory and crash the service.

### Finding Description
**Exact code path:**

`rosetta/app/services/construction_service.go`, `getOperationSlice()`, lines 432–465:
```go
operationSlice := make(types.OperationSlice, 0, len(operations))
for _, operation := range operations {
    ...
    operationSlice = append(operationSlice, types.Operation{...})
}
``` [1](#0-0) 

There is no check on `len(operations)` before the `make()` call. The function is called unconditionally from `ConstructionPayloads()` at line 232: [2](#0-1) 

**Root cause — no body size limit anywhere in the stack:**

The HTTP server in `rosetta/main.go` configures only timeout values (`ReadTimeout`, `WriteTimeout`, etc.) but never applies `http.MaxBytesReader` to the request body: [3](#0-2) 

No middleware in `rosetta/app/middleware/` applies a body size cap either. The metrics middleware only instruments request sizes; it does not reject large bodies: [4](#0-3) 

**Why the asserter does not help:**

The rosetta-sdk-go `Asserter` (passed to `server.NewConstructionAPIController`) validates each operation's *type* and *content*, but imposes no maximum *count* on the operations array. JSON deserialization of the entire body into `[]*rTypes.Operation` occurs before the asserter runs, so the heap allocation from parsing already happens regardless of asserter outcome. [5](#0-4) 

**Why the Traefik middleware does not help:**

The `inFlightReq` and `rateLimit` Traefik middleware are optional Kubernetes deployment artifacts, not part of the application itself. They are only applied when `global.middleware` is enabled and are absent in direct deployments, Docker Compose, or offline mode: [6](#0-5) 

**Why `ReadTimeout` is insufficient:**

The default `ReadTimeout` is 5 seconds (`5000000000` ns). On a 100 Mbps connection, ~62 MB can be sent in 5 seconds. Each minimal valid operation JSON entry is ~150 bytes, yielding ~416,000 operations per request. Each `types.Operation` struct occupies ~200 bytes in memory, so a single request can force ~83 MB of allocation in `getOperationSlice` alone, on top of the JSON deserialization overhead. Multiple concurrent connections multiply this linearly with no concurrency cap at the application layer. [7](#0-6) 

### Impact Explanation
A successful attack causes the Go process to exhaust available heap memory, triggering an OOM kill or runtime panic. This prevents all new transaction construction (`/construction/payloads`, `/construction/preprocess`) from being processed, constituting a total denial of service for the construction API. Because the rosetta service is a single process with no per-request memory quota, a single well-crafted request from an unauthenticated attacker can trigger the condition.

### Likelihood Explanation
The endpoint is publicly reachable with no authentication. The attacker needs only a standard HTTP client and a fast network connection. The attack is trivially scriptable: generate a JSON array of N minimal valid operations and POST it. It is repeatable — each request independently triggers the allocation. No special knowledge of the system is required beyond the Rosetta API specification.

### Recommendation
1. Apply `http.MaxBytesReader` to every request body in the server handler or as middleware, e.g., wrap the handler with a middleware that calls `r.Body = http.MaxBytesReader(w, r.Body, maxBodyBytes)` before JSON decoding.
2. Add an explicit operation count guard at the top of `getOperationSlice()`:
   ```go
   const maxOperations = 1000 // tune to actual transaction limits
   if len(operations) > maxOperations {
       return nil, errors.ErrInvalidArgument
   }
   ```
3. Consider enforcing this limit at the application level regardless of deployment topology, so it is not dependent on optional infrastructure middleware.

### Proof of Concept
```bash
# Generate a payload with 500,000 operations
python3 -c "
import json, sys
ops = [{'operation_identifier':{'index':i},'type':'CRYPTOTRANSFER',
        'account':{'address':'0.0.1'},
        'amount':{'value':'1','currency':{'symbol':'HBAR','decimals':8}}}
       for i in range(500000)]
body = {'network_identifier':{'blockchain':'Hedera','network':'mainnet'},
        'operations': ops,
        'metadata':{'node_account_id':'0.0.3','valid_until_nanos':'9999999999999999999'}}
sys.stdout.write(json.dumps(body))
" > large_payload.json

# Send to the construction/payloads endpoint
curl -s -X POST http://<rosetta-host>:5700/construction/payloads \
  -H 'Content-Type: application/json' \
  -d @large_payload.json
```
Repeat with multiple concurrent connections (`-parallel` or `xargs`) to amplify heap pressure until the process is OOM-killed or becomes unresponsive.

func (c *constructionAPIService) getOperationSlice(operations []*rTypes.Operation) (
	types.OperationSlice,
	*rTypes.Error,
) {
	operationSlice := make(types.OperationSlice, 0, len(operations))
	for _, operation := range operations {
		var accountId types.AccountId
		if operation.Account != nil {
			var err error
			accountId, err = types.NewAccountIdFromString(operation.Account.Address, c.systemShard, c.systemRealm)
			if err != nil || accountId.IsZero() {
				return nil, errors.ErrInvalidAccount
			}
		}

		var amount types.Amount
		var rErr *rTypes.Error
		if operation.Amount != nil {
			if amount, rErr = types.NewAmount(operation.Amount); rErr != nil {
				return nil, rErr
			}
		}

		operationSlice = append(operationSlice, types.Operation{
			AccountId: accountId,
			Amount:    amount,
			Index:     operation.OperationIdentifier.Index,
			Metadata:  operation.Metadata,
			Type:      operation.Type,
		})
	}

	return operationSlice, nil
}
```
