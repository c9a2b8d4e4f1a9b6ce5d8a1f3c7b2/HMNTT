### Title
Unauthenticated Enumeration of Block Node Service Endpoints via `/api/v1/network/registered-nodes`

### Summary
The `GET /api/v1/network/registered-nodes` endpoint in the `rest-java` service exposes all active registered node service endpoints — including `ipAddress`, `port`, and `requiresTls` — without any authentication or authorization requirement. An unprivileged external attacker can filter by `type=BLOCK_NODE` to enumerate the full set of active block node network addresses, enabling targeted network-layer attacks against those nodes.

### Finding Description

**Exact code path:**

`NetworkController.java` lines 173–187 defines the endpoint with no `@PreAuthorize`, `@Secured`, or any Spring Security annotation:

```java
@GetMapping("/registered-nodes")
RegisteredNodesResponse getRegisteredNodes(@RequestParameter RegisteredNodesRequest request) {
    final var registeredNodes = networkService.getRegisteredNodes(request);
    ...
}
``` [1](#0-0) 

The underlying repository query in `RegisteredNodeRepository.java` lines 14–22 returns all non-deleted nodes filtered by type:

```sql
select * from registered_node
where registered_node_id >= :lowerBound
and registered_node_id <= :upperBound
and deleted is false
and (:type is null or type @> array[:type]::smallint[])
``` [2](#0-1) 

**Root cause:** No authentication gate exists on this controller or at the Spring Security layer (no `SecurityConfig` was found in the `rest-java` module). The `authHandler.js` found in the Node.js `rest` service only governs rate-limit customization — it does not block unauthenticated access to any endpoint, and it is entirely separate from the Java `rest-java` service. [3](#0-2) 

**Why the partial mitigation is insufficient:** `RegisteredNodeMapper.java` lines 58–61 deliberately omits only the nested `BlockNodeEndpoint` sub-object (for API stability reasons, per the comment), but the parent `RegisteredServiceEndpoint` — which carries `ipAddress`, `port`, and `requiresTls` — is fully mapped and returned via `toRegisteredServiceEndpoint()`: [4](#0-3) [5](#0-4) 

The `blockNodeForRest()` method returns `Collections.emptyMap()` only for the `BlockNodeEndpoint` nested field — it does not suppress the base service endpoint fields (IP, port, TLS flag) that identify the node on the network.

### Impact Explanation
An attacker who enumerates all active block nodes via `?type=BLOCK_NODE` obtains a complete, authoritative list of their network addresses and ports directly from the mirror node database. This intelligence can be used to mount coordinated volumetric (DDoS) or protocol-level attacks against ≥30% of block nodes simultaneously, degrading or halting their block processing capacity without requiring any brute-force or credential compromise. Block nodes are a critical component of the Hiero network's data pipeline; disrupting a sufficient fraction of them impairs block streaming and downstream mirror node ingestion.

### Likelihood Explanation
The exploit requires zero privileges, zero credentials, and a single HTTP GET request. The endpoint is publicly reachable on any deployed mirror node instance. The `type=BLOCK_NODE` filter (value `1` per the mapper) is documented in the API model. Any motivated attacker — including competitors, activists, or financially incentivized parties — can execute this in seconds and repeat it on demand to track topology changes.

### Recommendation
1. **Add authentication/authorization** to `getRegisteredNodes()` using Spring Security (e.g., `@PreAuthorize("hasRole('ADMIN')")` or an API-key filter) so that only authorized operators can query registered node endpoints.
2. **Suppress sensitive network fields** (`ipAddress`, `port`) from the public response, or introduce a separate privileged response model that includes them.
3. **Rate-limit** the endpoint independently of authentication to slow enumeration even if partial exposure is accepted.
4. Ensure the `rest-java` module has a `SecurityConfig` that enforces access control on sensitive `/network/*` sub-paths.

### Proof of Concept

```
# No credentials required
curl -s "https://<mirror-node-host>/api/v1/network/registered-nodes?type=BLOCK_NODE&limit=100"
```

Expected response: a paginated JSON list of all active block nodes, each containing their `serviceEndpoints` array with `ipAddress`, `port`, and `requiresTls` fields, sourced directly from the `registered_node` table. Iterate using the `links.next` cursor to exhaust the full set.
