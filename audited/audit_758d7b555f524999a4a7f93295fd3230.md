### Title
External Access to `/metrics` Silently Suppressed to Debug Level, Bypassing Audit Logging

### Summary
`TracingMiddleware` in `rosetta/app/middleware/trace.go` unconditionally logs all requests matching `internalPaths` (including `/metrics`) at `log.Debug` level. Because `/metrics` is served on the same public port as the Rosetta API with no authentication or IP restriction, any unprivileged external user can repeatedly access it while their requests are never recorded at the default `info` log level, defeating the audit trail entirely.

### Finding Description
**Exact code path:**

In `rosetta/app/middleware/trace.go` lines 55–58: [1](#0-0) 

```go
if internalPaths[path] {
    log.Debug(message)
} else {
    log.Info(message)
}
```

`internalPaths` is defined at line 19: [2](#0-1) 

`metricsPath = "/metrics"` is defined in `metrics.go` line 16: [3](#0-2) 

The `/metrics` route is registered with **no authentication** in `metrics.go` lines 64–73: [4](#0-3) 

The entire middleware stack — including `TracingMiddleware` — is bound to a **single public port** in `main.go` lines 217–227: [5](#0-4) 

The default log level is `info` (set at startup line 156, then overridden by config line 166): [6](#0-5) 

**Root cause and failed assumption:**

The design assumes `internalPaths` entries are only ever reached by internal infrastructure (Kubernetes liveness/readiness probes, internal monitoring scrapers). This assumption is false for `/metrics`: it is registered as a public route on the same port as the Rosetta API, with no authentication middleware and no network-level restriction. The `log.Debug` suppression therefore applies to external traffic, not just internal traffic.

**Exploit flow:**

1. Attacker sends `GET /metrics HTTP/1.1` to the public Rosetta port.
2. `path := request.URL.RequestURI()` resolves to `/metrics`.
3. `internalPaths["/metrics"]` evaluates to `true`.
4. `log.Debug(message)` is called — the access record is emitted only at debug level.
5. With the default (or any `info`-or-above) log level, the record is discarded by the logger before reaching any log shipper/aggregator.
6. The attacker receives a full Prometheus metrics response (request counts, durations, inflight gauges, response sizes) with zero audit footprint.

**Why existing checks are insufficient:**

There is no authentication on the `/metrics` handler (`promhttp.Handler()` is used directly). There is no separate internal-only listener or IP allowlist. The only "protection" is the debug-level log suppression, which is the vulnerability itself.

### Impact Explanation
An external attacker can perform unlimited reconnaissance of the node's operational metrics (request rates, error rates, response sizes, inflight counts) without generating any `info`-level log entries. This breaks the audit trail for the mirror node's log aggregation pipeline. In a security incident, defenders cannot determine whether, when, or how frequently an attacker scraped metrics. Additionally, the metrics data itself leaks operational intelligence (e.g., transaction volumes, error patterns) that can inform further attacks.

### Likelihood Explanation
Preconditions are minimal: the attacker needs only network access to the public Rosetta port (the same port used for all API calls). No credentials, tokens, or special headers are required. The attack is trivially repeatable with a single `curl` command. Any operator running the default `info` log level — which is the hardcoded startup default — is affected.

### Recommendation
1. **Separate the metrics port**: Serve `/metrics` on a distinct, non-public port (e.g., a localhost-only or cluster-internal listener), consistent with the assumption that `internalPaths` are internal.
2. **Or add authentication**: Protect the `/metrics` route with at least HTTP Basic Auth before it reaches the router.
3. **Or log at `info` regardless of path**: If network separation is not feasible, remove the debug-level suppression for `/metrics` so external accesses are always recorded:
   ```go
   // Always log at info; suppress only liveness/readiness probes if desired
   if internalPaths[path] && path != metricsPath {
       log.Debug(message)
   } else {
       log.Info(message)
   }
   ```

### Proof of Concept
```bash
# Precondition: Rosetta node running on default port (e.g., 8080), log level = info

# Step 1: Access /metrics as an unprivileged external user
curl -s http://<rosetta-host>:8080/metrics | head -20
# Returns Prometheus metrics output — no credentials required

# Step 2: Verify no info-level log entry was produced
# On the server, grep the log output:
grep "GET /metrics" <(journalctl -u rosetta) 
# Returns nothing — the access was logged only at debug level and dropped

# Step 3: Repeat indefinitely — each request is invisible to the audit log
for i in $(seq 1 100); do curl -s http://<rosetta-host>:8080/metrics > /dev/null; done
# 100 accesses, zero log records at info level
```
