### Title
Prometheus Metrics Cardinality Explosion via Unbounded User-Controlled Path Labels on Unmatched Routes

### Summary
In `metricsHandler.js`, the `toOpenApiPath` function falls back to the raw user-supplied `req.path` when a request does not match any registered Express route. This user-controlled string is used directly as a Prometheus label value, meaning every unique path an attacker sends creates a new time series in the in-process Prometheus registry. An unprivileged attacker can flood the service with requests to unique paths, causing unbounded memory growth and an eventual OOM crash of the Node.js process.

### Finding Description

**Code path:**

`recordRequestPath` in `rest/routes/index.js` (lines 16–21) sets `res.locals[requestPathLabel]` only when `req.route?.path` is truthy — i.e., only when the request matched a registered route:

```js
// rest/routes/index.js:16-21
const recordRequestPath = async (req, res) => {
  const path = req.route?.path;
  if (path && !path.startsWith(apiPrefix) && !res.locals[requestPathLabel]) {
    res.locals[requestPathLabel] = `${req.baseUrl}${req.route.path}`.replace(/\/+$/g, '');
  }
};
``` [1](#0-0) 

For unmatched routes (404s), `req.route` is `null`, so `res.locals[requestPathLabel]` is never set.

`toOpenApiPath` in `rest/middleware/metricsHandler.js` (lines 13–31) then falls through to `req.path` — the raw, user-supplied URL path — as its label source:

```js
// metricsHandler.js:13-31
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // not set for 404s

  if (!path) {
    if (!req.route) {
      path = req.path;                       // ← user-controlled input
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }
  ...
};
``` [2](#0-1) 

This path is then used as the `path` label in four per-route Prometheus instruments on every `res.on('finish', ...)` event:

```js
// metricsHandler.js:192-210
const labels = {method, path, code};
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
``` [3](#0-2) 

Each unique `{method, path, code}` label combination creates a new time series stored in the OpenTelemetry `MeterProvider`'s in-memory registry. There is no cardinality cap, no path length limit, and no rate limiting visible in the middleware stack. [4](#0-3) 

### Impact Explanation

The Prometheus registry accumulates one set of time series per unique label combination. With four instruments (counter + three histograms, each with multiple bucket entries), each unique path generates dozens of in-memory data points. An attacker sending `N` unique paths causes `O(N * buckets)` memory growth with no upper bound. This leads to an OOM kill of the Node.js process — a complete REST API outage, preventing the network from confirming new transactions.

### Likelihood Explanation

The attack requires zero authentication, zero special knowledge, and zero rate limiting. A single script sending sequential GET requests to `/api/v1/x1`, `/api/v1/x2`, ... at network speed is sufficient. The attack is trivially repeatable and automatable. Any internet-accessible deployment is at risk.

### Recommendation

1. **Normalize unknown paths to a sentinel label**: In `toOpenApiPath`, when `req.route` is null (no route matched), return a fixed string such as `"unknown"` or `apiPrefix + "/unknown"` instead of `req.path`. This collapses all 404 traffic into a single time series.
2. **Add a path length guard**: If the raw path must be used in any context, truncate it to a safe maximum (e.g., 200 characters) before using it as a label.
3. **Enforce cardinality limits**: Use an OpenTelemetry view with a cardinality limit on the per-route instruments to cap the number of unique label sets.

### Proof of Concept

```bash
# Send 10,000 requests to unique paths (no auth required)
for i in $(seq 1 10000); do
  curl -s "http://<mirror-node-host>/api/v1/nonexistent/path$i" &
done
wait

# Each request creates a new Prometheus time series for:
#   api_request_total{method="GET", path="/api/v1/nonexistent/pathN", code="404"}
#   api_request_duration_milliseconds_bucket{...pathN...} (per bucket)
#   api_request_size_bytes_bucket{...pathN...}
#   api_response_size_bytes_bucket{...pathN...}
#
# Memory grows unboundedly until OOM kill.
```

The root cause is at [5](#0-4)  where `path = req.path` is assigned without any sanitization or fallback to a fixed sentinel value.
