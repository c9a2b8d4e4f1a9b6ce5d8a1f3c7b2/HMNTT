### Title
Historical Block Gas Estimation Bypass via Old EVM Version Selection in `estimateGas()`

### Summary
When `estimate=true` is combined with `block=earliest` (or any old block number) in a request to `POST /api/v1/contracts/call`, the `processCall()` method routes through `estimateGas()` while the `ContractCallContext` is in historical mode. This causes `getSemanticEvmVersion()` to select an old EVM version (e.g., `EVM_VERSION_0_30` on MAINNET), which uses pre-Berlin gas rules with significantly cheaper opcode costs. The resulting gas estimate is artificially low and does not reflect what the current Hedera network will charge, causing real transactions submitted with this estimate to fail with `INSUFFICIENT_GAS`.

### Finding Description

**Exact code path:**

`ContractController.call()` accepts any `block` value including `"earliest"` and passes it directly to `ContractExecutionService.processCall()`. [1](#0-0) 

Inside `processCall()`, when `params.isEstimate()` is true, it calls `estimateGas(params, ctx)` with no check on whether the block is historical: [2](#0-1) 

`estimateGas()` calls `callContract(params, context)`, which sets `callServiceParameters` on the context with the historical block: [3](#0-2) 

`ContractCallContext.useHistorical()` returns `true` for any block that is not `BlockType.LATEST`, including `EARLIEST`: [4](#0-3) 

`BlockType.EARLIEST` has `number = 0L`: [5](#0-4) 

`EvmProperties.getSemanticEvmVersion()` then calls `getEvmVersionForBlock(0)` for the historical context: [6](#0-5) 

On MAINNET, block 0 maps to `EVM_VERSION_0_30`: [7](#0-6) 

`TransactionExecutorFactory.get()` reads the EVM version from the context and selects the v0.30 executor: [8](#0-7) 

The v0.30 executor is created with `contracts.evm.version=v0.30`, which uses pre-Berlin/pre-London gas rules (e.g., `SLOAD` costs 200 gas instead of 2100, `BALANCE`/`CALL` to cold addresses are far cheaper). The binary search in `BinaryGasEstimator` then converges on a gas value valid only under v0.30 rules: [9](#0-8) 

**Root cause:** No guard in `processCall()` or `estimateGas()` prevents `estimate=true` from being combined with a historical block. The test service explicitly acknowledges this is unsupported (`// estimate gas is not supported for historical blocks`) but the production code has no enforcement. [10](#0-9) 

### Impact Explanation

A user who calls `estimateGas` with `block=earliest` on a MAINNET-configured mirror node receives a gas estimate computed under EVM v0.30 rules. For storage-heavy contracts, `SLOAD` alone is 10.5x cheaper (200 vs 2100 gas under EIP-2929). The returned estimate can be an order of magnitude lower than what the current Hedera network actually requires. When the user submits a real transaction with this estimate, it fails with `INSUFFICIENT_GAS`. On Hedera, failed transactions still consume fees — the user loses the transaction fee without any execution. This constitutes a misleading fee quote that causes direct financial loss to users who trust the estimate.

### Likelihood Explanation

Any unprivileged user can trigger this with a single unauthenticated HTTP POST to the public `/api/v1/contracts/call` endpoint. No special credentials, tokens, or elevated access are required. The `block` field is a standard documented parameter. The attack is trivially repeatable and requires no on-chain interaction. It is exploitable on any MAINNET-configured deployment of the mirror node.

### Recommendation

1. **Reject `estimate=true` with historical blocks**: In `ContractExecutionService.processCall()` or `estimateGas()`, add a guard that throws `InvalidParametersException` when `params.isEstimate() && params.getBlock() != BlockType.LATEST`.
2. **Alternatively**, force the EVM version to the current version when `isEstimate=true`, regardless of the block parameter, since gas estimation is always forward-looking.
3. **Document the restriction** in the OpenAPI spec: the `block` field should note that it is ignored (or rejected) when `estimate=true`.

### Proof of Concept

**Precondition:** Mirror node configured with `hiero.mirror.web3.evm.network=MAINNET` (or any network with old EVM versions mapped to early blocks).

**Step 1 — Get misleading estimate (v0.30 gas rules):**
```http
POST /api/v1/contracts/call
Content-Type: application/json

{
  "block": "earliest",
  "estimate": true,
  "to": "0x<storage-heavy-contract>",
  "data": "0x<function-selector-that-reads-storage>"
}
```
Response: `{"result": "0x<artificially_low_gas_hex>"}` — computed under EVM v0.30 (SLOAD=200 gas).

**Step 2 — Compare with correct estimate (current EVM rules):**
```http
POST /api/v1/contracts/call
Content-Type: application/json

{
  "block": "latest",
  "estimate": true,
  "to": "0x<same-contract>",
  "data": "0x<same-calldata>"
}
```
Response: `{"result": "0x<correct_gas_hex>"}` — computed under EVM v0.67 (SLOAD=2100 gas cold).

**Step 3 — Submit real transaction with the low estimate from Step 1 to the Hedera network.** The transaction fails with `INSUFFICIENT_GAS`. The transaction fee is still charged.

**Observable difference:** For a contract that performs 10 cold `SLOAD` operations, Step 1 returns ~2000 gas lower bound while Step 2 returns ~21000 gas. The discrepancy is ~19000 gas per 10 SLOADs, directly traceable to the EVM version mismatch.

    public String processCall(final ContractExecutionParameters params) {
        return ContractCallContext.run(ctx -> {
            var stopwatch = Stopwatch.createStarted();
            var stringResult = "";

            try {
                updateGasLimitMetric(params);

                Bytes result;
                if (params.isEstimate()) {
                    result = estimateGas(params, ctx);
                } else {
                    final var ethCallTxnResult = callContract(params, ctx);
                    result = Objects.requireNonNullElse(
                            Bytes.fromHexString(ethCallTxnResult.contractCallResult()), Bytes.EMPTY);
                }

                stringResult = result.toHexString();
            } finally {
                log.debug("Processed request {} in {}: {}", params, stopwatch, stringResult);
            }

            return stringResult;
        });
    }

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
