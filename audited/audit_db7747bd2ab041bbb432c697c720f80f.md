### Title
Unbounded Repeated DB Queries on Block Lookups Enable Unauthenticated Griefing DoS

### Summary
The `Block()` handler in `rosetta/app/services/block_service.go` performs no caching at the block-repository level. Any unauthenticated caller can repeatedly request the same block (e.g., the second-latest block) and each request unconditionally issues multiple raw SQL queries against the database. There is no rate limiting, no block-level cache, and no request coalescing anywhere in the call chain.

### Finding Description
**Call chain per request:**

1. `Block()` (`block_service.go:51`) → `RetrieveBlock()` (`base_service.go:71-90`)
2. Depending on the `BlockIdentifier` fields supplied:
   - Index only → `blockRepo.FindByIndex()` → `findBlockByIndex()` (`block.go:210-224`) — raw SQL `selectRecordBlockByIndex` every call, no cache.
   - Hash only → `blockRepo.FindByHash()` → `findBlockByHash()` (`block.go:226-241`) — raw SQL `selectByHashWithIndex` every call, no cache.
   - Neither → `blockRepo.RetrieveLatest()` (`block.go:190-208`) — raw SQL `selectLatestWithIndex` every call, no cache.
3. `FindBetween()` (`base_service.go:54-60`) — second raw SQL query for transactions in the block's time range.
4. `updateOperationAccountAlias()` (`block_service.go:69`) — per-operation DB calls for any account not in the LRU entity cache.

**Root cause:** The only caching that exists is:
- `initGenesisRecordFile` uses `sync.Once` to cache the genesis block once (`block.go:243-263`).
- `entityCache` is an LRU cache for account aliases only (`block_service.go:22`).

There is **no block-level cache** for `RetrieveLatest`, `FindByIndex`, or `FindByHash`. Every invocation of `Block()` unconditionally hits the database at least twice (block lookup + transaction range scan).

**Failed assumption:** The design assumes callers will not hammer the same block repeatedly, or that an upstream layer (reverse proxy, API gateway) will enforce rate limits. Neither is enforced in the application code.

### Impact Explanation
An attacker who knows (or discovers via a single prior call) the index or hash of the second-latest block can flood the `/block` endpoint with requests for that block. Each request generates at minimum two expensive DB queries (`ORDER BY index` + time-range transaction scan). With a high-transaction-volume block, the `FindBetween` query is especially costly. This degrades database performance for all users of the mirror node (Rosetta and non-Rosetta), constituting a griefing DoS with no economic cost to the attacker.

### Likelihood Explanation
The Rosetta API is a public, unauthenticated HTTP endpoint by design (Coinbase Rosetta spec). No credentials, tokens, or special network access are required. The second-latest block index is trivially obtained by calling `/network/status` or `/block` with an empty identifier. The attack is trivially scriptable with any HTTP client and is repeatable indefinitely.

### Recommendation
1. **Add a block-level cache** in `blockRepository` (or a caching wrapper implementing `interfaces.BlockRepository`) keyed on block index/hash, with a short TTL (e.g., 1–5 seconds) to absorb burst traffic for the same block.
2. **Cache `RetrieveLatest` results** with a TTL matching the expected block production interval, since the second-latest block changes infrequently.
3. **Apply rate limiting** at the HTTP handler or reverse-proxy layer per source IP, independent of application-level caching.
4. **Coalesce in-flight requests** (singleflight pattern) for identical block identifiers to prevent thundering-herd amplification under concurrent load.

### Proof of Concept
```
# Step 1: discover the second-latest block index
curl -s -X POST http://<mirror-node>:8082/block \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"block_identifier":{}}' \
  | jq .block.block_identifier.index
# => e.g. 12345678

# Step 2: flood with requests for that specific block (no auth required)
while true; do
  curl -s -X POST http://<mirror-node>:8082/block \
    -H 'Content-Type: application/json' \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"block_identifier":{"index":12345678}}' \
    > /dev/null &
done
```

func (s *blockAPIService) Block(
	ctx context.Context,
	request *rTypes.BlockRequest,
) (*rTypes.BlockResponse, *rTypes.Error) {
	block, err := s.RetrieveBlock(ctx, request.BlockIdentifier)
	if err != nil {
		return nil, err
	}

	if block.Transactions, err = s.FindBetween(ctx, block.ConsensusStartNanos, block.ConsensusEndNanos); err != nil {
		return nil, err
	}

	var otherTransactions []*rTypes.TransactionIdentifier
	if len(block.Transactions) > s.maxTransactionsInBlock {
		otherTransactions = make([]*rTypes.TransactionIdentifier, 0, len(block.Transactions)-s.maxTransactionsInBlock)
		for _, transaction := range block.Transactions[s.maxTransactionsInBlock:] {
			otherTransactions = append(otherTransactions, &rTypes.TransactionIdentifier{Hash: transaction.Hash})
		}
		block.Transactions = block.Transactions[0:s.maxTransactionsInBlock]
	}

	if err = s.updateOperationAccountAlias(ctx, block.Transactions...); err != nil {
		return nil, err
	}

	return &rTypes.BlockResponse{Block: block.ToRosetta(), OtherTransactions: otherTransactions}, nil
}

func (b *BaseService) RetrieveBlock(ctx context.Context, bIdentifier *rTypes.PartialBlockIdentifier) (
	*types.Block,
	*rTypes.Error,
) {
	if !b.IsOnline() {
		return nil, errors.ErrInternalServerError
	}

	if bIdentifier.Hash != nil && bIdentifier.Index != nil {
		h := tools.SafeRemoveHexPrefix(*bIdentifier.Hash)
		return b.blockRepo.FindByIdentifier(ctx, *bIdentifier.Index, h)
	} else if bIdentifier.Hash == nil && bIdentifier.Index != nil {
		return b.blockRepo.FindByIndex(ctx, *bIdentifier.Index)
	} else if bIdentifier.Index == nil && bIdentifier.Hash != nil {
		h := tools.SafeRemoveHexPrefix(*bIdentifier.Hash)
		return b.blockRepo.FindByHash(ctx, h)
	} else {
		return b.blockRepo.RetrieveLatest(ctx)
	}
}

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}

func (br *blockRepository) findBlockByIndex(ctx context.Context, index int64) (*types.Block, *rTypes.Error) {
	if index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectRecordBlockByIndex, sql.Named("index", index)).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	return rb.ToBlock(br.genesisBlock), nil
}

func (br *blockRepository) initGenesisRecordFile(ctx context.Context) *rTypes.Error {
	if br.genesisBlock.ConsensusStart != genesisConsensusStartUnset {
		return nil
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	var rb recordBlock
	if err := db.Raw(selectGenesis, sql.Named("treasury_entity_id", br.treasuryEntityId.EncodedId)).
		First(&rb).Error; err != nil {
		return handleDatabaseError(err, hErrors.ErrNodeIsStarting)
	}

	br.once.Do(func() {
		br.genesisBlock = rb
	})

	log.Infof("Fetched genesis record file, index - %d", br.genesisBlock.Index)
	return nil
}
