### Title
Unauthenticated Historical Gas Estimation Triggers 21x EVM Execution Amplification via BinaryGasEstimator

### Summary
Any unprivileged caller can POST to `/api/v1/contracts/call` with `estimate=true` and a historical `block` parameter. The production code contains no guard preventing this combination, so `ContractExecutionService.estimateGas()` runs one initial EVM execution plus up to 20 binary-search iterations (default `maxGasEstimateRetriesCount = 20`) — all against the same historical database snapshot. The gas-bucket throttle is consumed only once at the HTTP layer, so the 21x EVM amplification is invisible to the rate limiter.

### Finding Description

**Entry point — no historical+estimate guard:**

`ContractController.call()` accepts any combination of `estimate` and `block` from an unauthenticated caller: [1](#0-0) 

`constructServiceParameters` passes both fields through verbatim with zero validation: [2](#0-1) 

**Throttle consumed once, before amplification:**

`ThrottleManagerImpl.throttle()` deducts gas tokens once per HTTP request based on `request.getGas()`: [3](#0-2) 

`doProcessCall()` called with `estimate=true` explicitly skips `restoreGasToBucket` and metric recording, so none of the binary-search iterations are visible to the throttle: [4](#0-3) 

**Amplification loop:**

`ContractExecutionService.estimateGas()` performs one initial EVM call, then unconditionally launches the binary search regardless of whether the block is historical: [5](#0-4) 

`BinaryGasEstimator.search()` loops up to `maxGasEstimateRetriesCount` (default **20**) additional EVM executions: [6](#0-5) 

**The only existing note about this combination is in test-only code** (`TestWeb3jService.java` line 233: *"estimate gas is not supported for historical blocks so we ignore the estimate gas logic in historical context"*) — it is never enforced in production: [7](#0-6) 

**Default configuration values:**

`maxGasEstimateRetriesCount = 20`, `maxGasLimit = 15_000_000`, `requestsPerSecond = 500`: [8](#0-7) [9](#0-8) 

### Impact Explanation
Each request with `estimate=true` + historical block causes **21 full EVM executions** (1 initial + up to 20 binary-search iterations) against the same historical database snapshot. Historical EVM calls are more expensive than latest-state calls because they require time-range-filtered queries to reconstruct state. The gas throttle accounts for only 1× the declared gas, so the effective database amplification factor is **21×** per request. At the default 500 req/s rate limit, an attacker can drive up to ~10,500 historical EVM executions per second, exhausting database connection pools and degrading or denying service to all historical-state consumers.

### Likelihood Explanation
The attack requires no credentials, no tokens, no special knowledge — only a valid HTTP POST to the public `/api/v1/contracts/call` endpoint with `"estimate": true` and any past block number. The payload is trivially constructable. The amplification is deterministic and repeatable. A single attacker with a modest HTTP client can sustain the attack indefinitely.

### Recommendation
1. **Reject the combination at the controller layer**: In `ContractController.constructServiceParameters()` or `ContractExecutionService.processCall()`, throw `InvalidParametersException` when `isEstimate() == true` and `block` is not `LATEST`/`PENDING`/`SAFE`/`FINALIZED`.
2. **Alternatively, throttle per-iteration**: Pass each `doProcessCall` invocation inside the binary-search loop through `throttleManager` so the gas bucket reflects actual EVM work performed.
3. **Reduce the default `maxGasEstimateRetriesCount`** or make it configurable per call type (historical vs. latest).

### Proof of Concept
```bash
# No authentication required
curl -X POST https://<mirror-node>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{
    "to":   "0x0000000000000000000000000000000000000001",
    "data": "0x",
    "gas":  15000000,
    "estimate": true,
    "block": "0x1"
  }'
```

Repeat in a tight loop (e.g., 500 concurrent clients). Each request triggers 21 EVM executions against the historical block-1 state. The gas throttle is consumed once (1,500 scaled tokens) but 21 database-heavy EVM calls are issued, amplifying database load by 21× relative to what the throttle accounts for.

```java
            CallServiceParameters params, long estimatedGas, boolean estimate) throws MirrorEvmTransactionException {
        EvmTransactionResult result = null;
        var status = ResponseCodeEnum.SUCCESS.toString();

        try {
            result = transactionExecutionService.execute(params, estimatedGas);

            if (!estimate) {
```

```java
    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
```
