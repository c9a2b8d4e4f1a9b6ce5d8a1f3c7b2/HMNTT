### Title
Whitespace-Only `X-Real-IP` Header Bypasses IP Fallback Logic, Masking Attacker Source in Logs

### Summary
The `getClientIpAddress()` function in `rosetta/app/middleware/trace.go` uses `len(ipAddress) == 0` to decide whether to fall back to `X-Forwarded-For` or `RemoteAddr`. A string consisting entirely of whitespace (e.g., `"   "`) has `len > 0`, so it is accepted as a valid IP and returned directly. An unprivileged attacker can exploit this to have whitespace logged as their client IP, completely hiding their true source address during a DoS campaign.

### Finding Description
**Code location:** `rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74.

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)       // line 64

    if len(ipAddress) == 0 {                        // line 66 — only guards against empty string
        ipAddress = r.Header.Get(xForwardedForHeader)
    }

    if len(ipAddress) == 0 {                        // line 70
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    return ipAddress
}
```

**Root cause:** `len(ipAddress) == 0` is `false` for any non-empty string, including `"   "` (spaces), `"\t"`, or `"\n"`. No `strings.TrimSpace()`, no `net.ParseIP()` validation, and no format check is applied. The function returns the raw, unvalidated header value directly.

**Exploit flow:**
1. Attacker sends HTTP request with header `X-Real-IP: " "` (one or more whitespace characters).
2. `r.Header.Get(xRealIpHeader)` returns `" "`.
3. `len(" ") == 1 > 0` → the first `if` block is skipped.
4. The second `if` block is also skipped.
5. `getClientIpAddress` returns `" "`.
6. `TracingMiddleware` logs: `"   POST /network/status (200) in 1ms"` — the IP field is invisible whitespace.
7. `r.RemoteAddr` (the true source) is never consulted.

**Why existing checks fail:** The only guards are two `len == 0` comparisons. There is no IP address parsing, no trimming, and no allowlist/denylist of header values. [1](#0-0) 

### Impact Explanation
An attacker flooding the Rosetta API (DoS) can suppress their real IP from every log line for the duration of the attack. Incident responders relying on logs to identify and block the source will see only whitespace, delaying or preventing mitigation. Because the logged message is used for both `log.Info` and `log.Debug` paths, all non-internal endpoints are affected. [2](#0-1) 

### Likelihood Explanation
- **No authentication required.** Any client that can reach the Rosetta HTTP port can set arbitrary request headers.
- **Trivially reproducible** with `curl -H "X-Real-IP: " ...` or any HTTP client.
- **Deployment dependency:** If a reverse proxy (nginx, Envoy) is in front and strips/overwrites `X-Real-IP`, the attacker cannot control the header. However, the code itself provides zero enforcement of this, and direct-exposure deployments are fully vulnerable. The Rosetta spec does not mandate a proxy.

### Recommendation
Apply `strings.TrimSpace` before the length check, and validate the result with `net.ParseIP`:

```go
import "strings"

func getClientIpAddress(r *http.Request) string {
    ipAddress := strings.TrimSpace(r.Header.Get(xRealIpHeader))
    if len(ipAddress) == 0 || net.ParseIP(ipAddress) == nil {
        ipAddress = strings.TrimSpace(r.Header.Get(xForwardedForHeader))
        // X-Forwarded-For may be comma-separated; take the first entry
        if idx := strings.Index(ipAddress, ","); idx != -1 {
            ipAddress = strings.TrimSpace(ipAddress[:idx])
        }
    }
    if len(ipAddress) == 0 || net.ParseIP(ipAddress) == nil {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }
    return ipAddress
}
```

This ensures only syntactically valid IP addresses are accepted from headers, and always falls back to the TCP-level `RemoteAddr` when headers are absent or invalid.

### Proof of Concept

```bash
# Direct exposure (no proxy):
curl -s -X POST http://<rosetta-host>:8082/network/status \
     -H "Content-Type: application/json" \
     -H "X-Real-IP:    " \
     -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"metadata":{}}'

# Observe server log — the client IP field will be blank/whitespace:
# "    POST /network/status (200) in 3ms"
# Compare with a normal request where the real IP appears:
# "203.0.113.42 POST /network/status (200) in 3ms"
```

Repeating this at high request rate constitutes a DoS with full log-source anonymization.
