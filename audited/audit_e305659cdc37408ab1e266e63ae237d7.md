### Title
Untrusted `firstAvailableBlock` from Fallback Block Node Causes Permanent Block Skip When `EARLIEST_AVAILABLE_BLOCK_NUMBER` Is Used

### Summary
When `startBlockNumber` is configured to `-1` (`EARLIEST_AVAILABLE_BLOCK_NUMBER`), `BlockNodeSubscriber.hasBlock()` unconditionally sets the local `nextBlockNumber` to whatever `firstAvailableBlock` the first responding block node reports. If an attacker forces the primary block node offline (e.g., via DDoS), the importer falls back to a secondary node that may only retain a recent block window, causing the importer to permanently skip all earlier blocks.

### Finding Description

**Code path:**

`AbstractBlockSource.get()` → `getNextBlockNumber()` → `doGet(blockNumber)` → `getNode(nextBlockNumber)` → `hasBlock(nextBlockNumber, node)`

`AbstractBlockSource.getNextBlockNumber()` returns `EARLIEST_AVAILABLE_BLOCK_NUMBER` (-1) when `startBlockNumber` is explicitly configured to -1: [1](#0-0) 

In `doGet()`, a fresh local `AtomicLong nextBlockNumber` is initialized to `blockNumber` (-1) and passed into `getNode()`: [2](#0-1) 

`getNode()` iterates the node list and calls `hasBlock()` for each active node: [3](#0-2) 

Inside `hasBlock()`, when `nextBlockNumber == EARLIEST_AVAILABLE_BLOCK_NUMBER`, it **unconditionally** overwrites `nextBlockNumber` with whatever `blockRange.lowerEndpoint()` the node reports, with no cross-validation: [4](#0-3) 

`blockRange.lowerEndpoint()` is sourced directly from the block node's `serverStatus` gRPC response field `firstAvailableBlock`, with no floor/sanity check: [5](#0-4) 

**Root cause / failed assumption:** The code assumes all block nodes in the list have the same `firstAvailableBlock`. In practice, nodes may have different retention windows. The code trusts the first responding node's self-reported lower bound without any validation.

**Why existing checks fail:**
- `blockStreamVerifier.verify()` validates block content and hash chains but does not enforce a minimum starting block number; it accepts any valid block as a starting point.
- `shouldGetBlock()` only checks against `endBlockNumber`, not a minimum: [6](#0-5) 
- Once a block is successfully verified and persisted, `getNextBlockNumber()` reads `lastIndex + 1` from the database on the next cycle, permanently anchoring the importer to the wrong starting point.

### Impact Explanation
The importer permanently skips all blocks between the true genesis (or configured start) and the fallback node's `firstAvailableBlock`. For a blockchain mirror node, this is a critical data-integrity failure: the mirror's database will have a gap in block history that cannot be recovered without a full re-sync. Downstream consumers (APIs, analytics) will silently serve incomplete historical data.

### Likelihood Explanation
The preconditions are:
1. `startBlockNumber = -1` is configured (operator opt-in, but documented as a valid mode for "start from earliest available").
2. At least two block nodes are configured with different block retention windows (common in production deployments with archival vs. pruned nodes).
3. The attacker can make the primary (archival) node unreachable — a volumetric DDoS against a public gRPC endpoint requires no authentication or privileged access.

The attack is repeatable: the attacker only needs to sustain the partition long enough for one `doGet()` cycle to complete against the fallback node and persist one block to the database.

### Recommendation
1. **Validate the resolved block number against a known floor.** When `EARLIEST_AVAILABLE_BLOCK_NUMBER` is used, after `hasBlock()` resolves `nextBlockNumber`, compare it against the configured `startBlockNumber` (if set) or 0 (genesis). Reject any node whose `firstAvailableBlock` is higher than the required starting point.
2. **Require consensus across nodes.** Before accepting a `firstAvailableBlock`, verify that a quorum of configured nodes agree on the same lower bound.
3. **Do not mutate `nextBlockNumber` inside `hasBlock()` for the `EARLIEST_AVAILABLE_BLOCK_NUMBER` case without first checking all active nodes.** Collect all candidates' lower endpoints and select the minimum (or the one matching the required start), rather than accepting the first responding node's value.

### Proof of Concept
**Preconditions:**
- Importer configured with `startBlockNumber = -1` and two block nodes: Node A (archival, blocks 0–N) and Node B (pruned, blocks 1000–N).
- No blocks have been processed yet (fresh start).

**Steps:**
1. Attacker floods Node A's gRPC port with traffic, causing it to time out. `BlockNode.getBlockRange()` catches the exception and returns `EMPTY_BLOCK_RANGE` for Node A.
2. Importer's `getNode()` loop skips Node A (empty range → `hasBlock()` returns false).
3. `hasBlock()` is called for Node B. Node B responds with `firstAvailableBlock = 1000`. `nextBlockNumber.set(1000)` is executed; `hasBlock()` returns true.
4. `doGet()` calls `node.streamBlocks(1000, ...)`. Node B streams valid blocks starting at 1000. `blockStreamVerifier.verify()` passes for each block.
5. Block 1000 is persisted to the database. `getNextBlockNumber()` now returns 1001 on the next cycle.
6. Attacker stops the DDoS. Node A is reachable again, but the importer now requests block 1001 from it — blocks 0–999 are permanently missing from the mirror database.

```java
    private BlockNode getNode(final AtomicLong nextBlockNumber) {
        final var nodeList = getBlockNodes();
        final var inactiveNodes = new ArrayList<BlockNode>();
        for (final var node : nodeList) {
            if (!node.tryReadmit(false).isActive()) {
                inactiveNodes.add(node);
                continue;
            }

            if (hasBlock(nextBlockNumber, node)) {
                return node;
            }
        }

        // find the first inactive node with the block and force activating it
        for (final var node : inactiveNodes) {
            if (hasBlock(nextBlockNumber, node)) {
                node.tryReadmit(true);
                return node;
            }
        }

        throw new BlockStreamException("No block node can provide block " + nextBlockNumber.get());
    }
```

```java
    private static boolean hasBlock(final AtomicLong nextBlockNumber, final BlockNode node) {
        final var blockRange = node.getBlockRange();
        if (blockRange.isEmpty()) {
            return false;
        }

        if (nextBlockNumber.get() == EARLIEST_AVAILABLE_BLOCK_NUMBER) {
            nextBlockNumber.set(blockRange.lowerEndpoint());
            return true;
        } else {
            return blockRange.contains(nextBlockNumber.get());
        }
    }
```
