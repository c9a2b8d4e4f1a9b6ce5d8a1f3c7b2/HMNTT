### Title
Integer Overflow in `NumberRangeParameter.getInclusiveValue()` Bypasses `registeredNodeId` Filter

### Summary
When an unprivileged user supplies `registeredNodeId=gt:9223372036854775807` (`Long.MAX_VALUE`), `getInclusiveValue()` computes `Long.MAX_VALUE + 1`, which overflows to `Long.MIN_VALUE`. `Math.max(0L, Long.MIN_VALUE)` evaluates to `0`, so `lowerBound` is silently reset to `0` instead of rejecting the request, causing the query to return all non-deleted registered nodes from ID `0` to `Long.MAX_VALUE`. The question's stated trigger value (`9223372036854775806`) is off by one â€” that value produces `Long.MAX_VALUE` (no overflow); the actual overflow trigger is `9223372036854775807`.

### Finding Description

**Code path:**

1. `NumberRangeParameter.getNumberValue()` â€” [1](#0-0) 
   `Long.parseLong("9223372036854775807")` = `Long.MAX_VALUE`. The only guard is `value < 0`, which passes because `Long.MAX_VALUE` is positive.

2. `NumberRangeParameter.getInclusiveValue()` â€” [2](#0-1) 
   For `RangeOperator.GT`, returns `value + 1`. `Long.MAX_VALUE + 1` silently wraps to `Long.MIN_VALUE` in Java (no exception).

3. `NetworkServiceImpl.resolveRegisteredNodeIdBounds()` â€” [3](#0-2) 
   For a GT parameter, `range.hasLowerBound()` is true, so: `lowerBound = Math.max(0L, Long.MIN_VALUE)` = `0L`. The overflow value is silently discarded and the lower bound is reset to `0`.

4. Repository query â€” [4](#0-3) 
   Executes `WHERE registered_node_id >= 0 AND registered_node_id <= Long.MAX_VALUE AND deleted IS FALSE`, returning all non-deleted registered nodes.

**Root cause:** No upper-bound check on the parsed `Long` value before arithmetic in `getInclusiveValue()`. The negative-value guard in `getNumberValue()` only prevents negative *inputs*, not overflow *results*.

**Why existing checks fail:**
- `getNumberValue()` rejects `value < 0` but `Long.MAX_VALUE` is positive â€” check passes. [5](#0-4) 
- `resolveRegisteredNodeIdBounds()` checks `lowerBound > upperBound` after `Math.max`, but after the overflow `lowerBound = 0` and `upperBound = Long.MAX_VALUE`, so `0 > Long.MAX_VALUE` is false â€” no exception thrown. [6](#0-5) 

### Impact Explanation
An unauthenticated caller can bypass the `registeredNodeId` range filter entirely. A request intended to return zero results (no node can have ID > `Long.MAX_VALUE`) instead returns all non-deleted registered nodes up to the configured `limit`. If registered node records contain operationally sensitive data (e.g., pending nodes, internal configuration), this constitutes an unauthorized data disclosure. The `deleted IS FALSE` filter and pagination `limit` remain in effect, partially bounding the exposure.

### Likelihood Explanation
No authentication or special privilege is required. The input is a standard query parameter accepted by the public REST API. The exact trigger value (`9223372036854775807`) is trivially discoverable by any attacker familiar with Java `Long` boundaries. The attack is stateless, repeatable, and requires a single HTTP request.

### Recommendation
Add an explicit upper-bound guard in `getInclusiveValue()` or in `getNumberValue()` to reject values that would overflow on `+1`/`-1`:

```java
// In getNumberValue(), after parsing:
if (value == Long.MAX_VALUE) {
    throw new IllegalArgumentException("Value too large");
}
```

Or use `Math.addExact` in `getInclusiveValue()` to throw `ArithmeticException` on overflow:

```java
public long getInclusiveValue() {
    if (operator == RangeOperator.GT) {
        return Math.addExact(value, 1);   // throws on overflow
    } else if (operator == RangeOperator.LT) {
        return Math.subtractExact(value, 1);
    } else {
        return value;
    }
}
```

### Proof of Concept

```
GET /api/v1/network/nodes/registered?registeredNodeId=gt:9223372036854775807
```

**Preconditions:** Public network access to the mirror node REST API; no authentication needed.

**Trigger:**
1. `NumberRangeParameter.valueOf("gt:9223372036854775807")` parses successfully (value is positive).
2. `getInclusiveValue()` returns `Long.MAX_VALUE + 1 = Long.MIN_VALUE`.
3. `Math.max(0L, Long.MIN_VALUE)` = `0L` â†’ `lowerBound = 0`.
4. Query: `registered_node_id >= 0 AND registered_node_id <= 9223372036854775807`.

**Expected result:** Empty list (no node ID can exceed `Long.MAX_VALUE`).
**Actual result:** All non-deleted registered nodes (up to the page limit) starting from ID 0.

```java
    private static Range<Long> resolveRegisteredNodeIdBounds(List<NumberRangeParameter> registeredNodeIdRanges) {
        long lowerBound = 0L;
        long upperBound = MAX_VALUE;

        for (final var range : registeredNodeIdRanges) {
            if (range.operator() == RangeOperator.EQ) {
                if (registeredNodeIdRanges.size() > 1) {
                    throw new IllegalArgumentException("The 'eq' operator cannot be combined with other operators");
                }
                return Range.closed(range.value(), range.value());
            } else if (range.hasLowerBound()) {
                lowerBound = Math.max(lowerBound, range.getInclusiveValue());
            } else if (range.hasUpperBound()) {
                upperBound = Math.min(upperBound, range.getInclusiveValue());
            }
        }

        if (lowerBound > upperBound) {
            throw new IllegalArgumentException("Invalid range: lower bound exceeds upper bound");
        }

        return Range.closed(lowerBound, upperBound);
    }
```
