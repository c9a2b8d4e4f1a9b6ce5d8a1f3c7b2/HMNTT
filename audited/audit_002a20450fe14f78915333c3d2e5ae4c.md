### Title
FIFO Substitution Defeats Liveness Probe via Unvalidated `os.Stat` on `/tmp/alive`

### Summary
`pinger/cmd/healthcheck/main.go` calls `os.Stat("/tmp/alive")` and trusts only the existence and `ModTime()` of the path, without verifying it is a regular file. An attacker with write access to `/tmp` inside the container can replace `/tmp/alive` with a named pipe (FIFO), causing the liveness probe to permanently report healthy while the pinger's heartbeat goroutine is silently blocked — defeating Kubernetes' ability to restart a stuck pod.

### Finding Description

**Exact code path:**

`pinger/cmd/healthcheck/main.go`, `main()`, lines 27–39:
```go
fi, err := os.Stat("/tmp/alive")   // succeeds on a FIFO — no file-type check
if err != nil {
    fmt.Fprintln(os.Stderr, err)
    os.Exit(1)
}
age := time.Since(fi.ModTime())    // FIFO mtime is attacker-controlled
if age > 2*time.Minute {
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
os.Exit(0)                         // probe reports HEALTHY
```

**Root cause:** `os.Stat` returns successfully for any filesystem object (regular file, FIFO, socket, device). The code never checks `fi.Mode()&os.ModeNamedPipe` or `fi.Mode().IsRegular()`. The only validation is existence and `ModTime()`, both of which are trivially satisfied by a FIFO whose mtime is kept fresh.

**Exploit flow:**

1. Attacker (with `/tmp` write access) removes any existing `/tmp/alive` regular file.
2. Attacker creates `/tmp/alive` as a FIFO: `syscall.Mknod("/tmp/alive", syscall.S_IFIFO|0644, 0)` — no root privilege required; `mknod` for FIFOs is permitted for unprivileged users and is not blocked by the `RuntimeDefault` seccomp profile.
3. Attacker opens the FIFO for reading in a background goroutine/process (to unblock the pinger's writer).
4. `pinger/main.go` line 36 — `os.WriteFile("/tmp/alive", ...)` — opens the FIFO for writing (unblocks because reader is present), writes succeed, and the FIFO's `mtime` is updated on every heartbeat tick (every 15 s).
5. `healthcheck live` calls `os.Stat("/tmp/alive")` → succeeds; `fi.ModTime()` is always fresh → `os.Exit(0)`.
6. When the attacker wants to suppress a restart: they kill/block the pinger process but keep the reader end of the FIFO open and periodically call `utimensat("/tmp/alive")` to refresh `mtime`. The liveness probe continues to return exit 0 indefinitely.

**Why existing checks fail:**

- `os.Stat` is explicitly documented to follow symlinks and return metadata for any inode type; it does not distinguish regular files from FIFOs.
- The `age > 2*time.Minute` guard is bypassed because the attacker controls `mtime` via `utimensat` (no privilege required for the file owner) or via writes through the FIFO.
- `readOnlyRootFilesystem: true` does not protect `/tmp`; it is a separate writable `tmpfs` mount.
- `capabilities: drop: [ALL]` and `runAsNonRoot: true` do not prevent `mknod(S_IFIFO)` or `utimensat` — neither requires elevated capabilities.

### Impact Explanation
The liveness probe is the sole mechanism by which Kubernetes detects and restarts a stuck or dead pinger pod. By defeating it, an attacker can prevent automatic recovery: a pinger that has stopped submitting Hedera transactions will appear healthy indefinitely. The monitoring/alerting function of the pinger is silently lost. Classified as griefing — no direct economic damage to network users, but operational visibility and self-healing are eliminated.

### Likelihood Explanation
**Precondition:** write access to `/tmp` inside the container. In the current deployment (`FROM scratch`, single-binary, no shell), this requires pre-existing code execution within the container (e.g., supply-chain compromise of the pinger binary, or a shared `emptyDir` volume with a compromised sidecar). This is a non-trivial barrier. However, once the precondition is met, the exploit is fully deterministic, requires no privileges, and is repeatable across pod restarts (the attacker re-creates the FIFO after each restart). The `RuntimeDefault` seccomp profile and dropped capabilities impose no obstacle to `mknod(S_IFIFO)` or `utimensat`.

### Recommendation
Replace the bare `os.Stat` check with an explicit regular-file type assertion:

```go
fi, err := os.Stat("/tmp/alive")
if err != nil {
    fmt.Fprintln(os.Stderr, err)
    os.Exit(1)
}
if !fi.Mode().IsRegular() {
    fmt.Fprintln(os.Stderr, "heartbeat path is not a regular file")
    os.Exit(1)
}
age := time.Since(fi.ModTime())
if age > 2*time.Minute {
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
os.Exit(0)
```

Additionally, in `pinger/main.go`, do not silently discard the `os.WriteFile` error — log it so that a blocked write (e.g., to a FIFO with no reader) is observable:
```go
if err := os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644); err != nil {
    log.Printf("heartbeat write failed: %v", err)
}
```

### Proof of Concept

```bash
# Inside the container (as UID 1000, with /tmp write access):

# 1. Remove the legitimate heartbeat file
rm /tmp/alive

# 2. Create /tmp/alive as a FIFO (no root required)
#    In Go (e.g., injected via supply-chain or shared volume):
#    syscall.Mknod("/tmp/alive", syscall.S_IFIFO|0644, 0)
mkfifo /tmp/alive   # equivalent syscall

# 3. Open the read end to unblock pinger's WriteFile
cat /tmp/alive &    # keeps read end open; pinger writes succeed; mtime stays fresh

# 4. Verify liveness probe reports healthy
/healthcheck live   # exits 0

# 5. Simulate pinger death — kill the pinger process
kill $(pgrep pinger)

# 6. Refresh mtime manually to keep probe happy
touch /tmp/alive    # utimensat; no privilege needed; probe still exits 0

# 7. Confirm Kubernetes never restarts the pod
/healthcheck live   # still exits 0 — pod stuck, no restart triggered
