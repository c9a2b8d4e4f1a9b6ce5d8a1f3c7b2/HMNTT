### Title
Unsanitized `X-Real-IP` / `X-Forwarded-For` Headers Allow Audit Log Injection in `getClientIpAddress()`

### Summary
The `getClientIpAddress()` function in `rosetta/app/middleware/trace.go` accepts the `X-Real-IP` and `X-Forwarded-For` headers verbatim with no IP format validation. Any client that can reach the service (directly or through a proxy that does not strip/override these headers) can inject an arbitrary string — including hostnames, log-format escape sequences, or spoofed IPs — which is then written to the mirror-node trace log as the authoritative client address, corrupting IP-based audit trails.

### Finding Description
**Exact code path:** `rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74.

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)          // line 64 – raw header, no validation

    if len(ipAddress) == 0 {                           // line 66 – only emptiness check
        ipAddress = r.Header.Get(xForwardedForHeader)  // line 67 – same issue
    }

    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr) // line 71 – only fallback uses real addr
    }

    return ipAddress                                   // line 74 – arbitrary string returned
}
```

**Root cause:** The only guard is `len(ipAddress) == 0`. There is no call to `net.ParseIP()` or any regex/format check. Whatever the client supplies in `X-Real-IP` is returned and immediately interpolated into the log message at line 52–53 via `fmt.Sprintf`.

**Failed assumption:** The code assumes `X-Real-IP` is always set by a trusted reverse proxy. In practice, if the Rosetta endpoint is reachable without a proxy (common in dev/staging, or when a proxy is misconfigured to forward the header rather than override it), the client fully controls this value.

**Exploit flow:**
1. Attacker sends: `GET /network/status HTTP/1.1` with header `X-Real-IP: attacker-hostname` (or `X-Real-IP: 192.168.1.1` to impersonate an internal host, or `X-Real-IP: 1.1.1.1 GET /admin 200` for log injection).
2. `getClientIpAddress()` returns the raw string without validation.
3. `TracingMiddleware` logs: `attacker-hostname GET /network/status 200 in 3ms` — a non-IP value appears as the client address in the mirror-node trace record.

### Impact Explanation
- **Audit trail corruption:** IP-based forensics, rate-limiting analysis, and incident response become unreliable because the logged "client IP" is attacker-controlled.
- **Log injection / log forgery:** A crafted value like `1.2.3.4 POST /admin 200 in 0s\n` can inject fake log lines, potentially hiding real attacker activity or framing legitimate users.
- **Spoofed IP attribution:** An attacker can impersonate internal IPs (e.g., `127.0.0.1`, `10.0.0.1`) to make malicious requests appear to originate from trusted infrastructure.

Severity: **Medium** — does not directly compromise data integrity or availability, but undermines the security monitoring and audit capability of the mirror node.

### Likelihood Explanation
- **No authentication required** — any HTTP client can set arbitrary headers.
- **Directly exploitable** whenever the Rosetta service is exposed without a proxy that unconditionally overwrites `X-Real-IP` (e.g., nginx `proxy_set_header X-Real-IP $remote_addr`).
- Even behind a proxy, many default configurations *append* or *pass through* client-supplied headers rather than replacing them.
- Fully repeatable and automatable; requires zero privileges.

### Recommendation
Validate the extracted value with `net.ParseIP()` before using it. Fall back to `r.RemoteAddr` if the header value is not a valid IP:

```go
func getClientIpAddress(r *http.Request) string {
    for _, header := range []string{xRealIpHeader, xForwardedForHeader} {
        val := r.Header.Get(header)
        // X-Forwarded-For may be a comma-separated list; take the first entry
        if idx := strings.IndexByte(val, ','); idx != -1 {
            val = strings.TrimSpace(val[:idx])
        }
        if ip := net.ParseIP(strings.TrimSpace(val)); ip != nil {
            return ip.String()
        }
    }
    ipAddress, _, _ := net.SplitHostPort(r.RemoteAddr)
    return ipAddress
}
```

This ensures only syntactically valid IP addresses are ever logged.

### Proof of Concept
```bash
# Inject a hostname string as the client IP
curl -H "X-Real-IP: attacker-hostname" http://<rosetta-host>/network/status

# Expected log line (vulnerable):
# INFO attacker-hostname GET /network/status (200) in 2ms

# Inject a fake log line (log forgery)
curl -H $'X-Real-IP: 10.0.0.1 POST /admin 200 in 0s\nINFO' \
     http://<rosetta-host>/network/status

# Spoof an internal IP to bypass IP-based allow-lists in log analysis
curl -H "X-Real-IP: 127.0.0.1" http://<rosetta-host>/network/status
