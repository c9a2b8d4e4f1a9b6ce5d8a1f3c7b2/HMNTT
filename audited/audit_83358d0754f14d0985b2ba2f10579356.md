### Title
Gas Throttle Bypass via `scaleGas()` Zero-Token Floor Allows Rate-Limit Monopolization with Sub-10,000 Gas Requests

### Summary
`ThrottleProperties.scaleGas()` returns `0L` for any gas value ≤ 10,000, and `ThrottleManagerImpl.throttle()` passes this zero directly to `gasLimitBucket.tryConsume(0)`, which always succeeds in bucket4j without consuming any tokens. An unprivileged attacker can submit up to `requestsPerSecond` (default 500) requests per second with gas=1, consuming zero gas-bucket tokens while exhausting the shared rate-limit bucket, effectively blocking all legitimate users from submitting any requests.

### Finding Description

**Root cause — `ThrottleProperties.scaleGas()`:** [1](#0-0) 

`GAS_SCALE_FACTOR = 10_000L`. The method returns `0L` for any `gas` in the range `[0, 10_000]` (inclusive). This is an intentional scaling decision, but the return value is used unchecked as a bucket4j token count.

**Exploit path — `ThrottleManagerImpl.throttle()`:** [2](#0-1) 

When `scaleGas(request.getGas())` returns `0`, the call becomes `gasLimitBucket.tryConsume(0)`. In bucket4j, consuming 0 tokens always returns `true` without touching the bucket's state. The gas-budget guard is completely bypassed.

**Contrast with `restore()`:** [3](#0-2) 

`restore()` explicitly guards against adding 0 tokens (`if (tokens > 0)`), demonstrating the developer was aware of the zero-return case — but no equivalent guard exists on the consumption side in `throttle()`.

**Exploit flow:**
1. Attacker sends HTTP requests to the `eth_call` endpoint with `gas=1` (or any value 1–10,000).
2. `rateLimitBucket.tryConsume(1)` succeeds — 1 rate-limit token consumed.
3. `gasLimitBucket.tryConsume(0)` always returns `true` — 0 gas tokens consumed.
4. Request is processed at zero gas cost to the attacker's gas budget.
5. Attacker repeats at the full `requestsPerSecond` rate (default 500 req/s).
6. The shared global `rateLimitBucket` is exhausted; all legitimate users receive `REQUEST_PER_SECOND_LIMIT_EXCEEDED`.

**Why existing checks are insufficient:** [4](#0-3) 

Both `rateLimitBucket` and `gasLimitBucket` are single global buckets — there is no per-IP or per-caller partitioning. A single attacker can consume the entire global rate-limit allowance. The `@Min(21_000)` constraint visible in `ThrottleProperties` applies only to the `gasPerSecond` *configuration* field, not to the gas value in incoming requests. [5](#0-4) 

### Impact Explanation
An attacker monopolizes the global rate-limit bucket with zero-gas-cost requests, causing `REQUEST_PER_SECOND_LIMIT_EXCEEDED` for all other callers. Legitimate high-gas transactions (e.g., complex contract simulations) are denied service even though the gas bucket remains full. This is a protocol-level denial-of-service: the gas-budget throttle — the primary defense against expensive computation — is rendered irrelevant for the attacker's traffic class.

### Likelihood Explanation
No authentication, staking, or privileged access is required. Any external caller can craft an HTTP request with `gas=1`. The attack is trivially scriptable (e.g., `curl` in a loop), repeatable indefinitely, and requires no on-chain funds or special tooling. The default `requestsPerSecond=500` means 500 free-gas slots per second are available to the attacker.

### Recommendation
1. **Guard zero in `throttle()`** — mirror the pattern already used in `restore()`:
   ```java
   long scaledGas = throttleProperties.scaleGas(request.getGas());
   if (scaledGas > 0 && !gasLimitBucket.tryConsume(scaledGas)) {
       throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
   }
   ```
   This alone does not fix the monopolization issue but prevents the silent bypass.

2. **Enforce a minimum gas floor on incoming requests** — reject requests with `gas < GAS_SCALE_FACTOR` (10,000) or the Ethereum minimum (21,000) at the validation layer before throttling.

3. **Add per-caller (per-IP) rate limiting** — the global bucket design means any single caller can exhaust the shared allowance regardless of gas value.

4. **Treat `scaleGas() == 0` as a sentinel** — if the scaled value is 0, either reject the request or consume a minimum of 1 token from the gas bucket to ensure every request has a non-zero cost.

    // Necessary since bucket4j has a max capacity and fill rate of 1 token per nanosecond
    public long getGasPerSecond() {
        return scaleGas(gasPerSecond);
    }

    public long scaleGas(long gas) {
        if (gas <= GAS_SCALE_FACTOR) {
            return 0L;
        }
        return Math.floorDiv(gas, GAS_SCALE_FACTOR);
    }
