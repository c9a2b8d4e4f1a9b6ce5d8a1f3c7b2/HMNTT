### Title
Unauthenticated Enumeration of Block Node Service Endpoints via `/api/v1/network/registered-nodes`

### Summary
The `GET /api/v1/network/registered-nodes` endpoint in the rest-java service requires no authentication or authorization. Any external user can query it with `?type=BLOCK_NODE` to retrieve all active block node service endpoints — including IP addresses, ports, and TLS requirements — directly from the `registered_node` table. This provides a complete network topology map of block nodes, enabling targeted network-layer attacks.

### Finding Description

**Exact code path:**

`NetworkController.java` exposes the endpoint at line 173–187 with a plain `@GetMapping` and no security annotation:

```java
@GetMapping("/registered-nodes")
RegisteredNodesResponse getRegisteredNodes(@RequestParameter RegisteredNodesRequest request) {
    final var registeredNodes = networkService.getRegisteredNodes(request);
    ...
}
``` [1](#0-0) 

`NetworkServiceImpl.getRegisteredNodes()` calls the repository with the caller-supplied `type` filter and no access control:

```java
return registeredNodeRepository.findByRegisteredNodeIdBetweenAndDeletedIsFalseAndTypeIs(
        lowerBound, upperBound, nodeTypeId, page);
``` [2](#0-1) 

`RegisteredNodeRepository.findByRegisteredNodeIdBetweenAndDeletedIsFalseAndTypeIs()` executes a raw SQL query returning all non-deleted nodes of the requested type, including their full `serviceEndpoints` column: [3](#0-2) 

`RegisteredNodeMapper` maps `serviceEndpoints` (containing IP address, port, TLS flag) directly into the REST response. The only intentional omission is the `BlockNodeEndpoint` sub-field (API credentials), but the network-layer endpoint data is fully exposed: [4](#0-3) 

**Root cause:** No Spring Security configuration exists in the rest-java module (no `SecurityConfig`, no `WebSecurityConfigurerAdapter`, no `@PreAuthorize`/`@Secured` on the controller or method). The only servlet filters present are `LoggingFilter` and `MetricsFilter`, neither of which enforces authentication. [5](#0-4) 

CORS is configured to allow all origins (`allowedOrigins("*")`), meaning browser-based attackers can also reach this endpoint: [6](#0-5) 

**Why checks fail:** There are no checks. The endpoint is fully public by design or omission.

### Impact Explanation
An unauthenticated attacker obtains a complete, paginated list of all active block node service endpoints (IP, port, TLS) by issuing a single HTTP GET. With this map, the attacker can direct volumetric or protocol-level attacks (TCP SYN flood, TLS exhaustion, application-layer DoS) at specific block nodes. Because the full set of active block nodes is enumerable, an attacker can target enough nodes to exceed the ≥30% disruption threshold, degrading or halting block processing on the network without needing to brute-force any addresses.

### Likelihood Explanation
The exploit requires zero privileges, zero credentials, and a single HTTP request. It is repeatable on demand, automatable, and executable from any internet-connected host or browser (due to open CORS). The barrier to exploitation is effectively zero for any attacker who reads the API documentation or discovers the endpoint through passive reconnaissance.

### Recommendation
1. **Require authentication** on `GET /api/v1/network/registered-nodes` using Spring Security (e.g., `@PreAuthorize`, a `SecurityFilterChain` bean, or an API-key filter).
2. **Restrict sensitive fields**: If the endpoint must remain public, strip `serviceEndpoints` (IP/port/TLS) from the response for unauthenticated callers, returning only non-sensitive metadata.
3. **Rate-limit** the endpoint to slow bulk enumeration even if some public access is intended.
4. **Audit CORS policy**: Replace `allowedOrigins("*")` with an explicit allowlist for `/api/v1/network/registered-nodes`.

### Proof of Concept

```bash
# No authentication header required
curl -s "https://<mirror-node-host>/api/v1/network/registered-nodes?type=BLOCK_NODE&limit=100" \
  | jq '.registeredNodes[] | {registeredNodeId, serviceEndpoints}'
```

Expected response (abridged):
```json
{
  "registeredNodes": [
    {
      "registeredNodeId": 1,
      "serviceEndpoints": [
        { "ipAddressV4": "203.0.113.10", "port": 8080, "requiresTls": true }
      ]
    },
    ...
  ],
  "links": { "next": "?registered_node_id=gt:100&type=BLOCK_NODE&limit=100" }
}
```

Paginate using the `links.next` cursor to enumerate all block nodes. The attacker now has a complete target list for network-layer attacks.

        response.setRegisteredNodes(registeredNodeDtos);
        response.setLinks(links);

        return response;
    }
