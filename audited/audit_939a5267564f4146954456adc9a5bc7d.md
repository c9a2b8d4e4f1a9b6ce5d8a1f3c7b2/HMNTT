### Title
Unauthenticated DoS via Missing Rate Limiting on `GET /api/v1/network/registered-nodes`

### Summary
The `GET /api/v1/network/registered-nodes` endpoint in the `rest-java` module has no rate limiting, authentication, or request throttling of any kind. Any unprivileged external attacker can flood this endpoint with thousands of requests per second, each triggering a synchronous database query, exhausting the Spring Boot thread pool and database connection pool and rendering the mirror node unresponsive to all registered-node topology discovery requests.

### Finding Description
**Exact code path:**

`NetworkController.java` lines 173–187 defines the handler with no throttle guard: [1](#0-0) 

Each request synchronously calls `networkService.getRegisteredNodes(request)`, which executes a native SQL query via: [2](#0-1) 

**Root cause — no rate limiting in `rest-java`:**

The only servlet filters registered in the `rest-java` module are `LoggingFilter` (request logging) and `MetricsFilter` (byte metrics). Neither enforces any request rate limit: [3](#0-2) [4](#0-3) [5](#0-4) 

The `WebMvcConfiguration` adds only argument resolvers and formatters — no interceptors that throttle: [6](#0-5) 

**Rate limiting exists only in `web3`, not `rest-java`:**

The bucket4j-based `ThrottleConfiguration` and `ThrottleManagerImpl` live exclusively in the `web3` module and guard only EVM contract-call endpoints: [7](#0-6) [8](#0-7) 

No equivalent bean, filter, or interceptor exists anywhere under `rest-java/src/main/java/`.

**OpenAPI spec confirms the endpoint is public** (no security scheme referenced): [9](#0-8) 

### Impact Explanation
An attacker flooding `GET /api/v1/network/registered-nodes` forces a native SQL query per request against the `registered_node` table. With thousands of concurrent requests, the Tomcat thread pool (default 200 threads) and the HikariCP database connection pool are exhausted. The mirror node stops responding to all REST API calls, including topology discovery requests from block nodes, mirror nodes, and RPC relay nodes registered in the network. Nodes that rely on the mirror node as their authoritative source for peer topology updates will be unable to refresh their view of the network, effectively isolating them from topology changes. Given that registered nodes of all types (block nodes, mirror nodes, RPC relays) use this endpoint for discovery, a sustained attack can isolate ≥30% of registered infrastructure nodes from topology updates without any brute-force or cryptographic attack.

### Likelihood Explanation
Preconditions: none — the endpoint is unauthenticated and publicly reachable. A single attacker with a modest botnet or even a single high-bandwidth host running `ab`, `wrk`, or a simple loop can sustain thousands of requests per second. The attack is trivially repeatable, requires no special knowledge, and leaves no persistent state to clean up between runs. The absence of any IP-based or global rate limiting makes mitigation at the application layer impossible without a code change.

### Recommendation
1. Add a bucket4j rate-limiting filter to the `rest-java` module mirroring the pattern in `web3/src/main/java/org/hiero/mirror/web3/config/ThrottleConfiguration.java`, scoped to `/api/v1/network/**` endpoints.
2. Apply a per-IP rate limit (e.g., via a `HandlerInterceptor` or `OncePerRequestFilter`) to reject requests exceeding a configurable threshold (e.g., 100 req/s per IP) with HTTP 429.
3. Enforce a global RPS cap on the `rest-java` Tomcat connector or via an ingress/API gateway (e.g., nginx `limit_req`, Kubernetes ingress rate limiting) as a defense-in-depth layer.
4. Consider adding a short-lived cache (e.g., 5–10 seconds) on `getRegisteredNodes()` results to reduce DB pressure even if rate limiting is bypassed.

@Configuration(proxyBeanMethods = false)
@RequiredArgsConstructor
public class ThrottleConfiguration {

    public static final String GAS_LIMIT_BUCKET = "gasLimitBucket";
    public static final String RATE_LIMIT_BUCKET = "rateLimitBucket";
    public static final String OPCODE_RATE_LIMIT_BUCKET = "opcodeRateLimitBucket";

    private final ThrottleProperties throttleProperties;

    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    public void throttle(ContractCallRequest request) {

        } else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {

        for (var requestFilter : throttleProperties.getRequest()) {
            if (requestFilter.test(request)) {
                action(requestFilter, request);
            }
        }
    }
