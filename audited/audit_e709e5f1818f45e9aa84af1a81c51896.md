### Title
Unauthenticated HTTP Mirror-Node Bootstrap Allows DNS/MITM Attacker to Redirect Signed Transactions to Attacker-Controlled gRPC Endpoints

### Summary
`fetchMirrorNodeNetwork()` fetches the Hedera node topology over a plain, unauthenticated HTTP connection with no TLS enforcement, no certificate pinning, and no response-integrity verification. An attacker who can poison DNS for the mirror-node hostname (or perform a network-path MITM) can return a fabricated `nodesEnvelope` that replaces every gRPC endpoint with attacker-controlled addresses. The pinger then signs real `TransferTransaction`s with the operator's private key and submits them to those attacker endpoints, where they can be captured and replayed on the live Hedera network.

### Finding Description

**Code path:**

`pinger/mirror_node_client.go` line 46 constructs the HTTP client with no transport security:
```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
```
No `TLSClientConfig`, no certificate pool, no HTTPS enforcement — confirmed by the complete absence of any `tls.`, `x509`, or `CertPool` usage in the entire `pinger/` directory.

`fetchMirrorNodeNetwork()` (lines 74–131) issues a plain `GET` to whatever URL was configured, decodes the JSON body into `nodesEnvelope`, and builds the network map:
```go
addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
network[addr] = nodeAccountId
```
The only validation performed is syntactic: non-empty `NodeAccountID`, parseable `AccountID`, non-empty host, non-zero port (lines 103–119). There is no allowlist of legitimate Hedera node IPs/domains, no signature over the response, and no cross-check against a known-good source.

`sdk_client.go` lines 18–22 pass this map directly to `hiero.ClientForNetwork(netmap)`, then line 45 calls `client.SetOperator(opID, opKey)`, binding the operator's private key to a client whose every gRPC endpoint is attacker-controlled.

`transfer.go` lines 29–33 then sign and execute a `TransferTransaction` against that client:
```go
cryptoTransfer := hiero.NewTransferTransaction()...
resp, err := cryptoTransfer.Execute(client)
```

**Root cause:** The bootstrap URL defaults to `http://` (config.go line 37 example: `http://mirror-rest:5551`), and the code never enforces HTTPS or validates the authenticity of the returned topology. The fabricated `nodesEnvelope` is accepted unconditionally as long as it is syntactically valid JSON with at least one parseable endpoint.

### Impact Explanation
The attacker's gRPC server receives fully signed `TransferTransaction` protobuf messages authenticated with the operator's real private key. These can be immediately replayed against the real Hedera network, draining the operator account. Additionally, the pinger's liveness/health-check purpose is silently defeated: the attacker can return fake gRPC success responses, making the pinger report healthy while no real transactions ever reach the network. Severity: **High** — direct financial loss (token theft via replay) and silent monitoring failure.

### Likelihood Explanation
The attack requires DNS poisoning or a network-path MITM for the mirror-node hostname. In a Kubernetes deployment (the intended environment, per the `Dockerfile` and `/tmp/ready` probe), internal CoreDNS can be poisoned by any compromised pod in the same namespace — an unprivileged position. For deployments where `HIERO_MIRROR_PINGER_REST` points to an external hostname over plain HTTP, any on-path network attacker (ISP, cloud provider, BGP hijack) suffices. The default example URL uses `http://`, making plain-HTTP deployments the common case. No special privileges on the pinger host are required.

### Recommendation
1. **Enforce HTTPS**: Reject any `mirrorRest` URL that does not use the `https://` scheme at config-load time (`config.go`).
2. **Pin the CA or certificate**: Configure the `http.Client` with a `tls.Config` that restricts acceptable CAs to the known mirror-node certificate authority.
3. **Validate returned endpoints**: Maintain a compile-time or config-time allowlist of legitimate Hedera node account IDs and/or IP ranges; reject any `nodesEnvelope` entry that does not match.
4. **Sign the bootstrap response**: Require the mirror node to sign the `/network/nodes` response (e.g., with a known public key), and verify the signature before trusting any endpoint.

### Proof of Concept

**Preconditions:**
- Pinger deployed with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551`
- Attacker can poison DNS for `mirror-rest` (e.g., via a compromised pod in the same Kubernetes namespace)

**Steps:**

1. Stand up a malicious HTTP server on attacker-controlled IP `1.2.3.4:8080` serving:
```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{
      "domain_name": "1.2.3.4",
      "ip_address_v4": "",
      "port": 50211
    }]
  }],
  "links": {"next": null}
}
```
2. Stand up a malicious gRPC server on `1.2.3.4:50211` that accepts `CryptoTransfer` RPCs, records the signed transaction bytes, and returns a synthetic `TransactionResponse` with `OK`.
3. Poison DNS so `mirror-rest` resolves to `1.2.3.4`.
4. Start (or restart) the pinger. `fetchMirrorNodeNetwork()` fetches from `1.2.3.4:8080`, accepts the fabricated envelope, and `newClient()` builds a `hiero.Client` pointing solely at `1.2.3.4:50211`.
5. On the next ticker tick, `submitWithRetry()` signs a `TransferTransaction` with the operator key and sends it to `1.2.3.4:50211`.
6. Replay the captured signed transaction bytes against `mainnet-node03.hedera.com:50211` (the real node `0.0.3`) — the transaction is valid and executes, transferring funds from the operator account.
