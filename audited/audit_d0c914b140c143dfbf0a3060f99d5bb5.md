### Title
Unauthenticated Mirror REST Response Allows Endpoint Injection into SDK Network Map

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` makes a plain, unauthenticated HTTP GET to the operator-configured mirror REST URL and unconditionally trusts the returned JSON to build the SDK client's entire node network map. An attacker who can intercept or spoof that HTTP response (via MITM, DNS poisoning, or mirror-node compromise) can inject attacker-controlled `ip_address_v4`/`domain_name` values for any subset of real `node_account_id` entries, causing the SDK client to route all transactions to attacker-controlled endpoints and starving the real consensus nodes of traffic.

### Finding Description
**Exact code path:**

`pinger/sdk_client.go` `newClient()` (line 18–22): when `network=other`, calls `buildNetworkFromMirrorNodes()` → `fetchMirrorNodeNetwork()`, then passes the result directly to `hiero.ClientForNetwork(netmap)` with no further validation.

`pinger/mirror_node_client.go` `fetchMirrorNodeNetwork()` (lines 74–131):
- Line 84: issues a plain `http.Client.Do(req)` — no TLS requirement, no certificate pinning.
- Lines 95–98: decodes the raw HTTP response body into `nodesEnvelope` — no signature, no HMAC, no content integrity check.
- Lines 102–124: for every `nodeEntry` in `payload.Nodes`, takes `ep.DomainName` or `ep.IPAddressV4` and `ep.Port` verbatim and maps `host:port → nodeAccountId`. There is zero validation that the IP/domain belongs to the legitimate node.
- Lines 127–129: the only guard is that the resulting map must be non-empty — it does not verify that the endpoints are legitimate.

**Root cause / failed assumption:** The code assumes the mirror REST endpoint is a trusted, integrity-protected source of truth. In reality it is an unauthenticated HTTP service; the response carries no cryptographic proof that the `service_endpoints` match the canonical on-chain address book (file 0.0.102). Any party who can influence the bytes returned by that HTTP call can substitute arbitrary endpoints.

**Why existing checks fail:**
- HTTP status-code check (lines 90–93): only rejects non-2xx responses; a 200 OK with poisoned JSON passes.
- Non-empty network check (lines 127–129): satisfied as long as at least one entry has a valid host+port, even if all are attacker-controlled.
- No TLS enforcement in `config.go` (line 37): the flag description explicitly shows `http://mirror-rest:5551` as the example, and nothing in `loadConfig()` rejects an `http://` URL.

### Impact Explanation
The network map returned by `fetchMirrorNodeNetwork()` is the **sole** source of consensus-node addresses when `network=other`. If an attacker substitutes attacker-controlled endpoints for ≥30% of the node account IDs, the SDK client (via `hiero.ClientForNetwork`) will send all transactions to those fake endpoints. The real nodes receive zero traffic from this pinger, causing them to accumulate no successful responses and eventually be marked unhealthy. Because the poisoning happens at startup (one-shot call in `newClient()`), the client never self-corrects during its lifetime.

### Likelihood Explanation
- **Precondition**: attacker must be able to influence the HTTP response from the mirror REST URL. Realistic vectors: (a) network-level MITM on the same cluster/VPC where the pinger runs against a plain `http://` endpoint; (b) internal DNS poisoning of the hostname (e.g., `mirror-rest`); (c) compromise of the mirror node process itself.
- **No privilege on the pinger host is required** — the attack is entirely external/network-layer.
- The attack is **repeatable**: every time the pinger restarts it re-fetches the network map, so a persistent MITM yields persistent poisoning.
- The default example URL (`http://mirror-rest:5551`) is plain HTTP, making in-cluster MITM the lowest-effort path.

### Recommendation
1. **Enforce HTTPS** for `mirrorRest`: reject any URL that does not use the `https` scheme in `loadConfig()`.
2. **Pin or verify the TLS certificate** of the mirror REST endpoint (e.g., via a configured CA bundle).
3. **Cross-validate returned endpoints** against the canonical on-chain address book (file 0.0.102) or a locally configured allowlist of known-good IP ranges before accepting them into the network map.
4. **Implement pagination**: `fetchMirrorNodeNetwork()` currently fetches only the first page; a poisoned response can omit nodes entirely by not including them — follow `links.next` and require a minimum expected node count.
5. As a defense-in-depth measure, add a **response signature** (e.g., JWS) to the mirror REST `/network/nodes` endpoint and verify it in the pinger.

### Proof of Concept
1. Stand up a fake HTTP server at the address the pinger will resolve for `HIERO_MIRROR_PINGER_REST` (e.g., via DNS poisoning of `mirror-rest` or ARP spoofing on the same subnet).
2. Configure the fake server to respond to `GET /api/v1/network/nodes` with:
```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [{"ip_address_v4": "192.0.2.1", "port": 50211}]
    },
    {
      "node_account_id": "0.0.4",
      "service_endpoints": [{"ip_address_v4": "192.0.2.2", "port": 50211}]
    }
  ],
  "links": {"next": null}
}
```
   where `192.0.2.1` and `192.0.2.2` are attacker-controlled listeners (or black holes).
3. Start the pinger with `HIERO_MIRROR_PINGER_NETWORK=other HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551`.
4. `fetchMirrorNodeNetwork()` decodes the response and returns `{"192.0.2.1:50211": 0.0.3, "192.0.2.2:50211": 0.0.4}`.
5. `hiero.ClientForNetwork(netmap)` is initialized with only these two fake endpoints.
6. All subsequent `submitWithRetry()` calls route to the attacker's listeners; the real nodes `0.0.3` and `0.0.4` receive no traffic and are marked unhealthy. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

func fetchMirrorNodeNetwork(
	ctx context.Context,
	httpClient *http.Client,
	url string,
) (map[string]hiero.AccountID, bool, error) {
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		return nil, false, err
	}

	resp, err := httpClient.Do(req)
	if err != nil {
		return nil, true, fmt.Errorf("GET %s failed: %w", url, err)
	}
	defer resp.Body.Close()

	if resp.StatusCode < 200 || resp.StatusCode >= 300 {
		retry := resp.StatusCode == http.StatusTooManyRequests || resp.StatusCode >= 500
		return nil, retry, fmt.Errorf("GET %s returned %s", url, resp.Status)
	}

	var payload nodesEnvelope
	if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
		return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
	}
