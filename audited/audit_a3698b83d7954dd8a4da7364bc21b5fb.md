### Title
Unauthenticated Enumeration of GENERAL_SERVICE Node Infrastructure via `/api/v1/network/registered-nodes`

### Summary
The `GET /api/v1/network/registered-nodes` endpoint requires no authentication and exposes full `serviceEndpoints` data (IP addresses, ports, domain names, TLS configuration) for all registered nodes. By supplying `?type=GENERAL_SERVICE`, any unprivileged external user can enumerate every GENERAL_SERVICE infrastructure node and its network addresses, producing a complete target list for subsequent coordinated disruption attacks.

### Finding Description
**Code location:**
- Controller: `rest-java/src/main/java/org/hiero/mirror/restjava/controller/NetworkController.java`, lines 173–187
- Repository: `rest-java/src/main/java/org/hiero/mirror/restjava/repository/RegisteredNodeRepository.java`, lines 14–22
- Domain model: `common/src/main/java/org/hiero/mirror/common/domain/node/AbstractRegisteredNode.java`, lines 29–49

**Root cause:** The `@GetMapping("/registered-nodes")` handler at `NetworkController.java:173` carries no `@PreAuthorize`, no Spring Security role check, and no API-key guard. The `rest-java` config directory (`JacksonConfiguration`, `LoggingFilter`, `MetricsConfiguration`, `MetricsFilter`, `NetworkProperties`, `RestJavaConfiguration`, `RuntimeHintsConfiguration`, `WebMvcConfiguration`) contains no `SecurityConfig` or `WebSecurityConfig` class whatsoever. The OpenAPI spec (`rest/api/v1/openapi.yml:969–989`) declares no `security` requirement on this path.

**Query path:**
```
GET /api/v1/network/registered-nodes?type=GENERAL_SERVICE&limit=100&order=asc
```
This reaches `NetworkServiceImpl.getRegisteredNodes()` (lines 140–152), which calls:
```java
registeredNodeRepository
  .findByRegisteredNodeIdBetweenAndDeletedIsFalseAndTypeIs(
      0, MAX_VALUE, GENERAL_SERVICE.getId(), page);
```
The native SQL query (`RegisteredNodeRepository.java:14–22`) applies only `deleted is false` and the optional type array-containment filter — no ownership, no role, no tenant check. The full `serviceEndpoints` JSON column (containing `ipAddress`, `port`, `domainName`, `requiresTls`) is returned verbatim through `RegisteredNodeMapper` and serialised into the HTTP response.

**Why existing checks fail:** The `authHandler` in the Node.js REST layer (`rest/middleware/authHandler.js`) is optional (unauthenticated requests simply proceed) and applies only to the Node.js service, not to the Spring Boot `rest-java` service that serves this endpoint. There is no rate-limiting, no IP allowlist, and no field-level redaction of `serviceEndpoints`.

### Impact Explanation
A single unauthenticated HTTP GET call returns every non-deleted GENERAL_SERVICE node with its IP address, port, domain name, and TLS flag. An attacker can page through all results using the `next` link cursor and build a complete, current map of GENERAL_SERVICE infrastructure. With that map, a targeted volumetric or application-layer attack against a subset of those endpoints can be launched without any brute-force discovery phase. If GENERAL_SERVICE nodes represent ≥30% of network processing capacity (as the scope statement asserts), coordinated disruption of the enumerated targets meets the Medium severity threshold.

### Likelihood Explanation
Exploitation requires only an HTTP client and zero credentials. The endpoint is publicly documented in the OpenAPI spec. Pagination via `registerednode.id=gt:<last_id>` allows complete enumeration in a small number of requests. The attack is repeatable, automatable, and leaves no authentication artefact. Any internet-connected actor can perform it.

### Recommendation
1. **Add authentication/authorization** to `GET /api/v1/network/registered-nodes` in `NetworkController.java` — at minimum require a valid API key or restrict to known operator CIDRs via Spring Security.
2. **Redact or omit `serviceEndpoints.ipAddress`** from the public response; expose only domain names or a stable node identifier.
3. **Apply rate limiting** (e.g., via a gateway or Spring's `HandlerInterceptor`) to this endpoint to slow bulk enumeration even if some disclosure is accepted.
4. **Add a `security` requirement** to the OpenAPI definition at `rest/api/v1/openapi.yml:969` so client generators and API gateways enforce authentication.

    RegisteredNodesResponse getRegisteredNodes(@RequestParameter RegisteredNodesRequest request) {
        final var registeredNodes = networkService.getRegisteredNodes(request);
        final var registeredNodeDtos = registeredNodeMapper.map(registeredNodes);

        final var sort = Sort.by(request.getOrder(), REGISTERED_NODE_ID);
        final var pageable = PageRequest.of(0, request.getLimit(), sort);
        final var links = linkFactory.create(registeredNodeDtos, pageable, REGISTERED_NODE_EXTRACTOR);

        final var response = new RegisteredNodesResponse();
        response.setRegisteredNodes(registeredNodeDtos);
        response.setLinks(links);

        return response;
    }
