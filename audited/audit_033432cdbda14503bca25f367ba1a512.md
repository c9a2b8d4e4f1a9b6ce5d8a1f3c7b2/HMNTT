### Title
Unauthenticated DoS via qs Nested Object Injection in `requestQueryParser` for `order`/`result` Parameters

### Summary
The `requestQueryParser` function in `rest/middleware/requestHandler.js` configures `qs` with `depth: 1`, which permits one level of bracket nesting (e.g., `order[key]=val`). When `order` or `result` receive an object value this way, `canonicalizeValue` calls `lowerCaseQueryValue` on the object, which silently returns it unchanged (non-string passthrough). The resulting object stored in `req.query` causes a `TypeError` crash in every downstream consumer that calls `.split(':')` on the value, producing an unhandled 500 for any unauthenticated request.

### Finding Description

**Exact code path:**

1. **Parser configuration** — `depth: 1` with `strictDepth: true` allows exactly one bracket level. The input `order[key]=val` has depth 1 and is accepted, producing `{order: {key: 'val'}}`. [1](#0-0) 

2. **Silent object passthrough in `canonicalizeValue`** — The value `{key: 'val'}` is not an array, so `lowerCaseQueryValue` is called directly on it. [2](#0-1) 

3. **`lowerCaseQueryValue` non-string guard** — Returns the object unchanged because `typeof {key:'val'} === 'string'` is false. No error is raised; the object is stored in `req.query.order`. [3](#0-2) 

4. **Crash in `buildComparatorFilter`** — Every endpoint using `buildAndValidateFilters` → `buildFilters` reaches this line with the object as `filter`, throwing `TypeError: filter.split is not a function`. [4](#0-3) 

5. **Crash in `paramValidityChecks`** — Endpoints using `validateReq` hit the same crash at `opAndVal.split(':')`. [5](#0-4) 

6. **Crash in `isValidValueIgnoreCase`** — If validation is reached before `.split`, `value.toLowerCase()` throws on the object. [6](#0-5) 

**Root cause:** The `queryCanonicalizationMap` handler for `order`/`result` assumes the value is always a string or array of strings. The `depth: 1` qs option contradicts this assumption by allowing objects. `lowerCaseQueryValue`'s non-string guard silently propagates the object instead of rejecting it.

**Why existing checks fail:** `strictDepth: true` only throws when depth *exceeds* the limit; depth 1 is explicitly permitted. There is no type guard in `canonicalizeValue` or `requestQueryParser` to reject non-string, non-array values before they enter `req.query`.

### Impact Explanation
Any REST API endpoint that accepts `order` or `result` as a query parameter (virtually all list endpoints) will throw an unhandled `TypeError` and return HTTP 500. This is a complete denial-of-service for those endpoints, repeatable on every request, with no rate-limit bypass needed beyond normal HTTP access.

### Likelihood Explanation
Zero authentication or privilege is required. The payload is a single standard HTTP query string (`?order[key]=val`) that any HTTP client can send. It is trivially repeatable and automatable, making sustained DoS straightforward.

### Recommendation
Apply one or both of the following:

1. **Set `depth: 0`** in `queryOptions`. This prevents all bracket nesting, so `order[key]=val` is treated as the literal key `order[key]` and never produces an object. [1](#0-0) 

2. **Add a type guard in `canonicalizeValue`** (or `requestQueryParser`) to reject or stringify non-string, non-array values before storing them in the result map, e.g.:
   ```js
   if (typeof value !== 'string' && !Array.isArray(value)) {
     return value; // or throw / skip
   }
   ``` [2](#0-1) 

### Proof of Concept

```
GET /api/v1/transactions?order[x]=asc HTTP/1.1
Host: <mirror-node-host>
```

**Steps:**
1. Send the above request to any list endpoint (e.g., `/api/v1/transactions`, `/api/v1/accounts`, `/api/v1/contracts/results`).
2. `qs.parse('order[x]=asc', {depth:1, strictDepth:true})` → `{order: {x: 'asc'}}` (no throw, depth 1 is allowed).
3. `canonicalizeValue('order', {x:'asc'})` → calls `lowerCaseQueryValue({x:'asc'})` → returns `{x:'asc'}` unchanged.
4. `req.query.order` is now `{x:'asc'}`.
5. `buildComparatorFilter('order', {x:'asc'})` → `{x:'asc'}.split(':')` → `TypeError: filter.split is not a function`.
6. Server responds with HTTP 500.

Repeat indefinitely to sustain DoS on all paginated endpoints.
