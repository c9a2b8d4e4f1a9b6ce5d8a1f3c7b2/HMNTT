### Title
Unauthenticated IP Spoofing via Unvalidated `X-Forwarded-For`/`X-Real-IP` Headers in Trace Log

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` unconditionally trusts the `X-Real-IP` and `X-Forwarded-For` HTTP headers supplied by the client, with no trusted-proxy validation. Any unprivileged external attacker can set `X-Forwarded-For: 127.0.0.1` (or `::1`) and have that value recorded verbatim in the mirror node trace log, completely hiding their real source IP and making the request appear to originate from a loopback/internal address.

### Finding Description
**Exact code path** — `rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)          // line 64 – blindly trusted

    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader) // line 67 – blindly trusted
    }

    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr) // only fallback
    }

    return ipAddress
}
```

The returned value is immediately embedded in the log line by `TracingMiddleware` (line 52–53) and written at `Info` level for every non-internal path. There is no check that the request actually arrived through a known/trusted reverse proxy before the header value is accepted.

**Root cause / failed assumption**: The code assumes that `X-Forwarded-For` / `X-Real-IP` are only set by a trusted upstream proxy. In practice, any HTTP client can set arbitrary request headers. The test suite (`trace_test.go` lines 52–58) even encodes this blind-trust behavior as the expected outcome, confirming no validation was ever intended.

**Exploit flow**:
1. Attacker sends a direct HTTP request to the Rosetta endpoint with a crafted header:
   ```
   GET /network/list HTTP/1.1
   Host: <mirror-node>
   X-Forwarded-For: 127.0.0.1
   ```
2. `getClientIpAddress()` reads `X-Forwarded-For`, finds `"127.0.0.1"`, and returns it immediately (line 67).
3. `TracingMiddleware` logs: `127.0.0.1 GET /network/list (200) in 3ms` — the attacker's real IP never appears.

**Why existing checks are insufficient**: There are none. No allowlist of trusted proxy CIDRs, no comparison against `r.RemoteAddr`, no IP format/range validation, and no stripping of the header for untrusted sources.

### Impact Explanation
Every trace log entry for attacker-controlled requests will record a fabricated IP (`127.0.0.1`, `::1`, or any other value). This directly corrupts the mirror node's audit trail:
- Incident response and forensics cannot identify the real source of malicious or anomalous requests.
- IP-based alerting, rate-limiting, or blocking rules that consume these logs will be evaded.
- An attacker can impersonate internal/loopback addresses, potentially bypassing any downstream tooling that grants elevated trust to local-origin log entries.

### Likelihood Explanation
Precondition: none beyond network access to the Rosetta HTTP port. The attack requires a single crafted HTTP header, is trivially repeatable, and requires zero authentication or special privileges. Any script-kiddie-level attacker can exploit this.

### Recommendation
Validate the header only when the request's `r.RemoteAddr` belongs to a configured set of trusted proxy CIDRs. If no trusted proxy is configured, fall back directly to `r.RemoteAddr`. Example pattern:

```go
func getClientIpAddress(r *http.Request, trustedProxies []*net.IPNet) string {
    remoteIP, _, _ := net.SplitHostPort(r.RemoteAddr)
    if isTrustedProxy(remoteIP, trustedProxies) {
        if ip := r.Header.Get(xRealIpHeader); ip != "" {
            return ip
        }
        if ip := r.Header.Get(xForwardedForHeader); ip != "" {
            return strings.TrimSpace(strings.Split(ip, ",")[0])
        }
    }
    return remoteIP
}
```

If the service is always behind a fixed proxy (e.g., a Kubernetes ingress), the trusted CIDR list can be a static configuration value.

### Proof of Concept
```bash
# Direct request spoofing loopback address
curl -H "X-Forwarded-For: 127.0.0.1" http://<mirror-node-rosetta>:5700/network/list

# Observe the mirror node trace log:
# INFO  127.0.0.1 GET /network/list (200) in 4ms
# Real attacker IP is absent from the log.

# Also works with X-Real-IP:
curl -H "X-Real-IP: ::1" http://<mirror-node-rosetta>:5700/network/list
# INFO  ::1 GET /network/list (200) in 3ms
```
