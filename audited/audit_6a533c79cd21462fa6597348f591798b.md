### Title
Missing Null-Byte Sanitization on `ContractResult.errorMessage` Causes PostgreSQL COPY Failure in Batch Insert

### Summary
The `errorMessage` field in `ContractResult` is populated directly from the protobuf `ContractFunctionResult.getErrorMessage()` without calling `DomainUtils.sanitize()`, which exists specifically to strip null bytes (U+0000) that PostgreSQL rejects in `text` columns. The `BatchInserter` uses `COPY ... FROM STDIN WITH CSV` to persist records. A user-deployed contract that reverts with a null byte in its revert reason will produce an `errorMessage` containing `\u0000`, causing the PostgreSQL COPY operation to throw an exception that crashes the importer's batch insert pipeline and halts record stream processing on every mirror node importer instance.

### Finding Description

**Root cause — missing sanitization call:**

`DomainUtils.sanitize()` exists precisely for this problem:

```java
// DomainUtils.java:232-241
/**
 * Cleans a string of invalid characters that would cause it to fail when inserted into the database.
 * In particular, PostgreSQL does not allow the null character (0x0000) to be inserted.
 */
public static String sanitize(String input) {
    return StringUtils.isNotEmpty(input) ? input.replace(NULL_CHARACTER, NULL_REPLACEMENT) : input;
}
``` [1](#0-0) 

It is applied to `memo` in `AbstractEntity` and `AbstractToken`, but **never** to `ContractResult.errorMessage`.

**Unsanitized assignment in `ContractResultServiceImpl`:**

```java
// ContractResultServiceImpl.java:251
contractResult.setErrorMessage(functionResult.getErrorMessage()); // no sanitize()
``` [2](#0-1) 

Same pattern in the block stream transformer:

```java
// AbstractBlockTransactionTransformer.java:148
contractResultbuilder.setErrorMessage(evmTransactionResult.getErrorMessage()); // no sanitize()
``` [3](#0-2) 

**Batch insert via PostgreSQL COPY:**

`BatchInserter` serializes all fields via Jackson CsvMapper and streams them directly into PostgreSQL using the binary COPY protocol:

```java
// BatchInserter.java:85
sql = String.format("COPY %s(%s) FROM STDIN WITH CSV", this.tableName, columnsCsv);
// ...
writer.writeValue(pgCopyOutputStream, items); // null byte in errorMessage → PostgreSQL error
``` [4](#0-3) 

PostgreSQL unconditionally rejects `\u0000` in `text` columns (`ERROR: invalid byte sequence for encoding "UTF8": 0x00`). The exception propagates as a `ParserException`: [5](#0-4) 

The `error_message` column is typed `text null` with no database-level constraint that would strip null bytes before the COPY fails: [6](#0-5) 

### Impact Explanation
Every mirror node importer instance processes the same canonical record stream. A single crafted transaction that produces a null-byte `errorMessage` will cause every importer to fail its batch insert on that record, halt stream processing, and be unable to advance past that consensus timestamp. This effectively stops all mirror node importers from ingesting new data, degrading the entire mirror node fleet simultaneously.

### Likelihood Explanation
Any unprivileged user can deploy a Solidity contract and call it. The EVM `REVERT` opcode accepts arbitrary bytes as revert data. Hedera consensus nodes decode the revert reason and place it in `ContractFunctionResult.error_message`. A contract that executes `revert("\x00poison")` produces a null byte in the recorded `error_message`. No special privileges, no brute force, and no rate-limit bypass is required — a single transaction is sufficient to trigger the condition on all importer instances simultaneously.

### Recommendation
Apply `DomainUtils.sanitize()` to `errorMessage` at the point of assignment in `ContractResultServiceImpl.processContractResult()` and in `AbstractBlockTransactionTransformer.transformSmartContractResult()`:

```java
contractResult.setErrorMessage(DomainUtils.sanitize(functionResult.getErrorMessage()));
```

```java
contractResultbuilder.setErrorMessage(DomainUtils.sanitize(evmTransactionResult.getErrorMessage()));
```

This mirrors the existing pattern used for `memo` fields in `AbstractEntity` and `AbstractToken`.

### Proof of Concept
1. Deploy a Solidity contract on Hedera testnet:
   ```solidity
   contract Poison {
       function trigger() external pure {
           revert("\x00null-byte-crash");
       }
   }
   ```
2. Submit a `ContractCall` transaction invoking `trigger()`.
3. The consensus node records `\x00null-byte-crash` in `ContractFunctionResult.error_message`.
4. The mirror node importer calls `ContractResultServiceImpl.processContractResult()`, which calls `contractResult.setErrorMessage("\x00null-byte-crash")` with no sanitization.
5. `BatchInserter.persistItems()` streams the CSV row containing `\x00` into `COPY contract_result(...) FROM STDIN WITH CSV`.
6. PostgreSQL returns `ERROR: invalid byte sequence for encoding "UTF8": 0x00`.
7. `ParserException` is thrown; the importer halts processing at that consensus timestamp.
8. All mirror node importer instances fail identically on the same record, permanently stalling the fleet until the record is manually skipped or the code is patched.
