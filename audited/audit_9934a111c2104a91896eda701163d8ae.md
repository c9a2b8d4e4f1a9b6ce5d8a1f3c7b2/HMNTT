### Title
`TracingMiddleware` `internalPaths` Check Bypassed by Query String, Enabling Log Injection

### Summary
In `rosetta/app/middleware/trace.go`, `TracingMiddleware` uses `request.URL.RequestURI()` to obtain the request path, which includes the raw query string. The `internalPaths` map is keyed on bare paths (e.g., `"/metrics"`), so any request to `/metrics?anything` fails the map lookup and is logged at `Info` level instead of `Debug`. Because the full attacker-controlled URI is interpolated unsanitized into the log message via `fmt.Sprintf`, this also constitutes a log injection primitive.

### Finding Description
**File:** `rosetta/app/middleware/trace.go`
**Function:** `TracingMiddleware()`, lines 47 and 55

```go
// line 47
path := request.URL.RequestURI()   // returns "/metrics?injected=block_hash_abc"

// line 55
if internalPaths[path] {           // map key is "/metrics" — lookup returns false
    log.Debug(message)
} else {
    log.Info(message)              // attacker-controlled path lands here at Info level
}
```

`internalPaths` is defined at line 19 as:
```go
var internalPaths = map[string]bool{livenessPath: true, metricsPath: true, readinessPath: true}
```
where `metricsPath = "/metrics"` (defined in `metrics.go` line 16).

Go's `URL.RequestURI()` returns the encoded path **plus** query string. An exact map lookup against `"/metrics"` will never match `"/metrics?foo=bar"`. The log message is built with `fmt.Sprintf` at line 52–53 and includes `path` verbatim — no sanitization of the query string occurs before it is written to the log.

**Exploit flow:**
1. Attacker sends `GET /metrics?level=info+fake+block+hash+abc123 HTTP/1.1` to the Rosetta API port (publicly accessible, same port as the API per `main.go` lines 217–219).
2. `path` becomes `"/metrics?level=info+fake+block+hash+abc123"`.
3. `internalPaths[path]` → `false`.
4. `log.Info(message)` fires with the full attacker-controlled string embedded.

The existing test in `trace_test.go` (lines 47–50) only covers the case of a suffix mismatch (`/metricss`), not a query-string mismatch, confirming this path was never tested.

### Impact Explanation
- **Log level bypass:** Prometheus scraper traffic (or attacker-crafted requests) that should be suppressed at `Debug` level in production now appears in `Info`-level logs, polluting operational log streams.
- **Log injection:** The query string is written unsanitized into structured log output. An attacker can embed arbitrary text (fake log entries, misleading status strings, fake IP addresses) into the log record. This can corrupt log-based audit trails, confuse SIEM/alerting systems, and obscure real events. It does **not** affect blockchain state or funds directly.

Severity: **Low–Medium** (log integrity / audit trail corruption; no direct data or fund impact).

### Likelihood Explanation
The `/metrics` endpoint is served on the same port as the Rosetta API with no authentication or network-level restriction visible in the codebase. Any unprivileged external client that can reach the API port can trigger this with a single HTTP GET request. It is trivially repeatable and requires zero credentials.

### Recommendation
Replace the `RequestURI()` call with `request.URL.Path` for the `internalPaths` lookup, so query strings are never considered:

```go
// trace.go line 47
path := request.URL.RequestURI()   // keep for logging the full URI if desired
cleanPath := request.URL.Path      // use only the path for the map lookup

// line 55
if internalPaths[cleanPath] {
    log.Debug(message)
} else {
    log.Info(message)
}
```

Additionally, sanitize `path` before interpolating it into the log message (strip or encode control characters and newlines) to eliminate the log injection surface.

### Proof of Concept
```bash
# Trigger Info-level log with attacker-controlled string
curl "http://<rosetta-host>:<port>/metrics?FAKE_ENTRY=block_hash_abc"

# Resulting log line (Info level, visible in production):
# time="..." level=info msg="<ip> GET /metrics?FAKE_ENTRY=block_hash_abc (200) in ..."
```
