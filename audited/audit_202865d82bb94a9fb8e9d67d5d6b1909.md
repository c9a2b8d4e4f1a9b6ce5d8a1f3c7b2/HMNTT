### Title
Reflected User-Controlled Input in GraphQL Error Response via `decodeEvmAddress` Exception Handling

### Summary
In `GraphQlUtils.decodeEvmAddress()`, when `Hex.decodeHex()` throws a `DecoderException` for invalid input (e.g., whitespace, special characters), the catch block constructs an `IllegalArgumentException` whose message directly concatenates the attacker-supplied `evmAddress` string. `CustomExceptionResolver` then unconditionally forwards `ex.getMessage()` into the GraphQL error response, reflecting the raw user input back to the caller.

### Finding Description

**Code path:**

`AccountController.account()` (line 52-54) passes the raw `evmAddress` argument directly to `entityService.getByEvmAddressAndType()`, which calls `GraphQlUtils.decodeEvmAddress()`. [1](#0-0) 

Inside `decodeEvmAddress`, after stripping the optional `0x` prefix, `Hex.decodeHex(evmAddress)` is called. For any input containing non-hex characters (whitespace, `<`, `>`, `"`, etc.), it throws `DecoderException`. The catch block re-throws as:

```java
throw new IllegalArgumentException("Unable to decode evmAddress: " + evmAddress);
```

The local variable `evmAddress` at this point holds the post-`removeStart` user input verbatim. [2](#0-1) 

`CustomExceptionResolver.resolveToSingleError()` explicitly matches `IllegalArgumentException` and returns `ex.getMessage()` directly into the GraphQL error: [3](#0-2) 

**Root cause:** No sanitization or truncation of user input before embedding it in the exception message; `CustomExceptionResolver` blindly forwards the full message to the client.

**Failed assumption:** The code assumes `evmAddress` will always be a valid hex string before reaching the error path, but no pre-validation enforces this.

### Impact Explanation
The attacker's raw input is reflected verbatim in the GraphQL JSON error response under the `message` field. While the attacker already knows what they sent, this pattern enables:
- **Reflected injection probing**: Confirming which characters survive processing (useful for chaining with downstream parsers or log injection).
- **Log injection**: If the error message is logged server-side, attacker-controlled strings (including newlines, ANSI codes) appear in logs.
- **Client-side trust issues**: Downstream clients that render GraphQL error messages without escaping could be affected if the reflected value contains markup.

Severity: **Low–Medium** (CWE-209 / reflected input in error message).

### Likelihood Explanation
No authentication is required to call the `account` GraphQL query. Any unauthenticated or unprivileged user can supply an arbitrary `evmAddress` string. The trigger is trivially reproducible with a single GraphQL request. The endpoint is publicly reachable by design.

### Recommendation
1. **Sanitize the error message**: Do not include raw user input in exception messages. Replace with a generic message:
   ```java
   throw new IllegalArgumentException("Invalid evmAddress: must be a 40-character hex string");
   ```
2. **Pre-validate input length and character set** before calling `Hex.decodeHex()`, e.g., with a regex `^(0x)?[0-9a-fA-F]{40}$`, throwing the generic error on mismatch.
3. **Audit `CustomExceptionResolver`**: Consider stripping or escaping `ex.getMessage()` before forwarding it, or mapping exception types to static messages.

### Proof of Concept

Send the following GraphQL request to the mirror node GraphQL endpoint (no authentication needed):

```graphql
{
  account(input: { evmAddress: "   <script>alert(1)</script>   " }) {
    balance
  }
}
```

**Expected response:**
```json
{
  "errors": [
    {
      "message": "Unable to decode evmAddress:    <script>alert(1)</script>   ",
      "locations": [...],
      "path": ["account"],
      "extensions": { "classification": "ValidationError" }
    }
  ]
}
```

The attacker-controlled string `   <script>alert(1)</script>   ` (minus the `0x` prefix strip, which is a no-op here) appears verbatim in the `message` field of the response, confirming reflected input.

    public static byte[] decodeEvmAddress(String evmAddress) {
        if (evmAddress == null) {
            return ArrayUtils.EMPTY_BYTE_ARRAY;
        }

        try {
            evmAddress = Strings.CS.removeStart(evmAddress, HEX_PREFIX);
            return Hex.decodeHex(evmAddress);
        } catch (DecoderException e) {
            throw new IllegalArgumentException("Unable to decode evmAddress: " + evmAddress);
        }
    }
