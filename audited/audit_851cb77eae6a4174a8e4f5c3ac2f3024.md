### Title
Contract Filter Bypass via Zero Address Recipient in `MirrorOperationActionTracer.tracePostExecution()`

### Summary
When tracing is enabled with a contract filter configured, a call whose EVM message frame has `Address.ZERO` as the recipient causes `commonEntityAccessor.get()` to return `Optional.empty()`. Because `contractFilterCheck()` is only invoked when `recipientNum.isPresent()` is true, the filter is silently skipped and `log.info()` fires unconditionally for every opcode in that frame, bypassing the operator-configured allowlist.

### Finding Description

**Exact code path:**

In `MirrorOperationActionTracer.tracePostExecution()` (lines 59–68):

```java
final var recipientAddress = frame.getRecipientAddress();
final var recipientNum = recipientAddress != null
        ? commonEntityAccessor.get(recipientAddress, Optional.empty())
        : Optional.empty();

if (recipientNum.isPresent()
        && traceProperties.contractFilterCheck(
                CommonUtils.hex(toEvmAddress(((Entity) recipientNum.get()).getId())))) {
    return;
}

log.info(...);   // fires unconditionally when recipientNum is empty
```

**Root cause — zero address resolves to `Optional.empty()`:**

`CommonEntityAccessor.get(Address address, ...)` (lines 29–36) checks `ConversionUtils.isLongZeroAddress()`. `Address.ZERO` (all 20 bytes zero) satisfies this check, so it calls `getEntityByMirrorAddressAndTimestamp()` (line 73–78), which extracts entity num = 0 via `entityIdNumFromEvmAddress()` and queries `entityRepository.findByIdAndDeletedIsFalse(0)`. Entity ID 0 never exists in the database, so the result is `Optional.empty()`.

**Failed assumption:**

The guard at line 64 uses `recipientNum.isPresent() && contractFilterCheck(...)`. When the entity lookup returns empty, the short-circuit evaluation means `contractFilterCheck()` is **never called**, regardless of what the operator has configured in `TraceProperties.contract`.

`contractFilterCheck()` itself (lines 33–35 of `TraceProperties.java`) is correct in isolation:
```java
public boolean contractFilterCheck(String contract) {
    return !getContract().isEmpty() && !getContract().contains(contract);
}
```
It returns `true` (skip) when the filter is non-empty and the contract is not in it. But it is never reached for zero-address frames.

**Exploit flow:**

1. Operator enables tracing and sets a contract allowlist, e.g. `hiero.mirror.web3.evm.trace.contract=["0x000000000000000000000000000000000000abcd"]`.
2. Attacker deploys a contract containing a loop that issues `CALL` to `address(0)` (ECRecover precompile address).
3. Each iteration creates a child `MessageFrame` with `recipientAddress = Address.ZERO`.
4. For every opcode executed in that frame, `tracePostExecution` is invoked; `recipientNum` is `Optional.empty()`; the filter is skipped; `log.info(...)` fires.
5. Attacker repeats calls at will, flooding the application log with trace entries that the operator intended to suppress.

### Impact Explanation

The operator-configured contract filter — the only mechanism to limit which contracts produce trace output — is completely ineffective for zero-address frames. An attacker can generate an unbounded volume of `INFO`-level log entries, potentially exhausting disk space, overwhelming log aggregation pipelines, or masking legitimate trace output. There is no economic damage to network participants; the impact is purely operational (griefing / availability degradation of the mirror node's logging subsystem).

### Likelihood Explanation

No special privileges are required. Any user who can submit a contract call (a standard, unauthenticated JSON-RPC `eth_call` or `eth_sendRawTransaction`) can trigger this. The only prerequisite is that the operator has enabled tracing with a non-empty contract filter — a configuration that exists precisely to limit trace volume, making it a realistic deployment scenario. The attack is trivially repeatable and requires no on-chain state.

### Recommendation

Replace the short-circuit guard with explicit handling of the empty-entity case. When `recipientNum` is absent, treat the frame as **not matching** any configured filter (i.e., skip logging when a filter is active):

```java
// Proposed fix
if (!traceProperties.getContract().isEmpty()) {
    if (recipientNum.isEmpty()) {
        return;   // no entity → not in the allowlist → suppress
    }
    if (traceProperties.contractFilterCheck(
            CommonUtils.hex(toEvmAddress(((Entity) recipientNum.get()).getId())))) {
        return;
    }
}
```

Alternatively, invert the logic so that logging only proceeds when the entity is found **and** passes the filter, rather than proceeding whenever the entity is absent.

### Proof of Concept

**Preconditions:**
- Mirror node running with `hiero.mirror.web3.evm.trace.enabled=true`
- `hiero.mirror.web3.evm.trace.contract` set to any non-empty address list (e.g., `["0x0000000000000000000000000000000000001234"]`)

**Steps:**

1. Deploy a contract:
   ```solidity
   contract ZeroAddrSpam {
       function spam(uint256 n) external {
           for (uint256 i = 0; i < n; i++) {
               address(0).call("");   // CALL to Address.ZERO each iteration
           }
       }
   }
   ```
2. Call `spam(10000)` via `eth_call` or `eth_sendRawTransaction`.
3. Observe that the mirror node emits `log.info(...)` trace entries for every opcode in each zero-address child frame, despite `Address.ZERO` not being in the configured contract allowlist.
4. Repeat indefinitely to flood logs.

**Expected (correct) behavior:** No log output, because `Address.ZERO` is not in the configured allowlist.
**Actual behavior:** Unrestricted log output for every opcode in zero-address frames. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    private Optional<Entity> getEntityByMirrorAddressAndTimestamp(Address address, final Optional<Long> timestamp) {
        final var entityId = entityIdNumFromEvmAddress(address);
        return timestamp
                .map(t -> entityRepository.findActiveByIdAndTimestamp(entityId, t))
                .orElseGet(() -> entityRepository.findByIdAndDeletedIsFalse(entityId));
    }
