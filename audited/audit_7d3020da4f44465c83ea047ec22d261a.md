### Title
Missing Application-Level Rate Limiting in `newBlockchainOnlineRouter()` Enables Resource Exhaustion via Sustained Expensive Requests

### Summary
The `newBlockchainOnlineRouter()` function in `rosetta/main.go` assembles a router via `server.NewRouter()` with no rate-limiting middleware applied at the application layer. The only defined rate-limiting protection exists in the Helm chart's Traefik middleware configuration, but it is **disabled by default** (`global.middleware: false`). Any unprivileged external user can therefore send an unbounded stream of structurally valid, DB-heavy requests to endpoints such as `/block` or `/account/balance`, driving aggregate CPU and memory consumption well above the 30% baseline threshold without triggering brute-force detection.

### Finding Description

**Exact code path:**

`rosetta/main.go`, `newBlockchainOnlineRouter()` (lines 61–120): the function constructs all API controllers and returns `server.NewRouter(...)` with no rate-limiting wrapper. [1](#0-0) 

`rosetta/main.go`, `main()` (lines 217–219): the only middleware applied after the router is `MetricsMiddleware`, `TracingMiddleware`, and `CorsMiddleware` — none of which throttle request rates. [2](#0-1) 

`rosetta/app/middleware/` contains only `health.go`, `metrics.go`, and `trace.go` — no rate-limiting implementation exists at the application level.


**Root cause — failed assumption:**

The project assumes that Traefik middleware (defined in `charts/hedera-mirror-rosetta/values.yaml` lines 149–166) will always be active. However, the guard condition in `middleware.yaml` is `{{ if and .Values.global.middleware .Values.middleware }}`, and `global.middleware` defaults to `false`. [3](#0-2) [4](#0-3) 

This means the `inFlightReq` (5 concurrent per IP) and `rateLimit` (10 req/s per host) Traefik rules are **never deployed** in a default installation.

**Expensive endpoints exposed without any throttle:**

- `/block` → `blockAPIService.Block()` calls `RetrieveBlock`, `FindBetween` (full DB range scan over consensus timestamps), and `updateOperationAccountAlias` (per-operation DB lookup for every operation in every transaction in the block). [5](#0-4) 

- `/account/balance` → `AccountAPIService.AccountBalance()` calls `RetrieveBlock` or `RetrieveLatest` followed by `RetrieveBalanceAtBlock` (multi-table DB query including NFT transfer joins). [6](#0-5) 

### Impact Explanation

With no application-level rate limiting and infrastructure-level protection disabled by default, an attacker can open many concurrent HTTP connections and POST valid Rosetta requests (correct `network_identifier`, valid block indices) to `/block` and `/account/balance`. Each request triggers multiple PostgreSQL queries against large tables. Sustained concurrent load saturates the Go goroutine pool, exhausts the DB connection pool (`max_connections = 600`), and drives CPU and memory above the 30% baseline threshold. The HPA scales up to 10 replicas, amplifying DB load rather than mitigating it. The attack does not resemble brute-force (no repeated identical payloads, no auth failures), so no existing alert fires.

### Likelihood Explanation

Preconditions are minimal: network access to the Rosetta port, knowledge of a valid `network_identifier` (publicly documented for Hedera mainnet/testnet), and any valid block index (obtainable from `/network/status`). The attack is fully automatable with standard HTTP tooling (`wrk`, `hey`, `curl` in a loop). Any operator who deploys the Helm chart without explicitly setting `global.middleware: true` is exposed. Direct (non-Kubernetes) deployments have no protection at all.

### Recommendation

1. **Add application-level rate limiting** in `newBlockchainOnlineRouter()` (or in `main()` before the HTTP server starts) using a middleware such as `golang.org/x/time/rate` or `github.com/ulule/limiter`. This ensures protection regardless of deployment topology.
2. **Change the default** of `global.middleware` in `charts/hedera-mirror-rosetta/values.yaml` from `false` to `true` so the Traefik `inFlightReq` and `rateLimit` rules are active out of the box.
3. **Add a concurrency limiter** (e.g., a semaphore) inside `blockAPIService.Block()` to cap simultaneous in-flight DB-heavy requests independent of the HTTP layer.

### Proof of Concept

```bash
# 1. Discover a valid network identifier (public knowledge for Hedera mainnet)
# 2. Get a recent block index
curl -s -X POST http://<rosetta-host>:<port>/network/status \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' \
  | jq '.current_block_identifier.index'

# 3. Flood /block with valid, expensive requests using 200 concurrent workers
hey -n 100000 -c 200 -m POST \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"block_identifier":{"index":<BLOCK_INDEX>}}' \
  http://<rosetta-host>:<port>/block

# 4. Observe CPU/memory on the rosetta pod and the PostgreSQL instance
# Expected: CPU and memory spike >30% above 24h baseline within seconds;
# no brute-force alert fires because all requests are structurally valid and distinct.
```

```go
func (s *blockAPIService) Block(
	ctx context.Context,
	request *rTypes.BlockRequest,
) (*rTypes.BlockResponse, *rTypes.Error) {
	block, err := s.RetrieveBlock(ctx, request.BlockIdentifier)
	if err != nil {
		return nil, err
	}

	if block.Transactions, err = s.FindBetween(ctx, block.ConsensusStartNanos, block.ConsensusEndNanos); err != nil {
		return nil, err
	}

	var otherTransactions []*rTypes.TransactionIdentifier
	if len(block.Transactions) > s.maxTransactionsInBlock {
		otherTransactions = make([]*rTypes.TransactionIdentifier, 0, len(block.Transactions)-s.maxTransactionsInBlock)
		for _, transaction := range block.Transactions[s.maxTransactionsInBlock:] {
			otherTransactions = append(otherTransactions, &rTypes.TransactionIdentifier{Hash: transaction.Hash})
		}
		block.Transactions = block.Transactions[0:s.maxTransactionsInBlock]
	}

	if err = s.updateOperationAccountAlias(ctx, block.Transactions...); err != nil {
		return nil, err
	}

	return &rTypes.BlockResponse{Block: block.ToRosetta(), OtherTransactions: otherTransactions}, nil
```

```go
func (a *AccountAPIService) AccountBalance(
	ctx context.Context,
	request *rTypes.AccountBalanceRequest,
) (*rTypes.AccountBalanceResponse, *rTypes.Error) {
	accountId, err := types.NewAccountIdFromString(request.AccountIdentifier.Address, a.systemShard, a.systemRealm)
	if err != nil {
		return nil, errors.ErrInvalidAccount
	}

	var block *types.Block
	var rErr *rTypes.Error
	if request.BlockIdentifier != nil {
		block, rErr = a.RetrieveBlock(ctx, request.BlockIdentifier)
	} else {
		block, rErr = a.RetrieveLatest(ctx)
	}
	if rErr != nil {
		return nil, rErr
	}

	balances, accountIdString, publicKey, rErr := a.accountRepo.RetrieveBalanceAtBlock(ctx, accountId, block.ConsensusEndNanos)
	if rErr != nil {
		return nil, rErr
	}

	metadata := make(map[string]any)
	if accountId.HasAlias() && accountIdString != "" {
		metadata["account_id"] = accountIdString
	}
	if isEd25519PublicKey(publicKey) {
		metadata["public_key"] = tools.SafeAddHexPrefix(hex.EncodeToString(publicKey))
	}

	return &rTypes.AccountBalanceResponse{
		BlockIdentifier: block.GetRosettaBlockIdentifier(),
		Balances:        balances.ToRosetta(),
		Metadata:        metadata,
	}, nil
}
```
