### Title
SSRF via Unvalidated `DomainName`/`IPAddressV4` in `buildNetworkFromMirrorNodes()`

### Summary
The pinger's `fetchMirrorNodeNetwork()` function fetches node endpoint data from a configurable REST URL and blindly trusts the `DomainName` (or `IPAddressV4`) and `Port` fields returned in the JSON response. No allowlist, RFC-1918 block, or hostname validation is applied before those values are assembled into gRPC dial targets and handed to the Hiero SDK. An attacker who controls the HTTP server that `HIERO_MIRROR_PINGER_REST` resolves to can therefore redirect all outbound gRPC connections to any host reachable from the pinger's network, achieving full SSRF.

### Finding Description
**Exact code path:**

`pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 113–124:

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)   // line 114 – raw attacker value
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4) // line 116 – raw attacker value
    }
    if host == "" || ep.Port == 0 {
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port)) // line 122
    network[addr] = nodeAccountId
}
```

The returned `network` map is passed directly to `hiero.ClientForNetwork(netmap)` in `pinger/sdk_client.go` line 22, which causes the Hiero SDK to open gRPC connections to every address in the map.

**Root cause:** The code assumes the REST endpoint is trusted infrastructure. The only checks applied to `DomainName`/`IPAddressV4` are `strings.TrimSpace` and an empty-string guard. There is no:
- IP allowlist / denylist (no RFC-1918 / loopback / link-local blocking)
- Hostname validation (no suffix check against known public domains)
- Port range restriction

**Failed assumption:** The developer assumes `cfg.mirrorRest` always points to a legitimate, operator-controlled mirror node. If that assumption breaks (misconfiguration, DNS hijack, compromised upstream), the attacker gains full control over which hosts the pinger dials.

### Impact Explanation
The pinger process typically runs inside a Kubernetes pod or container with access to the cluster-internal network. An attacker who can serve a crafted response can:
1. **Port-scan** internal services by observing connection timing/errors.
2. **Reach internal gRPC services** (e.g., other Hedera nodes, internal APIs) that are not exposed externally.
3. **Hit cloud metadata endpoints** (e.g., `169.254.169.254:80`) to exfiltrate IAM credentials, depending on cloud provider and metadata service configuration.
4. **Pivot** to any TCP endpoint reachable from the pinger's network namespace.

Severity: **High** — the pinger has broad internal network reachability and the SSRF is unconditional once the precondition is met.

### Likelihood Explanation
The precondition — `HIERO_MIRROR_PINGER_REST` pointing to an attacker-controlled URL — can arise from:
- **Misconfiguration** (operator typo, wrong ConfigMap value in Kubernetes).
- **DNS hijack / BGP hijack** of the mirror-node hostname.
- **Compromise of the upstream mirror-node REST service** (supply-chain style).
- **Man-in-the-middle** on an unencrypted `http://` REST URL (the default example in config.go is `http://mirror-rest:5551`).

The attacker does not need any credentials or access to the pinger's deployment environment — they only need to be able to serve HTTP responses from the URL the pinger is configured to contact. The attack is fully repeatable on every startup of the pinger (and on any periodic refresh if one is added).

### Recommendation
1. **Validate returned hostnames/IPs before use.** After parsing the JSON, check each `host` value:
   - Reject RFC-1918 addresses (`10.x`, `172.16–31.x`, `192.168.x`), loopback (`127.x`), and link-local (`169.254.x`).
   - Optionally enforce an allowlist of expected domain suffixes.
2. **Restrict the port range** to known Hedera gRPC ports (e.g., 50211, 50212).
3. **Enforce TLS** for `cfg.mirrorRest` in production to prevent MitM injection.
4. **Pin the mirror-node URL** to a static, operator-verified value and document that it must not be set to untrusted URLs.

### Proof of Concept
**Preconditions:**
- `HIERO_MIRROR_PINGER_NETWORK=other`
- `HIERO_MIRROR_PINGER_REST=http://attacker.example.com`

**Attacker's HTTP server responds to `GET /api/v1/network/nodes` with:**
```json
{
  "nodes": [
    {
      "node_account_id": "0.0.3",
      "service_endpoints": [
        {
          "domain_name": "169.254.169.254",
          "ip_address_v4": "",
          "port": 80
        }
      ]
    }
  ],
  "links": { "next": null }
}
```

**Trigger:** Start the pinger normally. `buildNetworkFromMirrorNodes()` fetches the crafted response, `fetchMirrorNodeNetwork()` constructs `addr = "169.254.169.254:80"`, and `hiero.ClientForNetwork` dials that address. The pinger now sends gRPC traffic (including operator credentials in metadata) to the AWS/GCP/Azure instance-metadata endpoint — or any other internal host substituted in `domain_name`.

**Observable result:** TCP connection attempt (and potential gRPC handshake) to `169.254.169.254:80` originates from the pinger pod, confirming SSRF.
