### Title
Unbounded Unthrottled Gap-Recovery Query via Safety Check Path Enables Resource Exhaustion DoS

### Summary
An unprivileged gRPC subscriber can trigger the safety-check code path in `TopicMessageServiceImpl.subscribeTopic()` to issue an unthrottled, limit-free database polling loop against any topic. Because the `gapFilter` built in `missingMessages(topicContext, null)` inherits the original filter's absent limit and absent endTime, the unthrottled retriever will execute up to `maxPolls+1` (13) database queries of up to `maxPageSize` (5,000) rows each — 65,000 rows per subscription — with no per-subscriber cap. With many concurrent subscriptions this exhausts database connection pool, CPU, and heap.

### Finding Description

**Exact code path:**

`ConsensusController.subscribeTopic()` → `TopicMessageService.subscribeTopic()` → safety-check Flux → `missingMessages(topicContext, null)` → `topicMessageRetriever.retrieve(gapFilter, false)`
