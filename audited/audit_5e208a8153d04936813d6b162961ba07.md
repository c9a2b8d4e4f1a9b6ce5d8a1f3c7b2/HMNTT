### Title
Unbounded Prometheus Label Cardinality via User-Controlled Path in `toOpenApiPath()`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to `req.path` for unmatched routes (where `req.route` is `null`). Because `req.path` is fully attacker-controlled, an unprivileged user can generate an unbounded number of unique `path` label values in per-route Prometheus instruments, causing label cardinality explosion that exhausts Prometheus/OpenTelemetry SDK memory and degrades or destroys the monitoring plane.

### Finding Description
**Exact code path:**

`rest/middleware/metricsHandler.js`, `toOpenApiPath()`, lines 13â€“31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // line 14 â€“ set only for matched sub-router routes

  if (!path) {
    if (!req.route) {
      path = req.path;                        // line 18 â€“ raw, user-supplied path
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }

  path = path.replace(/:([^/]+)/g, '{$1}');

  if (!path.startsWith(apiPrefix)) {          // apiPrefix = '/api/v1'
    return apiPrefix + '/' + path;            // line 27 â€“ concatenation, not normalization
  }
  return path;
};
```

`apiPrefix` is `/api/v1` (constants.js line 19). `requestPathLabel` (`'requestPath'`) is only written by `recordRequestPath` in `rest/routes/index.js` for sub-router-matched requests (line 19), and by `responseHandler` only for matched routes. For any request that does not match a registered route, both `res.locals[requestPathLabel]` and `req.route` are `null`, so `path = req.path` â€” the raw, user-supplied URL path.

This value is then used verbatim as the `path` label in all four per-route instruments (lines 207â€“210):

```js
const labels = {method, path, code};
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
```

**Doubled-prefix sub-case (as asked):** If an attacker sends `GET /foo` (not starting with `/api/v1`), line 27 returns `/api/v1//foo`. If the attacker sends `GET /foo/api/v1/accounts`, line 27 returns `/api/v1//foo/api/v1/accounts` â€” a label value that contains `apiPrefix` twice and is distinct from every legitimate route label. Each such unique string registers a new time-series in the OpenTelemetry SDK's in-process accumulation map.

**Root cause / failed assumption:** The code assumes that only a bounded set of known route patterns will ever reach the `!req.route` branch. In reality, every 404 response passes through this branch with an arbitrary attacker-supplied path.

### Impact Explanation
Each unique `{method, path, code}` triple causes the OpenTelemetry SDK (and downstream Prometheus scraper) to allocate a new time-series accumulator. With four instruments (`requestTotalCounter`, `durationHistogram`, `requestSizeHistogram`, `responseSizeHistogram`) each having multiple histogram buckets, a single unique path generates O(tens) of in-memory objects. Sending N distinct paths generates O(N Ă— tens) objects. At scale this exhausts Node.js heap memory, crashes the mirror-node REST process, and destroys the metrics pipeline â€” directly impairing the ability to observe and operate the network. The severity is **High** (availability impact on the monitoring/API plane).

### Likelihood Explanation
No authentication is required. Any HTTP client can send GET requests to arbitrary paths. The attack is trivially scriptable: a loop generating `GET /api/v1/<random-uuid>` requests produces a new unique label on every iteration. The metrics middleware is applied globally (`app.useExt(metricsHandler())`, server.js line 91) and fires on every request including 404s. The only gate is `config.metrics.enabled`, which is expected to be `true` in production deployments.

### Recommendation
Replace the raw-path fallback with a fixed sentinel for unmatched routes:

```js
if (!path) {
  if (!req.route) {
    path = 'unmatched';          // fixed label â€” no user input
  } else {
    path = (req.baseUrl ?? '') + req.route.path;
  }
}
```

Alternatively, maintain an allowlist of known OpenAPI path patterns and map any unrecognized path to a single `unknown` bucket. Never use `req.path` or any other user-supplied value as a Prometheus label.

### Proof of Concept
```bash

  curl -s "http://localhost:5551/api/v1/nonexistent-$(uuidgen)" &

# Scrape metrics and count unique path label values:
