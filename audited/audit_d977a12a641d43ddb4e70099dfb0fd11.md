### Title
Unauthenticated `/health/readiness` Endpoint Triggers Unbounded Internal NetworkList + NetworkStatus Amplification, Enabling DoS Against Smart Contract Transaction Processing

### Summary
The `GET /health/readiness` endpoint in `rosetta/app/middleware/health.go` is unauthenticated, has no application-level rate limiting, and no response caching. Every inbound request synchronously triggers two internal Rosetta API calls (`NetworkList` + `NetworkStatus`) plus multiple database queries. An unprivileged attacker who floods this endpoint can exhaust the server's goroutine pool and database connection pool, degrading or blocking smart contract transaction processing on the same Rosetta instance.

### Finding Description
**Code path:**

`NewHealthController` registers a `readinessHealth` object with two checks — `postgresql` and `network` — with no `WithCacheDuration` option: [1](#0-0) 

The `network` check is `checkNetworkStatus`, which on every invocation creates a full round-trip to the same server: [2](#0-1) 

`NetworkStatus` itself performs three synchronous database queries — `RetrieveGenesis`, `RetrieveLatest`, and `Entries` (address book): [3](#0-2) 

`Routes()` exposes both endpoints with no authentication or throttling middleware: [4](#0-3) 

**Root cause:** The `hellofresh/health-go/v4` library executes all registered checks synchronously on every HTTP request when `WithCacheDuration` is not set. The code never sets it.

**Why existing checks fail:**

The Traefik-level rate limiter (`average: 10`) is:
1. Disabled by default (`global.middleware: false`): [5](#0-4) 
2. Even when enabled, it only covers `/rosetta/*` ingress paths — `/health/readiness` is not in that list: [6](#0-5) 

The server port (5700) is directly exposed externally: [7](#0-6) 

There is no application-level rate limiting or concurrency cap on the health handler in `rosetta/app/middleware/`.


### Impact Explanation
Each `GET /health/readiness` request fans out into: 1 internal HTTP `POST /network/list` + 1 internal HTTP `POST /network/status` + 3 DB queries (genesis block, latest block, address book entries). At 100 req/s from a single attacker, this generates 200 internal API calls and 300 DB queries per second. The shared DB connection pool and Go HTTP server goroutine pool become saturated, causing legitimate smart contract transaction requests (which also require DB access and the same network service) to time out or queue indefinitely. No funds are directly at risk, but the network layer becomes unavailable for smart contract execution, matching the Medium severity classification.

### Likelihood Explanation
The attack requires no credentials, no special knowledge, and no prior access — only network reachability to port 5700. The port is documented as externally exposed. A single attacker with a basic HTTP flood tool (e.g., `wrk`, `hey`, `ab`) can trigger the condition. The Traefik middleware that would mitigate this is disabled by default and does not cover the `/health/*` path even when enabled. Repeatability is unlimited.

### Recommendation
1. **Add `WithCacheDuration`** to the readiness health object so checks are not re-executed on every request:
   ```go
   readinessHealth, err := health.New(
       health.WithChecks(readinessChecks...),
       health.WithCacheDuration(10*time.Second),
   )
   ```
2. **Add a concurrency limiter** (e.g., `golang.org/x/time/rate` or a semaphore) inside the health handler or as a wrapping middleware specific to `/health/*` routes.
3. **Restrict network exposure** of `/health/*` to cluster-internal traffic only (e.g., via a separate internal-only port or NetworkPolicy), separate from the externally-facing Rosetta API port.
4. **Enable and extend the Traefik `inFlightReq`/`rateLimit` middleware** to cover `/health/*` paths, or enforce it at the application level unconditionally.

### Proof of Concept
```bash
# Precondition: Rosetta server running and port 5700 reachable externally
# No credentials needed

# Step 1: Confirm the endpoint is open and unauthenticated
curl -s http://<target>:5700/health/readiness

# Step 2: Flood the readiness endpoint (single attacker, no special tooling)
hey -n 100000 -c 200 http://<target>:5700/health/readiness &

# Step 3: Simultaneously attempt a legitimate smart contract call
curl -s -X POST http://<target>:5700/call \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{...},"contract_identifier":{...},"data":"..."}'

# Expected result under flood:
# - /call and other Rosetta endpoints time out or return 503
# - DB connection pool exhausted (visible in server logs)
# - Each readiness request generates NetworkList + NetworkStatus internal calls
#   (visible via: grep "Readiness check" server logs)
```

func checkNetworkStatus(port uint16) func(ctx context.Context) error {
	serverUrl := fmt.Sprintf("http://localhost:%d", port)
	cfg := client.NewConfiguration(serverUrl, "readiness-check", nil)
	rosettaClient := client.NewAPIClient(cfg)

	return func(ctx context.Context) (checkErr error) {
		networkList, _, err := rosettaClient.NetworkAPI.NetworkList(ctx, &types.MetadataRequest{})
		if err != nil {
			log.Errorf("Readiness check, /network/list failed: %v", err)
			return err
		}

		network := networkList.NetworkIdentifiers[0]
		_, _, err = rosettaClient.NetworkAPI.NetworkStatus(ctx, &types.NetworkRequest{NetworkIdentifier: network})
		if err != nil {
			log.Errorf("Readiness check, /network/status failed: %v", err)
			return err
		}

		return
	}

```go
func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
```
