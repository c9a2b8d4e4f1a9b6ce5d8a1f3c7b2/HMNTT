### Title
Unauthenticated DoS via DB Connection Exhaustion on `/network/status` with Rate-Limiting Disabled by Default

### Summary
`NetworkStatus()` in `rosetta/app/services/network_service.go` executes up to four sequential database queries per request with no application-level concurrency or rate control. The Traefik middleware chain that provides `inFlightReq` and `rateLimit` protection is gated behind `global.middleware: false` in the default Helm values, meaning it is **off by default**. An unprivileged attacker can flood the endpoint with concurrent requests, exhaust the finite DB connection pool (`MaxOpenConnections`), and deny service to all legitimate clients.

### Finding Description

**Code path — per-request DB load:**

`NetworkStatus()` at [1](#0-0)  makes three sequential repository calls:
1. `n.RetrieveGenesis(ctx)` → `blockRepo.RetrieveGenesis(ctx)`
2. `n.RetrieveLatest(ctx)` → `blockRepo.RetrieveLatest(ctx)`
3. `n.addressBookEntryRepo.Entries(ctx)` → up to **two** SQL queries (iterates over file IDs 101 and 102)

`Entries()` at [2](#0-1)  acquires a DB handle via `aber.dbClient.GetDbWithContext(ctx)` and runs raw SQL for each file ID in a loop, consuming a connection slot for the duration of each query.

**DB connection pool — finite resource:**

`ConnectToDb` at [3](#0-2)  calls `sqlDb.SetMaxOpenConns(dbConfig.Pool.MaxOpenConnections)`. This is a hard ceiling. When all slots are occupied, new queries block until a slot is freed or the context times out.

**No application-level rate limiting:**

`main.go` assembles the middleware stack as: [4](#0-3) 
```
MetricsMiddleware → TracingMiddleware → CorsMiddleware → Router
```
There is no rate limiter, semaphore, or in-flight request cap anywhere in the Go application.

**Traefik middleware disabled by default:**

The Helm chart defines `inFlightReq` (5 concurrent per IP) and `rateLimit` (10 req/s) in `values.yaml`: [5](#0-4) 

But the middleware CRDs are only created when **both** `global.middleware` and `middleware` are truthy: [6](#0-5) 

The default value is: [7](#0-6) 
```yaml
global.middleware: false
```
With this default, no Traefik Middleware CRDs are deployed, so **no rate limiting or concurrency limiting is active**.

**Additional amplification when middleware IS enabled:**

The `retry` middleware is configured with `attempts: 3`: [8](#0-7)  This means each attacker request can trigger up to 3 upstream DB-hitting requests, tripling the load. Furthermore, `inFlightReq` uses `ipStrategy: depth: 1` (reads `X-Forwarded-For[0]`), which an attacker can spoof to bypass the per-IP cap: [9](#0-8)  The `rateLimit` uses `sourceCriterion: requestHost: true` — a single global bucket shared by all clients, not per-source-IP: [10](#0-9) 

### Impact Explanation
An attacker who exhausts `MaxOpenConnections` causes all subsequent DB operations across the entire Rosetta service to block or fail. This affects not only `/network/status` but every other online-mode endpoint (`/block`, `/account/balance`, etc.) that shares the same GORM pool. The result is a complete service outage for all legitimate clients — a network partition from the perspective of any Rosetta consumer (e.g., exchange integrations, validators). No authentication is required; the endpoint is publicly reachable by design per the Rosetta API specification.

### Likelihood Explanation
The attack requires only an HTTP client capable of sending concurrent POST requests to `/network/status` — no credentials, no special knowledge, no exploit chain. The default Helm deployment ships with `global.middleware: false`, meaning the vast majority of deployments have zero ingress-level protection. Even operators who notice the middleware values and set `global.middleware: true` face the `X-Forwarded-For` spoofing bypass for `inFlightReq` and the retry amplification. The attack is trivially scriptable with tools like `ab`, `wrk`, or `hey` and is fully repeatable.

### Recommendation
1. **Application layer (primary fix):** Add a semaphore or `golang.org/x/time/rate` token-bucket limiter inside the Go HTTP handler chain in `main.go`, before the router, to cap concurrent in-flight requests and per-IP request rate independently of infrastructure configuration.
2. **DB pool:** Set a conservative `MaxOpenConnections` and ensure `StatementTimeout` is always configured so runaway queries release connections promptly.
3. **Helm default:** Change `global.middleware` default to `true` so the Traefik middleware chain is active out of the box.
4. **Fix `inFlightReq` bypass:** Use `ipStrategy: depth: 0` (real peer IP) or add trusted proxy configuration to prevent `X-Forwarded-For` spoofing.
5. **Fix `rateLimit` criterion:** Change `sourceCriterion` from `requestHost` to `ipStrategy` so the rate limit is per source IP, not a shared global bucket.
6. **Remove `retry`** from the middleware chain for this service, or scope it only to network errors — retrying on 5xx responses amplifies DB load from both legitimate errors and attacker-induced failures.

### Proof of Concept
```bash
# Precondition: Rosetta running in online mode, default Helm deploy (global.middleware=false)
# Tool: wrk (or hey/ab)

wrk -t 50 -c 500 -d 60s \
  -s <(echo 'wrk.method="POST"; wrk.body="{}"; wrk.headers["Content-Type"]="application/json"') \
  http://<rosetta-host>/network/status

# Expected result:
# - DB connection pool slots exhausted within seconds
# - Subsequent requests to /block, /account/balance, /network/status all return 500
#   (ErrDatabaseError) or hang until StatementTimeout fires
# - Service effectively unavailable to all legitimate clients
```

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (aber *addressBookEntryRepository) Entries(ctx context.Context) (*types.AddressBookEntries, *rTypes.Error) {
	db, cancel := aber.dbClient.GetDbWithContext(ctx)
	defer cancel()

	nodes := make([]nodeServiceEndpoint, 0)
	// address book file 101 has service endpoints for nodes, resort to file 102 if 101 doesn't exist
	for _, fileId := range []int64{aber.addressBook101.EncodedId, aber.addressBook102.EncodedId} {
		if err := db.Raw(
			latestNodeServiceEndpoints,
			sql.Named("file_id", fileId),
		).Scan(&nodes).Error; err != nil {
			log.Error("Failed to get latest node service endpoints", err)
			return nil, errors.ErrDatabaseError
		}

		if len(nodes) != 0 {
			break
		}
	}

	entries := make([]types.AddressBookEntry, 0, len(nodes))
	for _, node := range nodes {
		entries = append(entries, node.toAddressBookEntry())
	}

	return &types.AddressBookEntries{Entries: entries}, nil
}
