### Title
Global Gas-Bucket Exhaustion DoS via Concurrent Max-Gas `POST /api/v1/contracts/call` Requests

### Summary
The `gasLimitBucket` in `ThrottleManagerImpl.throttle()` is a single, global, unauthenticated token-bucket with a capacity of exactly 750,000 scaled tokens. A single unprivileged attacker can send 500 concurrent `POST /api/v1/contracts/call` requests each with `gas=15_000_000`, atomically consuming all 750,000 tokens in one burst. Because token restoration (`restoreGasToBucket`) only occurs after EVM execution completes — which can take up to the 10-second `requestTimeout` — every legitimate request during that window receives `ThrottleException("Gas per second rate limit exceeded.")`. The attack is repeatable every second because the rate-limit bucket also refills at 500/second.

### Finding Description

**Exact code path:**

`ThrottleProperties.scaleGas()` divides raw gas by `GAS_SCALE_FACTOR = 10_000`: [1](#0-0) 

`getGasPerSecond()` returns `scaleGas(7_500_000_000)` = **750,000 tokens**. The bucket is configured with that as both capacity and refill rate: [2](#0-1) 

`ThrottleManagerImpl.throttle()` consumes `scaleGas(request.getGas())` tokens atomically. For `gas=15_000_000` that is `15_000_000 / 10_000 = 1_500` tokens per request: [3](#0-2) 

The rate-limit bucket capacity is `requestsPerSecond = 500`. Therefore `500 requests × 1,500 tokens = 750,000 tokens` — exactly the full gas bucket — can be consumed in a single burst by one attacker.

Token restoration only happens inside `ContractCallService.doProcessCall()` in the `finally` block, **after** EVM execution finishes: [4](#0-3) 

`restoreGasToBucket` restores `min(gasLimit - gasUsed, gasLimitToRestoreBaseline)` for successful calls, meaning gas actually consumed by the EVM is **never returned**: [5](#0-4) 

`ContractController.call()` only restores gas on `InvalidParametersException`, not on `ThrottleException` (which is thrown before any processing): [6](#0-5) 

**Root cause:** Both `rateLimitBucket` and `gasLimitBucket` are single global Spring beans with no per-IP, per-user, or per-connection partitioning. The bucket capacity is arithmetically equal to what 500 max-gas requests consume in one burst, and the restoration path is deferred until after execution.

**Exploit flow:**
1. Attacker sends 500 concurrent `POST /api/v1/contracts/call` with `gas=15000000` targeting a gas-heavy contract (e.g., a loop-heavy or storage-heavy function).
2. All 500 pass `rateLimitBucket.tryConsume(1)` (consuming all 500 rate-limit tokens).
3. All 500 pass `gasLimitBucket.tryConsume(1500)` (consuming all 750,000 gas tokens).
4. Gas bucket is now at 0. Any new request hits `ThrottleException("Gas per second rate limit exceeded.")`.
5. EVM execution of the 500 requests proceeds for up to `requestTimeout = 10,000 ms`.
6. During that window, all legitimate `eth_call` requests are rejected.
7. After 1 second the rate-limit bucket refills; attacker sends another 500 requests before gas is restored, sustaining the DoS.

**Why existing checks are insufficient:**
- The `gasLimitRefundPercent = 100` refund only helps for failed transactions (full refund) or successful ones (partial refund of unused gas). It does not prevent the temporal depletion window.
- The `requestsPerSecond = 500` rate limit is itself the attack vector — it is calibrated to allow exactly enough requests to exhaust the gas bucket.
- There is no IP-based, session-based, or authenticated rate limiting anywhere in the throttle stack.

### Impact Explanation
All `POST /api/v1/contracts/call` (eth_call simulation) requests from legitimate users receive HTTP 429 `ThrottleException` for the duration of the attack. Since the mirror node's web3 module is the sole interface for EVM contract-call simulation (used by wallets, dApps, and tooling), this constitutes a complete denial of service for that API surface. The attack does not require any on-chain interaction or funds.

### Likelihood Explanation
The attack requires no credentials, no special network position, and no prior knowledge beyond the public API. A single machine with a standard HTTP client library can issue 500 concurrent requests. The attack is repeatable every second indefinitely. The attacker bears no cost (no gas fees, no tokens). This is trivially scriptable.

### Recommendation
1. **Per-IP rate limiting**: Introduce a per-source-IP token bucket (e.g., via a Spring `HandlerInterceptor` or a reverse-proxy rule) so a single client cannot consume the global gas budget.
2. **Decouple bucket capacity from max-rate × max-gas**: Increase `gasPerSecond` (or reduce `maxGasLimit`) so that the product `requestsPerSecond × scaleGas(maxGasLimit)` is significantly less than the bucket capacity, leaving headroom for legitimate traffic.
3. **Eager restoration on throttle path**: If a request is throttled at the gas-bucket check after the rate-limit check has already passed, ensure no tokens are silently lost (currently `tryConsume` is atomic so this is fine, but document the invariant).
4. **Async/queue-based admission**: Accept requests into a bounded queue and admit them to the EVM only as gas tokens become available, rather than rejecting outright.

```java
    private void restoreGasToBucket(EvmTransactionResult result, long gasLimit) {
        // If the transaction fails, gasUsed is equal to gasLimit, so restore the configured refund percent
        // of the gasLimit value back in the bucket.
        final var gasLimitToRestoreBaseline = (long) (gasLimit * throttleProperties.getGasLimitRefundPercent() / 100f);
        if (result == null || (!result.isSuccessful() && gasLimit == result.gasUsed())) {
            throttleManager.restore(gasLimitToRestoreBaseline);
        } else {
            // The transaction was successful or reverted, so restore the remaining gas back in the bucket or
            // the configured refund percent of the gasLimit value back in the bucket - whichever is lower.
            final var gasRemaining = gasLimit - result.gasUsed();
            throttleManager.restore(Math.min(gasRemaining, gasLimitToRestoreBaseline));
        }
    }
```
