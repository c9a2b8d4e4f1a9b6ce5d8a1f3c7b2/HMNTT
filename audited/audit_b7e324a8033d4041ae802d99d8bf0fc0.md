### Title
Unauthenticated IP Spoofing via Unvalidated `X-Forwarded-For` Header in `getClientIpAddress()`

### Summary
`getClientIpAddress()` in `rosetta/app/middleware/trace.go` blindly returns the raw `X-Forwarded-For` header value without parsing or validation. Any unprivileged external client can inject a multi-value string (e.g., `8.8.8.8, 192.168.1.1`), causing the entire unsanitized string to be logged as the client IP. Because the application is deployed with no trusted-proxy layer enforcing XFF stripping (as seen in `main.go`), this is exploitable by default with zero privileges.

### Finding Description
**Exact code path:**

`rosetta/app/middleware/trace.go`, `getClientIpAddress()`, lines 63–74:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)          // line 64

    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader)  // line 67 — raw, unvalidated
    }

    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    return ipAddress  // returned and logged verbatim
}
``` [1](#0-0) 

The returned value is logged verbatim at line 52–53:

```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
``` [2](#0-1) 

**Root cause:** The function performs only a length check (`len(ipAddress) == 0`) — it never:
- Parses the first (leftmost) or last (rightmost) IP from a comma-separated XFF list
- Validates that the value is a syntactically valid IP address
- Restricts trust to requests arriving from a known trusted proxy

**Deployment context:** In `main.go` lines 217–219, `TracingMiddleware` is applied directly to the router with no upstream proxy trust configuration and no XFF stripping:

```go
metricsMiddleware := middleware.MetricsMiddleware(router)
tracingMiddleware := middleware.TracingMiddleware(metricsMiddleware)
corsMiddleware := server.CorsMiddleware(tracingMiddleware)
``` [3](#0-2) 

This means any external client can set `X-Forwarded-For` to an arbitrary value and it will be accepted as the authoritative client IP.

**Test gap:** The test suite only exercises single-IP XFF values and never tests multi-value or malformed inputs, so this path has never been validated: [4](#0-3) 

### Impact Explanation
The logged "client IP" becomes an attacker-controlled arbitrary string. Any security control that consumes these logs for IP-based decisions (firewall rules, SIEM alerting, rate-limit enforcement, incident response blocking) will operate on the injected string rather than the real source IP. An exact-match rule against `8.8.8.8` will never match the logged value `8.8.8.8, 192.168.1.1`, so the attacker's real IP is never blocked. This enables sustained request flooding that cannot be mitigated through log-derived controls, which can exhaust the node's resources and partition it from the network. Severity: **High** — complete bypass of log-based IP blocking with zero privileges required.

### Likelihood Explanation
Preconditions are minimal: the attacker only needs HTTP access to the Rosetta port (the default deployment exposes this directly). No authentication, no special role, no prior access is required. The attack is trivially repeatable with a single curl command. Any deployment that relies on log-based firewall rules or SIEM-driven blocking is immediately affected.

### Recommendation
Fix `getClientIpAddress()` to extract only the **first** IP from the XFF header (the leftmost entry, representing the original client in a correctly configured proxy chain) and validate it as a well-formed IP:

```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)

    if len(ipAddress) == 0 {
        xff := r.Header.Get(xForwardedForHeader)
        if len(xff) > 0 {
            // Take only the first (leftmost) entry
            parts := strings.SplitN(xff, ",", 2)
            ipAddress = strings.TrimSpace(parts[0])
        }
    }

    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    // Validate it is actually an IP
    if net.ParseIP(ipAddress) == nil {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }

    return ipAddress
}
```

Additionally, if a trusted reverse proxy is deployed in front of the service, configure it to strip and rewrite `X-Forwarded-For` before requests reach the application, so client-supplied values are never trusted.

### Proof of Concept

```bash
# Attacker sends a request with a spoofed XFF header
curl -X POST http://<rosetta-node>:<port>/network/list \
  -H "Content-Type: application/json" \
  -H "X-Forwarded-For: 8.8.8.8, 192.168.1.1" \
  -d '{"network_identifier": {"blockchain": "Hedera", "network": "mainnet"}, "metadata": {}}'

# Resulting log entry (from TracingMiddleware):
# time="..." level=info msg="8.8.8.8, 192.168.1.1 POST /network/list (200) in 3ms"

# A firewall rule blocking exact IP "8.8.8.8" will NOT match "8.8.8.8, 192.168.1.1"
# The attacker's real source IP never appears in the log; blocking is impossible.
```
