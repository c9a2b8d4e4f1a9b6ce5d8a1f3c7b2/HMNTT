### Title
Unauthenticated DoS via Connection Pool Exhaustion on `/api/v1/network/nodes` Due to Missing Rate Limiting and Expensive Unbounded CTE Query

### Summary
The `GET /api/v1/network/nodes` endpoint in `NetworkController` is publicly accessible with no rate limiting in the `rest-java` module. Each request executes a complex multi-CTE native SQL query in `NetworkNodeRepository.findNetworkNodes()` that performs multiple full-table scans and a correlated subquery regardless of the `limit` cap. An unprivileged attacker flooding this endpoint with concurrent requests will exhaust the database connection pool, making the node address book unavailable and preventing clients from discovering consensus nodes.

### Finding Description

**Code path:**

`NetworkController.getNodes()` (line 152–171) accepts any unauthenticated GET request and delegates to `NetworkServiceImpl.getNetworkNodes()` (lines 99–137), which calls `NetworkNodeRepository.findNetworkNodes()` (lines 104–105). [1](#0-0) 

The `rest-java` configuration directory contains only `MetricsFilter`, `LoggingFilter`, `RestJavaConfiguration` (ETag + protobuf), and `WebMvcConfiguration` — **no rate-limiting filter exists**: [2](#0-1) 

The only throttle implementation in the codebase lives in the `web3` module and is not applied to `rest-java`: [3](#0-2) 

The `limit` is capped at 25 by `getEffectiveLimit()`: [4](#0-3) 

However, the native SQL query in `findNetworkNodes()` executes three CTEs unconditionally on every call regardless of the limit:
- `latest_address_book`: scans `address_book` ordered by `start_consensus_timestamp DESC`
- `latest_node_stake`: performs `SELECT max(consensus_timestamp) FROM node_stake` (full table scan) then fetches all rows at that timestamp
- `node_info`: full scan of `node` table
- A **correlated subquery** on `address_book_service_endpoint` per result row [5](#0-4) 

The `limit :limit` clause only bounds the result set rows returned; it does **not** prevent the CTEs from scanning their full tables. Each concurrent request holds a JDBC connection for the full query duration.

`NetworkServiceImpl.getNetworkNodes()` passes `lowerBound=0` and `upperBound=Long.MAX_VALUE` by default when no `node.id` filter is supplied: [6](#0-5) 

**Root cause:** The failed assumption is that `limit=25` is sufficient to bound per-request database cost. It is not — the CTE structure causes full-table scans on every invocation, and no rate limiting prevents concurrent request flooding.

### Impact Explanation

With a default HikariCP connection pool (typically 10 connections), an attacker sending ~20–50 concurrent requests per second will exhaust all available connections. Subsequent legitimate requests receive connection-pool-timeout errors. The `/api/v1/network/nodes` endpoint is the canonical mechanism for clients (SDKs, wallets, dApps) to discover consensus node endpoints and account IDs needed to submit transactions. Unavailability of this endpoint prevents new transaction submission, constituting a network-level denial of service. Severity: **High** (availability impact on critical infrastructure discovery path, no authentication required).

### Likelihood Explanation

The attack requires zero privileges, zero authentication, and only a standard HTTP client. The endpoint is publicly documented in the OpenAPI spec: [7](#0-6) 

The attack is trivially repeatable with tools like `ab`, `wrk`, or `hey`. No special knowledge of the protocol is needed. The absence of any rate-limiting infrastructure in `rest-java` (confirmed by directory listing of `config/`) means there is no server-side defense. Likelihood: **High**.

### Recommendation

1. **Add rate limiting to `rest-java`**: Implement a `OncePerRequestFilter` using Bucket4j (already a transitive dependency via `web3`) scoped to `/api/v1/network/**`, mirroring the pattern in `web3/src/main/java/org/hiero/mirror/web3/config/ThrottleConfiguration.java`.
2. **Add query-level timeout**: Set a `javax.persistence.query.timeout` or Spring `@QueryHints` timeout on `findNetworkNodes()` to bound per-query wall-clock time.
3. **Cache the result**: The address book changes infrequently. Apply Spring `@Cacheable` on `NetworkServiceImpl.getNetworkNodes()` with a short TTL (e.g., 30s) to serve repeated requests from memory rather than hitting the database.
4. **Optimize the CTE**: Materialize `latest_node_stake` with `WITH ... AS MATERIALIZED` or pre-aggregate via a scheduled job to avoid the full `node_stake` scan on every request.

### Proof of Concept

```bash

# Observe: after pool exhaustion, legitimate requests return 500/503

# Variant: maximize query cost with wide range (default, no params needed)

  curl -s "https://<mirror-node-host>/api/v1/network/nodes" &
```

```java
                where consensus_timestamp = (select max(consensus_timestamp) from node_stake)
            ),
            node_info as (
                select account_id, admin_key, associated_registered_nodes, decline_reward, grpc_proxy_endpoint, node_id
                from node
```

```java
        long lowerBound = 0L;
        long upperBound = Long.MAX_VALUE;

        for (final var nodeIdParam : nodeIdParams) {
            if (nodeIdParam.operator() == RangeOperator.EQ) {
                nodeIds.add(nodeIdParam.value());
            } else if (nodeIdParam.hasLowerBound()) {
                lowerBound = Math.max(lowerBound, nodeIdParam.getInclusiveValue());
            } else if (nodeIdParam.hasUpperBound()) {
                upperBound = Math.min(upperBound, nodeIdParam.getInclusiveValue());
            }
        }

        if (lowerBound > upperBound) {
            throw new IllegalArgumentException("Invalid range provided for node.id");
        }

        final Long[] nodeIdArray;
        if (!nodeIds.isEmpty()) {
            final var range = Range.closed(lowerBound, upperBound);
            nodeIdArray = nodeIds.stream().filter(range::contains).toArray(Long[]::new);
            if (nodeIdArray.length == 0) {
                return List.of();
            }
        } else {
            nodeIdArray = EMPTY_NODE_ID_ARRAY;
        }

        return networkNodeRepository.findNetworkNodes(
                fileId, nodeIdArray, lowerBound, upperBound, orderDirection, limit);
```
