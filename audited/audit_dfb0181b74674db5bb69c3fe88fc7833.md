### Title
Prometheus Label Cardinality Explosion via Unmatched Route Path Injection in `toOpenApiPath()`

### Summary
When an HTTP request does not match any Express route (`req.route` is `null`), `toOpenApiPath()` falls back to using the raw `req.path` as the Prometheus metric label. Because this label is recorded into four unbounded per-route instruments with no sanitization, length cap, or application-level rate limiting, an unauthenticated attacker can flood the server with requests to unique paths, creating an unbounded number of unique time-series entries in the `MeterProvider` registry, exhausting heap memory and crashing the Node.js process.

### Finding Description

**Exact code path:**

In `rest/middleware/metricsHandler.js`, `toOpenApiPath()` at lines 13–31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // line 14 — usually unset for 404s

  if (!path) {
    if (!req.route) {
      path = req.path;                       // line 18 — raw attacker-controlled value
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }
  path = path.replace(/:([^/]+)/g, '{$1}'); // line 24 — no length/cardinality guard
  ...
  return path;
};
```

The returned value is used directly as the `path` label at lines 192 and 207–210:

```js
const labels = {method, path, code};        // line 192
requestTotalCounter.add(1, labels);         // line 207
durationHistogram.record(duration, labels); // line 208
requestSizeHistogram.record(..., labels);   // line 209
responseSizeHistogram.record(..., labels);  // line 210
```

**Root cause:** The assumption that `req.route` will always be set (i.e., only matched routes reach the `finish` handler) is false. The `metricsMiddleware` is registered globally via `app.useExt(metricsHandler())` at `rest/server.js` line 91, *before* any route definitions, so it intercepts every request — including those that result in 404s — and its `res.on('finish', ...)` callback fires unconditionally.

**Why existing checks fail:**
- `res.locals[requestPathLabel]` is only set by route-specific handlers; it is `undefined` for unmatched paths.
- `req.route` is `null` for any path that does not match a registered Express route.
- There is no rate limiting, path length cap, or cardinality guard anywhere in the application middleware stack (`rest/server.js` lines 68–129).
- The `@opentelemetry/sdk-metrics` `MeterProvider` accumulates a new attribute-set entry in its internal `HashMap` for every unique `{method, path, code}` triple. Without an explicit `cardinalityLimit` configured on any instrument (none is set in `initMetrics()`, lines 55–135), the registry grows without bound.

### Impact Explanation
Each unique path creates **four** new time-series entries (one per instrument). An attacker sending N unique paths consumes O(4N) heap entries in the `MeterProvider` registry. Node.js has a default heap limit of ~1.5 GB; at ~1–4 KB per entry, tens of thousands of unique paths are sufficient to trigger OOM. A Node.js OOM crash terminates the entire REST API process, taking down all transaction confirmation endpoints (`/api/v1/transactions`, `/api/v1/accounts`, etc.) — a complete service outage. The process does not recover automatically unless an external supervisor restarts it, and the attack can be repeated immediately after restart.

### Likelihood Explanation
The attack requires zero authentication, zero special headers, and zero knowledge of the application internals. Any HTTP client can send GET requests to `/api/v1/<random_string>` in a tight loop. A single machine with a modest network connection can generate tens of thousands of unique paths per second. The attack is fully repeatable and survives process restarts. No WAF or ingress-level rate limit is configured in the REST API deployment (the Traefik rate-limit config visible in the repo applies only to the Rosetta service, not the REST API).

### Recommendation
1. **Normalize unmatched paths to a fixed sentinel label** — in `toOpenApiPath()`, when `req.route` is `null` and `res.locals[requestPathLabel]` is unset, return a fixed string such as `"unknown"` or `apiPrefix + "/unknown"` instead of `req.path`.
2. **Set an explicit cardinality limit** on all per-route instruments via the `cardinalityLimit` advice option in `initMetrics()`.
3. **Add application-level rate limiting** (e.g., `express-rate-limit`) before the metrics middleware in `rest/server.js`.

### Proof of Concept

```bash
# Send 100,000 requests with unique paths (no auth required)
for i in $(seq 1 100000); do
  curl -s "http://<mirror-node-host>/api/v1/$(cat /dev/urandom | tr -dc 'a-z0-9' | head -c 32)" &
done
wait
```

Each request results in `req.route === null`, causing `toOpenApiPath()` to return the unique raw path, which is recorded as a new label set across all four instruments. After sufficient unique paths, the Node.js heap is exhausted and the process crashes with `FATAL ERROR: Reached heap limit Allocation failed - JavaScript heap out of memory`.
