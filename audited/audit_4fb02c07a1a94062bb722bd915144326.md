### Title
User-Supplied `evmAddress` Reflected Verbatim in GraphQL Error Response via `CustomExceptionResolver`

### Summary
When an unprivileged user submits a GraphQL query with an invalid `evmAddress` (e.g., non-hex characters), `decodeEvmAddress()` in `GraphQlUtils.java` throws an `IllegalArgumentException` whose message directly embeds the raw user-supplied string. `CustomExceptionResolver` catches this exception and forwards `ex.getMessage()` — unsanitized — into the GraphQL error response returned to the client, reflecting arbitrary attacker-controlled input back to the caller.

### Finding Description
**Exact code path:**

1. `AccountController.account()` receives the user-supplied `evmAddress` string and calls `entityService.getByEvmAddressAndType(evmAddress, EntityType.ACCOUNT)` with no input pre-validation. [1](#0-0) 

2. `EntityServiceImpl.getByEvmAddressAndType()` immediately calls `decodeEvmAddress(evmAddress)` with no try/catch, so any exception propagates upward. [2](#0-1) 

3. `GraphQlUtils.decodeEvmAddress()` strips the `0x` prefix and calls `Hex.decodeHex(evmAddress)`. On a `DecoderException`, it throws:
   ```java
   throw new IllegalArgumentException("Unable to decode evmAddress: " + evmAddress);
   ```
   The raw, unmodified user input is concatenated directly into the exception message. [3](#0-2) 

4. `CustomExceptionResolver.resolveToSingleError()` catches `IllegalArgumentException` and calls `.message(ex.getMessage())` — passing the exception message, which contains the raw user input, directly into the GraphQL error response with no sanitization or redaction. [4](#0-3) 

**Root cause:** The failed assumption is that `ex.getMessage()` is safe to return to clients. Because the exception message is constructed by concatenating user-controlled input, the resolver becomes a reflection vector.

### Impact Explanation
Any unauthenticated user can craft a GraphQL `account` query with an arbitrary string as `evmAddress`. The string — including any payload up to the length accepted by the GraphQL layer — is reflected verbatim in the JSON error response under the `"message"` field. While this does not expose server-side secrets, it:
- Confirms internal error message format and class names to attackers for reconnaissance.
- Enables reflected-content abuse (e.g., injecting misleading or confusing strings into error logs or monitoring systems that consume the GraphQL response).
- Matches the declared scope of "griefing with no economic damage."

### Likelihood Explanation
No authentication or special privilege is required. The GraphQL endpoint is publicly accessible. The trigger is trivially reproducible by any user who can send a GraphQL query — a single request with `evmAddress: "INVALID!!!"` is sufficient. The behavior is deterministic and repeatable.

### Recommendation
In `CustomExceptionResolver.resolveToSingleError()`, replace `ex.getMessage()` with a static, generic error message for `IllegalArgumentException` (e.g., `"Invalid input"`), or sanitize the message to strip any user-supplied content before including it in the response. Alternatively, in `GraphQlUtils.decodeEvmAddress()`, replace the concatenated message with a static string:
```java
throw new IllegalArgumentException("Invalid evmAddress: value contains non-hex characters");
```
This ensures no user-controlled data is ever embedded in exception messages that flow to the client.

### Proof of Concept
Send the following GraphQL query to the mirror node GraphQL endpoint (no authentication required):

```graphql
query {
  account(input: { evmAddress: "NOTVALIDHEX!!ATTACKER_CONTROLLED_STRING" }) {
    balance
  }
}
```

**Expected response:**
```json
{
  "errors": [
    {
      "message": "Unable to decode evmAddress: NOTVALIDHEX!!ATTACKER_CONTROLLED_STRING",
      "locations": [...],
      "path": ["account"],
      "extensions": { "classification": "ValidationError" }
    }
  ]
}
```

The attacker-controlled string `NOTVALIDHEX!!ATTACKER_CONTROLLED_STRING` is reflected verbatim in the `"message"` field of the error response, confirming unsanitized input reflection.
