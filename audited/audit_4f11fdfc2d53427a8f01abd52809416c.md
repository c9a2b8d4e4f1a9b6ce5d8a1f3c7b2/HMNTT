### Title
Unvalidated Loopback/Private Host Injection via Mirror REST Response in `fetchMirrorNodeNetwork()`

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` accepts `DomainName` values from the mirror REST API response without any validation against loopback addresses (`localhost`, `127.0.0.1`, `::1`) or private IP ranges. An attacker who controls the mirror REST server response — or who can perform a man-in-the-middle attack on the plaintext HTTP connection — can inject loopback entries that cause the Hiero SDK client to route all consensus traffic to the pinger's own loopback interface, creating a complete self-referential network partition outside design parameters.

### Finding Description

**Exact code location:** `pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 113–124.

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {
        continue
    }

    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))
    network[addr] = nodeAccountId
}
```

**Root cause:** The only guards are `host == ""` and `ep.Port == 0`. There is no check against loopback hostnames (`localhost`), loopback IPv4 (`127.0.0.0/8`), loopback IPv6 (`::1`), link-local, or RFC-1918 private ranges. Any non-empty string with a non-zero port is accepted verbatim and passed directly to `net.JoinHostPort`, then inserted into the network map handed to `hiero.ClientForNetwork()`.

**Failed assumption:** The code assumes the mirror REST server will only return valid, publicly-routable consensus node addresses. No such guarantee is enforced in code.

**HTTP transport amplifies the attack surface:** `pinger/config.go` line 37 shows the default `mirrorRest` value is `http://mirror-rest:5551` — a plaintext HTTP URL. The HTTP client constructed at line 46 of `mirror_node_client.go` performs no TLS enforcement:

```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
```

This means a network-adjacent attacker does not even need to compromise the mirror server — intercepting the plaintext HTTP response suffices.

**Exploit flow:**
1. Attacker controls the mirror REST server (or intercepts the HTTP response via MITM on the same container/pod network).
2. Returns a crafted `nodesEnvelope`:
   ```json
   {"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"domain_name":"localhost","port":50211}]}],"links":{"next":null}}
   ```
3. `fetchMirrorNodeNetwork()` passes all guards (`host = "localhost"`, `port = 50211`, both non-zero/non-empty).
4. `net.JoinHostPort("localhost", "50211")` → `"localhost:50211"` is inserted into the network map.
5. `hiero.ClientForNetwork(netmap)` in `sdk_client.go` line 22 creates an SDK client targeting the loopback interface.
6. Every consensus transaction the pinger submits is sent to `localhost:50211` — the pinger's own loopback — instead of real consensus nodes.

### Impact Explanation

The pinger holds a live operator private key (`cfg.operatorKey`) and submits signed transactions on a configurable interval. With the network map poisoned to loopback:

- All consensus transactions fail (connection refused or connect to an unrelated local listener).
- The pinger enters a permanent failure loop, reporting errors that mask real network health.
- If a local service happens to be listening on the injected port, signed transactions are delivered to it — a potential key-exposure or replay surface.
- The design intent of the `network=other` mode (monitoring a custom/private Hedera network) is completely subverted.

Severity: **High** — complete loss of pinger function and potential operator-key misuse.

### Likelihood Explanation

- **Mirror server compromise:** The mirror REST node is a less-privileged component (no operator keys). An attacker who compromises it gains influence over the more-privileged pinger. This is a classic privilege-escalation path across trust boundaries.
- **MITM without server compromise:** The default transport is plaintext HTTP (`http://mirror-rest:5551`). In Kubernetes or Docker Compose deployments, any container on the same overlay network can ARP-spoof or DNS-poison to intercept the HTTP call — no server compromise required.
- **Repeatability:** The pinger only calls `buildNetworkFromMirrorNodes()` once at startup (`sdk_client.go` line 18), so a single poisoned response permanently misconfigures the client for the lifetime of the process.

### Recommendation

1. **Reject loopback and private addresses** in `fetchMirrorNodeNetwork()` before inserting into the network map:
   ```go
   ip := net.ParseIP(host)
   if ip == nil {
       // resolve once to check
       addrs, _ := net.LookupHost(host)
       if len(addrs) > 0 { ip = net.ParseIP(addrs[0]) }
   }
   if ip != nil && (ip.IsLoopback() || ip.IsPrivate() || ip.IsLinkLocalUnicast()) {
       continue // reject
   }
   ```
2. **Enforce HTTPS** for the mirror REST URL and validate the server certificate; reject `http://` schemes in `loadConfig()`.
3. **Pin expected node account IDs** via configuration so injected entries for unknown account IDs are discarded.
4. **Validate port range** — consensus gRPC ports are well-known (50211/50212); reject ports outside an expected range.

python3 -c "
import http.server, json
class H(http.server.BaseHTTPRequestHandler):
    def do_GET(self):

            'service_endpoints':[{'domain_name':'localhost','ip_address_v4':'','port':50211}]}],
            'links':{'next':None}}).encode()
        self.send_response(200)
        self.send_header('Content-Type','application/json')
        self.end_headers()
        self.wfile.write(body)
