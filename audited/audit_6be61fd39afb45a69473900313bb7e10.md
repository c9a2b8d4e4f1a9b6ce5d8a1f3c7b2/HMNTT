### Title
Slow-Body DoS via Unconstrained `Http.ReadTimeout=0` on `/construction/metadata`

### Summary
The `Http` struct in `rosetta/app/config/types.go` places no minimum-value constraint on `ReadTimeout`. When this field is zero (either by explicit operator misconfiguration or by the embedded `application.yml` failing to supply a value), Go's `net/http.Server` treats it as "no timeout," allowing any unauthenticated remote client to hold a server goroutine open indefinitely by trickling the POST body to `/construction/metadata`. Repeated connections exhaust the goroutine pool and deny service to legitimate callers.

### Finding Description

**Code path**

`rosetta/app/config/types.go` lines 64–69 — the `Http` struct is a plain Go struct with no field-level validation:

```go
type Http struct {
    IdleTimeout       time.Duration `yaml:"idleTimeout"`
    ReadTimeout       time.Duration `yaml:"readTimeout"`       // zero is legal
    ReadHeaderTimeout time.Duration `yaml:"readHeaderTimeout"`
    WriteTimeout      time.Duration `yaml:"writeTimeout"`
}
```

`rosetta/app/config/config.go` lines 44–99 — `LoadConfig()` unmarshals YAML into this struct via `viper.Unmarshal` with no post-unmarshal validation step; a zero (or absent) `readTimeout` value is silently accepted.

`rosetta/main.go` lines 220–227 — the value is forwarded verbatim to `http.Server`:

```go
httpServer := &http.Server{
    ...
    ReadTimeout:       rosettaConfig.Http.ReadTimeout,   // 0 → no timeout
    ReadHeaderTimeout: rosettaConfig.Http.ReadHeaderTimeout,
    ...
}
```

**Root cause / failed assumption**

The design assumes operators will always supply a positive `readTimeout`. There is no `Validate()` call, no `max(value, minimumSafe)` clamp, and no startup assertion. Go's `net/http` documentation explicitly states that a zero `ReadTimeout` disables the deadline entirely.

**Exploit flow**

1. Attacker opens a TCP connection to port 5700.
2. Sends a well-formed HTTP/1.1 `POST /construction/metadata` with valid headers (Content-Type, Content-Length set to a large value) within the `ReadHeaderTimeout` window (default 3 s) — this satisfies the only active deadline.
3. After headers are accepted, the attacker sends one byte of the JSON body every ~60 s, keeping the connection alive indefinitely.
4. The server goroutine blocks in `net/http` body-read, waiting for the rest of the body before dispatching to `ConstructionMetadata`.
5. Repeating with hundreds of connections exhausts the goroutine pool; legitimate requests queue and time out.

**Why existing checks are insufficient**

- `ReadHeaderTimeout` (default 3 s) only covers the header phase; once headers are received the body-read phase has no deadline when `ReadTimeout = 0`.
- `WriteTimeout` is irrelevant — it starts after the response begins, which never happens.
- `IdleTimeout` applies only to keep-alive idle periods between requests, not to an in-progress body read.
- There is no per-IP connection rate limit or application-level `context.WithTimeout` wrapping the body read in the handler chain.
- The `/construction/metadata` endpoint requires no authentication, so any internet-reachable instance is exposed.

### Impact Explanation

A sustained slow-body attack prevents the Rosetta Construction API from returning fee metadata (`SuggestedFee`) and node account IDs. Clients building transactions cannot obtain the required metadata, effectively halting transaction construction workflows. Because the Rosetta server is single-process with a shared goroutine pool, full exhaustion also blocks all other endpoints (block, account, network) on the same port.

### Likelihood Explanation

The attack requires no credentials, no special knowledge beyond the public Rosetta API spec, and is executable with standard tools (`curl --limit-rate`, `slowhttptest`). The only precondition is that `readTimeout` is zero — which occurs if an operator omits the field, sets it to `0` explicitly, or if the embedded `application.yml` default is absent. The documentation lists a 5 s default, but the code itself provides no enforcement; a single misconfigured deployment is fully exploitable by any external party.

### Recommendation

1. **Add config validation** in `LoadConfig()` (or a dedicated `Validate()` method on `Config`) that rejects or clamps `Http.ReadTimeout` to a minimum safe value (e.g., 5 s):
   ```go
   if cfg.Http.ReadTimeout <= 0 {
       return nil, errors.New("http.readTimeout must be > 0")
   }
   ```
2. **Enforce a hard floor at server construction** in `main.go` as a defence-in-depth measure.
3. **Add an application-level body size limit** using `http.MaxBytesReader` in middleware so that oversized or trickled bodies are rejected early regardless of timeout configuration.
4. **Document the security implication** of setting `readTimeout` to zero in the configuration reference.

// ConstructionMetadata implements the /construction/metadata endpoint.
func (c *constructionAPIService) ConstructionMetadata(
	ctx context.Context,
	request *rTypes.ConstructionMetadataRequest,
) (*rTypes.ConstructionMetadataResponse, *rTypes.Error) {
	options := request.Options
	if options == nil || options[optionKeyOperationType] == nil {
		return nil, errors.ErrInvalidOptions
	}

	operationType, ok := options[optionKeyOperationType].(string)
	if !ok {
		return nil, errors.ErrInvalidOptions
	}

	maxFee, rErr := c.transactionHandler.GetDefaultMaxTransactionFee(operationType)
	if rErr != nil {
		return nil, rErr
	}

	metadata := maps.Clone(request.Options)
	delete(metadata, optionKeyAccountAliases)

	if resolved, rErr := c.resolveAccountAliases(ctx, options); rErr != nil {
		return nil, rErr
	} else if resolved != "" {
		metadata[metadataKeyAccountMap] = resolved
	}

	// node account id
	nodeAccountId, rErr := c.getRandomNodeAccountId()
	if rErr != nil {
		return nil, rErr
	}
	metadata[metadataKeyNodeAccountId] = nodeAccountId.String()

	// pass the value as a string to avoid being deserialized into data types like float64
	validUntilNanos := time.Now().UnixNano() + maxValidDurationNanos
	metadata[metadataKeyValidUntilNanos] = fmt.Sprintf("%d", validUntilNanos)

	return &rTypes.ConstructionMetadataResponse{
		Metadata:     metadata,
		SuggestedFee: []*rTypes.Amount{maxFee.ToRosetta()},
	}, nil
}
