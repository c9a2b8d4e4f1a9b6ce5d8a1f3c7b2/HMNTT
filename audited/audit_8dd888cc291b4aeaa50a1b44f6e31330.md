### Title
Unauthenticated Full Historical Scan via Unbounded Timestamp Range on Hook Storage Change Endpoint

### Summary
An unprivileged external user can supply `timestamp=gte:0&timestamp=lte:9223372036854775807` to `GET /api/v1/accounts/{ownerId}/hooks/{hookId}/storage`, forcing the server into the expensive historical query path against `hook_storage_change` with no time restriction. The `DISTINCT ON (key)` SQL query must scan every historical record for the given owner/hook before applying the output limit, enabling repeated full-table scans with no authentication required.

### Finding Description

**Exact code path:**

Because `primarySortField = false`, this check is **never executed** for timestamp bounds. With `timestamp=gte:0`, `lower` is set to the `gte:0` parameter; with `timestamp=lte:9223372036854775807`, `upper` is set to the `lte:MAX_VALUE` parameter. `Bound.isEmpty()` (`Bound.java:126-128`) returns `false` because both `lower` and `upper` are non-null.

`HookStorageRequest.isHistorical()` (`HookStorageRequest.java:66-68`) returns `!timestamp.isEmpty()` → `true`, routing execution to `HookServiceImpl.getHookStorageChange()` (`HookServiceImpl.java:84-118`):

```java

final long timestampUpperBound = timestamp.adjustUpperBound();            // → Long.MAX_VALUE
```

`getAdjustedLowerRangeValue()` (`Bound.java:86-97`) returns `0` for `gte:0`. `adjustUpperBound()` (`Bound.java:63-74`) returns `Long.MAX_VALUE` for `lte:9223372036854775807`.

These values are passed directly to the repository query (`HookStorageChangeRepository.java:15-39`):

```sql
select distinct on (key) ...
from hook_storage_change
where owner_id = :ownerId
  and hook_id = :hookId
  and key >= :keyLowerBound
  and key <= :keyUpperBound
  and consensus_timestamp between :timestampLowerBound and :timestampUpperBound
```

With `timestampLowerBound=0` and `timestampUpperBound=9223372036854775807`, this is a **full historical scan** of `hook_storage_change` for the given owner/hook. PostgreSQL's `DISTINCT ON (key)` with `ORDER BY key ASC, consensus_timestamp DESC` must process and sort **all** matching rows before the `LIMIT` clause is applied. The output limit (`MAX_LIMIT = 100`) constrains returned rows, not the scan scope.

**Why existing checks fail:**

- `@Size(max = 2)` on `timestamps` (`HooksController.java:110`) only limits the count of timestamp parameters to 2, not their values.
- `@Max(MAX_LIMIT)` on `limit` (`HooksController.java:112`) caps output rows at 100 but does not restrict the number of rows the database must scan.
- No authentication or authorization annotations (`@PreAuthorize`, `@Secured`, etc.) are present on the controller or endpoint.
- No minimum timestamp window (e.g., max 24-hour range) is enforced anywhere in the call chain.

### Impact Explanation

Any unauthenticated caller who knows a valid `ownerId` and `hookId` (both are public blockchain data) can force a full sequential scan of the `hook_storage_change` table partition for that owner/hook. On a busy network with many historical hook storage changes, this scan is O(N) in the number of historical records. Repeated concurrent requests constitute a denial-of-service against the database, degrading availability for all users. Additionally, the response discloses the complete gossip-prevention storage history (all `key`/`value_written` pairs across all time) for the targeted hook, which is the sensitive data the historical table is meant to expose only under controlled conditions.

### Likelihood Explanation

The attack requires no credentials, no special knowledge beyond a valid `ownerId`/`hookId` (discoverable via the public `GET /api/v1/accounts/{ownerId}/hooks` endpoint), and no complex tooling — a single `curl` command suffices. It is trivially repeatable and scriptable. The timestamp values `gte:0` and `lte:9223372036854775807` are the natural extremes of a `long`, making them an obvious first probe. Any attacker performing basic API fuzzing would discover this within minutes.

### Recommendation

1. **Enforce a maximum timestamp window**: In `hookStorageChangeRequest()` or `HookServiceImpl.getHookStorageChange()`, reject requests where `timestampUpperBound - timestampLowerBound` exceeds a configured maximum (e.g., 7 days in nanoseconds). Throw `IllegalArgumentException` if the window is too large.
2. **Require at least one bound**: Reject requests where the timestamp `Bound` has no lower bound (i.e., `lower == null`), forcing callers to anchor queries to a specific time range rather than scanning from epoch 0.
3. **Enable range validation for timestamp bounds**: Pass `primarySortField = true` in `Bound.of()`, or add an explicit `adjustedLower <= adjustedUpper` check that also validates the window size.
4. **Add authentication/authorization**: Restrict the historical storage endpoint to authenticated operators or the hook owner.

    private HookStorageRequest hookStorageChangeRequest(
            EntityIdParameter ownerId,
            long hookId,
            List<SlotRangeParameter> keys,
            TimestampParameter[] timestamps,
            int limit,
            Direction order) {
        final var keyFilters = new ArrayList<byte[]>();

        var lowerBound = MIN_KEY_BYTES;
        var upperBound = MAX_KEY_BYTES;

        for (final var key : keys) {
            final byte[] value = key.value();

            if (key.hasLowerBound()) {
                if (key.operator() == RangeOperator.EQ) {
                    keyFilters.add(value);
                } else if (Arrays.compareUnsigned(value, lowerBound) > 0) {
                    lowerBound = value;
                }
            } else if (key.hasUpperBound()) {
                if (Arrays.compareUnsigned(value, upperBound) < 0) {
                    upperBound = value;
                }
            }
        }

        final var bound = Bound.of(timestamps, TIMESTAMP, HookStorageChange.HOOK_STORAGE_CHANGE.CONSENSUS_TIMESTAMP);

        return HookStorageRequest.builder()
                .hookId(hookId)
                .keys(keyFilters)
                .limit(limit)
                .keyLowerBound(lowerBound)
                .keyUpperBound(upperBound)
                .order(order)
                .ownerId(ownerId)
                .timestamp(bound)
                .build();
    }

    public long adjustUpperBound() {
        if (this.upper == null) {
            return Long.MAX_VALUE;
        }

        long upperBound = this.upper.value();
        if (this.upper.operator() == RangeOperator.LT) {
            upperBound--;
        }

        return upperBound;
    }

    public RangeParameter<Long> adjustLowerRange() {
        if (this.hasEqualBounds()) {
            // If the primary param has a range with a single value, rewrite it to EQ
            lower = new NumberRangeParameter(EQ, this.getAdjustedLowerRangeValue());
            upper = null;
        }

        return lower;
    }

    public long getAdjustedLowerRangeValue() {
        if (this.lower == null) {
            return 0;
        }

        long lowerBound = this.lower.value();
        if (this.lower.operator() == RangeOperator.GT) {
            lowerBound++;
        }

        return lowerBound;
    }

    private HookStorageResult getHookStorageChange(HookStorageRequest request) {
        final var page = request.getPageRequest();
        final var ownerId = entityService.lookup(request.getOwnerId());
        final long hookId = request.getHookId();

        final var keys = request.getKeys();
        final boolean requestHasKeys = !keys.isEmpty();
        final var keysInRange = request.getKeysInRange();

        if (keysInRange.isEmpty() && requestHasKeys) {
            return new HookStorageResult(ownerId, List.of());
        }

        final var timestamp = request.getTimestamp();
        final long timestampLowerBound = timestamp.getAdjustedLowerRangeValue();
        final long timestampUpperBound = timestamp.adjustUpperBound();

        List<HookStorage> changes;

        if (requestHasKeys) {
            changes = hookStorageChangeRepository.findByKeyInAndTimestampBetween(
                    ownerId.getId(), hookId, keys, timestampLowerBound, timestampUpperBound, page);
        } else {
            changes = hookStorageChangeRepository.findByKeyBetweenAndTimestampBetween(
                    ownerId.getId(),
                    hookId,
                    request.getKeyLowerBound(),
                    request.getKeyUpperBound(),
                    timestampLowerBound,
                    timestampUpperBound,
                    page);
        }

        return new HookStorageResult(ownerId, changes);
    }
