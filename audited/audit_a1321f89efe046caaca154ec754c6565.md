### Title
Global Opcode Rate Limit Monopolization via Single-Client Starvation (No Per-Client Isolation)

### Summary
`ThrottleManagerImpl.throttleOpcodeRequest()` enforces a single application-wide token bucket with a default capacity of 1 token/second for the `/contracts/results/{transactionIdOrHash}/opcodes` endpoint. Because the bucket is a global Spring singleton with no per-IP or per-client partitioning, any single unprivileged user sending exactly one request per second permanently exhausts the entire global capacity, causing all other users to receive HTTP 429 for the duration of the attack.

### Finding Description
**Code path:**

- `ThrottleProperties.java` line 29: `private long opcodeRequestsPerSecond = 1;` — default capacity is 1 token/second.
- `ThrottleConfiguration.java` lines 47–55: a single `opcodeRateLimitBucket` Spring bean is created with `capacity(rateLimit)` and `refillGreedy(rateLimit, Duration.ofSeconds(1))` where `rateLimit = 1`.
- `ThrottleManagerImpl.java` lines 52–56: every incoming opcode request calls `opcodeRateLimitBucket.tryConsume(1)` against this single shared bucket.

```java
// ThrottleManagerImpl.java lines 52-56
public void throttleOpcodeRequest() {
    if (!opcodeRateLimitBucket.tryConsume(1)) {
        throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
    }
}
```

**Root cause:** The `opcodeRateLimitBucket` is a singleton bean shared across all callers. There is no per-IP, per-session, or per-user bucket. The bucket has a maximum capacity of 1 token, which refills at 1 token/second. A single attacker consuming that 1 token/second leaves zero tokens for any other caller.

**Failed assumption:** The design assumes the 1 RPS global limit is sufficient to protect the server from abuse, but does not account for a single client monopolizing the entire allowance, starving all other legitimate users.

**Why existing checks fail:** The only check is `opcodeRateLimitBucket.tryConsume(1)` — a global non-partitioned test. There is no IP-based rate limiting at the application layer. The Helm chart (`charts/hedera-mirror-web3/values.yaml` lines 79–81) shows the opcodes route is commented out, meaning no gateway-level per-client protection is active by default.

### Impact Explanation
When `hiero.mirror.web3.opcode.tracer.enabled=true`, a single attacker sending 1 GET request per second to `/api/v1/contracts/results/{hash}/opcodes` with `Accept-Encoding: gzip` permanently denies all other users access to the opcode debugging endpoint. Every other caller receives HTTP 429 for the entire duration of the attack. The attacker requires no credentials, no special network position, and no elevated privileges.

### Likelihood Explanation
The attack requires only a standard HTTP client capable of sending 1 request per second — trivially achievable with `curl`, `wget`, or any scripting language. The attacker needs no authentication, no knowledge of valid transaction hashes (any syntactically valid hash triggers the throttle check before the service layer), and no special network access. The attack is indefinitely sustainable at negligible cost and is not detectable as anomalous traffic (1 req/sec is indistinguishable from normal usage).

### Recommendation
Replace the single global bucket with per-client buckets keyed by client IP (extracted from `X-Forwarded-For` or `RemoteAddr`). For example, use a `ConcurrentHashMap<String, Bucket>` or a Caffeine-backed cache with TTL eviction, creating one bucket per IP with the same 1 RPS limit. This preserves the server-protection intent while preventing any single client from monopolizing the global allowance. Additionally, enforce a separate global cap to bound total server load.

### Proof of Concept
**Precondition:** `hiero.mirror.web3.opcode.tracer.enabled=true` (operator must enable).

**Attacker terminal (runs indefinitely):**
```bash
while true; do
  curl -s -o /dev/null -w "%{http_code}\n" \
    -H "Accept-Encoding: gzip" \
    "https://<mirror-node>/api/v1/contracts/results/0xaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa/opcodes"
  sleep 1
done
```

**Victim terminal (any other user, same second):**
```bash
curl -H "Accept-Encoding: gzip" \
  "https://<mirror-node>/api/v1/contracts/results/<valid-tx-hash>/opcodes"
# Returns: HTTP 429 {"_status":{"messages":[{"message":"Requests per second rate limit exceeded"}]}}
```
