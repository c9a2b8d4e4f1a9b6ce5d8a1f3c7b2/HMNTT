### Title
Unbounded `X-Real-IP` Header Logged Without Truncation Enables Log Disk Exhaustion (DoS)

### Summary
The `getClientIpAddress()` function in `rosetta/app/middleware/trace.go` reads the `X-Real-IP` (or `X-Forwarded-For`) header value verbatim and returns it without any length validation or truncation. `TracingMiddleware` then embeds this raw string directly into a log message emitted via `log.Info()` for every non-internal request. An unauthenticated attacker can repeatedly send requests with a near-maximum-size header value (~1 MB, Go's default `MaxHeaderBytes`), causing unbounded log growth and eventual disk exhaustion.

### Finding Description
**Exact code path:**

In `rosetta/app/middleware/trace.go`, `getClientIpAddress()` (lines 63–74):
```go
func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)          // raw, unbounded read
    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader) // same issue
    }
    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }
    return ipAddress  // no length cap, no validation
}
```

The returned value is immediately embedded in a log line (lines 52–59):
```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
log.Info(message)   // written to stdout on every non-internal request
```

**Root cause:** No maximum-length enforcement on the header value before it is written to the log. The `http.Server` in `rosetta/main.go` (lines 220–227) sets `IdleTimeout`, `ReadHeaderTimeout`, `ReadTimeout`, and `WriteTimeout`, but does **not** set `MaxHeaderBytes`, so Go's default of `1 << 20` (1 MB) applies per request. Each request can therefore inject up to ~1 MB into the log stream.

**Why existing checks fail:** The test suite (`trace_test.go`, lines 52–58) only exercises normal short IP strings. There is no server-side or middleware-level cap on the header length before logging.

### Impact Explanation
Log output is directed to `os.Stdout` (`rosetta/main.go` line 54). In typical container/Kubernetes deployments, stdout is captured by a logging driver and written to disk or a log aggregator with finite storage. An attacker sending a sustained flood of requests—each carrying a ~1 MB `X-Real-IP` header—can fill the log volume, causing:
- Log-driver or container crashes due to out-of-disk-space errors.
- Potential host-level disk exhaustion if logs are stored on the root filesystem.
- Loss of legitimate log data, impairing incident response.

### Likelihood Explanation
The Rosetta API is a public-facing HTTP service with no authentication on any endpoint. Any network-reachable client can craft and send the malicious header. The attack requires no credentials, no special knowledge of the application, and is trivially repeatable with standard HTTP tooling (`curl`, `wrk`, etc.). Rate limiting is not present in the middleware stack shown.

### Recommendation
1. **Truncate the header value** before use: cap `ipAddress` to a safe maximum (e.g., 45 characters for the longest valid IPv6 address) immediately inside `getClientIpAddress()`.
2. **Validate the value** with `net.ParseIP()` or a regex; log a placeholder (e.g., `"invalid"`) if it does not parse as a valid IP.
3. **Set `MaxHeaderBytes`** on the `http.Server` in `rosetta/main.go` to an explicit, conservative value (e.g., `8192`) to limit total header size at the transport layer.

Example fix for `getClientIpAddress()`:
```go
const maxIPLength = 45 // max IPv6 length

func getClientIpAddress(r *http.Request) string {
    ipAddress := r.Header.Get(xRealIpHeader)
    if len(ipAddress) == 0 {
        ipAddress = r.Header.Get(xForwardedForHeader)
    }
    if len(ipAddress) == 0 {
        ipAddress, _, _ = net.SplitHostPort(r.RemoteAddr)
    }
    if len(ipAddress) > maxIPLength {
        ipAddress = ipAddress[:maxIPLength]
    }
    if net.ParseIP(ipAddress) == nil {
        ipAddress = "invalid"
    }
    return ipAddress
}
```

### Proof of Concept
```bash
# Generate a ~900 KB X-Real-IP value (comma-separated fake IPs)
LARGE_IP=$(python3 -c "print(','.join(['1.2.3.4']*100000))")

# Send a flood of requests; each causes ~900 KB to be written to the log
for i in $(seq 1 500); do
  curl -s -o /dev/null \
    -H "X-Real-IP: $LARGE_IP" \
    -X POST http://<rosetta-host>:<port>/network/list \
    -H "Content-Type: application/json" \
    -d '{"metadata":{}}' &
done
wait
# Result: ~450 MB written to the log in a single burst; repeat to exhaust disk.
```
