### Title
Unauthenticated `/network/status` Endpoint Triggers Multiple Unbounded DB Queries Per Request, Enabling Resource Exhaustion

### Summary
The `NetworkStatus()` handler in `rosetta/app/services/network_service.go` executes 2–3 database queries on every invocation with no application-level rate limiting or authentication. Any unprivileged external user can flood this endpoint to exhaust database connection pool resources, degrading or denying service to the mirror node's database backend.

### Finding Description

**Exact code path:**

`NetworkStatus()` at [1](#0-0)  sequentially calls three repository methods:

1. `n.RetrieveGenesis(ctx)` → `blockRepository.RetrieveGenesis()` → `initGenesisRecordFile()` which uses a `sync.Once` [2](#0-1)  — this DB query fires only once per process lifetime and is then cached. After initialization, `RetrieveGenesis` costs **0 DB queries**.

2. `n.RetrieveLatest(ctx)` → executes `selectLatestWithIndex` raw SQL against `record_file` on **every request** with no caching. [3](#0-2) 

3. `n.addressBookEntryRepo.Entries(ctx)` → executes `latestNodeServiceEndpoints` SQL against `address_book_entry` on **every request**, iterating over up to two file IDs (101 and 102), meaning **1–2 DB queries per call**. [4](#0-3) 

**Net DB queries per request (steady state): 2–3, with no caching, no throttle, no auth.**

**Root cause — failed assumption:** The application assumes an external infrastructure layer (Traefik ingress) will enforce rate limiting. The Helm chart defines a `rateLimit` middleware: [5](#0-4)  However, this middleware is only rendered when **both** `global.middleware` and `.Values.middleware` are truthy: [6](#0-5)  This is an opt-in, infrastructure-dependent control. The Go application itself — `main.go` — applies only `MetricsMiddleware`, `TracingMiddleware`, and `CorsMiddleware` with zero rate limiting: [7](#0-6) 

**No authentication is required.** The endpoint accepts any POST with a valid `NetworkRequest` body.

### Impact Explanation
An attacker sending a sustained flood of POST requests to `/network/status` forces 2–3 DB queries per request against PostgreSQL. At sufficient request rate, this exhausts the database connection pool, causes query queuing/timeouts, and makes the mirror node DB unavailable to all other components (importer, REST API, gRPC). This constitutes a network partition from the mirror node's perspective — it can no longer read chain state — matching the described impact. Severity is **High** (availability impact, no confidentiality/integrity impact, no privileges required).

### Likelihood Explanation
The attack requires only network access to the rosetta port and knowledge of the Rosetta API spec (publicly documented). No credentials, tokens, or special headers are needed. The attack is trivially repeatable with standard tools (`curl`, `wrk`, `k6` — the project itself ships a k6 load test for this exact endpoint at `tools/k6/src/rosetta/test/networkStatus.js`). [8](#0-7)  Deployments without Traefik ingress (bare-metal, direct port exposure, non-Helm deployments) have zero rate limiting. Even with Traefik, the `rateLimit` is keyed on `requestHost` (not source IP), which an attacker can trivially satisfy. Likelihood is **High**.

### Recommendation
1. **Application-level rate limiting**: Add a per-IP token-bucket middleware in `rosetta/main.go` (e.g., `golang.org/x/time/rate`) wrapping the router before `MetricsMiddleware`, so rate limiting is enforced regardless of deployment topology.
2. **Cache `RetrieveLatest` result**: Introduce a short TTL cache (e.g., 1–2 seconds) for the latest block result in `blockRepository`, similar to the existing `sync.Once` pattern used for genesis. This reduces DB amplification even under legitimate load.
3. **Cache `Entries` result**: The address book changes infrequently; cache `Entries()` with a TTL of minutes.
4. **Do not rely solely on optional infrastructure controls**: The Helm middleware is opt-in and absent in non-Kubernetes deployments. Defense-in-depth requires the application to protect itself.

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}

func (aber *addressBookEntryRepository) Entries(ctx context.Context) (*types.AddressBookEntries, *rTypes.Error) {
	db, cancel := aber.dbClient.GetDbWithContext(ctx)
	defer cancel()

	nodes := make([]nodeServiceEndpoint, 0)
	// address book file 101 has service endpoints for nodes, resort to file 102 if 101 doesn't exist
	for _, fileId := range []int64{aber.addressBook101.EncodedId, aber.addressBook102.EncodedId} {
		if err := db.Raw(
			latestNodeServiceEndpoints,
			sql.Named("file_id", fileId),
		).Scan(&nodes).Error; err != nil {
			log.Error("Failed to get latest node service endpoints", err)
			return nil, errors.ErrDatabaseError
		}

		if len(nodes) != 0 {
			break
		}
	}

	entries := make([]types.AddressBookEntry, 0, len(nodes))
	for _, node := range nodes {
		entries = append(entries, node.toAddressBookEntry())
	}

	return &types.AddressBookEntries{Entries: entries}, nil
}
