### Title
Unbounded Metric Label Cardinality via Raw Request Path Injection in `metricsMiddleware`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath` function falls back to the raw user-controlled `req.path` when no Express route is matched (`req.route` is `null`). Every unique path an attacker sends to a non-existent endpoint creates a new label tuple `{method, path, code}` in the OpenTelemetry SDK's in-memory store for `durationHistogram`, `requestTotalCounter`, `requestSizeHistogram`, and `responseSizeHistogram`. Because there is no cap on unique label combinations and no rate-limiting in the REST middleware layer, an unprivileged attacker can drive unbounded memory growth and OOM-kill the process.

### Finding Description

**Exact code path:**

`rest/middleware/metricsHandler.js`, `toOpenApiPath` (lines 13–31):
```js
if (!path) {
  if (!req.route) {
    path = req.path;          // ← raw attacker-controlled value
  } else {
    path = (req.baseUrl ?? '') + req.route?.path;   // ← safe template
  }
}
```

`metricsMiddleware` `res.on('finish', ...)` handler (lines 185–211):
```js
const path = toOpenApiPath(req, res);
const code = String(res.statusCode);
const method = req.method;
const labels = {method, path, code};

durationHistogram.record(duration, labels);      // new time-series per unique tuple
requestTotalCounter.add(1, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(..., labels);
```

**Root cause:** The design assumption is that `req.route` will always be set for instrumented requests, so `req.route.path` (a fixed template string like `/api/v1/transactions/:id`) bounds the `path` label. This assumption fails for any request that does not match a registered Express route (resulting in a 404). For those requests `req.route` is `null`, so `path = req.path` — the literal URL path string supplied by the client — is used verbatim as a metric label.

**Why existing checks fail:**
- `toOpenApiPath` does apply a `:param → {param}` substitution (line 24), but this only helps when `req.route` is set. For unmatched routes the substitution has nothing to act on.
- `res.locals[requestPathLabel]` is only populated by `recordRequestPath` in `rest/routes/index.js` (line 19) and by specific controller workarounds (e.g., `contractController.js` line 791). It is never set for 404 paths.
- The REST service has no application-level rate limiting (the throttle machinery found in `web3/` is entirely separate and does not apply here).
- The OpenTelemetry JS SDK (`@opentelemetry/sdk-metrics`) has no built-in cardinality cap by default; every unique attribute set allocates a new accumulator with per-bucket counters.

### Impact Explanation
Each unique `{method, path, code}` tuple causes the OTel SDK to allocate a new histogram accumulator (one counter per bucket boundary plus sum/count). With the default 4-bucket configuration (`[25, 100, 250, 500]`) each accumulator is small, but with millions of unique paths the aggregate heap grows without bound. When the Node.js heap limit is reached the process is OOM-killed. Because the mirror-node REST service is the public query interface for Hedera transaction data, killing it prevents clients from querying transaction gossip state — directly matching the stated scope of "preventing gossip of a transaction or multiple transactions."

### Likelihood Explanation
The attack requires no authentication, no special headers, and no knowledge of the application internals. A single attacker with a modest script can generate millions of unique paths (e.g., `/api/v1/xxxxxxxx` with a random suffix) at high throughput. The REST endpoint is publicly reachable by design. The attack is repeatable across process restarts because the SDK state is re-initialized fresh each time, meaning the attacker can re-trigger OOM after each restart. No existing network-layer rate limiting is configured by default in the REST chart values.

### Recommendation
1. **Normalize unknown paths to a sentinel value.** In `toOpenApiPath`, replace the `path = req.path` fallback with a fixed string such as `"unknown"` or `"/api/v1/unknown"`:
   ```js
   if (!req.route) {
     path = `${apiPrefix}/unknown`;   // bounded sentinel
   }
   ```
2. **Enable OTel cardinality overflow protection.** Configure `@opentelemetry/sdk-metrics` with a `cardinalityLimit` on the view for per-route instruments (available since SDK v1.9).
3. **Add ingress-level rate limiting** to the REST service (e.g., via the Traefik `rateLimit` middleware already used for the Rosetta service) to slow the rate at which unique paths can be injected.

### Proof of Concept
```bash
# Requires: curl, bash, a running mirror-node REST instance at $HOST
HOST=http://localhost:5551
for i in $(seq 1 500000); do
  curl -s -o /dev/null "$HOST/api/v1/$(cat /dev/urandom | tr -dc 'a-z0-9' | head -c 16)" &
  # throttle to ~500 concurrent connections
  [ $((i % 500)) -eq 0 ] && wait
done
# Monitor: watch -n1 'curl -s http://localhost:5551/swagger/metrics/ | grep api_request_duration'
# Expected: label set grows without bound; Node.js heap exhausted → process killed with ENOMEM
```
