### Title
Malicious Block Node Can Permanently Evade Selection via Crafted `ServerStatus` Response Without Triggering Circuit Breaker

### Summary
In `BlockNode.getBlockRange()`, a block node returning `firstAvailableBlock > lastAvailableBlock` causes `Range.closed()` to throw `IllegalArgumentException`, which is silently caught and returns `EMPTY_BLOCK_RANGE`. Critically, unlike `streamBlocks()`, this code path never calls `onError()`, so the malicious node is never marked inactive and never enters the readmit cooldown — it is simply skipped on every selection cycle indefinitely, causing permanent block ingestion failure if it is the sole or last available node.

### Finding Description

**Exact code location:** `importer/src/main/java/org/hiero/mirror/importer/downloader/block/BlockNode.java`, `getBlockRange()`, lines 99–112.

```java
public Range<Long> getBlockRange() {
    try {
        ...
        final long firstBlockNumber = response.getFirstAvailableBlock();
        return firstBlockNumber != -1
                ? Range.closed(firstBlockNumber, response.getLastAvailableBlock())  // line 106
                : EMPTY_BLOCK_RANGE;
    } catch (Exception ex) {                          // line 108 — catches IllegalArgumentException
        log.error("Failed to get server status for {}", this, ex);
        return EMPTY_BLOCK_RANGE;                     // line 110
    }
}
``` [1](#0-0) 

**Root cause:** Guava's `Range.closed(lower, upper)` requires `lower <= upper`; it throws `IllegalArgumentException` otherwise. The broad `catch (Exception ex)` swallows this without distinguishing a malformed response from a transient network error.

**Contrast with `streamBlocks()`**, which calls `onError()` on every exception, potentially marking the node inactive: [2](#0-1) 

`getBlockRange()` has no equivalent `onError()` call. The node's `active` flag and `errors` counter are never touched.

**Selection path in `BlockNodeSubscriber`:** `getNode()` calls `hasBlock()`, which calls `node.getBlockRange()`. An empty range causes `hasBlock()` to return `false`, and the node is skipped: [3](#0-2) 

If no node passes `hasBlock()`, `getNode()` throws `BlockStreamException("No block node can provide block X")`: [4](#0-3) 

### Impact Explanation

If an attacker controls all configured block nodes (or the single configured node), they can halt block ingestion permanently. The mirror node will repeatedly query each node's status, receive `EMPTY_BLOCK_RANGE` every time, and throw `BlockStreamException` on every `doGet()` call. Because `onError()` is never invoked from `getBlockRange()`, the circuit breaker never fires, and the node is never marked inactive — the system loops indefinitely without self-healing. This is a complete denial of block data ingestion.

### Likelihood Explanation

Block node operators are semi-trusted external parties from the mirror node's perspective. A malicious or compromised block node operator requires no special privilege within the mirror node itself — they only need to control the gRPC endpoint the mirror node is configured to contact. The attack is trivially repeatable: the malicious `ServerStatus` response (e.g., `firstAvailableBlock=100`, `lastAvailableBlock=50`) can be returned on every status poll with zero additional effort.

### Recommendation

1. **Validate the response before calling `Range.closed()`:** Check `firstBlockNumber <= lastAvailableBlock` explicitly and treat a violation as a node error (call `onError()`), not a silent empty range.
2. **Call `onError()` on unexpected exceptions in `getBlockRange()`** so the circuit breaker applies uniformly to all failure modes, including malformed responses.
3. **Distinguish malformed responses from transient errors** in logging and metrics so operators can detect this attack pattern.

```java
public Range<Long> getBlockRange() {
    try {
        ...
        final long first = response.getFirstAvailableBlock();
        final long last  = response.getLastAvailableBlock();
        if (first == -1) return EMPTY_BLOCK_RANGE;
        if (first > last) {
            log.warn("Malformed ServerStatus from {}: first={} > last={}", this, first, last);
            onError();          // trigger circuit breaker
            return EMPTY_BLOCK_RANGE;
        }
        return Range.closed(first, last);
    } catch (Exception ex) {
        onError();              // also penalize on unexpected exceptions
        log.error("Failed to get server status for {}", this, ex);
        return EMPTY_BLOCK_RANGE;
    }
}
```

### Proof of Concept

1. Stand up a gRPC server implementing `BlockNodeService` at the endpoint configured in the mirror node.
2. Implement `serverStatus()` to always return `ServerStatusResponse` with `firstAvailableBlock=1000` and `lastAvailableBlock=1`.
3. Configure the mirror node to use only this block node.
4. Start the mirror node and observe it attempting to ingest blocks.
5. **Expected result:** Every call to `getBlockRange()` throws `IllegalArgumentException` inside `Range.closed(1000, 1)`, is caught, returns `EMPTY_BLOCK_RANGE`, `hasBlock()` returns `false`, `getNode()` throws `BlockStreamException`, and block ingestion halts permanently. The node is never marked inactive; the error counter never increments; the readmit cooldown never triggers.
