### Title
Liveness Heartbeat Goroutine Decoupled from Transaction Submission Enables Silent On-Chain History Gaps

### Summary
The heartbeat goroutine in `pinger/main.go` writes to `/tmp/alive` every 15 seconds independently of the transaction submission loop. If the transaction loop goroutine blocks indefinitely (e.g., the Hiero SDK `Execute` call hangs without a deadline), the heartbeat continues firing, the liveness probe passes, and the orchestrator never restarts the pod — while zero transactions are submitted to Hashgraph. An attacker who can also write to `/tmp/alive` (shared `/tmp` volume, same-UID sidecar, or container escape) can sustain this silent gap even after the pinger process is killed.

### Finding Description
**Exact code path:**

`pinger/main.go` lines 28–39 — the heartbeat goroutine:
```go
go func() {
    t := time.NewTicker(15 * time.Second)
    defer t.Stop()
    for {
        select {
        case <-ctx.Done():
            return
        case <-t.C:
            _ = os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644)
        }
    }
}()
``` [1](#0-0) 

`pinger/main.go` lines 57–70 — the transaction loop runs in the **main goroutine**, completely separate from the heartbeat goroutine:
```go
for {
    select {
    case <-ctx.Done(): ...
    case <-ticker.C:
        if err := submitWithRetry(ctx, client, cfg); err != nil { ... }
    }
}
``` [2](#0-1) 

`pinger/transfer.go` line 33 — the SDK call has no explicit per-call deadline:
```go
resp, err := cryptoTransfer.Execute(client)
``` [3](#0-2) 

`pinger/cmd/healthcheck/main.go` lines 27–38 — the liveness check only inspects the file's modification time, with no cryptographic proof or process-identity check:
```go
fi, err := os.Stat("/tmp/alive")
...
age := time.Since(fi.ModTime())
if age > 2*time.Minute {
    fmt.Fprintln(os.Stderr, "heartbeat stale")
    os.Exit(1)
}
``` [4](#0-3) 

**Root cause / failed assumption:** The design assumes that a recently-modified `/tmp/alive` implies the transaction loop is healthy. It does not. The heartbeat goroutine is a completely independent goroutine that shares only the `ctx` cancellation signal with the transaction loop. As long as the process is alive and no OS signal is received, the heartbeat fires every 15 s regardless of whether `submitWithRetry` is making progress.

**Exploit flow (no external attacker needed — architectural path):**
1. A network partition, slow consensus node, or SDK-internal deadlock causes `cryptoTransfer.Execute(client)` to block indefinitely (no per-call context deadline is set).
2. The main goroutine is stuck inside `submitWithRetry`; no new transactions are submitted.
3. The heartbeat goroutine continues to write to `/tmp/alive` every 15 s.
4. `healthcheck live` sees a fresh file, exits 0 — liveness probe passes.
5. The orchestrator never restarts the pod. The on-chain gap grows silently.

**External-write amplification path (requires write access to `/tmp/alive`):**
The file is created with permissions `0644`. [5](#0-4) 
The container runs as `USER 1000:1000` from a `scratch` base. [6](#0-5) 
Any process running as UID 1000 with access to the same `/tmp` (shared `emptyDir` volume, sidecar, or container escape) can `touch /tmp/alive` every <2 minutes after the pinger process is killed, keeping the liveness probe green while the orchestrator is deceived into not restarting the pod.

**Why existing checks are insufficient:** The 2-minute staleness window only bounds how stale the file can be; it does not verify that the file was written by the pinger process, that any transaction was submitted, or that the transaction loop goroutine is unblocked. [7](#0-6) 

### Impact Explanation
The pinger's sole purpose is to submit periodic transactions to Hashgraph so that the mirror node's ingestion pipeline can be continuously validated. A silent gap — where the orchestrator believes the pinger is healthy but no transactions are landing on-chain — directly undermines that guarantee. Monitoring dashboards show green, alerting is suppressed, and the gap in on-chain history is only discovered after the fact (if at all). Severity: **High** for availability/integrity of the on-chain audit trail.

### Likelihood Explanation
The architectural path (SDK call blocking) requires no attacker at all — it is a latent bug triggered by any network-level anomaly or SDK regression. The external-write amplification requires either a misconfigured shared `emptyDir` volume (common in Kubernetes multi-container pods) or a same-UID sidecar, both of which are realistic in real deployments. Container escape is a higher bar but not required for the primary path.

### Recommendation
1. **Couple the heartbeat to transaction success**: Only write to `/tmp/alive` after `submitWithRetry` returns `nil`, not on a free-running timer.
2. **Add a per-call deadline**: Wrap `cryptoTransfer.Execute` with a `context.WithTimeout` derived from `ctx` so a hung SDK call is bounded and the main loop can recover.
3. **Add a watchdog timestamp**: Record the last successful transaction time in an atomic variable; the heartbeat goroutine should write to `/tmp/alive` only if `time.Since(lastSuccess) < threshold`.
4. **Restrict `/tmp/alive` permissions**: Use `0600` instead of `0644` to prevent same-UID writes from other processes.
5. **Use a Unix domain socket or pipe** for the liveness signal instead of a world-readable file, so only the pinger process can signal liveness.

### Proof of Concept

**Architectural path (no external attacker):**
```bash
# 1. Simulate a hung SDK call by introducing an indefinite block in Execute
#    (e.g., iptables DROP to the Hedera node IPs, no TCP RST)
iptables -I OUTPUT -d <hedera-node-ip> -j DROP

# 2. Observe: pinger main loop goroutine blocks in Execute, no transactions submitted
# 3. Observe: /tmp/alive is still updated every 15 s by the heartbeat goroutine
watch -n 5 'stat /tmp/alive'

# 4. Observe: liveness probe continues to pass
/healthcheck live; echo $?   # exits 0

# 5. Confirm: no new transactions appear on-chain via mirror REST API
curl https://<mirror-rest>/api/v1/transactions?limit=5
```

**External-write amplification path:**
```bash
# Precondition: shared emptyDir volume mounted at /tmp in a sidecar container
# running as UID 1000

# 1. Kill the pinger process (e.g., via SIGKILL from another privileged container)
kill -9 <pinger-pid>

# 2. From the sidecar (UID 1000, shared /tmp), keep touching the file
while true; do
    echo "$(date -u +%Y-%m-%dT%H:%M:%S.%NZ)" > /tmp/alive
    sleep 60
done

# 3. Liveness probe continues to pass; orchestrator does not restart pinger
/healthcheck live; echo $?   # exits 0

# 4. No transactions are submitted to Hashgraph; on-chain gap grows silently
```
