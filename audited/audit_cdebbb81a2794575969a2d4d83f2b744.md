### Title
ETH_ESTIMATE_GAS Binary Search Bypasses Gas Throttle — Up to 21x EVM Execution Amplification

### Summary
The `ContractController.call()` endpoint applies gas-bucket throttling once per HTTP request based on the declared gas limit, but `estimate=true` requests internally trigger one initial EVM execution plus up to 20 additional binary-search EVM executions, none of which interact with the gas bucket. An unprivileged attacker can therefore cause up to 21× more real EVM work per throttle token than a normal `ETH_CALL`, enabling a low-volume sender to saturate the mirror node's EVM execution pipeline and render the web3 API unavailable.

### Finding Description

**Entry point — throttle is charged once:**

`ContractController.call()` calls `throttleManager.throttle(request)` before any processing. [1](#0-0) 

`ThrottleManagerImpl.throttle()` consumes exactly `scaleGas(request.getGas())` tokens from the gas bucket and 1 token from the rate bucket — both consumed once, at HTTP-request entry. [2](#0-1) 

**Estimate path — 1 + up to 20 EVM executions:**

When `params.isEstimate()` is true, `ContractExecutionService.estimateGas()` first calls `callContract()` (initial execution), then hands control to `BinaryGasEstimator.search()` for up to `maxGasEstimateRetriesCount = 20` additional iterations. [3](#0-2) 

The binary search loop calls `doProcessCall(params, gas, true)` — note the `true` flag — on every iteration. [4](#0-3) 

**Root cause — `estimate=true` skips `restoreGasToBucket` and all gas-bucket interaction:**

`doProcessCall()` only calls `restoreGasToBucket` (and records metrics) when `estimate` is `false`. When `estimate=true`, the EVM still executes at line 115, but the `finally` block is a no-op with respect to the throttle. [5](#0-4) 

The initial `callContract()` call inside `estimateGas()` uses `doProcessCall(params, params.getGas(), false)` (estimate=false), so it does restore gas. But all 20 binary-search iterations use `estimate=true` and restore nothing. [6](#0-5) 

**Net throttle accounting per request:**

| Request type | Gas bucket charged | EVM executions |
|---|---|---|
| `ETH_CALL` | `scaleGas(gasUsed)` (net after refund) | 1 |
| `ETH_ESTIMATE_GAS` | `scaleGas(gasUsed_initial)` (net after refund of initial call only) | 1 + up to 20 |

The 20 binary-search EVM executions are completely invisible to the gas bucket.

**Why existing checks fail:**

- `rateLimitBucket` (default 500 req/s): limits HTTP requests, not EVM executions per request. 500 estimate requests/s → up to 10,500 EVM executions/s.
- `gasLimitBucket` (default 7.5 B gas/s): charged once at entry, refunded after the initial call. For a contract using minimal gas (e.g., 21,000), the net bucket consumption is ~2 scaled tokens, leaving the bucket nearly full while 20 more EVM runs proceed.
- `maxGasEstimateRetriesCount = 20`: caps amplification at 21× but does not prevent it. [7](#0-6) [8](#0-7) 

### Impact Explanation
An attacker who saturates the 500 req/s rate limit exclusively with `estimate=true` requests forces up to 10,500 EVM executions per second on the mirror node, compared to 500 for normal `ETH_CALL`. Because the gas bucket is barely consumed (gas is refunded after the initial call), the gas-based throttle offers no meaningful protection. The EVM execution thread pool becomes the bottleneck; legitimate `ETH_CALL` and `ETH_ESTIMATE_GAS` requests queue indefinitely, making the mirror node's web3 API unresponsive. This is a denial-of-service against the mirror node's read layer; it does not directly halt Hedera consensus, but it disables the primary JSON-RPC interface used by wallets, dApps, and relayers.

### Likelihood Explanation
No authentication or API key is required. The attacker needs only a standard HTTP client and a valid (or zero-address) contract target. The attack is repeatable at the rate limit (500 req/s) from a single IP, or distributed across multiple IPs to bypass any upstream IP-level rate limiting. The payload is a trivial JSON body with `"estimate": true` and `"gas": 15000000`. The amplification is deterministic and bounded at 21×, making it easy to calibrate.

### Recommendation
1. **Charge the gas bucket per EVM execution, not per HTTP request.** Before each binary-search iteration in `BinaryGasEstimator.search()`, consume `scaleGas(mid)` tokens from the gas bucket. If the bucket is exhausted, abort the search and return the current best estimate.
2. **Alternatively, apply a multiplier at throttle entry for estimate requests.** In `ThrottleManagerImpl.throttle()`, detect `request.isEstimate()` and consume `scaleGas(request.getGas()) * maxGasEstimateRetriesCount` tokens upfront, refunding the unused portion after the search completes.
3. **Reduce `maxGasEstimateRetriesCount`** from 20 to a lower value (e.g., 10) to halve the worst-case amplification while still providing accurate estimates for most contracts.
4. **Add per-IP rate limiting** at the ingress layer (e.g., nginx, API gateway) as a defense-in-depth measure.

### Proof of Concept

```bash
# Attacker script — no credentials required
# Target: POST /api/v1/contracts/call with estimate=true and max gas

for i in $(seq 1 500); do
  curl -s -X POST https://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{
      "from": "0x0000000000000000000000000000000000000000",
      "to":   "0x<any_deployed_contract>",
      "gas":  15000000,
      "data": "0x<selector_of_gas_intensive_function>",
      "estimate": true
    }' &
done
wait
```

**Expected result:** Each of the 500 concurrent requests triggers 1 initial EVM execution + up to 20 binary-search EVM executions = up to 10,500 EVM executions/second. The gas bucket is barely consumed because the initial call's unused gas is refunded. The mirror node's EVM thread pool saturates; subsequent legitimate requests receive timeouts or 429 errors from the rate-limit bucket before the gas bucket is ever exhausted.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);

                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
