### Title
Unvalidated `domain_name` in Mirror Node Response Allows IPv6 Zone-ID Injection Leading to Pinger DoS

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` accepts `domain_name` values from the mirror node JSON response and passes them directly to `net.JoinHostPort` without any format validation beyond an empty-string check. An attacker who controls the mirror node response (e.g., via MITM or a rogue mirror node) can inject IPv6 addresses with zone IDs (e.g., `fe80::1%eth0`) into every `service_endpoints[].domain_name` field, causing `net.JoinHostPort` to produce addresses like `[fe80::1%eth0]:50211` that are structurally accepted but permanently unroutable, silently poisoning the entire network map and rendering the pinger unable to submit any transactions.

### Finding Description

**Exact code path** — `pinger/mirror_node_client.go`, lines 113–124:

```go
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)   // line 114 – only whitespace stripped
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)
    }
    if host == "" || ep.Port == 0 {            // line 118 – only guard is non-empty + non-zero port
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))  // line 122
    network[addr] = nodeAccountId                                // line 123
}
```

**Root cause** — The only rejection criteria are an empty `host` string and a zero port. No format validation (RFC 1123 hostname, dotted-decimal IPv4, or bracket-enclosed IPv6) is applied to `domain_name`.

**Exploit flow:**

1. Attacker returns a crafted JSON body from the mirror node endpoint where every `service_endpoints` entry has `"domain_name": "fe80::1%eth0"` and a non-zero port.
2. `strings.TrimSpace("fe80::1%eth0")` → `"fe80::1%eth0"` (non-empty, passes the guard).
3. `net.JoinHostPort("fe80::1%eth0", "50211")` detects the embedded `:` and wraps the host in brackets → `"[fe80::1%eth0]:50211"`. This is syntactically accepted by Go's `net` package.
4. The poisoned address is inserted into `network` and returned to `newClient()` → `hiero.ClientForNetwork(netmap)` (sdk_client.go line 22).
5. The Hiero SDK / gRPC layer attempts to dial `[fe80::1%eth0]:50211`. The `%eth0` zone ID references a local interface; the address is link-local and unreachable from any remote host. Every dial attempt times out or is immediately rejected.
6. Because `len(network) > 0` (line 127), the "no usable endpoints" error is never triggered — the pinger believes it has a valid network map and proceeds, but every subsequent `submitWithRetry` call fails.

**Why existing checks fail:**

- The empty-string check (line 115/118) passes because `"fe80::1%eth0"` is non-empty.
- There is no IP-address parsing (`net.ParseIP`), no hostname regex, and no rejection of the `%` character.
- `net.JoinHostPort` does not validate the host — it only decides whether to add brackets.

### Impact Explanation

The pinger is the liveness/readiness probe for the Hiero mirror node deployment. With a fully poisoned network map the pinger can never submit a transfer transaction; `main.go` logs `transfer failed` on every tick (line 67) and the `/tmp/ready` file is never refreshed, causing Kubernetes liveness probes to eventually kill and restart the pod in a crash loop. Monitoring data is lost and operators receive false "network down" signals. Because the pinger targets the `other` network mode (used for custom/private deployments that may represent significant stake), the impact can extend to any deployment that relies on a dynamically discovered address book.

### Likelihood Explanation

The precondition is control of the mirror node HTTP response. This is achievable by:
- A network-level MITM on an unencrypted (HTTP) mirror REST endpoint (the default example URL in config.go is `http://mirror-rest:5551`).
- A compromised or malicious mirror node operator.
- DNS poisoning of the mirror node hostname.

The attack requires no authentication, no special protocol knowledge, and is fully repeatable — the pinger re-fetches the address book on every startup and after every crash-loop restart, so the DoS is self-sustaining.

### Recommendation

1. **Validate `domain_name` format** before use. Accept only strings that pass `net.ParseIP` (for bare IP literals) or a strict RFC 1123 hostname regex. Explicitly reject any value containing `%`, `[`, `]`, or other non-hostname characters.
2. **Reject IPv6 literals in `domain_name`** entirely — the field is semantically a DNS name; raw IPv6 addresses belong in `ip_address_v4` (or a dedicated `ip_address_v6` field).
3. **Validate port range** (1–65535) rather than just `!= 0`.
4. **Enforce TLS** on the mirror REST endpoint to prevent MITM injection.

Example guard to add before line 122:

```go
if net.ParseIP(host) != nil && strings.ContainsAny(host, "%[]") {
    continue // reject zone-ID or bracket-enclosed IPv6 in domain_name
}
if strings.ContainsAny(host, "%[]") {
    continue
}
```

### Proof of Concept

Stand up a mock HTTP server that returns:

```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [
      {"domain_name": "fe80::1%eth0", "ip_address_v4": "", "port": 50211},
      {"domain_name": "fe80::2%eth0", "ip_address_v4": "", "port": 50212}
    ]
  }],
  "links": {"next": null}
}
```

Point the pinger at it:

```
HIERO_MIRROR_PINGER_NETWORK=other \
HIERO_MIRROR_PINGER_REST=http://<attacker-host>:8080 \
HIERO_MIRROR_PINGER_OPERATOR_ID=0.0.2 \
HIERO_MIRROR_PINGER_OPERATOR_KEY=<any valid key> \
./pinger
```

**Observed result:** `newClient` succeeds (network map is non-empty), but every `submitWithRetry` call fails with a gRPC dial error against `[fe80::1%eth0]:50211`. The pinger loops indefinitely logging `transfer failed`, `/tmp/ready` is never updated after the initial write, and the liveness probe eventually kills the process — crash-loop DoS.
