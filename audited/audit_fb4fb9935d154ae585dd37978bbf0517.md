### Title
Prometheus High-Cardinality Route Label via Unmatched URL Paths Causes Unbounded Memory Growth (OOM)

### Summary
`MetricsMiddleware()` in `rosetta/app/middleware/metrics.go` delegates route-label extraction to `middleware.RouteMatcher.MatchRoute()` from `github.com/weaveworks/common/middleware`. For requests that do not match any registered gorilla/mux route, `MatchRoute` falls back to the raw `r.URL.Path` value, which is then stored verbatim as the `route` label in all four `HistogramVec`/`GaugeVec` instruments. Because Prometheus allocates a new time-series per unique label combination, an unauthenticated attacker who sends requests to a large number of distinct URL paths causes the registry to grow without bound, eventually OOM-killing the process.

### Finding Description

**Exact code path:**

`rosetta/app/middleware/metrics.go`, `MetricsMiddleware()`, lines 76–83: [1](#0-0) 

```go
func MetricsMiddleware(next http.Handler) http.Handler {
    return middleware.Instrument{
        Duration:         requestDurationHistogram,   // labels: method, route, status_code, ws
        InflightRequests: requestInflightGauge,       // labels: method, route
        RequestBodySize:  requestBytesHistogram,      // labels: method, route
        ResponseBodySize: responseBytesHistogram,     // labels: method, route
        RouteMatcher:     next.(middleware.RouteMatcher),
    }.Wrap(next)
}
```

`requestDurationHistogram` is declared with 6 duration buckets and 4 label dimensions: [2](#0-1) 

**Root cause:**

`middleware.Instrument.Wrap()` (weaveworks `v0.0.0-20230728070032-dd9e68f319d5`) calls `i.RouteMatcher.MatchRoute(r)` on every request. The underlying gorilla/mux router (`v1.8.1`, pulled in transitively via `coinbase/rosetta-sdk-go v0.11.0`) returns the registered path template when a route matches, but returns `r.URL.Path` verbatim when no route matches. There is no sanitisation, truncation, or normalisation of this value before it is passed to `prometheus.Labels{"route": route}`. Each distinct path string causes Prometheus to allocate a new set of time-series (one per bucket × 4 instruments), retaining them in memory for the lifetime of the process. [3](#0-2) [4](#0-3) 

**Failed assumption:** The code assumes `RouteMatcher` will always return one of the small, fixed set of registered Rosetta API paths. It does not account for the fallback-to-raw-URL behaviour for unmatched requests.

### Impact Explanation

Each unique URL path sent by an attacker creates a permanent entry in the Prometheus default registry. For `requestDurationHistogram` alone, each new `route` value allocates `6 buckets + _sum + _count = 8` float64 counters multiplied by the number of `method × status_code × ws` combinations. Across all four instruments the per-unique-path cost is on the order of tens of kilobytes of heap. Sending ~100 000 distinct paths (trivially achievable with a short script) can consume gigabytes of heap, triggering the Linux OOM killer and terminating the rosetta process — a complete denial of service.

### Likelihood Explanation

**Preconditions:** None. The rosetta HTTP port is publicly reachable (it is the API endpoint). No authentication is required to send arbitrary GET/POST requests. The attacker needs only a network path to the service.

**Feasibility:** A single machine with a modest network connection can generate hundreds of thousands of unique URL paths per minute. The Traefik ingress rate-limit (`average: 10` req/s per host) defined in `charts/hedera-mirror-rosetta/values.yaml` is an optional, Kubernetes-level control: [5](#0-4) 

It is not enforced at the application layer, is trivially bypassed by rotating source IPs, and is not present in non-Kubernetes deployments. The attack is fully repeatable and requires no special knowledge of the application.

### Recommendation

1. **Normalise the route label before recording.** Wrap `RouteMatcher.MatchRoute()` in a helper that returns a fixed sentinel (e.g. `"unmatched"`) whenever the returned string is not in the known set of registered Rosetta paths.
2. **Alternatively**, implement a custom `RouteMatcher` that only returns the path template for known routes and returns `"unmatched"` for everything else, preventing any attacker-controlled string from reaching the Prometheus registry.
3. **Add an application-level request rate limiter** (e.g. `golang.org/x/time/rate`) that is independent of the ingress layer.
4. **Set a maximum URL length** at the HTTP server level (`http.Server.ReadHeaderTimeout` is already set, but `http.MaxBytesReader` or a custom handler should also reject oversized request URIs before they reach the middleware).

### Proof of Concept

```bash
# Send 100 000 requests each with a unique path, bypassing the small set of
# registered Rosetta routes so every path falls through to the raw-URL fallback.
for i in $(seq 1 100000); do
  curl -s -o /dev/null "http://<rosetta-host>:<port>/attack-$(openssl rand -hex 16)" &
done
wait

# Observe memory growth:
curl -s http://<rosetta-host>:<port>/metrics | grep hiero_mirror_rosetta_request_duration_bucket | wc -l
# The line count grows linearly with the number of unique paths sent.
# Monitor process RSS until OOM kill occurs.
```
