### Title
Unauthenticated Access to Full EVM Opcode Trace Endpoint Exposes Sensitive Transaction Data

### Summary
The `getContractOpcodes()` endpoint in `OpcodesController` performs no authentication or authorization check before replaying any historical transaction and returning its complete EVM execution trace — including full memory, stack, and storage contents. Any anonymous internet user who knows (or can enumerate) a transaction ID or hash can retrieve the complete low-level execution state of that transaction, including any sensitive data that was present in EVM memory or storage during execution.

### Finding Description

**Exact code path:**

`web3/src/main/java/org/hiero/mirror/web3/controller/OpcodesController.java`, lines 52–68:

```java
@GetMapping(value = "/{transactionIdOrHash}/opcodes")
OpcodesResponse getContractOpcodes(
        @PathVariable TransactionIdOrHashParameter transactionIdOrHash,
        @RequestParam(required = false, defaultValue = "true") boolean stack,
        @RequestParam(required = false, defaultValue = "false") boolean memory,
        @RequestParam(required = false, defaultValue = "false") boolean storage,
        @RequestHeader(value = HttpHeaders.ACCEPT_ENCODING) String acceptEncoding) {
    if (properties.isEnabled()) {
        validateAcceptEncodingHeader(acceptEncoding);
        throttleManager.throttleOpcodeRequest();
        final var request = new OpcodeRequest(transactionIdOrHash, stack, memory, storage);
        return opcodeService.processOpcodeCall(request);
    }
    throw new ResponseStatusException(HttpStatus.NOT_FOUND);
}
``` [1](#0-0) 

**Root cause:** The method has no `@PreAuthorize`, no Spring Security filter, and no identity/ownership check. No `SecurityConfig` class exists in the web3 module. The only guards are:

1. A feature flag (`properties.isEnabled()`) — `false` by default per `OpcodesProperties`, but **must be set to `true`** for the feature to function at all. This is a deployment toggle, not a security control.
2. A `gzip` `Accept-Encoding` header check — trivially satisfied by any HTTP client.
3. A rate limiter (`throttleManager.throttleOpcodeRequest()`) — limits request rate but does not prevent data disclosure. [2](#0-1) 

**What the tracer captures:** `AbstractOpcodeTracer.captureMemory()`, `captureStack()`, and `captureStorage()` dump the raw EVM memory words, stack items, and all storage slot reads/writes at every opcode step during the replay. [3](#0-2) [4](#0-3) 

**CORS is open:** `GenericControllerAdvice` configures `allowedOrigins("*")` for all `/api/v1/contracts/**` paths, meaning browser-based cross-origin requests are also permitted. [5](#0-4) 

**Failed assumption:** The design assumes that enabling the feature flag is sufficient protection because it is an operator-controlled toggle. This assumption fails because once enabled, the endpoint is reachable by the entire internet with no identity check. Transaction IDs and hashes are public on the Hedera ledger.

### Impact Explanation

When the opcode tracer is enabled, any unauthenticated caller can replay any historical contract transaction with `stack=true&memory=true&storage=true`. For a healthcare contract that processes patient data (e.g., patient identifiers, diagnoses, or records passed as ABI-encoded calldata), that data is loaded into EVM memory during execution. The full memory dump at every opcode step is returned verbatim in the `OpcodesResponse`. Storage slot values written or read during the transaction are also returned. This constitutes a complete, step-by-step disclosure of all data the contract processed — equivalent to attaching a debugger to the original execution. Severity is **Critical** when the feature is enabled on a node serving sensitive contract data.

### Likelihood Explanation

- Transaction hashes are publicly visible on the Hedera mirror node REST API (`/api/v1/contracts/results`), requiring no privilege to enumerate.
- The only precondition is that the operator has set `hiero.mirror.web3.opcode.tracer.enabled=true`, which is the required configuration for any legitimate use of this debugging endpoint.
- The attack requires a single HTTP GET request with `Accept-Encoding: gzip` — reproducible by any HTTP client, script, or browser.
- Rate limiting slows bulk enumeration but does not prevent targeted disclosure of a known transaction.

### Recommendation

1. **Require authentication** on this endpoint. Add a Spring Security configuration that restricts `/api/v1/contracts/results/*/opcodes` to authenticated and authorized roles (e.g., operator/admin only). No unauthenticated caller should be able to trigger an EVM replay.
2. **Add an `@PreAuthorize` annotation** or a dedicated security filter that validates a bearer token or API key before `processOpcodeCall` is invoked.
3. Consider whether this endpoint should ever be exposed on a public-facing mirror node, or whether it should be restricted to loopback/internal network access via network policy in addition to application-level auth.
4. Do not rely on the feature flag as a security boundary — it is a functional toggle, not an access control mechanism.

### Proof of Concept

**Preconditions:** `hiero.mirror.web3.opcode.tracer.enabled=true` is set on the target mirror node (required for the feature to work).

**Steps:**

1. Enumerate a contract transaction hash from the public mirror node REST API:
   ```
   GET /api/v1/contracts/results?limit=1
   ```
   Extract the `hash` field from the response.

2. Submit the opcode replay request as an unauthenticated user:
   ```
   GET /api/v1/contracts/results/0x<txhash>/opcodes?stack=true&memory=true&storage=true
   Accept-Encoding: gzip
   ```

3. Decompress the gzip response. The JSON body contains an `opcodes` array where each entry includes:
   - `memory`: array of 32-byte hex words — the full EVM memory at that step
   - `stack`: array of hex values — the EVM stack at that step
   - `storage`: map of slot key → value — all storage accesses

4. Decode the `memory` entries as ABI-encoded data to recover any patient records, identifiers, or other sensitive values that were passed as calldata or computed during the transaction.

No credentials, tokens, or special network access are required beyond the ability to reach the mirror node's HTTP port.

    protected final List<String> captureMemory(final MessageFrame frame, final OpcodeContext options) {
        if (!options.isMemory()) {
            return Collections.emptyList();
        }
        var size = frame.memoryWordSize();
        var hexMemoryEntries = new ArrayList<String>(size);

        final var wordSize = 32;
        final var memory = frame.readMutableMemory(0L, (long) wordSize * size).toArrayUnsafe();
        final var hex = HexFormat.of();

        for (var i = 0; i < size; i++) {
            var startIndex = i * wordSize;
            final var result = HEX_PREFIX + hex.formatHex(memory, startIndex, startIndex + wordSize);
            hexMemoryEntries.add(result);
        }

        return hexMemoryEntries;
    }

    protected Map<String, String> captureStorage(
            final MessageFrame frame, final OpcodeContext options, final ContractCallContext context) {
        if (!options.isStorage()) {
            return Collections.emptyMap();
        }

        try {
            if (context.getOpcodeContext().getRootProxyWorldUpdater() == null) {

                var worldUpdater = frame.getWorldUpdater();
                var parent = worldUpdater.parentUpdater().orElse(null);
                while (parent != null) {
                    worldUpdater = parent;
                    parent = worldUpdater.parentUpdater().orElse(null);
                }

                if (!(worldUpdater instanceof RootProxyWorldUpdater rootProxyWorldUpdater)) {
                    // The storage updates are kept only in the RootProxyWorldUpdater.
                    // If we don't have one -> something unexpected happened and an attempt to
                    // get the storage changes from a ProxyWorldUpdater would result in a
                    // NullPointerException, so in this case just return an empty map.
                    return Collections.emptyMap();
                }

                context.getOpcodeContext().setRootProxyWorldUpdater(rootProxyWorldUpdater);
            }

            final var rootProxyWorldUpdater = context.getOpcodeContext().getRootProxyWorldUpdater();
            final var updates = rootProxyWorldUpdater
                    .getEvmFrameState()
                    .getTxStorageUsage(true)
                    .accesses();

            if (updates.isEmpty()) {
                return Collections.emptyMap();
            }

            final var result = new TreeMap<String, String>();
            for (final var storageAccesses : updates) {
                for (final var access : storageAccesses.accesses()) {
                    final var key = hexCache.get(access.key(), Bytes::toHexString);
                    if (!result.containsKey(key)) {
                        final var value = access.writtenValue() != null
                                ? hexCache.get(access.writtenValue(), Bytes::toHexString)
                                : hexCache.get(access.value(), Bytes::toHexString);
                        result.put(key, value);
                    }
                }
            }
            return result;
