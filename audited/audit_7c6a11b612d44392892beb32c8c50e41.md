### Title
Unauthenticated Thundering Herd via Nil BlockIdentifier in `AccountBalance()` Causing Unbounded DB Queries

### Summary
Any unauthenticated caller can POST to `/account/balance` with a valid account address and omit `block_identifier`, causing `RetrieveLatest()` to issue a raw SQL query against the `record_file` table on every single request with no caching, deduplication, or application-level rate limiting. Flooding this endpoint with concurrent requests saturates the database connection pool and degrades performance for all users of the Rosetta API.

### Finding Description

**Exact code path:**

In `rosetta/app/services/account_service.go`, `AccountBalance()` at lines 52â€“56: [1](#0-0) 

When `request.BlockIdentifier == nil`, the code unconditionally calls `a.RetrieveLatest(ctx)`.

`BaseService.RetrieveLatest()` in `rosetta/app/services/base_service.go` at lines 100â€“106 performs no caching â€” it directly delegates to the repository: [2](#0-1) 

`blockRepository.RetrieveLatest()` in `rosetta/app/persistence/block.go` at lines 190â€“208 executes a raw SQL query on every invocation with no result caching, no singleflight coalescing, and no TTL: [3](#0-2) 

The SQL executed is: [4](#0-3) 

This is a `SELECT ... FROM record_file ORDER BY index DESC LIMIT 1` â€” a sequential scan with sort on a table that grows continuously.

**Root cause:** There is no in-process cache, singleflight group, or request coalescing for the "latest block" lookup in the Rosetta Go service. Every concurrent request with a nil `BlockIdentifier` issues an independent DB query.

**Why existing checks fail:**

The only guard in `RetrieveLatest()` is `IsOnline()`, which merely checks `b.blockRepo != nil` â€” it is not a rate limit: [5](#0-4) 

The Traefik middleware (rate limit + inFlightReq) in the Helm chart is **conditionally applied** only when `global.middleware` and `middleware` values are both set: [6](#0-5) 

It is not enforced at the application layer. Deployments without Traefik or with middleware disabled have zero protection. The throttling code found in the codebase (`ThrottleManagerImpl`, `ThrottleConfiguration`) belongs to the `web3` Java module, not the Rosetta Go service: [7](#0-6) 

### Impact Explanation
Concurrent unauthenticated requests with nil `BlockIdentifier` each consume one DB connection for the duration of the `ORDER BY index DESC LIMIT 1` query. At sufficient concurrency, the DB connection pool is exhausted, causing all Rosetta API requests (including those from legitimate users with specific block identifiers) to queue or fail. This is a griefing/availability attack with no economic damage to on-chain state, consistent with the Medium severity classification.

### Likelihood Explanation
Preconditions: none. The endpoint is public, requires no authentication, and accepts any syntactically valid account address (e.g., `0.0.1`). The attack is trivially reproducible with any HTTP load-testing tool (`wrk`, `hey`, `ab`). The attacker needs only a single machine with moderate network bandwidth. The attack is repeatable indefinitely.

### Recommendation
1. **Add a singleflight/coalescing layer** for `RetrieveLatest()` so that N concurrent calls in-flight result in exactly one DB query, with all waiters receiving the same result (Go's `golang.org/x/sync/singleflight` package).
2. **Add a short TTL in-process cache** (e.g., 500 ms) for the latest-block result, analogous to the Caffeine cache used in the `web3` Java module (`CACHE_MANAGER_RECORD_FILE_LATEST` with 500 ms `expireAfterWrite`): [8](#0-7) 
3. **Enforce application-level rate limiting** in the Rosetta Go HTTP handler, independent of the optional Traefik middleware.

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}
