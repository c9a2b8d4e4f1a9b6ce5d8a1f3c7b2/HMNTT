### Title
Query-String Bypass of Internal-Path Log-Level Filter in `TracingMiddleware`

### Summary
`TracingMiddleware` uses `request.URL.RequestURI()` to obtain the request path, which includes any query string. The `internalPaths` map is keyed on bare paths only. Any unprivileged caller who appends a query string to a health-check path (e.g., `/health/liveness?x=1`) causes the map lookup to miss, and the request is logged at `log.Info` instead of `log.Debug`. An attacker can exploit this to flood the Info-level log with synthetic health-check entries, burying real transaction log lines.

### Finding Description
**File:** `rosetta/app/middleware/trace.go`  
**Function:** `TracingMiddleware`, lines 43–61

```
path := request.URL.RequestURI()          // line 47 — includes query string
...
if internalPaths[path] {                  // line 55 — bare-path map lookup
    log.Debug(message)
} else {
    log.Info(message)                     // reached when query string present
}
```

`net/http`'s `RequestURI()` returns the unmodified request-target from the HTTP request line, including any `?query` component. The `internalPaths` map is populated with the three bare constants (`/health/liveness`, `/health/readiness`, `/metrics`). A request to `/health/liveness?x=1` produces `path = "/health/liveness?x=1"`, which is not a key in the map, so `internalPaths[path]` evaluates to `false` and the request is logged at `Info`. No authentication, no special headers, and no rate-limiting exist on the health endpoints.

### Impact Explanation
An attacker can continuously send `GET /health/liveness?<random>=<value>` requests, each producing an `Info`-level log line. Because real Rosetta API calls (e.g., `/block`, `/transaction`) are also logged at `Info`, the attacker's synthetic entries are indistinguishable in the log stream. This degrades the integrity of the operational audit trail: operators relying on Info logs to reconstruct transaction history will see it polluted with attacker-controlled noise, making incident response and forensic analysis unreliable.

### Likelihood Explanation
Preconditions: none. The health endpoints are publicly reachable by design (they are Kubernetes liveness probes). Any HTTP client can append a query string. The attack is trivially scriptable, repeatable at high frequency, and requires zero privileges or credentials.

### Recommendation
Replace the `RequestURI()` call with `request.URL.Path`, which strips the query string before the map lookup:

```go
// line 47 — use Path, not RequestURI()
path := request.URL.Path
```

The log message can still include the full URI for diagnostic purposes by keeping `request.URL.RequestURI()` only in the `message` format string, while using `request.URL.Path` exclusively for the `internalPaths` gate.

### Proof of Concept
```bash
# Normal health check — logged at Debug (suppressed in production)
curl http://<rosetta-host>/health/liveness

# Bypass — logged at Info, pollutes transaction audit log
curl "http://<rosetta-host>/health/liveness?injected=true"

# High-volume log flood
for i in $(seq 1 10000); do
  curl -s "http://<rosetta-host>/health/liveness?i=$i" &
done
```
Observe that the Info-level log is flooded with `GET /health/liveness?i=N (200)` entries, obscuring legitimate `Info`-level Rosetta API transaction entries.
