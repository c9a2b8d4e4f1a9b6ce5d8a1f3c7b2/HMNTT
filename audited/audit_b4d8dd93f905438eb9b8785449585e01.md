### Title
Unauthenticated `/metrics` Endpoint Exposes Real-Time Inflight Request Gauge, Enabling Timed Burst Attacks

### Summary
The Rosetta server registers `/metrics` on the same port as the public API with no authentication, no IP restriction, and no application-level access control. The `internalPaths` map in `trace.go` only suppresses the access log to `DEBUG` level — it does not block the request. An attacker who can reach the pod port (directly or via any non-ingress path) can poll `hiero_mirror_rosetta_request_inflight` in real time and time burst submissions to arrive when the server is near the Traefik `inFlightReq` cap, maximising the probability that legitimate transactions are dropped or gossip is suppressed.

### Finding Description

**Code path — no authentication on `/metrics`:**

`rosetta/app/middleware/metrics.go` lines 64–73 registers the route:
```go
server.Routes{{
    "metrics", "GET", metricsPath,
    promhttp.Handler().ServeHTTP,   // bare handler, zero auth
}}
``` [1](#0-0) 

`rosetta/main.go` lines 217–219 builds the middleware chain:
```go
metricsMiddleware := middleware.MetricsMiddleware(router)
tracingMiddleware := middleware.TracingMiddleware(metricsMiddleware)
corsMiddleware    := server.CorsMiddleware(tracingMiddleware)
```
There is no authentication middleware anywhere in the chain. [2](#0-1) 

**`internalPaths` provides zero access control:**

`trace.go` line 19 declares `internalPaths` and lines 55–59 use it only to choose between `log.Debug` and `log.Info`. `inner.ServeHTTP` is called unconditionally on line 50 — the response is already sent before the log-level branch is reached. [3](#0-2) [4](#0-3) 

**Traefik middleware (the only rate/inflight guard) is disabled by default:**

`values.yaml` line 95 sets `global.middleware: false`. The middleware CRD template (`middleware.yaml`) is only rendered when `global.middleware` is true, so the `inFlightReq` (5 req cap) and `rateLimit` (10 req/s) guards are absent in the default deployment. [5](#0-4) [6](#0-5) 

**Ingress path exclusion is the only barrier — and it is bypassable:**

The ingress exposes only `/rosetta/account`, `/rosetta/block`, etc. — `/metrics` is absent. [7](#0-6) 

However: the service type is `ClusterIP` with no `NetworkPolicy` (no such template exists in the chart). Any pod inside the cluster, any operator using `kubectl port-forward`, any NodePort/LoadBalancer override, or any misconfigured ingress rule reaches the same port and gets the full Prometheus output — including `hiero_mirror_rosetta_request_inflight` — with no challenge. [8](#0-7) 

**The gauge that leaks capacity state:**

`metrics.go` lines 34–37 define the gauge that directly reports current server load per route:
```go
requestInflightGauge = prometheus.NewGaugeVec(prometheus.GaugeOpts{
    Name: "hiero_mirror_rosetta_request_inflight",
    Help: "Current number of inflight HTTP requests.",
}, []string{"method", "route"})
``` [9](#0-8) 

### Impact Explanation

An attacker who can reach the pod port polls `GET /metrics` at high frequency (the endpoint is cheap — it is a synchronous in-memory read). When `hiero_mirror_rosetta_request_inflight{route="/construction/submit"}` approaches the `inFlightReq` cap (default 5), the attacker fires a coordinated burst of `/construction/submit` requests. This fills the remaining slots, causing Traefik to return 429 to legitimate submitters. Because Rosetta's `/construction/submit` is the path through which signed transactions are forwarded to the Hedera network for gossip, sustained slot exhaustion delays or prevents transaction gossip — a denial-of-service at the protocol layer, not just the HTTP layer. The attacker's own requests can be crafted to be slow (large payloads, targeting slow DB paths) to hold slots longer.

### Likelihood Explanation

**Precondition:** Network reachability to the pod port. This is satisfied for: any co-tenant pod in the same namespace/cluster (no NetworkPolicy exists), any operator who has ever run `kubectl port-forward` and left it open, any deployment that changed `service.type` to `NodePort` or `LoadBalancer` for debugging, or any ingress rule that uses a wildcard path. None of these require credentials.

**Skill level:** Trivial. `curl http://<pod-ip>:5700/metrics` returns the full gauge set. Polling at 1 Hz and scripting the burst is a few lines of shell.

**Repeatability:** Continuous — the gauge resets as requests complete, so the attacker can repeat the timing attack indefinitely.

### Recommendation

1. **Application layer (primary fix):** Wrap `promhttp.Handler()` with an authentication middleware (e.g., Bearer token or mutual TLS) before registering the route, or bind `/metrics` to a separate, non-public port (standard Prometheus practice).

2. **Infrastructure layer:** Add a `NetworkPolicy` to the Helm chart that restricts ingress on the application port to only the Prometheus scraper namespace/pod selector, blocking all other in-cluster access.

3. **Clarify `internalPaths`:** Document (or remove) the misleading implication that listing a path in `internalPaths` provides any access restriction — it does not; it only changes log verbosity.

4. **Enable middleware by default:** Change `global.middleware` default to `true` so that `inFlightReq` and `rateLimit` guards are active in all deployments, not opt-in.

### Proof of Concept

```bash
# Step 1: Reach the pod port (any of the following)
kubectl port-forward svc/hedera-mirror-rosetta 5700:80 &
# OR from any pod in the cluster:
# curl http://<rosetta-clusterip>:80/metrics

# Step 2: Poll the inflight gauge
watch -n 0.5 'curl -s http://localhost:5700/metrics | grep hiero_mirror_rosetta_request_inflight'

# Step 3: When gauge value approaches the inFlightReq cap (5),
# fire a burst of construction/submit requests to fill remaining slots:
for i in $(seq 1 5); do
  curl -s -X POST http://localhost:5700/construction/submit \
    -H "Content-Type: application/json" \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},"signed_transaction":"<large_payload>"}' &
done

# Result: legitimate submitters receive HTTP 429 from Traefik;
# their transactions are not gossiped to the Hedera network.
```
