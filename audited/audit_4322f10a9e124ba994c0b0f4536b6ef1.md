### Title
Unauthenticated `/mempool` Endpoint as Deterministic Error Oracle Enables Circuit Breaker Manipulation and Service-Wide Network Partition

### Summary
The `Mempool()` handler in `rosetta/app/services/mempool_service.go` unconditionally returns `errors.ErrNotImplemented`, which the Coinbase Rosetta SDK serializes as an HTTP 500 response. Because `/rosetta/mempool` is publicly exposed with no authentication and the Traefik ingress middleware applies a single shared circuit breaker (`ResponseCodeRatio(500, 600, 0, 600) > 0.25`) across all rosetta paths, an unprivileged attacker can sustain a request rate to `/mempool` that keeps the 500-response ratio above the 25% threshold, permanently tripping the circuit breaker and blocking all other rosetta endpoints.

### Finding Description

**Exact code path:**

`rosetta/app/services/mempool_service.go`, lines 22–27:
```go
func (m *mempoolAPIService) Mempool(
    _ context.Context,
    _ *types.NetworkRequest,
) (*types.MempoolResponse, *types.Error) {
    return nil, errors.ErrNotImplemented
}
```
`ErrNotImplemented` is defined in `rosetta/app/errors/errors.go` line 70 as `newError(NotImplemented, 111, false)`. The Coinbase Rosetta SDK (`github.com/coinbase/rosetta-sdk-go/server`) maps any non-nil `*types.Error` return to an HTTP 500 response — this is the Rosetta specification's mandated behavior for all errors.

**Ingress exposure** (`charts/hedera-mirror-rosetta/values.yaml`, line 131):
```yaml
paths:
  - "/rosetta/mempool"   # publicly routed, no auth annotation
```
No authentication middleware is applied to this path.

**Circuit breaker** (`charts/hedera-mirror-rosetta/values.yaml`, line 150–151):
```yaml
- circuitBreaker:
    expression: NetworkErrorRatio() > 0.25 || ResponseCodeRatio(500, 600, 0, 600) > 0.25
```
This is a **service-wide** Traefik circuit breaker. It evaluates the ratio of 5xx responses across **all** requests routed through the middleware chain, regardless of path. When tripped, Traefik returns 503 for every rosetta endpoint (`/rosetta/account`, `/rosetta/block`, `/rosetta/network`, etc.).

**Rate limit** (`charts/hedera-mirror-rosetta/values.yaml`, lines 157–160):
```yaml
- rateLimit:
    average: 10
    sourceCriterion:
      requestHost: true
```
`requestHost: true` means the 10 req/s budget is shared across **all clients** using the same Host header — it is a global cap, not a per-IP cap. An attacker sending ≥3 req/s to `/mempool` (3/10 = 30% > 25%) is sufficient to sustain the trip condition.

**Root cause:** The failed assumption is that a permanently-failing, unauthenticated endpoint can coexist with a service-wide circuit breaker whose threshold is computed over all paths. The `/mempool` endpoint is a guaranteed, zero-cost 500 generator that any anonymous caller can exploit as a calibrated error oracle.

### Impact Explanation
When the circuit breaker trips, Traefik returns HTTP 503 for every path in the rosetta service — `/rosetta/network`, `/rosetta/block`, `/rosetta/account`, `/rosetta/construction`, etc. This constitutes a complete network partition of the Rosetta API: block explorers, exchange integrations, and construction workflows all fail. The attacker does not need to overwhelm the server; they only need to maintain a fraction (>25%) of the allowed request budget on a single endpoint that is guaranteed to return 500. The impact is a targeted, low-bandwidth, sustained denial of service against the entire Rosetta API surface.

### Likelihood Explanation
The attack requires no credentials, no special knowledge, and no high-bandwidth capability. The attacker needs only:
1. Knowledge that `/rosetta/mempool` always returns 500 (trivially discoverable by a single request).
2. The ability to send ≥3 HTTP POST requests per second to a public endpoint.

The rate limit (`average: 10, requestHost: true`) does not protect against this — it is a global budget, not a per-attacker budget. The `inFlightReq: amount: 5, ipStrategy: depth: 1` limits concurrent connections per IP but does not prevent sustained sequential requests. The attack is repeatable, scriptable, and requires no amplification.

### Recommendation
1. **Remove `/rosetta/mempool` from the public ingress paths** entirely, or return HTTP 404 (not 500) so it does not contribute to the 5xx ratio. The Rosetta spec permits returning a 404 for unimplemented endpoints.
2. **Scope the circuit breaker per-path or exclude known-failing paths** from the ratio calculation, so a permanently-failing stub endpoint cannot influence the health signal for other endpoints.
3. **Change the rate limit `sourceCriterion` to `ipStrategy`** (per-IP) rather than `requestHost` (global), so a single attacker cannot consume the entire request budget.
4. **Add authentication or IP-based access control** to the `/mempool` and `/mempool/transaction` endpoints if they must remain routed.

### Proof of Concept
```bash
# Step 1: Confirm /mempool always returns HTTP 500
curl -s -o /dev/null -w "%{http_code}" -X POST https://<rosetta-host>/rosetta/mempool \
  -H "Content-Type: application/json" \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}'
# Output: 500

# Step 2: Sustain 4 req/s to keep 500-ratio above 25% (4/10 = 40%)
while true; do
  for i in 1 2 3 4; do
    curl -s -o /dev/null -X POST https://<rosetta-host>/rosetta/mempool \
      -H "Content-Type: application/json" \
      -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}' &
  done
  sleep 1
done

# Step 3: Observe circuit breaker trip — all other endpoints now return 503
curl -s -o /dev/null -w "%{http_code}" -X POST https://<rosetta-host>/rosetta/network/status \
  -H "Content-Type: application/json" \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"}}'
# Output: 503
```
