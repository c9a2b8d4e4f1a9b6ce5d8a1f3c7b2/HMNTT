### Title
Unbounded DB Read Amplification via Unauthenticated `/network/status` Flooding

### Summary
Every call to `NetworkStatus()` unconditionally executes at least two live database queries — one for the latest block and up to two for address book entries — with no response-level caching and no application-level rate limiting. An unprivileged external user can flood `POST /network/status` to amplify database reads linearly with request rate, exhausting DB connection pool resources and degrading service for all users.

### Finding Description

**Code path:**

`rosetta/app/services/network_service.go`, `NetworkStatus()`, lines 59–88: [1](#0-0) 

Each invocation unconditionally calls:

1. `n.RetrieveGenesis(ctx)` → `blockRepo.RetrieveGenesis(ctx)` → `initGenesisRecordFile()`. Genesis **is** cached via `sync.Once` after the first successful call, so subsequent calls skip the DB query. [2](#0-1) 

2. `n.RetrieveLatest(ctx)` → `blockRepo.RetrieveLatest(ctx)` — **always** executes `selectLatestWithIndex` raw SQL against `record_file` on every request, with no caching: [3](#0-2) 

3. `n.addressBookEntryRepo.Entries(ctx)` — **always** executes `latestNodeServiceEndpoints` (up to twice, once per address book file ID) on every request, with no caching: [4](#0-3) 

**No response caching exists in the Rosetta service.** The `responseCacheHandler.js` caching belongs to the separate REST API (Node.js). The Rosetta Go router in `main.go` only wraps with `MetricsMiddleware`, `TracingMiddleware`, and `CorsMiddleware` — no caching layer: [5](#0-4) 

**Rate limiting is disabled by default.** The Helm chart defines Traefik `rateLimit` middleware, but `global.middleware` defaults to `false`, meaning the middleware chain (including rate limiting) is not deployed in default installations: [6](#0-5) [7](#0-6) 

### Impact Explanation
Each `/network/status` request triggers 2–3 live DB queries. An attacker sending N requests/second forces N×(2–3) DB queries/second. With no connection-pool back-pressure at the application layer, this can saturate the PostgreSQL connection pool, increase query latency for all other endpoints (block queries, account queries, construction), and in extreme cases cause DB connection exhaustion, making the node unresponsive. The blockchain state itself is unaffected, matching the "griefing with no economic damage" classification, but service availability is degraded.

### Likelihood Explanation
The endpoint is unauthenticated, publicly reachable, and accepts a trivially small POST body (`{"network_identifier": {...}}`). No API key, token, or proof-of-work is required. Any attacker with network access and a basic HTTP client can sustain thousands of requests per second. Default deployments without Traefik middleware enabled have zero application-level protection.

### Recommendation
1. **Cache the `/network/status` response** in-process (e.g., a short TTL in-memory cache of 1–5 seconds) so repeated requests within the TTL window return the cached result without hitting the DB.
2. **Cache `addressBookEntryRepo.Entries()`** results with a longer TTL (address book changes infrequently).
3. **Enable the Traefik middleware by default** (`global.middleware: true`) or add application-level rate limiting (e.g., `golang.org/x/time/rate`) inside the handler.
4. Consider adding a `sync.Once`-style or time-bounded cache for `RetrieveLatest` results, similar to the existing genesis caching pattern.

### Proof of Concept
```bash
# Flood /network/status with no credentials required
# Replace <HOST> and <NETWORK> with deployment values

BODY='{"network_identifier":{"blockchain":"Hedera","network":"<NETWORK>"}}'

for i in $(seq 1 10000); do
  curl -s -o /dev/null -X POST http://<HOST>/network/status \
    -H 'Content-Type: application/json' \
    -d "$BODY" &
done
wait

# Observe on the DB side:
# SELECT count(*) FROM pg_stat_activity WHERE query LIKE '%record_file%';
# Each concurrent request maps to a live DB query against record_file and address_book_entry tables.
```

Each iteration of the loop causes the server to execute `SELECT ... FROM record_file ORDER BY index DESC LIMIT 1` and `SELECT ... FROM address_book_entry ...` with no deduplication or caching, confirming linear DB read amplification proportional to request rate.

func (n *networkAPIService) NetworkStatus(
	ctx context.Context,
	_ *rTypes.NetworkRequest,
) (*rTypes.NetworkStatusResponse, *rTypes.Error) {
	if !n.IsOnline() {
		return nil, errors.ErrEndpointNotSupportedInOfflineMode
	}

	genesisBlock, err := n.RetrieveGenesis(ctx)
	if err != nil {
		return nil, err
	}

	currentBlock, err := n.RetrieveLatest(ctx)
	if err != nil {
		return nil, err
	}

	peers, err := n.addressBookEntryRepo.Entries(ctx)
	if err != nil {
		return nil, err
	}

	return &rTypes.NetworkStatusResponse{
		CurrentBlockIdentifier: currentBlock.GetRosettaBlockIdentifier(),
		CurrentBlockTimestamp:  currentBlock.GetTimestampMillis(),
		GenesisBlockIdentifier: genesisBlock.GetRosettaBlockIdentifier(),
		Peers:                  peers.ToRosetta(),
	}, nil
}

func (br *blockRepository) RetrieveLatest(ctx context.Context) (*types.Block, *rTypes.Error) {
	if err := br.initGenesisRecordFile(ctx); err != nil {
		return nil, err
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	rb := &recordBlock{}
	if err := db.Raw(selectLatestWithIndex).First(rb).Error; err != nil {
		return nil, handleDatabaseError(err, hErrors.ErrBlockNotFound)
	}

	if rb.Index < br.genesisBlock.Index {
		return nil, hErrors.ErrBlockNotFound
	}

	return rb.ToBlock(br.genesisBlock), nil
}

func (br *blockRepository) initGenesisRecordFile(ctx context.Context) *rTypes.Error {
	if br.genesisBlock.ConsensusStart != genesisConsensusStartUnset {
		return nil
	}

	db, cancel := br.dbClient.GetDbWithContext(ctx)
	defer cancel()

	var rb recordBlock
	if err := db.Raw(selectGenesis, sql.Named("treasury_entity_id", br.treasuryEntityId.EncodedId)).
		First(&rb).Error; err != nil {
		return handleDatabaseError(err, hErrors.ErrNodeIsStarting)
	}

	br.once.Do(func() {
		br.genesisBlock = rb
	})

	log.Infof("Fetched genesis record file, index - %d", br.genesisBlock.Index)
	return nil
}

func (aber *addressBookEntryRepository) Entries(ctx context.Context) (*types.AddressBookEntries, *rTypes.Error) {
	db, cancel := aber.dbClient.GetDbWithContext(ctx)
	defer cancel()

	nodes := make([]nodeServiceEndpoint, 0)
	// address book file 101 has service endpoints for nodes, resort to file 102 if 101 doesn't exist
	for _, fileId := range []int64{aber.addressBook101.EncodedId, aber.addressBook102.EncodedId} {
		if err := db.Raw(
			latestNodeServiceEndpoints,
			sql.Named("file_id", fileId),
		).Scan(&nodes).Error; err != nil {
			log.Error("Failed to get latest node service endpoints", err)
			return nil, errors.ErrDatabaseError
		}

		if len(nodes) != 0 {
			break
		}
	}

	entries := make([]types.AddressBookEntry, 0, len(nodes))
	for _, node := range nodes {
		entries = append(entries, node.toAddressBookEntry())
	}

	return &types.AddressBookEntries{Entries: entries}, nil
}
