### Title
Unauthenticated, Unbounded Database Query on `/api/v1/network/stake` Enables DoS via DB Resource Exhaustion

### Summary
The `GET /api/v1/network/stake` endpoint in the `rest-java` module invokes `getLatestNetworkStake()`, which executes a correlated subquery against the database on every request with no caching and no rate limiting. Any unauthenticated external user can flood this endpoint to exhaust database connection pool resources, making stake data unavailable to clients that use it for weighted gossip node selection.

### Finding Description
**Exact code path:**

`NetworkController.getNetworkStake()` (lines 126–130) calls `networkService.getLatestNetworkStake()` with no throttle guard: [1](#0-0) 

`NetworkServiceImpl.getLatestNetworkStake()` (lines 52–56) directly delegates to the repository with no `@Cacheable` annotation: [2](#0-1) 

`NetworkStakeRepository.findLatest()` executes a correlated subquery on every call — no `@Cacheable` annotation is present: [3](#0-2) 

**Root cause:** The `rest-java` module has no rate-limiting infrastructure applied to this endpoint. The throttle system (`ThrottleConfiguration`, `ThrottleManagerImpl`) exists exclusively in the `web3` module and is scoped to `ContractCallRequest` processing: [4](#0-3) [5](#0-4) 

The `rest-java` config directory contains no equivalent rate-limiting or caching configuration for the network stake endpoint:


**Failed assumption:** The design assumes that infrastructure-level controls (e.g., a reverse proxy) will throttle this endpoint, but no such control is enforced in application code, and the `network_stake` query hits the DB on every request.

### Impact Explanation
Each request to `GET /api/v1/network/stake` issues a correlated subquery (`SELECT * FROM network_stake WHERE consensus_timestamp = (SELECT MAX(...) FROM network_stake)`). Under high concurrency, this exhausts the database connection pool, causing all DB-dependent endpoints to fail. Clients (including the Hiero monitor, which uses this endpoint for health checks) rely on stake data to perform weighted gossip node selection. If the endpoint is unavailable, clients fall back to unweighted or stale selection, degrading the quality of gossip node targeting. Severity: **Medium–High** (availability impact on a public, unauthenticated endpoint with secondary protocol effect). [6](#0-5) 

### Likelihood Explanation
Preconditions: none — the endpoint is public and unauthenticated. A single attacker with a modest botnet or even a single machine using concurrent HTTP clients (e.g., `ab`, `wrk`, or a simple script) can saturate the DB connection pool. The attack is trivially repeatable and requires no special knowledge of the system.

### Recommendation
1. **Add caching** to `NetworkStakeRepository.findLatest()` with a short TTL (e.g., 60 seconds) using `@Cacheable`, mirroring the pattern used in the `grpc` module's `NodeStakeRepository`: [7](#0-6) 
2. **Add rate limiting** to the `rest-java` module for public read endpoints, analogous to the `web3` throttle infrastructure.
3. Alternatively, apply a per-IP rate limit at the reverse proxy layer and document it as a required deployment constraint.

### Proof of Concept
```bash
# Flood the endpoint with 1000 concurrent requests
ab -n 100000 -c 1000 http://<mirror-node-host>/api/v1/network/stake

# Or with wrk
wrk -t 16 -c 500 -d 60s http://<mirror-node-host>/api/v1/network/stake
```
Expected result: database connection pool exhaustion, subsequent requests to all DB-backed endpoints begin returning 500 errors, and the `/api/v1/network/stake` endpoint becomes unavailable to legitimate clients.

@Configuration(proxyBeanMethods = false)
@RequiredArgsConstructor
public class ThrottleConfiguration {

    public static final String GAS_LIMIT_BUCKET = "gasLimitBucket";
    public static final String RATE_LIMIT_BUCKET = "rateLimitBucket";
    public static final String OPCODE_RATE_LIMIT_BUCKET = "opcodeRateLimitBucket";

    private final ThrottleProperties throttleProperties;

    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    @Bean(name = GAS_LIMIT_BUCKET)
    Bucket gasLimitBucket() {
        long gasLimit = throttleProperties.getGasPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(gasLimit)
                .refillGreedy(gasLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder()
                .withSynchronizationStrategy(SynchronizationStrategy.SYNCHRONIZED)
                .addLimit(limit)
                .build();
    }

    @Bean(name = OPCODE_RATE_LIMIT_BUCKET)
    Bucket opcodeRateLimitBucket() {
        long rateLimit = throttleProperties.getOpcodeRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    public void throttle(ContractCallRequest request) {
        if (!rateLimitBucket.tryConsume(1)) {
            throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
        } else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
            throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
        }

        for (var requestFilter : throttleProperties.getRequest()) {
            if (requestFilter.test(request)) {
                action(requestFilter, request);
            }
        }
    }
