### Title
Prometheus Label Cardinality DoS via Unbounded `req.path` in `toOpenApiPath()`

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to the raw `req.path` value when no route matches (`req.route` is null) and no pre-set `res.locals[requestPathLabel]` exists. This user-controlled string is then used as the `path` label on four per-route Prometheus instruments with no length cap, sanitization, or cardinality guard. An unauthenticated attacker can flood the service with requests to unique paths, creating an unbounded number of Prometheus time series and causing a cardinality-based denial of service against the monitoring stack.

### Finding Description

**Exact code path:**

`rest/middleware/metricsHandler.js`, `toOpenApiPath()`, lines 13–31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // only set by matched route handlers

  if (!path) {
    if (!req.route) {
      path = req.path;                       // ← raw user input, no sanitization
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }

  path = path.replace(/:([^/]+)/g, '{$1}'); // only normalises :param patterns
  ...
  return path;
};
```

This path is then used as a label on all four per-route instruments at lines 207–210:

```js
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
```

**Root cause and failed assumption:**

The code assumes `req.route` will be non-null for any request that reaches the `res.on('finish', ...)` callback. This is false. For unmatched routes, Express never sets `req.route`; `responseHandler.js` (line 28–31) throws a `NotFoundError` which is handled by an error middleware that sends the 404 — all without ever setting `req.route` or `res.locals[requestPathLabel]`. The fallback `path = req.path` is therefore reachable by any unauthenticated request to a non-existent path.

**Why existing checks fail:**

- `res.locals[requestPathLabel]` is only populated by `recordRequestPath` in `rest/routes/index.js` (lines 16–21), which itself requires `req.route?.path` to be truthy — impossible for unmatched routes.
- The regex `/:([^/]+)/g` only transforms `:param` patterns; it does not truncate or normalise arbitrary strings.
- No rate limiting exists on the REST API (confirmed: grep for `rateLimit`, `express-rate`, `throttle` returns only a test-file hit).
- No cardinality cap is configured on the OpenTelemetry `MeterProvider` or `PrometheusExporter` (lines 62–64).

### Impact Explanation

Each unique `path` label value creates a new time series in every one of the four per-route instruments (`api_request_total`, `api_request_duration_milliseconds`, `api_request_size_bytes`, `api_response_size_bytes`). Prometheus stores all active time series in memory. Injecting millions of unique label combinations causes Prometheus memory exhaustion (OOM kill) or severe performance degradation, destroying all monitoring visibility — including alerting on transaction authorization failures, request latency, and error rates. The Grafana dashboards that query `api_request_total{...} by (path)` (e.g., `hedera-mirror-rest.json` line 614) become unusable.

### Likelihood Explanation

The attack requires zero authentication, zero privileges, and zero knowledge of the application internals. A single script sending HTTP GET requests to `/api/v1/<random_uuid>` at modest throughput (e.g., 1 000 req/s) generates 1 000 new label combinations per second. No existing network-layer rate limiting is present in the application code. The attack is trivially repeatable and automatable.

### Recommendation

1. **Normalize unmatched paths to a fixed sentinel label** — in `toOpenApiPath()`, replace the `path = req.path` fallback with a constant such as `path = 'unknown'` or `path = apiPrefix + '/unknown'`.
2. **Enforce a path length cap** before using any path as a label (e.g., truncate to 128 characters).
3. **Add application-level rate limiting** (e.g., `express-rate-limit`) to bound the request rate per IP.
4. **Configure a Prometheus cardinality limit** via relabeling rules in the `ServiceMonitor`/scrape config to drop or cap time series exceeding a threshold on the `path` label.

### Proof of Concept

```bash
# Send 100 000 requests to unique paths (no auth required)
for i in $(seq 1 100000); do
  curl -s "http://<mirror-node-rest>/api/v1/$(uuidgen)" &
done
wait

# Observe Prometheus memory growth:
curl http://<prometheus>/api/v1/query?query=count({__name__=~"api_request.*"})
# Returns an ever-growing cardinality count; Prometheus eventually OOMs or scrape times out
```
