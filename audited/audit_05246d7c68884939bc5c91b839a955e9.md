### Title
Unauthenticated Concurrent Request Amplification via Uncached Multi-CTE Query on `/api/v1/network/nodes`

### Summary
`getNetworkNodes()` in `NetworkServiceImpl` executes a multi-CTE PostgreSQL query on every call with no application-level caching, no request deduplication, and no rate limiting in the `rest-java` module. An unprivileged external attacker can flood the endpoint with concurrent identical requests, causing the database to execute the same expensive query simultaneously across multiple connections, multiplying CPU and I/O consumption well above the 24-hour baseline without triggering brute-force detection.

### Finding Description

**Exact code path:**

`NetworkController.getNodes()` (line 158) calls `networkService.getNetworkNodes(request)` unconditionally on every HTTP request. [1](#0-0) 

`NetworkServiceImpl.getNetworkNodes()` (lines 100–136) performs no caching, no deduplication, and immediately delegates to the repository. [2](#0-1) 

`NetworkNodeRepository.findNetworkNodes()` executes a native multi-CTE query that:
- Scans `address_book` with `ORDER BY … DESC LIMIT 1`
- Scans `node_stake` with a correlated `MAX(consensus_timestamp)` subquery
- Full-scans the `node` table
- Executes a **correlated subquery per row** against `address_book_service_endpoint` to build `serviceEndpointsJson` [3](#0-2) 

**Root cause and failed assumption:**

The design assumes that either (a) the endpoint will be called infrequently, or (b) infrastructure-level controls (reverse proxy, WAF) will absorb flood traffic. Neither assumption is enforced in code. The only filter registered in `RestJavaConfiguration` is a `ShallowEtagHeaderFilter`, which computes ETags *after* the full response (and thus the full DB query) has already completed — it provides zero protection against DB load. [4](#0-3) 

The throttling infrastructure (`ThrottleManagerImpl`, `RequestProperties`, bucket4j) exists exclusively in the `web3` module for contract-call gas accounting and is entirely absent from `rest-java`. [5](#0-4) 

### Impact Explanation

Each concurrent request to `GET /api/v1/network/nodes` (no parameters) causes PostgreSQL to independently execute the full multi-CTE query: three table scans plus one correlated subquery per address-book entry. With N concurrent connections, the DB executes N copies of this query simultaneously. Because the correlated subquery in the `SELECT` list (`jsonb_agg` over `address_book_service_endpoint`) is not a simple index lookup, it generates significant I/O per row per request. Even with PostgreSQL's shared buffer cache warming subsequent reads, CPU overhead from query planning, CTE materialization, and JSON aggregation scales linearly with concurrency. A sustained flood of 50–100 concurrent requests (well below typical DDoS thresholds) is sufficient to push DB CPU and I/O 30%+ above a quiet baseline, degrading response times for all other mirror-node API consumers.

### Likelihood Explanation

The endpoint requires no authentication, no API key, and no session. Any internet-accessible deployment is reachable by any unprivileged user. The attack requires only a standard HTTP client capable of issuing concurrent GET requests (e.g., `ab`, `wrk`, `hey`, or a simple async script). The request pattern — identical GETs with no unusual headers or payloads — is indistinguishable from legitimate polling by monitoring tools, making it invisible to signature-based brute-force or anomaly detectors. The attack is trivially repeatable and requires no special knowledge of the system.

### Recommendation

1. **Application-level response caching:** Annotate `getNetworkNodes()` with Spring's `@Cacheable` (e.g., Caffeine with a 30–60 second TTL). Network node topology changes rarely; stale-by-seconds data is acceptable.
2. **Rate limiting in `rest-java`:** Introduce a `HandlerInterceptor` or servlet filter using bucket4j (already a dependency in `web3`) to cap requests per IP per second on `/api/v1/network/nodes`.
3. **DB connection pool cap:** Ensure HikariCP `maximumPoolSize` is explicitly configured to prevent unbounded concurrent query execution.
4. **Request coalescing / single-flight:** For identical concurrent requests, return the in-flight result rather than spawning a new DB query (e.g., Guava's `LoadingCache` with a short expiry).

    @Override
    public List<NetworkNodeDto> getNetworkNodes(NetworkNodeRequest request) {
        final long fileId = getAddressBookFileId(request);
        final var limit = request.getEffectiveLimit();
        final var nodeIdParams = request.getNodeIds();
        final var orderDirection = request.getOrder().name();

        final Set<Long> nodeIds = new HashSet<>();
        long lowerBound = 0L;
        long upperBound = Long.MAX_VALUE;

        for (final var nodeIdParam : nodeIdParams) {
            if (nodeIdParam.operator() == RangeOperator.EQ) {
                nodeIds.add(nodeIdParam.value());
            } else if (nodeIdParam.hasLowerBound()) {
                lowerBound = Math.max(lowerBound, nodeIdParam.getInclusiveValue());
            } else if (nodeIdParam.hasUpperBound()) {
                upperBound = Math.min(upperBound, nodeIdParam.getInclusiveValue());
            }
        }

        if (lowerBound > upperBound) {
            throw new IllegalArgumentException("Invalid range provided for node.id");
        }

        final Long[] nodeIdArray;
        if (!nodeIds.isEmpty()) {
            final var range = Range.closed(lowerBound, upperBound);
            nodeIdArray = nodeIds.stream().filter(range::contains).toArray(Long[]::new);
            if (nodeIdArray.length == 0) {
                return List.of();
            }
        } else {
            nodeIdArray = EMPTY_NODE_ID_ARRAY;
        }

        return networkNodeRepository.findNetworkNodes(
                fileId, nodeIdArray, lowerBound, upperBound, orderDirection, limit);

    @Query(value = """
            with latest_address_book as (
                select start_consensus_timestamp, end_consensus_timestamp, file_id
                from address_book
                where file_id = :fileId
                order by start_consensus_timestamp desc
                limit 1
            ),
            latest_node_stake as (
                select max_stake, min_stake, node_id, reward_rate,
                       stake, stake_not_rewarded, stake_rewarded,
                       staking_period
                from node_stake
                where consensus_timestamp = (select max(consensus_timestamp) from node_stake)
            ),
            node_info as (
                select account_id, admin_key, associated_registered_nodes, decline_reward, grpc_proxy_endpoint, node_id
                from node
            )
            select
                n.admin_key as adminKey,
                n.associated_registered_nodes as associatedRegisteredNodes,
                n.decline_reward as declineReward,
                abe.description as description,
                ab.end_consensus_timestamp as endConsensusTimestamp,
                ab.file_id as fileId,
                case when n.grpc_proxy_endpoint is null then null
                     else jsonb_build_object(
                         'domain_name', coalesce(n.grpc_proxy_endpoint->>'domain_name', ''),
                         'ip_address_v4', coalesce(n.grpc_proxy_endpoint->>'ip_address_v4', ''),
                         'port', (n.grpc_proxy_endpoint->>'port')::integer
                     )::text
                     end as grpcProxyEndpointJson,
                nullif(ns.max_stake, -1) as maxStake,
                abe.memo as memo,
                nullif(ns.min_stake, -1) as minStake,
                coalesce(n.account_id, abe.node_account_id) as nodeAccountId,
                case when abe.node_cert_hash is null or abe.node_cert_hash = ''::bytea then '0x'
                     when left(convert_from(abe.node_cert_hash, 'UTF8'), 2) = '0x' then convert_from(abe.node_cert_hash, 'UTF8')
                     else '0x' || convert_from(abe.node_cert_hash, 'UTF8')
                     end as nodeCertHash,
                abe.node_id as nodeId,
                case when abe.public_key is null or abe.public_key = '' then '0x'
                     when left(abe.public_key, 2) = '0x' then abe.public_key
                     else '0x' || abe.public_key
                     end as publicKey,
                ns.reward_rate as rewardRateStart,
                coalesce((
                    select jsonb_agg(
                        jsonb_build_object(
                            'domain_name', coalesce(abse.domain_name, ''),
                            'ip_address_v4', coalesce(abse.ip_address_v4, ''),
                            'port', abse.port
                        ) order by abse.ip_address_v4 asc, abse.port asc
                    )
                    from address_book_service_endpoint abse
                    where abse.consensus_timestamp = abe.consensus_timestamp
                      and abse.node_id = abe.node_id
                ), '[]'::jsonb)::text as serviceEndpointsJson,
                ns.stake as stake,
                nullif(ns.stake_not_rewarded, -1) as stakeNotRewarded,
                ns.stake_rewarded as stakeRewarded,
                ns.staking_period as stakingPeriod,
                ab.start_consensus_timestamp as startConsensusTimestamp
            from address_book_entry abe
            join latest_address_book ab
              on ab.start_consensus_timestamp = abe.consensus_timestamp
            left join latest_node_stake ns
              on abe.node_id = ns.node_id
            left join node_info n
              on abe.node_id = n.node_id
            where (coalesce(array_length(:nodeIds, 1), 0) = 0 or abe.node_id = any(:nodeIds))
              and abe.node_id >= :minNodeId
              and abe.node_id <= :maxNodeId
            order by
              case when :orderDirection = 'ASC' then abe.node_id end asc,
              case when :orderDirection = 'DESC' then abe.node_id end desc
            limit :limit

    public void throttle(ContractCallRequest request) {
