### Title
Unauthenticated Mirror REST Response Allows Attacker-Controlled Node Injection via Semantically Unchecked Account IDs

### Summary
`fetchMirrorNodeNetwork()` in `pinger/mirror_node_client.go` fetches network topology from an operator-configured mirror REST URL over a plain, unauthenticated HTTP client. The function filters only empty and syntactically unparseable `NodeAccountID` values; any syntactically valid account ID (e.g., `0.0.999`) paired with attacker-controlled `service_endpoints` passes all checks and is inserted directly into the network map handed to `hiero.ClientForNetwork()`. An attacker who can influence the mirror REST response — via MITM on an HTTP deployment, DNS poisoning, or server compromise — can silently redirect all pinger gRPC traffic to attacker infrastructure.

### Finding Description
**Exact code path:**

`pinger/sdk_client.go` line 18 calls `buildNetworkFromMirrorNodes()` only when `cfg.network == "other"`. That function calls `fetchMirrorNodeNetwork()` (`pinger/mirror_node_client.go` lines 74–132).

Inside `fetchMirrorNodeNetwork()`:
- Line 103–105: skips entries where `NodeAccountID == ""`
- Line 107–110: skips entries where `AccountIDFromString()` returns an error
- Lines 113–124: for every remaining entry, iterates `ServiceEndpoints` and inserts `addr → nodeAccountId` into the `network` map with **no further validation**

The resulting map is returned to `buildNetworkFromMirrorNodes()` and passed verbatim to `hiero.ClientForNetwork(netmap)` (sdk_client.go line 22), which the SDK uses as the authoritative node roster for all subsequent gRPC calls.

**Root cause:** The two `continue` guards are purely syntactic. There is no semantic check — no comparison against a known-good node list, no on-chain verification, no TLS pinning, no HMAC/signature on the REST response. Any syntactically valid account ID such as `0.0.3` paired with `"ip_address_v4": "198.51.100.1", "port": 50211` passes both guards and is accepted as a legitimate consensus node.

**HTTP client construction** (mirror_node_client.go line 46):
```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
```
No TLS configuration, no certificate pinning, no authentication header. The config example (`http://mirror-rest:5551`) uses plain HTTP.

**Exploit flow:**
1. Attacker positions themselves to MITM the HTTP connection (ARP spoof, DNS hijack, BGP prefix hijack, or rogue DNS for the mirror hostname).
2. Attacker intercepts the GET `/api/v1/network/nodes` response and replaces it with a crafted JSON body:
```json
{
  "nodes": [{
    "node_account_id": "0.0.3",
    "service_endpoints": [{"ip_address_v4": "198.51.100.1", "port": 50211}]
  }],
  "links": {"next": null}
}
```
3. `AccountIDFromString("0.0.3")` succeeds; the entry is inserted into `network`.
4. `hiero.ClientForNetwork(network)` routes all pinger gRPC calls to `198.51.100.1:50211`.
5. The pinger's `submitWithRetry()` sends signed `CryptoTransfer` transactions to the attacker's endpoint.

### Impact Explanation
The attacker's gRPC endpoint receives every signed transaction the pinger emits, exposing: operator account ID, destination account, transfer amount, and transaction timing. The attacker can silently drop all transactions, causing the pinger to log continuous failures and triggering false monitoring alerts — disrupting the health-check function the pinger is designed to provide. Because the transactions are signed with the real operator key and submitted to a fake node, they never reach the real network; funds are not transferred but the monitoring signal is completely subverted. This matches the stated impact scope: reorganization/disruption of transaction history without direct fund theft.

### Likelihood Explanation
The attack is feasible for any network-adjacent unprivileged attacker. The code explicitly supports HTTP URLs (the documented example is `http://mirror-rest:5551`), making passive MITM trivial on shared infrastructure (cloud VPC, Kubernetes cluster-internal DNS, container network). DNS poisoning requires no special privilege. The attack is repeatable on every pinger startup (client is built once at startup, mirror REST is queried once) and requires no interaction with the pinger operator.

### Recommendation
1. **Enforce HTTPS and validate the server certificate** — reject `http://` mirror REST URLs at config load time; use a TLS-aware `http.Client` with a pinned CA or system roots.
2. **Validate returned node account IDs against a trusted set** — maintain a configurable allowlist of expected node account IDs (e.g., `0.0.3`, `0.0.4`, …) and reject any entry not in the list.
3. **Add response authentication** — require the mirror REST endpoint to sign its `/network/nodes` response, or compare the returned topology against a secondary trusted source before use.
4. **Fail closed** — if any returned node entry fails semantic validation, reject the entire response rather than silently skipping the entry.

### Proof of Concept
**Preconditions:** Pinger deployed with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551`. Attacker has DNS control or network-level MITM capability.

**Steps:**
1. Stand up a fake gRPC server on `198.51.100.1:50211` that accepts `CryptoTransfer` RPCs and logs all received transactions.
2. Intercept DNS for `mirror-rest` and resolve it to an attacker-controlled HTTP server.
3. Serve the following from `GET /api/v1/network/nodes`:
```json
{"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"ip_address_v4":"198.51.100.1","port":50211}]}],"links":{"next":null}}
```
4. Start (or restart) the pinger.
5. `fetchMirrorNodeNetwork()` parses `"0.0.3"` successfully (passes both `continue` guards), inserts `198.51.100.1:50211 → AccountID{0,0,3}` into the network map.
6. `hiero.ClientForNetwork()` uses this map; all subsequent `CryptoTransfer.Execute()` calls in `submitWithRetry()` are sent to the attacker's gRPC server.
7. Observe signed transactions arriving at the fake node; pinger logs show failures (no valid receipt returned), triggering monitoring alerts. [1](#0-0) [2](#0-1) [3](#0-2)
