### Title
Prometheus Label Cardinality Injection via Unsanitized `req.path` on Unmatched Routes

### Summary
In `rest/middleware/metricsHandler.js`, the `toOpenApiPath()` function falls back to using the raw `req.path` as a Prometheus label value when no Express route is matched (`req.route` is `undefined`). Because any unprivileged external user can send requests to arbitrary unregistered paths, an attacker can generate an unbounded number of unique Prometheus time series, exhausting Prometheus memory and polluting operator dashboards with crafted paths that mimic legitimate transaction history routes.

### Finding Description
**Exact code path:**

`rest/middleware/metricsHandler.js`, `toOpenApiPath()`, lines 13–31:

```js
const toOpenApiPath = (req, res) => {
  let path = res.locals[requestPathLabel];   // only set for matched routes

  if (!path) {
    if (!req.route) {
      path = req.path;                       // LINE 18: raw attacker-controlled value
    } else {
      path = (req.baseUrl ?? '') + req.route?.path;
    }
  }

  path = path.replace(/:([^/]+)/g, '{$1}'); // only normalizes :param patterns
  ...
  return path;
};
```

This `path` value is then used directly as a Prometheus label at lines 192 and 207–210:

```js
const labels = {method, path, code};
requestTotalCounter.add(1, labels);
durationHistogram.record(duration, labels);
requestSizeHistogram.record(..., labels);
responseSizeHistogram.record(responseSize, labels);
```

**Root cause:** The fallback at line 18 assumes that unmatched-route requests are rare or benign. There is no allowlist, no normalization to a fixed sentinel value (e.g., `"unknown"`), no path length cap, and no cardinality guard.

**Why existing checks fail:**
- `res.locals[requestPathLabel]` is only populated by `recordRequestPath` in `rest/routes/index.js` (line 19), which only runs when `req.route?.path` exists — i.e., only for matched routes. For 404 paths it is always empty.
- The `replace(/:([^/]+)/g, '{$1}')` regex only converts Express `:param` tokens to `{param}` style. It does not normalize arbitrary path segments like `0.0.1234` or `fake`.
- The metrics middleware is registered unconditionally before route handlers (`rest/server.js` line 91), so every request — including 404s — passes through it and triggers the `res.on('finish', ...)` label recording.

### Impact Explanation
Each unique `path` label value creates a new time series in every per-route Prometheus instrument (`api_request_total`, `api_request_duration_milliseconds`, `api_request_size_bytes`, `api_response_size_bytes`). Prometheus stores all active time series in memory. Flooding with N unique paths creates 4×N new time series, leading to:
1. **Prometheus OOM / scrape failure / DoS** — the monitoring plane becomes unavailable.
2. **Dashboard pollution** — Grafana panels (e.g., "Request Rate by Path") display attacker-crafted paths such as `/api/v1/transactions/0.0.1234/fake` alongside legitimate routes, misleading operators about which transaction history endpoints were actually accessed and masking real anomalies.

### Likelihood Explanation
The attack requires zero privileges — only the ability to send HTTP GET requests to the public API. No authentication, no special headers, no account needed. The exploit is trivially scriptable: a loop sending requests to `/api/v1/transactions/<incrementing-id>/fake` is sufficient. It is repeatable indefinitely and can be sustained at low request rates to avoid rate-limit triggers while still accumulating thousands of unique time series over time.

### Recommendation
Replace the raw `req.path` fallback with a fixed sentinel string so unmatched routes do not contribute unique label values:

```js
if (!req.route) {
  path = 'unknown';   // or apiPrefix + '/unknown'
} else {
  path = (req.baseUrl ?? '') + req.route?.path;
}
```

Alternatively, maintain an explicit allowlist of known OpenAPI paths and map any unrecognized path to `"unknown"`. Additionally, consider enforcing a Prometheus cardinality cap at the SDK or scrape level as a defense-in-depth measure.

### Proof of Concept
```bash
# Send 10,000 requests to unique unregistered paths (no auth required)
for i in $(seq 1 10000); do
  curl -s "https://<mirror-node-host>/api/v1/transactions/0.0.${i}/fake" &
done
wait

# Scrape Prometheus metrics and observe unbounded unique 'path' label values:
curl http://<mirror-node-host>/swagger/metrics/ | grep 'api_request_total' | grep 'fake'
# Output: thousands of unique time series like:
# api_request_total{method="GET",path="/api/v1/transactions/0.0.1/fake",code="404"} 1
# api_request_total{method="GET",path="/api/v1/transactions/0.0.2/fake",code="404"} 1
# ...
```

Each iteration adds 4 new time series (one per histogram/counter instrument). At 10,000 unique paths that is 40,000 new time series injected by an unprivileged attacker.
