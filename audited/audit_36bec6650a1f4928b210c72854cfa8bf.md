### Title
Log Path Mismatch via Percent-Encoded URI and Query-String Bypass of `internalPaths` Check in `TracingMiddleware`

### Summary
`TracingMiddleware` captures the log path using `request.URL.RequestURI()` (line 47), which returns the raw, percent-encoded URI including any query string. Go's HTTP mux routes requests based on the decoded `request.URL.Path`. This creates a persistent discrepancy between the path recorded in the audit log and the handler that actually processed the request. Additionally, the `internalPaths` map lookup at line 55 uses the raw `RequestURI()` value, so any request to an internal path with an appended query string bypasses the Debug-level suppression and is promoted to Info-level logging.

### Finding Description
**Code location:** `rosetta/app/middleware/trace.go`, lines 47 and 55.

```
path := request.URL.RequestURI()   // line 47 — raw, percent-encoded, includes query string
...
if internalPaths[path] {           // line 55 — exact-string map lookup
```

`internalPaths` is defined as:
```go
var internalPaths = map[string]bool{livenessPath: true, metricsPath: true, readinessPath: true}
// keys: "/health/liveness", "/health/readiness", "/metrics"
```

**Root cause — two related issues:**

1. **Percent-encoded path mismatch.** Go's `net/http` server populates `URL.Path` with the decoded, cleaned path (used by gorilla/mux for routing) and `URL.RawPath` with the original encoded form. `RequestURI()` returns `RawPath` when it is set and valid. A request for `/netw%6Frk/list` (`%6F` = `o`) is routed to the `/network/list` handler, but the log records `/netw%6Frk/list`. The audit trail does not reflect the handler that ran.

2. **`internalPaths` bypass via query string.** `RequestURI()` appends the raw query string. A request to `/health/liveness?x=1` is routed to the liveness handler, but `internalPaths["/health/liveness?x=1"]` evaluates to `false`, so the entry is emitted at `log.Info` instead of `log.Debug`. Any unprivileged caller can force health-check traffic into the Info log stream.

**Why existing checks are insufficient:** The `internalPaths` map performs a verbatim string comparison against the raw URI. No normalization (query-string stripping, percent-decoding) is applied before the lookup. The test suite (`trace_test.go` lines 26–85) only exercises exact, unencoded paths and never tests encoded variants or paths with query strings, so the gap is untested.

### Impact Explanation
- **Audit trail integrity:** Operators and security tooling that parse the Info log to reconstruct which Rosetta API endpoints were called will see obfuscated or incorrect paths. An attacker can make repeated calls to `/network/list` appear as calls to `/netw%6Frk/list`, defeating pattern-based alerting.
- **Log-level manipulation:** By appending a query string to `/health/liveness`, `/health/readiness`, or `/metrics`, an attacker can force those requests into the Info log, inflating log volume and potentially masking real traffic in noisy environments.
- Severity is **low-medium**: no authentication bypass or data exposure occurs, but the audit trail — the primary forensic artifact for this middleware — is unreliable.

### Likelihood Explanation
No privileges are required. Any HTTP client can send a percent-encoded path or append a query string. The technique is trivially repeatable, requires no special tooling, and is invisible to the application (the handler still executes normally). The Rosetta API is exposed on a public port (default 5700).

### Recommendation
1. **Normalize the logged path before use.** Replace `request.URL.RequestURI()` with `request.URL.Path` (already decoded and cleaned by Go's HTTP server) for the log message and the `internalPaths` lookup. If the query string must be logged, append it separately after the map check:
   ```go
   path := request.URL.Path          // decoded, cleaned, no query string
   rawURI := request.URL.RequestURI() // for the log message only
   ...
   if internalPaths[path] {
       log.Debug(...)
   } else {
       log.Info(...)
   }
   ```
2. **Strip query strings from the `internalPaths` lookup** even if `RequestURI()` is kept for the log message.
3. **Add test cases** for percent-encoded paths and paths with query strings to `trace_test.go`.

### Proof of Concept
```bash
# 1. Force a health-check request into Info-level logs (internalPaths bypass)
curl -s "http://<rosetta-host>:5700/health/liveness?x=1"
# Expected: log entry at level=info instead of level=debug

# 2. Log path mismatch via percent-encoding (%6F = 'o')
curl -s -X POST "http://<rosetta-host>:5700/netw%6Frk/list" \
     -H "Content-Type: application/json" \
     -d '{"metadata":{}}'
# Expected: request routed to /network/list handler (HTTP 200),
