### Title
Unauthenticated `/metrics` Endpoint Exposes Per-Route Request Telemetry in Rosetta Service

### Summary
The Rosetta service registers the `/metrics` endpoint via `promhttp.Handler()` with no authentication guard, making the full Prometheus exposition — including per-route histogram `_count` fields labeled by `route` — accessible to any HTTP client that can reach the service. This allows an unprivileged user to enumerate exactly which API routes are active, their request volumes, durations, and payload sizes, enabling precise attack surface mapping.

### Finding Description
In `rosetta/app/middleware/metrics.go`, the `Routes()` function registers the `/metrics` path directly with the raw `promhttp.Handler()`:

```go
// rosetta/app/middleware/metrics.go, lines 64-73
func (c *metricsController) Routes() server.Routes {
    return server.Routes{
        {
            "metrics",
            "GET",
            metricsPath,
            promhttp.Handler().ServeHTTP,  // no auth wrapper
        },
    }
}
``` [1](#0-0) 

The four registered histograms all carry a `route` label:
- `hiero_mirror_rosetta_request_duration` (`method`, `route`, `status_code`, `ws`)
- `hiero_mirror_rosetta_request_bytes` (`method`, `route`)
- `hiero_mirror_rosetta_response_bytes` (`method`, `route`)
- `hiero_mirror_rosetta_request_inflight` (`method`, `route`) [2](#0-1) 

In `rosetta/main.go`, the middleware chain applied is `MetricsMiddleware → TracingMiddleware → CorsMiddleware` — none of which add authentication to the `/metrics` route: [3](#0-2) 

The `metricsController` is included in both online and offline routers with no conditional access control: [4](#0-3) 

**Contrast with the REST service**, which explicitly implements an `authenticate()` check before serving metrics: [5](#0-4) 

The rosetta service has no equivalent guard.

### Impact Explanation
A successful `GET /metrics` response exposes:
- Exact `_count` per `{route, method, status_code}` combination — reveals which endpoints have ever been called and how many times
- `_sum` and bucket distributions — reveals response time and payload size profiles per route
- Inflight gauge — reveals current load distribution

This gives an attacker a precise map of active vs. dormant routes, request rates, and timing characteristics, directly aiding targeted exploitation (e.g., focusing on high-traffic routes for DoS, or identifying rarely-monitored routes for stealthy abuse).

### Likelihood Explanation
The Kubernetes `values.yaml` defaults to `ClusterIP`, but:
1. Any ingress rule or `NodePort`/`LoadBalancer` override exposes the port publicly
2. Any attacker or compromised workload inside the cluster can reach `ClusterIP` services without restriction
3. The exploit requires a single unauthenticated HTTP GET — zero preconditions, fully repeatable [6](#0-5) 

### Recommendation
Wrap `promhttp.Handler()` with an authentication middleware before registering it, mirroring the pattern already used in the REST service (`rest/middleware/metricsHandler.js` lines 137–144). Concretely:

1. Add a configurable `metrics.authentication` flag to the Rosetta config (analogous to the REST config).
2. Create an auth-wrapping handler that checks HTTP Basic credentials before delegating to `promhttp.Handler()`.
3. Replace the bare `promhttp.Handler().ServeHTTP` in `Routes()` with the wrapped handler.

Alternatively, if metrics are only intended for internal scraping (e.g., Prometheus in-cluster), restrict access at the network layer (NetworkPolicy) and document that the endpoint must not be exposed externally.

### Proof of Concept

const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;
  }
  const credentials = basicAuth(req);
  return credentials && tsscmp(credentials.name, username) && tsscmp(credentials.pass, password);
};

const metricsHandler = () => {
  initMetrics();
  const serializer = new PrometheusSerializer();
  // Maintain backward-compatible path: uriPath="/swagger" → endpoint at /swagger/metrics/
  const metricsPath = `${config.metrics.config.uriPath}/metrics/`;

  return function metricsMiddleware(req, res, next) {
    const {pathname} = new URL(req.url, 'http://localhost');
    const normalizedPath = pathname.endsWith('/') ? pathname : `${pathname}/`;

    if (normalizedPath === metricsPath) {
      if (!authenticate(req)) {
        res.set('WWW-Authenticate', 'Basic realm="Metrics"');
        return res.status(401).send('Unauthorized');
      }
      return exporter.collect().then(({resourceMetrics}) => {
        res.set('Content-Type', 'text/plain; charset=utf-8');
        res.send(serializer.serialize(resourceMetrics));
      });
```
