### Title
Unauthenticated HTTP Node-List Fetch Allows MITM to Inject Truncated Network Map in `newClient()`

### Summary
When `cfg.network == "other"`, `newClient()` calls `buildNetworkFromMirrorNodes()`, which fetches the consensus node list over a plain, unauthenticated HTTP connection with no minimum-node-count validation. An attacker who controls the network path between the pinger and the mirror REST API can intercept and replace the `nodesEnvelope` JSON response with a crafted payload containing only 1–2 node entries. The resulting `netmap` is passed directly to `hiero.ClientForNetwork(netmap)`, causing the pinger to route all transactions exclusively to the attacker-chosen nodes and starving ≥30% of real consensus nodes of pinger traffic.

### Finding Description

**Exact code path:**

`pinger/sdk_client.go`, `newClient()`, lines 17–22:
```go
case "other":
    netmap, err := buildNetworkFromMirrorNodes(context.Background(), cfg)
    if err != nil {
        return nil, err
    }
    client = hiero.ClientForNetwork(netmap)
``` [1](#0-0) 

`pinger/mirror_node_client.go`, `buildNetworkFromMirrorNodes()`, lines 36–72:
The HTTP client is constructed as:
```go
httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}
``` [2](#0-1) 

No TLS configuration, no certificate pinning, and no custom `Transport` is set. The URL is assembled from `cfg.mirrorRest`, whose documented default/example is `http://mirror-rest:5551` — plain HTTP. [3](#0-2) 

**Root cause:** Two compounding failures:
1. The node-list endpoint is fetched over plain HTTP (no TLS enforcement, no cert pinning), making the response trivially interceptable and replaceable.
2. The returned `nodesEnvelope.Nodes` slice is consumed without any minimum-length guard before being handed to `hiero.ClientForNetwork(netmap)`. A response with 1–2 entries is accepted as valid. [4](#0-3) 

**Why existing checks fail:**
- The retry loop (`attempts = max(cfg.mirrorNodeClientMaxRetries + 1, 1)`) only retries on transport/HTTP errors, not on a semantically valid but truncated JSON response. [5](#0-4) 
- `loadConfig()` validates that `mirrorRest` is non-empty when `network == "other"`, but performs no scheme enforcement (HTTP vs HTTPS) and no TLS verification requirement. [6](#0-5) 
- There is no post-fetch assertion on the size of `netmap` before `hiero.ClientForNetwork(netmap)` is called. [7](#0-6) 

### Impact Explanation
The pinger is a liveness/health monitoring component. After a successful MITM injection, `hiero.ClientForNetwork(netmap)` builds a client aware of only the attacker-supplied 1–2 nodes. All subsequent periodic tinybar transfers are routed exclusively to those nodes. The remaining ≥30% of real consensus nodes receive zero pinger transactions for the lifetime of the process (the client is built once at startup and never refreshed). This corrupts monitoring telemetry: those nodes appear unreachable or unhealthy to any alerting system consuming pinger results, while the attacker-chosen nodes appear healthy. This satisfies the stated Medium severity scope: ≥30% of network processing nodes are effectively excluded from pinger-driven health observation without any brute-force action.

### Likelihood Explanation
**Precondition:** Attacker controls a network path between the pinger pod and the mirror REST service. This is realistic in several scenarios:
- The pinger is deployed in a Kubernetes cluster where an attacker has compromised a sidecar, a CNI plugin, or a co-tenant pod capable of ARP/NDP spoofing on the pod network.
- The `mirrorRest` URL uses plain `http://` (the documented example), making passive interception or transparent proxy injection trivial — no TLS to break.
- DNS spoofing of the mirror REST hostname is sufficient if the attacker can influence cluster DNS (e.g., CoreDNS compromise).

No credentials, no privileged OS access to the pinger host, and no cryptographic material are required. The attack is repeatable: if the pinger process restarts, the attacker can re-inject on the next startup fetch.

### Recommendation
1. **Enforce HTTPS with certificate verification** for `mirrorRest`. Reject any URL with an `http://` scheme at config-load time, or configure the `http.Client` with a strict `tls.Config` that disallows insecure connections.
2. **Add a minimum node count guard** after `buildNetworkFromMirrorNodes` returns. For example:
   ```go
   const minNodes = 3 // tune to ≥ 70% of expected network size
   if len(netmap) < minNodes {
       return nil, fmt.Errorf("mirror node returned only %d nodes, expected at least %d", len(netmap), minNodes)
   }
   ```
3. **Consider response signing or a trusted bootstrap list** to cross-validate the returned node set against a known-good set of account IDs.

### Proof of Concept
1. Deploy the pinger with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551`.
2. Position a transparent HTTP proxy (e.g., `mitmproxy`) on the network path between the pinger and `mirror-rest:5551`.
3. Configure the proxy to intercept `GET /api/v1/network/nodes` and return:
   ```json
   {"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"ip_address_v4":"1.2.3.4","port":50211}],"grpc_proxy_endpoint":null}],"links":{"next":null}}
   ```
4. Start the pinger. Observe via logs or network capture that all gRPC transaction traffic goes only to `1.2.3.4:50211`.
5. Confirm that all other real consensus node IPs receive zero pinger transactions for the duration of the process lifetime.

func buildNetworkFromMirrorNodes(ctx context.Context, cfg config) (map[string]hiero.AccountID, error) {
	base := strings.TrimRight(strings.TrimSpace(cfg.mirrorRest), "/")

	var url string
	if strings.HasSuffix(base, "/api/v1") {
		url = base + "/network/nodes"
	} else {
		url = base + "/api/v1/network/nodes"
	}

	httpClient := &http.Client{Timeout: cfg.mirrorNodeClientTimeout}

	attempts := max(cfg.mirrorNodeClientMaxRetries + 1, 1)

	var lastErr error

	for attempt := 1; attempt <= attempts; attempt++ {
		network, retry, err := fetchMirrorNodeNetwork(ctx, httpClient, url)
		if err == nil {
			return network, nil
		}

		lastErr = fmt.Errorf("attempt %d/%d: %w", attempt, attempts, err)
		if !retry || attempt == attempts {
			break
		}

		backoff := cfg.mirrorNodeClientBaseBackoff * time.Duration(1<<(attempt-1))
		select {
		case <-ctx.Done():
			return nil, ctx.Err()
		case <-time.After(backoff):
		}
	}

	return nil, lastErr
}
```
