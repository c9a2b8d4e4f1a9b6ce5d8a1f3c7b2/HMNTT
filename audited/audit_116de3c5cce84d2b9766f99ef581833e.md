### Title
Gas Throttle Bypass via `scaleGas()` Dead Zone for Gas Values ≤ 10,000

### Summary
`ThrottleProperties.scaleGas()` returns `0L` for any gas value ≤ `GAS_SCALE_FACTOR` (10,000). Because `ThrottleManagerImpl.throttle()` passes this scaled value directly to `gasLimitBucket.tryConsume()`, and bucket4j's `tryConsume(0)` always returns `true`, any request with `gas ≤ 10,000` consumes zero tokens from the gas bucket and is never throttled by the gas limiter. An unprivileged attacker can exploit this to send `eth_estimateGas` requests at the full rate-limit ceiling (default 500 req/s) without depleting the gas bucket, triggering repeated EVM executions and starving legitimate high-gas requests of their fair share of the rate limit.

### Finding Description

**Exact code path:**

`ThrottleProperties.scaleGas()` — `web3/src/main/java/org/hiero/mirror/web3/throttle/ThrottleProperties.java`, lines 42–47: [1](#0-0) 

```java
public long scaleGas(long gas) {
    if (gas <= GAS_SCALE_FACTOR) {   // GAS_SCALE_FACTOR = 10_000
        return 0L;                   // ← returns 0 for any gas ≤ 10,000
    }
    return Math.floorDiv(gas, GAS_SCALE_FACTOR);
}
```

`ThrottleManagerImpl.throttle()` — `web3/src/main/java/org/hiero/mirror/web3/throttle/ThrottleManagerImpl.java`, line 40: [2](#0-1) 

```java
public void throttle(ContractCallRequest request) {
    if (!rateLimitBucket.tryConsume(1)) {
        throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
    } else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
        // ↑ tryConsume(0) is called when gas ≤ 10,000 — bucket4j always returns true for 0
        throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
    }
```

**Root cause:** The `scaleGas()` function has an intentional dead zone (values ≤ 10,000 map to 0) to avoid fractional token issues with bucket4j. The failed assumption is that gas values this low are harmless or impossible in practice. However, bucket4j's `tryConsume(0)` is a no-op that always succeeds, so the gas throttle guard is completely skipped for these requests.

**Exploit flow:**

1. Attacker sends `eth_estimateGas` with `gas=10000` (or any value ≤ 10,000).
2. `throttle()` is called once per HTTP request (in `ContractController`).
3. `scaleGas(10000)` returns `0`; `gasLimitBucket.tryConsume(0)` returns `true` unconditionally.
4. The request proceeds to `ContractExecutionService.processCall()` → `estimateGas()`.
5. `estimateGas()` first calls `callContract(params, context)` (initial EVM execution with `estimate=false`).
6. If `gasUsedByInitialCall < 10000`, `binaryGasEstimator.search()` runs up to `maxGasEstimateRetriesCount` additional EVM executions — none of which re-check the throttle. [3](#0-2) 
7. The attacker repeats this at up to 500 req/s (the `rateLimitBucket` ceiling), consuming zero gas tokens each time.

**Why existing checks fail:**

- The `rateLimitBucket` (500 req/s) is the only remaining guard, but it is shared across all users. An attacker saturating it at 500 req/s blocks all other users.
- The `restore()` path in `ThrottleManagerImpl` also calls `scaleGas()` and guards with `if (tokens > 0)`, so no tokens are ever restored either — the bucket is never touched at all for these requests. [4](#0-3) 
- There is no minimum-gas validation on `ContractCallRequest` that would reject `gas=10000`.

### Impact Explanation

The gas throttle exists to bound total EVM compute per second across all users. By bypassing it, an attacker can occupy the full 500 req/s rate limit with cheap-but-real EVM executions, starving legitimate users. While gas=10,000 limits the depth of each individual execution, the binary search loop multiplies the number of EVM invocations per request. At 500 req/s × up to `maxGasEstimateRetriesCount` iterations each, this is a practical amplified DoS against the EVM execution layer. The "direct loss of funds" framing in the scope label is inaccurate — this is a resource-exhaustion / availability impact, not a fund-theft vector.

### Likelihood Explanation

No authentication or special privilege is required. The attacker only needs to send standard JSON-RPC `eth_estimateGas` requests with `gas` set to any value ≤ 10,000. This is trivially scriptable with any HTTP client. The attack is repeatable and stateless.

### Recommendation

1. **Fix `scaleGas()` dead zone:** Instead of returning `0` for gas ≤ 10,000, return `1` (minimum token cost) so every request consumes at least one gas token:
   ```java
   public long scaleGas(long gas) {
       if (gas <= 0) return 0L;
       return Math.max(1L, Math.floorDiv(gas, GAS_SCALE_FACTOR));
   }
   ```
2. **Enforce a minimum gas floor on `ContractCallRequest`:** Reject requests with `gas < 21000` (the EVM minimum for a transaction) at the validation layer before throttle checks.
3. **Consider per-IP rate limiting** in addition to the global `rateLimitBucket` to prevent a single attacker from exhausting the shared limit.

### Proof of Concept

```bash
# Send 500 eth_estimateGas requests per second with gas=10000
# Each bypasses the gas bucket (tryConsume(0) = true always)
for i in $(seq 1 500); do
  curl -s -X POST http://<mirror-node>/api/v1/contracts/call \
    -H 'Content-Type: application/json' \
    -d '{
      "data": "0x<simple_contract_calldata>",
      "to": "0x<deployed_contract>",
      "gas": 10000,
      "estimate": true
    }' &
done
wait
# Result: all 500 requests admitted by gas throttle (0 tokens consumed),
# each triggering EVM execution + binary search iterations,
# exhausting the shared rateLimitBucket and blocking legitimate users.
```

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
