### Title
Unauthenticated Metrics Endpoint Leaks Per-Route Latency Histograms, Enabling DB Partition Oracle

### Summary
The `metricsHandler` in `rest/middleware/metricsHandler.js` serves a Prometheus metrics endpoint on the **same public port** as the REST API. Authentication is skipped entirely when `config.metrics.config.authentication` is falsy (the default/unconfigured state). An unprivileged external attacker can poll this endpoint and read `api_request_duration_milliseconds` histogram bucket distributions — labeled by route, method, and HTTP status code — to detect when DB queries are timing out, confirming a network partition attack is succeeding.

### Finding Description

**Exact code path:**

`rest/middleware/metricsHandler.js`, `authenticate()` (lines 137–144):
```js
const authenticate = (req) => {
  const {authentication, username, password} = config.metrics.config;
  if (!authentication) {
    return true;          // ← bypasses all auth when not configured
  }
  ...
};
``` [1](#0-0) 

When `authentication` is falsy (not set in config), `authenticate()` unconditionally returns `true`, granting access to any caller.

The metrics endpoint is registered as a path check inside the **global middleware** applied to the same Express app and port as the public API: [2](#0-1) 

The endpoint path is `${config.metrics.config.uriPath}/metrics/` (e.g., `/swagger/metrics/`), served on the same public-facing port. [3](#0-2) 

On every completed request, `durationHistogram.record(duration, labels)` records the wall-clock duration with labels `{method, path, code}`: [4](#0-3) 

The histogram uses explicit bucket boundaries (default: 25, 100, 250, 500 ms): [5](#0-4) 

**Root cause:** The failed assumption is that operators will always configure authentication. In practice, if `config.metrics.config.authentication` is absent or falsy, the guard is a no-op and the full Prometheus scrape payload — including per-route latency histograms and 5xx counters — is publicly readable.

**Exploit flow:**
1. Attacker discovers the metrics path (default or via enumeration).
2. Attacker sends `GET /swagger/metrics/` with no credentials.
3. Response includes `api_request_duration_milliseconds_bucket` lines broken down by route and status code.
4. During a DB network partition, DB queries time out: requests accumulate in the `+Inf` bucket, the `>500ms` bucket fills, and `api_all_server_error_total` rises sharply.
5. Attacker polls every few seconds and observes the histogram shift in real time — a direct oracle confirming the partition is causing DB-level failures.

### Impact Explanation
An attacker gains a **timing oracle** for internal DB health without any credentials. They can:
- Confirm a network partition attack is causing DB query timeouts (latency spike in high buckets + 5xx surge).
- Identify which specific API routes are DB-heavy (per-route labels).
- Time their attack window precisely using the histogram as feedback.

This converts a blind infrastructure attack into a **closed-loop, observable attack**, significantly increasing its effectiveness. Severity: **Medium** (information disclosure enabling attack amplification; no direct data exfiltration, but meaningful operational intelligence leak).

### Likelihood Explanation
- **No credentials required** when `authentication` is not configured (the default).
- The endpoint is on the same public port — no firewall bypass needed.
- The metrics path is predictable (`/swagger/metrics/` or similar).
- Polling is trivially automated with `curl` or any HTTP client.
- Realistic in any deployment where the operator did not explicitly set `config.metrics.config.authentication = true`.

### Recommendation
1. **Default-deny:** Invert the authentication logic so that if `authentication` config is absent, access is **denied**, not granted:
   ```js
   if (!authentication) {
     return false; // deny by default
   }
   ```
2. **Separate the metrics port:** Serve the metrics endpoint on a dedicated internal-only port (e.g., `9090`) not exposed to the public internet, following the Prometheus operator pattern.
3. **Suppress sensitive labels in default config:** Avoid exposing per-route, per-status-code breakdown unless explicitly enabled, to limit the oracle value even if the endpoint is accidentally exposed.

### Proof of Concept
```bash
# No credentials needed when authentication is not configured
curl -s http://<mirror-node-host>:<port>/swagger/metrics/

# During a DB partition, observe histogram shift:
# api_request_duration_milliseconds_bucket{method="GET",path="/api/v1/transactions",code="500",le="+Inf"} <rising>
# api_request_duration_milliseconds_bucket{method="GET",path="/api/v1/transactions",code="500",le="500"} <flat>
# → confirms requests are exceeding 500ms and returning 5xx = DB timeout signal

# Poll every 5 seconds to track attack progress:
watch -n5 'curl -s http://<host>:<port>/swagger/metrics/ | grep duration'
```
