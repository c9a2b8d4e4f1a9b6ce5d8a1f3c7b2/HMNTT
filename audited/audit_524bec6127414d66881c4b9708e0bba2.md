### Title
Unsynchronized Genesis Cache Check Enables Concurrent DB Query Flooding via Unauthenticated `/block` Requests

### Summary
`initGenesisRecordFile()` in `rosetta/app/persistence/block.go` guards the genesis cache with an unsynchronized struct-field read before executing the expensive `selectGenesis` cross-table query. Because the guard is not protected by a mutex, any number of concurrent goroutines can pass it simultaneously and all issue the `selectGenesis` query against the database. Since the `/block` endpoint requires no authentication and the application applies no rate limiting, an unprivileged attacker can flood the server during the startup window, exhausting the DB connection pool and delaying genesis initialization well beyond normal block time.

### Finding Description

**Exact code path:**

`rosetta/main.go` → `newBlockchainOnlineRouter()` (line 72) creates a single `blockRepository` via `persistence.NewBlockRepository()`. Every `/block` POST request reaches `blockAPIService.Block()` → `BaseService.RetrieveBlock()` → `blockRepo.FindByIndex()` → `initGenesisRecordFile()`.

`rosetta/app/persistence/block.go`, lines 243–263:

```go
func (br *blockRepository) initGenesisRecordFile(ctx context.Context) *rTypes.Error {
    if br.genesisBlock.ConsensusStart != genesisConsensusStartUnset {  // line 244 — plain field read, NO mutex
        return nil
    }

    db, cancel := br.dbClient.GetDbWithContext(ctx)
    defer cancel()

    var rb recordBlock
    if err := db.Raw(selectGenesis, sql.Named("treasury_entity_id", br.treasuryEntityId.EncodedId)).
        First(&rb).Error; err != nil {                                  // line 252-253 — expensive cross-join
        return handleDatabaseError(err, hErrors.ErrNodeIsStarting)
    }

    br.once.Do(func() {
        br.genesisBlock = rb                                            // line 258 — sync.Once only guards the write
    })
    ...
}
```

**Root cause:** The fast-path check at line 244 reads `br.genesisBlock.ConsensusStart` without any synchronization. `sync.Once` (line 257) only serializes the *write*; it does not prevent multiple goroutines from passing the check and all executing the `selectGenesis` query concurrently. This is a TOCTOU race: N goroutines all observe `ConsensusStart == -1`, all proceed past line 246, and all issue the query simultaneously.

**`selectGenesis` query** (lines 49–72): a CTE that queries `account_balance` filtered by `treasury_entity_id`, then cross-joins with `record_file` on `consensus_end > genesis.timestamp`, ordered by `consensus_end`, `LIMIT 1`. On a production database with millions of rows this is a non-trivial query.

**No application-level rate limiting:** `newBlockchainOnlineRouter()` (lines 61–120) wraps the router only with `MetricsMiddleware`, `TracingMiddleware`, and `CorsMiddleware`. There is no authentication, no per-IP connection limit, and no request throttle inside the Go application. The Traefik middleware in `charts/hedera-mirror-rosetta/values.yaml` (lines 149–166) is an optional Kubernetes-layer control that is not enforced by the application itself and is trivially bypassed from multiple source IPs.

**DB connection pool:** Default `maxOpenConnections = 100` (`docs/configuration.md` line 658, `rosetta/app/db/db.go` line 33). An attacker can saturate all 100 connections with concurrent `selectGenesis` queries, causing every subsequent DB call (including legitimate ones) to queue or time out.

**Exploit flow:**
1. Server restarts; `br.genesisBlock.ConsensusStart` is initialized to `-1` (line 130).
2. Attacker immediately sends a flood of concurrent POST `/block` requests with `BlockIdentifier.Index = 0` (valid: `FindByIndex` only rejects `index < 0`, line 172).
3. All goroutines reach line 244 before any query completes; all read `-1`; all proceed.
4. All issue `selectGenesis` concurrently, saturating the 100-connection pool.
5. The pool is exhausted; new DB calls block waiting for a connection.
6. Genesis initialization is delayed proportionally to the DB saturation duration.
7. All block-related endpoints (`FindByIndex`, `FindByHash`, `RetrieveLatest`, `RetrieveGenesis`) are stalled because every one of them calls `initGenesisRecordFile` first.

### Impact Explanation

During the startup window (before genesis is first cached), the entire block-serving subsystem is effectively frozen. Hedera's average block time is ~1–2 seconds. With 100 concurrent `selectGenesis` queries saturating the pool, each query's latency multiplies by the queue depth, easily producing delays of 10–20 seconds (500–1000% of normal block time). All Rosetta API consumers (exchanges, indexers, wallets) receive errors or stalled responses for the duration of the attack. Because the attacker can repeat the attack on every server restart or pod rollout, this constitutes a persistent, repeatable availability impact against the Rosetta API.

### Likelihood Explanation

The attack requires zero privileges: the `/block` endpoint is unauthenticated and publicly reachable. The only precondition is that the server has recently started (or restarted), which is a predictable event (deployments, rolling updates, crash recovery). A single attacker with a modest HTTP flood tool (e.g., `wrk`, `hey`, or a simple script) can saturate the pool. Using multiple source IPs trivially bypasses the optional Traefik per-IP `inFlightReq` limit of 5. The attack is fully repeatable on every restart.

### Recommendation

Replace the unsynchronized field-read guard with a `sync.RWMutex` or restructure `initGenesisRecordFile` to use `sync.Once` for the *entire* initialization block (query + write), not just the write:

```go
br.once.Do(func() {
    db, cancel := br.dbClient.GetDbWithContext(ctx)
    defer cancel()
    var rb recordBlock
    if err := db.Raw(selectGenesis, ...).First(&rb).Error; err != nil {
        // handle error; reset once if needed via a separate flag
        return
    }
    br.genesisBlock = rb
})
```

This ensures exactly one goroutine executes the DB query regardless of concurrency. Additionally, add application-level concurrency controls (e.g., a semaphore or `golang.org/x/time/rate` limiter) inside `newBlockchainOnlineRouter` so that the Rosetta API is not solely dependent on optional infrastructure-layer rate limiting.

func newBlockchainOnlineRouter(
	asserter *rosettaAsserter.Asserter,
	dbClient interfaces.DbClient,
	network *rTypes.NetworkIdentifier,
	mirrorConfig *config.Mirror,
	serverContext context.Context,
	version *rTypes.Version,
) (http.Handler, error) {
	systemEntity := domain.NewSystemEntity(mirrorConfig.Common)
	accountRepo := persistence.NewAccountRepository(dbClient, systemEntity.GetTreasuryAccount())
	addressBookEntryRepo := persistence.NewAddressBookEntryRepository(systemEntity.GetAddressBook101(), systemEntity.GetAddressBook102(), dbClient)
	blockRepo := persistence.NewBlockRepository(dbClient, systemEntity.GetTreasuryAccount())
	transactionRepo := persistence.NewTransactionRepository(dbClient, systemEntity.GetStakingRewardAccount())

	baseService := services.NewOnlineBaseService(blockRepo, transactionRepo)

	networkAPIService := services.NewNetworkAPIService(baseService, addressBookEntryRepo, network, version)
	networkAPIController := server.NewNetworkAPIController(networkAPIService, asserter)

	blockAPIService := services.NewBlockAPIService(
		accountRepo,
		baseService,
		mirrorConfig.Rosetta.Cache[config.EntityCacheKey],
		mirrorConfig.Rosetta.Response.MaxTransactionsInBlock,
		serverContext,
	)
	blockAPIController := server.NewBlockAPIController(blockAPIService, asserter)

	mempoolAPIService := services.NewMempoolAPIService()
	mempoolAPIController := server.NewMempoolAPIController(mempoolAPIService, asserter)

	constructionAPIService, err := services.NewConstructionAPIService(
		accountRepo,
		baseService,
		mirrorConfig,
		construction.NewTransactionConstructor(),
	)
	if err != nil {
		return nil, err
	}
	constructionAPIController := server.NewConstructionAPIController(constructionAPIService, asserter)

	healthController, err := middleware.NewHealthController(&mirrorConfig.Rosetta)
	metricsController := middleware.NewMetricsController()
	if err != nil {
		return nil, err
	}

	return server.NewRouter(
		networkAPIController,
		blockAPIController,
		mempoolAPIController,
		constructionAPIController,
		accountAPIController,
		healthController,
		metricsController,
	), nil
}
