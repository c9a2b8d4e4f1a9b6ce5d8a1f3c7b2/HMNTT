### Title
Entity Existence Oracle via Differential Error Messages in gRPC `subscribeTopic`

### Summary
The `topicExists()` method in `TopicMessageServiceImpl` throws two semantically distinct exceptions depending on whether a queried entity ID exists in the database or not. These exceptions are mapped by `ProtoUtil.toStatusRuntimeException()` to different gRPC status codes and their raw messages are forwarded verbatim to the caller. Any unauthenticated external user can exploit this differential to determine whether an arbitrary entity ID exists in the mirror node database and confirm it is a non-topic entity type.

### Finding Description

**Code path:**

`ConsensusController.subscribeTopic()` (line 44–48) calls `toFilter()` then `topicMessageService::subscribeTopic`, with all errors mapped via `ProtoUtil::toStatusRuntimeException`.

`toFilter()` (lines 55–73) performs no entity type validation — it simply wraps the caller-supplied `topicID` into an `EntityId` and builds a `TopicMessageFilter`. [1](#0-0) 

`TopicMessageServiceImpl.topicExists()` (lines 94–106) then performs a database lookup and branches on two distinct outcomes:
- **Entity not found** → throws `EntityNotFoundException(topicId)`, message: `"0.0.<num> does not exist"`
- **Entity found but type ≠ TOPIC** → throws `IllegalArgumentException("Not a valid topic")` [2](#0-1) 

`ProtoUtil.toStatusRuntimeException()` maps these to **different gRPC status codes** and forwards the raw exception message to the client:
- `IllegalArgumentException` → `Status.INVALID_ARGUMENT` + `t.getMessage()` = `"Not a valid topic"`
- `EntityNotFoundException` → `Status.NOT_FOUND` + `t.getMessage()` = `"0.0.X does not exist"` [3](#0-2) 

The `EntityNotFoundException` message template is `"%s does not exist"`, confirming the entity ID is embedded in the response. [4](#0-3) 

**Root cause:** The failed assumption is that both "entity not found" and "entity found but wrong type" are treated as equivalent invalid inputs from a security perspective. They are not — they leak orthogonal facts about the database state. The `checkTopicExists` configuration flag only governs the "not found" branch; the "wrong type" branch always emits the distinguishing `"Not a valid topic"` message unconditionally.

### Impact Explanation
An attacker gains a binary oracle over the entity table: for any entity ID `N`, a `subscribeTopic` call with `topicNum=N` returns either `NOT_FOUND` (entity absent) or `INVALID_ARGUMENT: Not a valid topic` (entity present, non-topic). This allows systematic enumeration of which entity IDs exist in the mirror node database and confirmation that those entities are accounts, contracts, files, or tokens — not topics. While Hedera entity IDs are sequential and some information is public via REST, the gRPC endpoint provides a dedicated, unauthenticated, high-throughput channel for this enumeration with no rate limiting enforced at the application layer.

### Likelihood Explanation
No authentication or authorization is required. The gRPC `subscribeTopic` endpoint is publicly reachable. The attacker needs only a standard gRPC client (e.g., `grpcurl`) and knowledge of the protobuf schema, both of which are publicly available. The attack is trivially scriptable, repeatable, and produces deterministic results. Entity IDs are small integers, making exhaustive enumeration practical.

### Recommendation
Normalize the error response for both "entity not found" and "entity found but not a topic" into a single, indistinguishable error. Replace the two-branch logic in `topicExists()` with a unified error:

```java
.switchIfEmpty(Mono.error(new IllegalArgumentException("Not a valid topic")));
// applied to BOTH the not-found and wrong-type cases
```

Or, if the `NOT_FOUND` distinction must be preserved internally, ensure `ProtoUtil.toStatusRuntimeException()` maps `EntityNotFoundException` to the same `INVALID_ARGUMENT` status and a generic message (e.g., `"Not a valid topic"`) rather than exposing the entity-specific `"0.0.X does not exist"` string and a different status code.

### Proof of Concept

**Preconditions:** Mirror node gRPC endpoint is reachable. Entity `0.0.1000` is an account (not a topic). Entity `0.0.9999999` does not exist.

**Expected response:** `NOT_FOUND: 0.0.9999999 does not exist`
→ Confirms entity `0.0.9999999` **does not exist** in the database.

**Step 3 — Enumerate:** Script steps 1–2 across a range of `topicNum` values to build a map of existing non-topic entities.

        var topicId = filter.getTopicId();
        return Mono.justOrEmpty(entityRepository.findById(topicId.getId()))
                .switchIfEmpty(
                        grpcProperties.isCheckTopicExists()
                                ? Mono.error(new EntityNotFoundException(topicId))
                                : Mono.just(Entity.builder()
                                        .memo("")
                                        .type(EntityType.TOPIC)
                                        .build()))
                .filter(e -> e.getType() == EntityType.TOPIC)
                .switchIfEmpty(Mono.error(new IllegalArgumentException("Not a valid topic")));
    }

    public static StatusRuntimeException toStatusRuntimeException(Throwable t) {
        if (Exceptions.isOverflow(t)) {
            return clientError(t, Status.DEADLINE_EXCEEDED, OVERFLOW_ERROR);
        } else if (t instanceof ConstraintViolationException
                || t instanceof IllegalArgumentException
                || t instanceof InvalidEntityException) {
            return clientError(t, Status.INVALID_ARGUMENT, t.getMessage());
        } else if (t instanceof EntityNotFoundException) {
            return clientError(t, Status.NOT_FOUND, t.getMessage());
        } else if (t instanceof TransientDataAccessException || t instanceof TimeoutException) {
            return serverError(t, Status.RESOURCE_EXHAUSTED, DB_ERROR);
        } else if (t instanceof NonTransientDataAccessResourceException) {
            return serverError(t, Status.UNAVAILABLE, DB_ERROR);
        } else {
            return serverError(t, Status.UNKNOWN, UNKNOWN_ERROR);
        }
    }
