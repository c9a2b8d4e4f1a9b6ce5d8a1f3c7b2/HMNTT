### Title
Unauthenticated Endpoint Flood Enables False-Positive DOWN Health Alerts via `SubscriberHealthIndicator`

### Summary
The monitor module's `SubscriberHealthIndicator` uses the HTTP status code of the public, unauthenticated, and unrate-limited `GET /api/v1/network/stake` endpoint as the **primary gate** for the entire mirror node health signal. Any 5xx response from that endpoint immediately causes the monitor to report `Status.DOWN`, bypassing all other health checks. Because the `rest-java` module applies no rate limiting to this endpoint and `NetworkStakeRepository.findLatest()` issues an uncached correlated SQL subquery on every request, an unprivileged attacker can flood the endpoint to exhaust database connections or application threads, inducing 500/503 responses that the monitor misinterprets as a genuine infrastructure failure.

### Finding Description

**Code path 1 – the unprotected endpoint:**

`rest-java/src/main/java/org/hiero/mirror/restjava/controller/NetworkController.java`, lines 126–130:
```java
@GetMapping("/stake")
NetworkStakeResponse getNetworkStake() {
    final var networkStake = networkService.getLatestNetworkStake();
    return networkStakeMapper.map(networkStake);
}
```
No authentication, no rate-limiting annotation, no throttle guard. The rate-limiting infrastructure present in the codebase (`web3` module's `ThrottleManagerImpl`, `RequestFilter`) is entirely absent from `rest-java`.

**Code path 2 – the uncached DB query:**

`rest-java/src/main/java/org/hiero/mirror/restjava/repository/NetworkStakeRepository.java`, lines 12–19:
```java
@Query(value = """
    select *
    from network_stake
    where consensus_timestamp = (
        select max(consensus_timestamp) from network_stake
    )
    """, nativeQuery = true)
Optional<NetworkStake> findLatest();
```
A correlated subquery (`SELECT MAX(...)`) executes on every HTTP request. `NetworkServiceImpl.getLatestNetworkStake()` carries no `@Cacheable` annotation (confirmed by grep). Under flood conditions this saturates the JDBC connection pool.

**Code path 3 – the health gate:**

`monitor/src/main/java/org/hiero/mirror/monitor/health/SubscriberHealthIndicator.java`, lines 59–63 and 93–106:
```java
public Mono<Health> health() {
    return restNetworkStakeHealth()          // evaluated FIRST
        .flatMap(health ->
            health.getStatus() == Status.UP
                ? publishing().switchIfEmpty(subscribing())
                : Mono.just(health));        // short-circuits on non-UP
}

// inside restNetworkStakeHealth():
var status = statusCode.is5xxServerError() ? Status.DOWN : Status.UNKNOWN;
```
Any 5xx from the endpoint → `Status.DOWN` → the `flatMap` short-circuits → the entire mirror node is reported as DOWN without ever consulting publish/subscribe rates.

**Root cause / failed assumption:** The design assumes that `/api/v1/network/stake` can only return 5xx when the mirror node is genuinely unhealthy. This assumption fails because the endpoint is publicly reachable with no rate limiting, and its per-request DB cost is non-trivial.

### Impact Explanation
A sustained flood causes the monitor to emit `Status.DOWN` and set the `hiero.mirror.monitor.health{type=cluster}` gauge to `0`. Downstream Grafana/PagerDuty alerts keyed on this metric fire as if the mirror node is down. On-call engineers are paged, incident bridges are opened, and rollback/restart procedures may be initiated — all for a condition that does not reflect actual mirror node health. The impact is operational disruption and alert fatigue with no economic damage to the Hedera network itself, consistent with the stated Medium scope.

### Likelihood Explanation
The attack requires only an HTTP client and a publicly routable IP. No credentials, tokens, or privileged access are needed. The flood can be sustained from a single host using async HTTP tooling (e.g., `wrk`, `hey`, `ab`) or trivially distributed. The monitor polls health on a short interval, so even a brief flood window is sufficient to trigger an alert. The attack is fully repeatable.

### Recommendation
1. **Decouple the health signal from the public endpoint.** Replace `getNetworkStakeStatusCode()` with an internal actuator check (e.g., a direct DB ping or a Spring `DataSourceHealthIndicator`) that is not reachable from the public internet.
2. **Add rate limiting to `rest-java` network endpoints.** Apply a per-IP or global rate limiter (Spring Cloud Gateway, Bucket4j, or a sidecar) to `/api/v1/network/stake` and other `NetworkController` routes.
3. **Cache `findLatest()`.** Annotate `NetworkServiceImpl.getLatestNetworkStake()` with `@Cacheable` with a short TTL (e.g., 30 s) to bound DB load regardless of request rate.
4. **Treat 5xx as UNKNOWN, not DOWN.** If the public endpoint must remain the health probe, map 5xx to `Status.UNKNOWN` rather than `Status.DOWN` so that transient load spikes do not trigger critical alerts.

### Proof of Concept

**Preconditions:** Mirror node is running and healthy; monitor is polling health.

**Step 1 – Flood the endpoint from any unprivileged host:**
```bash
# Using 'hey' (https://github.com/rakyll/hey)
hey -n 100000 -c 500 -q 0 https://<mirror-node-host>/api/v1/network/stake
```

**Step 2 – Observe DB connection pool exhaustion.** The JDBC pool (default HikariCP, typically 10 connections) is saturated by concurrent `SELECT MAX(consensus_timestamp) FROM network_stake` subqueries. New requests receive HTTP 500 or 503.

**Step 3 – Monitor's health check fires during the flood window.**
`SubscriberHealthIndicator.restNetworkStakeHealth()` receives a 5xx status code:
```
statusCode.is5xxServerError() == true  →  status = Status.DOWN
```

**Step 4 – Health endpoint reports DOWN:**
```
GET /actuator/health  →  {"status":"DOWN"}
```
The `hiero.mirror.monitor.health{type=cluster}` gauge drops to `0`, triggering any configured alert rule.

**Step 5 – Flood stops; monitor recovers automatically** on the next poll cycle, leaving no persistent damage but having generated a false incident. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

    private Mono<Health> restNetworkStakeHealth() {
        return restApiClient
                .getNetworkStakeStatusCode()
                .flatMap(statusCode -> {
                    if (statusCode.is2xxSuccessful()) {
                        return UP;
                    }

                    var status = statusCode.is5xxServerError() ? Status.DOWN : Status.UNKNOWN;
                    var statusMessage =
                            String.format("Network stake status is %s with status code %s", status, statusCode.value());
                    log.error(statusMessage);
