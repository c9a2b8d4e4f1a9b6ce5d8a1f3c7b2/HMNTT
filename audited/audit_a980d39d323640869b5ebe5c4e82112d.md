### Title
Unauthenticated Multi-Query Fan-Out in `getContractResultsByTimestamp` Enables DB Resource Exhaustion

### Summary
The `GET /contracts/:contractId/results/:consensusTimestamp` endpoint is publicly accessible with no authentication or application-level rate limiting. Each successful request triggers up to 7 sequential/parallel database queries. A single attacker flooding this endpoint with valid path parameters can exhaust the PostgreSQL connection pool, degrading or denying service for all users.

### Finding Description

**Exact code path:**

`rest/controllers/contractController.js`, lines 1002â€“1042, `getContractResultsByTimestamp()`:

```
1011: ContractService.getInvolvedContractsByTimestampAndContractId(timestamp, contractId)  // Query 1
1016-1017: this.getDetailedContractResults(contractDetails, contractId)                    // Queries 2-6 (Promise.all)
1022-1023: FileDataService.getLatestFileDataContents(ethTransaction.callDataId, ...)       // Query 7 (conditional)
```

`getDetailedContractResults` (lines 1205â€“1220) fires **5 queries in parallel** via `Promise.all`:
- `ContractService.getContractResultsByTimestamps`
- `TransactionService.getEthTransactionByTimestampAndPayerId`
- `RecordFileService.getRecordFileBlockDetailsFromTimestamp`
- `ContractService.getContractLogsByTimestamps`
- `ContractService.getContractStateChangesByTimestamps`

If `ethTransaction.callDataId` passes `utils.isValidUserFileId()` (any file entity ID > 1000), `FileDataService.getLatestFileDataContents` fires a 7th query â€” itself a CTE-based aggregation query against the `file_data` table.

**Root cause â€” failed assumption:** The design assumes infrastructure-level rate limiting (Traefik middleware) will always be deployed and enabled. In code, `rest/server.js` (lines 68â€“98) registers no rate-limiting middleware. The `authHandler` (`rest/middleware/authHandler.js`, lines 15â€“36) only sets a custom pagination *limit* for authenticated users; it does not block or throttle unauthenticated requests. The response cache (`responseCacheHandler.js`) is only active when `config.cache.response.enabled && config.redis.enabled` â€” both optional. The Traefik middleware (`charts/hedera-mirror-rest/templates/middleware.yaml`) is conditional on `global.middleware` being true, which is not enforced at the application layer.

**Exploit flow:**
1. Attacker queries `/api/v1/contracts` to enumerate valid `contractId` values.
2. Attacker queries `/api/v1/contracts/:contractId/results` to collect valid `consensusTimestamp` values.
3. Attacker sends a high-concurrency flood of `GET /api/v1/contracts/:contractId/results/:consensusTimestamp` requests using valid pairs.
4. Each request that finds a matching record in `contract_transaction` fires 5 parallel DB queries + 1 optional file data query.
5. The PostgreSQL connection pool (finite size) is saturated; legitimate requests queue or time out.

### Impact Explanation
Each successful request consumes up to 7 database connections simultaneously (5 via `Promise.all`). With a standard pool size (e.g., 10â€“20 connections), as few as 3â€“4 concurrent attacker requests can saturate the pool. All other API endpoints sharing the same pool â€” account lookups, transaction queries, token queries â€” experience increased latency or connection-wait timeouts. This constitutes a griefing-class denial of service with no economic cost to the attacker and no on-chain footprint.

### Likelihood Explanation
The preconditions are trivially satisfied: valid `contractId` and `consensusTimestamp` pairs are public blockchain data discoverable via the same API. No credentials, tokens, or special network access are required. The attack is fully automatable with any HTTP load tool (e.g., `wrk`, `ab`, `hey`). It is repeatable indefinitely. The only mitigations are optional infrastructure components that may not be deployed in all environments.

### Recommendation
1. **Add application-level rate limiting** unconditionally in `rest/server.js`, independent of infrastructure configuration. Use `express-rate-limit` or equivalent, keyed by IP, applied before route handlers.
2. **Add a DB query concurrency guard**: limit the number of in-flight requests to this endpoint using a semaphore or connection-pool-aware queue.
3. **Make response caching mandatory** (not opt-in) for this endpoint, since contract results at a specific timestamp are immutable once finalized.
4. **Do not rely solely on Traefik middleware** for rate limiting; treat it as defense-in-depth, not the primary control.
