### Title
Entity ID Enumeration Oracle via Differential Error Responses in `topicExists()`

### Summary
The `topicExists()` method in `TopicMessageServiceImpl` returns two semantically distinct gRPC error codes depending on whether a queried entity ID is absent from the database versus present but not of type `TOPIC`. Any unauthenticated caller can exploit this difference to enumerate which entity IDs exist in the mirror node's database and infer their types, with no credentials or privileges required.

### Finding Description
**Code location:** `grpc/src/main/java/org/hiero/mirror/grpc/service/TopicMessageServiceImpl.java`, `topicExists()`, lines 94–106.

```java
private Mono<?> topicExists(TopicMessageFilter filter) {
    var topicId = filter.getTopicId();
    return Mono.justOrEmpty(entityRepository.findById(topicId.getId()))
            .switchIfEmpty(
                    grpcProperties.isCheckTopicExists()
                            ? Mono.error(new EntityNotFoundException(topicId))   // path A
                            : Mono.just(Entity.builder().memo("").type(EntityType.TOPIC).build()))
            .filter(e -> e.getType() == EntityType.TOPIC)
            .switchIfEmpty(Mono.error(new IllegalArgumentException("Not a valid topic")));  // path B
}
```

**Path A** — entity ID not found in DB (default: `checkTopicExists = true`):
`EntityNotFoundException` → `ProtoUtil.toStatusRuntimeException()` line 51–52 → gRPC `Status.NOT_FOUND` with description `"0.0.X does not exist"`.

**Path B** — entity ID found but `type != TOPIC`:
`IllegalArgumentException` → `ProtoUtil.toStatusRuntimeException()` line 47–50 → gRPC `Status.INVALID_ARGUMENT` with description `"Not a valid topic"`.

**Path C** — valid topic: subscription stream opens, no error.

`checkTopicExists` defaults to `true` in `GrpcProperties.java` line 19, so Path A is active in all default deployments. No authentication gate exists before `subscribeTopic` is called; the `@Validated` annotation only enforces filter format constraints.

The oracle is:
| Response | Meaning |
|---|---|
| `NOT_FOUND` "0.0.X does not exist" | Entity ID absent from DB |
| `INVALID_ARGUMENT` "Not a valid topic" | Entity ID exists, is not a TOPIC (account, contract, file, etc.) |
| Stream opens | Entity ID is a valid TOPIC |

### Impact Explanation
An attacker can iterate over entity IDs (Hedera IDs are sequential: `0.0.1`, `0.0.2`, …) and, for each, determine: (a) whether the ID is allocated at all, and (b) whether it belongs to a non-topic entity type. This leaks the mirror node's internal entity inventory — which IDs are live, which are unallocated, and which are non-topic entities — without any credentials. While Hedera entity IDs are broadly public on-chain, this oracle provides a fast, programmatic, unauthenticated enumeration channel directly against the mirror node's database state, which may diverge from or lead the public explorer state (e.g., during sync lag). Severity is **low-to-medium**: no funds or private keys are exposed, but entity existence and type are disclosed.

### Likelihood Explanation
Exploitation requires only the ability to send gRPC requests to the mirror node's `ConsensusService` endpoint (port 5600 by default), which is publicly exposed by design. No account, token, or special network access is needed. The attack is fully automatable, repeatable, and produces deterministic results. Any developer with a gRPC client (e.g., `grpcurl`, Hedera SDK) can execute it immediately.

### Recommendation
Normalize error responses so that both "entity not found" and "entity exists but is not a topic" return the same gRPC status code and a generic message (e.g., `Status.NOT_FOUND` with `"Topic not found"`). Specifically, change the `IllegalArgumentException` in the second `switchIfEmpty` to also throw `EntityNotFoundException(topicId)` (or a unified `TopicNotFoundException`), so both paths produce identical `NOT_FOUND` responses indistinguishable to the caller:

```java
.switchIfEmpty(Mono.error(new EntityNotFoundException(topicId)));
```

Optionally, add rate-limiting on the `subscribeTopic` endpoint to slow bulk enumeration regardless of error normalization.

### Proof of Concept
**Preconditions:** gRPC access to the mirror node's ConsensusService (default port 5600); `checkTopicExists=true` (default).

**Steps:**

1. Send a `subscribeTopic` request for a known non-existent ID, e.g., `topicID = 0.0.99999999`:
   ```
   grpcurl -plaintext -d '{"topicID":{"topicNum":99999999}}' \
     <mirror-node>:5600 com.hedera.mirror.api.proto.ConsensusService/subscribeTopic
   ```
   **Expected response:** `NOT_FOUND: 0.0.99999999 does not exist` → ID is absent.

2. Send a `subscribeTopic` request for a known account ID (e.g., `0.0.2`, the treasury account):
   ```
   grpcurl -plaintext -d '{"topicID":{"topicNum":2}}' \
     <mirror-node>:5600 com.hedera.mirror.api.proto.ConsensusService/subscribeTopic
   ```
   **Expected response:** `INVALID_ARGUMENT: Not a valid topic` → ID exists, is not a TOPIC.

3. Automate iteration over `topicNum` values 1–N, classifying each ID as: absent (`NOT_FOUND`), non-topic entity (`INVALID_ARGUMENT`), or valid topic (stream opens).     public static StatusRuntimeException toStatusRuntimeException(Throwable t) {
        if (Exceptions.isOverflow(t)) {
            return clientError(t, Status.DEADLINE_EXCEEDED, OVERFLOW_ERROR);
        } else if (t instanceof ConstraintViolationException
                || t instanceof IllegalArgumentException
                || t instanceof InvalidEntityException) {
            return clientError(t, Status.INVALID_ARGUMENT, t.getMessage());
        } else if (t instanceof EntityNotFoundException) {
            return clientError(t, Status.NOT_FOUND, t.getMessage());
        } else if (t instanceof TransientDataAccessException || t instanceof TimeoutException) {
            return serverError(t, Status.RESOURCE_EXHAUSTED, DB_ERROR);
        } else if (t instanceof NonTransientDataAccessResourceException) {
            return serverError(t, Status.UNAVAILABLE, DB_ERROR);
        } else {
            return serverError(t, Status.UNKNOWN, UNKNOWN_ERROR);
        }
    }
