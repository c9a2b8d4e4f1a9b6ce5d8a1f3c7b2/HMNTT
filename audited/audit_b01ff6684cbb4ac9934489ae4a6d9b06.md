### Title
Gas Throttle Bypass via Binary Search Amplification in `estimate=true` Requests

### Summary
The gas throttle in `ContractController.call()` is charged exactly once per HTTP request based on the user-supplied `gas` value, but when `estimate=true` is set, `ContractExecutionService.estimateGas()` triggers up to 21 sequential EVM executions (1 initial probe + up to 20 binary search iterations). No authentication is required. An unprivileged attacker can supply `estimate=true`, a historical block, and `gas=15000000` to cause up to 21 full EVM re-executions against historical state per single throttle token deduction, achieving a 21x amplification of server-side compute and database load.

### Finding Description

**Exact code path:**

`ContractController.call()` (line 40) calls `throttleManager.throttle(request)` once, deducting `scaleGas(request.getGas())` tokens from the gas bucket: [1](#0-0) 

`ThrottleManagerImpl.throttle()` (line 40) consumes `floor(gas / 10_000)` tokens — for `gas=15_000_000` that is 1,500 tokens, charged once: [2](#0-1) 

`restore()` is only called in the `catch (InvalidParametersException)` branch — it is never called after a successful estimate path: [3](#0-2) 

After the single throttle deduction, `ContractExecutionService.estimateGas()` fires one initial `callContract()` and then passes control to `BinaryGasEstimator.search()`: [4](#0-3) 

`BinaryGasEstimator.search()` loops up to `maxGasEstimateRetriesCount = 20` times, each iteration invoking a full EVM execution via `safeCall()`: [5](#0-4) 

The default `maxGasEstimateRetriesCount` is 20: [6](#0-5) 

**Root cause:** The throttle model assumes 1 EVM execution per request. For `estimate=true`, the actual execution count is 1 + up to 20 = **21 EVM executions**, but the gas bucket is only debited once. The failed assumption is that `gas` tokens consumed ≈ server work performed.

**Why existing checks fail:**
- The gas bucket (`gasPerSecond = 7,500,000,000`, scaled to 750,000 tokens) is sized for 1 execution per request. With 21 executions per request, the effective throughput the server must handle is 21× what the bucket enforces.
- The RPS bucket (`requestsPerSecond = 500`) limits HTTP requests, not EVM executions. 500 estimate requests/sec × 21 iterations = 10,500 EVM executions/sec.
- There is no per-IP rate limiting, no authentication requirement, and no flag distinguishing estimate from non-estimate in the gas bucket accounting. [7](#0-6) 

### Impact Explanation
Each request with `estimate=true`, `gas=15000000`, and a historical block forces the server to:
1. Load historical state from the database for the target block (expensive I/O).
2. Execute the EVM up to 21 times against that historical state.
3. Produce intermediate `ContractCallContext` snapshots (reset on each iteration via `contractCallContext.reset()`).

A single attacker staying within the 500 RPS limit can force 10,500 full EVM executions per second instead of 500, exhausting CPU, JVM heap, and database connection pool. This constitutes a practical denial-of-service against the web3 service without requiring any privileged access or special credentials.

### Likelihood Explanation
The endpoint `/api/v1/contracts/call` is a public, unauthenticated HTTP POST endpoint. No wallet, token, or API key is required. The attack requires only crafting a JSON body with `"estimate": true`, a valid historical block number, and `"gas": 15000000`. It is trivially scriptable and repeatable. The attacker does not need to know any contract address — `to` can be omitted (defaults to zero address) and the binary search will still run all 20 iterations before converging.

### Recommendation
1. **Multiply the gas token cost for estimate calls** by `maxGasEstimateRetriesCount + 1` (i.e., 21×) in `ThrottleManagerImpl.throttle()` when `request.isEstimate()` is true, so the gas bucket correctly reflects the worst-case server work.
2. Alternatively, **call `throttleManager` inside each binary search iteration** (inside `BinaryGasEstimator.search()`) to deduct gas tokens per actual EVM execution rather than per HTTP request.
3. **Add per-source-IP rate limiting** as a defense-in-depth layer so a single client cannot exhaust the global RPS budget.
4. Consider **capping `maxGasEstimateRetriesCount`** to a lower value (e.g., 10) to reduce the amplification ceiling.

### Proof of Concept

```bash
# No authentication required. Repeat at up to 500 req/s to saturate server.
curl -X POST https://<mirror-node-host>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{
    "to": "0x0000000000000000000000000000000000000167",
    "data": "0x<any-valid-calldata>",
    "gas": 15000000,
    "estimate": true,
    "block": "0x1"
  }'
```

**Steps:**
1. Identify any deployed contract address on the network (or use a precompile address).
2. Send the above request with `estimate=true`, `gas=15000000`, and a historical block number (`"block": "0x1"` or any past block).
3. The server deducts 1,500 gas tokens from the throttle bucket (1× charge).
4. Internally, `estimateGas()` fires 1 initial EVM call + up to 20 binary search iterations = 21 EVM executions against historical state.
5. Flood at 500 req/s (within the RPS limit) to force 10,500 EVM executions/sec while the throttle only accounts for 500.
6. Observe server CPU saturation, increased DB query load, and eventual request timeouts or OOM errors.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
