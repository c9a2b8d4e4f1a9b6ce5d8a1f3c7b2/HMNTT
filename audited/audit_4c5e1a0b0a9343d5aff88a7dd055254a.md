### Title
Synchronous Thread Pool Exhaustion via Concurrent `estimate=true` Requests in `ContractController.call()`

### Summary
The `/api/v1/contracts/call` endpoint processes `estimate=true` requests synchronously, holding a Tomcat servlet thread for the full duration of up to 21 EVM executions (1 initial + 20 `BinaryGasEstimator` iterations). The gas/RPS throttle is checked once at request entry via a non-blocking `tryConsume()` and is not re-evaluated during execution. The bucket capacity allows a burst of ~500 max-gas requests to pass simultaneously, far exceeding Tomcat's default 200-thread pool, enabling an unauthenticated attacker to exhaust all servlet threads and deny service to all subsequent requests.

### Finding Description

**Entry point — `ContractController.call()` (lines 38–51):** [1](#0-0) 

The method is a plain blocking `@PostMapping` handler. It calls `throttleManager.throttle(request)` once, then synchronously invokes `contractExecutionService.processCall(params)`, holding the servlet thread until the call completes.

**Throttle is non-blocking and checked only once — `ThrottleManagerImpl.throttle()` (lines 37–48):** [2](#0-1) 

`tryConsume()` is a non-blocking token-bucket check. If tokens are available, the request is admitted immediately and the thread is never re-checked. The gas bucket capacity equals `gasPerSecond` (default 7,500,000,000 / 10,000 = 750,000 tokens). Each max-gas request (15,000,000 gas / 10,000 = 1,500 tokens) consumes 1,500 tokens. A full bucket admits **500 max-gas requests in a single burst** before the bucket empties.

**Up to 21 blocking EVM executions per estimate request — `ContractExecutionService.estimateGas()` (lines 81–98) and `BinaryGasEstimator.search()` (lines 35–58):** [3](#0-2) [4](#0-3) 

`maxGasEstimateRetriesCount = 20` (default). Each iteration calls `doProcessCall()` synchronously. All 21 calls execute on the same servlet thread.

**`requestTimeout` only enforced at DB query boundaries — `HibernateConfiguration.statementInspector()` (lines 31–47):** [5](#0-4) 

The timeout check fires only when Hibernate is about to issue a new SQL statement. During CPU-bound EVM bytecode execution between DB queries, the thread is held with no timeout enforcement. The `db.statementTimeout = 3000ms` adds up to 3 additional seconds per statement after the 10s wall-clock timeout is reached.

**`requestTimeout` default:** [6](#0-5) 

**`maxGasEstimateRetriesCount` and `maxGasLimit` defaults:** [7](#0-6) 

### Impact Explanation

Tomcat's default thread pool is 200 threads. An attacker who sends 200 concurrent `estimate=true` requests with `gas=15000000` (the maximum) will:
1. Pass the gas bucket check (bucket capacity = 750,000 tokens; 200 requests × 1,500 tokens = 300,000 tokens — well within capacity).
2. Pass the RPS check (200 requests in <1 second is within the 500 req/s limit).
3. Hold all 200 servlet threads for up to ~10–13 seconds each.
4. Cause all subsequent legitimate requests to queue (up to Tomcat's `acceptCount`, default 100) and then be refused.

This constitutes a complete denial of service for the web3 API for the duration of the attack window, which is continuously renewable by the attacker.

### Likelihood Explanation

No authentication or account registration is required. The attacker needs only HTTP access to the public endpoint. The attack requires sending ~200 concurrent HTTP POST requests — trivially achievable with `curl`, `ab`, `wrk`, or any scripting language. The attack is repeatable: as soon as the thread pool drains (after ~10s), the attacker re-sends the burst. No special knowledge of the contract system is needed; targeting the zero address (`to: "0x0"`) with a computationally expensive calldata is sufficient.

### Recommendation

1. **Enforce a per-request concurrency limit** (e.g., a `Semaphore` or async dispatch with a bounded executor) so that at most N estimate requests execute concurrently, independent of the thread pool size.
2. **Move estimate execution off the servlet thread** using Spring's async request processing (`DeferredResult` / `Callable`) with a dedicated, bounded thread pool for EVM execution, so servlet threads are not held during computation.
3. **Enforce the `requestTimeout` wall-clock deadline inside `BinaryGasEstimator.search()`** at each iteration boundary, not only at DB query boundaries. Check `ContractCallContext.get().getStartTime()` at the top of each loop iteration and abort early if elapsed ≥ timeout.
4. **Add a per-source-IP rate limit** (e.g., via an ingress/gateway layer or a per-IP bucket in `ThrottleManagerImpl`) to prevent a single client from consuming the full RPS/gas budget.
5. **Reduce `maxGasEstimateRetriesCount`** or cap the total CPU time per estimate request.

### Proof of Concept

```bash
# Send 200 concurrent estimate requests with max gas to exhaust the thread pool
for i in $(seq 1 200); do
  curl -s -X POST http://<host>/api/v1/contracts/call \
    -H 'Content-Type: application/json' \
    -d '{
      "to": "0x0000000000000000000000000000000000000167",
      "gas": 15000000,
      "estimate": true,
      "data": "0x<expensive_calldata>"
    }' &
done
wait

# Immediately after, attempt a legitimate request — expect timeout or connection refused:
curl -v -X POST http://<host>/api/v1/contracts/call \
  -H 'Content-Type: application/json' \
  -d '{"to":"0x0000000000000000000000000000000000000167","gas":21000,"estimate":false}'
```

All 200 concurrent requests pass the gas bucket check (300,000 tokens consumed out of 750,000 capacity) and the RPS check (200 < 500). Each holds a servlet thread for up to 10–13 seconds. The final legitimate request receives a connection timeout or HTTP 503 for the duration of the attack window.

    private Bytes estimateGas(final ContractExecutionParameters params, final ContractCallContext context) {
        final var processingResult = callContract(params, context);
        final var gasUsedByInitialCall = processingResult.gasUsed();

        // sanity check ensuring gasUsed is always lower than the inputted one
        if (gasUsedByInitialCall >= params.getGas()) {
            return Bytes.ofUnsignedLong(gasUsedByInitialCall);
        }

        final var status = ResponseCodeEnum.SUCCESS.toString();
        final var estimatedGas = binaryGasEstimator.search(
                (totalGas, iterations) -> updateMetrics(params, totalGas, iterations, status),
                gas -> doProcessCall(params, gas, true),
                gasUsedByInitialCall,
                params.getGas());

        return Bytes.ofUnsignedLong(estimatedGas);
    }
