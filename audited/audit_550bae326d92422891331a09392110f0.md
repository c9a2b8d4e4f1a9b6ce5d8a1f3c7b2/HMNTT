### Title
Global Gas Bucket Exhaustion via Default Gas Value in ContractCallRequest Enables DoS on Mirror Node Contract Call Endpoint

### Summary
`ContractCallRequest` defaults the `gas` field to `15,000,000` when omitted by the caller. The global `gasLimitBucket` is sized at `scaleGas(gasPerSecond)` = 750,000 tokens (default `gasPerSecond` = 7,500,000,000 / 10,000). Each request with default gas consumes `scaleGas(15_000_000)` = 1,500 tokens at throttle-check time, before any execution or gas restoration occurs. An unauthenticated attacker sending 500 concurrent requests (the global `requestsPerSecond` cap) consumes all 750,000 tokens in a single burst, exhausting the bucket and causing every subsequent request to receive `GAS_PER_SECOND_LIMIT_EXCEEDED` until the bucket refills.

### Finding Description

**Code path:**

- `ContractCallRequest.java:37` — `private long gas = 15_000_000L;` — default applied when caller omits the field.
- `ContractController.java:40` — `throttleManager.throttle(request)` is called first, before execution.
- `ThrottleManagerImpl.java:40` — `gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))` — consumes scaled tokens synchronously.
- `ThrottleProperties.java:42-47` — `scaleGas(15_000_000)` = `Math.floorDiv(15_000_000, 10_000)` = **1,500 tokens** per request.
- `ThrottleConfiguration.java:35-44` — `gasLimitBucket` capacity = `scaleGas(gasPerSecond)` = `scaleGas(7_500_000_000)` = **750,000 tokens**.
- `ThrottleProperties.java:35` — `requestsPerSecond = 500` — global (not per-IP) RPS cap.

**Root cause:** Both the `rateLimitBucket` (500 tokens) and `gasLimitBucket` (750,000 tokens) are single global beans with no per-IP partitioning. Token consumption from `gasLimitBucket` happens at throttle-check time, before EVM execution and before any gas restoration via `restoreGasToBucket`. A single attacker can therefore consume the entire gas budget in one burst.

**Exploit flow:**
1. Attacker sends 500 concurrent POST `/api/v1/contracts/call` requests, each omitting the `gas` field (or explicitly setting it to 15,000,000).
2. All 500 requests pass the `rateLimitBucket` check (consuming all 500 RPS tokens).
3. All 500 requests pass the `gasLimitBucket` check, consuming 500 × 1,500 = 750,000 tokens — the entire bucket.
4. Any request arriving after this burst receives `ThrottleException("Gas per second rate limit exceeded.")` until the bucket refills (1 second).
5. Attacker repeats every second, keeping the bucket perpetually exhausted.

**Why the gas restoration check fails:** `restoreGasToBucket` in `ContractCallService.java:140-151` runs only after EVM execution completes. The throttle check at line 40 of `ThrottleManagerImpl` is synchronous and immediate. In a burst of 500 concurrent requests, all 500 token deductions occur before any single execution finishes and restores tokens. The restoration is therefore always racing behind the depletion. For calls that consume significant gas (complex contracts, loops, storage writes), `gasRemaining` is small and restoration is minimal, making the sustained attack viable without relying solely on the burst window.

### Impact Explanation
All users of the `/api/v1/contracts/call` endpoint on the mirror node are denied service for the duration of the attack. The `gasLimitBucket` refills at 750,000 tokens/second, but the attacker can re-exhaust it every second with a new burst of 500 requests. This results in continuous rejection of legitimate contract simulation requests. The mirror node is a read-only service and does not affect Hedera consensus directly; however, complete unavailability of `eth_call` simulation breaks all dApps, wallets, and tooling that rely on the mirror node for read queries, effectively making the network unusable from an application layer perspective.

### Likelihood Explanation
No authentication, API key, or IP-based rate limiting is required. Any unauthenticated HTTP client can send 500 POST requests per second. The attack requires no special knowledge of contract internals — simply omitting the `gas` field is sufficient. The attack is trivially scriptable with standard HTTP tooling (curl, wrk, ab) and is fully repeatable every second. The global (non-per-IP) nature of both buckets means a single attacker machine can saturate the entire service.

### Recommendation
1. **Per-IP rate limiting:** Introduce a per-client-IP bucket (e.g., via a servlet filter or Spring `HandlerInterceptor`) so one source cannot consume the global gas budget.
2. **Lower the default gas:** Reduce `ContractCallRequest.gas` default from `15_000_000` to a value representative of typical calls (e.g., `300_000`), reducing per-request token cost from 1,500 to 30.
3. **Decouple throttle from gas limit:** Consume tokens based on actual gas used (post-execution) rather than the declared gas limit, or use a two-phase reservation with a tight pre-execution cap.
4. **Enforce a lower per-request gas cap at throttle time:** Regardless of the declared gas, cap the tokens consumed per request to a fraction of the bucket (e.g., max 1% of bucket capacity per request).

### Proof of Concept
```bash
# Send 500 concurrent requests with no gas field (defaults to 15,000,000)
# Requires: wrk or parallel curl

# Using GNU parallel:
seq 500 | parallel -j500 curl -s -o /dev/null -w "%{http_code}\n" \
  -X POST http://<mirror-node>/api/v1/contracts/call \
  -H "Content-Type: application/json" \
  -d '{"to":"0x0000000000000000000000000000000000000001","data":"0x"}'

# Expected: first ~500 requests return 200 OK
# Immediately after: all requests return 429 with body containing
# "Gas per second rate limit exceeded."

# Repeat every second to sustain the DoS:
while true; do
  seq 500 | parallel -j500 curl -s -o /dev/null \
    -X POST http://<mirror-node>/api/v1/contracts/call \
    -H "Content-Type: application/json" \
    -d '{"to":"0x0000000000000000000000000000000000000001","data":"0x"}'
  sleep 0.9
done
```

**Verification:** A legitimate request sent during the attack window returns HTTP 429 with `"Gas per second rate limit exceeded."` confirming the bucket is exhausted.
