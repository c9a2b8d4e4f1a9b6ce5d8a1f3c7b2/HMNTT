### Title
Log Flooding via Exact-Match Bypass in `internalPaths` Suppression — Unprivileged Log I/O Amplification

### Summary
The `TracingMiddleware` in `rosetta/app/middleware/trace.go` suppresses verbose logging for internal monitoring paths using an exact-match map lookup. Any request whose URL is not an exact key in `internalPaths` — including prefix variants (`/health`), suffix variants (`/health/liveness/extra`), or query-string-appended variants (`/health/liveness?x=1`) — falls through to `log.Info`, generating a persistent log entry per request. With no rate limiting anywhere in the rosetta stack, an unauthenticated external attacker can trivially flood the log output and drive disk/stdout I/O above the 30% threshold.

### Finding Description
**File:** `rosetta/app/middleware/trace.go`, lines 55–59
**Function:** `TracingMiddleware`

```go
// line 47: path is the full RequestURI, including query string
path := request.URL.RequestURI()
// ...
if internalPaths[path] {   // line 55 — exact map key lookup only
    log.Debug(message)
} else {
    log.Info(message)      // line 58 — fires for ANY non-exact path
}
```

`internalPaths` (line 19) contains exactly three keys:
- `/health/liveness`
- `/health/readiness`
- `/metrics`

The lookup is a plain Go map key test — no prefix matching, no suffix matching, no query-string stripping. Therefore:

| Attacker-sent path | `internalPaths[path]` | Log level |
|---|---|---|
| `/health` | `false` | `log.Info` |
| `/health/liveness/extra` | `false` | `log.Info` |
| `/health/liveness?x=1` | `false` | `log.Info` |
| `/metrics/prometheus` | `false` | `log.Info` |
| `/metricss` | `false` | `log.Info` |

The test suite at `rosetta/app/middleware/trace_test.go` line 49 explicitly confirms this: `metricsPath + "s"` (`/metricss`) is asserted to produce `level=info` output.

The middleware is applied unconditionally to every inbound connection before the router (`main.go` lines 217–219):
```go
metricsMiddleware := middleware.MetricsMiddleware(router)
tracingMiddleware := middleware.TracingMiddleware(metricsMiddleware)   // wraps all traffic
corsMiddleware := server.CorsMiddleware(tracingMiddleware)
```

The router returns 404 for unregistered paths, but `TracingMiddleware` runs and logs **before** the response is inspected, so 404 responses are logged at `log.Info` just like valid ones.

No rate limiting, connection throttling, or request-per-IP cap exists anywhere in the rosetta codebase (confirmed: zero matches for `RateLimit`, `ratelimit`, `throttle` in `rosetta/**/*.go`).

### Impact Explanation
Each `log.Info` call emits a formatted line (~150–250 bytes) to stdout (configured at `main.go` line 54: `log.SetOutput(os.Stdout)`), which in containerized deployments is captured by the container runtime and written to disk. At 5,000 req/s — well within reach of a single commodity machine — the attacker generates ~1–1.25 MB/s of additional log I/O. On a node whose baseline log throughput is a few MB/min (normal health-check + API traffic), this easily exceeds a 30% I/O increase. The attack also consumes CPU for string formatting and goroutine scheduling per request, compounding the resource impact.

### Likelihood Explanation
The Rosetta API is designed to be publicly reachable (it is a blockchain node interface). No authentication is required for any endpoint. The attacker needs only an HTTP client and knowledge that the server exists. The attack is trivially repeatable, requires no credentials, no special protocol knowledge, and no prior access. A single `ab -n 1000000 -c 200 http://target/health/liveness/extra` invocation is sufficient to trigger the condition.

### Recommendation
1. **Strip query strings before the map lookup** — use `request.URL.Path` instead of `request.URL.RequestURI()` at line 47 so that `/health/liveness?x=1` is correctly suppressed.
2. **Use prefix matching** — replace the exact map lookup with a prefix check so that `/health/liveness/extra` is also suppressed:
   ```go
   isInternal := false
   for p := range internalPaths {
       if path == p || strings.HasPrefix(path, p+"/") || strings.HasPrefix(path, p+"?") {
           isInternal = true
           break
       }
   }
   ```
3. **Add HTTP rate limiting** — apply a per-IP or global request-rate middleware (e.g., `golang.org/x/time/rate`) before `TracingMiddleware` to bound the maximum log generation rate regardless of path.
4. **Log 404s at Debug level** — requests that result in a 404 (unregistered paths) could be demoted to `log.Debug` to reduce the value of flooding with invalid paths.

### Proof of Concept
```bash
# Target: publicly reachable Rosetta node at TARGET_HOST:8080
# No credentials required.

# 1. Confirm /health/liveness is suppressed (logged at Debug, silent at default Info level):
curl -s http://TARGET_HOST:8080/health/liveness
# → no Info log entry on server

# 2. Confirm prefix variant bypasses suppression:
curl -s http://TARGET_HOST:8080/health/liveness/extra
# → server emits: level=info msg="<IP> GET /health/liveness/extra (404) in Xms"

# 3. Flood with suffix/prefix variants to amplify log I/O:
ab -n 500000 -c 300 http://TARGET_HOST:8080/health/liveness/x
# → ~300 req/s sustained; each produces a log.Info line
# → monitor server disk/stdout I/O: expect >30% increase over baseline
# → same effect with /metrics/prometheus, /health?probe=1, etc.
