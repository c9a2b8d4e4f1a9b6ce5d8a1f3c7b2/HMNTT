### Title
DNS-Poisoning-to-Attacker-gRPC-Endpoint Injection via Unauthenticated Mirror REST Bootstrap (`network=other`)

### Summary
When `network=other`, `newClient()` bootstraps the Hiero SDK's node map by fetching `GET <mirrorRest>/api/v1/network/nodes` over plain HTTP with no TLS enforcement and no validation of the returned gRPC endpoints. An attacker who can poison DNS for the mirror REST hostname receives the HTTP request, returns a crafted `nodesEnvelope`, and causes `hiero.ClientForNetwork(netmap)` to route every subsequent signed transaction to attacker-controlled gRPC servers, enabling transaction suppression or relay/replay.

### Finding Description

**Exact code path:**

`pinger/sdk_client.go` lines 17–22 — the `"other"` branch unconditionally calls `buildNetworkFromMirrorNodes`, which calls `fetchMirrorNodeNetwork`: [1](#0-0) 

`pinger/mirror_node_client.go` lines 46–47 — the HTTP client is constructed with no TLS requirement; the URL is operator-supplied and the config comment explicitly shows `http://mirror-rest:5551` (plain HTTP): [2](#0-1) 

`pinger/mirror_node_client.go` lines 79–98 — the HTTP GET is issued with no certificate pinning, no HTTPS enforcement, and the full response body is decoded into `nodesEnvelope` without any integrity check: [3](#0-2) 

`pinger/mirror_node_client.go` lines 100–124 — every `ServiceEndpoint` in the attacker-controlled response is inserted verbatim into the `network` map that is returned to `hiero.ClientForNetwork`: [4](#0-3) 

**Root cause:** The bootstrap HTTP call has no transport-layer authentication (no HTTPS requirement, no certificate pinning, no HMAC/signature on the response). The only guard is `if len(network) == 0` (line 127), which the attacker trivially bypasses by returning one well-formed entry. [5](#0-4) 

**Failed assumption:** The code assumes the mirror REST endpoint is trusted and reachable only by legitimate infrastructure. It does not verify that the returned gRPC addresses belong to any known/expected set.

### Impact Explanation

After `hiero.ClientForNetwork(netmap)` is called with attacker-supplied endpoints, `client.SetOperator(opID, opKey)` attaches the operator's private key to that client: [6](#0-5) 

Every transaction the pinger submits is signed with the operator key and sent to the attacker's gRPC server. Concrete impacts:

- **Transaction suppression** — attacker silently drops all transactions; the pinger's liveness/readiness probes may not detect this.
- **Signed-transaction capture** — attacker receives fully signed, valid transactions and can replay them against the real network at will (Hiero transactions have a validity window; within that window replay is possible).
- **Operator key exposure** — the attacker observes the signed bytes; combined with known transaction structure, this can facilitate offline key-recovery attacks.

Severity: **High** — full compromise of the pinger's transaction integrity and potential replay on the real ledger.

### Likelihood Explanation

The `network=other` mode is the intended path for private/custom network deployments. In such deployments `cfg.mirrorRest` is commonly an internal hostname (e.g., `http://mirror-rest:5551`). Preconditions for the attack:

- **Kubernetes/container environments**: an attacker with the ability to create a pod in the same namespace can manipulate CoreDNS or perform ARP/NBNS poisoning on the pod network — no cluster-admin privileges required in many default configurations.
- **Plain HTTP**: the config default and documented example use `http://`, so TLS is not in the path; DNS poisoning alone is sufficient (no need to forge a TLS certificate).
- **One-shot bootstrap**: the poisoning only needs to succeed at startup; once `hiero.ClientForNetwork` is called the malicious map is locked in for the lifetime of the process.

Realistic likelihood: **Medium-High** in containerised deployments where the mirror REST URL is an internal DNS name served over HTTP.

### Recommendation

1. **Enforce HTTPS** for `cfg.mirrorRest` when `network=other`; reject `http://` URLs at config-load time (`pinger/config.go`).
2. **Pin the mirror REST TLS certificate** or at minimum validate against a configurable CA bundle so DNS poisoning cannot be combined with a self-signed cert.
3. **Validate returned endpoints** against a configurable allowlist of expected IP ranges or domain suffixes before passing them to `hiero.ClientForNetwork`.
4. **Re-fetch and diff** the node map periodically; alert and halt if endpoints change unexpectedly.
5. Consider using `hiero.ClientForName` with a hardcoded well-known network instead of the dynamic bootstrap for production deployments.

### Proof of Concept

**Preconditions:**
- Pinger is deployed with `HIERO_MIRROR_PINGER_NETWORK=other` and `HIERO_MIRROR_PINGER_REST=http://mirror-rest:5551` (plain HTTP, internal hostname).
- Attacker can respond to DNS queries for `mirror-rest` (e.g., via a rogue pod in the same Kubernetes namespace, or via ARP poisoning on the pod network).

**Steps:**

1. Attacker stands up an HTTP server on their controlled IP that responds to `GET /api/v1/network/nodes` with:
   ```json
   {
     "nodes": [{
       "node_account_id": "0.0.3",
       "service_endpoints": [{
         "domain_name": "attacker.example.com",
         "ip_address_v4": "",
         "port": 50211
       }]
     }],
     "links": {"next": null}
   }
   ```
2. Attacker poisons DNS so `mirror-rest` resolves to their IP.
3. Pinger starts (or is restarted). `buildNetworkFromMirrorNodes` issues `GET http://mirror-rest:5551/api/v1/network/nodes` — DNS resolves to attacker IP.
4. Attacker's HTTP server returns the crafted payload. `fetchMirrorNodeNetwork` decodes it; `len(network) == 1 > 0`, so no error is returned.
5. `hiero.ClientForNetwork({"attacker.example.com:50211": AccountID{0,0,3}})` is called; the SDK client now targets the attacker's gRPC server.
6. Attacker's gRPC server (listening on `attacker.example.com:50211`) receives every signed `CryptoTransfer` transaction. It can drop them (suppression) or forward them to the real network (replay within the transaction validity window).
