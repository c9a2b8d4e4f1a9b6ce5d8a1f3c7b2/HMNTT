### Title
Rate Token Consumed Without Rollback When Gas Bucket Check Fails — DoS via Rate Limit Exhaustion

### Summary
In `ThrottleManagerImpl.throttle()`, the `rateLimitBucket` token is consumed before the `gasLimitBucket` is checked. When the gas check fails, the already-consumed rate token is never restored. An unprivileged attacker can exploit this ordering to exhaust the shared global rate limit for all users without any of their requests being processed, causing a sustained denial of service.

### Finding Description

**Exact code path:**

`web3/src/main/java/org/hiero/mirror/web3/throttle/ThrottleManagerImpl.java`, lines 38–42:

```java
if (!rateLimitBucket.tryConsume(1)) {                                          // line 38 — rate token consumed here
    throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
} else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {  // line 40
    throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);                // line 41 — rate token already gone
}
```

`bucket4j`'s `tryConsume()` is an atomic consume-or-fail operation. When it returns `true`, the token is permanently deducted from the bucket. There is no transactional rollback. When line 38 succeeds and line 40 fails, the rate token is irrecoverably consumed.

**Root cause — failed assumption:** The code assumes that a gas-check failure is rare enough that the "wasted" rate token is inconsequential. In reality, an attacker can deliberately keep the gas bucket empty, turning every subsequent request into a rate-token drain with zero gas cost.

**No restore path for rate tokens:** The `restore()` method only refills `gasLimitBucket`:

```java
@Override
public void restore(long gas) {
    long tokens = throttleProperties.scaleGas(gas);
    if (tokens > 0) {
        gasLimitBucket.addTokens(tokens);   // only gas, never rate
    }
}
```

There is no equivalent mechanism to return the consumed rate token.

**Both buckets are global singletons** (Spring `@Bean`s in `ThrottleConfiguration`), shared across all callers with no per-IP partitioning.

**Exploit flow:**

1. **Phase 1 — exhaust the gas bucket.** Default `gasPerSecond = 7_500_000_000`, scaled by `GAS_SCALE_FACTOR = 10_000` → 750,000 bucket tokens. Default `maxGasLimit = 15_000_000` → 1,500 scaled tokens per request. The attacker sends ~500 max-gas requests in one second, draining the gas bucket entirely. The rate limit is also 500/second, so both buckets are drained simultaneously in one burst.

2. **Phase 2 — sustained rate-token drain.** With the gas bucket empty, the attacker continues sending requests at the full rate limit (500 req/s). Each request:
   - Passes the rate check → 1 rate token consumed (line 38)
   - Fails the gas check → `ThrottleException` thrown (line 41)
   - Rate token is **not** restored

3. **Result.** The attacker holds all 500 rate tokens/second. Legitimate users receive HTTP 429 for every request. The gas bucket refills at 750,000 tokens/second but the attacker never needs to consume gas tokens again — they only need to keep the rate bucket drained.

### Impact Explanation

The `rateLimitBucket` is the primary admission gate for all contract call requests to the mirror node's web3 endpoint. Exhausting it blocks all users from having their contract calls processed (EVM execution, and any downstream transaction submission to the Hedera network). This is a complete, sustained denial of service of the web3 API with no collateral cost to the attacker beyond sending cheap HTTP requests.

### Likelihood Explanation

- **No authentication required.** Any network-reachable client can call the web3 endpoint.
- **No special knowledge required.** The attacker only needs to know the endpoint exists and send requests with a high `gas` field.
- **Trivially repeatable.** A single script sending 500 HTTP POST requests per second is sufficient to maintain the attack indefinitely.
- **Low cost.** The attacker's requests are rejected at the gas check, so no EVM execution occurs on their behalf — the attack is computationally cheap.

### Recommendation

Check the gas bucket **before** consuming the rate token, or restore the rate token if the gas check fails:

**Option A — swap check order** (gas first, then rate):
```java
if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
    throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
} else if (!rateLimitBucket.tryConsume(1)) {
    throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
}
```
This ensures a rate token is only consumed when gas is available. (Note: the symmetric problem — gas consumed but rate fails — is less severe because gas tokens are already restored via `restore()` after processing.)

**Option B — restore rate token on gas failure:**
```java
if (!rateLimitBucket.tryConsume(1)) {
    throw new ThrottleException(REQUEST_PER_SECOND_LIMIT_EXCEEDED);
} else if (!gasLimitBucket.tryConsume(throttleProperties.scaleGas(request.getGas()))) {
    rateLimitBucket.addTokens(1);   // restore the consumed rate token
    throw new ThrottleException(GAS_PER_SECOND_LIMIT_EXCEEDED);
}
```

**Option C — add per-IP rate limiting** as a defense-in-depth measure so a single attacker cannot exhaust the global bucket.

### Proof of Concept

```python
import requests, threading

TARGET = "http://<mirror-node-host>/api/v1/contracts/call"
PAYLOAD = {
    "data": "0x",
    "to": "0x0000000000000000000000000000000000000001",
    "gas": 15000000,   # max gas — exhausts gas bucket fast
    "estimate": False
}

def flood():
    while True:
        requests.post(TARGET, json=PAYLOAD)

# Phase 1: exhaust gas bucket (500 requests in 1 second)
threads = [threading.Thread(target=flood) for _ in range(50)]
for t in threads: t.start()

# After ~1 second the gas bucket is empty.
# Every subsequent request consumes a rate token but fails at gas check.
# Legitimate users now receive HTTP 429 for all requests.
```

    @Bean(name = RATE_LIMIT_BUCKET)
    Bucket rateLimitBucket() {
        long rateLimit = throttleProperties.getRequestsPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(rateLimit)
                .refillGreedy(rateLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder().addLimit(limit).build();
    }

    @Bean(name = GAS_LIMIT_BUCKET)
    Bucket gasLimitBucket() {
        long gasLimit = throttleProperties.getGasPerSecond();
        final var limit = Bandwidth.builder()
                .capacity(gasLimit)
                .refillGreedy(gasLimit, Duration.ofSeconds(1))
                .build();
        return Bucket.builder()
                .withSynchronizationStrategy(SynchronizationStrategy.SYNCHRONIZED)
                .addLimit(limit)
                .build();
    }
