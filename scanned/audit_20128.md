# [M-01] Chainlink - Use latestRoundData instead of latestAnswer to run more validations

## Summary
This bug report is about UniswapV3Oracle.sol, a smart contract used to get the last WETH price. The issue is that when the method `latestAnswer` is called, it will return the last value, but it won't be able to check if the data is fresh. To check if the data is fresh, the method `latestRoundData` should be called instead. This method will return a tuple with the roundId, rawPrice, updateTime, and answeredInRound. The code should also include extra validations to ensure that the rawPrice is greater than 0, the updateTime is not 0, and that the answeredInRound is greater than or equal to the roundId. This will ensure that the data is fresh and up to date. Further information can be found in the Chainlink FAQ.

## Impact

`UniswapV3Oracle.sol` is calling `latestAnswer` to get the last WETH price. This method will return the last value, but you won't be able to check if the data is fresh.
On the other hand, calling the method `latestRoundData` allow you to run some extra validations

```
       (
          roundId,
          rawPrice,
          ,
          updateTime,
          answeredInRound
        ) = AggregatorV3Interface(XXXXX).latestRoundData();
        require(rawPrice > 0, "Chainlink price <= 0");
        require(updateTime != 0, "Incomplete round");
        require(answeredInRound >= roundId, "Stale price");
```

More information:  https://docs.chain.link/docs/faq/#how-can-i-check-if-the-answer-to-a-round-is-being-carried-over-from-a-previous-round
