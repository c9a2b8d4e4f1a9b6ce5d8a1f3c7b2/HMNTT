# [M-04] _deposit resetting user rewards can be used to grief them and make them loose rewards via depositForMember

## Summary
A bug has been identified in the code of the 2021-07-spartan GitHub repository. The bug is related to the function _deposit, which sets the variable mapMemberSynth_lastTime to a date in the future. This variable is also used to calculate rewards earned. The bug allows anyone to use the function depositForMember to make a donation for the member, which causes the member to lose all their accrued rewards. This can't be used for personal gain, but can be used to bring misery to others.

To mitigate this bug, it is recommended to harvest for the user right before changing mapMemberSynth_lastTime. This is the second submission under the same exploit.
