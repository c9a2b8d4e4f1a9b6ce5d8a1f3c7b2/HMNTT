# Possible assets loss

## Summary
This bug report is about the WETHGateway.sol contract in the Gearbox-protocol repository on Github. It states that if the repayAmount is greater than the amount, then the Gateway repays credit from its own assets. The recommendation is to call revert in this case. 

In the WETHGateway.sol contract, the Gateway repays credit from its own assets when the repayAmount is greater than the amount. This is a potential security risk, as the Gateway could be overpaying for the credit and losing money. To prevent this, the recommendation is to call revert when the repayAmount is greater than the amount. This will ensure that the Gateway does not overpay for the credit.

## Details
##### Description
If `repayAmount > amount`, this means that Gateway repays credit from its own assets:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/WETHGateway.sol#L191
##### Recommendation
We recommend to call revert in case `repayAmount > amount`.
