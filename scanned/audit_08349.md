# LACK OF RESOURCES AND RATE LIMITING

## Summary
The report describes a vulnerability in the BenQi API, which is a tool used to make requests to the website https://api.benqi.fi/. The vulnerability occurs when the API receives too many requests at the same time, causing it to use up too much of its resources (network, CPU, memory, and storage). This can be exploited by an attacker to overload the API, making it unavailable or unresponsive to new requests. The impact of this vulnerability is rated as a 3 out of 5, and the likelihood of it being exploited is rated as a 4 out of 5.

## Details
##### Description

Requests to the `BenQi` API (`https://api.benqi.fi/`) consume resources such as network, CPU, memory, and storage. This vulnerability occurs when too many requests arrive simultaneously, and the API does not have enough compute resources to handle those requests. An attacker could exploit this vulnerability to overload the API by sending more requests than it can handle. As a result, the API becomes unavailable or unresponsive to new requests.
