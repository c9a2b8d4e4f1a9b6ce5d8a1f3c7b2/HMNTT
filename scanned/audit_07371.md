# Funds Locked Due to the Whitelist

## Summary
The bug report discusses an issue with the `onlyAllowedProviders` check in the `ExternalRequestsManagerBetaV1.sol` contract. This check is applied when a provider tries to cancel a pending request, but if the provider gets delisted after creating the request, they are unable to cancel it. This can happen if the whitelist is inactive and then activated. The recommended solution is to remove the `onlyAllowedProviders` check from the `cancelMint()` and `cancelBurn()` functions.

## Details

- https://github.com/resolv-im/resolv-contracts/blob/a36e73c4be0b5f233de6bfc8d2c276136bf67573/contracts/beta/ExternalRequestsManagerBetaV1.sol#L148

The `onlyAllowedProviders` check is applied in cancel operations. If a provider gets delisted after creating a pending request, they won't be able to cancel it.

This situation may arise when the whitelist was inactive and then got activated via `setWhitelistEnabled()`.

##### Recommendation

We recommend removing the `onlyAllowedProviders` check from `cancelMint()` and `cancelBurn()`.
