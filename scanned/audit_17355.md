# No money back.

## Summary
This bug report talks about an issue with the AccessTokenContract.sol code, specifically with the paidMint() and referralMint() functions. The problem is that there is no handling for when a user pays more than the required amount, resulting in the extra funds not being returned to the user. The recommendation is to add processing for this case and to be cautious of the reentrancy vulnerability. The re-audit comment states that the issue has been resolved. The post-audit recommendations include adding processing to the referral Mint method, moving storage changes to the end of the function, and using a different function to transfer funds.

## Details
**Description**

AccessTokenContract.sol, paidMint() and referralMint()

There is no handling in case the user pays more than required. Thus extra funds are not returned to the user.

**Recommendation**

Add the processing of this case, returning the balance of the necessary funds to the user. Be careful and consider the reentrancy vulnerability.

**Re-audit comment**

Resolved.

Post-audit:

1. Add processing to the Metropolis World Passport.referral Mint method

2. Move storage changes to the end of the function.

3. Use function safeTransferETH or Address.sendValue() instead of transfer
