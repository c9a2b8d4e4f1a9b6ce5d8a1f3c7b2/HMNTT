# Use SafeMath in all math operations

## Summary
This bug report is about a vulnerability in the Brave International's Basic Attention Token Crowdsale. The vulnerability was related to some unchecked math operations. The risk of this vulnerability was that it could potentially lead to a security breach. To fix this issue, Brave International released a commit, 09a1038c2ac0148b93876057fc056ae2a872fb88, which checked the math operations and made them safe. Thus, the vulnerability was fixed and the security breach was avoided. It is important to always perform checked operations to ensure that safety and security are maintained.

## Details
Most math operations are safe, but there’s still a few that are unchecked ([these two](https://github.com/brave-intl/basic-attention-token-crowdsale/blob/17a5f8440a256a6dc5d8dd894b9615182c2901b2/contracts/BAToken.sol#L84-L85), for instance). It’s always better to be safe and perform checked operations.


**Update**: fixed in commit [09a1038c2ac0148b93876057fc056ae2a872fb88](https://github.com/brave-intl/basic-attention-token-crowdsale/commit/09a1038c2ac0148b93876057fc056ae2a872fb88).
