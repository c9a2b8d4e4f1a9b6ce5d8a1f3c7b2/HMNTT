# `expectedLiquidityLimit` can be equal to zero

## Summary
This bug report concerns the PoolService.sol contract in the Gearbox Protocol. The issue is that the `expectedLiquidityLimit` in the contract is not set to a value in the constructor, and it can be equal to zero. This can lead to issues with the functioning of the contract.

The recommendation is to set a value to the `expectedLiquidityLimit` in the constructor. This will ensure that the contract functions as intended and prevent any issues from arising.
