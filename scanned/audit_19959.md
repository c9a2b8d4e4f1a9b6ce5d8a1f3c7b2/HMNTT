# [M-02] Crash Eth Oracle On Any LogicCallEvent

## Summary
This bug report discusses a vulnerability with a medium severity and high likelihood of happening. In the code path `eth_oracle_main_loop`, the method `get_last_checked_block` is called, which can potentially lead to a panic. This is due to the method `from_log` in the struct `LogicCallExecutedEvent` being unimplemented. This can also be triggered in the method `check_for_events` in the same struct. As a result, attestations will be frozen until the bug is patched.

The recommended mitigation steps for this bug are to implement the method `from_log` in the struct `LogicCallExecutedEvent`. This will help prevent the panic from happening, and allow attestations to be processed normally.

## Vulnerability details

**Severity: Medium**
**Likelihood: High**

In `eth_oracle_main_loop`, `get_last_checked_block` is called. Followed by:
```rust=
let logic_call_executed_events = web3
            .check_for_events(
                end_search.clone(),
                Some(current_block.clone()),
                vec![gravity_contract_address],
                vec![LOGIC_CALL_EVENT_SIG],
            )
            .await;
```
and may hit the code path:
```rust=
        for event in logic_call_executed_events {
            match LogicCallExecutedEvent::from_log(&event) {
                Ok(call) => {
                    trace!(
                        "{} LogicCall event nonce {} last event nonce",
                        call.event_nonce,
                        last_event_nonce
                    );
                    if upcast(call.event_nonce) == last_event_nonce && event.block_number.is_some()
                    {
                        return event.block_number.unwrap();
                    }
                }
                Err(e) => error!("Got ERC20Deployed event that we can't parse {}", e),
            }
        }
```

But will panic at `from_log` here:
```rust=
impl LogicCallExecutedEvent {
    pub fn from_log(_input: &Log) -> Result<LogicCallExecutedEvent, GravityError> {
        unimplemented!()
    }
    // snip...
}
```
It can/will also be triggered here in `check_for_events`: 
```rust=
let logic_calls = LogicCallExecutedEvent::from_logs(&logic_calls)?;
```

Attestations will be frozen until patched.

## Recommendation
Implement the method.
