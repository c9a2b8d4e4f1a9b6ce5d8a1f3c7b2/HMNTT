# [M-05]

## Summary
This bug report, identified as M-05, was submitted by jessicapointing. It is about an inactive KNOT being able to successfully stake. The expected behaviour of the system is that the stake function should revert if an inactive KNOT is trying to stake, however the stake function succeeds and the inactive KNOT is able to stake. The property violated is <code>cannotStakeIfKnotIsInActive</code>, which sets the <code>isActive</code> state of the <code>blsPubKey</code> to <code>false</code>. The stake function checks if KNOT is not registered (<code>!isKNOTRegistered</code>) and if it is no longer part of the syndicate (<code>!isNoLongerPartOfSyndicate</code>) and reverts accordingly, but it does not check whether the KNOT is inactive.

The potential fix suggested is to check the <code>isActive</code>/<code>isKnotRegistered</code>/<code>isNoLongerPartOfSyndicate</code> state of a KNOT before executing the function. This principle can be applied to other functions in the code which may reveal more bugs.

The assessment of the bug is low severity as staking without an active knot does not earn rewards, and the stake can be retrieved. The bug will be fixed by checking for active status when staking. However, teryanarmen (Certora) commented that inactive knots can still earn rewards since all rewards are compiled together, and deregistering a knot can be triggered at any time by anyone, making it medium severity.

## Details
### Description of the problem

An inactive KNOT can successfully stake.

Expected behaviour of the system: The stake function should revert if an inactive KNOT is trying to stake.

Actual behaviour of the system: The stake function succeeds and the inactive KNOT stakes.

Property Violated: `cannotStakeIfKnotIsInActive`

In this rule, the `isActive` state of the `blsPubKey` is set to `false`. The stake function is then called with this `blsPubKey` and the expected behaviour is that the function would revert because the KNOT is inactive. However, the stake function succeeds and an inactive KNOT can therefore successfully stake. The stake function checks if KNOT is not registered (`!isKNOTRegistered`) and if it is no longer part of the syndicate (`!isNoLongerPartOfSyndicate`) and reverts accordingly but does not check whether the KNOT is inactive.

Lines of code: <https://github.com/Certora/2023-01-blockswap-fv/blob/certora/contracts/syndicate/Syndicate.sol# L216>

### Potential Fix
```

(,,,,,bool isActive) = getStakeHouseUniverse().stakeHouseKnotInfo(blsPubKey);
if (!isKnotRegistered[_blsPubKey] || isNoLongerPartOfSyndicate[_blsPubKey] || !isActive) revert KnotIsNotRegisteredWithSyndicate();
```
