# The Admin Can Take the Fees From the `FeeCollector`

## Summary
The client has marked a bug as "Fixed" in the file `vesting-contracts-v3/contracts/NativeTokenVestingManager.sol` with the code `c94e904607eaf9315963b9d98fe3eb312e1a5596`. The bug was reported because the `NativeTokenVestingManager.withdrawAdmin()` function, which is only supposed to be used by the admin, was incorrectly calculating the withdrawable amount. This could allow the admin to withdraw funds intended for the fee collector. The recommendation is to use a separate variable to track fees and restrict admin withdrawals to avoid this issue.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `c94e904607eaf9315963b9d98fe3eb312e1a5596`. The client provided the following explanation:

> Added tracking of the fee reserve.

**File(s) affected:**`vesting-contracts-v3/contracts/NativeTokenVestingManager.sol`

**Description:** The `NativeTokenVestingManager.withdrawAdmin()` function, callable by the admin, computes the withdrawable amount as `address(this).balance - numTokensReservedForVesting` via `amountAvailableToWithdrawByAdmin()`. However, this is the same calculation used in `withdrawGasFee()`, which is intended for the fee collector. Since fees are not tracked separately, the admin could inadvertently or maliciously withdraw the fee collector’s funds.

**Recommendation:** Consider using a separate variable to track fees intended for the fee collector and restrict admin withdrawals to `address(this).balance - numTokensReservedForVesting - reservedForFees`.
