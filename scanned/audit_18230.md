# Logic issue.

## Summary
The bug report is about an issue in a code file called StakingB_1.sol. On line 512, there is a problem with the variable "lastRewardBlock" not being initialized when the pool is set. This leads to the first reward being calculated incorrectly, causing problems with the reward amount. The recommendation is to make sure that lastRewardBlock is initialized when setting up the pool.

## Details
**Description**

StakingB_1.sol Line 512
“lastRewardBlock” must be initialized when the pool is set. In other case first reward will be
counted as a reward in range [0; last block number] that is a numerous amount.

**Recommendation**:

Initialize lastRewardBlock in pool setting.
