# Broken account must be deleted

## Summary
This bug report is about an issue with the CreditManager.sol contract. If a user attempts to transfer tokens from an account and the transfer is reverted, then the account must be deleted. This is to ensure that the user is not able to continue to make transfers from the same account. The recommendation is that if a user attempts to transfer tokens and the transfer is reverted, the account should be deleted. This will help to prevent further attempts to transfer tokens from the same account.

## Details
##### Description
In case if some tokens cannot be transferred from the account, then this account must be deleted:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L451
##### Recommendation
We recommend to delete account in case some transfers are reverted.
