# Lack of incentives to liquidate small positions

## Summary
The report states that there is a bug in the protocol that affects the liquidation of positions when their margin value falls below a certain threshold. This bug has a high severity level and has been resolved. The issue is that liquidators are not being incentivized to carry out these liquidations, which could result in bad debt remaining in the protocol. This is because the gas cost of liquidation may exceed the profits earned from liquidating positions with very low margins. To fix this, the recommendation is to set a minimum position size to ensure that liquidations are profitable.

## Details
**Severity**: High

**Status**: Resolved

**Description**

The protocol expects positions to be liquidated when their margin value falls below the liquidation threshold. Liquidators should be incentivized to carry out these liquidations. If there is no profit to be earned, no one will undertake the liquidation. However, if a user opens many positions with very low margins, and these become undercollateralized, the gas cost of liquidation may exceed the profits from liquidating these positions. This could result in bad debt remaining in the protocol.

**Recommendation**: 

Set a minimum position size to ensure that liquidations are profitable.
