# Logic issue.

## Summary
The bug report is about a problem with the StakingB_1.sol code, specifically on Line 512. The issue is that the "lastRewardBlock" must be initialized when the pool is set, otherwise the first reward will be counted as a reward in a range that is too large. The recommendation is to initialize the lastRewardBlock when setting the pool.

## Details
**Description**

StakingB_1.sol Line 512
“lastRewardBlock” must be initialized when the pool is set. In other case first reward will be
counted as a reward in range [0; last block number] that is a numerous amount.

**Recommendation**:

Initialize lastRewardBlock in pool setting.
