# A swap with permit can be blocked if a frontrunner swaps using copied permit

## Summary
This bug report is about the SwapFacade contract in the Argon project. This contract does not check that the msg.sender of a transaction is the owner of a permit. This means that a frontrunner can use permits of other users available from mempool, causing their transactions to revert. This would result in the user having to build new calldata without a permit.

The recommendation is to decode permits to extract their owner, then check that msg.sender is this owner. An example of this is the SafeERC20 contract from the 1inch project. 

In summary, this bug report is about a security issue with the SwapFacade contract in the Argon project. A workaround is to decode permits to extract their owner and check that msg.sender is this owner. This can be done using the 1inch project's SafeERC20 contract as an example.

## Details
##### Description

Facade does not check that msg.sender is the owner of a permit.
So, a frontrunner can use permits of other users available from mempool, so that their transactions would revert (as permit are used by the frontrunner).

- https://github.com/BarterLab/argon/blob/22239aaccdbb78d4aa7ac5c4d0859a9b31c0fc00/contracts/SwapFacade.sol#L58-L64

A user will receive revert and will have to build new calldata, without permit.

##### Recommendation

We recommend decoding permits to extract their owner, then check that msg.sender is this owner. Some example used by 1inch:
- https://github.com/1inch/solidity-utils/blob/60fa6eba06f50f2acac5286dea5fb54f206e2434/contracts/libraries/SafeERC20.sol#L158-L245
