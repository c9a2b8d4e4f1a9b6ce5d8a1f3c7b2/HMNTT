# Missing test coverage report

## Summary
A test coverage report is an important tool for software developers as it shows which parts of the code have been tested and which have not. Without this report, it is impossible to know if there are parts of the code never executed by the automated tests. This means that for every change, a full manual test suite has to be executed to make sure that nothing is broken or misbehaving.

To prevent this, it is recommended to add the test coverage report and make sure that it covers at least 95% of the source code. This will provide developers with an overview of the code that has been tested and which parts have not been tested yet. This makes it easier to identify any potential issues and make sure that all parts of the code are tested.
