# Centralization Risk

## Summary
This bug report discusses a medium severity issue that has been resolved. It mentions that the contract owner has the ability to change certain parameters that can affect how the contracts work. This includes functions related to rewards and general settings. The report also mentions a more serious issue where the contract owner can withdraw ERC20 assets, which is concerning. The recommendation is to use multisig wallets or a governance module to prevent this issue from happening.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

There are multiple instances where the contract owner can change protocol parameters that affect the behavior of the contracts. These include calls to several functions like updating rewarding details and general setters/mutators. More severely though is the action to with draw ERC20 asset from the contract, this alleviates the concern, hence recommendation on how to deal with it should be followed.

**Recommendation**: 

Consider using multisig wallets or having a governance module
