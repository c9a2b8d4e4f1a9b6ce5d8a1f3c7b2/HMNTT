# Unchecked possible zero address

## Summary
This bug report is about a high severity issue that has been resolved. The problem is in a contract called Identity.sol, specifically in a function called execute. There is a possibility that the "to" function parameter could be zero, and this is not being checked. This means that if the "to" address is zero, the value being sent will be lost. The recommendation is to add a check to make sure the "to" address is not zero, and if it is, the transaction should be reverted.

## Details
**Severity**: High

**Status**: Resolved

**Description**

In contract Identity.sol, at line 90 in function execute the to function parameter can be zero and is not checked. In case the to address is zero, the following to.call{value: value}(data) would return true and send the value to the zero address.

**Recommendation**: 

Add a sanity check for the to address variable to not be zero and revert otherwise.
