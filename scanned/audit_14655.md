# Precision Loss Issue In Weighted Math

## Summary
The math module has a bug that affects the calculation of swap amounts. This is due to a precision error in the log_exp_math::pow function used for exponentiation. This can lead to incorrect values being returned, causing issues with the swap amounts. The bug can be replicated by inputting specific values and observing the incorrect output. To fix this, the precision of log_exp_math::pow needs to be improved and the return value should be set to x when log_exp_math::pow(x, 1) is called. Additionally, the invariant value should be checked to ensure it does not decrease after a swap. This bug has been fixed in the latest patch, 572dabb.

## Details
## Vulnerability in math Module

The math module calculates the amount taken in during a swap based on the amount given out, the balances in the pool, and the weights of the assets. `calc_in_given_out_internal` is responsible for this calculation, which involves using `log_exp_math::pow` to perform the required exponentiation. 

`log_exp_math::pow` used by `calc_in_given_out_internal` is vulnerable to precision errors, which may return incorrect values. For instance, the function may incorrectly calculate `1.0000000002 ** 1` as `1.0`. This precision issue may be exploited in `calc_in_given_out_internal`, leading to a return value of zero despite a non-zero `amount_out` value.

## Proof of Concept

1. Assume \( b_I = b_O = 10,000,000,000 \) ($10,000) and weights of the assets, \( w_I = w_O = 50 \).
2. When attempting to calculate the `amount_in` for 100 as the `amount_out`, the returned value is 97, whereas the expected value is 10.
3. The loss of precision becomes more significant as the asset balances increase.
4. Let’s assume \( b_I = b_O = 1,000,000,000,000 \) ($1,000,000) and \( w_I = w_O = 50 \).
5. If the `amount_out` is set to 200, the returned value is zero, indicating that we may perform a swap by giving zero tokens and receiving 200 tokens in return.

## Remediation

- Improve the precision of `log_exp_math::pow` and set the return value to \( x \) when `log_exp_math::pow(x, 1)` is called.
- Additionally, ensure that the invariant value does not decrease after a swap by asserting its non-decreasing nature, helping to avoid any rounding problems that may lead to a loss of value in the pool.

## Patch

Fixed in `572dabb` by improving precision in `log_exp_math::pow` and short-circuiting the return value to \( x \) when `log_exp_math::pow(x, 1)` is called.
