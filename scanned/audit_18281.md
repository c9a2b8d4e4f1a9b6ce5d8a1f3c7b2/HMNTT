# Risk of overflow.

## Summary
This bug report discusses an issue with the calculation of a value called accTokenPerShare. The calculation uses a very large number, 1e62, which is close to the maximum size of an unsigned integer, 2**256-1. This can cause an overflow, especially when the pool rate is high and the rateDenominator is low. The recommendation is to handle or prevent this overflow exception.

## Details
**Description**

accTokenPerShare calculation utilizes 1e12*1e18*1e32 that is equal to 1e62. Max size of
unsigned integer (256) is 2**256-1 that is ~1e77. With high pool rate and low
rateDenominator such amount can cause an overflow.

**Recommendation**:

Handle or prevent an overflow exception.
