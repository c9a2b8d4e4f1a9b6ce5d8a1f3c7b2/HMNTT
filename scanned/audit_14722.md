# Admin can bypass the upgrade delay by `delegate` and `emergencyUpgrade`

## Summary
This bug report is about a vulnerability that allows an admin to bypass the upgrade delay flow and immediately change the address of the wallet core. This vulnerability is caused by a sequence of steps that involves the use of the `transferDelegation`, `acceptDelegation`, `upgradeFallback` and `emergencyUpgrade` functions.

The recommendation for this bug is to improve the upgrade delay flow to prevent the admin from bypassing it. This can be done by making changes to the sequence of steps that were used to exploit the vulnerability.

Overall, this bug report highlights a vulnerability in the upgrade delay flow that allows an admin to bypass it and immediately change the address of the wallet core. The recommendation is to improve the upgrade delay flow to fix this vulnerability.

## Details
##### Description
The admin can bypass the upgrade delay flow and immediately change the address of the wallet core by following the sequence:
- `transferDelegation` (to a new `delegate` account, controlled by the admin)
- `acceptDelegation` (from the `delegate` account)
- `upgradeFallback` (from the admin account)
- `emegrencyUpgrade` (from the `delegate` account)
##### Recommendation
We recommend improving the upgrade delay flow.
