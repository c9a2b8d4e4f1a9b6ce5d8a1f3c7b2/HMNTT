# Use of Transfer event to log token creation

## Summary
This bug report is about the crowdsale contract, which uses the Transfer event to log the creation of tokens. The Transfer event of the ERC20 standard has been overloaded, which could be an issue as other software (such as wallets or exchanges) could be relying on these semantics. This is an ongoing discussion in the ERC20 standardization process, and the stance is that a different event should be used. The bug has been fixed and the new event is available in the given link.

## Details
The crowdsale contract [uses the Transfer event](https://github.com/erkmos/moeda/blob/b2bf23119d563e251b6f16b29b642bac43e76a64/contracts/MoedaToken.sol#L55) to log the creation of tokens, with a *from* field set to the zero address. This is overloading the semantics of the `Transfer` event of the ERC20 standard, which might not be a good idea as other software (such as wallets or exchanges) could be relying on these semantics. This is [an ongoing discussion](https://github.com/ethereum/EIPs/pull/621) in the ERC20 standardization process, but our stance on it is that [a different event](https://github.com/OpenZeppelin/zeppelin-solidity/blob/609869f08732fce8cd946996d326c15889e441c8/contracts/token/MintableToken.sol#L20) should be used.


**Update:** Created a new event, fixed in <https://github.com/erkmos/moeda/commit/68ff6544976c2bfa655cae28e69ab09d813dbc46>.
