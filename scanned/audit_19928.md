# The contract owner has too many permissions

## Summary
The bug report states that there is an issue with the functions "addBlacklist()" and "destroyBlackFunds()" in a contract. These functions allow the contract owner to add any address to a blacklist and burn user tokens without the user's permission. The recommendation is to only allow the burning of user tokens if the user has given permission or to provide more information on why this function is necessary.

## Details
**Description**

Contract owners can add any address to the blacklist with function addBlacklist() and burn user
tokens with function destroyBlackFunds() without user allowance.

**Recommendation**:

Burn user tokens only when it’s allowed by user or share additional information why this
function needs to be included.
