# Possible duplication of data

## Summary
This bug report is about a potential issue with the Gearbox Protocol's AccountFactory contract. If the configurator calls a particular function multiple times, it is possible for data duplication to occur. To prevent this from happening, the report recommends adding a check that only new data is pushed to the array. This will ensure that no duplicate data is added, and the issue is resolved.

## Details
##### Description
If configurator calls function several times, then data duplication is possible:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/AccountFactory.sol#L301
##### Recommendation
We recommend adding a check that only new data is pushed to array.
