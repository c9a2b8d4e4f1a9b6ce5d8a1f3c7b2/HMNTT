# Locked Ether

## Summary
The client has marked a bug as "Fixed" and provided an explanation. The bug affects three files, namely `NFTMarketSimple`, `NFT721`, and `NFT1155`. The bug causes transfers of Ether to ignore their success, resulting in the Ether being stored in unintended contracts without a withdraw function. This means the Ether will be locked forever. To fix this, the client recommends adding checks for success in three functions: `NFT721.redeem()`, `NFT1155.redeem()`, and `NFTMarketSimple.sendAssets()`. They also suggest either adding a revert function in `NFTMarketSimple` or an `ownerOnly` withdraw function to prevent this issue.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `3630c5af28e7e8489e2e77106bb3a478a477cd5e`. The client provided the following explanation:

> As recommended, we have added checks for the success of sending funds to the NFT721.redeem(), NFT1155.redeem(), and NFTMarketSimple.sendAssets() functions.

**File(s) affected:**`NFTMarketSimple`, `NFT721`, `NFT1155`

**Description:** Various transfers of Ether ignore the success of a transfer. Therefore, the Ether will be stored in a contract that was not intended to store Ether. The lack of a withdraw function will cause the Ether transferred to these contracts to be locked forever. Return values for `call()` should be checked to ensure success. This occurs in:

*   `NFT721.redeem()`
*   `NFT1155.redeem()`
*   `NFTMarketSimple.sendAssets()`

**Recommendation:** Contracts without owners, such as `NFT721` and `NFT1155`, should revert if the transfer to the NFT creator fails. Either the `sendAssets()` function in `NFTMarketSimple` should revert or an `ownerOnly` withdraw function should be added.
