# Event is probably missing

## Summary
A bug has been identified in the CoverPoolFactory.sol contract, which is part of the CoverProtocol/cover-core-v2 repository on GitHub. The method updateDeployGasMin does not emit an event, unlike other methods for updates, such as updateCoverPoolImpl and updateCoverImpl.

It is recommended that an event be created and emitted in the method updateDeployGasMin, in order to make the contract more consistent with the other methods for updates. This will ensure that all methods are emitting events when they are used.

## Details
##### Description
At line https://github.com/CoverProtocol/cover-core-v2/tree/513f5e502a8e8a623729c2c3480fca4e80fdef39/contracts/CoverPoolFactory.sol#L89 `CoverPoolFactory`'s method `updateDeployGasMin` should probably emit an event as well as other methods for update (e.g. `updateCoverPoolImpl` or `updateCoverImpl`, etc.) do.

##### Recommendation
Create a suit event and emit the one in the method.
