# No validation of the address parameter value in contract constructor

## Summary
This bug report is about a problem in the BaseWrapper.sol code in the Yearn Vaults repository. The code assigns the variable "token" the value of the constructor input parameter "_token". However, this parameter is not checked beforehand, so if the value is zero, the contract must be redeployed in order to set the variable.

The recommendation is to add a check of the input parameter to zero before initializing the variables. This will ensure that the value of the parameter is not zero and the contract does not have to be redeployed.

## Details
##### Description
The variable is assigned the value of the constructor input parameter. But this parameter is not checked before this. If the value turns out to be zero, then it will be necessary redeploy the contract, since there is no other functionality to set this variable.

- At the line https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L45 the `token` variable is set to the value of the `_token` input parameter.

##### Recommendation
It is necessary to add a check of the input parameter to zero before initializing the variables.
