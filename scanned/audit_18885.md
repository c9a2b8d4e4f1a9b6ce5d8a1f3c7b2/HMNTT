# [M-09] no use of safeMint() as safe guard for users 

## Summary
A bug was discovered in Vault.sol, a smart contract used in the Sandclock project. The deposit() function calls claimers.mint() and depositers.mint(), which does not guarantee that the receiver of the NFT can accept them, leaving them vulnerable to reentrancy attacks. A manual code review was done to find the bug. To fix the issue, the developer should use _safeMint() instead of mint(), which checks if a user can properly accept an NFT and reverts otherwise. This will provide a guard to protect the user from reentrancy attacks.

## Impact
In Vault.sol the deposit() function eventually calls claimers.mint() and depositers.mint().  Calling mint this way does not ensure that the receiver of the NFT is able to accept them.  _safeMint() should be used with reentrancy guards as a guard to protect the user as it checks to see if a user can properly accept an NFT and reverts otherwise.
