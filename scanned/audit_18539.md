# State variables saleOpen & presaleOpen of Smart Contract CryptoBearWatchClub are allocated to regulate current state of contract.

## Summary
The bug report is about a smart contract called CryptoBearWatchClub.sol. The issue is that there are two state variables, saleOpen and presaleOpen, which are used to indicate the current phase of the contract. The recommendation is to use only one state variable with an enum data type that has values for each phase (PRIVATE_SALE, PRESALE, AUCTION). This will make the code more organized and easier to understand.

## Details
**Description**

CryptoBearWatchClub.sol, saleOpen & presaleOpen

**Recommendation**:

use one state variable to indicate current phase, e.g. state variable with data type enum with
values (PRIVATE_SALE, PRESALE, AUCTION).
