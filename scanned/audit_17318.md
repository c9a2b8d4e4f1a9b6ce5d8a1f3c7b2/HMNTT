# Lack of event emission

## Summary
This bug report is about two functions in the Origin Protocol's OUSD Governance project: the `setRewardsTarget` and `setInflation` functions. These functions do not emit relevant events after executing sensitive actions. This means that off-chain clients following the contracts' activity are not notified of the changes.

To fix this bug, the Origin Protocol has issued a pull request (#96) which has now been implemented. Additionally, they suggest indexing the event parameters to further help tracking. 

In conclusion, this bug report was about the lack of relevant events being emitted after sensitive changes in the OUSD Governance project. This bug has now been fixed in pull request #96, and the Origin Protocol suggests indexing the event parameters for better tracking.

## Details
The following functions do not emit relevant events after executing sensitive actions:


* The [`setRewardsTarget` function](https://github.com/OriginProtocol/ousd-governance/blob/2b9761606d4ac4062b69367ebbad88220cea45ce/contracts/RewardsSource.sol#L122) changes the address `OGV` [tokens are minted to](https://github.com/OriginProtocol/ousd-governance/blob/2b9761606d4ac4062b69367ebbad88220cea45ce/contracts/RewardsSource.sol#L44) as part of the staking reward system.
* The [`setInflation` function](https://github.com/OriginProtocol/ousd-governance/blob/2b9761606d4ac4062b69367ebbad88220cea45ce/contracts/RewardsSource.sol#L95) deletes and optionally updates the rewards slopes.


Consider emitting events after sensitive changes take place to facilitate tracking and notify off-chain clients following the contracts’ activity.


**Update**: *Fixed in pull request [#96](https://github.com/OriginProtocol/ousd-governance/pull/96/). In addition, consider indexing the event parameters.*
