# Discrepancy between comment and code

## Summary
The bug report is about the CentrallyIssedToken contract. It has a constructor parameter named _owner which is used as the upgradeMaster. The comment in line 29 of the contract states an intent to allocate the initial balance to the owner, but the balance is actually allocated to msg.sender. This means that either the code is incorrect and should be changed to use the owner variable, or the comment should be updated to accurately reflect the code.

## Details
`CentrallyIssedToken` has a constructor parameter named `_owner`, which is used as the `upgradeMaster`. The comment in [line 29](https://github.com/Storj/storj-contracts/blob/2bdeb27c0216d2f0889b6e7363d8a84b54cd7f39/contracts/CentrallyIssuedToken.sol#L29) states an intent to allocate the initial balance to the *owner*; however, the balance is allocated to `msg.sender`. If it’s an error, use the owner variable; otherwise, update the comment.
