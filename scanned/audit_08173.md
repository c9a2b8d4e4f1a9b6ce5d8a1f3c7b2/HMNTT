# ADMIN CAN CHANGE SESSION DETAILS AFTER THE START OF A SESSION

## Summary
This bug report describes a problem with the `setType` function in the `NewStakingSaloon.sol` code. The function allows the owner of the contract to change details about running sessions, which could lead to users unknowingly burning their NFTs if the owner abuses their privileges or if their private key is stolen. The code location of the function is provided, along with a recommendation to remove the function. The impact of this bug is rated as a 5 out of 10 and the likelihood of it occurring is rated as a 1 out of 10. The Seascape team has since solved the issue by removing the function.

## Details
##### Description

Testing revealed that the `setType` function allowed the owner of the contract to change details regarding running sessions. Should this privilege be abused or the private key of the owner be stolen, users might risk of burning their NFTs without their knowledge when unstaking. This is due to the function allowing to set the NFTs in a certain session to burn and whether users get a bonus by playing the game.

Code Location
-------------

`NewStakingSaloon.sol` Lines#

```
//change type about session
  function setType(uint256 _sessionId, bool _burn, bool _specify) external onlyOwner{
      Session storage _session = sessions[_sessionId];
      require(block.timestamp <= (_session.startTime + _session.period), "saloon: session end");
      _session.burn    = _burn;
      _session.specify = _specify;
  }

```

##### Score

Impact: 5  
Likelihood: 1

##### Recommendation

***SOLVED***: The `Seascape team` removed the function.
