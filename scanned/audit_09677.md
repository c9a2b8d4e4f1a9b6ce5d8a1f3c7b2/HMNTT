# [M-04] Wrong usage of revBp() in deposit() in fee calculation

## Summary
The bug report states that there is an issue with the fee calculation in the `_deposit()` function. This is considered a low severity issue because it only affects the accounting of fees and not the functionality of the code. However, it is likely to occur in every call to the `deposit` or `mint` function. The code currently calculates the fee by slicing it from the amount using the `revBp()` logic, but this is incorrect as it does not take into account the full amount being deposited. The recommendation is to use a different calculation method that takes into account the full amount and the fee.

## Details
**Severity**

**Impact:** Low, because wrong accounting of fee

**Likelihood:** High, because it will happen in each call to deposit/mint

**Description**

In `_deposit()` function code calculates fee like this:

```solidity
        // slice the fee from the amount (gas optimized)
        if (!exemptionList[_receiver])
            claimableAssetFees += _amount.revBp(fees.entry);
```

And the `revBp()` logic is:

```solidity
    function revBp(
        uint256 amount,
        uint256 basisPoints
    ) internal pure returns (uint256) {
        return mulDiv(amount, basisPoints, BP_BASIS - basisPoints);
    }
```

As the `amount` in the deposit is not sliced and it is `deposit + fee` so the calculation for fee is wrong.

**Recommendations**

Fee calculation should be:

```solidity
             claimableAssetFees  += _amount * fees.entry / (BP_BASIS  + fees.entry)
```
