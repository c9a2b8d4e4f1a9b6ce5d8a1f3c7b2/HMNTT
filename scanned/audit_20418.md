# [H-06] Incorrect type conversion in the contract ABC makes users unable to burn FSD tokens

## Summary
A bug has been found in the function `_calculateDeltaOfFSD` of contract `ABC`. This bug occurs when the parameter `_reserveDelta` is negative. When this happens, the function incorrectly converts the `int256` type parameter to `uint256`, resulting in an extremely large number. This large number causes a SafeMath operation `sub` at line 43 to revert, and thus the FSD tokens cannot be burned. The bug can be triggered by simply calling `fsd.burn` after a successful `fsd.mint`. To mitigate the issue, it is recommended to use the solidity function `abs` to get the `_reserveDelta` absolute value.

## Vulnerability details

## Impact

The function `_calculateDeltaOfFSD` of contract `ABC` incorrectly converts an `int256` type parameter, `_reserveDelta`, to `uint256` by explicit conversion, which in general results in an extremely large number when the provided parameter is negative. The extremely large number could cause a SafeMath operation `sub` at line 43 to revert, and thus the FSD tokens cannot be burned. (`_reserveDelta` is negative when burning FSD tokens)

## Proof of Concept

Simply calling `fsd.burn` after a successful `fsd.mint` will trigger this bug.
