# Gas overflow during iteration (DoS)

## Summary
This bug report is about a problem that occurs in the PieFactoryContract.sol, CallFacet.sol, and BasketFacet.sol code. Each iteration of the cycle requires a gas flow, and if more gas is required than it is allocated to record one block, all iterations of the loop will fail. The specific lines of code affected are listed in the report.

The recommendation given is to add a check for the maximum possible number of elements of the arrays. This will help to ensure that the gas allocated is sufficient for the task at hand, and that the loop will not fail due to lack of gas.

## Details
##### Description
Each iteration of the cycle requires a gas flow.
A moment may come when more gas is required than it is allocated to record one block. In this case, all iterations of the loop will fail.

##### Recommendation
We recommend adding a check for the maximum possible number of elements of the arrays.
