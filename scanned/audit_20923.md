# It is possible to process a non-existing array element or skip an array element

## Summary
This bug report identifies two issues in the code of the Sushiswap Bentobox project. 

The first issue is in the `LendingPair.sol` file at line 467. This line is working with the elements of the `borrowFractions` array in a loop. For each element of the `users` array, there must be an element of the `borrowFractions` array. However, if an error is made when transferring data for these arrays, it could result in referring to a nonexistent element of the array or not processing any element at all. This would cause the `liquidate()` function to work incorrectly.

The second issue is in the `BentoBox.sol` file at line 122. This line is working with the elements of the `amounts` array in a loop. For each element of the `tos` array, there must be an element of the `amounts` array. Again, if an error is made when transferring data for these arrays, it could result in referring to a nonexistent element of the array or not processing any element at all. This would cause the `transferMultipleFrom()` function to work incorrectly.

The recommendation to fix these issues is to add a condition so that the length of the `users` array is equal to the length of the `borrowFractions` array. This would ensure that the functions are working correctly and that there are no errors when transferring data for these arrays.
