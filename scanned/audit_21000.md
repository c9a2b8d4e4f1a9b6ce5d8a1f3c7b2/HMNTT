# Solidity files has no license declaration.

## Summary
The bug report suggests that it is important to include a license in every Solidity file. This is a recommendation and not a bug, but it is still important to follow. A Solidity file is a type of file used in blockchain technology, specifically in the programming language used to create smart contracts. A license is a legal document that outlines the terms and conditions for using the code in the file. By including a license in every Solidity file, it ensures that anyone who uses the code is aware of the terms and conditions and can use it legally. This is important because without a license, there may be confusion or potential legal issues regarding the use of the code. Therefore, it is recommended to always include a license in every Solidity file to avoid any complications.

## Details
**Recommendation**:

Specify license in every Solidity file.
