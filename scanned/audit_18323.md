# [M-19] sharesToTokenAmount: Division by zero

## Summary
This bug report is about a vulnerability in the public 'sharesToTokenAmount' function, which does not check if the denominator 'totalSharesMinted[_tokenAddress]' is zero. This can cause the function to revert, which can lead to an interruption in the services provided by the contract. The recommended mitigation step for this vulnerability is to return 0 in case 'totalSharesMinted[_tokenAddress]' is zero. This will ensure that the function does not revert and the services provided by the contract are not interrupted.

## Vulnerability details

## Impact
The public `sharesToTokenAmount` function does not check if the denominator `totalSharesMinted[_tokenAddress]` is zero.
Neither do the callers of this function. The function will revert.
Calling functions like `getFeeAccumulatedOnNft` and `sharesToTokenAmount` from another contract should never revert.

## Recommended Mitigation Steps
Return 0 in case `totalSharesMinted[_tokenAddress]` is zero.
