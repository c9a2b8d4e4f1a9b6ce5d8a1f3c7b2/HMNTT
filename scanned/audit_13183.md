# Centralization of setFEYProductImplementation()

## Summary
This bug report discusses a medium severity issue in the contract FEYProductFactory. The problem is that the implementation address of the Product contract can be changed by a malicious admin, which could lead to users losing their funds if a malicious implementation contract is used to deploy new products. The bug has been resolved, but it is recommended to prevent the changing of the implementation contract and add more decentralization to the contract through the use of a governance mechanism or a multisig.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In contract FEYProductFactory, the implementation address of the Product contract can be changed anytime by a malicious admin. This can result in users losing their funds to the attacker via a malicious implementation contract, if new products are deployed using this implementation address.


**Recommendation**: 

It is advised to disallow changing of the implementation contract. It is also advised to add more decentralization to the contract and roles, such as using a governance mechanism or a multisig.
