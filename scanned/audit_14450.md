# Centralization Risk

## Summary
This bug report highlights an issue where the admin of a project has too much power and control over its contracts. This can potentially cause problems as the admin can make changes that affect the project's tokenomics without proper oversight. The recommendation is to implement governance methods or use multisig wallets to prevent this issue. The developers have acknowledged the problem and plan to address it by utilizing multisig wallets in the future.

## Details
**Severity**: Medium

**Status**: Acknowledged

**Description**

Admin enjoys too much authority in contracts. The general theme is that admin has power to call several state changing functions also affecting the tokenomics of the project. Some functions can be highly severe to be left out, controlled by one wallet.

**Recommendation** 

Apply governance methods or use multisig wallets.

**Fix** - Developers stated they are planning to utilize multisig wallets.
