# Improper Price Deviation Calculation Formula

## Summary
The function get_price_diff_pct in the thala-protocol-v1/sources/oracle.move RUST file is not correctly calculating the percentage of price deviation. The current implementation uses the new_price as the denominator if new_price is greater than old_price. This is incorrect and should use old_price as the denominator in both cases. This bug has been fixed in the latest patch, bd1e275.

## Details
## Price Deviation Calculation

The `get_price_diff_` function is responsible for computing price deviation. To calculate the percentage of price deviation, the formula should be:

\[
\text{Percentage Deviation} = \left( \frac{\text{diff(new\_price, old\_price)}}{\text{old\_price}} \right) \times 100
\]

The current implementation incorrectly uses `new_price` as the denominator if `new_price` is greater than `old_price`.

## Implementation in RUST

The function to get the difference between two prices `a` and `b` in percentage is defined as follows:

```rust
// Get the difference between two prices a and b in percentage. Result is rounded to nearest integer
fun get_price_diff_pct(a: FixedPoint64, b: FixedPoint64): u64 {
    if (fp64::gt(&a, &b)) {
        fp64::decode(fp64::mul(fp64::div_fp(fp64::sub_fp(a, b), a), 100))
    } else if (fixed_point64::lt(&a, &b)) {
        fp64::decode(fp64::mul(fp64::div_fp(fp64::sub_fp(b, a), b), 100))
    } else {
        0
    }
}
```

## Remediation

To fix the issue, use `b` (the old price) as the denominator in both cases.

## Patch

The fix was implemented in commit `bd1e275` by using `b` as the denominator in both cases.
