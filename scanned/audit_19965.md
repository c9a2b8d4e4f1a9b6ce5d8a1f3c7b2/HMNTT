# Update Solidity version

## Summary
The bug report states that the project is using outdated versions of Solidity, a programming language used for creating smart contracts. The recommended versions to use are 0.6.12, 0.7.6, or 0.8.6, but the project is using 0.5.17 and 0.4.23. This is considered a medium issue and the recommendation is to update to the latest stable version of Solidity and use only one version within the project's contracts.

## Details
**Description**

The project utilises Solidity 0.5.17 and 0.4.23 versions which are marked as outdated. The
standard auditor’s checklist states for the usage of the latest version of the Solidity - 0.6.12,
0.7.6 or 0.8.6. Thus the issue is marked as Medium.

**Recommendation**:

Use the latest stable release of Solidity. Also, use the single Solidity version within the
contracts set.
