# [M-05] Add a timelock to BaseStrategy:setPerfFeePct

## Summary
This bug report is about a vulnerability in the BaseStrategy.sol file of the 2022-01-sandclock repository on GitHub. The vulnerability is that functions that set key/critical variables are not put behind a timelock. This means that users are not given the trust they deserve, as the key/critical variables can be changed without restriction. The vulnerability was discovered using VS Code and the proof of concept can be found in the link provided. The recommended mitigation step is to add a timelock to setter functions of key/critical variables, so that users can be given more trust.
