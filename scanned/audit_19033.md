# Possible incorrect setting value for `maxLeverageFactor`

## Summary
This bug report relates to the Gearbox Protocol Contracts code on Github. Specifically, the `maxLeverageFactor` can be set to 0 which is not desirable. To fix this issue, the code should be amended to include a check that `_maxLeverageFactor > 0`. This would ensure that the `maxLeverageFactor` cannot be set to 0 and any other value that is not desirable.

## Details
##### Description
`maxLeverageFactor` can be set to 0:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L722
##### Recommendation
We recommend adding a check that `_maxLeverageFactor > 0`.
