# `SubnetKeys[]` Should Be Removed

## Summary
This bug report describes an issue with the `kill()` function in the `GatewayManagerFacet.sol` contract. When a subnet is killed, the `totalSubnets` variable is decreased and the corresponding `subnets` entry is deleted. However, the `subnetKeys` array, which holds the keys of registered subnets, is not updated, leading to potential errors. The recommendation is to also update the `subnetKeys` array when a subnet is killed. The bug has been resolved and the severity is considered medium.

## Details
**Severity** - Medium

**Status** - Resolved

**Description**


Inside the `kill()` function in the contract `GatewayManagerFacet.sol` , we decrease the `totalSubnets` at L135 , then delete `subnets[subnet.id.toHash()]` .  
Since `subnetKeys` array hold the keys of the registered subnets , `subnetKeys[subnet.id]` should also be removed when a subnet is killed.

**Recommendation**:

Along with `totalSubnets`  and `subnets[]` also update the `subnetKeys[]` when a subnet is killed
