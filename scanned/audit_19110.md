# [M-08] Wrong comment in `getFee`

## Summary
The bug report is about a discrepancy between the comments and the code in the `ThreePieceWiseLinearPriceCurve.getFee` function. The comment states that the total and input must be less than the cap, but the code only checks if the input is less than the cap. The recommended mitigation steps include clarifying the desired behavior and reconciling the code with the comments. The severity of the issue is debated, with some saying it is a medium issue and others saying it is a low issue. The bug has been fixed in line 92.

## Details
_Submitted by cmichel_

The `ThreePieceWiseLinearPriceCurve.getFee` comment states that the total + the input must be less than the cap:

> If dollarCap == 0, then it is not capped. Otherwise, **then the total + the total input** must be less than the cap.

The code only checks if the input is less than the cap:

```solidity
// @param _collateralVCInput is how much collateral is being input by the user into the system
if (dollarCap != 0) {
    require(_collateralVCInput <= dollarCap, "Collateral input exceeds cap");
}
```

#### Recommended Mitigation Steps

Clarify the desired behavior and reconcile the code with the comments.
