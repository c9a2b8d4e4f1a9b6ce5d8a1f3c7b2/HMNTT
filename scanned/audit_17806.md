# [M-12] User will lose funds

## Summary
A bug was discovered in the AuraClaimZap.sol contract located at line 224-226. The bug results in users losing funds due to a missing else condition. This bug occurs when a user calls the claimRewards function with the option LockCvx set to false. The code will then transfer the user's cvxBalance to the contract, but not lock it since the else condition is missing. This leaves the user's cvx funds stuck in the contract. 

The recommended mitigation steps for this bug are to add an if condition that checks if the user has enabled the lock for cvx before transferring the funds. This will prevent users from losing their funds due to the bug. 

The severity of the bug was initially deemed to be a 3, but 0xMaharishi (Aura Finance) disagreed and suggested that it should be a 2 at most. LSDan (judge) then decreased the severity to medium, as the bug does not open up an attack vector which would allow a malicious actor to lock a user's funds. 0xMaharishi (Aura Finance) then resolved the bug with the code fixes in the code-423n4/2022-05-aura#6 and aurafinance/aura-contracts#84 pull requests.
