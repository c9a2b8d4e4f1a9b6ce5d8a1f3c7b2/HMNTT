# Missing call to _disableInitializers

## Summary
This bug report states that there is a medium severity issue with an uninitialized contract that can be exploited by an attacker. This issue affects both a proxy and its implementation contract, potentially causing harm to the proxy. To prevent this from happening, it is recommended to use the _disableInitializers() function in the constructor. More information on how to do this can be found in the provided link. It is advised to consider implementing this solution to avoid any potential attacks. The bug has been resolved.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

An uninitialized contract can be taken over by an attacker. This applies to both a proxy and its implementation contract, which may impact the proxy. To prevent the implementation contract from being used, you should invoke the _disableInitializers() function in the constructor to automatically lock it when it is deployed, more information can be found here.

**Recommendation**: 

Consider calling _disableInitializers()in the constructor.
