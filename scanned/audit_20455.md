# There is no check on the result of the function

## Summary
This bug report is about an issue with the `approve()` function in the ERC20 standard. According to the standard, the function should return a boolean value, but when it is called from the `_approveAll()` function in the Strategy.sol contract, this value is not being processed. This could lead to the function returning a False value. 

To fix this issue, it is recommended to add a check of the return value. This would ensure that the function is returning the expected value and that any errors are caught and handled appropriately.

## Details
##### Description
According to the ERC20 standard, the `approve()` function returns a boolean value. But in the contract at lines https://github.com/orbxball/weth-maker/blob/39cfbee59bcebfce19a5b9ac6f11fb84f3ab7b23/contracts/Strategy.sol#L68-L75, after the call to the `_approveAll()` function, this values are not processed. A situation may arise that a False will return. 

##### Recommendation
It is recommended to add a check of the return value.
