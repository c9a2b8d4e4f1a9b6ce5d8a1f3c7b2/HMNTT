# Missing test coverage report

## Summary
This bug report is about the lack of an automated test coverage report. This report is necessary to know if parts of the code are never executed by the automated tests. Without this report, a full manual test suite has to be executed every time a change is made in the code. The bug report suggests adding the test coverage report and making it reach at least 95% of the source code.

The bug report has been updated, and a coverage report has been added. However, this report needs to be generated every time the code is changed. Therefore, it is suggested to use a reporting service such as Codecov or Coveralls. Additionally, although the reported line coverage is 100%, the branch coverage is still 53%.

## Details
There is no automated test coverage report. Without this report it is impossible to know whether there are parts of the code never executed by the automated tests; so for every change, a full manual test suite has to be executed to make sure that nothing is broken or misbehaving.


Consider adding the test coverage report, and making it reach at least 95% of the source code.


***Update:*** *Fixed. A* [*coverage report*](https://github.com/propsproject/props-token-distribution/tree/7ead8c55da3770c4d39db8c89a8aa20ec62e97b1/coverage) *has been added. Note that this report has to be generated every time the code is changed, so consider removing it from the repository and using instead a reporting service like* [*Codecov*](https://codecov.io/) *or* [*Coveralls*](https://coveralls.io/)*. Also note that although reported line coverage is 100%, branch coverage is 53%.*
