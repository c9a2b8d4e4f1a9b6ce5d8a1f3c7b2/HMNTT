# Presence Of Infinite Loop

## Summary
The function prepareSettleRaffles in RaffleLib is used to prepare a list of raffles that can be settled based on certain criteria. However, there is a bug where if a raffle has not sold any tickets, the loop does not increment the counter and continues to check the same NFT, causing an infinite loop. This bug has been fixed in the code with the patch 4e7c0b8 and the loop counter is now properly incremented in this case.

## Details
## prepareSettleRaffles in RaffleLib

`prepareSettleRaffles`, in RaffleLib, prepares a list of raffles eligible for settlement based on specific criteria. It takes a collection state and a list of NFT IDs (`nftIds`) and checks whether each NFT satisfies the conditions for settlement.

## Raffle.sol (SOLIDITY)

```solidity
        if (raffle.endTime >= nowTs) revert Errors.ActivityHasNotCompleted();
        if (raffle.ticketSold == 0) {
            continue;
        }
        [...]
    }
    [...]

```

For a particular NFT, if the raffle has sold no tickets, i.e., `raffle.ticketSold == 0`, it will not satisfy the condition `raffle.ticketSold > 0`; it will not be considered eligible for settlement. However, the loop does not increment the loop counter (`i`) when `raffle.ticketSold == 0`, which results in the loop continuously checking the same NFT, creating an infinite loop.

## Remediation

Ensure to increment the loop counter in the case `raffle.ticketSold == 0`.

## Patch

Fixed in commit `4e7c0b8`.
