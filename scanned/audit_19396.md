# [H-01] YaxisVaultAdapter.sol#withdraw() will most certainly fail

## Summary
This bug report is about a vulnerability in the code of the WatchPug project. The code in question involves the `vault.withdraw()` and `IDetailedERC20(_token).safeTransfer(_recipient, _amount)` functions. The issue is that due to precision loss in `_tokensToShares()` and `vault.withdraw()`, the actual token withdrawn from `vault.withdraw()` will be less than the `_amount` specified. This will cause `IDetailedERC20(_token).safeTransfer(_recipient, _amount)` to revert due to insufficant balance. The simulation that was run shows that this issue will fail `99.99%` of the time unless the pps is equal to 1e18.

The recommendation is to change the code to the following:

```solidity=68
function withdraw(address _recipient, uint256 _amount) external override onlyAdmin {
    address _token = vault.getToken();
    uint256 beforeBalance = IDetailedERC20(_token).balanceOf(address(this));
    
    vault.withdraw(_tokensToShares(_amount));

    IDetailedERC20(_token).safeTransfer(
        _recipient,
        IDetailedERC20(_token).balanceOf(address(this)) - beforeBalance
    );
}
```
