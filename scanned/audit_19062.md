# `creditManager` isn't checked

## Summary
This bug report is about a possible exploit in the Gearbox Protocol's contract code. It is possible to give an unlimited amount of approval to a poisoned contract. This could allow malicious actors to exploit the code and cause harm to the protocol. To fix this, the report recommends adding a check that the creditManager is a system contract. This would ensure that the code is safe and secure from malicious actors and their attempts to exploit it.

## Details
##### Description
It is possible to give unlimited approve to poisoned contract:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/LeverageActions.sol#L222
##### Recommendation
We recommend adding a check that `creditManager` is a system contract.
