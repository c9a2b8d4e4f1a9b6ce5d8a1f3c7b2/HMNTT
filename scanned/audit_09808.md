# Centralized Power / A Malicious Admin Can Deny Claiming

## Summary
The bug report is about a protocol that allows for an unlimited number of admins to make changes to the current merkleRoot and currentVersion variables. This can be used to prevent users from claiming in a given round. The client has marked the bug as "Fixed" and provided an explanation that they have added a recommendation to address the issue. The recommendation is to only allow the owner to invoke the function, which will greatly reduce the risk.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `1204484`. The client provided the following explanation:

> Added the same recommendation for gracePeriod it doesn't apply to owner though in case there is a problem with a distribution

Review comment: by ensuring admins cannot update `merkleRoot` again within a `lockdownPeriod`, now the risk is reduced.

**Description:** The protocol allows for an unlimited number of admins. An admin has the power to invoke the function `createDistribution`, which changes the current `merkleRoot` and the `currentVersion` variable. This can be used to prevent users from claiming in a given round.

**Recommendation:** Consider if this is intended, if not, then only allow the owner to invoke this function. This will only allow one address to be able to invoke this function, thus greatly reducing the risk.
