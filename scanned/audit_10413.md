# Chainlink’s `latestRoundData` might return stale or incorrect results

## Summary
This bug report discusses an issue with the PriceAggregator contract using Chainlink oracle to retrieve price data. The problem is that there is no check in place to ensure that the data returned is not stale or incorrect. This could lead to inaccurate prices being used. The recommendation is to add a check for the updatedAt parameter returned from latestRoundData() and compare it to a staleness threshold. The bug has been resolved.

## Details
**Severity**: Medium

**Status**:  Resolved

**Description**
 
The fulfill function in the PriceAggregator contract use Chainlink oracle as a second oracle to retrieve price data by calling latestRoundData. However, there's a risk that this data may be stale or incorrect due to various reasons related to Chainlink oracles. There is no check if the return value indicates stale data. This could lead to stale prices according to the Chainlink documentation:
https://docs.chain.link/docs/historical-price-data/#historical-rounds

**Recommendation**: 

Check the updatedAt parameter returned from latestRoundData() and compare it to a staleness threshold.
