# Possible reentrancy

## Summary
This bug report is about a potential reentrancy issue in the Gearbox Protocol's code. Reentrancy is a type of vulnerability in which a malicious actor can call a function multiple times before it completes its intended task. In this case, the issue is found in the following functions: CurveV1, UniswapV2, UniswapV3, YearnV2, and WETHGateway. The recommendation to fix this issue is to add a 'nonReentrant' modifier to the code.
