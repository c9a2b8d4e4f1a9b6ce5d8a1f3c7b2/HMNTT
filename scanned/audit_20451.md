# There is no check on the result of the function

## Summary
This bug report describes an issue with the `approve()` function in the contract found at the given link. This function is part of the ERC20 standard, and is expected to return a boolean value. However, the `_approveAll()` function does not process this return value, which could lead to a False value being returned. It is recommended to add a check of the return value to ensure the expected outcome.

## Details
##### Description
According to the ERC20 standard, the `approve()` function returns a boolean value. But in the contract on lines https://github.com/orbxball/yfi-maker/blob/63af6fcfa536073f00d652f49befd18e429b5500/contracts/Strategy.sol#L69-L76, after the call to the `_approveAll()` function, this values are not processed. A situation may arise that a False will return. 

##### Recommendation
It is recomended to add a check of the return value.
