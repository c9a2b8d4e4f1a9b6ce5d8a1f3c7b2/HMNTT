# [M-13] manager.allowedVaults check missing for add/remove strategy

## Summary
This bug report is about a vulnerability in the code-423n4/2021-09-yaxis repository. It is classified as a medium severity issue because it allows a strategist to accidentally or maliciously add or remove strategies on unauthorized vaults. The proof of concept is provided in the form of 4 Github links. Manual Analysis was used to find this vulnerability. The recommended mitigation steps to fix this vulnerability is to add a manager.allowedVaults check in addStrategy() and removeStrategy() functions.

## Vulnerability details

## Impact
The manager.allowedVaults check is missing for add/remove strategy like how it is used in reorderStrategies(). This will allow a strategist to accidentally/maliciously add/remove strategies on unauthorized vaults.

Given the critical access control that is missing on vaults here, this is classified as medium severity.
