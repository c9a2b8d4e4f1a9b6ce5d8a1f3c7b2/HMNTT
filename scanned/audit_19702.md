# [M-05] createBasket re-entrancy

## Summary
This bug report is about a vulnerability in the function createBasket in Factory. The vulnerability is that this function should also be nonReentrant as it interacts with various tokens inside the loop and these tokens may contain callback hooks. This means that the function could be called multiple times and cause errors. The recommended mitigation step for this vulnerability is to add the nonReentrant modifier to the declaration of createBasket. This will prevent the function from being called multiple times, thus avoiding any potential errors.

function `createBasket` in Factory should also be `nonReentrant` as it interacts with various tokens inside the loop and these tokens may contain callback hooks.
