# Centralization risk as the owner can disqualify all minted token Ids

## Summary
This bug report discusses an issue with the protocol where the owner can disqualify all token IDs for the reward and withdraw all reward tokens after a wave has ended. This can cause a problem where users who have minted badges and won rewards will no longer be considered winners. The recommendation is to update the logic to prevent the owner from disqualifying all token IDs and to use a multi-sig account for the owner. This will help avoid this issue in the future.

## Details
**Severity**: Medium

**Status**: Acknowledged

**Description**

In the protocol, there can be a state where the owner can disqualify all token IDs for the reward and then withdraw all the reward tokens once the wave has ended.

This state will invalidate the requirement that users can be winners if they have minted badges and win rewards.

**Recommendation**: 

update the logic to not allow admin to disqualify all tokenIds. Also, use multi-sig account for the owner.
