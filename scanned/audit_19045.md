# Unnecessary inheritance from `Proxy`

## Summary
A bug has been identified in the `yVault` contract, which is part of the Gearbox Protocol. All functions called from the proxy are not working, as the storage is not the same as specified in the code. The issue can be resolved by removing the inheritance from the `Proxy`. This will ensure that the functions are working properly, and the bug is fixed.

## Details
##### Description
All functions for `yVault` called from proxy wouldn't work because the storage is not the same:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/integrations/yearn/YearnPriceFeed.sol#L17
##### Recommendation
We recommend removing inheritance from `Proxy`.
