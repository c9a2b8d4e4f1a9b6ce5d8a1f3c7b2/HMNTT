# Wrong accounting for totalBorrowed in the InfinityPool.writeOff function ✓ Fixed

## Summary
This bug report is about the `InfinityPool.writeOff` function in the `code/src/Pool/InfinityPool.sol` file. The part of the code in question is from line 271 to 287. The issue is that when the `totalBorrowed` is decreased, it is decreased by the `lostAmt` value instead of the original `account.principal` value. This does not acknowledge the loss, and needs to be changed. The resolution to this bug is that the `totalBorrowed` should be decreased by the original `account.principal` value. This has been fixed in the pull request at the following link: https://github.com/glif-confidential/pools/pull/440/files.

## Details
#### Resolution



[Fixed](https://github.com/glif-confidential/pools/pull/440/files).


#### Description


Here is a part of the `InfinityPool.writeOff` function:


**code/src/Pool/InfinityPool.sol:L271-L287**



```
// transfer the assets into the pool
// whatever we couldn't pay back
uint256 lostAmt = principalOwed > recoveredFunds ? principalOwed - recoveredFunds : 0;

uint256 totalOwed = interestPaid + principalOwed;

asset.transferFrom(
 msg.sender,
 address(this),
 totalOwed > recoveredFunds ? recoveredFunds : totalOwed
);
// write off only what we lost
totalBorrowed -= lostAmt;
// set the account with the funds the pool lost
account.principal = lostAmt;

account.save(router, agentID, id);

```
The `totalBorrowed` is decreased by the `lostAmt` value. Instead, it should be decreased by the original `account.principal` value to acknowledge the loss.
