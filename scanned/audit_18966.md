# Incorrect length of input data

## Summary
There is a problem with the code in the CreditManager contract. The length of a certain array, called "paths", can be less than the allowed number of tokens in another array called "creditFilter". This can cause unexpected behavior and errors. It is recommended to add a check to make sure both arrays have the same length.

## Details
##### Description
`paths.length` can be < `creditFilter.allowedTokensCount()`:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L816
##### Recommendation
We recommend adding a check that arrays have equal lengths.
