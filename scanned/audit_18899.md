# Update Solidity version.

## Summary
This bug report highlights an issue with using an outdated version of Solidity (programming language for smart contracts) in contracts and the hardhat config. The recommended solution is to update to the latest stable version 0.8.10 to avoid potential problems.

## Details
**Description**

Contracts utilize “^0.8.0” version and the hardhat config uses 0.8.0 version which is already
outdated, since the OpeZeppelin team has fixed several issues up to the version 0.8.10

**Recommendation**:

Use the latest stable Solidity version 0.8.10.
