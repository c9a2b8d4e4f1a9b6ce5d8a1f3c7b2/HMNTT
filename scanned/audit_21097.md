# [M03] Lack of event emission after sensitive actions

## Summary
This bug report is about the Notional Finance contracts. It states that certain functions do not emit relevant events after sensitive actions. The `initialize` function of the `Governed` contract should emit a `DirectorySet` event, the `setContract` function of the `Governed` contract should emit a `SetContract` event, the `_setDependencies` function of the `Governed` contract should emit a `SetContract` event, the `setParameter` function of `CashMarket.sol` should emit a `ParametersSet` event, the `setDependencies` function of the `Directory` contract should emit a `SetContract` event, and the `settleMaturedAssets` and `settleMaturedAssetsBatch` functions of `Portfolios.sol` should emit `SettleAccount` and `SettleAccountBatch` events, respectively.

In general, the bug report is suggesting that events should be emitted after state-changing sensitive actions take place, in order to facilitate tracking and notify off-chain clients of the contracts' activity. This would help to ensure that the relevant information is available to all parties involved.

## Details
The following functions do not emit relevant events after sensitive actions.


* The [`initialize` function](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/utils/Governed.sol#L21) of the `Governed` contract should emit a `DirectorySet` event.
* The [`setContract` function](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/utils/Governed.sol#L33) of the `Governed` contract should emit a `SetContract` event, different from the one in the `Directory` contract.
* The [`_setDependencies` function](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/utils/Governed.sol#L38) of the `Governed` contract should emit a `SetContract` event.
* The [`setParameter` function](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/CashMarket.sol#L94) of `CashMarket.sol` should emit a `ParametersSet` event, showing the new values of `G_MATURITY_LENGTH` and `G_NUM_MATURITIES`, or emit an individual event for each variable updated (e.g.: `MaturityLengthSet`, `NumMaturitiesSet`), showing the old and new values for each of them.
* The [`setDependencies` function](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/utils/Directory.sol#L29) of the `Directory` contract should emit a `SetContract` event, as is being done in the [`setContract` function](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/utils/Directory.sol#L62) of the same contract.
* In `Portfolios.sol`, when calling [`settleMaturedAssets`](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/Portfolios.sol#L571) a [`SettleAccount` event is emitted](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/Portfolios.sol#L575)  if the account has any assets that were settled. However, when calling [`settleMaturedAssetsBatch`](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/Portfolios.sol#L584) no `SettleAccount` events are emitted. If the function is not `calledByEscrow`, then the [`SettleAccountBatch` event is emitted](https://github.com/notional-finance/contracts/blob/b6fc6be4622422d0e34c90e77f2ec9da18596b8c/contracts/Portfolios.sol#L591), but it simply lists all accounts that were provided in the batch – with no way to distinguish which accounts were actually settled.


Consider emitting events after state-changing sensitive actions take place, to facilitate tracking and notify off-chain clients following the contracts’ activity.
