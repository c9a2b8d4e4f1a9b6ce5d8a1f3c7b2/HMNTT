# Missing/limited Test Suite

## Summary
The Reach team has added a test suite for the `ReachFactory` and `ReachDistributionV2` contracts, but the tests for the token contract still need improvement. The report recommends adding more tests for all the audited contracts and aiming for 100% code coverage. There is also a test that relies on random values, which can cause issues for automated analysis tools. The report suggests adding a minimum offset to these random values to ensure all tests run successfully every time.

## Details
**Update**
The Reach team added a test suite with good coverage for both the `ReachFactory` and `ReachDistributionV2` contracts. However, we still recommend improving the tests for the token contract.

**Description:** The project does not implement tests for the new functionalities related to the `ReachDistribution` and `ReachFactory` contracts. Additionally, the test for the `Reach` token contract is very limited, only testing its deployment. Tests may express requirements and are necessary to validate the software's intent.

**Recommendation:** We highly recommend improving the test suite. Add tests for all of the audited smart contracts functionality. 100% code coverage, though not foolproof, should be targeted as it is a robust heuristic for gauging test thoroughness.

Additionally, there is one test that relies on random values to be successful and as such would fail sometimes. Such practices affect the use of automated analysis tools that rely on all tests running successfully each time. We recommend making sure all tests run successfully on every run even though random values are used. In this particular case, the test could add a minimum offset to each random value.
