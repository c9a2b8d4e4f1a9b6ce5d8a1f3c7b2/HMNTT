# Possible loss of assets by mistake

## Summary
This bug report is about a function in the Gearbox Protocol Contracts repository. If a liquidator mistakenly calls this function with the parameters `to = address(0), force = true`, they will lose all assets. To prevent this, the report recommends adding a check that `to != address(0)`. This will ensure that liquidators don't accidentally lose all their assets when calling this function.

## Details
##### Description
If a liquidator calls this function by mistake with the next parameters: `to = address(0), force = true`, then he loses all assets:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L302
##### Recommendation
We recommend adding a check that `to != address(0)`.
