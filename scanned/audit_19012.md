# `signatory` not checked

## Summary
This bug report is about the GearToken.sol contract in the Gearbox-protocol/gearbox-contracts repository. The issue is that the signatory is not being checked. This means that if someone has the correct signature, they can access the contract without the correct signatory.

To fix this issue, the report recommends adding a check that the correct signatory was recovered. This would ensure that only the correct signatory can access the contract.

This bug report is important because it highlights a potential security issue in the GearToken.sol contract. If the signatory is not checked, anyone with the correct signature could access the contract. This could lead to malicious actors taking advantage of the contract and potentially stealing funds.

In conclusion, this bug report highlights an important issue in the GearToken.sol contract. To fix the issue, the report recommends adding a check that the correct signatory was recovered. This would help ensure that only the correct signatory can access the contract and prevent malicious actors from taking advantage of it.
