# Malicious user can pay less to pool

## Summary
This bug report discusses a malicious user who can add a "poisoned pool" to the `paths[i]` to minimize profits and pay less to the pool. This could be a security risk for the Gearbox Protocol. To prevent this issue, the report recommends checking the total value of the user before any transfers are made. This would help ensure that the user is not attempting to exploit the system by adding a poisoned pool. By taking this precaution, the Gearbox Protocol can help protect itself from malicious actors.
