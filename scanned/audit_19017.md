# `head` can't be taken out

## Summary
This bug report is about an issue with the AccountFactory.sol contract code on the Gearbox-protocol/gearbox-contracts repository. The code at line 257 is trying to get the `prev` account for the `head` but the `prev` account does not exist. 

The recommendation to resolve this issue is to add a special check for when the `creditAccount` is equal to the `head`. This will ensure that the `prev` account is always accounted for.
