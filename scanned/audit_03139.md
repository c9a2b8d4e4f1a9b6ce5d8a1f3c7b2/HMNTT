# Reserves can be re-added

## Summary
This bug report discusses an issue with the LendingPool and MiniPool contracts in the code files LendingPool.sol and MiniPool.sol. The bug can cause important parameters to be rewritten and the liquidity and borrow indices to be reset to one RAY if the reserve for a specific asset and reserveType has already been added. The severity of this bug is considered to be medium risk. The recommendation is to revert the action if the reserve has already been added. The bug has been fixed in commit 4fb61a93 and has been verified by Spearbit.

## Description
If the reserve for that specific asset and reserveType has already been added, revert instead. If we don't, some of the important parameters can be rewritten, and also the liquidity and borrow indices would be reset to one RAY.

## Recommendation
It would be best to revert if `reserveAlreadyAdded` is true.
