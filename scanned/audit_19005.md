# Address not checked

## Summary
This bug report is about the `_interestRateModel` in the Gearbox Protocol Contracts Pool Service. The bug is that the `_interestRateModel` can be set to zero, meaning that it does not have a valid address. In order to fix the issue, the report recommends adding necessary checks for the model address. This means that when the `_interestRateModel` is set, it should be checked to make sure that it is a valid address. This will help to ensure that the `_interestRateModel` is always set to an appropriate address.

## Details
##### Description
`_interestRateModel` can be equal to zero address:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/pool/PoolService.sol#L453
##### Recommendation
We recommend adding necessary checks for model address.
