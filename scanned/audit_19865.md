# [M-08] Controller.inCaseStrategyGetStuck does not update balance

## Summary
This bug report is about an issue with the `Controller.inCaseStrategyGetStuck` function in the code. This function withdraws from a strategy, but does not call the `updateBalance(_vault, _strategy)` afterwards. This means that the `_vaultDetails[_vault].balances[_strategy]` variable does not correctly track the actual strategy balance. Additionally, this can lead to `Controller.withdraw` attempting to withdraw too much.

The recommended mitigation step for this issue is to call `updateBalance(_vault, _strategy)` in `inCaseStrategyGetStuck`. This should help to ensure that the `_vaultDetails[_vault].balances[_strategy]` variable is correctly updated and that `Controller.withdraw` does not attempt to withdraw too much.
