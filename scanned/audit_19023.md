# `params.path` length not checked

## Summary
This bug report discusses an issue with the `_extractTokens()` function in the Gearbox Protocol’s UniswapV3.sol file. The issue is that the length of `params.path` is not checked, which can lead to an underflow. This means that the function can't accurately process the data it's given and may return incorrect results.

The recommendation is to add a check for the `params.path` length. This will ensure that the function can accurately process the data it's given and return the correct results.

## Details
##### Description
`params.path` length is not checked, so underflow can occur in `_extractTokens()` function:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/adapters/UniswapV3.sol#L102
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/adapters/UniswapV3.sol#L200
##### Recommendation
We recommend adding a check for the `params.path` length.
