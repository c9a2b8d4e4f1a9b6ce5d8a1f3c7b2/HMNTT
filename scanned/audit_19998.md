# Explicitly mark visibility of state

## Summary
This bug report is about a missing visibility setting in a contract called OrderBook.sol. This can cause problems when interacting with the contract and is an important issue that should be addressed. The recommendation is to explicitly set the visibility of the state to avoid any potential issues.

## Details
**Description**

contracts/OrderBook.sol#20, withdrawAddress
The storage variable has no visibility set. Such an issue is included to the standard auditor s
checklist. Missed visibility may lead to incorrect interaction with the smart contract.

**Recommendation**:
Explicitly mark visibility of state.
