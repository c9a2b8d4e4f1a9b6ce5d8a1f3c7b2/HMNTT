# Possible ddos attack

## Summary
This bug report is about the Limit Order Protocol on the 1inch platform. The issue is that malicious makers can potentially block bots from filling orders by calling an unbounded operation, which is an infinite loop, in the notifyFillOrder function. To fix this issue, the recommendation is to use a gas limit for the function. A gas limit is a limit on the amount of gas that a transaction can use, and it is an important security measure. This will ensure that malicious makers can't call unbounded operations, as the gas limit will prevent them from doing so.

## Details
##### Description
Malicious maker can block bots for filling order if he calls unbounded operation (infinite loop) in `notifyFillOrder`:
https://github.com/1inch/limit-order-protocol/tree/a14bde6a260458de5083cee117d734221e1cbc05/contracts/LimitOrderProtocol.sol#L338
##### Recommendation
We recommend to use gas limit for this function.
