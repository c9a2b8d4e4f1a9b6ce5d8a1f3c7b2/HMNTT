# The contract owner has too many permissions

## Summary
The rewardCutoff function allows the owner to change the last block number for when users can receive rewards. This means that the owner can add a new pool, wait for investors to invest, and then change the rewardCutoff to a current block plus one. This could cause unexpected issues for potential investors. To prevent this, it is recommended to only allow the owner to increase the rewardCutoff, not decrease it. Additionally, implementing a multisig system for the owner would prevent them from removing all rewards by themselves.

## Details
**Description**

**Recommendation**:

Make changes of rewardCutoff in a way when the owner can’t set less then initial value.
