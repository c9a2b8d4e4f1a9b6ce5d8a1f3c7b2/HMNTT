# Centralization due to `destroy()` method

## Summary
This bug report is about a medium severity issue that has already been resolved. The problem was found in a contract called TokenSale.sol, where a method called `destroy()` was used to self-destruct the contract and send all funds to an admin wallet. This means that only the admin had control over the funds, which is not ideal for security reasons. The recommendation is to decentralize the use of this function by using a multi-sig wallet with trusted users or a secure governance mechanism. This will help improve the security of the funds in the contract.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In Contract TokenSale.sol, the method `destroy()` self-destructs the contracts and sends all funds to the admin wallet. 

Since only the admin can call this method, the security of the funds in the contract is very much centralized.

**Recommendation**: 

It is advised to decentralize the usage of these functions by using a multi-sig wallet with at least 2/3 or a 3/5 configuration of trusted users. Alternatively, a secure governance mechanism can be utilized for the same.
