# Incorrect Accounting For Fee-On-Transfer Tokens

## Summary
A bug has been reported regarding the base token in a project. The base token is meant to be a stable coin like USDT, but there is a possibility that the fee on transfer mechanism for the token will be turned on in the future. This could cause incorrect accounting in the `contribute()` function, as the total contributed and account base contribution would be lower than the actual amount sent to the contract due to the fee being deducted. The recommendation is to check the balance of the base token before and after the transfer and account for the difference. The client has also mentioned that wETH will be used as the base token. The severity of this bug is medium and it has been acknowledged by the team.

## Details
**Severity** - Medium

**Status** - Acknowledged

**Description**

The base token is supposed to be a stable coin like USDT , though not currently but it is possible in the future that the fee on transfer mechanism for the token is turned on . If so , the accounting would be incorrect in the `contribute()` function . This is because `totalContributed` and `account_BaseContribute`d would be lower than the actual amount sent to the contract since a part of the amount would be sent out as fee.

**Recommendation:**

Check the balance of the base token before and after the transfer and account for the difference.

**Client comment**: wETH will be used as a base token.
