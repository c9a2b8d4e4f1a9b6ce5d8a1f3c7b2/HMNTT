# [M-02] Admin can't increase claim balance when `hasAllowanceMechanism` has been set in Claim contract

## Summary
The bug report is about a problem with creating a DAO (Decentralized Autonomous Organization). When users create a DAO, they can specify the amount of tokens that can be claimed, and they can later increase this amount. However, when a certain setting is enabled, called "hasAllowanceMechanism", it becomes impossible to increase the claim balance. This means that the amount of tokens available for claiming will remain the same as what was set during the creation of the DAO, and it will not be possible to increase it. The recommendation is to allow admins to still be able to increase the claim balance even when the "hasAllowanceMechanism" setting is enabled. This will prevent the issue of a constant claim balance and allow for the total amount of tokens available for claiming to be increased.

## Details
## Severity

**Impact:** Low

**Likelihood:** High

## Description

When users creates DAO they specify the `claimBalance` and later they can increase it. The issue is that when `hasAllowanceMechanism` is set there's no way to increase `claimBalance` because `depositTokens()` won't allow it. This would cause constant `claimBalance` that admin set during the DAO creation and it won't be possible to increase airdrop total amount.

## Recommendations

Allow admins to increase `claimBalance` when `hasAllowanceMechanism` is set.
