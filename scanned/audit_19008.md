# Transfer of 0 funds

## Summary
This bug report is about the PoolService.sol contract in the Gearbox-protocol GitHub repository. The issue is that when the withdrawFee is set to 0, the contract will still attempt to transfer 0 funds. This could create a security risk as it could allow malicious actors to exploit the contract. 

To fix this issue, the report recommends adding a check so that 0 funds wouldn't be transferred when the withdrawFee is set to 0. This would prevent malicious actors from exploiting the contract and ensure the safety of the funds.

## Details
##### Description
If `withdrawFee == 0`, then here 0 amount of asset transferred:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/pool/PoolService.sol#L191
##### Recommendation
We recommend adding a check, so 0 funds wouldn't be transferred.
