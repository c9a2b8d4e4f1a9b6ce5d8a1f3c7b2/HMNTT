# `path` length not checked

## Summary
This bug report is related to the Gearbox Protocol's smart contracts. It states that the length of the `path` is not checked in two locations within the code. This could potentially cause problems as the length of the `path` is an important factor in the code. Therefore, it is recommended to add a check for the `path` length in these two locations to ensure that the code runs correctly.

## Details
##### Description
`path` length is not checked:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/LeverageActions.sol#L107
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/LeverageActions.sol#L494
##### Recommendation
We recommend adding a check for `path` length.
