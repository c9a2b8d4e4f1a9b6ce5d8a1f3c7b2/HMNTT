# Unsupported fee-on-transfer tokens 

## Summary
The Transfers.sol implementation is supposed to work with different types of tokens, but it currently has a bug where fees are being charged twice for two-token transfers. This can cause problems when the input token is swapped to the recipient currency, as the maximum amount allowed to be paid does not account for the first fee. This can lead to failed transfers in certain situations. The function responsible for swapping tokens also returns incorrect information due to this bug. The recommendation is to either account for fee-on-transfer tokens in all transfer cases or explicitly revert when these tokens are used. Coinbase has already fixed this issue, and Cantina Managed has also addressed it.

## Details
## Transfers.sol

## Description
The implementation of Transfers.sol should be token agnostic and cater to most atypical token types. In most cases, two token transfers are made: firstly to the contract and secondly to the intended recipient. This causes a fee to be charged twice when it is really only necessary to make a single transfer from the payer to the merchant.

In the case where the input token is swapped to the recipient currency, the `maxAmountWillingToPay` argument is not adjusted to account for the fee charged in the first transfer. This leads to cases where the signed transfer intent may revert. This can happen when:

- Uniswap executes a swap where `amountInMax` is greater than the available amount to swap.
- Prior to swap execution, the input token is sent to the universal router contract in its entirety, which will fail because `maxAmountWillingToPay` includes the deducted fee.

Additionally, the `swapTokens()` function will potentially return incorrect informative swapped amounts as `payerBalanceBefore` makes use of `maxAmountWillingToPay`.

## Recommendation
Consider accounting for fee-on-transfer tokens in all transfer instances or explicitly revert when transfer intents on these atypical tokens are signed and executed.

## Coinbase
We chose to disallow fee-on-transfer tokens, including both the input and output tokens for swap payments. Fixed in commit `1aef11b1fcc57ee78a09eeb7fde71bb45d133f60`.

## Cantina Managed
Fixed.
