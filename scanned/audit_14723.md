# Admin can bypass the upgrade delay by `setDelay`

## Summary
This bug report is about the implementation of a special flow to upgrade a wallet core with a delay, but the admin can force an immediate upgrade by reducing the delay using setDelay(). This could deprive users of the ability to check implementations before applying. The recommendation is to improve the upgrade delay flow. 

A wallet core is a software program that stores and manages the user's private keys. These private keys are used to sign transactions and provide access to the user's cryptocurrency funds. The delay in the upgrade flow is to give users time to check the implementation before applying. 

The bug report is suggesting that the admin can reduce the delay, which would deprive users of the ability to check the implementation before applying. This could potentially lead to users unknowingly applying an implementation that is not in their best interest. 

The recommendation is to improve the upgrade delay flow to ensure that users have enough time to check the implementation before applying. This could be done by increasing the delay, or by creating a system that allows users to check the implementation in a secure manner before applying.
