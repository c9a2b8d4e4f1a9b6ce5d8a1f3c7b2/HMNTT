# Increase test coverage

## Summary
The pSTAKE Finance team created deep-dive tests for each unique scenario and tested them before deploying the code. However, the test coverage is limited and only covers the happy path. The LPStaking tests only cover the happy path and StakeLPCoreV8 has no tests. Furthermore, many of the test descriptions are inaccurate. Examples of this inaccuracy are tests that do not check to make sure that Stakers are the only group that can mint, tests that only show that an unauthorized address can’t use the stake function to mint tokens, and tests that actually test for the inverse case. 

It is recommended that the test coverage be increased for the entire codebase. Tests should be added for the inherited contracts from OpenZeppelin and should also test for edge cases and multiple expected cases. Additionally, the test descriptions should match the functionality that is actually tested.
