# [H-01] `MIMOEmptyVault.sol executeOperation()` does not transfer the Vault leftover assets to the owner, it is locked in the `MIMOEmptyVault`

## Summary
This bug report is about a vulnerability in the MIMOEmptyVault.sol smart contract. The executeAction() function is supposed to pay off the debt and return the leftover assets to the owner of the Vault. However, when tested, it was found that the emptyVault contract only pays back the flash loan, and does not transfer the leftover assets to the owner, and the assets are locked in the emptyVault contract. 

To test this, logs were added to the test case in the MIMOEmtpyVault.test.ts file. The logs showed that the balance of the emptyVault contract before and after the transaction were 0 and 44383268870065355782 respectively.

The recommended mitigation step is to add code to the executeOperation() function that will transfer the remaining balance back to the mimoProxy or owner. The code should look like this: 

vaultCollateral.safeTransfer(address(mimoProxy), vaultCollateral.balanceOf(address(this)) - flashloanRepayAmount);

Once the code is added, the function should be able to pay off the debt and return the leftover assets to the owner of the Vault.
