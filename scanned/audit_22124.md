# Use latest version of Solidity

## Summary
This bug report is about the current code written for old versions of solc (0.4.2). Recently, a storage overwriting vulnerability was found in versions of solc prior to 0.4.4. This means that the code may be compiled with vulnerable versions, which could pose a risk. It is recommended to change the solidity version pragma to the latest version (0.4.6) in order to enforce the use of the latest compiler version. This will help ensure that the code is compiled with the most up-to-date version of solc, reducing the risk of the vulnerability.

## Details
[Current code](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L5) is written for old versions of solc (0.4.2). With [the latest storage overwriting vulnerability](https://blog.ethereum.org/2016/11/01/security-alert-solidity-variables-can-overwritten-storage/) found on versions of solc prior to 0.4.4, there’s a risk this code can be compiled with vulnerable versions. We recommend changing the solidity version pragma for the latest version (`pragma solidity ^0.4.6;`) to enforce latest compiler version to be used.
