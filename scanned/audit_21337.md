# VotingEscrow ’s Admin can take whitelisted accounts hostage

## Summary
This bug report is about a data validation vulnerability in the ERC20CRV.vy contract. It has been identified as a high difficulty bug. The vulnerability allows the admin of the VotingEscrow contract to allow or disallow any contract to interact with VotingEscrow. This could be exploited by a malicious owner to ask for ransom from VotingEscrow's users. 

To prevent this, it is recommended that users should be made aware of the risk. In the long term, all possible actions for privileged accounts should be identified and documented, and users should be able to easily identify the risks associated with each account.

## Data Validation

## Target: ERC20CRV.vy

**Difficulty:** High

### Description

VotingEscrow’s admin can allow or disallow any contract to interact with VotingEscrow. A malicious owner can use this feature to ask for a ransom from VotingEscrow’s users.

```python
@public
def add_to_whitelist(addr: address):
    assert msg.sender == self.admin
    self.contracts_whitelist[addr] = True

@public
def remove_from_whitelist(addr: address):
    assert msg.sender == self.admin
    self.contracts_whitelist[addr] = False
```

_Figure 9.1: VotingEscrow.vy#L90-L99._

### Exploit Scenario

Eve is a malicious VotingEscrow owner. Eve allows Bob to use VotingEscrow’s multisig wallet. Bob deposits $1,000,000 worth of assets in the contract. Eve revokes Bob from the whitelist and asks him to pay $100,000 in ransom to withdraw his funds, which he does.

### Recommendation

- **Short term:** Make sure users are aware of this risk.
  
- **Long term:** Identify and document all possible actions for privileged accounts. Ensure users can easily identify the risks associated with every privileged account.
