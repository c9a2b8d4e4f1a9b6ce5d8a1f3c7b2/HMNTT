# Use SafeERC20

## Summary
The bug report suggests that there is an issue with transferring ERC20 tokens in three different contracts - BaseLiquidityPool.sol, FlashHedge.sol, and PerpetualMarket.sol. It is recommended to use 'safeTransfer' and 'safeTransferFrom' instead of regular ERC20 transfers to ensure that the tokens are transferred correctly, even in cases where the ERC20 implementation is non-standard, such as in USDT tokens.

## Details
**Description**

BaseLiquidityPool.sol
FlashHedge.sol
PerpetualMarket.sol
ERC20 tokens should be transferred with ‘safeTransfer’ and ‘safeTransferFrom’. SafeERC20
performs all the checks that tokens were transferred, including ono-standard ERC20
implementation(like in USDT tokens).

**Recommendation**:

Use ‘safeTransfer’ and ‘safeTransferFrom’ instead.
