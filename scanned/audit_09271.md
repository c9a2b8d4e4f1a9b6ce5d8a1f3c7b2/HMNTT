# Centralization risks

## Summary
The bug report states that the project administrator and manager have too much power and control over the project, which can lead to centralization risks. They have the ability to make changes to the contract code, project configuration, add minters, and make oracle appointments without any restrictions. To address this issue, it is recommended to implement a DAO system that has a time-delayed execution for updates. This will help prevent any potential misuse of power and promote decentralization within the project.

## Details
##### Description
The project administrator and manager have unrestricted rights, including:
- Contract code updates,
- Project configuration changes,
- Addition of an arbitrary number of RSETH minters (though minting capability is only used in LRTDepositPool),
- Oracle appointments.

These permissions pose centralization risks.
##### Recommendation
We recommend creating a DAO system with time-delayed execution of updates.
