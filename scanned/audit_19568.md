# [M-02] Null check in pricePerShare

## Summary
This report is about a vulnerability in the code of the Github repository "2021-10-badgerdao". The vulnerability is related to the Oracle, which could set a price of 0 for a share, resulting in a Denial Of Service. To prevent this, the report recommends adding a null check to ensure that the price is always greater than 0. This check should be implemented on every update to the code.

oracle can `0` as a price of the share, in that case, 0 will be the denominator in some calculations which can cause reverts from SafeMath (for e.g here: https://github.com/code-423n4/2021-10-badgerdao/blob/main/contracts/WrappedIbbtc.sol#L148 ) resulting in Denial Of Service.

Add a null check to ensure that on every update, the price is greater than 0.
