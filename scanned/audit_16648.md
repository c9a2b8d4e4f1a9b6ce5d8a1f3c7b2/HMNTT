# Centralization risk

## Summary
This bug report describes a high severity issue in a contract called "Identity". The problem is that any module can transfer funds from this contract to any other address, which could lead to centralization. The report recommends changing the contract's behavior so that only the owner of the funds can access them. The issue has been resolved.

## Details
**Severity**: High

**Status**: Resolved

**Description**

In contract Identity, in function execute, an arbitrary module can send the funds from Identity contract to any other address. This behavior creates a major risk of centralization.

**Recommendation**: 

Modify the behavior of the contract, so that only the owner of the funds can have access to them.
