# [M-02] Call to swapExactTokensForETH in liquidateDai() will always fail

## Summary
This bug report is about liquidateDai() calls Uniswap’s swapExactTokensForETH to swap Dai to ETH. It has been identified that the call to UniswapV2 router for swapping will fail because msg.sender, i.e. FSD contract, has not approved UniswapV2 with an allowance for the tokens being attempted to swap. The impact of this is that updateCostShareRequest() will fail and revert while working with stablecoin Dai. 

Proof of concept has been provided using manual analysis and the code snippets from the GitHub repository. The recommended mitigation step is to add FSD approval to UniswapV2 with an allowance for the tokens being attempted to swap. This will ensure that the call to UniswapV2 router for swapping will be successful and updateCostShareRequest() will not fail and revert while working with stablecoin Dai.

## Vulnerability details

## Impact

liquidateDai() calls Uniswap’s swapExactTokensForETH to swap Dai to ETH. This will work if msg.sender, i.e. FSD contract, has already given the router an allowance of at least amount on the input token Dai. 

Given that there is no prior approval, the call to UniswapV2 router for swapping will fail because msg.sender has not approved UniswapV2 with an allowance for the tokens being attempted to swap.

The impact is that updateCostShareRequest() will fail and revert while working with stablecoin Dai.
