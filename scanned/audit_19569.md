# [M-01] Unable to transfer WrappedIbbtc if Oracle go down

## Summary
This bug report deals with an issue in WrappedIbbtc, a smart contract. The issue is that users are not able to transfer if the oracle.pricePerShare() function (L124) reverts. This is because balanceToShares() is called in both transfer and transferFrom, which includes a call to pricePerShare(). Additionally, WrappedIbbtcEth behaves differently, as it uses a cached value in a local variable pricePerShare which is only updated upon call to updatePricePerShare(). Depending on the specification, one of them needs to be changed in order to resolve this issue.
