# Missing/Limited Test Suite

## Summary
The client has marked a previous issue as "fixed" but has provided a recommendation for further improvements. The project currently has a limited test suite, which can lead to serious bugs and security issues. While audits can help find bugs, they are not enough. The recommendation is to create a more comprehensive test suite with at least 90% branch coverage, unit and integration tests with a mainnet fork, and a variety of test cases including different scenarios and role permissions. The test suite should also verify state changes, invariants, and event emissions, and may also include fuzz testing.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `08ac9cbb07ba87ed6cefbf7d831f5bd88df2799e`. The client provided the following explanation:

> Modify according to recommendations.

**Description:** The project appears to have a limited test suite. Contracts that are not extensively tested are at high risk of serious bugs that can impact the protocol's proper functioning and security. While audits can help developers to find bugs, they are not a substitute for thorough unit and integration testing.

**Recommendation:** A test suite should be created, which:

*   Has at least `90%` branch coverage
*   Includes unit tests and integration tests with a mainnet fork
*   Includes a variety of test cases, including:
    *   All "happy path" scenarios
    *   Negative test cases
    *   Unexpected or edge cases
    *   All role permissions and access control for protected functions.

*   Verifies the following through assertions:
    *   All state changes
    *   Invariants
    *   Emission of events

*   May include fuzz testing
