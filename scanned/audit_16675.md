# [M-03] Possible DoS on vestingRecipients due to lack of disposal mechanism

## Summary
This bug report is about a vulnerability in the VTVLVesting.sol smart contract code. This vulnerability can cause a Denial of Service (DoS) attack when the smart contract starts to be used. The vulnerable lines of code are 224, 245, 302, and 317. 

The impact of this vulnerability is that the variable in storage vestingRecipients will start to be filled with addresses, and there is no mechanism to eliminate elements. This will cause the allVestingRecipients() function to generate a DoS attack. 

The recommended mitigation steps to fix this vulnerability are to add a mechanism to the withdraw() function that will remove elements from vestingRecipients that no longer have vesting. This would prevent the variable from growing without reducing elements.
