# Risk of underflow.

## Summary
The bug report is about a problem in a code for a system called StakingB_1.sol. In line 579 of the code, there is a function that allows users to "unstake," or withdraw, their funds. However, there is a problem where even if a user doesn't have any funds to withdraw, they can still use this function. This causes a decrease in a variable called "amountOfUsers," which can lead to an error and prevent other users from being able to withdraw their funds. 

The recommendation to fix this issue is to check if the user's account has already been deleted before decreasing the "amountOfUsers" variable. This will prevent the error from occurring and allow all users to withdraw their funds as intended.

## Details
**Description**


StakingB_1.sol Line 579
Each user can use the unstake function even if they have nothing to unstake, so everyone can
decrease the “amountOfUsers” variable that can cause an underflow and won’t allow other
users to unstake.

**Recommendation**:

Check if “userInfo[msg.sender]” has not been already deleted before decreasing an
“amountOfUsers”.
