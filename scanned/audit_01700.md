# [M-12] Merging or withdrawing VotingEscrow burns tokens without rewards

## Summary
The bug report states that there is a medium severity issue with the VotingEscrow merge(), split(), and withdraw() functions. These functions are supposed to allow users to merge, split, and withdraw their NFTs, but they are currently not working as intended. When a user performs one of these actions, their NFT is burned, which means it is permanently removed from their possession. However, in order to claim pending rewards from the bribe system, users need to own the NFT. Since the NFT is burned, it is impossible to access these rewards.

The report suggests that instead of directly burning the NFT, the token amounts should be reset. This would prevent users from losing their NFT and still allow them to claim their rewards.

## Details
## Severity

**Impact:** Medium

**Likelihood:** Medium

## Description

VotingEscrow merge(), split(), and withdraw() functions burn the user's NFT without claiming pending rewards from the bribe system (via getReward). Since claiming bribe rewards requires ownership of the NFT (isApprovedOrOwner check), burning the NFT makes it impossible to access unclaimed rewards afterward.

In the merge() function:
```
locked[_from] = LockedBalance(0, 0);
_checkpoint(_from, _locked0, LockedBalance(0, 0));
_burn(_from);
```
The getReward() function that users must call to claim bribe rewards checks ownership:

`require(IVotingEscrow(_ve).isApprovedOrOwner(msg.sender, tokenId));
`
Burning the NFT removes this ownership, making it impossible to later call getReward.


## Recommendations

Reset token amounts instead of directly burning nft.
