# Possible overflow

## Summary
The report describes a bug in the code of the Gearbox protocol, specifically in the CreditManager contract. The bug occurs when the total amount of tokens on a credit account is very large, causing an overflow. This can potentially lead to errors in the system. The recommendation is to use safeMath to prevent this bug from occurring.

## Details
##### Description
If overall amount of tokens on credit account is very big, then overflow can occur:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L593-L594
##### Recommendation
We recommend to use safeMath.
