# Fee-On-Transfer Tokens Lead to Improper Tracking

## Summary
This bug report is about a problem in the `RiskFundConverter` code. The code has a function called `updateAssetsState` that is supposed to track assets that are being transferred to a specific destination called `RiskFundV2`. However, if the asset being transferred is a fee-on-transfer token, the balance will not be properly tracked. This could cause off-chain scripts to fail due to incorrect balance information. The bug has been fixed in the latest update.

## Details
Untracked assets residing in `RiskFundConverter` can be tracked using the [`updateAssetsState` function](https://github.com/VenusProtocol/protocol-reserve/blob/ca8f8ba3dfcf6f13d2db4ac230ac9337950525f6/contracts/TokenConverter/RiskFundConverter.sol#L147-L171). If the asset is configured to be transferred directly to the destination (i.e., the [`RiskFundV2` vault](https://github.com/VenusProtocol/protocol-reserve/blob/ca8f8ba3dfcf6f13d2db4ac230ac9337950525f6/contracts/ProtocolReserve/RiskFundV2.sol)), the [`poolAssetsFunds`](https://github.com/VenusProtocol/protocol-reserve/blob/ca8f8ba3dfcf6f13d2db4ac230ac9337950525f6/contracts/ProtocolReserve/RiskFundStorage.sol#L13) in `RiskFundV2` [will be updated](https://github.com/VenusProtocol/protocol-reserve/blob/ca8f8ba3dfcf6f13d2db4ac230ac9337950525f6/contracts/ProtocolReserve/RiskFundV2.sol#L133-L140) using the [`updatePoolState` call](https://github.com/VenusProtocol/protocol-reserve/blob/ca8f8ba3dfcf6f13d2db4ac230ac9337950525f6/contracts/TokenConverter/RiskFundConverter.sol#L164). However, if this asset is a fee-on-transfer token, the balance will not be properly tracked as the amount of funds sent from the `RiskFundConverter` will not be the same as the amount of funds received by the `RiskFundV2`. This could lead off-chain scripts designed to transfer funds from `RiskFundV2` to fail due to insufficient balances if they are reading from [`poolAssetsFunds`](https://github.com/VenusProtocol/protocol-reserve/blob/ca8f8ba3dfcf6f13d2db4ac230ac9337950525f6/contracts/ProtocolReserve/RiskFundStorage.sol#L13). Consider tracking the actual amount of funds received by `RiskFundV2` instead of the amount of funds sent from `RiskFundConverter` to support fee-on-transfer tokens.


***Update:** Resolved at commit [4025db0](https://github.com/VenusProtocol/protocol-reserve/pull/24/commits/4025db080dbf0d9f19b38109e9da6f5e72cc9353).*
