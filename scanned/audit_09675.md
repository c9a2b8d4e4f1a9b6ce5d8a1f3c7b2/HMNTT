# [M-06] Fee on Transfer Token Will Break accounting

## Summary
This bug report states that there is an issue with the `mint()` and `deposit()` functions in the As4626.sol file. These functions use the `amount` variable for transferring and accounting, but this can cause problems with fee on transfer tokens. This is because the actual token received will be less than the amount, leading to incorrect accounting and affecting the `sharePrice`. The severity of this bug is high because it can lead to incorrect accounting and affect the `sharePrice`, but the likelihood is low since fee on transfer tokens are not commonly used. The report recommends using before and after balances to accurately reflect the true amount received and updating the `sharePrice` accordingly.

## Details
**Severity**

**Impact:** High, because the accounting will be incorrect, and the sharePrice will be affected

**Likelihood:** Low, because fee on transfer token is not commonly used

**Description**
