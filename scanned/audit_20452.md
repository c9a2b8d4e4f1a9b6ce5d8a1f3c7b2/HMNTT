# The approval value obtained in the constructor may not be enough for the long term of the smart contract

## Summary
A bug has been found in the smart contract code of the YFI Maker project. At line 69 of the Strategy.sol file, the constructor call of the smart contract is calling the `_approveAll()` function for different tokens. However, the value obtained from this process will only decrease, and if it decreases to zero, the tokens will remain locked in the contract permanently. To address this issue, it is recommended to add a function to increase the value of approvals.

## Details
##### Description
At line: https://github.com/orbxball/yfi-maker/blob/63af6fcfa536073f00d652f49befd18e429b5500/contracts/Strategy.sol#L69
the smart contract constructor call `_approveAll()` function for different tokens. But in the process of work, the obtained value will only decrease. If this value decreases to zero, then the tokens will remain locked in the contract forever.
  
##### Recommendation
It is recommended to add a function to increase the value of approvals.
