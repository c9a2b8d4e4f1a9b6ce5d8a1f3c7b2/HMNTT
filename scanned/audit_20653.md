# Gas overflow during iteration (DoS)

## Summary
This bug report relates to the Yearn Vaults smart contract. The issue is that in certain cases, more gas may be required than is allocated to record one block. This is due to the fact that the contract's logic does not check the length of the `vaults` address array when writing data to the blockchain, which consumes gas. As a result, all iterations of the loop will fail.

To fix this bug, it is recommended that a check for the maximum possible number of elements of the arrays should be added. This will ensure that the gas allocated is sufficient for all iterations of the loop.

## Details
##### Description
Each iteration of the cycle requires a gas flow.
A moment may come when more gas is required than it is allocated to record one block. In this case, all iterations of the loop will fail.

##### Recommendation
It is recommended adding a check for the maximum possible number of elements of the arrays.
