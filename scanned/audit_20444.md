# [M-03] timelockERC721Keys could exceed the block size limit

## Summary
This bug report is about a vulnerability found in a smart contract called Visor.sol. The vulnerability is found on line 504, where looping through the timelockERC721Keys could exceed the block size limit. No proof of concept was given, but the tool used to identify the vulnerability was Hardhat. The recommended mitigation step is to transfer by index instead of token ID. This means that instead of transferring the token ID, the transfer should be done using the index of the token ID. This should help to avoid exceeding the block size limit and prevent the vulnerability.
