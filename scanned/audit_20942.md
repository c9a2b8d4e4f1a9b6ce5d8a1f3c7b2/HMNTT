# [M13] Missing test coverage report

## Summary
This bug report is about the lack of an automated test coverage report for the `protocol` package. Without this report, it is not possible to know if parts of the code have not been tested by automated tests. As a result, a full manual test suite has to be executed for every change. It is suggested that the test coverage report should reach at least 95% of the source code. 

Update: This issue has not been fixed yet. cLabs’ statement is that the `celo-monorepo` configuration includes a coverage that runs once a day. It is not included on every PR/commit since it takes approximately 3 hours to run all tests with coverage enabled. This will be revisited before mainnet in order to improve the coverage tooling.

## Details
There is no automated test coverage report for the `protocol` package. Without this report it is impossible to know whether there are parts of the code never executed by the automated tests; so for every change, a full manual test suite has to be executed to make sure that nothing is broken or misbehaving.


Consider adding the test coverage report and making it reach at least 95% of the source code.
