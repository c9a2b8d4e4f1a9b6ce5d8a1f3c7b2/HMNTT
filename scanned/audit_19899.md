# Possible reentrancy

## Summary
This bug report is about a function found in the LimitOrderProtocol.sol file on the 1inch Github repository. The function in question is prone to reentrancy, which is when a function can be called again while it is already running. This can cause unexpected and undesired behavior, and can even lead to security risks.

To fix this issue, the report recommends adding a nonReentrant modificator. This modificator prevents the function from being called again while it is already running. By adding this modificator, the function will no longer be vulnerable to reentrancy, and the unexpected behavior and potential security risks can be avoided.

## Details
##### Description
In the following function reentrancy can occur:
https://github.com/1inch/limit-order-protocol/tree/a14bde6a260458de5083cee117d734221e1cbc05/contracts/LimitOrderProtocol.sol#L280-L345
##### Recommendation
We recommend to add `nonReentrant` modificator.
