# Missing docstrings

## Summary
This bug report is about the difficulty of understanding the code base in the project. It was found that the layout of the project made the problem worse because the reader had to jump through multiple files to understand a single function. To solve the problem, it was suggested to add Natspec docstrings to everything that is part of the contracts' public API, including structs and struct fields. It was also suggested to document any side-effects of the functions and the conditions that will make them revert in the code. If a new concept or business rule is introduced in a high-level function, it was suggested to briefly explain it and add a link to the user documentation for more details. This issue was fixed in pull request #234.

## Details
Some areas of the code base were difficult to understand or were not self-explanatory. The layout of the project makes this a bigger problem because the reader has to jump through many files to understand a single function.


Consider adding [Natspec docstrings](https://github.com/ethereum/wiki/wiki/Ethereum-Natural-Specification-Format) to everything that is part of the contracts’ public API, including structs and struct fields. In addition to that, consider documenting in the code any side-effects of the functions, and the conditions that will make them revert. If a new concept or a business rule is introduced in a high-level function, consider briefly explaining it and adding a link to the user documentation for more details.


***Update:*** *Fixed in* [*pull request #234*](https://github.com/dydxprotocol/solo/pull/234)*.*
