# `wethGateway` can't be changed

## Summary
This bug report is about the `wethGateway` feature of the Gearbox Protocol. Currently, the `wethGateway` can not be changed, meaning that if the contracts need to be redeployed, all contracts must be redeployed as well. This can cause problems, as it is time consuming and can be inconvenient. The recommendation is to add a function for updating the `wethGateway`, which would make it easier to redeploy contracts. This would save time and make the process more efficient.

## Details
##### Description
`wethGateway` can't be changed, so in case of redeploy, all contracts must be redeployed too:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L123
##### Recommendation
We recommend adding a function for updating `wethGateway`.
