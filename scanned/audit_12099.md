# Allowances Can Be Frontran

## Summary
This bug report is about two functions in the SubAccounts module, SubAccounts::setAssetAllowances() and SubAccounts::setSubIdAllowances(), which set allowances directly rather than increasing or decreasing the allowance. This can lead to the classic approve() frontrunning issue where the allowance can be spent twice through frontrunning when the user is attempting to set a new allowance. The development team was aware of the issue and considered it a reasonable trade-off in their use cases, so they decided not to fix it. As a result, it is important to be aware of this potential issue when using these functions.
