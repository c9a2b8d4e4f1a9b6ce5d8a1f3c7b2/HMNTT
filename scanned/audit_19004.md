# Possible overflow

## Summary
This bug report is about an overflow issue in the CreditManager.sol file of the Gearbox Protocol. This bug occurs when the overall amount of tokens on a credit account is very large. This can cause an overflow, which is when a calculation produces a result that is too large for the computer to store. In order to address this issue, the report recommends using safeMath. SafeMath is a library that is used to prevent overflow errors from occurring. It does this by limiting the size of the numbers that can be used in calculations. By using safeMath, the risk of an overflow occurring can be minimized.

## Details
##### Description
If overall amount of tokens on credit account is very big, then overflow can occur:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L593-L594
##### Recommendation
We recommend to use safeMath.
