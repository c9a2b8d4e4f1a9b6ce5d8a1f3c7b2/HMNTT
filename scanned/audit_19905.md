# Incorrect event argument

## Summary
The bug report is about a discrepancy in the displayed value in an event in a contract. The incorrect value is being displayed due to a decrease in the _amount variable at certain lines in the contract. The recommendation is to store the total _amount in another value and pass it to the event to ensure the correct value is displayed.

## Details
**Description**

At contracts/Cohort.sol:228
The _amount is decreasing at lines
contracts/Cohort.sol:222
contracts/Cohort.sol:225
contracts/Cohort.sol:213
So the wrong value will be displayed in the event.

**Recommendation**:

Store total _amount in the other value and pass it to the event.
