# [M-06] calcMint always return poolId=0 and idx=0

## Summary
This bug report is about a vulnerability in the `calcMint` function of Zap.sol, a smart contract used in the Badger-Finance project. The function is designed to return the most optimal route for users, but instead it always returns poolId=0 and idx=0, which leads to suboptimal zap. The proof of concept can be found in the Github link provided. It is recommended that the code be updated to ensure that the function returns the most optimal route for users.
