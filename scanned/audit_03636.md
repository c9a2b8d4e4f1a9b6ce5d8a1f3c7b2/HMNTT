# [M-02] Code should set `ethB` value in `_curveBuy()` when `curveLiquidityMet` is true

## Summary
This bug report discusses a problem with the `_curveBuy()` function in a smart contract. The severity of the bug is considered low, but the likelihood of it occurring is high. The issue is that when the `curveLiquidityMet` condition is met, the code buys all the tokens from pool B but sets the value of `ethA` instead of `ethB`. This means that the value of `ethB` will be incorrect, causing other functions that rely on this value to not work properly. The same issue is present in the `_curveBuyCalculation()` function. The recommendation is to set the value of `ethB` when the `curveLiquidityMet` condition is met.

## Details
## Severity

**Impact:** Low

**Likelihood:** High
