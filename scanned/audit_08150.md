# PRIVILEGED ADDRESS CAN BE TRANSFERRED WITHOUT CONFIRMATION

## Summary
The `update_config` function in certain smart contracts can cause the contract owner to lose control of the contracts and cannot be undone. This can happen when the owner changes the owner address without confirmation. The affected contracts are epoch-manager and distributor-v1, and the code can be found in contracts/distributor-v1/src/commands.rs and contracts/bbro-minter/src/commands.rs. The impact of this bug is a 4 out of 10 and the likelihood is a 2 out of 10. The issue has been solved in commit 79549c38936e99a89a1fa7aa7e38456032f47389.

## Details
##### Description

Incorrect use of the `update_config` function in contracts can set owner to have an invalid address and inadvertently lose control of the contracts, which cannot be undone in any way. Currently, the contract owner can change the **owner address** using the aforementioned function in a `single transaction` and `without confirmation` from the new address.

The affected smart contracts are the following:

* epoch-manager
* distributor-v1

Code Location
-------------

#### contracts/distributor-v1/src/commands.rs

```
let mut config = load_config(deps.storage)?;

if let Some(owner) = owner {
    config.owner = deps.api.addr_canonicalize(&owner)?;
}

```

#### contracts/bbro-minter/src/commands.rs

```
let mut config = load_config(deps.storage)?;

if let Some(owner) = owner {
    config.owner = deps.api.addr_canonicalize(&owner)?;
}

```
