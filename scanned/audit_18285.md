# Vulnerable dependencies in the Substrate parachain

## Summary
This bug report is regarding a data validation issue with the Parallel Finance parachain node. The node uses certain dependencies with known vulnerabilities, including chrono, lru, time, and net2. The vulnerabilities with lru, time, and net2 can be fixed by updating them to their newest versions, however, the chrono crate issue has not been mitigated and remains problematic. An attacker can exploit this vulnerability and perform a denial-of-service attack on the network by taking down all nodes in the network. 

In the short term, the team should update all dependencies to their newest versions and monitor the referenced GitHub thread regarding the chrono crate segfault issue. In the long term, the team should run cargo-audit as part of the CI/CD pipeline and ensure that they are alerted to any vulnerable dependencies that are detected.

### Description
The Parallel Finance parachain node uses the following dependencies with known vulnerabilities. (All of the dependencies listed are inherited from the Substrate framework.)

| Dependency | Version | ID | Description |
|------------|---------|----|-------------|
| chrono     | 0.4.19  | RUSTSEC-2020-0159 | Potential segfault in localtime_r invocations |
| lru        | 0.6.6   | RUSTSEC-2021-0130 | Use after free in lru crate |
| time       | 0.1.44  | RUSTSEC-2020-0071 | Potential segfault in the time crate |
| net2       | 0.2.37  | RUSTSEC-2020-0016 | net2 crate has been deprecated; use socket2 instead |

Other than `chrono`, all the dependencies can simply be updated to their newest versions to fix the vulnerabilities. The `chrono` crate issue has not been mitigated and remains problematic. A specific sequence of calls must occur to trigger the vulnerability, which is discussed in [this GitHub thread](https://github.com) in the `chrono` repository.

### Exploit Scenario
An attacker exploits a known vulnerability in the Parallel Finance node and performs a denial-of-service attack on the network by taking down all nodes in the network.

### Recommendations
- **Short term**: Update all dependencies to their newest versions. Monitor the referenced GitHub thread regarding the `chrono` crate segfault issue.
- **Long term**: Run `cargo-audit` as part of the CI/CD pipeline and ensure that the team is alerted to any vulnerable dependencies that are detected.
