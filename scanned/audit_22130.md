# Use latest version of Solidity

## Summary
A potential security vulnerability has been identified in the current code used by EtherCamp's Virtual Accelerator. This vulnerability, found in versions of solc prior to 0.4.4, involves the storage of data being overwritten. To prevent this, EtherCamp has recommended changing the solidity version pragma to the latest version (0.4.4) to ensure the latest compiler version is used. EtherCamp has already fixed the problem in their commits.

## Details
[Current code](https://github.com/ether-camp/virtual-accelerator/blob/master/contracts/ProjectKudos.sol#L1) is written for old versions of solc (0.4.0). With [the latest storage overwriting vulnerability](https://blog.ethereum.org/2016/11/01/security-alert-solidity-variables-can-overwritten-storage/) found on versions of solc prior to 0.4.4, there’s a risk this code can be compiled with vulnerable versions. We recommend changing the solidity version pragma for the latest version (`pragma solidity ^0.4.4;`) to enforce latest compiler version to be used.


EDIT: EtherCamp fixed this problem [on these commits](https://github.com/ether-camp/virtual-accelerator/compare/6665fdd4b71088443a74d1ed9fda52c6a8c8b975...9a2291ac505e3b92fa44a06658ac940fb8fc49e6).
