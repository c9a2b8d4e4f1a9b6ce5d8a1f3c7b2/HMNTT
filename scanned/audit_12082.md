# Missing Input Validation

## Summary
This bug report highlights three issues in the StakingContract.sol file. 

The first issue is that there is no check for a zero address in the `addOperator` function, which can cause problems when adding new operators. The function also does not check if the operator already exists, which can lead to duplicate entries.

The second issue is that the `setTreasury` function does not have a zero address check when updating the treasury address. This can cause issues when trying to change the treasury address.

The third issue is with the `activateOperator` and `deactivateOperator` functions. These functions allow the admin to activate or deactivate an operator, but they do not check if the operator is already activated or deactivated. This can cause false alarms for off-chain monitoring tools and unnecessary panic. 

These issues should be addressed to ensure the proper functioning of the StakingContract.

## Details
#### Description


* There is no zero address check in function `addOperator` for the operator address and fee recipient. Also, the function doesn’t check whether the operator already exists.
