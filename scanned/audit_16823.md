# [M-18] Precision is not enough for proposalThreshold and quorum. Collections with at least 20000 NFTs in total supply may have some trouble.

## Summary
A bug has been identified in the code of a project on GitHub, which could cause problems for collections with a total supply of at least 20,000 Non-Fungible Tokens (NFTs). The issue lies in the precision of the proposalThreshold and quorum, which cannot be set to 1 if the total supply is above 20,000. This is due to the division of the total supply by 10,000, which would return a value of 2 for a total supply of 20,000. 

The bug was identified manually by reviewing the code, which can be found at the link provided in the report. The recommended mitigation steps are to increase the division to a more precise value, such as 1e18, which would allow the threshold to be set to 1 for collections with a higher total supply. The code for this proposed solution is also provided in the report.

## Vulnerability details

## Impact
Precision is not enough for proposalThreshold and quorum. Collections with at least 20000 NFTs in total supply may have some trouble. These collections can't be set proposalThreshold = 1 or quorum = 1.

## Proof of Concept
```
    /// @notice The current number of votes required to submit a proposal
    function proposalThreshold() public view returns (uint256) {
        unchecked {
            return (settings.token.totalSupply() * settings.proposalThresholdBps) / 10_000;
        }
    }

    /// @notice The current number of votes required to be in favor of a proposal in order to reach quorum
    function quorum() public view returns (uint256) {
        unchecked {
            return (settings.token.totalSupply() * settings.quorumThresholdBps) / 10_000;
        }
    }
```

If totalSupply = 20000, with the lowest settings (settings.proposalThresholdBps, settings.quorumThresholdBps = 1), it would returns (20000 * 1) / 10000 = 2. And it can't be lowered more.

## Tools Used
Manual review

## Recommended Mitigation Steps
Increase division to a more precise value such as 1e18 to allow high total supply NFT to always set threshold as 1

```
    /// @notice The current number of votes required to submit a proposal
    function proposalThreshold() public view returns (uint256) {
        unchecked {
            return (settings.token.totalSupply() * settings.proposalThresholdBps) / 1e18;
        }
    }

    /// @notice The current number of votes required to be in favor of a proposal in order to reach quorum
    function quorum() public view returns (uint256) {
        unchecked {
            return (settings.token.totalSupply() * settings.quorumThresholdBps) / 1e18;
        }
    }
```
