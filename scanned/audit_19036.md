# Possible transfer of 0 funds

## Summary
This bug report concerns the CreditManager.sol contract, which is part of the Gearbox Protocol. The issue is that the liquidation process can allow for the `remainingFunds` to be equal to zero. This could cause problems in the system, so the recommendation is to add a check that `remainingFunds` is not equal to zero before any transfer is made. This would ensure that the system would not be affected by this issue.
