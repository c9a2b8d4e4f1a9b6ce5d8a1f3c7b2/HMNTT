# Using deprecated libraries

## Summary
The report states that the "GOVToken" contract uses a library called SafeMath to prevent errors caused by large numbers. However, the latest version of Solidity already has a built-in feature to prevent these errors, making the SafeMath library unnecessary. The recommendation is to remove the SafeMath library from the contract.

## Details
**Description**

Сontract “GOVToken” uses SafeMath library to prevent overflow after arithmetical operations.
Solidity from has reverted on overflow by default, the only reason to use SafeMath is to
send custom messages on reverts.

v0.8.0

**Recommendation**:

Remove unnecessary library.
