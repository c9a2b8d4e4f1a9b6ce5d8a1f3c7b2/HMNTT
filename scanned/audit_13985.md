# Missing/Limited Test Suite

## Summary
The bug report states that there are currently no tests in the code repository, which can lead to serious bugs and affect the proper functioning and security of the protocol. The team has added a test-suite with high coverage, but one of the tests is failing, possibly due to insufficient balance. The report recommends creating a test suite with at least 90% branch coverage, including unit and integration tests, and covering a variety of scenarios. Assertions should also be used to verify state changes, invariants, and event emissions.

## Details
**Update**
The team has added a test-suite with fairly high coverage at `94.21%` branch coverage. However, one of the tests fail with the following error message:

```
1) CreditCaller & CreditManager contract
       Test ETH #openLendCredit #repayCredit #liquidate:
     Error: VM Exception while processing transaction: reverted with reason string 'CreditCaller: Insufficient balance'
```

Furthermore, we noticed some critical paths of the code, perhaps partially due to the failed test, have not been covered, including liquidations. Among other tests, we would recommend testing several liquidation scenarios, including liquidations due to poor health and liquidations due to a position being held for longer than `365` days.

We also note that some important contracts had coverage level which can be improved. Once again, this may be due to the failing test:

*   `CreditCaller`: `84.62%` line coverage.
*   `CreditManager`: `59.68%` line coverage.

**Update**
The team added multiple tests, including several tests for liquidation scenarios. We encourage the team to think of more test cases as an ongoing process. The additional tests should cover not only different liquidation scenarios but also other workflows as well.

**Description:** There are currently no tests within the code repository. Contracts that are not extensively tested are at high risk of serious bugs that can impact the protocol's proper functioning and security. While audits can help developers to find bugs, they are not a substitute for thorough unit and integration testing.

**Recommendation:** A test suite should be created, which:

*   Has at least `90%` branch coverage
*   Includes unit tests and integration tests with a mainnet fork
*   Includes a variety of test cases, including:
    *   All "happy path" scenarios
    *   Negative test cases
    *   Unexpected or edge cases
    *   All role permissions and access control for protected functions.

*   Verifies the following through assertions:
    *   All state changes
    *   Invariants
    *   Emission of events

*   May include fuzz testing
