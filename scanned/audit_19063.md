# Incorrect change of state

## Summary
This bug report is about the CreditFilter.sol smart contract. It states that the state of the contract is incorrectly being changed to true and should be done only via the `allowToken()` function. The recommendation is to only allow changing the state to false. This bug report is important because it is related to the security of the smart contract. If the state is incorrectly changed, it could lead to unexpected results and potential security issues. Therefore, it is important to follow the recommendation and only allow changing the state to false.

## Details
##### Description
Changing state to true is incorrect and must be done via `allowToken()`:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditFilter.sol#L398
##### Recommendation
We recommend allowing changing state only to false.
