These CVEs represent publicly disclosed security flaws in software components the application directly or transitively depends on.

Many modern attacks exploit known vulnerabilities in third-party packages, especially those left unpatched in production. Even if a CVE seems unrelated to your usage, package managers and tools (e.g., `npm audit`) flag them for a reason: you can’t assume safety from partial usage of a module.
