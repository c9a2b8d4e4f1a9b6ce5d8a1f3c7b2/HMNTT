# User can't repay with force flag

## Summary
This bug report is about a problem with closing an account when a token is blocked. Currently, the user must wait for the account to be liquidated by someone else in order to close it. The recommendation is to add a parameter for the repay function that would allow the user to repay the account with a force flag. This would enable the user to close the account even if a token is blocked.

## Details
##### Description
If some token was blocked on account, user can't close this account and must wait until account would be liquidated by someone:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L368
##### Recommendation
We recommend adding a parameter for the repay function, so that user can repay account with force flag.
