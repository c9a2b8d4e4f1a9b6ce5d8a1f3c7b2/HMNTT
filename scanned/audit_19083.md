# [M-03] No way to remove GasThrottle from VaderPool after deployment

## Summary
This bug report is about a potential Denial of Service (DOS) attack on the VaderPool platform. The attack is enabled by a modifier called `validateGas` in the BasePool smart contract, which checks that the user's gas price is below the value returned by `_FAST_GAS_ORACLE`. If `_FAST_GAS_ORACLE` is compromised to always return zero, all swaps will fail, and there is no way to recover from this scenario. The recommended mitigation steps are to either remove GasThrottle.sol entirely, or allow governance to turn it off as is done in VaderPoolV2.sol.
