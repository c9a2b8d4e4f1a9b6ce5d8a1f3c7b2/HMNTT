# Incorrect taking out of `tail` account

## Summary
This bug report is about an issue with the AccountFactory.sol file in the Gearbox-protocol/gearbox-contracts repository. The issue is that if the `creditAccount` is set to `tail`, the `tail` is not updated properly, which causes the list of accounts to break. To fix this issue, the recommendation is to update `tail` if it was taken out.

## Details
##### Description
If `creditAccount == tail` then `tail` is not updated properly and this will break the list of accounts:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/AccountFactory.sol#L260
##### Recommendation
We recommend updating `tail` if it was taken out.
