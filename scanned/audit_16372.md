# [M-01] Unhandled return values of transfer and transferFrom

## Summary
This bug report is about an issue with the ERC20 implementation in the code located at the given GitHub links. The issue is that some implementations of the transfer and transferFrom functions may return ‘false’ on failure instead of reverting. This can lead to unexpected behavior and should be avoided. To investigate the issue, the codes should be read. To prevent this issue, the return value should be checked and the transaction should be reverted on 0/false. Alternatively, OpenZeppelin’s SafeERC20 wrapper functions can be used.

## Impact
ERC20 implementations are not always consistent. Some implementations of transfer and transferFrom could return ‘false’ on failure instead of reverting. It is safer to wrap such calls into require() statements to these failures.
