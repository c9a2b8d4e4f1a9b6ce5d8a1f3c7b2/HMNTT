# [M-04] Fee on transfer tokens do not work within the protocol

## Summary
A bug has been reported in the code-423n4/2021-10-defiprotocol repository on GitHub. The bug is related to the protocol not correctly calculating the amount of tokens transferred when using the 'feeOnTransferToken' feature. This is causing the protocol to request incorrect amounts when dealing with these tokens.

The bug has been acknowledged by frank-beard (Kuiper) and Alex the Entreprenerd (judge) commented that this is a medium severity finding as it is dependent on an external condition. It was suggested that the protocol should use stored token balances instead of transfer for calculating amounts.

## Details
_Submitted by anon_.

Fee on transfer tokens transfer less tokens in than what would be expect.
This means that the protocol request incorrect amounts when dealing with these tokens.

<https://github.com/code-423n4/2021-10-defiprotocol/blob/7ca848f2779e2e64ed0b4756c02f0137ecd73e50/contracts/contracts/Basket.sol#L256>

The protocol should use stored token balances instead of transfer for calculating amounts.
