# [M03] Missing test coverage report

## Summary
This bug report concerns the lack of an automated test coverage report for the EmptySetSquad team. This report is important as it would allow the team to know which parts of the code are never executed by the automated tests. Without it, the team has to manually test the entire suite of code every time a change is made. The team has acknowledged the issue but has stated that the Solidity coverage tool is currently incompatible with their setup. For this reason, they will not be fixing the issue at this time.

## Details
There is no automated test coverage report. Without this report it is impossible to know whether there are parts of the code never executed by the automated tests; so for every change, a full manual test suite has to be executed to make sure that nothing is broken or misbehaving.


Consider adding the test coverage report and making it reach at least 95% of the source code.
